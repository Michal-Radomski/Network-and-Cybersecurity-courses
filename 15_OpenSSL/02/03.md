Generating DSA keys using OpenSSL follows a very similar workflow to generating RSA keys. However, because of how the math
behind DSA works, it requires a two-step generation process: first, you generate a file containing the **DSA parameters**
(the primes and generator shared globally), and then you use those parameters to create the **private key**.

Here is how to do it using modern OpenSSL commands.

---

## Step 1: Generate the DSA Parameters

Before creating a key, OpenSSL needs to define the mathematical boundaries ($p$, $q$, and $g$) that your key will use.

Run this command to create a parameter file:

```bash
openssl genpkey -genparam -algorithm DSA -out dsa_param.pem -pkeyopt dsa_param_bits:2048 -pkeyopt dsa_param_q_bits:256

```

### Breaking down the command:

- `-genparam`: Tells OpenSSL to generate cryptographic parameters rather than an actual key.
- `-algorithm DSA`: Specifies the Digital Signature Algorithm.
- `-pkeyopt dsa_param_bits:2048`: Sets the prime $p$ size to 2048 bits (the secure standard for DSA).
- `-pkeyopt dsa_param_q_bits:256`: Sets the subprime $q$ size to 256 bits.

---

## Step 2: Generate the Private Key

Now that you have the parameters file (`dsa_param.pem`), you can use it to generate your actual DSA private key.

Run this command:

```bash
openssl genpkey -paramfile dsa_param.pem -out dsa_private.pem

```

### Breaking down the command:

- `-paramfile dsa_param.pem`: Instructs OpenSSL to seed the key generation using the parameters you created in Step 1.
- `-out dsa_private.pem`: Saves your new private key to this file. **Keep this file strictly confidential.**

> 💡 **Tip:** Once you have generated your private key, the `dsa_param.pem` file is no longer needed to use the key and can
> safely be deleted.

---

## Step 3: Extract the Public Key

Just like with RSA, you derive the public key directly from your private key so you can share it with others.

Run this command to extract it:

```bash
openssl pkey -in dsa_private.pem -pubout -out dsa_public.pem

```

- `-in dsa_private.pem`: Takes your private key as the input.
- `-pubout`: Tells OpenSSL to extract and output only the public key component.
- `-out dsa_public.pem`: Saves the public key to this file.

---

## The Legacy Alternative (Single Command)

If you are working with an older script or a legacy system, you might see people use the older `gendsa` utility. Note that
`gendsa` still requires a parameter file first (generated via `openssl dsaparam`).

```bash
# 1. Legacy parameter generation
openssl dsaparam -out old_param.pem 2048

# 2. Legacy private key generation
openssl gendsa -out old_private.pem old_param.pem

```

_Note: The newer `openssl genpkey` method shown in Steps 1 and 2 is highly recommended over this legacy method as it outputs
keys in the modern, more secure PKCS#8 format._
