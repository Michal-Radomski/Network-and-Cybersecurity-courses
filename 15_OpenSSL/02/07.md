Adding or removing a passphrase (encryption) on an existing RSA private key is a common task, usually done to secure a key
before storing it or to automate a script that requires an unencrypted key.

OpenSSL handles this smoothly by taking your existing key, processing it, and outputting it with or without encryption.

---

### 🔒 Adding Encryption to an Unencrypted Key

If you have a plain text (unencrypted) RSA private key and want to protect it with a passphrase, you can encrypt it using a
strong modern cipher like **AES-256**.

Run the following command:

```bash
openssl rsa -in unencrypted_priv.pem -aes256 -out encrypted_priv.pem

```

- `-in unencrypted_priv.pem`: Your current, unprotected private key.
- `-aes256`: Specifies the encryption algorithm to protect the key.
- `-out encrypted_priv.pem`: The new file that will contain the password-protected key.

> ⌨️ **What happens next:** OpenSSL will prompt you to enter a new passphrase and verify it.

---

### 🔓 Removing Encryption from a Key

If you have an encrypted RSA private key and want to strip the passphrase away (making it completely unencrypted), use this
command:

```bash
openssl rsa -in encrypted_priv.pem -out decrypted_priv.pem

```

- `-in encrypted_priv.pem`: Your current password-protected private key.
- `-out decrypted_priv.pem`: The new, unprotected file.

> ⌨️ **What happens next:** OpenSSL will prompt you to enter the _current_ passphrase to unlock the key before it can save
> the unencrypted version.

---

### 🔄 In-Place Editing (Overwriting the Original File)

If you don't want to create a brand new file and would rather just update your existing key file in place, simply point the
`-in` and `-out` flags to the **exact same filename**.

#### To add a password to an existing file:

```bash
openssl rsa -in my_key.pem -aes256 -out my_key.pem

```

#### To remove a password from an existing file:

```bash
openssl rsa -in my_key.pem -out my_key.pem

```

⚠️ **Warning:** Be careful when removing encryption. An unencrypted private key is highly sensitive; anyone who gains access
to that file can instantly impersonate you or decrypt your data. Ensure its file permissions are tightly restricted (e.g.,
`chmod 600 my_key.pem` on Unix-like systems).
