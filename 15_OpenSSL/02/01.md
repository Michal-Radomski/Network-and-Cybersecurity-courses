Generating RSA keys using OpenSSL is a straightforward process done through the command line. You will first generate a
private key, and then extract the corresponding public key from it.

Here is how to do it step-by-step.

---

## Step 1: Generate the Private Key

The private key is used to decrypt data or sign messages. It must be kept strictly secret.

Run the following command to generate a standard 2048-bit RSA private key:

```bash
openssl genpkey -algorithm RSA -out private_key.pem -pkeyopt rsa_keygen_bits:2048

```

### Breaking down the command:

- `genpkey`: Tells OpenSSL to generate a private key.
- `-algorithm RSA`: Specifies that you want an RSA key.
- `-out private_key.pem`: Saves the key to a file named `private_key.pem`.
- `-pkeyopt rsa_keygen_bits:2048`: Sets the key length to 2048 bits (the current standard for secure RSA keys). For higher
  security, you can change this to `4096`.

> 💡 **Tip (Optional): Protect your private key with a password** If you want to encrypt your private key file so that it
> requires a password every time it's used, add an encryption cipher like `-aes256`:
>
> ```bash
> openssl genpkey -algorithm RSA -out private_key.pem -pkeyopt rsa_keygen_bits:2048 -aes256
>
> ```

---

## Step 2: Extract the Public Key

The public key is shared freely with the world so others can encrypt data for you or verify your digital signatures. Because
it is mathematically bound to your private key, OpenSSL derives it directly from the private key file you just made.

Run this command to extract the public key:

```bash
openssl pkey -in private_key.pem -pubout -out public_key.pem

```

### Breaking down the command:

- `pkey`: Instructs OpenSSL to process a public or private key.
- `-in private_key.pem`: Specifies your existing private key as the input.
- `-pubout`: Tells OpenSSL that you want to output the public key component.
- `-out public_key.pem`: Saves the extracted public key to a file named `public_key.pem`.

---

## Verifying Your Keys

If you want to view the mathematical details or check the validity of your keys in plain text, you can read them with these
commands:

- **To view the private key structure:**

```bash
openssl pkey -in private_key.pem -text -noout

```

- **To view the public key structure:**

```bash
openssl pkey -pubin -in public_key.pem -text -noout

```

You will now have two files in your current directory: `private_key.pem` (keep this safe and hidden) and `public_key.pem`
(share this as needed).
