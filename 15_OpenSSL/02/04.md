### Where to Use RSA vs. DSA Keys

While both are asymmetric cryptographic algorithms, they excel at different things and have vastly different levels of
support in modern tech.

- **RSA (Rivest–Shamir–Adleman):** \* **Best Used For:** General-purpose encryption, digital signatures, SSL/TLS
  certificates, and SSH keys.
- **Why:** It is universally supported across almost all platforms, operating systems, and legacy devices. It supports both
  encryption (scrambling data) and signing (verifying identity).

- **DSA (Digital Signature Algorithm):** \* **Best Used For:** Legacy system compatibility only.
- **Why:** DSA can _only_ be used for digital signatures, not encryption. Furthermore, major security standards now consider
  original DSA obsolete because it maxes out at a 1024-bit key size in many older implementations, making it vulnerable to
  modern computing power.

---

### Which Key Should You Use for SSH?

**Do not use DSA, and only use RSA if you have to.** For SSH, the industry standard has shifted. Here is the recommended
hierarchy of keys you should use for SSH today, ordered from best to worst:

| Recommendation        | Key Type                   | Why You Should (or Shouldn't) Use It                                                                                                                                                                                                                 |
| --------------------- | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **🥇 Best Choice**    | **Ed25519**                | It is incredibly fast, highly secure, and has a very small key size (making it clean to copy/paste). It is supported by default on almost all modern SSH clients and servers.                                                                        |
| **🥈 Second Choice**  | **RSA (3074 or 4096-bit)** | Use this if you are connecting to an older server that doesn't support Ed25519. Ensure the key size is at least 3072 or 4096 bits. _Note: OpenSSL and OpenSSH have deprecated older RSA SHA-1 signatures, so older 1024-bit RSA keys will not work._ |
| **❌ Do Not Use**     | **ECDSA**                  | While secure, it relies on NIST curves that have faced historical skepticism regarding potential backdoors, and it can leak the private key if the random number generator used during a session is flawed.                                          |
| **🚫 Strictly Avoid** | **DSA**                    | OpenSSH completely disabled support for DSA keys by default in version 7.0 (released way back in 2015). It is weak, outdated, and will outright fail to connect on modern systems.                                                                   |

### How to Generate the Recommended SSH Key

If you want to create a brand new, highly secure SSH key, use the **Ed25519** algorithm with this command:

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"

```

If you must use **RSA** for older compatibility, make sure to specify a strong bit length:

```bash
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

```
