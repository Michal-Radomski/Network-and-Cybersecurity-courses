The `pkey` utility in OpenSSL is a modern, unified command-line tool used to manage and process public and private
cryptographic keys.

Introduced in OpenSSL 1.0.0 to replace older, algorithm-specific tools (like `rsa`, `dsa`, and `ec`), `pkey` acts as a
"one-stop shop." Instead of needing to remember a different command for every type of key, `pkey` automatically detects the
algorithm being used and processes the file correctly.

---

### Why Use `pkey` Instead of `rsa` or `ec`?

- **Algorithm Agnostic:** It works seamlessly with RSA, DSA, ECDSA, Ed25519, and DH keys. You don't have to change your
  command just because you switched from an RSA key to an Elliptic Curve key.
- **Modern Cryptography Support:** Newer algorithms like **Ed25519** and **Ed448** _only_ work with the `pkey` (and
  `genpkey`) utilities; the legacy tools do not support them.
- **Consistent Syntax:** It unifies the flags and options, making scripting much cleaner.

---

### Common Tasks Using `pkey`

Here is how you perform everyday key management tasks using the modern `pkey` tool.

#### 1. Inspecting a Key (Text Component View)

Instead of using `openssl rsa -text`, you can use `pkey` to print the mathematical components of **any** key type to the
terminal:

```bash
openssl pkey -in key.pem -text -noout

```

#### 2. Extracting a Public Key

To grab the public key component out of a private key file, `pkey` handles it natively:

```bash
openssl pkey -in private_key.pem -pubout -out public_key.pem

```

#### 3. Adding Encryption (Password-Protecting a Key)

You can encrypt an unprotected key using a modern cipher like AES-256:

```bash
openssl pkey -in open_key.pem -aes256 -out encrypted_key.pem

```

#### 4. Removing Encryption (Decrypting a Key)

To strip a passphrase from a key, simply omit the cipher flag. OpenSSL will prompt you for the current password, then output
the decrypted key:

```bash
openssl pkey -in encrypted_key.pem -out decrypted_key.pem

```

#### 5. Converting Key Formats (e.g., PKCS#8)

By default, `pkey` outputs private keys in the modern, standard **PKCS#8** format. If you have an older PKCS#1 "traditional"
RSA key and want to upgrade it to the modern standard, simply running it through `pkey` converts it automatically:

```bash
openssl pkey -in traditional_rsa.pem -out modern_pkcs8.pem

```

---

### Summary of Differences

| Feature / Task           | Old Way (Legacy Utilities)      | Modern Way (`pkey`)              |
| ------------------------ | ------------------------------- | -------------------------------- |
| **Check an RSA Key**     | `openssl rsa -in key.pem -text` | `openssl pkey -in key.pem -text` |
| **Check an EC Key**      | `openssl ec -in key.pem -text`  | `openssl pkey -in key.pem -text` |
| **Check an Ed25519 Key** | _Not Supported_                 | `openssl pkey -in key.pem -text` |
| **Default Format**       | PKCS#1 (Traditional)            | PKCS#8 (Modern Standard)         |
