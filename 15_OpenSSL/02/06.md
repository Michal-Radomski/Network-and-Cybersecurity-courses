When you inspect an RSA private key using OpenSSL, you are peeling back the cryptographic layer to look at the raw
mathematical components that make the key work.

To inspect a key, you typically run a command like this:

```bash
openssl rsa -in rsa_priv.pem -text -noout

```

The output contains several large hexadecimal blocks of numbers. Because an RSA private key actually contains all the
information needed for both signing/decrypting (private) and verifying/encrypting (public), it includes a total of **nine
distinct components** derived from the underlying math.

---

### The Mathematical Components Explained

Here is exactly what you will see in the output, in order, and what each piece means:

#### 1. Private-Key: (2048 bit) / (4096 bit)

This header confirms the algorithm type and the strength (bit length) of the key.

#### 2. modulus ($n$)

The modulus is the product of two massive prime numbers ($n = p \times q$). This number is shared openly and forms the core
of your **public key**. Its length in bits determines the key size (e.g., 2048 bits).

#### 3. publicExponent ($e$)

Also part of the **public key**, this is almost always set to the static value `65537` (or `0x10001` in hex). It is chosen
because it makes the math for encrypting or verifying signatures incredibly fast while remaining cryptographically secure.

#### 4. privateExponent ($d$)

This is the actual **secret sauce** of your private key. It is a massive number calculated mathematically from $e$, $p$, and
$q$. Anyone who knows $d$ can decrypt messages meant for you or forge your digital signature.

#### 5. prime1 ($p$) and prime2 ($q$)

These are the two secret, randomly generated, massive prime numbers that were multiplied together to create the modulus
($n$). If an attacker ever factors your modulus back into $p$ and $q$, your key is broken.

#### 6. exponent1 ($d \bmod (p-1)$) and exponent2 ($d \bmod (q-1)$)

These are pre-calculated values used to speed up the math. RSA decryption is computationally heavy; computers use a
mathematical shortcut called the **Chinese Remainder Theorem (CRT)** to split the workload using these exponents, making
decryption up to four times faster.

#### 7. coefficient ($q^{-1} \bmod p$)

Another pre-calculated CRT coefficient used purely to optimize the speed of the private key operations.

---

### Example Output Structure

The terminal output will look structurally like this:

```text
RSA Private-Key: (2048 bit, 2 primes)
modulus:
    00:b4:92:c1... [a massive block of hex pairs]
publicExponent: 65537 (0x10001)
privateExponent:
    73:a1:0f:d4... [a massive block of hex pairs]
prime1:
    00:e3:c4:11... [half the size of the modulus]
prime2:
    00:cb:9f:aa... [half the size of the modulus]
exponent1:
    3a:bc:44:12...
exponent2:
    12:ff:dd:88...
coefficient:
    00:99:a1:bc...

```

> 🔒 **Security Reminder:** Never run the `-text` command on a production private key while sharing your screen or logging
> your terminal output, as anyone who copies these numbers down can instantly recreate your exact private key.
