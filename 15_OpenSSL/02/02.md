The command `openssl genrsa` is an older, legacy command used in OpenSSL to generate an **RSA private key**.

While it still works in modern versions of OpenSSL for backwards compatibility, it has largely been superseded by the newer,
more unified `openssl genpkey` command.

---

## How it works (Syntax)

A typical usage of the command looks like this:

```bash
openssl genrsa -out private_key.pem 2048

```

- **`openssl`**: Calls the OpenSSL tool.
- **`genrsa`**: Specifies the specific sub-command to generate an RSA private key.
- **`-out private_key.pem`**: Dictates the output file name where the key will be saved.
- **`2048`**: Specifies the key size in bits. (2048 is the standard for basic security; 4096 is used for higher security).

---

## Key Differences: `genrsa` vs. `genpkey`

OpenSSL updated its command structure to make things more consistent. Here is how `genrsa` compares to the modern `genpkey`
alternative:

| Feature            | Legacy Command (`genrsa`)                           | Modern Command (`genpkey`)                                                                  |
| ------------------ | --------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **Syntax Example** | `openssl genrsa -out key.pem 2048`                  | `openssl genpkey -algorithm RSA -out key.pem -pkeyopt rsa_keygen_bits:2048`                 |
| **Flexibility**    | Strictly limited to generating **RSA** keys.        | Can generate **any** type of key (RSA, ECC, Ed25519, DH) by changing the `-algorithm` flag. |
| **Output Format**  | Older, traditional SSLeay/PKCS#1 format by default. | Modern PKCS#8 format by default (which is more secure and widely supported).                |

---

## What do you get when you run it?

When you run `openssl genrsa`, it generates a text file containing a base64-encoded block of data wrapped in headers that
look like this:

```text
-----BEGIN RSA PRIVATE KEY-----
MIIEowIBAAKCAQEAnz8... (lots of random characters) ...
-----END RSA PRIVATE KEY-----

```

This file contains the private key components (the modulus, public exponent, private exponent, and prime factors). Anyone who
has access to this file can decrypt data intended for you or impersonate your digital identity, so it must be kept secure.
