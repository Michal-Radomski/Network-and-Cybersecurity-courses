Generating Elliptic Curve (EC) keys in OpenSSL is very similar to the DSA process, but much more secure and efficient. Like
DSA, it is generally done in two steps: choosing the elliptic curve (the parameters) and then generating the key pair.

---

### Step 1: Choose a Curve and Generate Parameters

First, you need to decide which elliptic curve to use. The most universally supported and standard curve for general web use
is **prime256v1** (also known as **secp256r1** or NIST P-256).

Run the following command to generate the EC parameter file:

```bash
openssl ecparam -name prime256v1 -out ec_param.pem

```

- `ecparam`: The OpenSSL tool for Elliptic Curve parameter manipulation.
- `-name prime256v1`: Specifies the structure of the curve you want to use.
- `-out ec_param.pem`: The file where the curve parameters will be saved.

> 💡 **Tip:** If you want to see a list of all available curves supported by your version of OpenSSL, you can run
> `openssl ecparam -list_curves`.

---

### Step 2: Generate the Private Key

Now that you have your curve parameters, you can generate the actual EC private key:

```bash
openssl ecparam -in ec_param.pem -genkey -noout -out ec_priv.pem

```

- `-in ec_param.pem`: Uses the parameters you generated in the previous step.
- `-genkey`: Instructs OpenSSL to generate an EC private key.
- `-noout`: Prevents OpenSSL from printing the text defense of the parameters to the terminal screen.
- `-out ec_priv.pem`: The file where your new **private key** will be saved.

_(Optional)_ If you want to password-protect your private key right away using AES-256 encryption, use this command instead:

```bash
openssl ecparam -in ec_param.pem -genkey -noout | openssl ec -aes256 -out ec_priv.pem

```

---

### Step 3: Extract the Public Key

Just like with RSA and DSA, the private key file contains the mathematical information needed to derive your public key. To
extract the **public key** so you can distribute it, run:

```bash
openssl ec -in ec_priv.pem -pubout -out ec_pub.pem

```

- `ec`: The OpenSSL EC key processing utility.
- `-in ec_priv.pem`: Your secret private key file.
- `-pubout`: Specifies that you want to output the public key component.
- `-out ec_pub.pem`: The file where your public key will be saved.

---

### 🚀 Shortcut: The One-Command Method

If you are using a modern version of OpenSSL (OpenSSL 1.1.0 or newer), you can skip the parameter file entirely and generate
the private key in a single, clean command using the `genpkey` utility:

```bash
openssl genpkey -algorithm EC -pkeyopt ec_paramgen_curve:prime256v1 -out ec_priv.pem

```

From there, you can extract the public key using the same **Step 3** command shown above.
