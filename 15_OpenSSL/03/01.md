Generating a Certificate Signing Request (CSR) and a certificate using OpenSSL is a standard process for securing web
servers. You can either generate a CSR to send to a Certificate Authority (CA) like Let's Encrypt or DigiCert, or you can
generate your own "self-signed" certificate for testing purposes.

Here is how to do both.

---

## Scenario 1: Generate a Private Key and a CSR

Use this method if you plan to buy or request a certificate from an official Certificate Authority.

### Step 1: Run the OpenSSL command

Open your terminal and run the following command. This creates both a new private key (`server.key`) and the CSR
(`server.csr`) at the same time.

```bash
openssl req -new -newkey rsa:2048 -nodes -keyout server.key -out server.csr

```

### Step 2: Fill out the information

OpenSSL will prompt you for details about your organization.

- **Country Name (2 letter code):** e.g., `US`
- **State or Province Name:** e.g., `California`
- **Locality Name (city):** e.g., `San Francisco`
- **Organization Name:** Your company name (or leave blank)
- **Organizational Unit Name:** e.g., `IT Department`
- **Common Name:** **Crucial step.** This must be the exact domain name you want to secure (e.g., `example.com` or
  `*.example.com` for a wildcard).
- **Email Address:** Your contact email.
- **A challenge password:** Leave this blank (press Enter) unless your CA explicitly requires it.

> ⚠️ **Important:** Keep your `server.key` private and secure. You will send the `server.csr` file to your Certificate
> Authority.

---

## Scenario 2: Generate a Self-Signed Certificate

Use this method for local development, testing, or internal environments where you don't need a public CA.

### Run the Single-Step Command

This command generates a private key (`server.key`) and a self-signed certificate (`server.crt`) valid for 365 days, skipping
the CSR process entirely.

```bash
openssl req -x509 -newkey rsa:2048 -nodes -keyout server.key -out server.crt -days 365

```

Just like Scenario 1, you will be prompted to enter your organizational details and your **Common Name** (domain or
`localhost`).

---

## Summary of Files Created

| File Extension  | What it is                      | What to do with it                                     |
| --------------- | ------------------------------- | ------------------------------------------------------ |
| `.key`          | **Private Key**                 | Keep it secret on your server. Never share it.         |
| `.csr`          | **Certificate Signing Request** | Send this to your CA to get an official certificate.   |
| `.crt` / `.pem` | **SSL/TLS Certificate**         | Install this on your web server (Nginx, Apache, etc.). |
