To inspect and extract specific information from a Certificate or a Certificate Signing Request (CSR) using OpenSSL, you can
use the `x509` and `req` commands along with specific text filtering flags.

---

## 1. Extracting Information from a Certificate (`.crt` or `.pem`)

By default, adding `-text -noout` will print the entire certificate in plain text. You can isolate specific details using the
following targeted flags or combining them with `grep`.

### Quick Specific Flags

Instead of printing everything, OpenSSL provides direct flags for common fields:

- **Subject (Domain/Owner):**

```bash
openssl x509 -in cert.pem -subject -noout

```

- **Issuer (CA who signed it):**

```bash
openssl x509 -in cert.pem -issuer -noout

```

- **Validity Dates (Not Before / Not After):**

```bash
openssl x509 -in cert.pem -dates -noout

```

- **Serial Number:**

```bash
openssl x509 -in cert.pem -serial -noout

```

- **MD5/SHA Fingerprint:**

```bash
openssl x509 -in cert.pem -fingerprint -noout

```

### Extracting Subject Alternative Names (SANs)

Because SANs (extensions) don't have a dedicated single flag like `-subject`, you can use `grep` to pull them out of the full
text:

```bash
openssl x509 -in cert.pem -text -noout | grep -A 1 "Subject Alternative Name"

```

---

## 2. Extracting Information from a CSR (`.csr`)

To extract details from a Certificate Signing Request before it gets signed, use the `openssl req` command instead of `x509`.

### Quick Specific Flags

- **Subject (Requested Subject Information):**

```bash
openssl req -in request.csr -subject -noout

```

- **Verify the CSR Signature (Check if it is valid/corrupted):**

```bash
openssl req -in request.csr -verify -noout

```

- **Show the Public Key details inside the CSR:**

```bash
openssl req -in request.csr -pubkey -noout

```

### Extracting Requested SANs from a CSR

Just like with certificates, you can filter the full text output to see requested extensions:

```bash
openssl req -in request.csr -text -noout | grep -A 1 "Subject Alternative Name"

```

---

## 💡 Pro-Tip: Checking if a Key and Certificate Match

If you ever need to verify that a private key matches a specific certificate or CSR, compare their modulus hashes. If the
output hashes match exactly, the files belong together.

```bash
openssl x509 -noout -modulus -in cert.pem | openssl md5
openssl req -noout -modulus -in request.csr | openssl md5
openssl pkey -noout -modulus -in private.key | openssl md5

```

_(Note: For Ed25519 keys, which do not use a standard mathematical "modulus" like RSA, you would compare the public key
components using `openssl x509 -pubkey` and `openssl pkey -pubout` instead)._
