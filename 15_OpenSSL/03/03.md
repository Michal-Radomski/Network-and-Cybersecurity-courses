## What is Ed25519?

**Ed25519** is a modern public-key signature system based on Edwards-curve Digital Signature Algorithm (EdDSA) using the
Curve25519 elliptic curve. Introduced by Daniel J. Bernstein and his team, it is designed for high speed, high security, and
resistance to side-channel attacks.

---

## Ed25519 vs. RSA

While RSA has been the internet standard for decades, Ed25519 offers significant advantages in modern cryptography.

| Feature                 | Ed25519                                          | RSA                                     |
| ----------------------- | ------------------------------------------------ | --------------------------------------- |
| **Algorithm Type**      | Elliptic Curve (EdDSA)                           | Prime Factorization                     |
| **Key Size (Standard)** | **256 bits**                                     | **3072 or 4096 bits**                   |
| **Security Level**      | Equivalent to RSA ~3000-bit                      | Depends heavily on key size             |
| **Performance**         | Extremely fast (signing and verification)        | Slower, resource-intensive              |
| **File Size**           | Very small (ideal for small packets/tokens)      | Large                                   |
| **Safety**              | Immune to many bad random number generator flaws | Vulnerable if random generation is weak |

### Key Differences Broken Down:

- **Size matters:** An Ed25519 public key is only 32 bytes (256 bits) long, whereas a secure RSA public key is typically 384
  or 512 bytes (3072 or 4096 bits). This means less bandwidth and storage.
- **Speed:** Ed25519 operations are significantly faster and consume less CPU, making it perfect for high-traffic servers or
  low-power devices.

## How to Generate Ed25519 Keys in OpenSSL

Generating Ed25519 keys requires OpenSSL 1.1.1 or newer. Unlike older algorithms (like RSA), Ed25519 does not require you to
specify a bit length because the key size is fixed at 256 bits.

### 1. Generate an Unencrypted Private Key

To generate a plain text private key directly (using the `-noenc` or `-nodes` flag we talked about earlier):

```bash
openssl genpkey -algorithm ed25519 -out ed25519_private.pem -noenc

```

### 2. Generate a Password-Protected Private Key (Recommended for Production)

If you want to encrypt the private key with a passphrase using AES-256:

```bash
openssl genpkey -algorithm ed25519 -out ed25519_private.pem -aes-256-cbc

```

### 3. Extract the Public Key

Once you have your private key, you can extract the corresponding public key with this command:

```bash
openssl pkey -in ed25519_private.pem -pubout -out ed25519_public.pem

```
