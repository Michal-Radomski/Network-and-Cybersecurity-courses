Yes, it is absolutely possible. OpenSSL can create certificates directly in other formats (like the binary DER format) and
can convert existing files between PEM, DER, and PFX.

Here is how to create a DER certificate directly, followed by the commands to convert between all three formats.

---

## 1. Creating a Certificate directly in DER format

By default, OpenSSL outputs text-based PEM files. However, you can use the `-outform` flag to tell OpenSSL to output a binary
DER file instead.

For a self-signed certificate:

```bash
openssl req -x509 -newkey rsa:4096 -keyout private.key -out cert.der -outform DER -days 365 -noenc

```

---

## 2. Converting Between Formats

### 🔄 PEM to DER

To convert a text-based PEM certificate into a binary DER certificate:

```bash
openssl x509 -in cert.pem -outform DER -out cert.der

```

### 🔄 DER to PEM

To convert a binary DER certificate back into a text-based PEM format:

```bash
openssl x509 -in cert.der -inform DER -outform PEM -out cert.pem

```

### 📦 PEM to PFX (Bundling Certificate + Private Key)

Because a PFX file is a password-protected bundle, you must provide both the certificate and its matching private key.
OpenSSL will prompt you to create a password during this process:

```bash
openssl pkcs12 -export -out certificate.pfx -inkey private.key -in cert.pem

```

_If you have an intermediate CA chain file (`chain.pem`), you can include it by adding `-certfile chain.pem` to the end of
the command._

### 🔓 PFX to PEM

If you have a PFX file from a Windows server and need to extract the components for Nginx or Apache, you can extract them all
into a single PEM file:

```bash
openssl pkcs12 -in certificate.pfx -out bundle.pem -nodes

```

_(You will be prompted for the password that was used to lock the PFX file)._
