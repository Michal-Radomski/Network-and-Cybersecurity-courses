Encryption is divided into two main categories based on how keys are used: **Symmetric** (same key for encrypting and
decrypting) and **Asymmetric** (different, paired keys for encrypting and decrypting).

Here are the most commonly used algorithms for each type.

---

## Symmetric Key Encryption Algorithms

In symmetric encryption, the sender and receiver share a **single, secret key**. It is incredibly fast and ideal for
encrypting large amounts of data.

- **AES (Advanced Encryption Standard):** The undisputed global standard. It is used by governments, banks, and security
  protocols worldwide. It operates on 128-bit blocks of data and supports key lengths of 128, 192, and 256 bits (with AES-256
  being the most secure). It is practically unbreakable with current technology.
- **ChaCha20:** A modern, high-speed stream cipher. It is exceptionally fast on devices that don't have dedicated hardware
  acceleration for AES (like older smartphones or budget smart devices). It is widely used by Google and Cloudflare to secure
  internet traffic.
- **3DES (Triple Data Encryption Standard):** A legacy algorithm that applies the older DES cipher three times to each data
  block. It is **deprecated** and being phased out globally because it is slow and vulnerable to modern security flaws.
- **Blowfish / Twofish:** Twofish is a highly secure, open-source symmetric cipher that was a finalist against AES to become
  the global standard. Blowfish is its older, faster predecessor, though it is now mostly replaced by Twofish or AES due to
  smaller block size vulnerabilities.

---

## Asymmetric Key Encryption Algorithms

Also known as **Public-Key Cryptography**, this method uses a **KeyPair**: a _Public Key_ (which anyone can see, used to
encrypt data) and a _Private Key_ (kept secret, used to decrypt data). It is mathematically complex and slower than symmetric
encryption, so it is mostly used for securing communication channels and digital signatures.

- **RSA (Rivest-Shamir-Adleman):** One of the oldest and most widely used asymmetric algorithms. It relies on the
  mathematical difficulty of factoring giant prime numbers. While still popular, it requires very large key sizes (at least
  2048 or 4096 bits) to remain secure today, making it computationally heavy.
- **ECC (Elliptic Curve Cryptography):** The modern standard for asymmetric encryption. Instead of large prime numbers, it
  uses the algebraic structure of elliptic curves over finite fields. ECC provides the same level of security as RSA but with
  drastically smaller key sizes (e.g., a 256-bit ECC key offers equivalent security to a 3072-bit RSA key), making it much
  faster and more efficient for mobile devices and web browsers.
- **Diffie-Hellman (DH):** While not used to encrypt data directly, this is a foundational asymmetric algorithm used for
  **key exchange**. It allows two parties who have never met to safely create a shared symmetric secret key over an unsecure
  public network.
- **DSA / ECDSA (Digital Signature Algorithm):** Variations of asymmetric algorithms specifically optimized not for hiding
  data, but for **digital signatures**—proving that a file or message was genuinely signed by the owner of the private key.

---

> **How They Work Together:** In the real world, systems like HTTPS (web browsing) use **Asymmetric encryption (like ECC or
> RSA)** to securely exchange a temporary key, and then immediately switch to **Symmetric encryption (like AES)** to scramble
> the actual data being sent. This provides both the security of public keys and the speed of symmetric keys.
