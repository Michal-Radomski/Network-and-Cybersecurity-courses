**Hashing** is the process of taking an input of any size (a string of text, a number, or even an entire file) and
transforming it into a fixed-size string of characters, usually a sequence of numbers and letters.

Think of it like a digital fingerprint. Just as a human fingerprint uniquely identifies a person without revealing everything
about them, a hash uniquely identifies a piece of data.

---

## How Hashing Works (and Its Core Rules)

A cryptographic **hash function** relies on a few strict rules to be useful:

- **Deterministic:** The same input will _always_ produce the exact same output. If you hash the word "apple" today, it will
  look identical to the hash of "apple" ten years from now.
- **Fixed Output Size:** No matter how big the input is, the output length never changes. Hashing a single letter or hashing
  the entire encyclopedia will result in a string of the exact same length.
- **One-Way (Irreversible):** You cannot reverse-engineer the original data from the hash. If you see a hash, you cannot
  "de-hash" it to find out the original word.
- **The Avalanche Effect:** A tiny change in the input changes the output completely. Changing "Apple" to "apple" (just a
  lowercase letter) results in a completely unrecognizable, radically different hash.

## Commonly Used Hashing Algorithms

Different algorithms are used depending on whether you need **security** (making it impossible to crack) or **speed** (for
quick data lookups).

### 1. Secure Hashing Algorithms (SHA Family)

These are the industry standards for security, cryptography, and data integrity.

- **SHA-256:** Part of the SHA-2 family, it generates a 256-bit (64 character) hash. It is highly secure and is famously used
  in the Bitcoin blockchain and for securing SSL/TLS certificates on the web.
- **SHA-3:** The newest member of the SHA family. It uses a completely different internal structure than SHA-2, making it an
  excellent fallback if SHA-2 is ever compromised.

### 2. Password Hashing Algorithms

Standard hash functions like SHA-256 are actually _too fast_, making them vulnerable to hackers using supercomputers to guess
millions of passwords a second. Special, slower algorithms are used for passwords:

- **bcrypt:** A popular, time-tested algorithm that deliberately uses a lot of computer processing power, making
  "brute-force" hacking attacks incredibly slow and impractical.
- **Argon2:** The modern gold standard for password hashing. It is highly customizable and allows developers to decide
  exactly how much memory and time it takes to compute a hash.

### 3. Legacy Algorithms (Avoid for Security)

- **MD5 & SHA-1:** These were once the most popular algorithms on the planet. However, they are now considered **broken**.
  Modern computers can easily find "collisions" (where two different inputs produce the exact same hash), so they are no
  longer used for security, though you might still see MD5 used to verify that a downloaded file isn't corrupted.

## Common Use Cases

- **Password Storage:** Websites never store your actual password. They hash it. When you log in, they hash your input and
  check if it matches the stored hash. If a hacker steals the database, they only get useless hashes.
- **Data Integrity:** When downloading large software files, creators often provide a SHA-256 hash. You can hash the file on
  your own computer; if your hash matches theirs, you know the file wasn't corrupted or tampered with by a hacker.
- **Data Structures (Hash Maps):** In programming, hashing is used to store and retrieve data in fractions of a millisecond
  by assigning data to specific slots based on its hash value.
