Self-reproducing code (often called a Quine) is a program that takes no input and produces a copy of its own source code as
its only output. [1, 2] In JavaScript, this is typically achieved by using a function that can inspect its own string
representation using the .toString() method or by using specific formatting tricks with strings.

## 1. The Classic .toString() Quine

This is the most readable version. It defines a function and then logs the function's own code followed by a call to itself.
[3, 4]

(function quine() { console.log('(' + quine.toString() + ')();'); })();

## 2. The Minimalist Quine (Functional)

A more advanced version uses a string to store the "instructions" for building the rest of the code. It avoids calling
.toString() on a named function, making it a more "pure" mathematical quine. [2]

(s = "console.log('(s = ' + JSON.stringify(s) + ')' + ';' + s)")();

## 3. The "Bio-Program" Style (Evolutionary)

While not a formal quine, "self-reproducing" can also refer to objects that instantiate new copies of themselves at
intervals, a concept sometimes used in simulations or "bio-programming". [1]

const BioObject = function() { setInterval(function() { console.log('Replicating...'); new BioObject(); }, 1000); }; new
BioObject();

## Summary of Techniques

- Introspection: Using .toString() to allow a function to see its own source code.
- Data vs. Execution: Storing the program logic in a string (the "data") and then writing code that prints that string
  twice—once as a string literal and once as executable code.
- Recursion vs. Replication: A standard quine produces its source code as text, whereas "bio-programs" produce new instances
  of themselves in memory. [1, 2, 3, 4]
