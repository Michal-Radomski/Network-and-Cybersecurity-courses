WebSocket is a communication protocol that allows for a permanent, two-way connection between a client (browser) and a
server. [1, 2, 3] Unlike standard HTTP, where the browser must ask for data and the server must wait for a request to send
it, WebSockets allow either side to send data at any time. [4, 5, 6, 7, 8]

## How it Works

The process follows a specific lifecycle:

1.  The Handshake: It starts as a standard HTTP request. The browser sends a special header saying, "I’d like to upgrade this
    connection to a WebSocket."
2.  The Upgrade: If the server supports it, it responds with an HTTP 101 Switching Protocols status code.
3.  The Persistent Tunnel: The HTTP connection is "upgraded" to a WebSocket. The "doors" stay open on both sides.
4.  Full-Duplex Communication: Data can now flow back and forth instantly as small "frames" without the overhead of heavy
    HTTP headers (like cookies or User-Agent strings) being resent every time. [9, 10, 11, 12, 13]

---

## WebSocket vs. HTTP

| Feature [14, 15, 16, 17, 18] | HTTP (Standard)                     | WebSocket                    |
| ---------------------------- | ----------------------------------- | ---------------------------- |
| Connection                   | Opens and closes for every request. | Stays open (Persistent).     |
| Direction                    | One-way (Client must ask).          | Two-way (Either can send).   |
| Real-time                    | Delayed (requires polling).         | Instant (Real-time).         |
| Overhead                     | High (headers sent every time).     | Very low (minimal metadata). |

## Common Use Cases

WebSockets are the "gold standard" for apps that require instant updates: [19, 20, 21]

- Chat Apps: Receiving a message the millisecond it's sent (WhatsApp Web, Slack).
- Live Sports/Stocks: Prices or scores updating live without refreshing.
- Multiplayer Games: Moving your character and seeing others move instantly.
- Collaborative Tools: Seeing where someone else's cursor is in a Google Doc. [22, 23, 24, 25, 26]

## A Quick Code Example (JavaScript)

// Open the connectionconst socket = new WebSocket('ws://://example.com'); // Listen for messages from the server
socket.onmessage = (event) => { console.log('Message from server:', event.data); }; // Send a message to the server
socket.send('Hello Server!');
