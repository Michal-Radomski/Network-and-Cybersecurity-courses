To understand how code runs, it helps to think of the Engine as the car, and the Interpreter and JIT as the two different
ways the engine handles the "fuel" (your code).

## 1. The Engine (The Whole System)

The Engine is the software program that executes JavaScript. It includes everything: the parser, the memory management, the
interpreter, and the compiler.

- Examples: V8 (Chrome/Node.js), SpiderMonkey (Firefox), JavaScriptCore (Safari).
- Role: It takes your source code and manages its entire lifecycle until the task is finished.

## 2. The Interpreter (The "Live" Translator)

The Interpreter reads your code line-by-line and executes it immediately.

- How it works: It converts your JavaScript (or bytecode) into actions on the fly.
- Pros: It starts running the code instantly without waiting for a "build" process.
- Cons: It is relatively slow because it has to re-translate the same line every time it runs (like in a loop).

## 3. JIT: Just-In-Time Compiler (The "Turbo" Optimizer)

The JIT Compiler is a modern hybrid strategy. Instead of just interpreting, it "watches" the code as it runs.

- How it works:

1. The Interpreter starts running the code quickly. 2. The Profiler monitors which functions are running frequently (called
   "Hot" code). 3. The JIT Compiler takes that "Hot" code and compiles it directly into Machine Code (binary) that the CPU
   can run at max speed.

- Result: You get the fast startup of an interpreter with the high performance of a compiled language.

## How they work together (The Workflow)

Imagine you are a chef (the Engine) following a recipe (the JS Code):

1.  Interpreter: You start cooking immediately by reading one instruction at a time.
2.  JIT: While cooking, you realize you have to chop 500 onions. Instead of reading "chop one onion" 500 times, you take a
    moment to set up a professional food processor (Optimization) to do the rest of the onions at lightning speed.
