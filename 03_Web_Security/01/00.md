While the terms are often used interchangeably in casual conversation, Web Security is a specific subset of the much broader
field of Internet Security.

## 1. Web Security (The "Specific" Scope)

Web security focuses exclusively on protecting websites, web applications, and browsers. It deals with the data moving
through the HTTP/HTTPS protocols.

- Primary Goal: To prevent hackers from crashing sites, stealing user data from databases, or hijacking user sessions.
- Common Threats:
- SQL Injection: Attacking a website's database.
  - Cross-Site Scripting (XSS): Stealing info (like document.cookie) via malicious scripts in the browser.
  - Cross-Site Request Forgery (CSRF): Tricking a user into performing actions they didn't intend to.
- Common Tools: Web Application Firewalls (WAF), SSL/TLS certificates (HTTPS), and secure coding practices.

## 2. Internet Security (The "Broad" Scope)

Internet security is the "umbrella" term. it covers everything related to sending and receiving information over the entire
global network, not just through a web browser.

- Primary Goal: To protect all hardware, software, and data that travels across the internet or local networks.
- Common Threats:
- Phishing: Deceptive emails.
  - Malware/Ransomware: Software that infects your computer.
  - Packet Sniffing: Intercepting data as it travels across a Wi-Fi network.
- Common Tools: Antivirus software, VPNs, Firewalls, and Email filters.

---

## Key Differences at a Glance

| Feature         | Web Security                        | Internet Security                      |
| --------------- | ----------------------------------- | -------------------------------------- |
| Focus           | Websites and Web Apps.              | Emails, Browsers, Apps, and Networks.  |
| Protocols       | Mostly HTTP, HTTPS, WebSocket.      | HTTP, FTP, SMTP (Email), IP, and more. |
| Example Target  | A specific online store's database. | Your entire laptop or your home Wi-Fi. |
| Who Manages It? | Web developers and site owners.     | Individual users and IT departments.   |

## Summary

If a hacker steals your password by injecting code into a site, that is a Web Security failure. If a hacker gets into your
computer because you downloaded a sketchy email attachment, that is an Internet Security failure.
