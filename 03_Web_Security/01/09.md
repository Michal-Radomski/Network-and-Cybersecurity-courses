Actually, Axios uses both. It is a common point of confusion, but it works like this: Axios is a library that performs AJAX
requests, and all AJAX requests are just a specific type of HTTP request. Think of it this way:

- HTTP is the "language" or protocol used to send messages.
- AJAX is the "technique" of sending those messages in the background without refreshing the page.
- Axios is the "tool" (library) you use to make that technique easier to write.

## How Axios works under the hood

Axios is clever because it changes its behavior depending on where it is running:

1.  In the Browser: Axios uses the XMLHttpRequest object (the classic way to do AJAX) to send HTTP requests.
2.  In Node.js: Since there is no browser or "page" to refresh, it uses the native Node.js http module to send HTTP requests.

## Why use Axios instead of standard AJAX?

While you could use the browser's built-in fetch() or XMLHttpRequest, developers prefer Axios because:

- Automatic JSON Transformation: It automatically turns the server's response into a JavaScript object (you don't have to
  call .json() like you do with fetch).
- Browser Support: It works consistently across older browsers.
- Interceptors: It allows you to "intercept" requests (e.g., automatically adding a login token to every header) before they
  are sent.

Summary: You are using Axios to make AJAX calls, which travel across the internet as HTTP requests.
