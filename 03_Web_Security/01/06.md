document.cookie is a property in JavaScript that allows you to read and write the HTTP cookies associated with the current
document. In simple terms, it is the browser's "memory" for a specific website, used to store small bits of data (like login
sessions, shopping cart items, or user preferences).

## How it Works## 1. Writing a Cookie

To save a piece of information, you assign a string to document.cookie.

document.cookie = "username=JohnDoe";

Note: This doesn't overwrite all cookies; it just adds a new one or updates the one named "username".

## 2. Reading Cookies

When you access document.cookie, it returns a single string containing all the cookies for that site, separated by
semicolons.

console.log(document.cookie); // Output: "username=JohnDoe; theme=dark; logged_in=true"

## Key Attributes

When setting a cookie, you usually add extra instructions to control its behavior:

- expires / max-age: Tells the browser when to delete the cookie (otherwise it expires when the tab closes).
- path: Limits the cookie to a specific folder on the site (e.g., path=/admin).
- Secure: Only sends the cookie over encrypted HTTPS connections.
- HttpOnly: A security setting that prevents JavaScript from accessing the cookie (making it invisible to document.cookie).

## Security Warning

Because document.cookie is accessible via JavaScript, it is a primary target for Cross-Site Scripting (XSS) attacks. If a
hacker runs a malicious script on your site, they can use document.cookie to steal session tokens and hijack user accounts.
This is why sensitive data (like passwords) is usually stored with the HttpOnly flag, which makes it "server-side only."
