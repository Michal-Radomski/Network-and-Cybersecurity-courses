Think of SOP (Same-Origin Policy) as the "default lock" on your browser, and CORS (Cross-Origin Resource Sharing) as the
"special key" that opens it for trusted guests.

## 1. SOP: The "Default Lock"

Same-Origin Policy is a built-in security rule in every browser. It says: "A script on Website A cannot read data from
Website B."

- Why it exists: To stop a malicious site from stealing your private data. If you are logged into facebook.com in one tab,
  and you visit evil-site.com in another, SOP prevents evil-site.com from making a request to Facebook and reading your
  private messages [1, 2].
- What it checks: It compares the Protocol (https), Domain (site.com), and Port (443). If even one is different, the browser
  blocks the access [2, 3].

## 2. CORS: The "Special Key"

CORS is a system that allows servers to tell the browser: "It's okay to let this specific outside website access my data."
Modern web apps are often split: a frontend at mysite.com and an API at ://mysite.com. Since these are "different origins,"
SOP would normally block them. CORS fixes this [4].

- How it works: The server adds special HTTP Headers to its response.
- The most famous header: Access-Control-Allow-Origin: https://mysite.com. When the browser sees this, it says, "Okay, the
  server trusts this site," and allows the data through [5].

## Key Differences at a Glance

| Feature     | SOP (Same-Origin Policy)                | CORS (Cross-Origin Sharing)                    |
| ----------- | --------------------------------------- | ---------------------------------------------- |
| Nature      | A restriction (Security Wall).          | A permission (The Doorway).                    |
| Location    | Enforced by the Browser.                | Configured on the Server.                      |
| Action      | Blocks cross-origin reading by default. | Allows specific cross-origin reading.          |
| Requirement | Always on; you don't "install" it.      | You must explicitly set it up on your backend. |

## The "Preflight" Check (OPTIONS)

For "risky" requests (like a DELETE or POST with JSON), the browser doesn't just send the data. It sends a tiny "Preflight"
request first using the OPTIONS method. It's basically asking: "Hey server, is it okay if I send this request from this other
domain?" If the server says yes via CORS headers, the real request is sent [6].
