In a browser engine like V8 (Chrome/Edge/Node.js), bytecode is a set of low-level instructions that the engine's interpreter,
[Ignition](https://medium.com/dailyjs/understanding-v8s-bytecode-317d46c94775), uses to execute your code. [1, 2]

## JavaScript vs. V8 Bytecode

If you have a simple function that adds a value to a variable, the engine breaks it down into steps centered around a special
"storage" spot called the Accumulator. [3, 4] Your JavaScript:

function addTen(x) { return x + 10; }

The V8 Bytecode version:

| Instruction [3, 4, 5, 6, 7] | Explanation                                                       |
| --------------------------- | ----------------------------------------------------------------- |
| Ldar a0                     | Load accumulator from register a0 (the parameter x).              |
| AddSmi [10]                 | Add a Small integer (10) to the current value in the accumulator. |
| Return                      | Take whatever is in the accumulator and return it.                |

## Real-world "Raw" Look

If you were to look at the "raw" bytes stored in memory for this function, they wouldn't even be words like Ldar—they would
be numeric codes (hexadecimal). For example: [8, 9]

- 0x0b might represent Ldar
- 0x40 might represent AddSmi
- 0xaa might represent Return

## How to see it yourself

If you have Node.js installed, you can see the actual bytecode the V8 engine generates for any file by running it with a
special flag in your terminal: node --print-bytecode yourFile.js [10] Would you like to see how these instructions change if
we add something more complex, like a loop or an if-statement?

import dis def add_numbers(a, b): return a + b

# Generate bytecode for the simple functionbytecode = dis.Bytecode(add_numbers)for instr in bytecode:

    print(f"{instr.offset:4} {instr.opname:16} ({instr.argrepr})")
