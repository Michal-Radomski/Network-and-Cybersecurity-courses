When you browse the internet, your browser constantly saves data to your disk to improve speed and remember who you are. The
difference between normal and incognito mode is primarily about how long that data stays on your computer. [1, 2, 3]

## What is Saved to the Disk?

In Normal Mode, the following items are permanently written to your hard drive until you manually delete them:

- Browsing History: A list of every URL you have visited.
- Cache: Large files like images, videos, and scripts. The browser keeps these so it doesn't have to download them again the
  next time you visit the same site.
- Cookies: Small text files that keep you logged into accounts and track your preferences.
- Form Data: Information you’ve typed into search bars or address fields (autocomplete).
- Download History: A log of every file you have downloaded. [4, 5, 6, 7, 8, 9, 10, 11, 12]

---

## Normal vs. Incognito Mode

The main difference is that Incognito Mode is a "temporary session" that forgets everything once you close the window. [4,
13]

| Feature [1, 4, 7, 11, 14, 15, 16, 17, 18, 19, 20] | Normal Mode                         | Incognito (Private) Mode                                                                                                        |
| ------------------------------------------------- | ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| History                                           | Saved permanently to disk.          | Deleted as soon as the window is closed.                                                                                        |
| Cookies                                           | Stay on disk; you remain logged in. | Deleted after the session ends; you must re-login.                                                                              |
| Cache                                             | Stored to speed up future visits.   | Stored temporarily in RAM[](https://hide.me/en/blog/how-to-see-and-delete-incognito-mode-history/) or temp folders, then wiped. |
| Downloads                                         | Saved to your "Downloads" folder.   | The file is saved, but the history list is hidden.                                                                              |
| Bookmarks                                         | Saved permanently.                  | Saved permanently.                                                                                                              |

## Important Misconceptions

- It is not invisible: Your Internet Service Provider (ISP), employer, or the websites themselves can still see your activity
  and IP address while you are browsing.
- Local privacy only: Incognito is designed to hide your activity from other people who use your computer, not from the
  internet at large.
- Traces remain: Some technical traces, like DNS Cache (a record of site names your computer looked up), may remain on the
  system and require a manual "flush" to clear completely. [9, 17, 18, 21, 22, 23]

DNS cache is a temporary database on your device (or within your browser) that stores the IP addresses of websites you've
recently visited. Its purpose is to speed up browsing by letting your computer skip the "lookup" process for a domain name it
has already resolved. [1]

## How to See and Delete in Firefox

Firefox maintains its own internal DNS cache that is separate from your operating system. [1]

1.  To See the Cache: Open a new tab in Firefox and type about:networking#dns in the address bar. This page will display a
    list of all currently active DNS entries stored in Firefox's memory.
2.  To Delete (Flush) the Cache: On that same page, click the Clear DNS Cache button.

- Note: Firefox stores its DNS cache only in memory; it is automatically cleared whenever you restart the browser. [2, 3, 4,
  5, 6, 7]

## How to Delete the System DNS Cache in Ubuntu

If you are still having connection issues after clearing Firefox, you may need to clear the system-level cache. Modern Ubuntu
versions use the systemd-resolved service. [8, 9]

- View Statistics: To see how many entries are currently cached by Ubuntu, run: resolvectl statistics
- Clear Cache (Ubuntu 22.04 and 24.04): sudo resolvectl flush-caches
- Clear Cache (Ubuntu 20.04 and older): sudo systemd-resolve --flush-caches [9, 10, 11, 12, 13]

## Why Clear It?

You might need to clear this cache if:

- A website moved: The site changed its server IP, but your computer is still trying to connect to the old, broken one.
- Privacy: You want to remove local traces of which domains you have looked up.
- Troubleshooting: You are getting "Page Not Found" errors despite having an active internet connection. [1, 14]

Ad blockers like AdBlock Plus or uBlock Origin act as a "security guard" for your browser, filtering out specific content
before it even appears on your screen. They primarily work using two methods: Communication Blocking and Element Hiding.

## 1. Communication Blocking (The "No Entry" List)

Ad blockers maintain massive "filter lists" (like EasyList). These lists contain the URLs of known ad servers, trackers, and
malware distributors.

- How it works: Every time a website tries to load a resource (like an image or a script), the ad blocker checks the
  destination URL against its list.
- Result: If the URL is on the list, the browser is told to cancel the request. The ad is never even downloaded, which saves
  you data and makes the page load faster.

## 2. Element Hiding (The "Cloak")

Sometimes ads are part of the website's own code and don't come from a separate URL. In this case, the blocker uses CSS
Injection.

- How it works: The blocker identifies the specific "box" or container in the HTML code where the ad lives (e.g.,
  <div class="sidebar-ad">).
- Result: It adds a style rule to the page that says display: none !important;. The ad is still "there" in the code, but it
  is invisible to you, and the rest of the page collapses to fill the empty space.

## 3. Script Snippet Injection

Advanced blockers can inject small pieces of JavaScript to "trick" websites.

- Example: Some sites have "ad-block detectors" that show a popup if an ad fails to load. The blocker can inject a script
  that simulates a successful ad load, so the detector is fooled and you can read the site in peace.

## Comparison of Popular Blockers

| Feature    | AdBlock Plus (ABP)                                                    | uBlock Origin                                                     |
| ---------- | --------------------------------------------------------------------- | ----------------------------------------------------------------- |
| Philosophy | "Acceptable Ads": Allows some "non-intrusive" ads unless you opt out. | Strict Blocking: Blocks everything by default; very light on CPU. |
| Control    | User-friendly, but less customizable.                                 | Highly advanced; allows for manual "element zapping."             |
| Privacy    | Owned by a for-profit company.                                        | Open-source and community-driven.                                 |

Pro Tip: If you are using a Chromium-based browser (Chrome, Edge), most ad blockers will become less effective soon due to a
change called Manifest V3. Many users are switching to uBlock Origin Lite or using the Firefox browser to keep full blocking
power.
