To understand the difference, think of a Standard HTTP Request as a page reload and an AJAX Request as a background update.
[1]

## 1. Standard HTTP Request (The "Traditional" Way)

When you click a link or type a URL and hit Enter, the browser performs a full HTTP request.

- How it works: The browser sends a request to the server, and the server sends back an entirely new HTML document.
- User Experience: The screen goes white for a split second, the loading spinner in the tab turns, and the whole page
  "refreshes."
- Usage: Navigating from Home to Contact Us.

## 2. AJAX Request (The "Modern" Way)

AJAX stands for Asynchronous JavaScript and XML. It is a technique that uses an HTTP request, but it is triggered by
JavaScript in the background. [2, 3, 4, 5, 6]

- How it works: JavaScript sends a request to the server behind the scenes. The server usually sends back a small piece of
  data (like JSON) rather than a full page. JavaScript then uses that data to update a specific part of the page.
- User Experience: The page never reloads. The update feels "instant" and smooth.
- Usage: Liking a post on Instagram, a Google search suggestion appearing as you type, or a live sports score updating. [7,
  8, 9, 10, 11]

---

## Key Differences

| Feature [12, 13, 14, 15, 16] | Standard HTTP Request                            | AJAX Request                                              |
| ---------------------------- | ------------------------------------------------ | --------------------------------------------------------- |
| Trigger                      | User actions (clicking links, submitting forms). | JavaScript code (timers, button clicks, scrolling).       |
| Page Reload                  | Yes. The entire page is discarded and rebuilt.   | No. Only specific parts of the page change.               |
| Response Type                | Full HTML document.                              | Usually small data (JSON or XML).                         |
| State                        | The browser "forgets" the previous page state.   | The page keeps its current state (scroll position, etc.). |
| Blocking                     | Synchronous (browser waits and freezes UI).      | Asynchronous (browser stays interactive while waiting).   |

## Summary

An AJAX request is an HTTP request, but it is handled by the browser's XMLHttpRequest object or the fetch() API instead of
the main navigation bar. It allows websites to act more like "apps" where only the data moves, not the whole interface. [17,
18, 19]
