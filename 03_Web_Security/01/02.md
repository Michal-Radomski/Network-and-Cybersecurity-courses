When a browser runs a JavaScript (JS) script, it uses a specialized JavaScript Engine (like Chrome's [V8](https://v8.dev/) or
Firefox's [SpiderMonkey](https://spidermonkey.dev/)) to translate your human-readable code into something the computer's
processor can execute. [1, 2] The process happens in four main stages:

## 1. Parsing and Tokenization

As soon as the browser encounters a <script> tag or loads a .js file, it reads the code as a stream of text. [3, 4]

- Tokenization: The engine breaks the code into small units called "tokens" (like keywords, operators, and variable names).
- Abstract Syntax Tree (AST): These tokens are organized into a tree-like structure called an AST, which ensures the code
  follows correct syntax rules. [1, 3, 5, 6]

## 2. Compilation (Just-In-Time)

Modern browsers use Just-In-Time (JIT) compilation. [7, 8]

- Instead of reading code line-by-line (interpreting), the engine compiles parts of the script into machine code (binary)
  right before it runs.
- It also has an optimizer that monitors the code. If it sees a specific function running many times, it "re-compiles" it
  into a highly efficient version to make it run faster. [2, 6, 9, 10, 11]

## 3. Execution and Memory Management

The engine executes the compiled code using two main internal structures: [12, 13, 14]

- Call Stack: This is where the engine keeps track of which function is currently running and which one comes next.
- Memory Heap: This is a large region in memory used to store objects, variables, and data needed by the script.
- Garbage Collection: The engine automatically scans the heap to find and delete data that is no longer being used to free up
  memory. [1, 14, 15, 16]

## 4. Interacting with the Page (Web APIs)

JavaScript itself is quite limited; it can't "see" your screen or "click" buttons on its own. It relies on Web APIs provided
by the browser environment. [16, 17, 18, 19]

- The DOM: The engine uses the Document Object Model (DOM) to change the text, colors, or structure of the webpage.
- The Event Loop: Since JavaScript is "single-threaded" (it can only do one thing at a time), the Event Loop manages
  asynchronous tasks like waiting for a timer or a network request so the browser doesn't freeze while waiting. [14, 18, 20,
  21]
