Yes, an attack can absolutely be performed using a QR code. This type of cyberattack is known as **Quishing** (a portmanteau
of "QR" and "phishing").

While a QR code itself is just a way to store data, it acts as a "physical-to-digital" bridge that attackers use to bypass
traditional security filters.

### Common Types of QR Code Attacks

| Attack Type             | How it Works                                                                                                                  | Common Scenario                                                                             |
| :---------------------- | :---------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------ |
| **Quishing (Phishing)** | The code leads to a fake website that looks like a real login page (e.g., Microsoft 365, a bank, or Netflix).                 | An email claims your MFA is expiring and asks you to scan a code to "re-verify."            |
| **Overlay Attacks**     | Attackers stick a malicious QR code sticker over a legitimate one.                                                            | A public parking meter or restaurant menu has a sticker covering the original payment code. |
| **Malware Download**    | The code triggers an immediate download of a malicious file (.apk or .dmg) or redirects to an infected site.                  | A "free coupon" or "discount" flyer in a public space.                                      |
| **Deep Link Exploits**  | The code uses "deep links" to open specific apps on your phone and trigger actions like making a payment or adding a contact. | A QR code that automatically opens your crypto wallet with a pre-filled recipient address.  |
| **Rogue Wi-Fi**         | The code contains instructions to automatically connect your phone to a malicious Wi-Fi hotspot.                              | A "Free Airport Wi-Fi" QR code that intercepts your data once you connect.                  |

---

### Why QR Attacks are Effective

- **Mobile Vulnerability:** Most mobile devices do not have the same level of anti-phishing protection (like browser plugins
  or advanced firewalls) that a desktop computer has.
- **Filter Evasion:** Email security scanners are great at reading text and links, but they often ignore images. A QR code is
  "invisible" to many basic spam filters.
- **URL Masking:** You cannot see where a QR code goes until _after_ you scan it. Attackers often use **URL shorteners** to
  hide the final malicious destination.

---

### How to Protect Yourself

1.  **Check the URL Preview:** Most modern camera apps show a small preview of the link before you tap it. **Inspect it
    carefully.** If the link looks suspicious (e.g., `bit.ly/3x8z` or `micros0ft-login.com`), do not open it.
2.  **Look for Physical Tampering:** If you see a QR code on a poster or parking meter, feel it. Is it a sticker placed over
    another code? If so, **do not scan it.**
3.  **Avoid Scanning in Emails:** Legitimate companies rarely ask you to scan a QR code within an email to "fix" an account
    issue. Go directly to their website via your browser instead.
4.  **Use a Secure Scanner:** Some security apps (like Norton, Kaspersky, or Trend Micro) offer QR scanners that check the
    link against a database of known malicious sites before opening it.
5.  **Never Download Apps via QR:** Only download apps directly from the official Apple App Store or Google Play Store.

> **The "Think Before You Scan" Rule:** Treat a QR code like a link in a suspicious email. If you weren't expecting it, or if
> it's pressuring you to act quickly, it’s probably a trap.
