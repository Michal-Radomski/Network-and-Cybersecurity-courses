Integrating iframes into a React application introduces several security risks, primarily **Cross-Site Scripting (XSS)** and
**Clickjacking**. To protect your app, you need to implement security layers at both the code level and the server level.

---

## 1. Use the `sandbox` Attribute

The `sandbox` attribute is your first line of defense. It applies extra restrictions to the content in the iframe. By
default, an empty `sandbox` attribute restricts everything (scripts, forms, popups). You should only "opt-in" to the
permissions you absolutely need.

```javascript
<iframe
  src="https://example.com"
  title="Secure Content"
  sandbox="allow-scripts allow-same-origin"
  // Avoid allow-top-navigation unless necessary
/>
```

- **`allow-scripts`**: Allows the iframe to run JavaScript.
- **`allow-same-origin`**: Allows the iframe to maintain its origin (access cookies/storage).
- **`allow-forms`**: Allows the iframe to submit forms.

---

## 2. Implement Content Security Policy (CSP)

A CSP header tells the browser which sources of content are trusted. You can use the `frame-src` directive to restrict which
URLs your React app is allowed to embed.

**Example CSP Header:** `Content-Security-Policy: frame-src 'self' https://trusted-partner.com;`

This prevents your app from accidentally loading malicious iframes if an attacker manages to inject code into your site.

---

## 3. Prevent Clickjacking

While you want to protect your app _from_ the iframe, you also want to ensure your app cannot be embedded _inside_ a
malicious iframe elsewhere.

- **`X-Frame-Options`**: Set this server-side header to `SAMEORIGIN` or `DENY`.
- **`frame-ancestors`**: Use this CSP directive for more modern control.
  - `Content-Security-Policy: frame-ancestors 'self';`

---

## 4. Validate and Sanitize URLs

If the `src` of your iframe is dynamic (e.g., based on user input or an API), you must validate it to prevent attackers from
injecting `javascript:alert(1)` or malicious domains.

```javascript
const SafeIframe = ({ userUrl }) => {
  // Simple validation to ensure it's a valid HTTPS link
  const isValid = userUrl.startsWith("https://");

  if (!isValid) return <p>Invalid source</p>;

  return <iframe src={userUrl} sandbox="allow-scripts" loading="lazy" title="User Content" />;
};
```

---

## 5. Summary of Security Layers

| Security Layer            | Purpose                                      | Implementation           |
| :------------------------ | :------------------------------------------- | :----------------------- |
| **Sandbox Attribute**     | Limits what the iframe can do                | HTML/React Attribute     |
| **frame-src (CSP)**       | Restricts where iframes can load from        | HTTP Header              |
| **frame-ancestors (CSP)** | Prevents your app from being framed          | HTTP Header              |
| **Referrer-Policy**       | Controls how much info is sent to the iframe | HTTP Header or Attribute |

---

### A Quick Note on `postMessage`

If you need to communicate between your React app and the iframe, **never** use `*` as the target origin. Always specify the
exact domain to prevent sensitive data from being intercepted.

**Bad:** `window.parent.postMessage(data, '*')` **Good:** `window.parent.postMessage(data, 'https://trusted-domain.com')`
