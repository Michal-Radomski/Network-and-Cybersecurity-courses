To protect a React/Node.js application hosted on Apache2 from clickjacking, you need to prevent your site from being rendered
inside an `<iframe>` on a malicious domain. This is best achieved using a "defense-in-depth" approach across your server
configuration and application code.

### 1. Server-Level Countermeasures (Apache2)

The most robust way to stop clickjacking is to configure your web server to send specific security headers. Since Apache2 is
your entry point, you should apply these rules in your virtual host file or `.htaccess`.

#### **Content-Security-Policy (CSP)**

This is the modern standard. The `frame-ancestors` directive tells the browser which parent pages are allowed to embed your
site.

- **To block all framing:** `frame-ancestors 'none';`
- **To allow only your own site:** `frame-ancestors 'self';`

**Apache Configuration:**

```apache
<IfModule mod_headers.c>
    Header always set Content-Security-Policy "frame-ancestors 'self';"
</IfModule>
```

#### **X-Frame-Options**

While CSP is preferred, `X-Frame-Options` provides legacy support for older browsers.

- **DENY:** No one can frame the site.
- **SAMEORIGIN:** Only your site can frame itself.

**Apache Configuration:**

```apache
<IfModule mod_headers.c>
    Header always set X-Frame-Options "SAMEORIGIN"
</IfModule>
```

_Note: Ensure `mod_headers` is enabled on your server (`sudo a2enmod headers && sudo systemctl restart apache2`)._

---

### 2. Application-Level Countermeasures (Node.js/Express)

If you prefer to handle security within your Node.js logic (useful if you have dynamic framing requirements), use the
**Helmet** middleware. It automatically sets these headers for you.

**Installation:**

```bash
npm install helmet
```

**Implementation:**

```javascript
const express = require("express");
const helmet = require("helmet");
const app = express();

// Use Helmet to set security headers
app.use(
  helmet({
    contentSecurityPolicy: {
      directives: {
        "frame-ancestors": ["'self'"], // Modern defense
      },
    },
    xFrameOptions: { action: "sameorigin" }, // Legacy defense
  })
);
```

---

### 3. Frontend Countermeasures (React)

While headers are the primary defense, you can add a "Frame Busting" script in your React app as a secondary layer. This
script detects if the app is being framed and "breaks" it out to the top level.

**In your `App.js` or a custom hook:**

```javascript
import { useEffect } from "react";

const useFrameBusting = () => {
  useEffect(() => {
    if (window.self !== window.top) {
      // If we are in a frame, force the top window to our URL
      window.top.location.href = window.self.location.href;
    }
  }, []);
};
```

---

### 4. Extra Protection: SameSite Cookies

Clickjacking often aims to trick a logged-in user into performing an action. By setting your session cookies to
`SameSite=Lax` or `Strict`, the browser won't send the user's session cookie when your site is loaded in a cross-site iframe,
rendering the attack useless.

**In Node.js (express-session):**

```javascript
app.use(
  session({
    cookie: {
      sameSite: "strict", // or 'lax'
      secure: true, // requires HTTPS
      httpOnly: true,
    },
  })
);
```

### Summary Checklist

| Layer       | Action                                                     |
| :---------- | :--------------------------------------------------------- |
| **Apache**  | Enable `mod_headers` and set `frame-ancestors 'self'`.     |
| **Node.js** | Use `helmet` middleware for automated header management.   |
| **React**   | Implement a frame-busting script for legacy protection.    |
| **Cookies** | Use `SameSite=Strict` to prevent credential use in frames. |
