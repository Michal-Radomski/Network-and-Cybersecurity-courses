To implement anti-CSRF tokens effectively, you need a "handshake" between your Express backend and React frontend. The goal
is to ensure that for every state-changing request (POST, PUT, DELETE), the client proves it has access to a secret token
that a malicious third-party site couldn't possibly know.

---

## 1. The Strategy: Double Submit Cookie

This is the most standard approach for modern Single Page Applications (SPAs).

1.  **Backend** sends a random token in a cookie (e.g., `XSRF-TOKEN`).
2.  **React** reads that cookie and sends the value back in a custom HTTP header (e.g., `X-XSRF-TOKEN`).
3.  **Backend** compares the header against the cookie. If they match, the request is authentic.

---

## 2. Backend Implementation (Express.js)

We'll use `csrf-csrf`, a modern and maintained library for this purpose.

### Installation

```bash
npm install csrf-csrf cookie-parser
```

### Server Setup

```typescript
import express from "express";
import cookieParser from "cookie-parser";
import { doubleCsrf } from "csrf-csrf";

const app = express();

app.use(express.json());
app.use(cookieParser("your-super-secret-key"));

// Configure CSRF protection
const { generateToken, doubleCsrfProtection } = doubleCsrf({
  getSecret: () => "your-super-secret-key",
  cookieName: "XSRF-TOKEN", // Name of the cookie sent to React
  cookieOptions: {
    httpOnly: false, // REQUIRED: React needs to read this cookie
    sameSite: "lax",
    secure: process.env.NODE_ENV === "production",
  },
  getTokenFromRequest: (req) => req.headers["x-xsrf-token"], // Where Express looks for the token
});

// 1. Provide the token to React
app.get("/api/csrf-token", (req, res) => {
  const token = generateToken(req, res);
  res.json({ token });
});

// 2. Protect sensitive routes
app.post("/api/update-data", doubleCsrfProtection, (req, res) => {
  res.json({ message: "Success! Request was protected." });
});

app.listen(3000);
```

## 3. Frontend Implementation (React)

You need to ensure your API client (Axios or Fetch) automatically grabs that cookie and attaches it to headers.

### Using Axios (Recommended)

Axios has built-in support for this pattern. You just need to point it to the right names.

```typescript
import axios from "axios";

const api = axios.create({
  baseURL: "http://localhost:3000",
  withCredentials: true, // Crucial for sending/receiving cookies
  xsrfCookieName: "XSRF-TOKEN", // Matches Express cookieName
  xsrfHeaderName: "x-xsrf-token", // Matches Express getTokenFromRequest
});

// usage in a component
const updateData = async () => {
  // First, ensure we have a token (usually done on app mount or login)
  await api.get("/api/csrf-token");

  // Subsequent POSTs will automatically include the 'x-xsrf-token' header
  await api.post("/api/update-data", { name: "New Name" });
};
```

## 4. Key Security Rules

### The Cookie Conflict

You might have heard that cookies should always be `httpOnly: true`. This is true for **Session Cookies** (which hold your
login state). However, for the **CSRF Token Cookie**, it **must** be `httpOnly: false`.

- **Why?** Because JavaScript (React) needs to read the value from the cookie to put it into the header.
- **Is it safe?** Yes, because even if an attacker tricks a user's browser into sending a request, the attacker cannot _read_
  the cookie from their malicious domain due to the **Same-Origin Policy**.

### Deployment Tip

If your React app and Express API are on different domains (e.g., `myapp.com` and `api.myapp.com`), you must:

1.  Configure **CORS** in Express to allow `credentials: true`.
2.  Set the cookie `domain` attribute to `.myapp.com` so the cookie is shared across subdomains.

## Summary Checklist

| Task               | Responsibility                                    |
| :----------------- | :------------------------------------------------ |
| **Generate Token** | Express (`generateToken`)                         |
| **Send Cookie**    | Express (with `httpOnly: false`)                  |
| **Extract Cookie** | React (via Axios or manual helper)                |
| **Send Header**    | React (attach `x-xsrf-token` to POST body/header) |
| **Validate**       | Express (`doubleCsrfProtection` middleware)       |
