SQL injection is just one flavor of a broader category called **Code Injection**. These attacks occur whenever an application
incorrectly treats untrusted data as part of an executable command or script.

Here are the most common "cousins" to SQLi:

---

## 1. Command Injection (OS Injection)

This happens when an application passes user-supplied data to a system shell. Instead of just providing a filename or IP
address, the attacker appends operating system commands.

- **The Target:** The underlying server's Operating System (Linux, Windows, etc.).
- **The Example:** A web tool that pings an IP address: `system("ping " + user_input)`.
- **The Attack:** An attacker enters `8.8.8.8; cat /etc/passwd`.
- **The Result:** The server pings the IP and then displays the contents of the sensitive password file.

---

## 2. Cross-Site Scripting (XSS)

XSS is often called "JavaScript Injection." Instead of attacking the server directly, the attacker injects malicious scripts
that execute in the **browsers of other users**.

- **The Target:** The end-user's browser and session.
- **Stored XSS:** The script is saved on the server (e.g., in a comment section). Every user who views the comment gets
  infected.
- **Reflected XSS:** The script is part of a URL (e.g., `search.php?q=<script>alert(1)</script>`). If a user clicks the link,
  the script runs.
- **The Goal:** Stealing session cookies, redirecting users, or defacing websites.

---

## 3. NoSQL Injection

As modern applications move toward non-relational databases like MongoDB, attackers have adapted. Since these databases often
use JSON or JavaScript for queries, the injection looks different but has the same goal.

- **The Target:** NoSQL Databases (MongoDB, CouchDB).
- **The Attack:** Instead of `' OR 1=1`, an attacker might use a JSON operator like `{"$gt": ""}` (greater than nothing).
- **The Result:** If used in a login field, this can trick the database into returning the first user it finds, bypassing the
  password check.

---

## 4. LDAP Injection

Lightweight Directory Access Protocol (LDAP) is used for "Address Book" styles of data, often for corporate logins or
employee directories.

- **The Target:** Corporate identity and access management systems.
- **The Attack:** Using characters like `*`, `&`, and `|` to manipulate the LDAP filter.
- **The Goal:** Gaining access to the full employee directory or escalating privileges within a corporate network.

---

## 5. Server-Side Template Injection (SSTI)

Many web frameworks use "templates" (like Jinja2, Twig, or FreeMarker) to dynamically generate HTML. If an attacker can
inject code into these templates, they can often gain full Remote Code Execution (RCE).

- **The Target:** The web application's template engine.
- **The Attack:** Entering syntax like `{{ 7 * 7 }}`. If the page displays `49`, the attacker knows the server is executing
  their code.
- **The Goal:** Pivoting from simple math to executing system commands via the template's built-in functions.

---

## Summary Table

| Attack Type           | Target             | Primary Impact                         |
| :-------------------- | :----------------- | :------------------------------------- |
| **SQLi**              | Database           | Data theft, modification               |
| **Command Injection** | Operating System   | Full server takeover                   |
| **XSS**               | User Browser       | Identity theft, malware delivery       |
| **SSTI**              | Template Engine    | Remote Code Execution (RCE)            |
| **LDAP Injection**    | Identity Directory | Credential theft, privilege escalation |

---

## The Universal Solution: Defense in Depth

Regardless of the language or database, the defense remains largely the same:

1.  **Context-Aware Output Encoding:** Ensure data is treated as text, not code, when displayed.
2.  **Input Validation:** Use strict "allow-lists" for all user input.
3.  **Use Safe APIs:** Use parameterized queries for SQL, or library-specific functions that automatically handle escaping
    for LDAP or OS commands.
