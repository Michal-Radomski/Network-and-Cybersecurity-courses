To find out exactly which SSH key pair your computer is using to connect to GitHub, you can use a few quick terminal
commands.

Here is how to check, depending on whether you want to see the active connection or look at your saved keys.

### 1. The Direct Test (Recommended)

The fastest way to see exactly which key GitHub is recognizing is to run the SSH staging command with **verbose mode**
enabled. Open your terminal or Git Bash and run:

```bash
ssh -vT git@github.com

```

- **What to look for:** Look through the output lines (there will be a lot of text). You are looking for lines that say
  `Offering public key` or `Sending public key`.
- It will list the exact file path of the key it successfully used (e.g., `/users/yourname/.ssh/id_ed25519` or
  `/users/yourname/.ssh/id_rsa`).
- At the end, if successful, it will say: _“Hi [YourUsername]! You've successfully authenticated...”_

---

### 2. Check Your Local SSH Directory

If you want to see what keys you have available on your machine, you can look inside your hidden `.ssh` folder.

- **On Mac / Linux / Git Bash:**

```bash
ls -al ~/.ssh

```

- **On Windows (Command Prompt):**

```cmd
dir %userprofile%\.ssh

```

**Common key names you might see:**

- `id_ed25519` (Private key) & `id_ed25519.pub` (Public key) — _Modern standard_
- `id_rsa` (Private key) & `id_rsa.pub` (Public key) — _Older standard_

> ⚠️ **Important Note:** Never share the files _without_ the `.pub` extension. Those are your private keys!

---

### 3. Check if an SSH Agent is Managing Your Keys

If you've added your keys to the SSH agent so you don't have to type your password every time, you can list the fingerprints
of the currently active keys by running:

```bash
ssh-add -l

```

This will print the cryptographic fingerprint and the file path of the keys currently loaded into your computer's memory. You
can compare this fingerprint to the ones listed in your **GitHub Settings -> SSH and GPG keys** to see which one matches.
