Here is a quick-reference cheat sheet for SSH commands, organized by task.

---

## 🔑 1. Connecting to a Server

- **Standard Connection:** Connects using your current local username.

```bash
ssh example.com

```

- **Connect as a Specific User:**

```bash
ssh username@hostname_or_ip

```

- **Connect via a Specific Port:** (Default is 22).

```bash
ssh -p 2222 username@hostname_or_ip

```

- **Connect using a Specific Private Key (Identity File):**

```bash
ssh -i /path/to/private_key username@hostname_or_ip

```

---

## 🛠️ 2. Key Management & Generation

- **Generate a New SSH Key Pair:** (Ed25519 is recommended for modern security).

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"

```

- **Copy Public Key to a Server:** (Enables passwordless login).

```bash
ssh-copy-id username@hostname_or_ip

```

- **Copy Public Key via Specific Port:**

```bash
ssh-copy-id -p 2222 username@hostname_or_ip

```

- **View a Key's Fingerprint:**

```bash
ssh-keygen -l -f ~/.ssh/id_ed25519

```

---

## 📦 3. File Transfer (SCP & SFTP)

- **Copy a Local File to a Remote Server (SCP):**

```bash
scp /path/to/local/file.txt username@hostname_or_ip:/path/to/remote/dir/

```

- **Copy a Remote File to Local Machine (SCP):**

```bash
scp username@hostname_or_ip:/path/to/remote/file.txt /path/to/local/dir/

```

- **Copy an Entire Directory Remotely (SCP):**

```bash
scp -r /path/to/local/dir username@hostname_or_ip:/path/to/remote/dir/

```

- **Interactive File Transfer (SFTP):**

```bash
sftp username@hostname_or_ip

```

---

## 🚀 4. Executing Remote Commands

- **Run a Single Command and Exit:**

```bash
ssh username@hostname_or_ip "df -h"

```

- **Run a Local Script on a Remote Server:**

```bash
ssh username@hostname_or_ip 'bash -s' < local_script.sh

```

---

## 🌐 5. Port Forwarding & Tunneling

- **Local Port Forwarding:** (Access a remote server's port on your local machine).
  > _Example: Accessing a remote database running on port 3306 via your local port 8080._

```bash
ssh -L 8080:localhost:3306 username@hostname_or_ip

```

- **Remote Port Forwarding:** (Allow a remote server to access a port on your local machine).

```bash
ssh -R 8080:localhost:3000 username@hostname_or_ip

```

- **Dynamic Port Forwarding (SOCKS Proxy):**

```bash
ssh -D 1080 username@hostname_or_ip

```

---

## ⚙️ 6. Useful SSH Config (`~/.ssh/config`)

Instead of typing long commands every time, add shortcuts to your local `~/.ssh/config` file:

```text
Host myserver
    HostName 192.168.1.50
    User ubuntu
    Port 2222
    IdentityFile ~/.ssh/id_ed25519

```

Once saved, you can connect simply by typing:

```bash
ssh myserver

```

---

## 💡 Quick Tips & Troubleshooting

- **Fix "Permissions are too open" error:** SSH keys must have strict permissions.

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519

```

- **Debug Connection Issues:** Use the verbose flag (`-v`, `-vv`, or `-vvv`) to see what is happening behind the scenes.

```bash
ssh -v username@hostname_or_ip

```

- **Disconnect a Frozen Session:** Type `~` followed by `.` (Tilde and Period).
