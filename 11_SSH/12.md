Setting up and using GPG keys on Ubuntu is a great way to secure your files and sign your Git commits.

Here is a step-by-step guide to generating, using, and managing GPG keys on Ubuntu.

---

## Step 1: Generate a New GPG Key Pair

Ubuntu comes with `gpg` pre-installed. Open your terminal and run the following command to start the key generation wizard:

```bash
gpg --full-generate-key

```

You will be prompted to make a few choices:

1. **Kind of key:** Press `Enter` to select the default (`1` for RSA and RSA).
2. **Key size:** Type `4096` for maximum security, then press `Enter`.
3. **Validity period:** Press `Enter` if you want the key to never expire, or specify a time (e.g., `1y` for one year).
4. **User ID details:** Enter your Real Name and Email Address. (If you are using this for GitHub, use the email address
   associated with your GitHub account).
5. **Passphrase:** Enter a strong password to protect your private key. **Do not forget this passphrase!**

---

## Step 2: List Your GPG Keys

To make sure your key was created successfully and to find its **Key ID**, run:

```bash
gpg --list-secret-keys --keyid-format=long

```

You will see output that looks like this:

```text
sec   rsa4096/3AA5C34371567BDD 2026-06-20 [SC] [expires: 2027-06-20]
uid                 [ultimate] Your Name <your.email@example.com>
ssb   rsa4096/42B317FD4F0D25E4 2026-06-20 [E]

```

> **Note:** Your **Key ID** is the alphanumeric string after the key type. In the example above, the Key ID is
> `3AA5C34371567BDD`.

---

## Step 3: Common Ways to Use Your GPG Key

### 1. Encrypt and Decrypt a File

To encrypt a sensitive file so that only you (or someone you send it to) can open it:

- **To Encrypt:**

```bash
gpg --encrypt --recipient your.email@example.com secret.txt

```

This creates a new file called `secret.txt.gpg`.

- **To Decrypt:**

```bash
gpg --decrypt secret.txt.gpg > secret.txt

```

You will be prompted to enter your passphrase to unlock the key.

### 2. Export Your Public Key (For GitHub/GitLab)

If you want to use your GPG key to sign Git commits, you need to share your public key with GitHub.

To print your public key in text format, run this command (replace `YOUR_KEY_ID` with your actual ID from Step 2):

```bash
gpg --armor --export YOUR_KEY_ID

```

Copy everything from `-----BEGIN PGP PUBLIC KEY BLOCK-----` to `-----END PGP PUBLIC KEY BLOCK-----` and paste it into your
GitHub/GitLab GPG settings.

### 3. Configure Git to Use Your GPG Key

To automatically sign all your Git commits on your Ubuntu machine, run these configuration commands:

```bash
# Tell Git which key to use
git config --global user.signingkey YOUR_KEY_ID

# Tell Git to sign all commits by default
git config --global commit.gpgsign true

```

---

## Step 4: Back Up Your GPG Keys (Crucial)

If your computer crashes, you lose your keys forever. It is highly recommended to export your private key and save it
somewhere secure (like an encrypted USB drive).

- **Backup Public Key:**

```bash
gpg --armor --export your.email@example.com > my_public_key.asc

```

- **Backup Private Key:**

```bash
gpg --armor --export-secret-keys your.email@example.com > my_private_key.asc

```
