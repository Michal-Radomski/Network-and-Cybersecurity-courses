To get the algorithm and bit length of an SSH key, you can use the `ssh-keygen` command-line tool. It reads the public key
file (the one ending in `.pub`) and displays its metadata.

Here is the quickest way to do it:

### The Standard Command

Open your terminal and run the following command, replacing `id_rsa.pub` with the actual name of your public key file:

```bash
ssh-keygen -l -f ~/.ssh/id_rsa.pub

```

### Understanding the Output

The output will look something like this:

```text
3072 SHA256:bcdEfgH123... user@hostname (RSA)

```

Here is how to decode that information:

- **`3072`**: The **bit length** (keysize).
- **`SHA256:...`**: The fingerprint of the key.
- **`(RSA)`**: The **algorithm** used to generate the key (e.g., RSA, ECDSA, ED25519).

---

### What if you only have the private key?

If you accidentally deleted the `.pub` file or only have the private key file (the one without an extension), you can still
extract the information by running:

```bash
ssh-keygen -y -f ~/.ssh/id_rsa

```

> **Note:** This will output the public key string. To get the bit length and algorithm directly from it, you can pipe it
> right back into the fingerprint command like this: `ssh-keygen -y -f ~/.ssh/id_rsa | ssh-keygen -l -f -`
