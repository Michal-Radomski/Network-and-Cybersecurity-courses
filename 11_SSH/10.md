An SSH key fingerprint is a condensed, shortened version of your **public** SSH key.

Because public keys are massive, complex strings of characters (often hundreds of lines long), they are nearly impossible for
a human to read or compare quickly. The fingerprint solves this by compressing that massive key into a short, unique string
of characters—acting just like a human fingerprint.

---

## 🔍 What it Looks Like

When you view a fingerprint, it typically looks like a string of hexadecimal characters or a Base64 string, usually preceded
by the hashing algorithm used to create it (most commonly SHA256).

**Example of an SHA256 Fingerprint:**

```text
SHA256:uR0W4h9wM6vP8kZ9xY2qT4bN1mE8vK3zC5xJ7qL0wA4

```

---

## 🛡️ Why Do We Need It?

The primary purpose of a fingerprint is **verification and security**.

When you connect to an SSH server for the very first time, your computer doesn't know if it can trust that server yet. To
prevent an attacker from intercepting your connection (a "Man-in-the-Middle" attack), the server presents its public key.

Instead of showing you the massive public key, your terminal shows you the server's **fingerprint** and asks:

> _"The authenticity of host 'example.com' can't be established. Are you sure you want to continue connecting?"_

If you want to be 100% secure, you can contact the server administrator via a secure channel (like an encrypted chat or
looking at a company dashboard), ask them for the server's official fingerprint, and verify that it matches exactly what is
on your screen.

---

## ⚙️ How it is Made (Hashing)

A fingerprint is created using a **cryptographic hash function** (usually SHA256).

```
[ Massive Public Key Data ]  ───►  ( SHA256 Hash Function )  ───►  [ Short Unique Fingerprint ]

```

This process is a one-way street:

- You can easily turn a public key into a fingerprint.
- You **cannot** reverse-engineer a fingerprint back into the public key.
- Even if a single character changes in the massive public key, the resulting fingerprint will look completely, radically
  different.

---

## 🛠️ How to View Your Key's Fingerprint

If you want to check the fingerprint of your own local SSH keys, you can run the following command in your terminal:

```bash
ssh-keygen -l -f ~/.ssh/id_ed25519.pub

```

_(This reads your public key file and prints out its hash algorithm, bit-length, fingerprint string, and the email associated
with it.)_
