Setting up SSH keys is one of the best ways to secure your server connections. It replaces vulnerable passwords with
cryptographic key pairs.

Here is how to generate your keys, add the private key to your local Ubuntu machine, and authorize the public key on your
Ubuntu server.

---

## Step 1: Generate the SSH Key Pair

You should run this command on your **local PC** (your everyday computer).

1. Open your terminal.
2. Run the following command to generate a highly secure Ed25519 key pair:

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"

```

_(Note: If you have an older system that doesn't support Ed25519, use `ssh-keygen -t rsa -b 4096` instead)._ 3. **Prompt -
Enter file in which to save the key:** Press `Enter` to accept the default location (`~/.ssh/id_ed25519`). 4. **Prompt -
Enter passphrase:** Type a secure passphrase. This adds an extra layer of security so that even if someone steals your
private key file, they can't use it without the passphrase.

This creates two files in your `~/.ssh/` directory:

- `id_ed25519` (Your **Private** key — _Never share this with anyone!_)
- `id_ed25519.pub` (Your **Public** key — _Safe to share._)

---

## Step 2: Add Your Private Key to the Ubuntu SSH Agent

To avoid typing your passphrase every single time you connect to your server, you can add your private key to the Ubuntu SSH
agent on your **local PC**.

1. Start the SSH agent in the background:

```bash
eval "$(ssh-agent -s)"

```

2. Add your private SSH key to the agent:

```bash
ssh-add ~/.ssh/id_ed25519

```

---

## Step 3: Add Your Public Key to the Ubuntu Server

Now, you need to copy the **public key** to your remote server so it recognizes your local PC. Choose **one** of the two
methods below:

### Method A: The Easy Way (Using `ssh-copy-id`)

If you currently have password access to your server, Ubuntu has a built-in tool that handles this automatically. Run this on
your **local PC**:

```bash
ssh-copy-id username@remote_server_ip

```

_Input your server user's password when prompted. The tool will automatically copy your public key to the correct file on the
server._

### Method B: The Manual Way

If `ssh-copy-id` isn't working or isn't available, you can do it manually:

1. On your **local PC**, view and copy your public key:

```bash
cat ~/.ssh/id_ed25519.pub

```

_(Highlight and copy the entire output string starting with `ssh-ed25519`...)_ 2. Log into your **remote server** (using your
password for the last time):

```bash
ssh username@remote_server_ip

```

3. Create the `.ssh` directory and set the correct permissions:

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh

```

4. Open (or create) the `authorized_keys` file with a text editor:

```bash
nano ~/.ssh/authorized_keys

```

5. Paste your public key into this file on a new line. Save and exit (Press `Ctrl+O`, `Enter`, then `Ctrl+X`).
6. Set the correct file permissions for the key file:

```bash
chmod 600 ~/.ssh/authorized_keys

```

---

## Step 4: Test Your Connection

From your **local PC**, try logging into the server. It should log you in instantly without asking for your server user
password (it may ask for your key's _passphrase_ if your SSH agent isn't running).

```bash
ssh username@remote_server_ip

```
