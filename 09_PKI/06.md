Encrypting and decrypting files with symmetric encryption (where the same key/password is used for both actions) is
straightforward using OpenSSL on Ubuntu. The current standard recommended algorithm is **AES-256-CBC** or **AES-256-GCM**.

Here is how to do it using the terminal.

---

## 1. Encrypting a File

To encrypt a file, use the `openssl enc` command.

```bash
openssl enc -aes-256-cbc -salt -pbkdf2 -in secrets.txt -out secrets.txt.enc

```

**What these flags mean:**

- `-aes-256-cbc`: The encryption cipher (AES with a 256-bit key in CBC mode).
- `-salt`: Adds a random salt to the file to protect against dictionary attacks.
- `-pbkdf2`: Uses Password-Based Key Derivation Function 2. This is **crucial** in modern OpenSSL to ensure your password is
  secure.
- `-in`: The original file you want to encrypt.
- `-out`: The name of the new, encrypted file.

> ⚠️ **Note:** After running the command, you will be prompted to enter and verify a password. Don't forget this password, as
> the file cannot be recovered without it!

---

## 2. Decrypting the File

To reverse the process and get your original file back, use the same command but add the `-d` (decrypt) flag.

```bash
openssl enc -d -aes-256-cbc -pbkdf2 -in secrets.txt.enc -out decrypted_secrets.txt

```

**What these flags mean:**

- `-d`: Tells OpenSSL to decrypt instead of encrypt.
- `-in`: The encrypted file.
- `-out`: The destination where your decrypted data will be saved.

> ⚠️ **Note:** You must use the exact same cipher (`-aes-256-cbc`) and derivation function (`-pbkdf2`) that you used to
> encrypt the file, and enter the correct password when prompted.

---

## Pro-Tip: Compression

If you are encrypting a large file or a folder, it's usually best to zip or tar it first, then encrypt the resulting archive:

```bash
# Tar a folder and encrypt it in one go
tar -czf - my_folder/ | openssl enc -aes-256-cbc -salt -pbkdf2 -out folder.tar.gz.enc

```
