To encrypt and decrypt a file using asymmetric encryption (a public/private key pair) with OpenSSL, you use the **public key
to encrypt** the file and the **private key to decrypt** it.

Here is how to do it step-by-step on Ubuntu.

---

## 1. Encrypting a File (Using the Public Key)

Anyone who has your public key can encrypt a file for you. To encrypt a file named `secrets.txt` using `public_key.pem`, run
the following command:

```bash
openssl pkeyutl -encrypt -pubin -inkey public_key.pem -in secrets.txt -out secrets.txt.enc

```

**What these flags mean:**

- `pkeyutl`: The OpenSSL public key utility command.
- `-encrypt`: Specifies that you want to encrypt data.
- `-pubin`: Informs OpenSSL that the input key is a public key.
- `-inkey public_key.pem`: Specifies the path to your public key file.
- `-in secrets.txt`: The original file you want to protect.
- `-out secrets.txt.enc`: The name of the resulting encrypted file.

---

## 2. Decrypting the File (Using the Private Key)

Only the holder of the private key can decrypt the file. To decrypt `secrets.txt.enc` using `private_key.pem`, run:

```bash
openssl pkeyutl -decrypt -inkey private_key.pem -in secrets.txt.enc -out decrypted_secrets.txt

```

**What these flags mean:**

- `-decrypt`: Specifies that you want to decrypt data.
- `-inkey private_key.pem`: Specifies the path to your private key file.
- `-in secrets.txt.enc`: The encrypted file.
- `-out decrypted_secrets.txt`: The destination where the recovered, original data will be saved.

---

## ⚠️ Important Limitation: File Size

Asymmetric encryption (RSA) is mathematically limited by the size of the key.

- For example, a standard 2048-bit RSA key can only encrypt files smaller than **245 bytes**.
- If you try to encrypt a large file (like a PDF or a ZIP file) this way, OpenSSL will throw an error:
  `data too large for key size`.

> **The Standard Workaround (Hybrid Encryption):** > If you need to encrypt a large file securely for someone else, you
> should encrypt the file using **symmetric encryption** (like AES-256, which has no size limit) using a random password, and
> then use their **public key** to encrypt just that password.
