To hash a text string using OpenSSL on Ubuntu, you can use the `openssl dgst` (digest) command.

Here is how to do it using different popular hashing algorithms right from your terminal.

---

## 1. The Standard Way (SHA-256)

To hash a string like `"hello world"`, echo the text and pipe it into OpenSSL:

```bash
echo -n "hello world" | openssl dgst -sha256

```

**What the flags mean:**

- `echo -n`: The `-n` flag is **crucial**. It prevents Linux from adding a hidden newline character (`\n`) to the end of your
  text. If you leave it out, you will get a completely different hash.
- `dgst`: Tells OpenSSL you want to use a message digest (hashing) utility.
- `-sha256`: Specifies the SHA-256 algorithm.

**Output:**

```text
(stdin)= b94d27b9934d3e08a52e52d7da7dabfac484efe37a5380ee9088f7ace2efcde9

```

---

## 2. Using Other Hashing Algorithms

You can easily swap out `-sha256` for other algorithms depending on your needs:

### SHA-512 (More secure/longer output)

```bash
echo -n "hello world" | openssl dgst -sha512

```

### SHA-3 (Next-generation standard)

```bash
echo -n "hello world" | openssl dgst -sha3-256

```

---

## 3. Getting Just the Clean Hash Output

By default, OpenSSL prefixes the output with `(stdin)=`. If you are using this in a bash script and only want the raw
hexadecimal hash string, add the `-r` (reverse) flag and parse it, or pass it directly like this:

```bash
echo -n "hello world" | openssl dgst -sha256 | awk '{print $2}'

```

**Output:**

```text
b94d27b9934d3e08a52e52d7da7dabfac484efe37a5380ee9088f7ace2efcde9

```
