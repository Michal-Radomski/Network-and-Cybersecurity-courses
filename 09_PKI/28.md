To get a free, trusted SSL certificate from Let's Encrypt on Ubuntu with Apache, the easiest and most secure method is to use
**Certbot**. Certbot is an automated tool developed by the Electronic Frontier Foundation (EFF) that handles obtaining,
installing, and automatically renewing the certificate.

Here is the step-by-step guide to setting it up.

---

## Prerequisites

Before you start, make sure:

1. You have a registered domain name (e.g., `example.com`).
2. Your domain's DNS settings are pointed to your Ubuntu server's public IP address.
3. You have an Apache Virtual Host file set up for your domain.

---

## Step 1: Install Certbot and its Apache Plugin

Ubuntu's package manager (`apt`) includes Certbot and the necessary Apache plugin. Run the following commands to update your
package list and install them:

```bash
sudo apt update
sudo apt install certbot python3-certbot-apache -y

```

---

## Step 2: Allow HTTPS Through the Firewall

If you have the UFW firewall enabled on Ubuntu, you need to make sure it allows secure web traffic (port 443).

Check your firewall profile by running:

```bash
sudo ufw status

```

If it says active, allow the **Apache Full** profile, which opens both standard HTTP (port 80) and secure HTTPS (port 443)
traffic:

```bash
sudo ufw allow 'Apache Full'

```

---

## Step 3: Run Certbot to Get and Install the Certificate

The Certbot Apache plugin makes the process incredibly easy. It will automatically scan your Apache configurations, find your
domain name, request the certificate from Let's Encrypt, and update your Apache configuration to use it.

Run this command:

```bash
sudo certbot --apache

```

### What happens next:

1. **Enter Email:** Certbot will ask for an email address. This is used for important renewal notices and security alerts.
2. **Terms of Service:** Press **A** to agree to the Let's Encrypt terms of service.
3. **Mailing List (Optional):** Press **Y** or **N** if you want to share your email with the EFF.
4. **Select Domain:** Certbot will display a list of the domain names found in your Apache Virtual Host files. Type the
   numbers corresponding to the domains you want to secure (or leave it blank to select all of them) and press **Enter**.
5. **Redirect Traffic:** Certbot will ask if you want to automatically redirect all standard HTTP traffic to secure HTTPS. It
   is highly recommended to choose **Redirect** (usually option `2`).

Once finished, Certbot will output a success message showing where your new certificate files are stored.

---

## Step 4: Verify the Installation

Open your web browser and navigate to your website using `https://` (e.g., `https://example.com`).

You should see a secure padlock icon in the address bar next to your URL. If you click on it, it will show that the
certificate was issued by **Let's Encrypt** and is perfectly trusted.

---

## Step 5: Test Automatic Renewal

Let's Encrypt certificates are valid for **90 days**. However, the Certbot installation automatically sets up a background
system timer (cron job / systemd timer) that checks your certificates twice a day and renews them automatically if they are
within 30 days of expiring.

You can verify that the automatic renewal system works flawlessly by running a "dry run":

```bash
sudo certbot renew --dry-run

```

If the command finishes without errors, your certificates will safely renew in the background forever without you having to
lift a finger again!
