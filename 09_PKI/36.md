Here is a cheat sheet of the most common and useful OpenSSL commands, categorized by what you need to accomplish.

---

## 1. Generating Keys and Certificate Requests (CSRs)

### Generate a standard RSA Private Key (2048-bit)

```bash
openssl genrsa -out private.key 2048

```

### Generate a modern Elliptic Curve (ECC) Private Key

```bash
openssl ecparam -name prime256v1 -genkey -out ecc-private.key

```

### Create a Certificate Signing Request (CSR) from an existing key

```bash
openssl req -new -key private.key -out request.csr

```

### Generate a Private Key and a Self-Signed Certificate simultaneously

```bash
openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout private.key -out certificate.crt

```

---

## 2. Viewing and Checking Certificates/Keys

Because these files are Base64 encoded, you use these commands to decode and read their plain-text contents.

### View the full details of a Certificate (.crt or .pem)

```bash
openssl x509 -in certificate.crt -text -noout

```

### View the details of a CSR (Certificate Signing Request)

```bash
openssl req -in request.csr -text -noout

```

### Check if a Private Key is valid and intact

```bash
openssl rsa -in private.key -check

```

---

## 3. Verifying Matching Keys (MD5 Matching)

When installing a certificate, the Certificate and the Private Key must cryptographically match. You can verify this by
checking if their MD5 hashes match exactly:

### Check MD5 hash of a Certificate:

```bash
openssl x509 -noout -modulus -in certificate.crt | openssl md5

```

### Check MD5 hash of a Private Key:

```bash
openssl rsa -noout -modulus -in private.key | openssl md5

```

---

## 4. Converting Certificate Formats

### Convert PEM (.pem/.crt text) to DER (.der binary)

```bash
openssl x509 -in cert.pem -outform der -out cert.der

```

### Convert DER (.der binary) to PEM (.pem text)

```bash
openssl x509 -inform der -in cert.der -outform pem -out cert.pem

```

### Combine a Certificate and Private Key into a PKCS#12 (.pfx / .p12) file

_(Commonly used for Windows servers or IIS)_

```bash
openssl pkcs12 -export -out bundle.pfx -inkey private.key -in certificate.crt

```

---

## 5. Testing Network Connections (Debugging)

OpenSSL can act like `telnet` or `curl` but specifically for testing secure handshakes.

### Test an HTTPS connection to a remote server

This displays the entire certificate chain and allows you to test connection issues.

```bash
openssl s_client -connect google.com:443

```

### Test a connection forcing a specific TLS version (e.g., TLS 1.3)

```bash
openssl s_client -connect google.com:443 -tls1_3

```
