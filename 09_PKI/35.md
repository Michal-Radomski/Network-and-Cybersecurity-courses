Configuring OpenSSL on Ubuntu involves managing its global configuration file, which determines default settings like
cryptographic algorithms, key lengths, and directory paths for certificates.

On Ubuntu, the main OpenSSL configuration file is located at `/usr/lib/ssl/openssl.cnf` (with a symlink often found at
`/etc/ssl/openssl.cnf`).

Here is how to safely view, edit, and apply changes to your OpenSSL configuration.

---

## Step 1: Backup the Original Configuration

Before making any changes to a core security file, it is vital to create a backup so you can easily revert if a syntax error
breaks your system's cryptographic utilities.

Run the following command:

```bash
sudo cp /etc/ssl/openssl.cnf /etc/ssl/openssl.cnf.bak

```

---

## Step 2: Understand the Structure of `openssl.cnf`

Open the file using a text editor like `nano`:

```bash
sudo nano /etc/ssl/openssl.cnf

```

The file is organized into distinct **sections** denoted by square brackets (e.g., `[ section_name ]`). The most important
sections include:

- **`[ca]`**: Defines default directories, database file locations, and serial numbers used when your system acts as a
  Certificate Authority.
- **`[req]`**: Controls the settings used when generating Certificate Signing Requests (CSRs) or self-signed certificates.
- **`[crypto_policy]`**: (In newer Ubuntu versions) Points to system-wide cryptographic policies restricting weak algorithms.

---

## Step 3: Common Configuration Customizations

Here are the most common edits administrators make to optimize or secure OpenSSL for their environment:

### 1. Changing Default Identity Information

Scroll down to the `[ req_distinguished_name ]` section. You can modify the default entries so you don't have to retype your
location or company name every time you generate a certificate:

```ini
[ req_distinguished_name ]
countryName_default             = US
stateOrProvinceName_default     = California
localityName_default            = San Francisco
0.organizationName_default      = My Company LLC
organizationalUnitName_default  = IT Department

```

### 2. Increasing Default Key Size

By default, older configurations might still target a 2048-bit RSA key. If you want to increase the standard security
baseline for newly generated keys to 4096 bits, find the `[ req ]` section and modify `default_bits`:

```ini
[ req ]
default_bits        = 4096
default_md          = sha256

```

### 3. Enabling Elliptic Curve Cryptography (ECC) by Default

If you want to transition your environment away from RSA and default to the modern, faster Elliptic Curve Cryptography, you
can change the default key type settings inside the `[ req ]` block:

```ini
[ req ]
default_bits        = 256
default_keyfile     = privkey.pem
# Tell OpenSSL to use the P-256 curve by default
new_key             = ec
ec_paramgen_curve   = prime256v1

```

---

## Step 4: Activating Advanced Options (Like SANs)

Modern browsers require certificates to use **Subject Alternative Names (SANs)** rather than relying strictly on the old
Common Name field. To configure OpenSSL to prompt you for SANs automatically during setup, add the following lines under the
`[ req ]` section:

```ini
req_extensions = v3_req

[ v3_req ]
# Extensions to add to a certificate request
basicConstraints = CA:FALSE
keyUsage = nonRepudiation, digitalSignature, keyEncipherment
subjectAltName = @alt_names

[ alt_names ]
DNS.1 = example.com
DNS.2 = www.example.com
IP.1  = 192.168.1.50

```

Save and exit the file (`Ctrl+O`, `Enter`, then `Ctrl+X`).

---

## Step 5: Test and Verify Your Configuration

To ensure your changes were formatted correctly and that OpenSSL reads the configuration file without crashing, try
generating a mock certificate using your new defaults:

```bash
openssl req -x509 -nodes -days 30 -newkey rsa -keyout /tmp/test.key -out /tmp/test.crt

```

If the prompt successfully reads your new default country, state, or key sizes without throwing a syntax error, your OpenSSL
configuration is completely ready and active system-wide.
