Generating a self-signed certificate on Ubuntu is a straightforward process using **OpenSSL**, a tool that comes
pre-installed on almost all Ubuntu systems.

Because it is self-signed, it won't be trusted by public web browsers (you will see a "Warning: Untrusted Certificate"
message), but it is perfect for local development, testing, or internal company servers.

Here is how to generate one using the terminal.

---

## Step 1: Open the Terminal and Create the Key and Certificate

You can generate both the private key and the digital certificate with a single, standard OpenSSL command.

Run the following command in your terminal:

```bash
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/ssl/private/nginx-selfsigned.key \
  -out /etc/ssl/certs/nginx-selfsigned.crt

```

### What this command means:

- **`req -x509`**: Tells OpenSSL we want to create a self-signed certificate rather than generating a Certificate Signing
  Request (CSR) to send to a CA.
- **`-nodes`**: (No DES) Tells OpenSSL to skip the option to secure the private key with a passphrase. This is important
  because it allows your web server (like Nginx or Apache) to restart automatically without requiring a human to type a
  password every time.
- **`-days 365`**: Sets the expiration date. In this case, the certificate will be valid for exactly one year.
- **`-newkey rsa:2048`**: Generates a new certificate and a new 2048-bit RSA private key simultaneously.
- **`-keyout`**: Specifies the directory and filename where the secret **private key** will be saved.
- **`-out`**: Specifies the directory and filename where the public **certificate** will be saved.

---

## Step 2: Fill Out the Identity Information

Once you hit enter, OpenSSL will ask you a series of questions to build the certificate's identity profile (the X.509
fields).

Fill them out or press **Enter** to leave them default. The most critical line is the **Common Name**:

```text
Country Name (2 letter code) [AU]:US
State or Province Name (full name) [Some-State]:California
Locality Name (eg, city) []:San Francisco
Organization Name (eg, company) [Internet Widgits Pty Ltd]:My Development Co
Organizational Unit Name (eg, section) []:IT
Common Name (e.g. server FQDN or YOUR name) []:localhost
Email Address []:admin@localhost

```

> 💡 **Important:** For the **Common Name (CN)**, type the exact domain name or IP address you will use to access the server
> (e.g., `localhost`, `127.0.0.1`, or `dev.mysite.local`).

---

## Step 3: Verify the Files Were Created

Your self-signed certificate pair is now ready. You can double-check that the files exist and are secure by running:

```bash
ls -l /etc/ssl/certs/nginx-selfsigned.crt
ls -l /etc/ssl/private/nginx-selfsigned.key

```

Because the private key file (`.key`) is highly sensitive, Ubuntu automatically restricts its permissions so that only the
system root/admin can read it.

---

## Next Steps: Configuring Your Web Server

Now that you have the `.crt` (the certificate) and the `.key` (the private key), you point your web server configuration file
to their paths.

- **For Nginx:** You would add these lines to your server block:

```nginx
ssl_certificate /etc/ssl/certs/nginx-selfsigned.crt;
ssl_certificate_key /etc/ssl/private/nginx-selfsigned.key;

```

- **For Apache:** You would add these lines to your VirtualHost block:

```apache
SSLCertificateFile /etc/ssl/certs/nginx-selfsigned.crt
SSLCertificateKey /etc/ssl/private/nginx-selfsigned.key

```
