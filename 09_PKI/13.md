Popular hashing algorithms are generally split into two categories based on how they are meant to be used: **Fast Hashes**
(for data integrity, digital signatures, and blockchain) and **Slow Hashes** (specifically designed for password security).

Here is a breakdown of the most popular algorithms used today, ranked by their current safety status.

---

## 1. High-Speed Cryptographic Hashes (For Integrity & Signatures)

These algorithms are designed to be computed as quickly as possible. They are perfect for verifying that a file wasn't
corrupted during a download or for signing digital certificates, but they are **terrible for password storage** because
hackers can guess billions of combinations per second.

### SHA-256 (Secure Hash Algorithm 2)

- **Status:** Strong and industry standard.
- **Output Size:** 256 bits (64 hexadecimal characters).
- **Common Uses:** Bitcoin and cryptocurrency mining, SSL/TLS certificates, and file integrity checks (like verifying Linux
  ISO downloads).
- **Why it's popular:** It is incredibly secure with no known practical cryptographic weaknesses. It belongs to the SHA-2
  family, which also includes its larger sibling, **SHA-512**.

### SHA-3

- **Status:** Strong (Next-Generation).
- **Output Size:** Flexible (commonly 256 or 512 bits).
- **Common Uses:** Modern security systems, Ethereum (Keccak-256 variant).
- **Why it's popular:** It was created by independent cryptographers and chosen by NIST as a backup standard. Internally, it
  uses a unique "sponge construction" mathematical layout, meaning that if a structural flaw is ever discovered in SHA-2,
  SHA-3 will remain completely unaffected.

---

## 2. Intentionally Slow Hashes (Specifically for Passwords)

When storing user passwords, speed is your enemy. These algorithms are designed to intentionally hog CPU or RAM resources to
make brute-force attacks painfully slow for attackers.

### Argon2 (Argon2id)

- **Status:** The gold standard.
- **Common Uses:** Modern authentication systems, password managers (like KeePass).
- **Why it's popular:** It won the Password Hashing Competition. It is "memory-hard," meaning it requires a massive amount of
  RAM to process. This completely neutralizes hackers who try to use specialized graphics cards (GPUs) or custom hacking
  chips (ASICs) to crack passwords in parallel. The `Argon2id` hybrid variant is the standard implementation choice.

### bcrypt

- **Status:** Strong and widely supported.
- **Common Uses:** Web frameworks (Node.js, Ruby on Rails, Django).
- **Why it's popular:** As we discussed earlier, it has an adjustable "Work Factor" (or cost factor). As computers grow
  faster over time, developers can simply bump up the cost factor to keep the hash slow and secure without altering the
  underlying code.

---

## 3. Outdated & Legacy Hashes (Do Not Use)

These older algorithms were highly popular for decades but have been cryptographically broken. They should only be used for
legacy compatibility or basic data sorting.

### MD5 (Message Digest 5)

- **Status:** Severely Broken.
- **Output Size:** 128 bits.
- **Why it's obsolete:** It suffers from **collisions**, which means an attacker can easily generate two completely different
  files that produce the exact same MD5 hash. Today, it is only used as a non-secure checksum to see if a file transferred
  correctly.

### SHA-1

- **Status:** Depreciated / Broken.
- **Output Size:** 160 bits.
- **Why it's obsolete:** Major tech companies like Google, Microsoft, and Mozilla completely stripped away support for SHA-1
  after successful collision attacks were executed against it. It has been phased out entirely in favor of SHA-256.

---

## Quick Summary

| Algorithm    | Type | Ideal Use Case               | Security Level   |
| ------------ | ---- | ---------------------------- | ---------------- |
| **Argon2id** | Slow | Best for User Passwords      | 🟢 **Excellent** |
| **bcrypt**   | Slow | Great for User Passwords     | 🟢 **Very Good** |
| **SHA-256**  | Fast | Data Integrity, Crypto, TLS  | 🟢 **Excellent** |
| **SHA-3**    | Fast | Next-gen Integrity, Ethereum | 🟢 **Excellent** |
| **SHA-1**    | Fast | Legacy compatibility only    | 🔴 **Broken**    |
| **MD5**      | Fast | Non-secure file checksums    | 🔴 **Broken**    |
