A **Cipher Suite** is a standardized set of cryptographic algorithms used to secure a network connection. When your web
browser connects to a secure website (using HTTPS), the cipher suite acts as a recipe book, telling both sides exactly which
mathematical formulas to use to encrypt the data, verify identities, and ensure the connection hasn't been tampered with.

During the initial connection (called the **TLS Handshake**), your browser and the website's server negotiate and agree upon
a single cipher suite that they both support.

---

## The Components of a Cipher Suite

A traditional cipher suite name looks complicated, but it is actually just a string of abbreviations broken down into four
distinct parts.

Take this common TLS 1.2 cipher suite as an example:

$$TLS\_ECDHE\_RSA\_WITH\_AES\_128\_GCM\_SHA256$$

Here is what each component does:

### 1. Key Exchange (`ECDHE`)

- **What it does:** Determines how the client (browser) and server will securely agree upon a shared secret password over an
  unsecure network.
- **Examples:** `ECDHE` (Elliptic Curve Diffie-Hellman Ephemeral), `DH` (Diffie-Hellman).

### 2. Authentication (`RSA`)

- **What it does:** Ensures that you are actually talking to the real server and not an impostor. It uses the website's
  digital signature and certificate to verify its identity.
- **Examples:** `RSA`, `ECDSA`.

### 3. Bulk Encryption (`AES_128_GCM`)

- **What it does:** This is the symmetric encryption algorithm used to scramble the actual data (like passwords, credit
  cards, or messages) being sent back and forth.
- **Components:** It lists the algorithm (`AES`), the key size (`128`-bit), and the block cipher mode (`GCM`).

### 4. Integrity / Hashing (`SHA256`)

- **What it does:** Uses a Message Authentication Code (MAC) or hashing function to create a "digital fingerprint" of the
  data. This guarantees that a hacker hasn't secretly altered the data mid-transit.
- **Examples:** `SHA256`, `SHA384`.

---

## Modern Shift: TLS 1.3 Cipher Suites

In **TLS 1.3** (the modern standard for web security), the process was drastically simplified. Because older algorithms had
security flaws, TLS 1.3 removed the Key Exchange and Authentication mechanisms from the cipher suite name entirely (handling
them automatically in the background).

A modern TLS 1.3 cipher suite looks much shorter:

$$TLS\_AES\_256\_GCM\_SHA384$$

It only specifies the **Bulk Encryption** algorithm and the **Hash/Integrity** algorithm.
