Windows and Linux Ubuntu offer built-in and third-party methods for securely deleting data by overwriting it multiple times,
making recovery difficult even with forensic tools. Note that effectiveness varies on SSDs due to wear-leveling; for those,
full-disk encryption or manufacturer secure erase is often preferable. [youtube](https://www.youtube.com/watch?v=5y1I2CPkshY)

## Windows Methods

Several command-line and GUI tools ensure data overwriting.

- **Cipher utility**: Built-in tool to wipe free space after deletion. Run `cipher /w:C:\path\to\folder` as admin to
  overwrite deleted files' space with three passes (zeros, ones, random data).
  [youtube](https://www.youtube.com/watch?v=5y1I2CPkshY)
- **SDelete (Sysinternals)**: Free Microsoft tool for DoD-compliant deletion (e.g., 3-7 passes). Download from Microsoft,
  then `sdelete -p 3 -s C:\path\to\file.txt` for files or `-c` for free space.
  [learn.microsoft](https://learn.microsoft.com/en-us/sysinternals/downloads/sdelete)
- **Batch scripts**: Combine `echo` for overwriting content, `del`, and `cipher` in a .bat file run as admin.
  [youtube](https://www.youtube.com/watch?v=5y1I2CPkshY)
- **DiskPart or BitLocker**: For full drives, `clean all` wipes disks (caution: erases everything); enable BitLocker first
  for encryption-based security. [youtube](https://www.youtube.com/watch?v=5y1I2CPkshY)
- **BleachBit**: Open-source GUI for file wiping and free space cleaning.
  [ssd.eff](https://ssd.eff.org/module/como-borrar-tu-informaci%C3%B3n-de-forma-segura-en-windows)

## Ubuntu Linux Methods

Coreutils and specialized packages provide overwriting capabilities.

- **Shred**: Built-in utility (GNU coreutils). Use `shred -u -v -n 3 file.txt` to overwrite 3 times, then delete (`-u`); `-z`
  adds final zero pass. Works on files/devices.
  [tecmint](https://www.tecmint.com/permanently-and-securely-delete-files-directories-linux/)
- **Wipe**: Install via `sudo apt install wipe`, then `wipe -r -i file.txt` for recursive wipe with confirmation.
  [geeksforgeeks](https://www.geeksforgeeks.org/linux-unix/tools-to-securely-delete-files-from-linux/)
- **Secure-Delete suite**: Install `sudo apt install secure-delete`. Key tools:
  - `srm -v file.txt`: Secure rm with multiple passes.
  - `sfill -v /path`: Wipe free space.
  - `sswap`: Securely wipe swap (disable first with `swapoff`).
  - `sdmem`: Wipe RAM. [linuxconfig](https://linuxconfig.org/quick-guide-to-securely-erasing-files-in-linux)
- **dd**: Manual overwrite, e.g., `dd if=/dev/urandom of=file.txt; shred -u file.txt` (risky for beginners).
  [geeksforgeeks](https://www.geeksforgeeks.org/linux-unix/tools-to-securely-delete-files-from-linux/)

## Key Considerations

Always run as root/admin, back up unrelated data, and test on non-critical files. On SSDs/NVMe, traditional overwriting is
less reliable—use TRIM (`fstrim` on Linux) or ATA Secure Erase via tools like `hdparm` or manufacturer software. For maximum
security, encrypt first (BitLocker/FileVault or LUKS) and delete keys.
[stackoverflow](https://stackoverflow.com/questions/62898222/secure-delete-files-on-windows-10)

These methods follow standards like DoD 5220.22-M or Gutmann, ensuring data is practically irrecoverable on HDDs.
[learn.microsoft](https://learn.microsoft.com/en-us/sysinternals/downloads/sdelete)

---

Standard file deletion (sending a file to the Recycle Bin or Trash) doesn't actually erase the data; it simply tells the
operating system that the space is "available" for new data. Until that space is overwritten, the original files can be
easily recovered using forensic tools.

To truly sanitize your storage, you need to overwrite the data sectors. Here is how to do it securely on both platforms.

---

## 🛡️ Important: SSDs vs. HDDs

Before you start, identify your drive type.

- **HDDs (Hard Disk Drives):** Responding well to "shredding" or overwriting methods.
- **SSDs (Solid State Drives):** These use **Wear Leveling**, which moves data around to extend the drive's life. Standard
  "shredding" tools can miss data fragments. For SSDs, the most secure method is a **"Secure Erase"** command via the
  manufacturer's utility or a BIOS tool.

---

## 🪟 On Windows

### 1. SDelete (Sysinternals)

Microsoft provides a command-line utility called **SDelete** as part of its Sysinternals suite. It is the gold standard for
secure deletion on Windows.

- **How to use:**
  1. Download SDelete from Microsoft.
  2. Open PowerShell or Command Prompt as Administrator.
  3. Run: `sdelete64.exe -p 3 -s C:\path\to\your\folder`
  - `-p 3` tells the tool to overwrite the data 3 times.

### 2. Cipher (Built-in)

Windows has a built-in tool to wipe "free space." This is useful if you have already deleted files and want to ensure they
cannot be recovered.

- **How to use:**
  1. Open Command Prompt.
  2. Run: `cipher /w:C:` (Replace `C:` with your drive letter).
     > **Note:** This only wipes space that isn't currently being used by existing files.

### 3. Third-Party Tools (GUI)

If you prefer a visual interface, **Eraser** is a popular open-source tool that adds a "Secure Erase" option to your
right-click context menu.

---

## 🐧 On Ubuntu (Linux)

### 1. `shred`

This is a standard utility built into most Linux distributions. It overwrites a file multiple times with random data.

- **How to use:**
  1. Open the Terminal.
  2. Run: `shred -u -n 3 filename`
  - `-u` deallocates and removes the file after overwriting.
  - `-n 3` specifies 3 overwrite passes.

### 2. `wipe`

`wipe` is a more advanced tool that was specifically designed to handle the complexities of modern file systems.

- **Installation:** `sudo apt install wipe`
- **How to use:** `wipe filename` or `wipe -r /path/to/directory`

### 3. `srm` (Secure Remove)

Part of the `secure-delete` package, `srm` is designed to be a secure drop-in replacement for the standard `rm` command.

- **Installation:** `sudo apt install secure-delete`
- **How to use:** `srm filename`

> **⚠️ A Note on Safety:** Secure deletion is permanent. Once you run these commands, there is no "undo" button and
> professional recovery services will likely be unable to help you. Always double-check your file paths before hitting Enter.
