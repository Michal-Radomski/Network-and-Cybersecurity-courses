At its core, **hashing** is the process of turning any amount of data—a word, a book, or an entire hard drive—into a
fixed-length string of characters. It acts like a digital fingerprint for data.

Think of it this way: if you bake a cake, you can't "un-bake" it to get the eggs and flour back. Hashing is that one-way
chemical reaction.

### What is Hashing?

A hashing algorithm (like SHA-256) takes an input and produces a unique "hash value." It has three critical properties:

1.  **Deterministic:** The same input always produces the exact same hash.
2.  **Fixed Length:** Whether the input is one letter or a million, the hash is always the same length.
3.  **Irreversible:** You cannot look at a hash and mathematically figure out the original data.

---

### Hashing vs. Encryption

While both involve scrambling data, their purposes are polar opposites. Encryption is a "two-way street" designed for
**confidentiality**, whereas hashing is a "one-way street" designed for **integrity**.

| Feature       | Hashing                                                          | Encryption (with a key)                             |
| :------------ | :--------------------------------------------------------------- | :-------------------------------------------------- |
| **Purpose**   | To verify data hasn't been tampered with (Integrity).            | To hide data from unauthorized eyes (Privacy).      |
| **Direction** | **One-way.** You can't get the original data back.               | **Two-way.** You can decrypt it back to plain text. |
| **Output**    | Always the same length for a specific algorithm.                 | Length usually grows with the size of the input.    |
| **Key Usage** | Does not require a key (though "salting" is used for passwords). | **Requires a key** to lock and unlock the data.     |

---

### The Key Difference: The "Secret Key"

In **Encryption**, the key is the most important part. If I encrypt a message to you, I use a key to scramble it. You must
use that same key (or a mathematically related one) to unscramble it. If you lose the key, the data is effectively gone
forever.

In **Hashing**, there is no "un-hashing." Instead, if you want to check if a file was corrupted during a download, you hash
the file you received. If your hash matches the one provided by the website, you know the file is identical.

### Common Use Cases

- **Hashing:** Storing passwords. Websites shouldn't know your actual password; they only store the hash. When you log in,
  they hash your input and see if it matches the hash they have on file.
- **Encryption:** Sending a credit card number over the internet or sending a private message on WhatsApp. You want that data
  to be readable again once it reaches its destination.
