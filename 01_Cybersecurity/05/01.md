Hardening an Ubuntu server involves reducing the "attack surface" by disabling unnecessary services, tightening access
controls, and automating security updates. Think of it as moving from a house with many open windows to a locked vault with a
single, guarded entrance.

Here is a roadmap for hardening Ubuntu:

---

## 1. Access Control & SSH Security

The most common entry point for attackers is SSH. Protecting this service is your first priority.

- **Disable Root Login:** Never allow direct login as the `root` user. Force the use of a standard user with `sudo`
  privileges.
- **Key-Based Authentication:** Disable password logins entirely in favor of SSH Keys. Passwords can be brute-forced;
  high-entropy keys cannot.
- **Change the Default Port:** While "security by obscurity" isn't a complete solution, moving SSH from port **22** to a
  non-standard port (e.g., **2274**) stops thousands of automated bot attacks.
- **Use Fail2Ban:** This service monitors logs for repeated failed login attempts and automatically bans the offending IP
  addresses at the firewall level.

---

## 2. Network Security (The Firewall)

Ubuntu comes with **UFW (Uncomplicated Firewall)**, which is simple but powerful.

- **Default Deny:** Set your firewall to deny all incoming traffic by default and only allow specific ports (e.g., SSH, HTTP,
  HTTPS).
- **Command sequence:**
  1. `sudo ufw default deny incoming`
  2. `sudo ufw default allow outgoing`
  3. `sudo ufw allow 22/tcp` (or your custom SSH port)
  4. `sudo ufw enable`

---

## 3. Software Updates & Patching

Vulnerabilities are discovered daily. Staying updated is the only way to mitigate "N-day" exploits.

- **Unattended Upgrades:** Enable the `unattended-upgrades` package. This ensures that security patches are downloaded and
  installed automatically without manual intervention.
- **Minimize Packages:** If you don't need a service (like an email server or a print server), uninstall it. Every installed
  package is a potential back door.

---

## 4. System & Filesystem Hardening

Locking down how the OS operates internally prevents an attacker from escalating their privileges if they do get in.

- **Secure Shared Memory:** Edit `/etc/fstab` to set `/run/shm` to `ro` (read-only) or `nosuid/noexec` to prevent malicious
  scripts from running in shared memory.
- **Password Policies:** Use the `libpam-cracklib` module to enforce strong password requirements (length, complexity, and
  age).
- **Kernel Hardening:** Use `sysctl` to prevent common networking attacks, such as IP spoofing and ICMP redirects.

---

## 5. Auditing and Monitoring

You cannot protect what you cannot see.

- **Lynis:** This is an excellent open-source security auditing tool. Running `sudo lynis audit system` will scan your Ubuntu
  server and give you a "Hardening Index" score along with specific suggestions for improvement.
- **Log Management:** Ensure logs are being kept in `/var/log/` and consider shipping them to a remote, secure log server so
  an attacker can't "wipe their tracks" by deleting local logs.

---

| Task                     | Tool/Command           |
| :----------------------- | :--------------------- |
| **Firewall**             | `ufw`                  |

---

> **A Note on Backups:** No amount of hardening is 100% effective. Always maintain an off-site, encrypted backup of your
> configuration files and data.
