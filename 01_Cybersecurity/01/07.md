In cybersecurity, access control is the process of regulating who can view or use resources in a computing environment. It is
typically divided into categories (the "how" and "where" it is applied) and types/models (the specific logic used to grant or
deny access). [1, 2]

## 1. Access Control Categories

Categories describe the method of implementation or the functional goal of the security measure. [3, 4]

## By Implementation Method

- Physical Access Control: Restricts access to physical locations like data centers, offices, or labs using tools like
  keycards, biometric scanners, and locks.
- Logical (Technical) Access Control: Restricts access to digital resources like networks, files, and applications through
  software-based mechanisms like passwords and encryption.
- Administrative (Managerial) Access Control: Defines the policies, procedures, and guidelines that dictate how security
  should be handled within an organization. [5, 6, 7, 8, 9, 10]

## By Functional Goal

- Preventive: Designed to stop a security incident from occurring (e.g., a firewall or a locked door).
- Detective: Designed to identify and record when an incident has occurred (e.g., motion sensors or audit logs).
- Deterrent: Meant to discourage potential attackers (e.g., "Authorized Personnel Only" signs or visible cameras).
- Corrective: Used to restore systems after a breach (e.g., backups or system patches). [11, 12, 13, 14]

---

## 2. Access Control Types (Models)

These are the logical frameworks used to determine how permissions are assigned to users. [9, 15]

| Type [2, 6, 7, 16, 17, 18, 19, 20, 21] | How it Works                                                                                                                    | Best For                                                                               |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| Discretionary (DAC)                    | The resource owner (e.g., the person who created a file) decides who has access.                                                | Flexibility; small teams.                                                              |
| Mandatory (MAC)                        | A central authority assigns security labels (e.g., "Top Secret") to users and data. Access is granted only if the labels match. | Military or government systems; high-security data.                                    |
| Role-Based (RBAC)                      | Access is tied to a job role (e.g., "HR Manager" or "IT Admin") rather than an individual.                                      | Scalability in large corporations.                                                     |
| Attribute-Based (ABAC)                 | Access is granted based on dynamic attributes like time of day, location, device type, or department.                           | Complex, context-aware security (e.g., "Doctors can only view records while on duty"). |
| Rule-Based                             | Access is granted based on specific technical rules set by admins, such as "No access after 5 PM".                              | Time-based or automated restriction needs.                                             |
