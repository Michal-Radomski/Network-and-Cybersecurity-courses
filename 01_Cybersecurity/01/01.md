These four terms overlap a lot, but they describe different “layers” or scopes of protection in the digital world. Here’s a
concise breakdown:

### Cybersecurity (biggest umbrella)

Cybersecurity is the broadest concept: it covers **all digital assets**—systems, networks, devices, applications, and
data—against cyber threats like malware, phishing, ransomware, and hacking.
[onlinedegrees.sandiego](https://onlinedegrees.sandiego.edu/cyber-security-information-security-network-security/) Think of
it as the overall strategy to protect any internet‑connected or computer‑based environment.
[anysecura](https://www.anysecura.com/knowledge/cybersecurity-vs-network-security.html)

### Network Security (inside the umbrella)

Network security is a **subset of cybersecurity**. It focuses on **protecting the network infrastructure** (routers,
switches, firewalls, VPNs) and the **data in transit** across that network.
[brightdefense](https://www.brightdefense.com/resources/cybersecurity-vs-network-security/) Its main job is to prevent
unauthorized access, denial‑of‑service attacks, and eavesdropping on communications inside or between networks.
[darktrace](https://www.darktrace.com/cyber-ai-glossary/network-security-vs-cybersecurity-vs-information-security)

### Internet Security (subset of cybersecurity)

Internet security is about **protecting systems and data that are exposed to the internet**, such as email, online banking,
and public‑facing servers.
[onlinedegrees.sandiego](https://onlinedegrees.sandiego.edu/cyber-security-information-security-network-security/) It
emphasizes defense against web‑based threats like phishing, malware‑laden downloads, and attacks on internet‑connected
services. [logmein](https://www.logmein.com/blog/network-security-vs-cyber-security-differences)

### Web Security (even more specific)

Web security is a **narrower subset**, focused on **web applications, websites, and APIs** (e.g., forms, logins, APIs
handling user data). [brightdefense](https://www.brightdefense.com/resources/cybersecurity-vs-network-security/) It deals
with threats like SQL injection, cross‑site scripting (XSS), broken authentication, and insecure APIs that target how
websites behave, not just the network they run on.
[thinkcloudly](https://thinkcloudly.com/blog/cyber-security-vs-network-security/)

### How they relate (hierarchy)

- **Cybersecurity** includes everything: network, internet, web apps, endpoints, and user behavior.
  [anysecura](https://www.anysecura.com/knowledge/cybersecurity-vs-network-security.html)
- **Network security** protects the transport layer and network devices.
  [geeksforgeeks](https://www.geeksforgeeks.org/difference-between-network-security-and-cyber-security/)
- **Internet security** adds rules for anything exposed to the public internet.
  [logmein](https://www.logmein.com/blog/network-security-vs-cyber-security-differences)
- **Web security** zooms in on the security of websites and web apps themselves.
  [darktrace](https://www.darktrace.com/cyber-ai-glossary/network-security-vs-cybersecurity-vs-information-security)

In short:

- **Cybersecurity** = “all‑around digital protection.”
- **Network security** = “protect the pipes (network).”
- **Internet security** = “protect internet‑facing services.”
- **Web security** = “protect websites and web apps.”
  [onlinedegrees.sandiego](https://onlinedegrees.sandiego.edu/cyber-security-information-security-network-security/)
