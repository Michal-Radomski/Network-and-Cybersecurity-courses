**GDPR Overview**  
The General Data Protection Regulation (GDPR) is a comprehensive EU law enacted in 2016 and effective from May 25, 2018,
governing the collection, processing, storage, and transfer of personal data for EU residents. It aims to protect individual
privacy rights, defining personal data broadly as any information relating to an identifiable person, including names, IDs,
or indirect identifiers like location. [cloudflare](https://www.cloudflare.com/learning/privacy/what-is-the-gdpr/)

Non-compliance can result in fines up to 4% of global annual turnover or €20 million, whichever is higher.
[fortinet](https://www.fortinet.com/resources/cyberglossary/gdpr)

## Programmer Obligations

Programmers in the EU (or handling EU data) must embed GDPR principles like data minimization, purpose limitation, and
security into code via "privacy by design." Key actions include conducting privacy impact assessments, implementing explicit
consent mechanisms (no pre-checked boxes), and ensuring data accuracy through validation.
[inspeerity](https://inspeerity.com/blog/how-to-ensure-gdpr-compliance-for-software-development-projects/)

| Principle    | Programmer Actions [codific](https://codific.com/gdpr-compliance-software-development/) |
| ------------ | --------------------------------------------------------------------------------------- |
| Consent      | Build opt-in interfaces; allow easy withdrawal.                                         |
| Minimization | Collect only necessary data; audit usage.                                               |
| Security     | Use encryption, access controls; log breaches.                                          |
| Rights       | Enable data access, portability, deletion (e.g., "right to be forgotten").              |
| Storage      | Set retention policies with auto-deletion.                                              |

Developers should also vet third-party APIs/services for compliance and provide user data export tools.
