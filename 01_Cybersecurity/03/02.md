Logical access control in cybersecurity refers to the rules, tools, and technologies that decide _who_ can access digital
resources (like systems, networks, files, or applications) and _what they can do_ once inside. It sits between the user and
the data, enforcing authentication and authorization so only properly identified and permitted users can reach sensitive
information.

### What logical access control does

Logical access control ensures that only authorized users can access computer‑based assets (e.g., databases, servers, cloud
apps) while keeping unauthorized users or attackers out. It supports the core security goals of confidentiality, integrity,
and availability (the CIA triad) by limiting who can see, change, or delete data and who can manage systems.
[pacificsecuritygroup.com](https://pacificsecuritygroup.com.au/blog/physical-access-control-vs-logical-access-control/)

### Core components

Three main steps underpin logical access control:

- **Identification**: a user provides a unique identifier (for example, username or ID).
  [pacificsecuritygroup.com](https://pacificsecuritygroup.com.au/blog/physical-access-control-vs-logical-access-control/)
- **Authentication**: the system verifies that identity using something you know (password), have (token, smart card), or are
  (biometrics or MFA).
  [cybersecurity-automation](https://www.cybersecurity-automation.com/what-is-logical-access-control-in-computer-security/)
- **Authorization**: once authenticated, the system checks permissions and lets the user perform only allowed actions (read,
  write, delete, admin) on specific resources.
  [sciencedirect](https://www.sciencedirect.com/topics/computer-science/logical-access-control)

### Common models and techniques

Organizations often use structured models such as:

- **Discretionary Access Control (DAC)**: owners of resources decide who can access them.
  [sciencedirect](https://www.sciencedirect.com/topics/computer-science/logical-access-control)
- **Mandatory Access Control (MAC)**: access is enforced by strict security labels (e.g., “Top Secret”).
  [sciencedirect](https://www.sciencedirect.com/topics/computer-science/logical-access-control)
- **Role‑Based Access Control (RBAC)**: permissions are assigned to roles, and users inherit those roles (for example,
  “accountant” or “admin”).
  [infosectrain](https://www.infosectrain.com/blog/isc2-cc-domain-3-3-2-understand-logical-access-controls)

Technically, logical access control is implemented via user accounts, groups, file‑system permissions, access‑control lists
(ACLs), and identity‑management systems integrated with operating systems, databases, and cloud platforms.
