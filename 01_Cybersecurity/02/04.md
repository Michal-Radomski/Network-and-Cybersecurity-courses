SIEM stands for **Security Information and Event Management**. It is a cybersecurity system that collects log and event data
from many sources, analyzes it, and helps security teams spot threats faster.
[microsoft](https://www.microsoft.com/en-us/security/business/security-101/what-is-siem)

## What it does

- Collects logs from servers, endpoints, firewalls, applications, and cloud services.
  [microsoft](https://www.microsoft.com/en-us/security/business/security-101/what-is-siem)
- Correlates events to find suspicious patterns, such as repeated failed logins or unusual file access.
  [cisco](https://www.cisco.com/site/us/en/learn/topics/security/what-is-siem.html)
- Sends alerts to security teams and can sometimes trigger automatic responses.
  [microsoft](https://www.microsoft.com/en-us/security/business/security-101/what-is-siem)
- Supports incident response, investigation, compliance, and forensic analysis by keeping historical records.
  [fortinet](https://www.fortinet.com/resources/cyberglossary/what-is-siem)

## Why it matters

SIEM gives organizations a central view of security activity, which makes it easier to detect attacks early and reduce alert
noise. It is especially useful in a Security Operations Center, where analysts need to monitor many systems at once.
[cynet](https://www.cynet.com/siem/)

## Simple example

If an account logs in from Warsaw, then minutes later from another country, and also downloads a large amount of data, a SIEM
can combine those events and flag it as suspicious.
[cisco](https://www.cisco.com/site/us/en/learn/topics/security/what-is-siem.html)
