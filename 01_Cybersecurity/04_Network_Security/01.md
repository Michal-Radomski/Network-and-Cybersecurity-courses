Quality of Service (QoS) is a set of technologies used in computer networking to manage and prioritize data traffic. Its
primary goal is to ensure that critical, time-sensitive applications—like video calls, online gaming, and Voice over IP
(VoIP)—receive the bandwidth and performance they need to function smoothly, even when the network is busy. [1, 2, 3]

## Core Mechanisms

QoS works by acting like a traffic officer for your data. It uses several methods to organize how information moves through a
network: [2, 4, 5]

- Classification and Marking: Data packets are "tagged" (often using headers like DSCP) so the network knows which type of
  traffic they belong to (e.g., video, email, or web browsing).
- Queuing: Routers create separate virtual "lines" for different types of traffic. High-priority data is placed in a "fast
  lane" and sent first, while less urgent data (like a background file download) waits its turn.
- Traffic Shaping & Policing: These techniques limit the rate of data flow. Shaping buffers and delays packets to smooth out
  traffic bursts, while policing drops or marks packets that exceed a set speed limit to prevent one user from hogging all
  the bandwidth. [1, 6, 7, 8, 9]

## Key Performance Metrics

QoS is measured and managed through four main technical indicators: [3, 10]

- Bandwidth: The maximum amount of data that can travel over a connection at once. QoS reserves specific amounts for
  important apps.
- Latency (Delay): The time it takes for a packet to travel from sender to receiver. High latency causes lag in video calls
  or games.
- Jitter: Variations in the arrival time of data packets. High jitter makes audio sound "choppy" or robotic.
- Packet Loss: When data packets fail to reach their destination due to network congestion. This leads to dropped calls or
  visual glitches in streaming. [1, 2, 11, 12, 13]

## Implementation Models

Depending on the network's needs, administrators typically use one of three models: [14, 15]

- Best-Effort: The default model where all data is treated equally. There are no performance guarantees.
- Integrated Services (IntServ): A rigid model that reserves exact network resources for a specific application before any
  data is sent. It uses the RSVP protocol.
- Differentiated Services (DiffServ): The most common modern approach. It sorts traffic into classes (like "Silver," "Gold,"
  or "Platinum") and treats each class differently without needing strict reservations for every single connection. [2, 8, 9,
  16, 17]
