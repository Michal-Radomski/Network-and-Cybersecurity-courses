To implement an IDS or IPS on Ubuntu, the two most popular open-source tools are Snort and Suricata. While Snort is a
long-standing industry standard, Suricata is often recommended for modern Ubuntu systems because it supports multi-threading
(utilizing all CPU cores) and generates rich metadata logs. [1, 2, 3]

## Option 1: Implementing Suricata (Recommended)

Suricata is a high-performance engine that functions as both an IDS and IPS. [4]

1. Installation It is best to use the official stable repository to get the latest version: [5, 6]

sudo add-apt-repository ppa:oisf/suricata-stable sudo apt update sudo apt install suricata -y

2. Basic Configuration Find your network interface name using ip a (e.g., eth0 or enp0s3). [7, 8, 9]

- Edit the configuration file: sudo nano /etc/suricata/suricata.yaml.
- Update the HOME_NET variable to your local network (e.g., [192.168.1.0/24]).
- Ensure the af-packet interface matches your actual interface name. [7, 8]

3. Update Rules and Start

- Download the latest threat signatures: sudo suricata-update.
- Start the service: sudo systemctl start suricata.
- Check the status: sudo systemctl status suricata. [8, 10]

4. Switching from IDS to IPS Mode By default, Suricata only monitors (IDS). To make it an IPS (block traffic), you must
   change its listen mode to nfqueue in /etc/default/suricata and restart the service. [11]

---

## Option 2: Implementing Snort

Snort is highly reliable and uses fewer resources in single-core environments. [2, 12]

1. Installation Install directly from the default Ubuntu repositories: [13]

sudo apt update sudo apt install snort -y

During installation, you will be prompted to enter your network range (e.g., 192.168.1.0/24). [14] 2. Configuration and
Testing

- Configuration files are located at /etc/snort/snort.conf.
- Test your configuration: sudo snort -T -c /etc/snort/snort.conf -i <your_interface>. [13, 15]

## Comparison for Ubuntu Users

| Feature [1, 2, 16] | Suricata                                   | Snort                                         |
| ------------------ | ------------------------------------------ | --------------------------------------------- |
| Performance        | Multi-threaded (faster on multi-core CPUs) | Single-threaded (lighter on low-end hardware) |
| Logging            | Detailed JSON logs (easy to read)          | Standardized alert logs                       |
| Rules              | Compatible with Snort rules                | Industry-standard rule set                    |
