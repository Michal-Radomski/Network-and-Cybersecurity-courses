**IPsec (Internet Protocol Security)** is a set of protocols used to **secure communication over IP networks** (like the
internet). It works by **encrypting and authenticating data packets**, so attackers can’t read or tamper with them.

### 🔐 What IPsec does

IPsec provides three main protections:

- **Encryption** → keeps data confidential
- **Authentication** → verifies who sent the data
- **Integrity** → ensures data hasn’t been altered

### ⚙️ How it works

IPsec operates at the **network layer (Layer 3)** of the OSI model, meaning it protects all traffic passing through it—no
matter what application is used.

It uses two main protocols:

- **AH (Authentication Header)**
  - Provides authentication and integrity
  - Does _not_ encrypt data

- **ESP (Encapsulating Security Payload)**
  - Provides encryption + authentication
  - Most commonly used in practice

### 🔄 Modes of operation

- **Transport mode**
  - Encrypts only the data portion of the packet
  - Used for direct device-to-device communication

- **Tunnel mode**
  - Encrypts the entire IP packet
  - Commonly used in VPNs

### 🌐 Where you see IPsec

IPsec is widely used in:

- **VPNs (Virtual Private Networks)**
- Secure site-to-site connections between offices
- Remote access to corporate networks

### 🧠 Simple example

When you connect to a company VPN using IPsec, your data is:

1. Encrypted before leaving your device
2. Sent securely across the internet
3. Decrypted only by the trusted server
