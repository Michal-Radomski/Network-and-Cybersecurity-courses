Address Resolution Protocol (ARP) is a fundamental networking protocol used to find the physical MAC address of a device when
only its IP address is known. [1, 2] In the OSI model, it acts as the bridge between Layer 3 (Network/IP) and Layer 2 (Data
Link/MAC). Without it, your computer might know who it wants to talk to (IP) but wouldn't know where to send the electronic
signal on the local wire (MAC). [3, 4, 5, 6]

## How ARP Works (The 4-Step Process)

When a device wants to send data to another device on the same local network:

1.  Check the Cache: The sender first looks at its own ARP Table (or cache) to see if it already knows the MAC address for
    that IP.
2.  Broadcast Request: If not found, the sender sends an ARP Request to every device on the local network (a "broadcast"). It
    essentially asks, "Who has IP 192.168.1.5? Please send me your MAC address".
3.  Unicast Reply: The device with that specific IP recognizes its own address and sends back an ARP Reply containing its MAC
    address directly to the requester (a "unicast").
4.  Update and Send: The requester saves this mapping in its ARP table for future use and finally sends the original data
    packet. [7, 8, 9, 10, 11, 12]

## Key Types of ARP

- Standard ARP: The basic request-response process for matching an IP to a MAC.
- Proxy ARP: A router answers an ARP request on behalf of a device on a different network, helping them communicate as if
  they were on the same segment.
- Gratuitous ARP: An unsolicited announcement where a device broadcasts its own IP/MAC mapping to detect IP conflicts or
  update other devices' caches.
- Reverse ARP (RARP): A legacy protocol used by devices to find their own IP address when they only know their MAC. It is now
  largely obsolete, replaced by
  [DHCP](https://www.checkpoint.com/cyber-hub/network-security/what-is-the-address-resolution-protocol-arp/). [1, 7, 11, 13,
  14, 15]

## Important Notes

- Local Scope Only: ARP only works within a single local network segment (subnet). It does not cross routers to the wider
  internet.
- IPv6 Equivalent: Modern IPv6 networks do not use ARP; they use the Neighbor Discovery Protocol (NDP) for the same purpose.
- Security Risk: ARP is inherently "trusting," making it vulnerable to ARP Spoofing (or poisoning), where an attacker sends
  fake ARP messages to intercept or redirect network traffic. [4, 16, 17, 18]
