Private IP addresses are reserved for use within local networks (like your home, office, or school) and are never routed on
the public internet. They allow multiple devices to share a single public IP address through a process called NAT (Network
Address Translation). [1, 2, 3, 4, 5] The Internet Assigned Numbers Authority (IANA) reserved three specific ranges for
private networks under RFC 1918: [6, 7]

## The 3 Private IP Ranges

| Class [8, 9, 10, 11, 12] | IP Range                      | Total Addresses | Typical Use Case                               |
| ------------------------ | ----------------------------- | --------------- | ---------------------------------------------- |
| Class A                  | 10.0.0.0 – 10.255.255.255     | 16,777,216      | Large enterprise networks and data centers.    |
| Class B                  | 172.16.0.0 – 172.31.255.255   | 1,048,576       | Medium-sized organizations and universities.   |
| Class C                  | 192.168.0.0 – 192.168.255.255 | 65,536          | Small offices and home networks (most common). |

## Important "Special" Private Range

- APIPA (Automatic Private IP Addressing): 169.254.0.0 – 169.254.255.255.
- If you see your device has an IP in this range, it usually means it failed to get an IP from a DHCP server (like your
  router) and assigned itself a temporary "link-local" address so it can still talk to other devices in the immediate
  vicinity. [13, 14, 15]

## Key Facts

- Non-Routable: You cannot reach a private IP address directly from the internet. A router must bridge the gap using a Public
  IP.
- Reusable: Because they are local, millions of homes can use 192.168.1.1 simultaneously without any conflict, as long as
  they aren't on the same physical network. [16, 17, 18, 19, 20]

A loopback IP address is a special, reserved IP address used by a computer to "talk to itself." When you send data to a
loopback address, it never leaves the device or travels across a network; instead, it is routed directly back to the device's
own network software. [1, 2, 3, 4] The most famous loopback address is 127.0.0.1, often referred to by the hostname
localhost. [5]

## Key Details

- The Reserved Range: While 127.0.0.1 is the most common, the entire 127.0.0.0 to 127.255.255.255 block is reserved for
  loopback purposes (though most systems only use the first one).
- IPv6 Equivalent: In the newer IPv6 standard, the loopback address is written as ::1.
- Internal Routing: Data sent to this address bypasses the physical network hardware (like your Wi-Fi or Ethernet card) and
  stays entirely within the operating system's TCP/IP stack. [6, 7, 8, 9, 10]

## Why Use a Loopback Address?

1.  Testing and Troubleshooting: It is the first step in network diagnostics. If you can "ping 127.0.0.1" and get a reply,
    you know your computer's internal network software (TCP/IP stack) is working correctly, even if your Wi-Fi is broken.
2.  Web Development: Developers use localhost to run websites or databases on their own machines before making them public.
    For example, typing http://localhost:8080 in a browser lets you view a site stored locally on your hard drive.
3.  App Communication: Many software programs use the loopback address to communicate with different parts of themselves
    (e.g., a "client" part of a program talking to a "server" part on the same PC).
4.  Security: It allows services to run "privately" on a machine so they are only accessible to the person sitting at that
    computer, rather than anyone on the internet. [11, 12, 13, 14, 15]

A popular saying among IT professionals is "There’s no place like 127.0.0.1," which is the tech version of "There's no place
like home." [16]
