The main difference is the number of attack sources used to overwhelm the target. While both aim to crash a service or
website, they differ significantly in their execution and impact. [1, 2, 3, 4]

## Key Differences at a Glance

| Feature [1, 2, 5, 6, 7, 8, 9, 10, 11] | DoS (Denial of Service)                   | DDoS (Distributed Denial of Service)         |
| ------------------------------------- | ----------------------------------------- | -------------------------------------------- |
| Origin                                | A single source (one computer or IP)      | Multiple sources (thousands of devices)      |
| Method                                | A script or tool run from one machine     | A coordinated botnet of "zombie" devices     |
| Traffic Volume                        | Lower; limited by one machine's bandwidth | Extremely high; can saturate entire networks |
| Detection                             | Easy to spot and block a single IP        | Hard to distinguish bots from real users     |
| Traceability                          | Easy to trace back to the attacker        | Very difficult to trace the actual origin    |

## Comparison Summary

- DoS Attacks are like one person standing in a doorway to block customers from entering a shop. Because there is only one
  source, a security admin can quickly identify and block that specific IP address to end the attack.
- DDoS Attacks are like thousands of people crowding the entrance at once from all directions. Since the traffic is coming
  from everywhere, blocking a few IPs won't stop the flood, making it much more dangerous and difficult to mitigate. [1, 7,
  8, 12, 13]
