IDS (Intrusion Detection System) and IPS (Intrusion Prevention System) are network security tools that monitor traffic for
malicious activity. While they use similar methods to find threats, their primary difference is that IDS detects and alerts,
while IPS detects and automatically blocks. [1, 2, 3, 4]

## Intrusion Detection System (IDS)

An IDS acts like a security alarm for your network. [5]

- Passive Monitoring: It typically sits "out-of-band," meaning it receives a copy of network traffic to analyze without being
  in the direct path.
- Alerting: When it finds something suspicious, it logs the event and notifies an administrator.
- Impact: Because it is not in the direct line of traffic, it does not slow down the network or accidentally block legitimate
  users. [2, 6, 7, 8]

## Intrusion Prevention System (IPS)

An IPS acts like a bouncer at a club. [6]

- Active Prevention: It sits "inline," meaning all network traffic must pass through it.
- Automated Action: It can immediately drop malicious packets, terminate connections, or reconfigure firewalls to stop an
  attack as it happens.
- Trade-off: Because it processes every packet in real-time, it can introduce slight latency (delay) and might accidentally
  block safe traffic if not tuned correctly. [2, 6, 8, 9, 10]

## Comparison of Key Features

| Feature [1, 2, 6, 7, 10] | IDS (Detection)           | IPS (Prevention)              |
| ------------------------ | ------------------------- | ----------------------------- |
| Response                 | Passive (Alerts only)     | Active (Blocks & Prevents)    |
| Placement                | Out-of-band (Listen-only) | Inline (In path of traffic)   |
| Performance              | Minimal impact on speed   | May introduce latency         |
| False Positives          | Only causes a false alarm | Can block legitimate services |

## How They Detect Threats

Both systems typically use two main methods to identify "bad" traffic: [5, 11]

- Signature-based: They compare traffic to a database of known attack patterns (like a virus signature).
- Anomaly-based: They establish a "baseline" of normal network behavior and flag anything that looks unusual (e.g., a massive
  data transfer at 3 AM). [12, 13, 14]

## Today, these functions are often combined into a single solution called an IDPS or integrated into [Next-Generation Firewalls (NGFW)](https://www.paloaltonetworks.com/cyberpedia/firewall-vs-ids-vs-ips). [3, 15]
