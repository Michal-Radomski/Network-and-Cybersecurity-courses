The OSI (Open Systems Interconnection) model is a conceptual framework that standardizes how computer systems communicate
over a network. Created by the International Organization for Standardization (ISO) in 1984, it divides the complex process
of data transmission into seven distinct layers. [1, 2, 3, 4] Each layer serves the one above it and is served by the one
below it, allowing different hardware and software vendors to build interoperable products. [1, 5]

## The 7 Layers of the OSI Model

| Layer [6, 7, 8, 9, 10] | Name         | Function                                                                         | Data Unit (PDU) |
| ---------------------- | ------------ | -------------------------------------------------------------------------------- | --------------- |
| 7                      | Application  | Direct interaction with software like web browsers and email clients.            | Data            |
| 6                      | Presentation | Translates, encrypts, and compresses data (ensures it's "readable").             | Data            |
| 5                      | Session      | Manages, maintains, and terminates connections between devices.                  | Data            |
| 4                      | Transport    | Handles end-to-end communication, error recovery, and flow control.              | Segments        |
| 3                      | Network      | Routes data between different networks using logical (IP) addressing.            | Packets         |
| 2                      | Data Link    | Transfers data between nodes on the same network using physical (MAC) addresses. | Frames          |
| 1                      | Physical     | Transmits raw bit streams over physical media (cables, radio waves).             | Bits            |

---

## Layer Breakdown

- 7. Application Layer: The closest layer to the end-user. It provides network services to applications like Google Chrome or
     Microsoft Outlook. Common protocols include HTTP, FTP, and DNS.
- 6. Presentation Layer: Acts as the "translator." It formats data for the application layer and handles tasks like SSL/TLS
     encryption and compression (e.g., JPEG, MPEG).
- 5. Session Layer: Responsible for opening and closing "sessions." It ensures that a connection stays open long enough to
     transfer data and closes it to save resources once finished.
- 4. Transport Layer: Takes data from the session layer and breaks it into smaller segments. It uses TCP (reliable,
     connection-oriented) or UDP (fast, connectionless) to ensure the message arrives correctly.
- 3. Network Layer: Breaks segments into packets and finds the best physical path for them to reach their destination
     (routing). This is where IP addresses live.
- 2. Data Link Layer: Breaks packets into frames for transmission between two physically connected nodes. It manages MAC
     addresses and handles error detection in the local link.
- 1. Physical Layer: The actual hardware level. It defines the electrical, optical, and mechanical specifications for
     transmitting raw bits (1s and 0s) over cables, fiber optics, or Wi-Fi. [1, 3, 6, 7, 11, 12, 13, 14, 15]

## Why It Matters

While the modern internet primarily uses the simpler TCP/IP model, the OSI model remains the gold standard for
troubleshooting. Engineers can isolate a problem by saying it is a "Layer 1 issue" (like a broken cable) or a "Layer 7 issue"
(an application bug), providing a universal language for network professionals. [1, 4, 16] A common mnemonic to remember the
layers (from top to bottom) is: All People Seem To Need Data Processing. [16, 17]

While the OSI model is a theoretical blueprint used for teaching and troubleshooting, the TCP/IP model is the actual protocol
suite that runs the internet today. [1, 2] The main difference is that the TCP/IP model is more streamlined, combining
several of the OSI model's seven layers into just four practical ones. [3, 4]

## Side-by-Side Comparison

| OSI Layer [2, 3, 5, 6] | TCP/IP Layer      | Function & Protocols                                                                                                      |
| ---------------------- | ----------------- | ------------------------------------------------------------------------------------------------------------------------- |
| 7. Application         | 4. Application    | Combined in TCP/IP to handle all user-facing services, data formatting, and connection management (HTTP, DNS, SMTP, FTP). |
| 6. Presentation        |                   |                                                                                                                           |
| 5. Session             |                   |                                                                                                                           |
| 4. Transport           | 3. Transport      | Functions almost identically in both models. Manages end-to-end data delivery via TCP or UDP.                             |
| 3. Network             | 2. Internet       | Handles logical addressing and routing. In TCP/IP, this is where IP addresses operate.                                    |
| 2. Data Link           | 1. Network Access | Combined in TCP/IP to manage physical transmission and local network hardware (Ethernet, Wi-Fi, MAC addresses).           |
| 1. Physical            |                   |                                                                                                                           |

Note: Some modern technical guides use a 5-layer version of the TCP/IP model that keeps the Physical and Data Link layers
separate, similar to OSI. [7, 8]

## Key Differences

- Practical vs. Theoretical: TCP/IP was built around established protocols for real-world use, whereas the OSI model was
  developed as a comprehensive standard to follow later.
- Layer Granularity: The OSI model is much more detailed at the top (Layers 5–7) and bottom (Layers 1–2). This makes it
  superior for troubleshooting specific failures, like identifying if a problem is a bad cable (Layer 1) or a data-formatting
  error (Layer 6).
- Internet Dominance: Since TCP/IP is the foundation of the modern internet, it is the more common model used by
  administrators for daily operations and network configuration. [1, 3, 9, 10, 11, 12]
