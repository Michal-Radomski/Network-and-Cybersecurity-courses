IPv4 and IPv6 are versions of the Internet Protocol that assign unique addresses to devices on a network, but IPv6 was
developed to overcome IPv4's limitations like address exhaustion. The key differences lie in address size, format, and
built-in features. [kinsta](https://kinsta.com/blog/ipv4-vs-ipv6/)

## Address Format

IPv4 uses 32-bit addresses written as four decimal numbers separated by dots (e.g., 192.168.0.1), supporting about 4.3
billion unique addresses. IPv6 employs 128-bit addresses in hexadecimal format with colons (e.g.,
2001:0db8:85a3:0000:0000:8a2e:0370:7334), enabling roughly 3.4 × 10^38 addresses.

## Key Features

IPv6 has mandatory IPsec for security, eliminating the need for Network Address Translation (NAT) used in IPv4, and supports
multicasting natively. It also simplifies packet headers (fixed 40 bytes vs. IPv4's variable 20-60 bytes) and handles
fragmentation only at the sender. [kinsta](https://kinsta.com/blog/ipv4-vs-ipv6/)

## Performance Impact

IPv6 can offer better routing efficiency and end-to-end connectivity due to its design, though real-world speed depends on
network support rather than the protocol alone. [siteground](https://www.siteground.com/kb/ipv4-vs-ipv6/)

| Feature        | IPv4                   | IPv6 [ibm](https://www.ibm.com/docs/en/i/7.4.0?topic=6-comparison-ipv4-ipv6) |
| -------------- | ---------------------- | ---------------------------------------------------------------------------- |
| Address Length | 32 bits                | 128 bits                                                                     |
| Address Style  | Dotted decimal         | Hexadecimal with colons                                                      |
| Security       | Optional IPsec         | Mandatory IPsec                                                              |
| Header Size    | Variable (20-60 bytes) | Fixed (40 bytes)                                                             |
| NAT Required   | Yes, often             | No                                                                           |

---

The main difference between IPv4 and IPv6 is the **address space**. We essentially ran out of IPv4 addresses because the
internet grew much faster than anyone in the 1980s anticipated. IPv6 was designed to solve that "exhaustion" problem while
making the internet more efficient.

### Key Technical Differences

#### 1. Capacity and Scalability

IPv4 uses a 32-bit addressing scheme, which provides roughly **4.3 billion** unique addresses. While that sounds like a lot,
every smartphone, laptop, smart fridge, and server needs an IP. IPv6 uses 128-bit addresses, providing a number so large
($3.4 \times 10^{38}$) that we could theoretically assign an IP address to every atom on the surface of the Earth and still
have plenty left over.

#### 2. Header Complexity

IPv4 headers are variable in length and include a "Checksum," which means every router the packet passes through has to
re-calculate that math, slowing things down slightly. IPv6 uses a **fixed header length**. It stripped out the non-essential
fields, allowing routers to process packets much faster.

#### 3. Network Address Translation (NAT)

Because IPv4 addresses are scarce, we use **NAT**. Your home router has one public IP, and it "hides" all your devices behind
it using private IPs (like `192.168.x.x`). In IPv6, NAT is unnecessary. Every single device can have its own unique,
public-facing IP address, restoring true "end-to-end" connectivity.

#### 4. Stateless Address Autoconfiguration (SLAAC)

In IPv4, you usually need a DHCP server to tell a device what its IP is. IPv6 devices can "self-assign" an address by looking
at the network prefix and using their own MAC address to complete the ID. It’s essentially "plug and play" for the internet
age.

---

### Is one "faster" than the other?

Technically, IPv6 can be slightly faster because headers are easier for routers to process and it avoids the overhead of NAT.
However, in the real world, your speed depends more on your ISP and physical hardware. Most modern systems use **Dual
Stack**, meaning they run both IPv4 and IPv6 simultaneously to ensure they can talk to any site on the web.
