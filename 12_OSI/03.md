Moving down the OSI model from the software layers, we hit the heart of network data movement: **Layer 4 (Transport)** and
**Layer 3 (Network)**. These two layers are responsible for figuring out how data gets from Point A to Point B across a
network.

---

## Layer 4: The Transport Layer

The **Transport Layer** is responsible for the end-to-end delivery of data between two devices. Think of it as a quality
control manager and a shipping coordinator rolled into one. It takes the large chunks of data from the upper layers and
breaks them down into smaller pieces called **segments**.

- **Primary Function:** Ensuring reliable, efficient, and error-free data transfer between hosts.
- **What it does:** \* **Segmentation and Reassembly:** It breaks big files into smaller segments for transmission and puts
  them back together in the correct order at the destination.
- **Flow Control:** It manages the speed of data transmission. If a powerful server sends data faster than a smartphone can
  process it, the Transport Layer tells the server to slow down.
- **Error Control:** It checks to see if segments are corrupted or lost during transit. If a segment is missing, it requests
  a retransmission.
- **Port Addressing:** It uses port numbers (like Port 80 for HTTP or Port 443 for HTTPS) to make sure data gets to the
  correct application on your device.

- **Common Protocols:** \* **TCP (Transmission Control Protocol):** Connection-oriented and highly reliable (used for web
  browsing, emails).
- **UDP (User Datagram Protocol):** Connectionless and fast, but doesn't guarantee delivery (used for live streaming,
  gaming).

---

## Layer 3: The Network Layer

While Layer 4 manages the connection between two applications, the **Network Layer** handles the actual routing of data
across different networks. It takes the segments from Layer 4 and packages them into **packets**. This is where **IP
addresses** live.

- **Primary Function:** Routing data packets from the source host to the destination host across multiple networks.
- **What it does:**
- **Logical Addressing:** It assigns source and destination IP addresses to every packet so routers know where the data came
  from and where it is going.
- **Routing:** Routers operate at this layer. They examine a packet's destination IP address and use complex algorithms to
  determine the best physical path for the data to take across the internet.
- **Packetization:** Enclosing the transport layer segments into network layer packets.

- **Common Protocols & Devices:** \* **IPv4 and IPv6** (Internet Protocol)
- **ICMP** (Used for diagnostic tools like `ping`)
- **Routers** (The physical hardware operating at this layer)
