## The OSI Model

The **OSI (Open Systems Interconnections) Model** is a conceptual framework created by the International Organization for
Standardization (ISO) in 1984. It breaks down how data moves across a network into **7 distinct layers**.

It is a theoretical model used primarily for teaching and troubleshooting network architectures.

Here is how the 7 layers work, from the user down to the physical hardware:

- **7. Application Layer:** Where human-computer interaction happens. This is the layer your software (like Google Chrome or
  Spotify) interacts with using protocols like HTTP, FTP, and SMTP.
- **6. Presentation Layer:** Ensures data is in a usable format. This layer handles **encryption, decryption, compression,
  and translation** of data (e.g., converting ASCII data into a format a website understands).
- **5. Session Layer:** Manages connections. It opens, maintains, and terminates communication sessions between two devices.
- **4. Transport Layer:** Responsible for end-to-end communication. It breaks data into smaller pieces (segments) and handles
  flow control and error checking. Major protocols here are **TCP** (reliable) and **UDP** (fast).
- **3. Network Layer:** Handles routing. It determines the best physical path for the data to take. It works with **IP
  addresses** and routes data packets across different networks.
- **2. Data Link Layer:** Establishes a link between two physically connected nodes. It corrects physical layer errors and
  frames data. This layer works with **MAC addresses** and network switches.
- **1. Physical Layer:** The actual hardware. This includes the physical cables (fiber optic, Ethernet), wireless signals
  (Wi-Fi), and the raw electrical binary signals ($1$s and $0$s) moving through them.

## The TCP/IP Model

The **TCP/IP Model** (Transmission Control Protocol/Internet Protocol) was developed by the US Department of Defense (DARPA)
in the 1970s.

Unlike the OSI model, TCP/IP is a **practical, real-world model**. The modern internet is entirely built based on the TCP/IP
protocol suite. It condenses the networking process into **4 layers**:

- **4. Application Layer:** Combines OSI Layers 5, 6, and 7. It gives applications access to network services (HTTP, DNS,
  SSH, FTP).
- **3. Transport Layer:** Matches OSI Layer 4. It manages host-to-host communication using TCP or UDP.
- **2. Internet Layer:** Matches OSI Layer 3. It handles the packaging, addressing, and routing of data packets using IP
  (IPv4 or IPv6).
- **1. Network Access Layer:** Combines OSI Layers 1 and 2. It defines how data is physically sent through the network
  hardware, dealing with MAC addresses, Ethernet cables, and hardware drivers.

## Key Differences: OSI vs. TCP/IP

While both models describe how data travels over a network, they approach the concept differently:

| Feature               | OSI Model                                                             | TCP/IP Model                                                                         |
| --------------------- | --------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| **Development**       | Developed by ISO _after_ protocols were invented (Theoretical guide). | Developed by DARPA _before_ the internet was fully built (Practical implementation). |
| **Structure**         | **7 layers** (More detailed and rigid).                               | **4 layers** (Simpler and more flexible).                                            |
| **Application Layer** | Separates application, presentation, and session into 3 layers.       | Combines them into 1 single Application layer.                                       |
| **Hardware Layer**    | Separates physical and data link layers.                              | Combines them into 1 Network Access layer.                                           |
| **Usage**             | Used as a reference tool for education and troubleshooting.           | Used as the actual functional standard for the internet.                             |

> **Summary Analogy:** Think of the **OSI Model** like a highly detailed blueprint of a house, explaining exactly what every
> single wall and beam does theoretically. The **TCP/IP Model** is the actual modular house that was built—it combines a few
> rooms to make the design practical, efficient, and livable.
