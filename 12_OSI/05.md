The **PPP (Point-to-Point Protocol)** is a fundamental data link layer protocol (Layer 2) used to establish a direct
connection between two networking nodes. It is most famous for its use in connecting home computers to an Internet Service
Provider (ISP) via dial-up or DSL modems.

Because it operates at Layer 2, it encapsulates network layer packets (like IP packets) so they can be transmitted over a
physical synchronous or asynchronous link.

---

## Key Features of PPP

Unlike simpler protocols, PPP was designed to be highly robust and versatile. It provides three main features:

- **Framing:** It clearly defines where a data frame begins and ends, handling error detection in the process.
- **Link Control Protocol (LCP):** This is used to automatically establish, configure, test, maintain, and terminate the
  point-to-point connection.
- **Network Control Protocols (NCPs):** PPP can carry traffic from multiple network layer protocols simultaneously (such as
  IPv4 and IPv6). A specific NCP is used to configure each network layer protocol.

---

## How a PPP Session Works

When two devices connect using PPP, the protocol goes through a specific lifecycle managed by the LCP:

1. **Link Establishment:** LCP opens a connection and negotiates link parameters (such as the maximum frame size).
2. **Authentication (Optional):** Before allowing data transfer, the link can demand identity verification. It usually uses
   one of two protocols:

- **PAP (Password Authentication Protocol):** Simple, but sends passwords in plain text (unsecure).
- **CHAP (Challenge Handshake Authentication Protocol):** Much more secure, using a three-way handshake and encryption to
  verify identity.

3. **Network-Layer Configuration:** The specific NCP configures the network protocols. For example, the IP Control Protocol
   (IPCP) might dynamically assign an IP address to the connecting device.
4. **Data Transfer:** Data frames are securely routed back and forth.
5. **Link Termination:** LCP tears down the link when the session is closed.

---

## PPP Frame Structure

A standard PPP frame packages data into a predictable format to ensure both sides understand the transmission:

- **Flag (1 byte):** A specific binary pattern (`01111110`) that indicates the start or end of a frame.
- **Address (1 byte):** Always set to `11111111` (broadcast address), because PPP is point-to-point and doesn't require
  individual station addresses.
- **Control (1 byte):** A fixed byte (`00000011`) that calls for unnumbered user data.
- **Protocol (2 bytes):** Identifies what kind of data is inside the information field (e.g., an IPv4 packet, an IPv6 packet,
  or LCP control data).
- **Information (Variable):** The actual payload being transmitted (e.g., the IP packet).
- **FCS (2 or 4 bytes):** The Frame Check Sequence, used for error detection.

---

## Modern Evolution: PPPoE

While pure PPP is rarely used on modern broadband connections, its legacy lives on in **PPPoE (Point-to-Point Protocol over
Ethernet)**. ISPs frequently use PPPoE to connect consumers to their networks via DSL or fiber, allowing them to use PPP's
excellent authentication, encryption, and compression features over a standard Ethernet network.
