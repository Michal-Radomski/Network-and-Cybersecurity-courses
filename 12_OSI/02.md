The top three layers of the OSI (Open Systems Interconnected) model—Layers 7, 6, and 5—are collectively known as the **Upper
Layers** or **Software Layers**. Unlike the lower layers that handle cables and routing, these three layers focus entirely on
application logic, data presentation, and user interaction.

Here is a breakdown of what each layer does, moving from the top down.

---

## Layer 7: The Application Layer

The **Application Layer** is the closest to the end-user. It doesn't refer to the applications themselves (like Chrome or
Discord), but rather the **protocols** that those applications use to communicate over a network.

- **Primary Function:** It acts as the window for applications to access network services.
- **What it does:** It identifies communication partners, determines resource availability, and synchronizes communication.
- **Common Protocols:** \* **HTTP/HTTPS** (Web browsing)
- **SMTP/IMAP** (Email)
- **FTP** (File transfers)
- **DNS** (Domain name resolution)

---

## Layer 6: The Presentation Layer

The **Presentation Layer** acts as the network's translator. It ensures that the data sent from the Application Layer of one
system can be read by the Application Layer of another system, even if they use different internal data formats.

- **Primary Function:** It handles data formatting, translation, encryption, and compression.
- **What it does:** \* **Translation:** Converts data between different formats (e.g., ASCII to EBCDIC).
- **Encryption/Decryption:** Handles security protocols like SSL/TLS to protect data in transit.
- **Compression:** Reduces the size of the data to speed up transmission.

- **Common Standards/Formats:** JPEG, MP3, GIF, JSON, XML, and SSL/TLS.

---

## Layer 5: The Session Layer

The **Session Layer** acts as the event coordinator. When two computers need to talk, this layer is responsible for opening a
"session" (a temporary connection), keeping it stable while data is being exchanged, and closing it when the communication is
over.

- **Primary Function:** Managing, establishing, maintaining, and terminating sessions between applications.
- **What it does:** \* **Authentication and Authorization:** Ensuring the entities connecting are allowed to do so.
- **Checkpointing/Syncing:** It places "checkpoints" in a data stream. If a 100MB file download drops at 50MB, the session
  layer allows it to resume from the last checkpoint instead of starting over.

- **Common Protocols/APIs:** NetBIOS, RPC (Remote Procedure Call), and SOCKS.
