In simple terms, **operator overloading** is a way to define how built-in operators (like `+`, `-`, `*`, or `==`) behave when
used with custom objects.

Instead of getting a "Type Error," you can teach your classes how to interact using standard mathematical or logical symbols.

---

## 1. Operator Overloading in Python

Python handles operator overloading through **Special Methods** (also known as "Dunder" methods, because they start and end
with double underscores).

When you use an operator like `+`, Python actually calls a specific method behind the scenes. For example, `a + b` is
essentially shorthand for `a.__add__(b)`.

### Common Python Dunder Methods

| Operator | Method                 | Purpose        |
| :------- | :--------------------- | :------------- |
| `+`      | `__add__(self, other)` | Addition       |
| `-`      | `__sub__(self, other)` | Subtraction    |
| `*`      | `__mul__(self, other)` | Multiplication |
| `==`     | `__eq__(self, other)`  | Equality       |
| `<`      | `__lt__(self, other)`  | Less than      |

### Example: Vector Addition

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    # Overloading the + operator
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

v1 = Vector(2, 4)
v2 = Vector(1, 3)

# This uses the __add__ method
print(v1 + v2)  # Output: Vector(3, 7)
```

## 2. The Equivalent in JS / TS

Here is the short answer: **JavaScript (and TypeScript) does not support true operator overloading.**

Unlike Python, C++, or C#, you cannot redefine how the `+` or `==` operators work for your custom classes. If you try to
"add" two objects in JS, the engine will usually convert them to strings or `NaN`.

### The "Workarounds" in JS/TS

While you can't overload operators, developers use these three common patterns to achieve similar results:

#### A. Method Chaining (The Standard Way)

Instead of `v1 + v2`, you define a method. This is the most common approach in libraries like Three.js or RxJS.

```typescript
class Vector {
  constructor(
    public x: number,
    public y: number
  ) {}

  add(other: Vector): Vector {
    return new Vector(self.x + other.x, self.y + other.y);
  }
}

const v3 = v1.add(v2);
```

#### B. The `valueOf` Hack (Limited)

JavaScript objects have a `valueOf()` method. If you return a primitive (like a number), you can use mathematical operators.
**Warning:** This is usually considered "clever but dangerous" code.

```javascript
class Price {
    constructor(public amount: number) {}
    valueOf() { return this.amount; }
}

const p1 = new Price(10);
const p2 = new Price(20);
console.log(p1 + p2); // Output: 30 (But it returns a number, not a Price object!)
```

#### C. Symbol.toPrimitive

Similar to `valueOf`, this allows you to define how an object is converted to a primitive. However, like `valueOf`, it
doesn't allow you to return a new instance of your class from the operation; it just helps the object "decay" into a number
or string.

---

## Summary Comparison

| Feature               | Python                         | JS / TS                                    |
| :-------------------- | :----------------------------- | :----------------------------------------- |
| **Native Support**    | **Yes** (via Dunder methods)   | **No**                                     |
| **Custom Arithmetic** | Highly flexible and readable   | Requires explicit methods (e.g., `.add()`) |
| **Equality Checks**   | Overload `__eq__`              | Must use methods like `.equals()`          |
| **Type Safety**       | Dynamic (check types manually) | Static (via TS interfaces/types)           |

> **A note on TypeScript:** Even though TypeScript adds types to JavaScript, it still compiles to JavaScript. Since the
> underlying engine doesn't support operator overloading, TS can't "fake" it without significantly changing the language
> syntax.
