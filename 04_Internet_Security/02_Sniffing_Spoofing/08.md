Writing a raw socket sniffer is a classic way to understand how the Linux kernel passes data to applications.

**Important Note:** To run this script, you must have **Root/Administrative privileges** because raw sockets allow you to
bypass the standard security layers of the OS. This script is designed for **Linux**; Windows handles raw sockets differently
and often requires third-party drivers like Npcap.

### The Python Raw Sniffer

This script creates a socket that listens for **every single IP packet** entering the machine and prints out the raw "Hex"
(the bytes) of the header.

```python
import socket
import struct

def start_sniffer():
    # 1. Create a raw socket
    # AF_PACKET: Low-level packet interface (Linux specific)
    # SOCK_RAW: We want the raw data, not processed TCP/UDP
    # ntohs(0x0003): ETH_P_ALL - capture all Ethernet protocols
    try:
        conn = socket.socket(socket.AF_PACKET, socket.SOCK_RAW, socket.ntohs(0x0003))
    except PermissionError:
        print("Error: You must run this script as root (sudo)!")
        return

    print("Sniffer started... Press Ctrl+C to stop.")

    while True:
        # 2. Receive the raw data
        raw_data, addr = conn.recvfrom(65536)

        # 3. Unpack the Ethernet Header (First 14 bytes)
        # 6s = Destination MAC (6 bytes)
        # 6s = Source MAC (6 bytes)
        # H  = Ethernet Type (2 bytes)
        dest_mac, src_mac, eth_proto = struct.unpack('! 6s 6s H', raw_data[:14])

        print(f"\n[+] New Frame Captured!")
        print(f"    Target MAC: {dest_mac.hex(':')} | Source MAC: {src_mac.hex(':')} | Protocol: {eth_proto}")

        # 4. Show the first 20 bytes of the IP Header (if it's an IP packet)
        if eth_proto == 8: # 8 is the code for IPv4
            ip_header = raw_data[14:34]
            print(f"    Raw IP Header Hex: {ip_header.hex(' ')}")

if __name__ == "__main__":
    start_sniffer()
```

---

### How this script works (The "Deep Dive")

1.  **`AF_PACKET`**: This is a special Linux-only domain. Unlike `AF_INET` (which works at the IP layer), `AF_PACKET` gives
    you access to the **Data Link Layer (Layer 2)**. This means you see the MAC addresses and the Ethernet frames.
2.  **`struct.unpack`**: Raw packets are just a long string of bytes ($01011100...$). The `struct` library acts as a "map."
    It tells Python: _"Take the first 6 bytes and treat them as the Destination MAC, then the next 6 as the Source."_ 3.
    **`ntohs(0x0003)`**: This is a bit of "Network Magic." It stands for **Network to Host Short**. It ensures the byte order
    (Big-Endian vs Little-Endian) is correct so the CPU reads the protocol numbers properly.

---

### Why this is different from your previous script

In your previous script, you used `SOCK_DGRAM` (UDP).

- When you sent a message, the OS added the Ethernet Header and the IP Header for you.
- When you received a message, the OS **stripped off** those headers and just handed you the "Hello" string.

In **this** script, nothing is stripped. You are seeing the "packaging" (the envelopes) as well as the letter inside.

### How to run it

1. Save it as `sniffer.py`.
2. Open your Linux terminal.
3. Run with sudo: `sudo python3 sniffer.py`.
4. While it's running, open a browser and go to any website. You will see your terminal flood with MAC addresses and Hex
   headers.

**Safety Warning:** Be careful when using raw sockets on public networks. Sniffing traffic that does not belong to you is
illegal in many jurisdictions. Since you are learning, always test on your own private "Lab" or home network!
