To write these scripts, we use Python’s built-in `socket` library. This is the standard way to handle network communication.

We will use **UDP (User Datagram Protocol)** for this example because it is the simplest "message-based" way to send a
packet. In UDP, we don't need a complex "handshake"—we just fire the data at an IP and Port.

---

### 1. The Receiver (The Server)

The receiver must "bind" to a specific port and wait (listen) for data to arrive.

```python
import socket

# 1. Define the IP and Port to listen on
# '127.0.0.1' is "localhost" (your own computer)
# Port 5005 is a random "unassigned" port
UDP_IP = "127.0.0.1"
UDP_PORT = 5005

# 2. Create the socket (AF_INET = IPv4, SOCK_DGRAM = UDP)
sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# 3. Bind the socket to our IP and Port
sock.bind((UDP_IP, UDP_PORT))

print(f"Waiting for packets on {UDP_IP}:{UDP_PORT}...")

while True:
    # 4. Receive data (buffer size is 1024 bytes)
    data, addr = sock.recvfrom(1024)
    print(f"Received message: {data.decode()} from {addr}")
```

---

### 2. The Sender (The Client)

The sender just needs to know where the receiver is located.

```python
import socket

# 1. Define the destination (must match the receiver)
UDP_IP = "127.0.0.1"
UDP_PORT = 5005
MESSAGE = b"Hello! This is a network packet."

print(f"Target IP: {UDP_IP}")
print(f"Target Port: {UDP_PORT}")

# 2. Create the socket
sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# 3. Send the packet
sock.sendto(MESSAGE, (UDP_IP, UDP_PORT))

print("Packet sent!")
```

---

### How to test this

1.  **Open two terminal windows** (or two command prompts).
2.  In the first window, run the **Receiver** script: `python receiver.py`. (It will sit there waiting).
3.  In the second window, run the **Sender** script: `python sender.py`.
4.  Switch back to the first window—you will see the "Hello!" message appear!

---

### Key Concepts for the Learner

- **`AF_INET`**: This tells Python we are using **IPv4** addresses (like 127.0.0.1).
- **`SOCK_DGRAM`**: This tells Python to use **UDP**. If you wanted to use **TCP** (the "phone call" version), you would use
  `SOCK_STREAM`.
- **`b"Message"`**: The `b` stands for **bytes**. Networks don't send "text"; they send raw numbers (bytes). We have to
  convert our text to bytes before sending and `.decode()` them when they arrive.
- **Loopback Address (127.0.0.1)**: This is a special IP that means "this very computer." It's perfect for testing scripts
  without needing a second physical machine.
