Exactly! If the ports don't match, the operating system looks at the packet, sees no "door" (socket) open for that specific
number, and simply drops it. It's like sending mail to Apartment 505 when the person actually lives in 600.

In **Node.js**, the equivalent of Python's `b"Message"` (bytes) is the **`Buffer`** object.

### The JavaScript/Node.js Equivalent

In modern JavaScript (especially within Node.js), you don't usually need a prefix like `b`. Instead, you convert a string
into a buffer or use a typed array.

| Language    | Code                   | Type     |
| :---------- | :--------------------- | :------- |
| **Python**  | `b"Hello"`             | `bytes`  |
| **Node.js** | `Buffer.from("Hello")` | `Buffer` |

---

### Node.js Sender & Receiver (UDP)

Node.js uses the built-in `dgram` module for UDP sockets. It handles the conversion of strings to buffers automatically in
many cases, but using `Buffer` is the best practice for raw network data.

#### The Receiver (`receiver.js`)

```javascript
const dgram = require("dgram");
const server = dgram.createSocket("udp4");

server.on("message", (msg, rinfo) => {
  // msg is a Buffer, we use .toString() to see the text
  console.log(`Received: ${msg.toString()} from ${rinfo.address}:${rinfo.port}`);
});

server.on("listening", () => {
  const address = server.address();
  console.log(`Listening on ${address.address}:${address.port}`);
});

server.bind(5005, "127.0.0.1");
```

#### The Sender (`sender.js`)

```javascript
const dgram = require("dgram");
const client = dgram.createSocket("udp4");

// Equivalent to b"Message"
const message = Buffer.from("Hello from Node.js!");

client.send(message, 5005, "127.0.0.1", (err) => {
  console.log("Packet sent!");
  client.close();
});
```

### Why "Bytes" Matter

Whether you are using Python or Node.js, the network hardware (your Wi-Fi card or Ethernet port) doesn't understand what
"letters" or "emojis" are. It only understands **binary** (ones and zeros).

- **Encoding:** Converting "Hello" $\rightarrow$ `[72, 101, 108, 108, 111]` (The Bytes).
- **Decoding:** Converting `[72, 101, 108, 108, 111]` $\rightarrow$ "Hello".

In Node.js, if you try to send a raw JSON object or a Number without putting it into a `Buffer` first, the `socket.send()`
method will usually throw an error because it doesn't know how to turn a complex object into a stream of electrical pulses.

### A Quick Tip for TS (TypeScript)

If you are using TypeScript, you'll notice that the `msg` parameter in the receiver is automatically typed as a `Buffer`.
This is very helpful because it reminds you that you can't treat it like a string until you call `.toString()`.
