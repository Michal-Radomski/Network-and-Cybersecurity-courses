Both **Promiscuous Mode** and **Monitor Mode** allow you to "see more than you should," but they operate at different layers
of the connection. Think of it as the difference between listening to everyone in a room (Promiscuous) and listening to the
radio waves in the air around the building (Monitor).

---

### 1. Promiscuous Mode (Wired & Connected Wi-Fi)

Promiscuous mode works when your network card is **associated** (connected) to a network.

- **What it does:** It tells the card to stop ignoring packets that aren't addressed to its specific MAC address.
- **The Limit:** You only see traffic that physically reaches your port. On a modern switch, this is usually just your own
  traffic unless you use "Spoofing" or "Port Mirroring." On Wi-Fi, you only see traffic for the specific network you are
  logged into.
- **Analogy:** You are inside a house, and you decide to listen to every conversation happening in every room.

### 2. Monitor Mode (Wi-Fi Only)

Monitor mode works **without connecting** to any specific access point.

- **What it does:** It allows the Wi-Fi card to "hover" on a specific radio frequency (channel). It captures every frame in
  the air, including management frames (like beacons and "handshakes") from routers you aren't even connected to.
- **The Detail:** You see the "invisible" traffic, like the SSID names of nearby routers and the MAC addresses of devices
  searching for Wi-Fi.
- **Analogy:** You are standing outside on the sidewalk with a high-powered microphone, picking up every sound from every
  house on the block.

---

### Comparison at a Glance

| Feature        | Promiscuous Mode                | Monitor Mode                           |
| :------------- | :------------------------------ | :------------------------------------- |
| **Connection** | Must be connected to a network. | Must NOT be connected.                 |
| **Medium**     | Wired (Ethernet) or Wi-Fi.      | Wi-Fi only.                            |
| **Data Type**  | Actual data packets (IP, TCP).  | Data packets + Management frames.      |
| **Usage**      | Standard sniffing (Wireshark).  | Wi-Fi auditing/cracking (Aircrack-ng). |

---

### 3. How to Switch Modes on Linux

#### Switching to Promiscuous Mode

This is done using the `ip` or `ifconfig` commands. Replace `eth0` with your interface name (find it using `ip link`).

**To Enable:**

```bash
sudo ip link set eth0 promisc on
```

**To Disable:**

```bash
sudo ip link set eth0 promisc off
```

---

#### Switching to Monitor Mode

This is more complex because many Wi-Fi drivers try to block this for security. You generally use the `iw` tool or the
`airmon-ng` tool (part of the Aircrack-ng suite).

**Using `iw` (The standard way):**

1. Stop the network manager (to prevent it from interfering):
   ```bash
   sudo systemctl stop NetworkManager
   ```
2. Set the mode:
   ```bash
   sudo ip link set wlan0 down
   sudo iw wlan0 set monitor control
   sudo ip link set wlan0 up
   ```

**Using `airmon-ng` (The "easy" way):**

```bash
sudo airmon-ng start wlan0
```

_This will create a new interface usually called `wlan0mon` which is locked in Monitor Mode._

---

### Why does this matter for your "Raw Sniffer"?

If you run your Python raw sniffer in **Promiscuous Mode**, you will see the IP traffic of your network. If you run it in
**Monitor Mode** (and use `AF_PACKET`), you will see the raw 802.11 Wi-Fi management frames, which look very different from
standard Ethernet frames!

**A quick warning for your learning:** Not all Wi-Fi hardware supports Monitor Mode. Many "cheap" USB Wi-Fi dongles are
locked by the manufacturer. If you try the commands above and get an "Operation not supported" error, it's likely a hardware
limitation.
