To understand the modern internet, you have to look at how we identify "homes" for data. An **IP (Internet Protocol)
Address** is that unique digital mailing address.

---

## 1. What is an IP Address?

An IP address is a numerical label assigned to each device connected to a computer network.

The version we most commonly see is **IPv4**, which consists of four numbers (octets) separated by dots, ranging from `0` to
`255`.

- **Example:** `192.168.1.1`
- **Structure:** Each octet represents 8 bits, totaling a **32-bit** address. This allows for about 4.3 billion unique
  addresses (which, in 2026, we have long since exhausted, leading to the rise of IPv6).

---

## 2. Classes of IP Addresses (The Old Way)

In the early days of the internet (the 1980s), IP addresses were divided into "Classes." This was a rigid system that
determined how many networks and hosts (devices) you could have.

| Class       | Range (First Octet) | Purpose                     | Max Hosts per Network |
| :---------- | :------------------ | :-------------------------- | :-------------------- |
| **Class A** | 1 – 126             | Giant corporations/Govs     | 16,777,214            |
| **Class B** | 128 – 191           | Medium-to-large companies   | 65,534                |
| **Class C** | 192 – 223           | Small businesses/Home LANs  | 254                   |
| **Class D** | 224 – 239           | Multicast (Streaming/Video) | N/A                   |
| **Class E** | 240 – 255           | Experimental/Future use     | N/A                   |

> **The Problem:** This was incredibly wasteful. If a company needed 300 addresses, they had to take a Class B block (65k
> addresses), wasting over 64,000 IPs.

---

## 3. CIDR: The Modern Solution

**CIDR (Classless Inter-Domain Routing)** was introduced in 1993 to replace the rigid Class system. It allows for "Variable
Length Subnet Masking," meaning we can slice and dice IP blocks into any size needed.

### How to Read CIDR Notation

CIDR adds a **slash (/)** and a number to the end of an IP address. This number tells you how many bits are "locked" for the
network identity, leaving the rest for devices.

- **Example: `192.168.1.0/24`**
  - The `/24` means the first 24 bits (the first three octets: `192.168.1`) are the **Network Name**.
  - The remaining 8 bits are for **Hosts** (giving you 256 addresses).

### Common CIDR Suffixes:

- **/32**: A single IP address (all bits locked).
- **/24**: A standard small network (256 IPs).
- **/16**: A large network (65,536 IPs).
- **/0**: The entire internet (no bits locked).

---

## 4. Why This Matters for Your App

When you deploy your Node.js app to a cloud provider like AWS, Google Cloud, or Azure, you will often deal with **VPCs
(Virtual Private Clouds)**. You'll be asked to define a CIDR block for your network.

If you choose a block that is too small (like a `/30`), you won't be able to scale your server instances because you'll run
out of "addresses" to give them! Most developers stick to a `/16` for their main cloud network and `/24` for individual
subnets.
