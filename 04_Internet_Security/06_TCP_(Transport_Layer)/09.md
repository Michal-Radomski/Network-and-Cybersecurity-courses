Denial-of-Service (DoS) and Distributed Denial-of-Service (DDoS) attacks have evolved significantly. While the "classic"
attacks we've discussed (like SYN Floods) are still common, modern attackers now use **Multi-Vector** and **AI-powered**
strategies to hit multiple layers of a target's infrastructure at once.

As of 2026, here are the most common ways these attacks are carried out:

### 1. Volumetric (Bandwidth) Attacks

These are the "brute force" attacks. The goal is to saturate the target's internet pipe with so much garbage traffic that
legitimate users can't get through.

- **UDP Floods:** The most dominant method. Attackers send a massive volume of UDP packets to random ports on the target. The
  target's system gets overwhelmed trying to check for applications on those ports and sending back "Destination Unreachable"
  messages.
- **DNS/NTP Amplification:** This is a "force multiplier" attack. The attacker sends a small request to a public server (like
  a DNS provider) but **spoofs** your IP address as the sender. The server sends a massive response to _you_. A tiny 60-byte
  request can turn into a 4,000-byte "bomb" hitting your network.

### 2. Protocol (State-Exhaustion) Attacks

These focus on consuming the actual resources of your hardware (like firewalls, load balancers, or the OS kernel).

- **SYN Flood:** As we discussed, this exploits the TCP handshake to fill up the server's connection memory.
- **Carpet Bombing:** A newer trend where attackers don't hit just one IP, but spread the attack across a whole range of IP
  addresses (a subnet). This makes it much harder for traditional "scrubbing" tools to detect because the traffic at any
  single IP looks relatively low.

### 3. Application Layer (Layer 7) Attacks

These are the "stealthiest" attacks. They don't need much bandwidth because they target specific functions of a website or
app.

- **HTTP Floods:** Similar to thousands of computers hitting "Refresh" on your website at the same time. This forces the
  server to do heavy work (like loading a database or rendering a page) for every fake request.
- **API Targeting:** Attackers now specifically target API endpoints (like `/login` or `/search`) because these often require
  more CPU power to process than a simple static homepage.
- **"Low and Slow" (e.g., Slowloris):** The attacker opens many connections but sends data incredibly slowly. The server
  keeps these "zombie" connections open, waiting for them to finish, until it eventually runs out of available slots for real
  users.

### 4. The 2026 "Next-Gen" Trends

- **Multi-Vector Attacks:** Over 65% of attacks now use two or more of the methods above simultaneously. For example, an
  attacker might start with a UDP flood to distract your IT team, while a stealthy HTTP flood actually crashes your database.
- **AI-Powered Botnets:** Modern botnets (like _Kimwolf_ or _AISURU_) use AI to monitor a target's defense in real-time. If
  they see you've blocked UDP traffic, the botnet automatically switches to an HTTP flood or a SYN flood without the attacker
  needing to intervene.
- **Reconnaissance Probing:** Attacks are now often preceded by "mini-attacks" (10-15 minutes) used to probe your defenses
  and find the specific limit of your firewall before the "real" attack begins.

---

### Comparison of Attack Categories

| Category        | Targeted Resource         | Measurement                 |
| :-------------- | :------------------------ | :-------------------------- |
| **Volumetric**  | Bandwidth (The "Pipe")    | Bits per second (Gbps/Tbps) |
| **Protocol**    | Server/Firewall CPU & RAM | Packets per second (Mpps)   |
| **Application** | Web Server / Database     | Requests per second (RPS)   |

**Pro Tip:** If you are running a server, the best defense in 2026 is no longer just a firewall. Most businesses now use
"Scrubbing Services" (like Cloudflare, Akamai, or AWS Shield) that act as a giant shield, absorbing the Tbps-level floods
before they ever reach your local network.
