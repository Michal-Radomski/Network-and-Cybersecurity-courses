This is a classic "how does the mail get to the right apartment?" problem. Even though both tabs might be visiting the same
website (like `google.com`), the browser and your operating system use a specific set of identifiers to keep the data streams
separate.

The secret sauce is a **4-tuple** (a set of four specific pieces of information) that makes every single TCP connection
unique.

---

## 1. The 4-Tuple Strategy

For every TCP connection, the network stack looks at these four labels:

1.  **Source IP Address:** Your computer's ID.
2.  **Source Port:** A random number assigned by your browser for that specific tab.
3.  **Destination IP Address:** The website server's ID.
4.  **Destination Port:** Usually **443** for HTTPS or **80** for HTTP.

### How it works in practice:

Imagine you have two tabs open to the same site. The Destination IP and Port will be identical for both, but your browser
will assign a different **Source Port** to each tab.

| Connection | Source IP   | **Source Port** | Dest IP     | Dest Port |
| :--------- | :---------- | :-------------- | :---------- | :-------- |
| **Tab 1**  | 192.168.1.5 | **52001**       | 142.250.1.1 | 443       |
| **Tab 2**  | 192.168.1.5 | **52002**       | 142.250.1.1 | 443       |

Because the **Source Port** is different, the operating system knows exactly which "socket" the incoming data belongs to.

---

## 2. The Role of the Operating System (Sockets)

When your browser opens a new tab, it asks the Operating System (OS) for a **Socket**. You can think of a socket as a
specific "mailbox" assigned to that 4-tuple.

- **Outgoing:** When Tab 1 sends a request, it uses Socket A (Port 52001).
- **Incoming:** When the server replies, it sends data back to Port 52001.
- **Delivery:** The OS sees the data is for Port 52001 and hands it specifically to the process/thread tied to Tab 1.

---

## 3. Modern Complexity: Multiplexing (HTTP/2 and HTTP/3)

While the 4-tuple is the foundation, modern browsers are even smarter.

Under **HTTP/2** or **HTTP/3**, a browser might actually use **one** single TCP connection (one 4-tuple) to handle multiple
requests for the same tab. In this case, each request is given a **Stream ID**.

- **Stream 1:** The HTML file.
- **Stream 3:** A CSS file.
- **Stream 5:** An image.

The browser's internal engine manages these IDs to make sure the image doesn't end up trying to render as text. However,
between **two different tabs**, the browser almost always maintains separate TCP connections or uses the OS-level port
separation described above to keep things from getting messy.

---

## Summary

The browser knows where the data goes because:

1.  **Each tab uses a unique Source Port** assigned by the OS.
2.  The **combination** of IP addresses and Ports (the 4-tuple) ensures no two connections are ever truly identical.
3.  The **Operating System** acts as the mail sorter, delivering data to the specific "socket" the browser opened for that
    tab.
