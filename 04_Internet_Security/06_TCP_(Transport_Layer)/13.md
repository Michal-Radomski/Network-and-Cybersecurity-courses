In Linux, the command `echo $$` is a built-in shell variable that returns the **Process ID (PID)** of the current shell
session you are using.

### How it Works

Every process running on a Linux system is assigned a unique identification number called a PID. When you type `echo $$`, the
shell looks up its own unique ID and prints it to the terminal.

- **`echo`**: A command that outputs the text or variables follow it.
- **`$$`**: A special shell variable (an internal parameter) that holds the PID of the process calling it.

---

### Why is this useful?

You will most commonly see `$$` used in **shell scripting** for the following reasons:

1. **Creating Unique Temporary Files**: To prevent multiple instances of a script from overwriting each other's data,
   programmers often append the PID to a filename.

- Example: `temp_data_$$` becomes `temp_data_1234`.

2. **Process Management**: If a script hangs or needs to be tracked, the script can log its own PID so a user can find it
   easily in the process list (using `ps` or `top`).
3. **Debugging**: It helps you verify if you are still in the same shell or if a command has started a "subshell" (a child
   process).

### Quick Comparison

To see this in action, you can compare it with the parent process ID:

| Command      | What it shows                                                     |
| ------------ | ----------------------------------------------------------------- |
| `echo $$`    | The PID of the **current** shell.                                 |
| `echo $PPID` | The PID of the **parent** process that started the current shell. |

---

> **Pro Tip:** If you open a new terminal window or type `bash` to start a nested shell, running `echo $$` again will result
> in a different number because a new process has been created.
