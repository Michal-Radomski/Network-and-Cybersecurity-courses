To check the current value of this setting, you use the `sysctl` command without the `-w` (write) flag.

### How to get the value:

Run this in your terminal:

```bash
sysctl net.ipv4.tcp_syncookies
```

Alternatively, you can read the value directly from the Linux filesystem:

```bash
cat /proc/sys/net/ipv4/tcp_syncookies
```

---

### What is this?

`net.ipv4.tcp_syncookies` is a kernel parameter in Linux that acts as a **primary defense against SYN Flooding attacks**.

When this feature is enabled, the server changes how it handles the "Three-Way Handshake" specifically when its connection
queue (the backlog) starts to fill up.

#### How it works ("The Cookie Trick"):

1.  **Standard Mode:** Normally, when a SYN arrives, the server saves state in memory and waits for an ACK.
2.  **SYN Cookie Mode:** If the server is under attack and the queue is full, it **stops** saving state in memory.
3.  **The "Cookie":** Instead of keeping a record, the server carefully crafts a **Sequence Number** (the "Cookie") based on
    the client's IP, port, and a secret timestamp, and sends that in the SYN-ACK.
4.  **Verification:** If the client is legitimate, it will send back an ACK containing that number. The server "decrypts" the
    number to verify it's valid. If it checks out, the server opens the connection **only then**.

#### Value Meanings:

- **`0`**: Disabled. The server is vulnerable to SYN floods if the backlog fills up.
- **`1`**: Enabled. The kernel will only use SYN cookies when the SYN backlog queue overflows.
- **`2`**: Force Enabled. (Note: This is rare and usually not recommended as it can interfere with certain TCP extensions
  like Window Scaling).

### Why use it?

It allows your server to stay reachable even while being blasted by a script like the one you shared earlier. It effectively
makes the server "stateless" during the handshake, so the attacker's fake requests don't consume any of the server's RAM.

**Note:** Changing this value (using `-w`) usually requires **root** or **sudo** privileges because you are modifying the
behavior of the operating system's networking core.
