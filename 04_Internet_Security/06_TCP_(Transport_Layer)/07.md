A **TCP Reset (RST) Attack** is a technique used to abruptly terminate an established connection between two parties. Unlike
a SYN flood, which prevents a connection from _starting_, a Reset attack kills a connection that is already _active_.

It exploits the **RST flag** in the TCP header, which is normally used to tell a sender: _"I have no record of this
connection, stop sending data."_

---

## 1. How the Attack Works

For a TCP Reset attack to succeed, the attacker must "spoof" a packet so that it looks like it came from one of the two
legitimate parties.

1.  **Eavesdropping:** The attacker monitors the traffic between Client A and Server B (often via a "Man-in-the-Middle"
    position).
2.  **Gathering Intelligence:** To convince the target that the packet is legitimate, the attacker needs three things:
    - **IP Addresses and Ports:** The 4-tuple we discussed earlier.
    - **The Current Sequence Number:** TCP will ignore any packet that falls outside the current "window" of expected
      sequence numbers.
3.  **The "Kill" Packet:** Once the attacker knows the next expected sequence number, they craft a fake TCP packet with the
    **RST flag set to 1**.
4.  **Injection:** The attacker sends this fake packet to the victim.
5.  **The Shutdown:** The victim receives the packet, sees the correct sequence number and source IP, and believes the other
    party has crashed or wants to hang up. It immediately closes the socket and stops communicating.

---

## 2. Famous Use Cases

While it sounds like a purely "criminal" tool, TCP Resets have been used in several famous (and controversial) contexts:

- **The "Great Firewall":** Some national-level firewalls use TCP Reset attacks to block users from accessing forbidden
  websites. When the firewall detects a "forbidden" keyword in a packet, it sends RST packets to both the user and the
  website, tricking them into thinking the connection failed.
- **ISP Traffic Shaping:** In the past, some Internet Service Providers used RST packets to interfere with Peer-to-Peer (P2P)
  file sharing, effectively killing the connection to save bandwidth.
- **Security Tools:** Some Intrusion Prevention Systems (IPS) use TCP Resets to kill a connection if they detect malicious
  behavior or a virus being uploaded.

## 3. Comparison with Other Attacks

| Attack Type    | Target Stage         | Goal                                                          |
| :------------- | :------------------- | :------------------------------------------------------------ |
| **SYN Flood**  | Handshake (Start)    | Prevent new users from connecting.                            |
| **RST Attack** | Established (Middle) | Kick an existing user off a connection.                       |
| **FIN Flood**  | Closing (End)        | Overwhelm the server by forcing it to process close requests. |

---

## 4. How to Prevent It

TCP Reset attacks are much harder to pull off today due to several defenses:

- **Encryption (HTTPS/TLS):** While TLS doesn't hide the TCP headers, it makes it much harder for an attacker to see what's
  happening inside the stream to guess the state of the connection.
- **Sequence Number Randomization:** Modern Operating Systems use highly random initial sequence numbers, making it nearly
  impossible for an attacker to "guess" the correct number unless they can see the live traffic.
- **TCP Authentication Option (TCP-AO):** This adds a cryptographic signature to the TCP header, ensuring that the RST packet
  actually came from the trusted peer.

> **Note:** If you ever see a "Connection Reset by Peer" error in your browser, it usually means the server actually crashed
> or timed out—but in a high-security or censored environment, it could be a sign of a Reset Attack.
