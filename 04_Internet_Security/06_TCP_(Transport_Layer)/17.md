The **Mitnick Attack**, executed by Kevin Mitnick against security researcher Tsutomu Shimomura in 1994, is a legendary
example of a **TCP Sequence Prediction Attack** combined with **IP Spoofing**.

It is a "blind" attack, meaning the attacker can't see the traffic between the targets, yet manages to hijack a trusted
relationship to gain unauthorized access.

---

## How the Attack Works

The attack relies on a vulnerability in how older operating systems generated **Initial Sequence Numbers (ISNs)** for TCP
connections. If the ISNs are predictable, an attacker can forge a connection.

### 1. Identify a Trusted Relationship

Mitnick targeted two machines: a **Target** (Shimomura’s workstation) and a **Trusted Host**. In many Unix systems at the
time, the Target was configured to allow logins from the Trusted Host without a password (using `rlogin`).

### 2. Silence the Trusted Host

To prevent the real Trusted Host from interfering, Mitnick launched a **SYN Flood (DDoS)** against it. This kept the machine
so busy that it couldn't respond to any incoming traffic.

### 3. Predict the Sequence Numbers

Mitnick sent several connection requests to the Target to see how it incremented its TCP Sequence Numbers. By analyzing the
patterns, he could predict what the _next_ sequence number would be.

### 4. The Spoofed Connection

With the Trusted Host "muted," Mitnick sent a SYN packet to the Target, but **spoofed the IP address** to look like it came
from the Trusted Host.

1. **The Forged SYN:** Mitnick sends a SYN to the Target (pretending to be the Trusted Host).
2. **The Target's SYN/ACK:** The Target sends a SYN/ACK back to the _actual_ Trusted Host (which is currently down and
   ignores it).
3. **The Blind ACK:** Mitnick, unable to see the Target's SYN/ACK, guesses the Target's sequence number and sends a forged
   ACK back to the Target.

Because the sequence number matched Mitnick's prediction, the Target believed the "Trusted Host" had established a valid
connection.

### 5. Planting the Backdoor

Once the connection was established, Mitnick sent a command to the Target (still blind) to disable security checks (e.g.,
`echo "+ +" > .rhosts`). This allowed him to log in later from any machine without a password.

---

## Why it Doesn't Work Today

The Mitnick attack was a wake-up call for the networking world, leading to several critical security upgrades:

- **Randomized ISNs:** Modern operating systems use cryptographically secure random number generators for TCP sequence
  numbers, making them nearly impossible to predict.
- The mathematical representation of a secure ISN involves high entropy, where $ISN \approx \text{random}(0, 2^{32}-1)$.

- **Cryptographic Authentication:** Protocols like `rlogin` and `telnet` have been replaced by **SSH** (Secure Shell), which
  uses public-key cryptography to verify identities, rendering IP spoofing useless.
- **Ingress/Egress Filtering:** Routers now often drop packets that claim to be from an internal IP address but arrive on an
  external interface.
- **Firewalls:** Modern firewalls are "stateful," meaning they track the state of a connection and would reject an ACK packet
  that didn't follow a legitimate SYN/ACK sequence they had recorded.

---

## The Aftermath

This attack is famous not just for its technical cleverness, but for the high-stakes digital manhunt that followed. Tsutomu
Shimomura tracked Mitnick across the United States using cellular frequency monitoring and logic, eventually leading to
Mitnick's arrest by the FBI in February 1995.
