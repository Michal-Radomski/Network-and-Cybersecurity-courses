While both TCP and UDP operate at the **Transport Layer** of the OSI model, they handle data very differently. The names
"segment" and "datagram" aren't just jargon; they reflect the fundamental philosophy of how each protocol moves information
across a network.

---

## At a Glance: The Core Differences

The primary distinction lies in **reliability** versus **speed**. TCP (Transmission Control Protocol) is like a certified
letter that requires a signature, while UDP (User Datagram Protocol) is more like a postcard tossed into a mailbox.

| Feature          | TCP Segment                                      | UDP Datagram                 |
| :--------------- | :----------------------------------------------- | :--------------------------- |
| **Connection**   | Connection-oriented (Handshake)                  | Connectionless               |
| **Reliability**  | Guaranteed delivery                              | Best-effort (No guarantee)   |
| **Ordering**     | Data arrives in sequence                         | Data may arrive out of order |
| **Overhead**     | High (20–60 bytes header)                        | Low (8 bytes header)         |
| **Flow Control** | Yes (prevents sender from overwhelming receiver) | No                           |

---

## 1. TCP Segment: The Reliable Stream

A **TCP Segment** is a piece of a larger data stream. Because TCP is "connection-oriented," it must establish a virtual
circuit using a **three-way handshake** before any data moves.

- **Sequence Numbers:** Every segment is numbered. If segments arrive out of order, the receiving device uses these numbers
  to stitch them back together correctly.
- **Acknowledgments (ACKs):** The receiver must tell the sender, "I got segment #4." If the sender doesn't hear back, it
  assumes the segment was lost and sends it again.
- **Error Checking:** It uses a checksum to ensure data isn't corrupted.

---

## 2. UDP Datagram: The Fast Messenger

A **UDP Datagram** is a self-contained, independent entity. It is "connectionless," meaning it simply fires the data at the
destination without checking if the receiver is ready or even exists.

- **No Handshake:** Data starts flowing immediately, reducing latency (ping).
- **Fire and Forget:** There are no sequence numbers or acknowledgments. If a packet is lost in transit, it’s gone forever.
- **Lightweight:** Because it doesn't track anything, the header is tiny (only 8 bytes), making it much more efficient for
  small tasks.

---

## When to Use Which?

### Use TCP (Segments) when...

Accuracy is more important than speed. If a single bit is out of place, the entire file might be useless.

- **Examples:** Web browsing (HTTP/HTTPS), Email (SMTP), File transfers (FTP).

### Use UDP (Datagrams) when...

Speed and real-time delivery are more important than 100% accuracy. If you lose a tiny fraction of data in a video call, the
screen might flicker for a millisecond, but the call continues.

- **Examples:** Online gaming, Live video streaming, VoIP (Voice over IP), DNS lookups.

---

### Summary of Data Units

To keep your networking terminology straight, remember that the name changes based on the layer:

1.  **Layer 4 (Transport):** Segment (TCP) or Datagram (UDP)
2.  **Layer 3 (Network):** Packet (IP)
3.  **Layer 2 (Data Link):** Frame
4.  **Layer 1 (Physical):** Bits
