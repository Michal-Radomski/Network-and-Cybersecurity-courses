In networking, think of an **IP address** as the street address of a building (your computer), and a **Port** as the specific
door or mailbox number within that building.

While the IP address gets the data to the right device, the port ensures the data reaches the right application or service,
such as your web browser, email client, or a video game.

### How Ports Work

Ports are virtual endpoints managed by your computer's operating system. They are identified by a 16-bit number, which means
there are exactly **65,536** available ports (ranging from 0 to 65535).

When data travels over the internet, it uses a transport protocol—usually **TCP** (Transmission Control Protocol) or **UDP**
(User Datagram Protocol)—to "plug into" these ports.

---

### Categories of Ports

The Internet Assigned Numbers Authority (IANA) divides these 65,536 ports into three distinct ranges based on their intended
use:

| Range Name                | Port Numbers    | Purpose                                                                     |
| :------------------------ | :-------------- | :-------------------------------------------------------------------------- |
| **Well-Known Ports**      | 0 – 1,023       | Reserved for standard system services and core protocols (e.g., HTTP, FTP). |
| **Registered Ports**      | 1,024 – 49,151  | Used by specific software applications (e.g., databases, gaming services).  |
| **Dynamic/Private Ports** | 49,152 – 65,535 | Temporary ports used by your device to initiate a connection (client-side). |

---

### Common Ports You Encounter Daily

Most of the "magic" of the internet happens on the **Well-Known Ports**. Here are a few heavy hitters:

- **Port 80 (HTTP):** Unsecured web traffic.
- **Port 443 (HTTPS):** Secured, encrypted web traffic (what you're likely using right now).
- **Port 25 (SMTP):** Used for sending emails.
- **Port 22 (SSH):** Securely logging into another computer remotely.
- **Port 53 (DNS):** Translates website names (google.com) into IP addresses.

### Why do we need so many?

If you only had one port, you could only do one thing at a time. Because we have thousands, you can simultaneously watch a
YouTube video (Port 443), download a file via FTP (Port 21), and have a Zoom call (various UDP ports) without the data
streams getting tangled up.
