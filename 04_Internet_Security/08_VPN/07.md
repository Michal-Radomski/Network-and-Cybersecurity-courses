## What is IPsec?

**IPsec (Internet Protocol Security)** is a robust suite of protocols used to secure network communication by authenticating
and encrypting each IP packet in a communication session. Think of it as a secure, encrypted "tunnel" that protects data as
it travels across an untrusted network, like the public internet.

It is the foundational technology behind most **Virtual Private Networks (VPNs)**, allowing remote workers to connect
securely to office servers or linking two separate offices over the web.

---

### Core Functions of IPsec

IPsec provides four critical security services to ensure data remains private and unaltered:

1. **Confidentiality:** It encrypts the data so that even if a hacker intercepts the packets, they cannot read the content.
2. **Integrity:** It ensures that the data has not been tampered with or changed during transit.
3. **Authentication:** It verifies that the data is actually coming from the person or device it claims to be from.
4. **Anti-Replay:** It prevents attackers from capturing a legitimate packet and "replaying" it later to trick the system
   (e.g., resending a "transfer funds" request).

---

### How It Works: The Two Main Protocols

IPsec uses two distinct protocols to handle the heavy lifting of security:

- **Authentication Header (AH):** This focuses on integrity and authentication. It signs the packet to prove it hasn't been
  modified, but it **does not** encrypt the data. Because it doesn't provide privacy, it is used less frequently today.
- **Encapsulating Security Payload (ESP):** This is the "gold standard." It provides encryption (confidentiality) as well as
  authentication and integrity. Most modern VPNs rely primarily on ESP.

---

### The Two Modes of Operation

Depending on how much of the data packet needs to be hidden, IPsec operates in one of two modes:

| Mode               | What is Encrypted?                                                                   | Common Use Case                                          |
| ------------------ | ------------------------------------------------------------------------------------ | -------------------------------------------------------- |
| **Transport Mode** | Only the **payload** (the actual message). The original IP headers stay visible.     | End-to-end communication between two specific computers. |
| **Tunnel Mode**    | The **entire packet** (payload + original header). A new "outer" IP header is added. | **Site-to-Site VPNs** connecting two different networks. |

### The Handshake: Internet Key Exchange (IKE)

Before data can be sent, the two devices must agree on "how" they will talk securely. This is handled by **IKE**.

1. They negotiate which encryption algorithms to use (like AES).
2. They exchange "keys" to lock and unlock the data.
3. They establish a **Security Association (SA)**, which is essentially a contract defining the security parameters for that
   specific session.

### Why Use IPsec?

- **Invisible to Users:** It operates at the Network Layer (Layer 3), meaning applications don't need to be specially
  configured to use it.
- **High Security:** It uses industry-standard, military-grade encryption.
- **Versatility:** It can secure any traffic that runs over IP, including web browsing, email, and file transfers.
