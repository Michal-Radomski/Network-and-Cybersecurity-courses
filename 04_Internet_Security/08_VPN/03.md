**SOCKS** (which stands for **SOCKets**) is an internet protocol that routes network packets between a client and a server
through a proxy server.

Think of it as a "middleman" that handles your data traffic. Unlike a standard HTTP proxy, which only understands web
traffic, SOCKS is much more versatile.

### How It Works

When you use a SOCKS proxy, your traffic is sent to the proxy server first. The proxy server then communicates with the
destination website or service on your behalf. To the destination, it looks like the traffic is coming from the proxy
server’s IP address, not yours.

---

### Key Features of SOCKS

- **Protocol Agnostic:** SOCKS doesn't care what kind of traffic you're sending. It works for web browsing (HTTP), email
  (SMTP/IMAP), file transfers (FTP), and even gaming or torrenting.
- **Layer 5 Operation:** It operates at the "Session Layer" of the OSI model. This means it sits between the application
  (like your browser) and the transport layer (TCP/UDP), allowing it to handle many different types of requests.
- **No Header Rewriting:** Unlike HTTP proxies, SOCKS doesn't interpret or modify the data headers. It simply "pipes" the
  data through, which often makes it faster and less likely to cause errors with specialized applications.

---

### SOCKS4 vs. SOCKS5

There are two main versions you will encounter today:

| Feature            | SOCKS4        | SOCKS5                              |
| ------------------ | ------------- | ----------------------------------- |
| **Authentication** | No            | **Yes** (Username/Password support) |
| **UDP Support**    | No (TCP only) | **Yes**                             |
| **IPv6 Support**   | No            | **Yes**                             |
| **Remote DNS**     | No            | **Yes** (Prevents DNS leaks)        |

---

### Common Use Cases

1. **Bypassing Firewalls:** If your local network blocks a specific service (like a gaming server), you can use a SOCKS proxy
   to route your traffic around the restriction.
2. **Privacy:** By masking your IP address, it adds a layer of anonymity, though it does **not** provide encryption on its
   own (unlike a VPN or an SSH tunnel).
3. **Accessing Geo-Restricted Content:** You can appear to be in a different country by connecting to a proxy server located
   there.
4. **SSH Tunneling:** As mentioned in our previous discussion, the `-D` flag in SSH creates a dynamic SOCKS proxy, which is a
   popular way to secure traffic on public Wi-Fi.

> **A Note on Security:** While SOCKS5 is great for routing, it is **not a replacement for a VPN**. A VPN encrypts all your
> device's traffic; a SOCKS proxy only routes the traffic of the specific application configured to use it, and it doesn't
> provide encryption unless it is wrapped inside another secure protocol (like SSH).
