A **Cyclic Redundancy Check (CRC)** is a powerful mathematical "checksum" used by digital networks and storage devices to
detect accidental changes to raw data. It is the gold standard for error detection at the **Data Link Layer** because it is
extremely efficient at catching common types of transmission errors, such as "bit flips" caused by electrical interference.

---

### How It Works: The "Remainder" Concept

CRC is based on **binary polynomial division**. Think of it like a long division problem where we don't care about the
quotient (the result), only the **remainder**.

1.  **The Generator:** Both the sender and receiver agree on a fixed "magic number" called a **Generator Polynomial** (e.g.,
    CRC-32).
2.  **The Calculation:** The sender treats the data bits as one giant number and divides it by the Generator.
3.  **The Tag:** The resulting **remainder** is attached to the end of the data frame (this is the **FCS** or Frame Check
    Sequence).
4.  **The Verification:** The receiver takes the entire received frame (data + remainder) and divides it by the same
    Generator.
    - If the new remainder is **zero**, the data is considered "clean."
    - If the remainder is **not zero**, the data was corrupted during transit.

---

### The Math Behind the Magic

CRC doesn't use standard schoolbook math; it uses **Modulo-2 Arithmetic**. In this system, addition and subtraction are
replaced by the **XOR (Exclusive OR)** logical operation.

- 1 + 1 = 0
- 1 + 0 = 1
- 0 + 0 = 0

This makes the math incredibly fast for hardware to calculate because there are no "carries" or "borrows" to worry about. The
division is essentially a series of XOR shifts.

---

### Why use CRC instead of a simple Sum?

Earlier systems used a "checksum" (simply adding up all the bytes). However, checksums are "weak." If two bits flip in a way
that cancels each other out, a simple checksum won't notice.

**CRC is superior because:**

- **Burst Error Detection:** It is excellent at detecting "burst errors," where a series of consecutive bits are corrupted
  (common in lightning strikes or power surges).
- **Hardware Efficiency:** It can be implemented using simple shift registers and XOR gates, making it lightning-fast for
  network cards.
- **Reliability:** A 32-bit CRC (CRC-32) has a $1$ in $2^{32}$ (about 1 in 4.3 billion) chance of failing to detect an error.

---

### What happens if an error is found?

It’s important to remember that **CRC only detects errors; it does not fix them.** In the Data Link Layer (Ethernet/Wi-Fi),
if a frame fails the CRC check, the hardware simply **drops the frame**. It doesn't ask for a retransmission—that is usually
left to higher-layer protocols like **TCP** at the Transport Layer to realize data is missing and request it again.
