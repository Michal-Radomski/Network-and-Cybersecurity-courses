A **Man-in-the-Middle (MitM)** attack using ARP Poisoning is a method where an attacker positions themselves virtually
between a victim and a network resource (like a router). By manipulating the ARP tables of both parties, the attacker ensures
that all data flowing between them passes through their machine first.

---

### The Three Players

To visualize this, imagine three entities on a local network:

1.  **The Victim (User):** Wants to access the internet.
2.  **The Gateway (Router):** The door to the internet.
3.  **The Attacker:** A malicious device sitting on the same Wi-Fi or Ethernet segment.

---

### Phase 1: The "Double Lie"

The attacker sends continuous, unsolicited ARP replies (Gratuitous ARPs) to both the Victim and the Router.

- **To the Victim:** The attacker says, _"I am the Router."_ The Victim’s computer updates its ARP cache, linking the
  Router’s IP to the **Attacker’s MAC address**.
- **To the Router:** The attacker says, _"I am the Victim."_ The Router updates its ARP cache, linking the Victim’s IP to the
  **Attacker’s MAC address**.

---

### Phase 2: Traffic Interception

Now that the "poisoning" is complete, the hardware-level routing is broken:

- When the Victim sends a request to a website, the Ethernet frame is addressed to the Attacker's MAC.
- The Attacker receives the frame, looks at the data, and then **forwards** it to the real Router so the connection doesn't
  drop (which would alert the victim).
- When the Router sends the website data back, it sends it to the Attacker's MAC, thinking it is the Victim.

---

### Phase 3: The Exploitation

Once the attacker is "in the middle," they can perform several malicious actions:

- **Eavesdropping:** Using tools like Wireshark to capture unencrypted data (HTTP passwords, FTP credentials, Telnet
  sessions).
- **Data Manipulation:** Altering the data in transit. For example, replacing a legitimate file download with a
  malware-infected version.
- **Session Hijacking:** Stealing session cookies to take over logged-in accounts without needing a password.
- **SSL Stripping:** Forcing the victim's browser to use an unencrypted HTTP connection instead of HTTPS, making encrypted
  data readable.

---

### How to Detect it on Ubuntu

If you suspect an MitM attack is happening on your network, you can look for "Duplicate MACs" in your ARP table. If two
different IP addresses (like your Router and a random laptop) share the exact same MAC address, you are likely being spoofed.

**Check your table:**

```bash
ip neighbor show
```

**Monitor for ARP changes in real-time:** You can use `tcpdump` to watch for suspicious ARP traffic:

```bash
sudo tcpdump -i [your_interface] arp
```

_Look for an unusual volume of ARP replies coming from a single device that you didn't request._

---

### Prevention Summary

| Defense                | How it Works                                                                                                    |
| :--------------------- | :-------------------------------------------------------------------------------------------------------------- |
| **Static ARP**         | Manually locking the Router's MAC address in your settings.                                                     |
| **VPN**                | Encrypts all data from your device to a remote server, making intercepted data unreadable to the MitM attacker. |
| **DAI (Switch level)** | The network switch identifies and blocks fake ARP packets before they reach you.                                |
| **HTTPS Everywhere**   | Ensures that even if the connection is intercepted, the content remains encrypted.                              |
