Think of an **Ethernet Frame** as the "digital envelope" used to transport data across a local area network (LAN). While the
Network Layer deals with IP addresses, the Ethernet Frame (at the Data Link Layer) deals with the actual hardware-to-hardware
delivery.

An Ethernet frame has a specific structure defined by the **IEEE 802.3** standard. Here is the breakdown of its components in
the order they travel across the wire:

---

### The Anatomy of an Ethernet Frame

| Section                | Size (Bytes) | Purpose                                                                                   |
| :--------------------- | :----------- | :---------------------------------------------------------------------------------------- |
| **Preamble**           | 7            | A pattern of alternating 1s and 0s that "wakes up" the receiving NIC.                     |
| **SFD**                | 1            | The **Start Frame Delimiter** marks the end of the preamble and the start of actual data. |
| **Destination MAC**    | 6            | The hardware address of the intended receiver.                                            |
| **Source MAC**         | 6            | The hardware address of the sender.                                                       |
| **EtherType / Length** | 2            | Indicates which protocol is inside (usually IPv4 or IPv6).                                |
| **Payload (Data)**     | 46 – 1500    | The actual data (packet) being sent.                                                      |
| **FCS**                | 4            | The **Frame Check Sequence** used to detect if the data was corrupted.                    |

---

### Key Components Explained

#### 1. MAC Addresses (The Headers)

The frame contains two physical addresses. Switches use the **Destination MAC** to decide which port to send the frame out
of. If the destination is unknown, the switch broadcasts it to everyone.

#### 2. The Payload (The "Letter" inside the Envelope)

The payload contains the data from the upper layers.

- **MTU (Maximum Transmission Unit):** Standard Ethernet frames have a maximum payload size of **1500 bytes**.
- **Padding:** If the data is too small (less than 46 bytes), "padding" (extra zeros) is added to ensure the frame meets the
  minimum required length for collision detection.

#### 3. Frame Check Sequence (The Seal)

The FCS uses a mathematical formula called **CRC (Cyclic Redundancy Check)**.

- The sender calculates a value based on the frame's contents and attaches it to the end.
- The receiver performs the same calculation.
- If the values don't match, the receiver knows the frame was damaged and **drops it** immediately. Ethernet doesn't fix
  errors; it just throws away the "broken" envelopes.

---

### Interesting Fact: The Interpacket Gap

Frames aren't sent back-to-back with zero space. There is a brief period of "silence" on the wire between frames, known as
the **Interpacket Gap (IPG)**. This gives the receiving hardware time to process the current frame before the next preamble
arrives.
