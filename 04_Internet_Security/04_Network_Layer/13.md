An **ICMP Redirect Attack** is a type of network attack where an adversary manipulates the routing table of a victim host. By
spoofing specific control messages, the attacker tricks the host into sending its traffic through a device controlled by the
attacker rather than the legitimate default gateway.

---

## How it Works

To understand the attack, you first need to understand the legitimate purpose of an **ICMP Redirect (Type 5)** message. These
messages are designed for efficiency: if a router receives a packet and realizes there is a "better" (shorter) path to the
destination on the same local network, it forwards the packet and sends an ICMP Redirect to the sender to update its local
routing cache.

### The Attack Steps:

1.  **Interception:** The attacker identifies a target host and the legitimate gateway on the local area network (LAN).
2.  **Spoofing:** The attacker sends a forged ICMP Redirect message to the victim. This packet claims to be from the
    **legitimate gateway**.
3.  **Redirection:** The message tells the victim: _"To reach [Destination X], use [Attacker's IP] instead of me."_
4.  **Route Update:** The victim’s operating system updates its routing table (or route cache) with the new, malicious entry.
5.  **Man-in-the-Middle (MitM):** Subsequent traffic from the victim destined for the target is now sent directly to the
    attacker. The attacker can then inspect, log, or modify the data before forwarding it to the real gateway to avoid
    detection.

---

## Technical Requirements

For this attack to be successful, several conditions usually must be met:

- **Local Network Access:** The attacker must be on the same broadcast domain (subnet) as the victim.
- **OS Vulnerability:** The victim's operating system must be configured to accept and process ICMP Redirect messages (many
  modern systems disable this by default for non-routers).
- **Knowledge of IPs:** The attacker needs the IP addresses of the victim, the gateway, and a target destination.

---

## Potential Impact

- **Data Exfiltration:** Capturing sensitive information like login credentials or unencrypted emails.
- **Traffic Alteration:** Injecting malicious code into unencrypted web traffic (HTTP).
- **Denial of Service (DoS):** Redirection to a non-existent IP address, effectively "blackholing" the victim's traffic.

---

## Mitigation and Defense

Modern network security relies on "Zero Trust" regarding ICMP control messages. Here is how to prevent these attacks:

### 1. Host-Level Configuration

Most modern operating systems allow you to ignore redirect messages.

- **Linux:** Set `net.ipv4.conf.all.accept_redirects` to `0` in `sysctl.conf`.
- **Windows:** Generally ignores redirects by default in newer versions, but this can be hardened via Registry settings.

### 2. Network-Level Security

- **Disabling Redirects on Routers:** Configure gateways to never send ICMP redirects.
- **Firewalls:** Use local firewalls to drop ICMP Type 5 packets.
- **Intrusion Detection Systems (IDS):** Set up alerts for unexpected ICMP Redirect messages, which are rare in
  well-configured modern networks.

### 3. Encryption

Using **HTTPS, VPNs, or SSH** ensures that even if an attacker successfully redirects the traffic, the data remains encrypted
and unreadable to them.
