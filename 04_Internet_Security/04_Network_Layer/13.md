The **Dynamic Host Configuration Protocol (DHCP)** is a network management protocol used to automate the process of
configuring devices on IP networks. Without it, you would have to manually assign an IP address to every single phone,
laptop, and smart toaster that joins your Wi-Fi—a task that would be, frankly, a nightmare.

### The Core Concept

DHCP works on a **client-server model**. A "DHCP Server" (usually your router at home) maintains a pool of IP addresses and
"leases" them to "DHCP Clients" (your devices) for a set period of time.

---

### The DORA Process

The magic happens in a four-step handshake known by the acronym **DORA**.

1.  **D - Discovery:** When your device connects to a network, it doesn't have an IP address yet. It sends out a
    **DHCPDISCOVER** broadcast message. It's essentially shouting, _"Is there a DHCP server out here? I need an address!"_
2.  **O - Offer:** The DHCP server hears the shout and responds with a **DHCPOFFER**. It says, _"I'm here! I have this IP
    address ($192.168.1.50$) available for you."_
3.  **R - Request:** The device receives the offer and sends back a **DHCPREQUEST**. This confirms it wants that specific
    address: _"Great, I'll take $192.168.1.50$. Please reserve it for me."_
4.  **A - Acknowledgment:** The server sends a final **DHCPACK**. It's the digital equivalent of a signed contract: _"You've
    got it. Here are your subnet mask, default gateway, and DNS settings. See you in 24 hours when the lease expires!"_

---

### Key Components of a DHCP Lease

When a server hands out an IP, it doesn't just give a number; it provides a full configuration suite:

| Component           | Purpose                                                               |
| :------------------ | :-------------------------------------------------------------------- |
| **IP Address**      | The unique identifier for the device on the network.                  |
| **Subnet Mask**     | Defines which part of the IP is the network and which is the host.    |
| **Default Gateway** | The "exit door" (router) the device uses to reach the internet.       |
| **DNS Servers**     | The "phonebook" that translates website names (google.com) into IPs.  |
| **Lease Time**      | How long the device can keep the address before asking for a renewal. |

---

### Why do we use it?

- **No IP Conflicts:** The server tracks who has what, so two devices never try to use the same address at once.
- **Automation:** You can walk into a coffee shop, open your laptop, and be online in seconds without touching a single
  setting.
- **Efficiency:** When you leave the network, your IP address goes back into the "pool" to be reused by someone else.

Setting up a Dynamic Host Configuration Protocol (DHCP) server on Ubuntu is a great way to manage IP addresses across your
network automatically. The most common and reliable tool for this is the `isc-dhcp-server` package.

Here is a step-by-step guide to getting it up and running.

---

## 1. Installation

First, update your package repository to ensure you are getting the latest version, then install the server package.

```bash
sudo apt update
sudo apt install isc-dhcp-server
```

> **Note:** Don't worry if the service fails to start immediately after installation. This is normal because it hasn't been
> configured for your network yet.

---

## 2. Identify the Network Interface

You need to know which network card (Ethernet or Wi-Fi) will be handing out the IP addresses. Run the following command:

```bash
ip link show
```

Look for an interface name like `eth0`, `enp0s3`, or `eno1`.

---

## 3. Define the Interface

Now, tell the DHCP server which interface to listen on. Edit the default configuration file:

```bash
sudo nano /etc/default/isc-dhcp-server
```

Find the line `INTERFACESv4=""` and add your interface name inside the quotes: `INTERFACESv4="enp0s3"`

---

## 4. Configure the DHCP Scope

This is the most important step where you define the range of IPs and network settings. Open the main configuration file:

```bash
sudo nano /etc/dhcp/dhcpd.conf
```

Find the section for your subnet or scroll to the bottom and add a block like this (adjust the numbers to match your
network):

```text
subnet 192.168.1.0 netmask 255.255.255.0 {
  range 192.168.1.100 192.168.1.200;      # Range of IPs to assign
  option routers 192.168.1.1;             # Your Gateway/Router IP
  option domain-name-servers 8.8.8.8, 8.8.4.4;
  default-lease-time 600;
  max-lease-time 7200;
}
```

### Key Parameters:

- **range**: The pool of IP addresses available for clients.
- **option routers**: The default gateway for the clients.
- **default-lease-time**: How long (in seconds) a device keeps an IP before asking for a renewal.

---

## 5. Start and Test the Service

After saving your changes, restart the DHCP service to apply them:

```bash
sudo systemctl restart isc-dhcp-server
```

To verify that the server is running correctly without errors, check its status:

```bash
sudo systemctl status isc-dhcp-server
```

---

## 6. Verification

To see which devices have been assigned an IP address by your new server, you can check the lease file:

```bash
cat /var/lib/dhcp/dhcpd.leases
```

### Quick Troubleshooting Tips:

- **Static IP**: The server running the DHCP service **must** have a static IP address assigned to it within the same subnet
  you are configuring.
- **Firewall**: If you have `ufw` enabled, allow DHCP traffic: `sudo ufw allow 67/udp`
