To understand the difference between segments, packets, and frames, you have to look at the **OSI Model**. These terms all
refer to the same data, but they are called different things depending on which "layer" of the network they are currently
traveling through.

Think of it like a letter: it starts as **thoughts** (Data), gets put in an **envelope** (Segment), addressed with a **house
number** (Packet), and finally placed in a **mail truck** (Frame) for physical delivery.

---

### The Data Encapsulation Hierarchy

As data moves down the network stack, each layer adds its own "header" (metadata) to the information. This process is called
**encapsulation**.

| Layer                   | Data Unit (PDU) | Responsibility                          | Addressing Used                     |
| :---------------------- | :-------------- | :-------------------------------------- | :---------------------------------- |
| **Layer 4 (Transport)** | **Segment**     | Reliable delivery & error correction    | Port Numbers (e.g., Port 80, 443)   |
| **Layer 3 (Network)**   | **Packet**      | Routing across different networks       | IP Addresses (Source & Destination) |
| **Layer 2 (Data Link)** | **Frame**       | Physical transfer between local devices | MAC Addresses                       |

---

### 1. Segments (Layer 4 - Transport)

At this stage, the raw data from your application (like a chunk of an email) is broken down into manageable pieces.

- **Protocol:** Mostly **TCP** (Transmission Control Protocol).
- **Key Info:** The header includes **Port Numbers**, which tell the receiving computer which specific application (like
  Chrome vs. Spotify) should receive the data.
- **Function:** It handles sequencing (putting pieces back in the right order) and flow control.

### 2. Packets (Layer 3 - Network)

The segment is "wrapped" inside a packet. This is the level where the internet actually functions.

- **Protocol:** Primarily **IP** (Internet Protocol).
- **Key Info:** The header contains the **Source and Destination IP Addresses**.
- **Function:** This is the logical addressing layer. Routers use the IP information in a packet to decide the best path
  across the globe to reach the destination.

### 3. Frames (Layer 2 - Data Link)

The packet is then "wrapped" inside a frame. This is the final software layer before the data hits the physical wire or Wi-Fi
signal.

- **Protocol:** Usually **Ethernet** or **Wi-Fi (802.11)**.
- **Key Info:** The header contains **MAC Addresses** (the physical hardware ID of your network card).
- **Function:** Frames are responsible for "hop-to-hop" delivery. While an IP address gets a packet to the right building, a
  MAC address gets the frame to the specific computer plugged into the switch.

---

### Summary Analogy

If you are ordering a pizza:

- **Segment:** The **Pizza slices** in the box (organized and ready to be consumed).
- **Packet:** The **Delivery Address** written on the box (the global location/IP).
- **Frame:** The **Delivery Vehicle** (the physical transport method—bike or car—that moves the box from point A to point B).
