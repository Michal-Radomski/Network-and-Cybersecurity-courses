Understanding how an IP packet travels from a sender to a receiver is like tracing a letter through a global postal system,
but at the speed of light. The process involves multiple layers of the networking stack, moving from high-level data to
physical signals.

---

## 1. The Source: Encapsulation

Before a packet ever hits the wire, it is built through a process called **encapsulation**.

- **Application Layer:** Your data (like an HTTP request) is created.
- **Transport Layer:** The data is broken into segments and given a **TCP** or **UDP** header (including source and
  destination ports).
- **Network Layer:** The segment is wrapped in an **IP Header**, becoming a **packet**. This header contains the source and
  destination IP addresses.
- **Data Link Layer:** The packet is wrapped in a **Frame** (usually Ethernet). This adds the **MAC addresses** of the next
  physical hop.

---

## 2. The Local Exit: Default Gateway

Once the packet is ready, the host checks the destination IP.

- **Same Subnet:** If the destination is local, the host uses **ARP (Address Resolution Protocol)** to find the receiver's
  MAC address and sends it directly.
- **Different Subnet:** If the destination is outside the local network, the host sends the packet to its **Default Gateway**
  (the local router).

---

## 3. The Journey: Routing and Hopping

This is where the "traversal" really happens. As the packet moves across the internet, it passes through several routers.

### The Routing Decision

Each router performs the following steps:

1.  **De-encapsulation:** The router strips off the Ethernet frame to look at the IP packet inside.
2.  **Route Lookup:** It checks its **Routing Table** for the best path to the destination IP.
3.  **TTL (Time to Live) Decrement:** The router reduces the TTL value by 1. If it hits 0, the packet is dropped (preventing
    infinite loops).
4.  **Re-encapsulation:** The router wraps the packet in a _new_ frame with the MAC address of the _next_ router and sends it
    out.

---

## 4. The Arrival: Delivery and Assembly

When the packet finally reaches the destination network, the final router recognizes the IP address as belonging to a local
host.

- **Final Delivery:** The router uses ARP to get the destination host’s MAC address and delivers the frame.
- **Decapsulation:** The receiving host strips the headers in reverse order:
  1.  **Link Layer:** Checks the MAC address.
  2.  **Network Layer:** Verifies the IP address.
  3.  **Transport Layer:** Uses the Port number to send the data to the correct application.
- **Reassembly:** If the data was split into multiple packets (fragmentation), the receiving host puts them back together.

---

## Summary Table: What Changes vs. What Stays?

| Element             | Changes during traversal? | Why?                                                   |
| :------------------ | :------------------------ | :----------------------------------------------------- |
| **Source/Dest IP**  | **No** (usually)          | These are the ultimate start and end points.           |
| **Source/Dest MAC** | **Yes**                   | These change at every "hop" to get to the next device. |
| **TTL Value**       | **Yes**                   | Decremented by 1 at every router to prevent loops.     |
| **Payload Data**    | **No**                    | The actual message remains untouched.                  |

> **Note on NAT:** In most home or office setups, **Network Address Translation (NAT)** will actually change the Source IP
> address at the first router to a public IP, and then reverse it on the way back.
