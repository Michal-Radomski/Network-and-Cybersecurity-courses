IP fragmentation is the "I’ll make it fit" mechanism of the internet. It happens when a packet is too large to pass through a
specific network link, so it has to be broken into smaller pieces (fragments) and reassembled at the destination.

---

## 1. The Why: Understanding MTU

Every network link has a limit on the maximum size of a packet it can carry, known as the **MTU (Maximum Transmission
Unit)**.

- **Ethernet Standard:** The most common MTU is **1500 bytes**.
- **The Problem:** If a router receives a packet that is 1500 bytes, but the next link in the path only supports an MTU of
  **1000 bytes**, the packet cannot pass as-is.

---

## 2. The How: Fragmentation Fields

When a router decides to fragment a packet, it uses three specific fields in the **IP Header** to make sure the pieces can be
put back together:

1.  **Identification (ID):** Every fragment from the original packet gets the same ID number. This tells the receiver, "All
    these pieces belong to the same original message."
2.  **Flags:**
    - **DF (Don't Fragment):** If this bit is set to 1 and the packet is too big, the router will drop it and send back an
      error.
    - **MF (More Fragments):** Set to 1 on all fragments except the very last one. This tells the receiver when it has
      finally received the whole "set."
3.  **Fragment Offset:** This tells the receiver exactly where this piece fits in the original data stream (measured in units
    of 8 bytes).

---

## 3. The Process: Breaking and Reassembling

### At the Router (The Breakup)

If a 1500-byte packet needs to go through a 1000-byte MTU link:

- The router splits the data into two chunks.
- It copies the original IP header onto _both_ chunks.
- It updates the **Offset** and **MF** bits so the receiver knows the first chunk is at the start and more are coming.

### At the Destination (The Reassembly)

Fragments are **not** reassembled by the intermediate routers. They stay fragmented until they reach the final destination
host.

- The destination host looks at the **ID** to group fragments.
- It uses the **Offset** to arrange them in the correct order.
- Once it has the "Last Fragment" (MF=0) and all the gaps are filled, it passes the data up to the Transport Layer (TCP/UDP).

---

## 4. Why Fragmentation is Avoided

While fragmentation works, network engineers try to avoid it whenever possible because:

- **Efficiency:** It adds overhead (more headers).
- **CPU Load:** Routers and hosts have to work harder to slice and dice data.
- **Fragility:** If just **one** fragment is lost, the entire original packet is useless and must be retransmitted.

### The Modern Solution: PMTUD

Most modern systems use **Path MTU Discovery (PMTUD)**. They send packets with the **DF (Don't Fragment)** bit set. If a
packet hits a small link, the router sends back an ICMP error saying "Too Big." The sender then shrinks its packets from the
start, avoiding fragmentation entirely.
