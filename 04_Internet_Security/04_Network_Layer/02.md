The **Internet Protocol (IP)** is the primary protocol of the Network Layer. It acts as the universal language of the
internet, providing the rules for how data packets are addressed and routed so they can travel across multiple networks to
reach their final destination.

IP is a **connectionless** and **"best-effort"** protocol. This means it doesn't stay on the line to make sure the data
arrived (that's TCP's job); it simply focuses on getting the packet to the right address.

---

## 1. Key Characteristics of IP

- **Connectionless:** IP does not establish a "handshake" before sending data. It just sends the packet.
- **Best-Effort Delivery:** IP does not guarantee that data will reach its destination. It doesn't handle flow control or
  error recovery for the data itself.
- **Media Independent:** It doesn't care if the data is traveling over fiber optics, copper wire, or satellite signals.

---

## 2. The IP Header

Every IP packet has a header containing metadata necessary for routing. While the data (payload) is the "letter," the header
is the "envelope."

### Core Header Fields:

- **Source IP Address:** The logical address of the sender.
- **Destination IP Address:** The logical address of the receiver.
- **Time to Live (TTL):** A hop counter that prevents packets from looping infinitely.
- **Protocol:** Identifies which protocol is inside the payload (e.g., TCP = 6, UDP = 17).
- **Version:** Identifies if it is IPv4 or IPv6.

---

## 3. IPv4 vs. IPv6

There are currently two versions of IP in use. Because the internet grew faster than anyone expected, the original version
(IPv4) ran out of addresses.

| Feature             | IPv4                                 | IPv6                                      |
| :------------------ | :----------------------------------- | :---------------------------------------- |
| **Address Size**    | 32-bit                               | 128-bit                                   |
| **Address Format**  | Decimal (e.g., `192.168.1.1`)        | Hexadecimal (e.g., `2001:0db8:85a3...`)   |
| **Total Addresses** | ~$4.3$ billion                       | ~$340$ undecillion (effectively infinite) |
| **Header**          | Variable length ($20$ to $60$ bytes) | Fixed length ($40$ bytes)                 |
| **Security**        | Optional (IPsec)                     | Built-in (IPsec support mandated)         |

---

## 4. IP Fragmentation

Different networks have different limits on how large a packet can be—this limit is called the **MTU (Maximum Transmission
Unit)**.

- If a packet is too big for a network it needs to cross, the **IP Protocol** allows routers to break the packet into smaller
  pieces (fragments).
- The receiving device then uses the **Identification** and **Fragment Offset** fields in the IP header to put the original
  "puzzle" back together.

---

## 5. Logical Addressing & Subnetting

IP doesn't just give every device a random number. IP addresses are hierarchical, split into:

1.  **Network Portion:** Identifies the specific network you are on.
2.  **Host Portion:** Identifies your specific device on that network.

**Subnet Masks** (like `255.255.255.0`) are used by the protocol to determine where the "Network" ends and the "Host" begins.
This allows routers to quickly send data toward the right network without needing to know exactly where every single laptop
is located.

---

### Summary

The IP Protocol is the foundation of the internet. It provides the **addressing** (who are you?) and the **fragmentation**
(how do we fit?) necessary for data to move between different hardware types and geographic locations.
