Think of **TTL (Time to Live)** as a self-destruct timer for a packet, and **traceroute** as the specialized tool that uses
those timers to map out the "hops" across the internet.

---

## 1. TTL (Time to Live)

Despite the name, TTL isn't measured in seconds or minutes; it is measured in **router hops**. It is an 8-bit field in the
IPv4 header (or a "Hop Limit" in IPv6) designed to prevent packets from circling the internet forever if a routing loop
occurs.

- **How it works:** Every time a packet arrives at a router, the router decrements (subtracts) **1** from the TTL value.
- **The Limit:** If a router receives a packet with a TTL of **1**, it drops it to **0** and discards the packet.
- **The Notification:** When the router drops the packet, it sends an **ICMP "Time Exceeded"** message back to the original
  sender to let them know the packet died.

---

## 2. Traceroute: The Network Mapmaker

Traceroute is a diagnostic tool that purposefully "kills" packets to discover every router between you and a destination. It
exploits the TTL mechanism to identify the path.

### How Traceroute Works (Step-by-Step)

1.  **Hop 1:** Traceroute sends a packet with **TTL = 1**. The first router it hits decrements the TTL to 0, drops the
    packet, and sends back an ICMP error. Traceroute records that router's IP and the time it took.
2.  **Hop 2:** It sends another packet with **TTL = 2**. The first router passes it (TTL becomes 1), and the second router
    drops it (TTL becomes 0). The second router sends back an ICMP error.
3.  **The Pattern:** This continues (TTL = 3, 4, 5...) until the packet finally reaches the destination or a predefined limit
    (usually 30 hops).

---

## 3. Comparison and Usage

| Feature       | TTL (The Field)                  | Traceroute (The Tool)                                |
| :------------ | :------------------------------- | :--------------------------------------------------- |
| **Role**      | A safety mechanism.              | A diagnostic utility.                                |
| **Location**  | Inside the IP Header.            | An application/command-line tool.                    |
| **Main Goal** | Prevent infinite routing loops.  | Map the path and measure latency.                    |
| **Outcome**   | Silent drop (unless using ICMP). | Visual list of router IPs and RTT (Round Trip Time). |

### Reading a Traceroute Output

When you run `traceroute google.com`, you'll see three time values (in milliseconds) for each hop. This is because traceroute
typically sends **three packets** per hop to get a better average of the latency and to account for potential packet loss.

- **Asterisks (`* * *`):** If you see stars instead of a hostname, it usually means a router is configured to ignore ICMP
  requests for security or to save processing power. It doesn't necessarily mean the network is down—just that the router is
  "shy."
