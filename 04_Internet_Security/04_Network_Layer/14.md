A **Smurf attack** is a type of distributed denial-of-service (DDoS) attack that exploits the Internet Control Message
Protocol (ICMP) to overwhelm a target’s network with traffic. It is named after a tool called "Smurf," which was widely used
in the late 1990s.

The attack essentially uses "amplification" to turn a small request into a massive flood of data.

---

### How a Smurf Attack Works

The attack involves three main components: the **attacker**, an **intermediary network** (the amplifier), and the **target
victim**.

1.  **Spoofing:** The attacker creates a fake ICMP "echo request" (a ping) packet. They change the "source IP" of this packet
    to the IP address of the intended victim.
2.  **Broadcasting:** The attacker sends this spoofed packet to an IP broadcast address of a large intermediary network.
3.  **Amplification:** Every single device on that intermediary network receives the request and, believing it came from the
    victim, sends an "echo reply" back to the victim’s IP address.
4.  **The Flood:** If the intermediary network has 500 devices, one packet from the attacker results in 500 packets hitting
    the victim. This quickly consumes all available bandwidth, crashing the victim’s system.

---

### Key Characteristics

- **Amplification Factor:** The effectiveness depends on the number of devices on the intermediary network. The more devices
  there are, the more "amplified" the attack becomes.
- **Resource Exhaustion:** It doesn't usually target software vulnerabilities; instead, it aims to exhaust the victim's
  network bandwidth and processing power.
- **Difficulty to Trace:** Because the traffic comes from many legitimate devices on an intermediary network, it is difficult
  to trace the attack back to the original source.

---

### Is it still common?

Today, Smurf attacks are **largely a legacy threat**. Most modern networks and operating systems have been patched to prevent
them:

- **Broadcast Disabling:** Routers are now configured by default to ignore IP broadcast requests from outside the network.
- **Ignoring Echo Requests:** Modern systems are often configured not to respond to ICMP requests sent to broadcast
  addresses.

However, the **logic** of the Smurf attack—using amplification and spoofing—is still used in modern DDoS attacks, such as
**DNS Amplification** or **NTP Amplification** attacks.

---

### How to Mitigate Smurf Attacks

- **Configure Routers:** Set routers and firewalls to deny packets sent to broadcast addresses.
- **Host Configuration:** Disable the ability of individual hosts to respond to ICMP broadcast requests.
- **Upstream Filtering:** Work with Internet Service Providers (ISPs) to filter out spoofed traffic before it enters the
  local network.
