Since you're diving into the darker corners of networking, it’s fascinating to see how hackers turn a "helper" feature like
fragmentation into a weapon. Because fragmentation forces a target to wait and "think" about how to put pieces back together,
it creates unique opportunities for mischief.

Here are the primary ways fragmentation is exploited:

---

## 1. IP Fragment Overlap (Teardrop Attack)

This is a classic Denial of Service (DoS) attack. The attacker sends fragmented packets with **overlapping offsets**.

- **How it works:** The attacker sends two fragments. Fragment A says it contains data from bytes 0 to 100. Fragment B says
  it contains data from bytes **50** to 150.
- **The Result:** When the receiving operating system tries to reassemble them, it gets confused by the overlapping data.
  Older or unpatched systems (like Windows 3.1/95 or early Linux kernels) didn't know how to handle the overlap and would
  crash, blue-screen, or reboot.

---

## 2. Tiny Fragment Attack (IDS/Firewall Evasion)

This attack aims to sneak malicious data past a firewall or Intrusion Detection System (IDS).

- **How it works:** The attacker forces the first fragment to be so tiny that it only contains a piece of the Layer 4 header
  (like half of the TCP port information).
- **The Evasion:** Many firewalls only inspect the first fragment because that's usually where the "permit" or "deny" rules
  (based on port numbers) are applied. If the port number is split across two fragments, the firewall might let the first one
  through because it doesn't see a "forbidden" port, and let the second one through because it's just a "fragment."
- **The Result:** The fragments reassemble _behind_ the firewall, completing the forbidden connection.

---

## 3. IP Reassembly Buffer Flooding

This is a resource exhaustion attack.

- **How it works:** The attacker sends a massive wave of initial fragments (MF=1) but **never sends the final fragment**
  (MF=0).
- **The Result:** The victim's server must set aside memory (a buffer) to hold these fragments while it waits for the rest of
  the message. If the attacker does this thousands of times, the server runs out of memory (RAM) trying to "remember" all
  these incomplete conversations, eventually crashing or slowing to a crawl.

---

## 4. Overlapping Fragment Attack (Data Substitution)

This is a more sophisticated version of the Teardrop, used to bypass security filters.

- **How it works:**
  1.  The attacker sends Fragment 1 with "Safe" data. The Firewall inspects it and says "Okay."
  2.  The attacker sends Fragment 2 with an offset that **overwrites** part of Fragment 1 with "Malicious" data.
- **The Result:** Some operating systems will prioritize the _new_ data over the _old_ data during reassembly. The firewall
  saw the safe version, but the end-host executed the malicious version.

---

## Summary of Defense Mechanisms

| Attack Type         | Defense Strategy                                                                                                              |
| :------------------ | :---------------------------------------------------------------------------------------------------------------------------- |
| **Teardrop**        | Modern OS kernels now check for overlapping offsets and discard malformed packets.                                            |
| **Tiny Fragment**   | Firewalls now use **Virtual Reassembly**, meaning they put the packet together themselves before deciding to let it in.       |
| **Buffer Flooding** | Shortening the "Reassembly Timeout" (the time a server waits for missing pieces) and limiting buffer sizes.                   |
| **Evasion**         | Normalizing traffic at the edge—forcing all packets to follow strict formatting rules before they enter the internal network. |

It's a cat-and-mouse game: developers build a feature for flexibility, and attackers find a way to use that flexibility to
create chaos.
