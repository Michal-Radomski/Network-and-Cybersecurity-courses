Since you’re looking at this from a routing perspective, the goal is to prevent **IP Spoofing**—where an attacker sends
packets with a fake source IP address to impersonate a trusted host or hide their identity during a DoS attack.

Routers are the "gatekeepers" of the network, and they use several techniques to verify that a packet is actually coming from
where it claims to be.

---

## 1. uRPF (Unicast Reverse Path Forwarding)

This is the most common and effective routing-based defense. Think of it as a "sanity check" for every incoming packet.

- **How it works:** When a router receives a packet, it looks at the **Source IP**. It then checks its own routing table:
  _"If I wanted to send a packet back to this Source IP, would I use the same interface this packet just arrived on?"_
- **The Logic:** If the router would send traffic out of Interface A to reach that IP, but the packet arrived on Interface B,
  the router assumes the packet is spoofed and drops it.

### uRPF Modes:

- **Strict Mode:** The packet must arrive on the _best_ return path. (Best for "stub" networks with only one way in/out).
- **Loose Mode:** The Source IP must simply exist in the routing table, regardless of which interface it arrived on. (Best
  for multi-homed networks with multiple paths).

---

## 2. Ingress and Egress Filtering (BCP 38)

**BCP 38** (Best Current Practice) is a famous internet guideline that suggests every ISP should filter traffic at the
network edge.

- **Ingress Filtering (Incoming):** A router at the edge of a network should only allow packets into the network if their
  destination IP belongs to that network.
- **Egress Filtering (Outgoing):** A router should only allow packets to _leave_ the network if their **Source IP** belongs
  to the internal network range.
  - _Example:_ If your internal network is `192.168.1.0/24`, and a router sees a packet leaving with a source of `8.8.8.8`,
    it knows that packet is a lie and kills it.

---

## 3. ACLs (Access Control Lists)

On a more granular level, administrators can manually define which IPs are allowed on which ports.

- **Standard ACLs:** Placed as close to the source as possible. They check the Source IP against a "Permit" or "Deny" list.
- **The Downside:** Unlike uRPF, ACLs are manual. If your network changes, you have to update your ACLs, or you might
  accidentally block legitimate users.

---

## 4. IP Source Guard (IPSG)

This is typically a Layer 2/3 feature found on high-end switches and routers to prevent spoofing within a local network.

- **How it works:** It builds a "binding table" using **DHCP Snooping**. It learns which MAC address and IP address are
  assigned to which physical port.
- **The Enforcement:** If a device on Port 5 tries to send a packet with an IP address that wasn't assigned to Port 5, the
  hardware drops the traffic instantly.

---

## Summary Comparison

| Technique           | Where it's used       | Best For...                                              |
| :------------------ | :-------------------- | :------------------------------------------------------- |
| **uRPF**            | Core/Edge Routers     | Automatically detecting spoofed traffic at scale.        |
| **BCP 38**          | ISP/Enterprise Edge   | Preventing your network from being used in DDoS attacks. |
| **ACLs**            | Specific Interfaces   | High-security environments with static IP ranges.        |
| **IP Source Guard** | Access Layer Switches | Preventing internal "insider" spoofing.                  |

### Why isn't spoofing extinct?

Even though these tools exist, not every ISP or organization implements them. If an attacker sits on a network that doesn't
use BCP 38 or uRPF, they can send "garbage" packets to the entire world.
