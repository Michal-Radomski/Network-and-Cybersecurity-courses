Think of these headers as the different layers of an envelope. The **IP header** is the outer envelope used by the postal
service to get the letter to the right house, while the **TCP** or **UDP** headers are the instructions inside that tell the
residents which person (or application) the message is for and how to handle it.

---

## 1. The IP Header (The Delivery Map)

Operating at **Layer 3 (Network)**, the IP header’s primary job is routing. It doesn't care about the content of the message;
it only cares about the "To" and "From" addresses.

- **Version:** Usually IPv4 or IPv6.
- **TTL (Time to Live):** A hop counter that prevents packets from circling the internet forever.
- **Protocol:** Tells the receiver what’s inside (e.g., 6 for TCP, 17 for UDP).
- **Source/Destination IP Addresses:** The 32-bit (IPv4) addresses of the sender and receiver.

---

## 2. The TCP Header (The Certified Letter)

Operating at **Layer 4 (Transport)**, TCP is "connection-oriented." It’s designed for reliability. If a packet goes missing,
TCP notices and asks for a redo.

- **Source/Destination Ports:** Directs the data to a specific app (e.g., Port 443 for HTTPS).
- **Sequence Number:** Used to put packets back in the correct order.
- **Acknowledgment Number:** Confirms that previous data was received.
- **Flags (SYN, ACK, FIN):** Manage the "handshake" and connection state.
- **Window Size:** Controls flow (how much data can be sent before waiting for an ACK).

---

## 3. The UDP Header (The Postcard)

Also at **Layer 4**, UDP is "connectionless." It’s the "send and forget" protocol. It’s lightweight and fast, making it
perfect for gaming or streaming where a dropped frame is better than a delayed one.

- **Source/Destination Ports:** Same as TCP.
- **Length:** The total size of the UDP segment.
- **Checksum:** A basic error check (though UDP doesn't fix errors, it just discards bad packets).

---

## Comparison Table: IP vs. TCP vs. UDP

| Feature           | IP Header                | TCP Header                | UDP Header              |
| :---------------- | :----------------------- | :------------------------ | :---------------------- |
| **Layer**         | Layer 3 (Network)        | Layer 4 (Transport)       | Layer 4 (Transport)     |
| **Primary Goal**  | Routing & Addressing     | Reliability & Ordering    | Speed & Efficiency      |
| **Standard Size** | 20 Bytes (minimum)       | 20 Bytes (minimum)        | 8 Bytes (fixed)         |
| **Key Field**     | IP Addresses             | Sequence Numbers/Flags    | Port Numbers            |
| **Reliability**   | Best-effort (unreliable) | Highly reliable (Retries) | Unreliable (No retries) |

---

## How they work together

When you send a message, it’s like a Russian Nesting Doll:

1.  **Your Data** is placed inside a **TCP or UDP Header** (creating a Segment).
2.  That Segment is then placed inside an **IP Header** (creating a Packet).
3.  The IP header gets it to the right computer, and the TCP/UDP header gets it to the right software on that computer.
