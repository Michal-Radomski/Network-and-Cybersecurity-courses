The **Network Layer** (Layer 3) is the "Post Office" of the OSI model. While the Data Link Layer (Layer 2) handles delivery
within a single room or building, the Network Layer is responsible for **routing** data across different networks,
potentially across the entire world.

Its primary job is to find the best path for data to travel from a source on one network to a destination on another.

---

## 1. Core Responsibilities

The Network Layer performs three critical functions to ensure data moves across the internet:

- **Logical Addressing:** It assigns unique IP addresses to every device. Unlike MAC addresses (which are permanent and
  physical), IP addresses are logical and change based on where you are connected.
- **Routing:** This is the "brain" work. Routers at this layer maintain tables to decide which "neighbor" to pass a packet to
  so it eventually reaches its destination.
- **Packetization:** It takes the "Segments" from the Transport Layer (Layer 4) and encapsulates them into **Packets**.

---

## 2. IP Addresses vs. MAC Addresses

To understand Layer 3, you must understand how it differs from Layer 2.

| Feature          | Data Link Layer (L2)              | Network Layer (L3)                |
| :--------------- | :-------------------------------- | :-------------------------------- |
| **Unit of Data** | Frame                             | Packet                            |
| **Addressing**   | Physical (MAC Address)            | Logical (IP Address)              |
| **Scope**        | Local Network (Same Wi-Fi/Switch) | Global Network (Internet/Routers) |
| **Device**       | Network Switch                    | Router                            |

---

## 3. Key Protocols of the Network Layer

The Network Layer is dominated by a few heavy hitters that keep the internet running:

- **IP (Internet Protocol):** The most common protocol. It defines the structure of the packet and the addressing scheme
  (IPv4 and IPv6).
- **ICMP (Internet Control Message Protocol):** Used for "network diagnostics." When you use the `ping` command to see if a
  server is alive, you are using ICMP.
- **IGMP (Internet Group Management Protocol):** Used for multicasting (sending data to a group of receivers simultaneously).

---

## 4. How Routing Works

Routers don't know the exact "map" of every single computer on Earth. Instead, they use **Routing Protocols** (like OSPF,
BGP, or RIP) to learn about available paths.

When a packet arrives at a router, the router looks at the **Destination IP**, checks its **Routing Table**, and sends the
packet out the best interface. If the destination is not in its local network, it hands it off to the "Next Hop" (another
router).

---

## 5. Packet Structure

An IP packet consists of two parts:

1.  **The Header:** Contains the source IP, destination IP, version (IPv4/v6), and TTL (Time to Live).
2.  **The Payload:** The actual data being carried (usually a TCP or UDP segment).

> **What is TTL?** > The **Time to Live** is a counter in the packet header. Every time a packet passes through a router, the
> TTL is reduced by 1. If the TTL reaches 0, the packet is deleted. This prevents packets from looping forever if there is a
> mistake in the network path.
