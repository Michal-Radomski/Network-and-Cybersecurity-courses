At its core, **routing** is the process of selecting a path for traffic in a network or between or across multiple networks.
If IP addresses are the "phone numbers" of the internet, routing is the global system of switchboards that ensures your data
doesn't just wander aimlessly.

---

## 1. The Core Concept: The Routing Table

Every router (from your home Wi-Fi box to the massive machines at Google) maintains a **Routing Table**. This is a database
of "maps" that tells the router: _"If a packet is going to Destination X, send it out through Interface Y."_

A typical routing table entry contains:

- **Destination Network:** The IP range the packet is trying to reach.
- **Next Hop:** The IP address of the next router in the chain.
- **Metric/Cost:** A value used to decide the "best" path if multiple exist (lower is usually better).

---

## 2. Static vs. Dynamic Routing

How does a router learn where to send things? There are two main ways:

### Static Routing

A human administrator manually types the routes into the router.

- **Pros:** Very secure, low overhead (no processing needed to "talk" to other routers).
- **Cons:** Doesn't scale. If a cable is cut, a human has to manually fix the route.

### Dynamic Routing

Routers use **Routing Protocols** to talk to each other and "learn" the topology of the network automatically.

- **Pros:** Adaptive. If a link goes down, the routers automatically find a detour.
- **Cons:** Consumes CPU and bandwidth to maintain the "conversation" between routers.

---

## 3. Levels of Routing (IGP vs. EGP)

Not all routing happens at the same scale. The internet is divided into **Autonomous Systems (AS)**—large networks managed by
a single entity (like an ISP or a tech giant).

### Interior Gateway Protocols (IGP)

Used for routing **inside** a single company or ISP.

- **OSPF (Open Shortest Path First):** Uses a "Link State" algorithm to build a complete map of the local network.
- **EIGRP:** A Cisco-proprietary protocol known for being very fast.

### Exterior Gateway Protocols (EGP)

Used to route data **between** different Autonomous Systems.

- **BGP (Border Gateway Protocol):** The "Protocol of the Internet." It manages how traffic moves between ISPs and countries.

---

## 4. The Routing Decision Process

When a packet arrives, the router follows a strict logic:

1.  **Longest Prefix Match:** The router looks for the most specific match in its table. If one entry is for `192.168.1.0/24`
    and another is for `192.168.1.5/32`, it will pick the `/32` because it's more specific.
2.  **Administrative Distance:** If it learns about a destination from two different protocols (e.g., OSPF vs. RIP), it picks
    the one it "trusts" more.
3.  **Metric:** If everything else is equal, it picks the path with the lowest cost (fewest hops or fastest speed).

---

## 5. Routing vs. Switching

People often confuse these two, but they happen at different layers of the OSI model:

| Feature        | Switching                | Routing                          |
| :------------- | :----------------------- | :------------------------------- |
| **OSI Layer**  | Layer 2 (Data Link)      | Layer 3 (Network)                |
| **Identifier** | MAC Addresses            | IP Addresses                     |
| **Scope**      | Local Area Network (LAN) | Wide Area Network (WAN/Internet) |
| **Device**     | Network Switch           | Router                           |

> **Pro-Tip:** Switches find **hosts** (your laptop, your printer); Routers find **networks** (your office, your city, the
> internet).
