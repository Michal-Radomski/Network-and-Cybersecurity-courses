The **Internet Control Message Protocol (ICMP)** is the "troubleshooter" of the Internet Protocol suite. While IP is
responsible for moving data from point A to point B, ICMP is responsible for reporting back when things go wrong along that
path.

It doesn’t carry user data (like your emails or web pages); instead, it carries administrative and error messages between
network devices.

---

## 1. How ICMP Fits in the Stack

Technically, ICMP is a **Layer 3 (Network Layer)** protocol. However, it’s a bit of a hybrid: it is encapsulated directly
inside an IP packet, just like a TCP or UDP segment would be, but it doesn't use port numbers.

---

## 2. Common ICMP Message Types

ICMP uses a **Type** and a **Code** system to communicate specific issues. Here are the ones you will encounter most often:

| Type   | Name                    | Purpose                                                              |
| :----- | :---------------------- | :------------------------------------------------------------------- |
| **0**  | Echo Reply              | The "Pong" in a Ping request.                                        |
| **3**  | Destination Unreachable | The packet can't get there (Network/Host/Port down).                 |
| **5**  | Redirect                | A router tells the host: "There is a better path, use that instead." |
| **8**  | Echo Request            | The "Ping" sent to check if a host is alive.                         |
| **11** | Time Exceeded           | The TTL hit 0 (used by Traceroute).                                  |

---

## 3. The Core Tools: Ping and Traceroute

ICMP is the engine behind the two most famous networking commands:

### Ping (Echo Request/Reply)

When you run `ping 8.8.8.8`, your computer sends an **ICMP Type 8** packet. If the destination is up and configured to
respond, it sends back an **ICMP Type 0**. This measures **latency** (how long it takes) and **availability**.

### Traceroute (Time Exceeded)

As we discussed earlier, Traceroute sends packets with low TTLs. When a router drops the packet, it sends back an **ICMP Type
11 (Time Exceeded)**. Traceroute uses the source IP of these error messages to build a list of every router in the path.

---

## 4. The Anatomy of an ICMP Packet

An ICMP header is very small and efficient, usually consisting of:

- **Type (8 bits):** The high-level category (e.g., Error vs. Informational).
- **Code (8 bits):** The specific subtype (e.g., _Why_ is the destination unreachable?).
- **Checksum (16 bits):** To ensure the header wasn't corrupted in transit.
- **Data Section:** For error messages, this often contains a copy of the **IP header** of the packet that caused the error,
  so the sender knows which message failed.

---

## 5. ICMP and Security

Because ICMP is so useful for discovery, it is also a target for attackers:

- **Ping of Death:** Sending an oversized ICMP packet that exceeds the maximum IP size (65,535 bytes), causing older systems
  to crash during reassembly.
- **ICMP Redirect Attack:** An attacker sends a fake Type 5 message to a host, tricking it into sending all its traffic
  through the attacker’s machine (Man-in-the-Middle).
- **Smurf Attack:** A DoS attack where the attacker sends a ping to a broadcast address with a spoofed source IP (the
  victim's IP), causing hundreds of devices to "reply" to the victim at once.

> **Blue Team Note:** Many modern firewalls are configured to block "Echo Requests" (Ping) from the outside world. This makes
> the network "stealthy," though it also makes troubleshooting much harder.
