In Linux, the `ip route` command is the modern standard for managing the routing table. It is part of the **iproute2**
package, which replaced the now-deprecated `route` and `netstat -r` commands.

If the routing table is the "map" we discussed earlier, `ip route` is the tool you use to read, draw, and edit that map.

---

## 1. Viewing the Routing Table

To see how your computer currently thinks it should send data, simply type: `ip route show` (or just `ip route`)

**Example Output:**

```text
default via 192.168.1.1 dev eth0 proto dhcp metric 100
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.45 metric 100
```

### Breaking down the components:

- **default:** This is the "catch-all" route. If a destination doesn't match any other specific rule, it goes here.
- **via 192.168.1.1:** The IP address of the router (Gateway).
- **dev eth0:** The specific hardware interface (Ethernet card) the data will exit through.
- **proto kernel:** Indicates this route was created automatically by the system.
- **metric:** The "cost." If there are two routes to the same place, Linux chooses the one with the lower metric.

---

## 2. Common Management Commands

You will often use this command to fix connectivity issues or redirect traffic through a VPN or specific gateway.

### Adding a Static Route

If you want to reach a specific network (e.g., `10.0.0.0/24`) through a specific gateway (`192.168.1.254`):
`sudo ip route add 10.0.0.0/24 via 192.168.1.254`

### Deleting a Route

If a route is incorrect or no longer needed: `sudo ip route del 10.0.0.0/24`

### Changing the Default Gateway

If you want to change your primary exit point for the entire internet: `sudo ip route add default via 192.168.1.1`

---

## 3. The "Get" Feature (Troubleshooting)

One of the most useful features for learners is `ip route get`. It tells you exactly which path a packet **would** take to
reach a specific IP without actually sending a ping.

**Command:** `ip route get 8.8.8.8`

**Result:** `8.8.8.8 via 192.168.1.1 dev eth0 src 192.168.1.45 uid 1000`

> This confirms that to reach Google (8.8.8.8), the traffic will go through your local router using your local IP
> `192.168.1.45`.

---

## 4. Comparison: Old vs. New

If you see older tutorials using the `route` command, here is how they translate to the modern `ip` tool:

| Goal            | Old Command (Deprecated) | New Command (iproute2) |
| :-------------- | :----------------------- | :--------------------- |
| **View Routes** | `route -n`               | `ip route`             |
| **Add Route**   | `route add -net ...`     | `ip route add ...`     |
| **Del Route**   | `route del -net ...`     | `ip route del ...`     |
| **Flush Table** | N/A                      | `ip route flush cache` |

---

### A Quick Warning

Commands run with `ip route` are **temporary**. If you reboot your computer, these manual changes will disappear. To make
them permanent, you usually have to edit configuration files like `/etc/network/interfaces` or use a tool like **Netplan**
(on Ubuntu) or **NetworkManager**.
