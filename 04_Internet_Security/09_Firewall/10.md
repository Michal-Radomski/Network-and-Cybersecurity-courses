Using SSH and VPN tunnels to bypass firewalls is a common technique used by developers and network administrators to access
restricted resources or secure their communications on untrusted networks. While these methods are powerful, they are
essentially ways of "wrapping" your traffic so the firewall cannot see what is inside.

---

## 1. SSH Tunneling (The "Swiss Army Knife")

SSH (Secure Shell) is more than just a remote terminal; it can act as a transport layer for other types of traffic. Because
SSH traffic is encrypted, a firewall can see that you are connected to an SSH server, but it cannot see the web traffic or
database commands hidden inside.

### Local Port Forwarding

You use this when you want to access a resource that the firewall blocks, but you have SSH access to a machine that _can_ see
that resource.

- **The Command:** `ssh -L 8080:target-website.com:80 user@ssh-server`
- **How it works:** You tell your local computer to listen on port `8080`. Anything you send to `localhost:8080` is
  encrypted, sent through the SSH "tunnel," and popped out by the SSH server to the destination.

### Dynamic Port Forwarding (SOCKS Proxy)

This is the most effective way to bypass a general web filter.

- **The Command:** `ssh -D 1080 user@ssh-server`
- **How it works:** This turns your SSH client into a SOCKS proxy. If you configure your browser to use `localhost:1080` as a
  proxy, all your browsing traffic is tunneled through the SSH connection. To the firewall, it looks like one single,
  continuous SSH session.

---

## 2. VPN Tunneling (The "Cloak")

While SSH tunnels are usually application-specific, a VPN (Virtual Private Network) tunnels **all** network traffic from your
device.

- **Layer 3 Tunneling:** VPNs like OpenVPN or WireGuard create a virtual network interface on your machine. All packets are
  encapsulated (wrapped) inside another protocol (like UDP or TCP).
- **Bypassing Deep Packet Inspection (DPI):** Modern firewalls use DPI to spot VPN signatures. To bypass this, "stealth" VPNs
  use techniques like:
- **Obfuscation:** Making VPN traffic look like standard HTTPS (web) traffic.
- **TCP Over Port 443:** Running the VPN on the same port used by secure websites, making it very difficult for firewalls to
  block without breaking the internet.

---

## 3. Comparison of Bypassing Methods

| Feature           | SSH Tunneling                        | VPN Tunneling                         |
| ----------------- | ------------------------------------ | ------------------------------------- |
| **Scope**         | Specific apps/ports.                 | Entire system traffic.                |
| **Ease of Setup** | Fast (requires only an SSH account). | Requires VPN software/server setup.   |
| **Visibility**    | Easy to spot (usually port 22).      | Can be obfuscated to look like HTTPS. |
| **Performance**   | High overhead for web browsing.      | Optimized for high-speed throughput.  |

---

## 4. How Firewalls Fight Back

Firewall administrators aren't defenseless against tunneling. They use several strategies:

- **Egress Filtering:** Only allowing outgoing traffic on specific ports (e.g., only 80 and 443), which blocks the default
  SSH port (22).
- **Traffic Analysis:** Even if traffic is encrypted, a firewall can look at packet sizes and timing to guess if an SSH
  connection is actually being used for web browsing.
- **IP Blacklisting:** Blocking the IP addresses of known commercial VPN providers and SSH relay services.
