Both **Port Forwarding** and **Port Triggering** are techniques used to allow external traffic from the internet to reach
specific devices on a private local network. Because your router acts as a NAT (Network Address Translation) firewall, it
normally blocks all unsolicited incoming traffic. These two methods create "holes" in that firewall, but they do so in very
different ways.

---

## 1. Port Forwarding (Static)

Port Forwarding is a **permanent** rule. You tell the router: _"Any traffic that hits our public IP on Port X must always be
sent to Device A on the local network."_

- **How it works:** You manually configure a map in the router settings (e.g., External Port 8080 → Internal IP 192.168.1.10,
  Port 80).
- **Availability:** The port is open 24/7, even if the device is offline.
- **Requirement:** The internal device **must** have a Static IP address. If the device’s IP changes, the rule breaks.
- **Security:** Less secure, as the port is always open, providing a permanent "door" for potential attackers.

**Best for:** Persistent servers like a Plex Media Server, a Minecraft server, or a web server.

---

## 2. Port Triggering (Dynamic)

Port Triggering is a **temporary** and **automated** version of port forwarding. It is "event-based."

- **How it works:** The router watches for **outbound** traffic from any local device on a specific "trigger" port. When it
  sees that traffic, it opens an "incoming" port and remembers which device made the request. Once the activity stops, the
  router closes the port again.
- **Availability:** The port is only open when it is actually being used.
- **Requirement:** Does **not** require a static IP. Any device on the network can "trigger" the rule.
- **Security:** More secure than forwarding because the "door" is locked until an internal device specifically asks to open
  it, and it closes automatically after a timeout.

**Best for:** Online gaming (where a game might send a request out and need a range of ports opened to receive data) or video
conferencing applications.

---

## Key Comparison

| Feature              | Port Forwarding                 | Port Triggering                               |
| -------------------- | ------------------------------- | --------------------------------------------- |
| **Duration**         | Permanent (always open)         | Temporary (opens on demand)                   |
| **Setup**            | Manual configuration            | Manual rule, but automated action             |
| **Internal IP**      | Requires Static IP              | Works with Dynamic IP (DHCP)                  |
| **Multiple Devices** | Tied to **one** specific device | Can be used by **any** device (one at a time) |
| **Security**         | Lower (static opening)          | Higher (dynamic opening)                      |

---

## Summary: Which one should you use?

- Use **Port Forwarding** if you want to host something that people from the outside world need to access at any time (like a
  Home Assistant setup or a private cloud).
- Use **Port Triggering** if you are playing a game or using a specific app that needs "help" getting through the firewall,
  but you don't want a permanent hole in your security.
