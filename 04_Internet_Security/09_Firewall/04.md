On Ubuntu, you don't really "install" Netfilter because it is built directly into the Linux kernel. However, you can check if
the kernel modules are active and verify the presence of the user-space tools (`iptables` and `nftables`).

Here is how you check each component:

---

## 1. Checking Netfilter (The Kernel Core)

Since Netfilter is a set of kernel modules, you check for its presence by looking at the loaded modules. Most modern systems
load these on demand.

Run this command to see if the core Netfilter modules are currently loaded:

```bash
lsmod | grep nf_tables

```

Or to see the classic connection tracking module:

```bash
lsmod | grep nf_conntrack

```

> **Note:** If these return nothing, it doesn't mean Netfilter is missing; it often just means no firewall rules have been
> set yet to trigger the module loading.

---

## 2. Checking Iptables

Ubuntu usually comes with `iptables` installed by default. To check the version and see if the binary exists, run:

```bash
iptables --version

```

To see if there are any active rules (this requires root privileges):

```bash
sudo iptables -L

```

### The "Legacy" vs "Nft" Distinction

On newer Ubuntu versions (20.04 and later), `iptables` is often just a "wrapper" that translates commands to `nftables`. You
can check which "backend" your iptables is using with:

```bash
sudo update-alternatives --display iptables

```

It will likely show `iptables-nft`, meaning your "iptables" commands are actually being handled by the `nftables` engine.

---

## 3. Checking Nftables

`nftables` is the modern standard for Ubuntu. To check if the utility is installed, run:

```bash
nft --version

```

If it is **not** installed, you will get a "command not found" error. You can install it using:
`sudo apt update && sudo apt install nftables`

To see if there are any active `nftables` rules:

```bash
sudo nft list ruleset

```

---

## Summary Checklist Table

| Component          | Command to Check            | Expected Output              |
| ------------------ | --------------------------- | ---------------------------- |
| **Netfilter**      | `lsmod                      | grep nf`                     |
| **Iptables**       | `iptables --version`        | `iptables v1.8.x`            |
| **Nftables**       | `nft --version`             | `nftables v0.9.x` (or newer) |
| **Service Status** | `systemctl status nftables` | `active` or `inactive`       |

---

### Pro-Tip: Checking the Service

Even if the tools are installed, the firewall might not be running. On Ubuntu, check the status of the nftables service:

```bash
sudo systemctl status nftables

```

If it says **"enabled"** and **"active"**, your ruleset is being applied automatically at boot.
