Border Gateway Protocol (BGP) is the backbone of the internet's routing infrastructure, but its complexity makes it prone to
misconfigurations, "route leaks," and "hijacks." To manage this, network engineers rely on a suite of tools for monitoring,
debugging, and visualization.

---

## 1. Analysis and Monitoring Platforms

These platforms provide a global view of how your prefixes (IP ranges) are being seen by the rest of the world.

- **BGPStream:** A free, open-source framework for analyzing historical and real-time BGP data. It is essential for detecting
  large-scale routing outages or hijacks.
- **Cisco BGPStream:** Not to be confused with the open-source version, this is a public service that provides a live feed of
  BGP incidents (leaks and hijacks) globally.
- **BGP.he.net (Hurricane Electric):** Perhaps the most popular web-based tool. It allows you to search for any AS
  (Autonomous System) or IP prefix to see its upstream providers, downstream peers, and connectivity history.
- **Kentik / ThousandEyes:** Enterprise-grade SaaS platforms that offer deep visibility into BGP path changes and their
  impact on application performance.

---

## 2. Looking Glasses and Route Servers

When you want to see how a specific router in a different part of the world views your network, you use a **Looking Glass
(LG)**.

- **PeeringDB:** The "LinkedIn of Peering." It’s a database where networks list their peering policies and Looking Glass
  URLs.
- **Route Views Project / RIPE RIS:** These projects provide access to global BGP routing tables from dozens of locations.
  You can telnet into their collectors to run `show ip bgp` commands as if you were sitting at a major internet exchange.

---

## 3. Visualization Tools

Understanding BGP often requires seeing the "AS Path"—the chain of networks data travels through.

- **RIPE Stat:** A web-based interface that provides "Routing History" widgets and "BGPlay," which creates an animated graph
  showing how a specific prefix has been announced or withdrawn over time.
- **Peering Toolbox:** A community-led site that helps new network operators understand how to set up peering and use BGP
  safely.

---

## 4. Security and Hygiene Tools

With the rise of BGP hijacking, these tools focus on **RPKI (Resource Public Key Infrastructure)** to cryptographically sign
routing announcements.

- **Routinator:** A popular RPKI relying party software (validator) that downloads and verifies RPKI certificates so your
  routers can drop "invalid" (likely hijacked) routes.
- **BGPalerter:** A self-hosted tool that monitors your prefixes and sends alerts (via Slack, Email, Telegram) if it detects
  a hijack, a visibility loss, or an RPKI expiration.
- **MANRS (Mutually Agreed Norms for Routing Security):** Not a software tool, but an initiative and a set of "observatory"
  tools that track whether an ISP is following security best practices.

---

## 5. Command Line Utilities

For the hands-on engineer, these CLI tools are indispensable:

| Tool                 | Purpose                                                                                         |
| -------------------- | ----------------------------------------------------------------------------------------------- |
| **mtr**              | A combination of `ping` and `traceroute` that shows packet loss at every hop in the BGP path.   |
| **bgpq4**            | Generates BGP filter lists (prefix-lists) based on data from Internet Routing Registries (IRR). |
| **Bird / FRRouting** | Open-source routing suites used to turn Linux/Unix servers into high-performance BGP speakers.  |
| **whois**            | Used to query IRR databases to see who owns an AS number or IP block.                           |

---

### Pro Tip: The "BGP Playbook"

If you are troubleshooting a connectivity issue, the standard workflow is:

1. Check **BGP.he.net** to see if your route is visible.
2. Use **RIPE Stat** to see if there was a recent "flap" (withdrawal/re-announcement).
3. Log into a **Looking Glass** near the affected user to see the specific path the traffic is taking.
