The `whois` command is a standard networking tool used to query databases that store the registered users or assignees of an
Internet resource, such as a domain name or an IP address block.

Essentially, it’s the "public record" lookup for the internet.

### 1. Installation

Most Linux distributions don't include `whois` by default. You can install it using your package manager:

- **Debian/Ubuntu/Mint:** `sudo apt install whois`
- **RHEL/CentOS/Fedora:** `sudo dnf install whois`
- **Arch Linux:** `sudo pacman -S whois`

---

### 2. Basic Usage

The simplest way to use the command is to provide a domain name or an IP address as the argument.

#### Lookup a Domain

To see registration details, name servers, and expiration dates for a website:

```bash
whois google.com

```

#### Lookup an IP Address

To find out which organization owns a specific IP (useful for tracking the source of web traffic or potential attacks):

```bash
whois 8.8.8.8

```

---

### 3. Common Options and Flags

While the basic command works for 90% of cases, these flags help refine your search:

| Flag | Description                                                               | Example                                       |
| ---- | ------------------------------------------------------------------------- | --------------------------------------------- |
| `-h` | Specify a specific WHOIS server to query.                                 | `whois -h whois.verisign-grs.com example.com` |
| `-p` | Specify a port (default is 43).                                           | `whois -p 43 google.com`                      |
| `-i` | Search for objects by their "inverse" attribute (usually for IP lookups). | `whois -i mnt-by MAINT-AS1234`                |
| `-H` | Hide the legal disclaimers often attached to the output.                  | `whois -H google.com`                         |

---

### 4. How to Read the Output

When you run a `whois` query, the output is typically broken into several key sections:

- **Registrar Info:** Who the domain was bought from (e.g., GoDaddy, Namecheap).
- **Dates:** When the domain was created, last updated, and when it expires.
- **Registrant Contact:** Information about the owner. **Note:** Nowadays, this is often masked as "Redacted for Privacy" due
  to GDPR regulations.
- **Name Servers:** The servers that tell the internet where the website is hosted (e.g., `ns1.google.com`).

---

### 5. Advanced Tips

- **Filter results:** Since the output can be long, pipe it into `grep` to find specific info:

```bash
whois google.com | grep -i "expiry date"

```

- **Check for Availability:** If a `whois` search returns "No match found," it usually means the domain name is currently
  available for registration.

> **Pro Tip:** If you are performing many lookups in a row, some servers might temporarily block your IP address to prevent
> scraping. Always be respectful of the server's rate limits.
