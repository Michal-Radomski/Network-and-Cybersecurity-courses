A BGP speaker is any networking device (typically a core router or multilayer switch) that runs the Border Gateway Protocol
(BGP), establishes peer relationships, and actively exchanges routing information with other BGP-enabled devices. [1, 2, 3]

## Core Functions

- Establishes Peering: Initiates TCP connections on port 179.
- Exchanges Routes: Sends Network Layer Reachability Information (NLRI).
- Applies Policies: Filters routes using custom path attributes.
- Selects Paths: Runs the BGP decision algorithm for best paths.
- Maintains Tables: Stores routes in the BGP routing table (Loc-RIB). [4, 5, 6, 7, 8]

## Peer Types

- Internal BGP (iBGP): Peers reside within the same Autonomous System (AS).
- External BGP (eBGP): Peers reside in different Autonomous Systems. [9, 10, 11, 12]

## Message Types Used

- Open: Establishes the initial neighbor relationship.
- Update: Advertises new routes or withdraws dead routes.
- Keepalive: Verifies the peer connection is still active.
- Notification: Reports errors and closes the peering session. [13, 14, 15, 16, 17]
