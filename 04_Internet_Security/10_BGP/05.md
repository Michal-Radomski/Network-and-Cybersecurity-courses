Border Gateway Protocol (BGP) is often called the "language of the internet," and unlike other routing protocols that just
look for the fastest path, BGP focuses on **policy** and **path attributes**.

Because BGP can see multiple ways to reach a single destination, it uses a strict, step-by-step tie-breaker list called the
**BGP Best Path Selection Algorithm**.

---

## The Pre-Flight Check

Before a route even enters the selection process, it must meet two criteria:

1. **Valid Next Hop:** The router must know how to reach the "next hop" IP address listed for that route.
2. **Synchronization (if applicable):** In older setups, BGP wouldn't advertise a route unless it was also learned via an
   Interior Gateway Protocol (IGP), though this is mostly disabled in modern networks.

---

## The Selection Steps (The Tie-Breakers)

BGP evaluates attributes in this specific order. As soon as one path "wins" a step, the process stops, and that path is
chosen.

### 1. Weight (Cisco Specific)

- **Highest wins.**
- This is local to the router and isn't advertised to neighbors. It’s a "manual override" for a specific router.

### 2. Local Preference

- **Highest wins.**
- This tells the entire Autonomous System (AS) which path to use to **exit** the network.

### 3. Locally Originated

- Routes generated by the router itself (via `network` or `aggregate` commands) are preferred over routes learned from
  others.

### 4. AS Path Length

- **Shortest wins.**
- BGP counts how many Autonomous Systems a route has to pass through. A shorter path is generally preferred, though many
  admins use "AS Path Prepending" to artificially make a path look longer and less desirable.

### 5. Origin Type

- BGP prefers paths with a lower origin code: **IGP (i)** is preferred over **Incomplete (?)**. (The old "EGP" code is
  effectively obsolete).

### 6. Multi-Exit Discriminator (MED)

- **Lowest wins.**
- This is used to suggest to **external** neighbors which path they should take to **enter** your network.

### 7. eBGP over iBGP

- BGP prefers routes learned from an **External BGP** neighbor over those learned from an **Internal BGP** neighbor.

### 8. IGP Metric to Next Hop

- **Lowest wins.**
- If everything else is a tie, the router looks at its internal routing table (like OSPF or IS-IS) to see which BGP next hop
  is physically "closer."

---

## The Final Tie-Breakers

If the paths are _still_ identical, the router uses these "arbitrary" markers to break the tie:

1. **Oldest Route:** Prefer the most stable (longest-lived) path.
2. **Lowest Router ID:** The neighbor with the lowest IP-based ID wins.
3. **Lowest Neighbor IP Address:** The final "last resort" tie-breaker.

---

### Summary Table

| Attribute            | Preference | Scope             |
| -------------------- | ---------- | ----------------- |
| **Weight**           | Highest    | Local Router Only |
| **Local Preference** | Highest    | Entire AS         |
| **AS Path**          | Shortest   | Global            |
| **MED**              | Lowest     | Between Two ASs   |
| **Next Hop Metric**  | Lowest     | Local AS          |

> **Pro Tip:** Most BGP routing decisions in the real world are decided by **Local Preference** or **AS Path**. If you see a
> network admin "tuning" BGP, they are usually messing with one of those two.
