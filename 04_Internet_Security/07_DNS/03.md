The Domain Name System (DNS) is essentially the "phonebook" of the internet. Since computers communicate via IP addresses
(like `192.0.2.1`) and humans prefer names (like `google.com`), the DNS query process is what bridges that gap.

Here is the step-by-step journey of a DNS query, often called a **DNS Resolution**.

---

## 1. The Local Check (The Browser & OS)

Before your computer even touches the internet, it looks for a shortcut.

- **Browser Cache:** Your browser stores DNS records for a short time for websites you've recently visited.
- **OS Cache / Hosts File:** If the browser doesn't have it, it asks the Operating System. The OS checks its own local cache
  and a file called the **hosts file**.

## 2. The Recursive Resolver (The ISP)

If the address isn't found locally, the computer sends a query to a **Recursive Resolver**. This is typically managed by your
Internet Service Provider (ISP) or a third-party service like Google (8.8.8.8).

> The Resolver's job is to go out and find the answer for you. It acts as the "middleman."

## 3. The Root Nameserver

If the Resolver doesn't have the IP cached, it starts at the top of the hierarchy: the **Root Nameserver**.

- The Root server doesn't know the specific IP address.
- Instead, it points the Resolver toward the correct **TLD (Top-Level Domain) Nameserver** based on the extension (e.g.,
  .com, .org, .edu).

## 4. The TLD Nameserver

The Resolver then asks the TLD Nameserver (for example, the server in charge of all `.com` domains).

- The TLD server still doesn't have the final IP.
- It provides the address of the **Authoritative Nameserver** for that specific domain.

## 5. The Authoritative Nameserver

This is the final stop. The Authoritative Nameserver is the official source of truth for that specific domain (usually
managed by the domain registrar or a hosting provider).

- The Resolver asks for the A Record (the IPv4 address).
- The Authoritative server provides the IP: **"google.com is at 142.250.190.46."**

## 6. Returning the Result

The Recursive Resolver takes that IP address back to your computer.

- **Storage:** The Resolver and your OS cache the result so they don't have to do this again for a while (governed by the
  **TTL** or Time To Live).
- **Connection:** Your browser now has the IP and can initiate an HTTP/HTTPS request to the web server.

---

### Summary Table: Who Does What?

| Component                | Role                                                                   |
| ------------------------ | ---------------------------------------------------------------------- |
| **DNS Recursor**         | The librarian who goes to find the specific book for you.              |
| **Root Server**          | The directory that points to different sections of the library (TLDs). |
| **TLD Server**           | The shelf that contains books for a specific category (.com, .net).    |
| **Authoritative Server** | The actual book that contains the specific information (the IP).       |
