Enabling **DNS over TLS (DoT)** is the final step in securing your DNS. While DNSSEC (which we just turned on) ensures the
data is **authentic**, it doesn't stop your ISP, a hacker on the Wi-Fi, or your router from **seeing** which websites you are
visiting.

DNS over TLS wraps your DNS queries in an encrypted "tunnel" (using the same technology as HTTPS), making your browsing
private.

---

## 1. How to Enable DoT on Ubuntu

Since you are already using Google DNS (`8.8.8.8`), you are in luck—Google's servers already support DoT. You just need to
tell Ubuntu to use it.

### Step 1: Open the configuration

```bash
sudo nano /etc/systemd/resolved.conf

```

### Step 2: Update the file

Look for the `DNSOverTLS` line. Change it to `yes`. Also, ensure your `DNS` and `FallbackDNS` lines point to servers that
support TLS (like Google or Cloudflare).

Your file should look like this:

```text
[Resolve]
DNS=8.8.8.8 8.8.4.4 2001:4860:4860::8888 2001:4860:4860::8844
DNSOverTLS=yes
DNSSEC=yes

```

### Step 3: Restart and Verify

```bash
sudo systemctl restart systemd-resolved

```

Now, run your status command again:

```bash
resolvectl status

```

You should now see **`+DNSOverTLS`** in the protocols list!

## 2. Why do both? (DNSSEC vs. DoT)

It's a common misconception that one replaces the other. They are a team:

| Feature                 | DNSSEC                                    | DNS over TLS (DoT)                                 |
| ----------------------- | ----------------------------------------- | -------------------------------------------------- |
| **Protects against...** | **Cache Poisoning** (Fake IPs).           | **Eavesdropping** (Privacy).                       |
| ** Analogy**            | A wax seal on a letter (proof of sender). | A locked armored truck (hides the letter).         |
| **Encryption**          | No (data is public, just signed).         | Yes (data is hidden).                              |
| **Integrity**           | High (verifies the source).               | Medium (ensures data wasn't changed _in transit_). |

## 3. Troubleshooting "The Blackout"

If you turn on DoT and suddenly you can't reach any websites, it is almost always because of a **Firewall** or an **Old
Router**.

- **The Reason:** Regular DNS uses **Port 53**. DNS over TLS uses **Port 853**.
- **The Fix:** Some routers block Port 853 because they want to "see" your DNS traffic (for parental controls or data
  collection). If this happens, you can set `DNSOverTLS=opportunistic`. This will try to use encryption but fall back to
  regular DNS if Port 853 is blocked.

### Checking your work

You can verify you are successfully "hidden" from eavesdroppers by using a packet sniffer or by visiting a tool like
[Cloudflare's Help Page](https://1.1.1.1/help), which will tell you if your DNS is encrypted.
