Denial of Service (DoS) and Distributed Denial of Service (DDoS) attacks on DNS are particularly devastating because DNS is
the "phonebook" of the internet. If DNS goes down, users cannot resolve domain names to IP addresses, effectively making
websites and services unreachable even if the web servers themselves are perfectly fine.

These attacks generally fall into two categories: **Resource Exhaustion** (crashing the DNS server) and **Amplification**
(using DNS to crash a different target).

---

## 1. DNS Amplification Attack (Reflection)

This is one of the most common and powerful DDoS methods. It uses the "reflection" principle to overwhelm a victim's network.

- **The Spoof:** The attacker sends thousands of small DNS queries to open DNS resolvers. Crucially, they **spoof the source
  IP address** to be the IP of the victim.
- **The Amplification:** The attacker asks for a very large record (like a DNSSEC `ANY` record). A tiny 60-byte query can
  trigger a 3000-byte response.
- **The Flood:** The DNS resolvers send these massive responses to the victim’s IP. The victim's bandwidth is quickly choked
  by a "wall" of unsolicited DNS data.

---

## 2. DNS NXDOMAIN Flood

In this attack, the goal is to exhaust the resources of a specific **Authoritative DNS Server** (the one that actually owns
the records for a domain like `example.com`).

- **The Tactic:** Attackers send a massive volume of queries for subdomains that do not exist (e.g., `ajshd123.example.com`,
  `992ksj.example.com`).
- **The Result:** 1. The recursive resolver can't find the answer in its cache, so it queries the Authoritative Server.

2. The Authoritative Server spends CPU and memory searching for these non-existent records and generating "NXDOMAIN"
   (Non-Existent Domain) responses.
3. Eventually, the server becomes too busy processing junk to handle legitimate requests.

---

## 3. "Phantom Domain" Attack

Similar to an NXDOMAIN flood, but targets the **Recursive Resolver** (like an ISP's DNS).

- **The Setup:** The attacker sets up multiple "phantom" domains on their own malicious authoritative servers.
- **The Attack:** They send queries for these phantom domains to a target resolver.
- **The Stall:** The attacker's authoritative servers are programmed to respond very slowly or not at all.
- **The Result:** The resolver's resources (waiting threads and memory) get tied up waiting for responses from the "slow"
  phantom servers, eventually causing the resolver to hang or crash for all users.

---

## 4. Random Subdomain Attack

Often launched via botnets (like Mirai), this attack is a hybrid of the NXDOMAIN flood. By using randomized subdomains, it
bypasses the DNS cache of the local resolver, forcing every single request to go all the way to the Authoritative Server,
effectively DDoS-ing the domain owner's infrastructure.

---

## Defensive Strategies

| Defense Mechanism                | How it Helps                                                                                                                                         |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Response Rate Limiting (RRL)** | Limits the number of responses a DNS server will send to a specific IP or range, blunting amplification attacks.                                     |
| **Anycast Routing**              | Distributes DNS traffic across a global network of servers. If one node is attacked, the others continue to function.                                |
| **Over-Provisioning**            | Simply having more bandwidth and processing power than the attacker can reasonably overwhelm (often handled by companies like Cloudflare or Akamai). |
| **Disabling 'ANY' Queries**      | Many modern DNS providers block or restrict the `ANY` query type to prevent high-factor amplification.                                               |

---

> **The "Water Torture" Attack:** This is a common nickname for the Random Subdomain attack. Like slow drips of water, the
> individual queries seem small, but the sheer volume and persistence eventually "break" the target's ability to function.
