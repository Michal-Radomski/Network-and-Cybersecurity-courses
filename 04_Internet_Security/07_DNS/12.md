The `dig` command (short for **Domain Information Groper**) is a powerful command-line tool used to query **DNS
nameservers**.

While regular users just type a URL into a browser, network administrators and developers use `dig` to see exactly what is
happening "under the hood" during the DNS lookup process.

---

## What can you do with it?

### 1. Basic IP Lookup

By default, `dig` looks for the **A Record** (IPv4 address) of a domain.

```bash
dig google.com

```

### 2. Query Specific Record Types

You can ask for specific information, like where a company receives email (**MX**) or the aliases for a site (**CNAME**).

```bash
dig google.com MX
dig shopify.com CNAME

```

### 3. Trace the Path

One of its coolest features is the `+trace` flag. It shows you the entire journey from the **Root Servers** to the **TLD
Servers** all the way down to the **Authoritative Server**.

```bash
dig google.com +trace

```

### 4. Query a Specific Server

If you want to see how a specific server (like Google's `8.8.8.8`) sees a domain compared to your ISP, you can specify it
with the `@` symbol.

```bash
dig @8.8.8.8 google.com

```

---

## Anatomy of a `dig` Output

When you run `dig`, the output is divided into sections that match the **DNS Packet Structure** we discussed earlier:

- **Header:** Shows the version of `dig`, the ID of the transaction, and the status (e.g., `NOERROR` means it worked).
- **Question Section:** Confirms what you asked for (e.g., `google.com. IN A`).
- **Answer Section:** The "gold." It provides the IP address and the **TTL** (how long you can cache it).
- **Statistics:** Tells you how long the query took (in milliseconds) and which server actually provided the answer.

---

## Why use `dig` instead of `nslookup`?

While both do similar things, `dig` is the industry standard because:

1. **More Detailed:** It shows the complete raw response from the server.
2. **Scriptable:** Its output is easier to parse for automated scripts.
3. **Accuracy:** It uses the OS's underlying resolver libraries more faithfully than the older `nslookup`.

### Pro Tip: The "Short" Version

If you don't want all the technical jargon and just want the IP address, use the `+short` flag:

```bash
dig google.com +short

```

_Output: `142.250.190.46_`
