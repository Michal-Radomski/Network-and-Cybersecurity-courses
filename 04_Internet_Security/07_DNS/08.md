The **DNS Root Servers** are the highest level of the Domain Name System (DNS) hierarchy. They are the first point of contact
for a recursive resolver (like Google DNS) when it needs to translate a domain name (e.g., `example.com`) into an IP address.

Think of them as the **master index** for the entire internet.

---

## 1. What Do They Actually Do?

Contrary to popular belief, root servers **do not** know the IP address of every website. Instead, they know who is
responsible for the **Top-Level Domains (TLDs)** like `.com`, `.org`, `.net`, and country codes like `.uk` or `.jp`.

- **The Request:** A resolver asks: "Where is `google.com`?"
- **The Response:** The Root Server says: "I don't know where `google.com` is, but I know the servers that handle all `.com`
  addresses. Go ask them at this address."

---

## 2. The "13 Servers" Myth

You will often hear that there are only **13 root servers** in the world. This is technically true and false at the same
time:

- **The 13 IP Addresses:** There are only 13 unique IP addresses assigned to the root server system (named
  `a.root-servers.net` through `m.root-servers.net`). This limit was originally due to technical constraints in the early
  architecture of the internet.
- **The Reality of Anycast:** While there are 13 _logical_ addresses, there are actually **hundreds of physical servers**
  located all over the globe. Using a technology called **Anycast**, when you query "m.root-servers.net," your request is
  automatically routed to the physical machine closest to you.

## 4. Why Are They Critical?

If the root server system were to fail completely, the internet would slowly "go dark."

1. **New Connections Fail:** You wouldn't be able to look up any website that wasn't already stored in your local cache.
2. **No New Addresses:** As cached records expire (based on their TTL), they would become unreachable because the resolver
   would have no "top" to start its search from.

Fortunately, because they are so widely distributed and managed by different organizations, a total failure has never
happened. They are designed to withstand massive DDoS attacks and physical disasters.

---

## 5. Security: DNSSEC

Because root servers are the "source of truth," they are a prime target for hackers. To prevent someone from hijacking a root
server and sending people to fake websites, the root zone is signed using **DNSSEC (DNS Security Extensions)**. This uses
digital signatures to prove that the data coming from the root server is authentic and hasn't been tampered with.
