On modern Ubuntu systems (18.04 and later), DNS is typically managed by a service called **systemd-resolved**.

Unlike a simple text file, this cache is stored in the system's memory. To see it, you don't "open" a file; instead, you use
the command line to tell the service to output its internal state.

---

## 1. View the Statistics

Before looking at the specific entries, you can see how well your cache is performing (hit/miss ratio) by running:

```bash
systemd-resolve --statistics

```

_(On newer versions of Ubuntu, use `resolvectl statistics`)_

---

## 2. Dump the Cache to Logs

Ubuntu doesn't have a command that simply prints the cache to the terminal window directly. Instead, you have to send a
"signal" to the resolver to dump its contents into the system log.

### Step A: Send the dump signal

Run this command to tell `systemd-resolved` to write its current cache to the internal system log:

```bash
sudo killall -USR1 systemd-resolved

```

_Note: This does not "kill" the process; the `-USR1` flag tells the service to perform a specific action (dumping the
cache)._

### Step B: Read the log

Now, view the end of the system log to see the dumped data:

```bash
sudo journalctl -u systemd-resolved > ~/dns_cache.txt

```

Or, to view it live in your terminal:

```bash
sudo journalctl -u systemd-resolved | grep -A 1000 "CACHE:"

```

---

## 3. How to Flush the Cache

If you are checking the cache because a website isn't loading or you've changed DNS settings, you might want to clear it out
and start fresh.

Run the following command:

```bash
sudo resolvectl flush-caches

```

---

## Summary of Commands

| Action          | Command                                  |
| --------------- | ---------------------------------------- |
| **Check Stats** | `resolvectl statistics`                  |
| **Dump Cache**  | `sudo killall -USR1 systemd-resolved`    |
| **View Dump**   | `sudo journalctl -u systemd-resolved -f` |
| **Clear Cache** | `sudo resolvectl flush-caches`           |

### Why is my cache empty?

If you see very little in your cache, it might be because your **browser** (Chrome or Firefox) is doing its own caching.
Browsers often bypass the OS cache entirely to save time. To see your browser's specific DNS cache, you can go to:

- **Chrome:** `chrome://net-internals/#dns`
- **Firefox:** `about:networking#dns`
