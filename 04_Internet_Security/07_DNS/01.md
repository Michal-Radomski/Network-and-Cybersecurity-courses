Think of the **Domain Name System (DNS)** as the "phonebook of the internet."

While humans access information online through easy-to-remember domain names like `google.com` or `wikipedia.org`, computers
communicate using **IP addresses** (Internet Protocol addresses), which look like `192.168.1.1` (IPv4) or more complex
alphanumeric strings (IPv6). DNS is the service that translates those human-friendly names into machine-readable IP
addresses.

---

## How the DNS Process Works

When you type a URL into your browser, a multi-step lookup process happens in milliseconds. It involves four primary types of
servers working together:

1. **DNS Recursor:** Think of this as a librarian you ask to find a specific book. It receives the query from your browser
   and does the legwork of talking to other DNS servers to find the address.
2. **Root Nameserver:** The first stop for the recursor. It doesn't know the IP, but it can point the recursor to more
   specific locations (like the .com or .net category).
3. **TLD Nameserver:** Top-Level Domain (TLD) servers manage specific domain extensions (.com, .org, .edu). It points the
   recursor toward the server that actually holds the specific domain record.
4. **Authoritative Nameserver:** This is the final stop. It holds the actual IP address for the domain. Once the recursor
   gets the IP from here, it passes it back to your browser.

## Key Components of DNS

### 1. DNS Caching

To speed up the internet, DNS information is often "cached" (stored temporarily) in various places:

- **Browser Cache:** Your browser remembers sites you've visited recently.
- **OS Cache:** Your computer's operating system keeps its own records.
- **ISP Cache:** Your Internet Service Provider keeps a massive cache to resolve popular requests faster for all their users.

### 2. Common Record Types

DNS doesn't just map names to IPs; it handles different types of requests using "records":

| Record Type     | Purpose                                                                                |
| --------------- | -------------------------------------------------------------------------------------- |
| **A Record**    | Maps a domain to an **IPv4** address.                                                  |
| **AAAA Record** | Maps a domain to an **IPv6** address.                                                  |
| **MX Record**   | Directs **email** to the correct mail server.                                          |
| **CNAME**       | Aliases one domain name to another (e.g., mapping `www.example.com` to `example.com`). |
| **TXT Record**  | Often used for security and domain verification (like proving you own a site).         |

---

## Why DNS Matters

- **Efficiency:** Without it, you would have to memorize a string of numbers for every website you want to visit.
- **Flexibility:** A website owner can change their hosting provider (changing their IP address) without the user ever
  knowing, as the DNS record is simply updated to point the domain name to the new IP.
- **Security:** Modern DNS includes features like **DNSSEC** (Domain Name System Security Extensions) to ensure that the data
  you receive hasn't been tampered with by a middleman.
