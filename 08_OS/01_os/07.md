## The Linux / Ubuntu Architecture

Ubuntu is a Linux distribution, which means it uses the **Linux kernel** at its core. Linux uses a **Monolithic Kernel**
architecture, but with a modern twist: it is **modular**.

While everything runs in the privileged Kernel Space for maximum speed, the kernel can dynamically load and unload pieces of
code—called **Loadable Kernel Modules (LKMs)**—on the fly. This means Ubuntu doesn't need to load drivers for hardware you
don't own, but the moment you plug in a new USB webcam, the kernel loads the required module instantly without needing a
system reboot.

## How Ubuntu Organizes the Layers

If you look under the hood of an Ubuntu system, the architecture is organized into distinct layers, moving from the user
interface down to the silicon.

```text
+-------------------------------------------------------------+
| 1. User Application Layer                                   |
|    (Firefox, LibreOffice, Python scripts, VLC)              |
+-------------------------------------------------------------+
| 2. GNU Utilities & Shell Layer                              |
|    (Bash terminal, apt package manager, systemd, ls, grep)  |
+-------------------------------------------------------------+
| 3. Desktop Environment (GUI) Layer                          |
|    (GNOME Shell, Display Server Display Server / Wayland)   |
+-------------------------------------------------------------+
                        ||  (System Calls / glibc)
===============================================================
                        ||
+-------------------------------------------------------------+
| 4. Linux Kernel Space                                       |
|    (VFS, Process Scheduler, Memory Manager, Dynamic Modules)|
+-------------------------------------------------------------+
                        ||
+-------------------------------------------------------------+
| 5. Physical Hardware                                        |
|    (Intel/AMD/ARM CPU, RAM, SSD, Graphics Card)             |
+-------------------------------------------------------------+

```

### 1. The User Application & GNU Layer

Ubuntu relies heavily on the **GNU toolset**. When you open a terminal and type commands like `ls` to list files or use `apt`
to install software, you are interacting with GNU utilities running in User Space.

### 2. The GNU C Library (`glibc`)

In Ubuntu, applications rarely make system calls to the Linux kernel directly. Instead, they use a wrapper library called
**`glibc`**. When a program wants to write data to a file, it calls a standard C function like `printf()` or `write()`.
`glibc` takes that request, packages it cleanly, and passes the official system call to the Linux kernel.

### 3. Desktop Environment (GNOME)

Unlike a CLI-only Linux server, desktop Ubuntu includes a heavy graphical layer. By default, Ubuntu uses **GNOME** as its
desktop environment, which runs on top of a display server (typically **Wayland** or **X11**). This layer translates your
mouse clicks and window drags into commands the underlying system can understand.

### 4. Inside the Ubuntu (Linux) Kernel

Once a request passes into Ubuntu's kernel space, it is handled by specific subsystems:

- **Virtual File System (VFS):** Linux treats almost everything as a file (even hardware devices). The VFS provides a unified
  interface so applications can talk to an ext4 SSD, a FAT32 USB drive, or a network drive using the exact same commands.
- **Memory Management (MM):** Manages Ubuntu's virtual memory, swapping data to the disk (swap space) if your physical RAM
  fills up.
- **Process Scheduler (CFS):** Ubuntu uses the _Completely Fair Scheduler_, which ensures that background system tasks and
  your open apps share CPU time smoothly without stuttering.
