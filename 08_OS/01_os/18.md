**Assembly language** is a low-level programming language that sits right above a computer's native machine code (the 1s and
0s). Instead of writing raw binary, Assembly uses short, human-readable abbreviations called **mnemonics** (like `MOV` for
move, `ADD` for addition, or `PUSH` for pushing data onto a stack) to command the CPU.

Because it is so close to the hardware, Assembly gives you total control over the processor and memory. However, it is
**architecture-specific**. Assembly written for an Intel/AMD processor (x86/x64) will not work on an ARM processor (like
those in phones or modern Macs).

---

## Assembly Language Example (x86_64)

Here is a classic "Hello, World!" program written for a 64-bit Ubuntu system using **NASM** (Netwide Assembler) syntax.

```nasm
section .data
    msg db "Hello, World!", 10      ; The string to print, 10 is the newline character
    msg_len equ $ - msg             ; Calculate the length of the string automatically

section .text
    global _start                   ; Entry point for the linker

_start:
    ; 1. Syscall to write to the screen
    mov rax, 1                      ; system call number 1 is 'sys_write'
    mov rdi, 1                      ; file descriptor 1 is stdout (the screen)
    mov rsi, msg                    ; pointer to our string
    mov rdx, msg_len                ; length of our string
    syscall                         ; call the kernel

    ; 2. Syscall to exit the program
    mov rax, 60                     ; system call number 60 is 'sys_exit'
    mov rdi, 0                      ; exit code 0 (success)
    syscall                         ; call the kernel

```

### How this code works:

- **`section .data`**: Where you declare initialized data or constants.
- **`section .text`**: Where the actual code goes.
- **Registers (`rax`, `rdi`, etc.)**: These are tiny, ultra-fast storage slots inside the CPU itself. In 64-bit assembly, we
  move data into specific registers to set up a "system call" (asking the Linux kernel to do something, like print text or
  close the program).

## How to Run Assembly on Ubuntu

To run the code above, you need an **assembler** (to turn the text into an object file) and a **linker** (to turn the object
file into an executable program).

Follow these steps using your terminal:

### Step 1: Install NASM

Ubuntu doesn't come with an assembler by default. Install NASM using `apt`:

```bash
sudo apt update
sudo apt install nasm -y

```

### Step 2: Create the File

Save the example code above into a file named `hello.asm`. You can use a text editor like Nano:

```bash
nano hello.asm

```

_(Paste the code, then press `Ctrl+O`, `Enter`, and `Ctrl+X` to save and exit)._

### Step 3: Assemble the Code

Convert your text file into a 64-bit ELF object file:

```bash
nasm -f elf64 hello.asm -o hello.o

```

### Step 4: Link the Object File

Link the object file to create the final executable:

```bash
ld hello.o -o hello

```

### Step 5: Run It

Now, execute your brand new binary program:

```bash
./hello

```

**Output:**

```text
Hello, World!

```
