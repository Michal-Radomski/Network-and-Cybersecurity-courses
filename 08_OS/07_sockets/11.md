**libuv** is a multi-platform support library with a focus on asynchronous I/O. It was primarily developed for use by
**Node.js**, but it's also used by other software like Luavit, Julia, and Neovim.

Think of it as the engine under the hood that allows Node.js to handle thousands of simultaneous connections without getting
bogged down.

---

### Core Features

- **Full-featured event loop:** Backed by epoll (Linux), kqueue (macOS), IOCP (Windows), and event ports (SunOS).
- **Asynchronous TCP and UDP sockets:** Handles networking smoothly across different operating systems.
- **Asynchronous DNS resolution:** Prevents domain name lookups from blocking the main thread.
- **Asynchronous file and file system operations:** Allows reading and writing files in the background.
- **Thread pool:** Used to handle tasks that can't be done asynchronously at the operating system level (like file I/O or
  specific crypto functions).

---

### How It Works: The Event Loop

At the heart of libuv is the **Event Loop**. Instead of creating a new thread for every single request (which consumes a lot
of memory and CPU), libuv operates on a single thread and uses a "non-blocking" approach.

```
[ Application / Node.js ]
          │
          ▼
     [ libuv ] ───► ( Checks for OS Events / Network I/O )
          │
          ├─► [ Thread Pool ] ──► ( Heavy tasks like File I/O )
          │
          ▼
[ Callback Queue ] ──► [ Executed on Main Thread ]

```

1. **Register:** Your application asks libuv to do something (e.g., "read this file" or "listen for a network request").
2. **Delegate:** libuv hands the task over to the Operating System kernel (for network stuff) or its internal Thread Pool
   (for file stuff).
3. **Notify:** Once the task is finished, libuv triggers a callback function to let your application know the data is ready.

---

### Why is it important?

Before libuv, writing cross-platform asynchronous code was a massive headache because Windows and Unix-based systems handle
I/O completely differently. Windows uses **IOCP** (Input/Output Completion Ports), while Linux uses **epoll**.

> **The Big Benefit:** libuv provides a neat, unified interface. Developers can write their code once, and libuv ensures it
> runs optimally on Windows, macOS, and Linux alike.
