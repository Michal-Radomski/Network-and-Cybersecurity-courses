In Ubuntu Server, **sockets** are the endpoints for network communication (TCP/UDP) or local inter-process communication
(Unix Domain Sockets). Efficiently managing them is crucial for troubleshooting connectivity issues, optimizing server
performance, and ensuring security.

Here is a practical guide to viewing, analyzing, and managing sockets in Ubuntu.

---

## 1. Checking Active Sockets (The Modern Way)

The old `netstat` tool is deprecated. The modern, much faster replacement is **`ss`** (socket statistics).

### Common `ss` Commands:

- **View all listening TCP and UDP sockets:**

```bash
ss -tuln

```

- `-t`: TCP sockets
- `-u`: UDP sockets
- `-l`: Listening sockets
- `-n`: Show numeric port numbers (instead of resolving names like "http")

- **Find which process is using a socket:** To see the actual application name and Process ID (PID), add the `-p` flag
  (requires `sudo`):

```bash
sudo ss -tulnp

```

- **View established (active) connections:**

```bash
ss -at

```

---

## 2. Managing Sockets via `systemd`

Ubuntu uses `systemd` to manage many system services. Some services use **systemd socket activation**. Instead of the service
running constantly, `systemd` listens on the socket and starts the service only when a connection arrives.

- **List all active systemd sockets:**

```bash
systemctl list-sockets

```

- **Stop or Disable a socket:** If a service is bound to a socket via systemd, stopping the service won't free the port
  permanently; you must stop the socket itself.

```bash
sudo systemctl stop cups.socket
sudo systemctl disable cups.socket

```

---

## 3. Finding and Closing Stubborn Sockets

If an application crashes, it might leave a socket in a `TIME_WAIT` or `CLOSE_WAIT` state, preventing you from restarting the
service (often causing an "Address already in use" error).

### Step 1: Identify the culprit

Use `lsof` (LiSt Open Files) to find the exact PID hogging the port (e.g., port 80):

```bash
sudo lsof -i :80

```

### Step 2: Kill the process

Once you have the PID (e.g., 1234), you can force-close the socket by killing the process:

```bash
sudo kill -9 1234

```

---

## 4. Tuning Socket Performance (Kernel Level)

For high-traffic Ubuntu servers, the default maximum socket limits might be too low, leading to "Connection refused" errors.
You can tune these via `/etc/sysctl.conf`.

### Key Parameters to Optimize:

- **`net.core.somaxconn`**: The maximum number of backlogged connections in the socket queue.
- **`net.ipv4.tcp_max_syn_backlog`**: Maximum number of remembered connection requests that have not received an
  acknowledgment from the client yet.

To increase these limits, edit the file:

```bash
sudo nano /etc/sysctl.conf

```

Add the following lines at the bottom:

```text
net.core.somaxconn = 1024
net.ipv4.tcp_max_syn_backlog = 2048

```

Apply the changes instantly without rebooting:

```bash
sudo sysctl -p

```
