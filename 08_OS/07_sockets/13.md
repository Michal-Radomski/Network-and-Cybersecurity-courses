The default TCP listen backlog value for Node.js is **`511`**.

When you create a TCP or HTTP server using the `net` or `http` modules (e.g., `server.listen(port)`), Node.js passes this
default value under the hood to the operating system's underlying `listen(fd, backlog)` system call.

### Important Nuances to Keep in Mind:

- **It is a maximum cap for _pending_ connections:** The backlog dictates the maximum length of the queue for connections
  that have been completed by the OS network stack but haven't been picked up (`accept`ed) by your Node.js application yet.
- **The OS has the final say:** The actual backlog limit is strictly bound by your operating system's configuration. On
  Linux, the operating system will silently truncate this value to the lower of your `sysctl` settings:
- `net.core.somaxconn` (The maximum backlog limit)
- `net.ipv4.tcp_max_syn_backlog` (The maximum length of the TCP SYN queue)

- **How to change it:** If you expect massive spikes in concurrent connection requests and need a larger queue, you can
  manually override the default by explicitly passing a backlog integer as an argument to the `.listen()` method:

```javascript
const net = require("net");
const server = net.createServer();

// Setting a custom backlog of 1023
server.listen(8080, "0.0.0.0", 1023, () => {
  console.log("Server is listening...");
});
```
