When two computers (hosts) talk to each other over a network, they rely on a structured process to ensure that data sent from
one side arrives safely and accurately at the other. This process is called **host-to-host communication**.

To understand how this works, it helps to look at the **OSI (Open Systems Interconnection) Model**, which breaks down the
communication into layers. Each layer has a specific job, passing data down at the sending host and back up at the receiving
host.

---

## The Network Layers at Work

While there are seven layers in total, host-to-host communication primarily focuses on how data moves across the network from
an end-to-end perspective.

Here is a breakdown of the core layers involved when Host A sends a message to Host B:

### 1. The Application & Presentation Layers (The Data)

- **What happens:** You type a message, click a link, or stream a video.
- **The Job:** The host creates raw data (like an HTTP request for a webpage) and formats it so the other host can understand
  it.

### 2. The Transport Layer (The Connection)

- **What happens:** This layer prepares the data for travel. It breaks large files into smaller, manageable chunks called
  **segments**.
- **The Job:** It adds a header to each segment containing **Port Numbers** (which identify the specific application, like
  port 80 for web traffic). This layer also decides _how_ to send the data:
- **TCP (Transmission Control Protocol):** Reliable and guarantees delivery (used for texting or web browsing).
- **UDP (User Datagram Protocol):** Fast but doesn't check for errors (used for live gaming or video calls).

### 3. The Network Layer (The Routing)

- **What happens:** The segments are packaged into **packets**.
- **The Job:** The network layer adds the logical addresses—the **Source IP address** (Host A) and the **Destination IP
  address** (Host B). Routers use these IP addresses to find the best path across the internet from one host to the other.

### 4. The Data Link & Physical Layers (The Delivery)

- **What happens:** Packets are put into **frames** and turned into physical signals.
- **The Job:** The Data Link layer adds physical addresses (**MAC addresses**) to get the data to the very next local device
  (like your home router). The Physical layer then converts everything into electrical signals, light pulses (fiber optics),
  or radio waves (Wi-Fi) to actually move the data across cables or the air.

---

## Step-by-Step: Sending a Message

To put it simply, imagine sending a letter:

1. **Encapulation (Sending):** You write the letter (Data), put it in an envelope with the recipient's room number (Port),
   write the street address (IP Address), and hand it to the mail carrier (Physical Layer).
2. **Transit:** The mail system (routers and switches) looks at the destination address and moves the letter from post office
   to post office until it reaches the right house.
3. **Decapsulation (Receiving):** Host B receives the physical signals, strips away the MAC and IP addresses, checks the port
   number to see which app it belongs to, and delivers the clean data to your screen.
