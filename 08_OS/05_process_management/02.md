# Process Control Block (PCB)

A **Process Control Block (PCB)** is a data structure maintained by the operating system that stores all information about a
process. It acts as the **process's identity card** and allows the OS to manage, schedule, and track the process.

When a process is created, the OS creates a PCB for it. When the process terminates, its PCB is removed.

---

## Purpose of PCB

The PCB helps the operating system:

- Track process state
- Perform process scheduling
- Manage memory allocation
- Save and restore process information during context switching
- Keep account of resources used by the process

---

## Information Stored in a PCB

### 1. Process Identification

- Process ID (PID)
- Parent Process ID (PPID)
- User ID

### 2. Process State

The current status of the process, such as:

- New
- Ready
- Running
- Waiting/Blocked
- Terminated

### 3. Program Counter (PC)

Stores the address of the next instruction to be executed.

### 4. CPU Registers

Contains the values of CPU registers when the process is interrupted.

Examples:

- General-purpose registers
- Stack Pointer (SP)
- Program Status Word (PSW)

### 5. CPU Scheduling Information

Used by the scheduler:

- Priority
- Scheduling queue pointers
- Time slice information

### 6. Memory Management Information

Details about memory assigned to the process:

- Base and limit registers
- Page tables
- Segment tables

### 7. I/O Status Information

Information about resources being used:

- Open files
- Allocated devices
- I/O requests

### 8. Accounting Information

Used for monitoring and billing:

- CPU time used
- Process creation time
- Resource usage statistics

## PCB and Context Switching

During a **context switch**:

1. The OS saves the current process's state into its PCB.
2. The OS selects another process.
3. The new process's state is loaded from its PCB.
4. Execution resumes from where that process previously stopped.

### Example

```text id="e08w4e"
Running Process A
       |
       | Interrupt
       v
Save A's state in PCB-A
Load B's state from PCB-B
       |
       v
Running Process B
```

Without PCBs, the OS would not be able to resume processes correctly.

---

## Importance of PCB

- Enables multitasking.
- Supports process scheduling.
- Facilitates context switching.
- Maintains process information securely.
- Helps manage CPU and memory resources efficiently.

---

## Summary

A **Process Control Block (PCB)** is the operating system's data structure for managing a process. It stores essential
information such as the process state, program counter, CPU registers, scheduling details, memory information, and I/O
status. During context switching, the OS saves and restores process information using the PCB, making multitasking possible.
