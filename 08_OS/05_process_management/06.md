In Node.js, JavaScript runs on a single thread. To take advantage of multi-core CPUs and handle CPU-intensive tasks without
freezing your main application, you can use the **`fork()`** method.

In Node.js, `fork()` is a specialized function built into the **`child_process`** module. Unlike the standard Unix `fork()`
which clones the entire process, Node's `fork()` spawns a completely new instance of the V8 engine and opens an **IPC
(Inter-Process Communication) channel** so the parent and child can easily talk to each other.

---

## How to Use `fork()`

To use it, you need two files:

1. **The Parent File (`server.js`):** Spawns the child and sends/receives messages.
2. **The Child File (`worker.js`):** Runs the heavy background task and sends results back.

### Step 1: Create the Child Script (`worker.js`)

This script listens for a message from the parent, does some work, and sends the result back using `process.send()`.

```javascript
// worker.js

// Listen for messages from the parent process
process.on("message", (message) => {
  console.log(`[CHILD] Received task from parent: ${message}`);

  // Simulate a heavy, time-consuming calculation
  let sum = 0;
  for (let i = 0; i < 1e9; i++) {
    sum += i;
  }

  // Send the result back to the parent process
  process.send({ result: sum });

  // Exit the child process when done
  process.exit();
});
```

### Step 2: Create the Parent Script (`server.js`)

This script uses `fork()` to launch `worker.js`, sends it data, and handles the response.

```javascript
// server.js
const { fork } = require("child_process");
const path = require("path");

console.log("[PARENT] Starting main application...");

// Spawn a new Node.js process running the worker script
const child = fork(path.join(__dirname, "worker.js"));

// Send a message to the child process
child.send("Start heavy calculation");

// Listen for messages (results) sent from the child process
child.on("message", (message) => {
  console.log(`[PARENT] Received result from child:`, message.result);
});

// Handle child process exit
child.on("exit", (code) => {
  console.log(`[PARENT] Child process exited with code ${code}`);
});

console.log("[PARENT] I am unblocked and can handle other requests!");
```

---

## Key Differences: Node.js `fork()` vs. Unix `fork()`

It is important to know that Node's `fork()` behaves differently than the traditional OS-level `fork()`:

- **No Memory Cloning:** A standard OS `fork()` duplicates the parent's memory. Node's `fork()` starts a **fresh** Node.js
  instance with its own allocated memory. They do not share variables directly.
- **Built-in Communication:** Node automatically sets up an IPC channel between the parent and child, allowing you to pass
  JSON messages back and forth using `.send()` and `.on('message')`.
- **Resource Intensive:** Because each child process has its own V8 environment, memory, and event loop, spawning hundreds of
  forks can quickly drain your system's RAM.

---

## When should you use `fork()`?

- **Do use it for:** Heavy mathematical computations, video/image processing, file compression, or executing long-running
  independent scripts.
- **Do NOT use it for:** Basic I/O tasks (like database queries or reading files). Node's asynchronous architecture already
  handles I/O brilliantly on a single thread. Spawning a fork for I/O adds unnecessary overhead.
