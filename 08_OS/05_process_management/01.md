# Process vs Thread

A **process** is an independent program in execution, while a **thread** is the smallest unit of execution within a process.

## Process

A process has its own:

- Memory space (address space)
- Code segment
- Data segment
- Heap
- System resources (files, handles, etc.)

### Characteristics

- Independent execution unit.
- Processes are isolated from each other.
- Communication between processes requires **Inter-Process Communication (IPC)** mechanisms such as pipes, sockets, or shared
  memory.
- Creation and context switching are relatively expensive.

### Example

Running:

- A web browser
- A music player
- A text editor

creates separate processes.

---

## Thread

A thread is a lightweight execution unit within a process.

Threads of the same process share:

- Code segment
- Data segment
- Heap memory
- Open files and resources

Each thread has its own:

- Program Counter (PC)
- Register set
- Stack

### Characteristics

- Lightweight compared to processes.
- Faster creation and switching.
- Threads can communicate directly through shared memory.
- Requires synchronization to avoid data inconsistencies.

### Example

In a web browser:

- One thread handles the user interface.
- Another thread loads web pages.
- Another thread executes scripts.

## Comparison Table

| Feature            | Process                           | Thread                                      |
| ------------------ | --------------------------------- | ------------------------------------------- |
| Definition         | Independent program in execution  | Smallest unit of execution within a process |
| Memory             | Separate memory space             | Shared memory within the process            |
| Resource Ownership | Own resources                     | Shares process resources                    |
| Communication      | IPC required                      | Direct through shared memory                |
| Creation Time      | Slower                            | Faster                                      |
| Context Switching  | More expensive                    | Less expensive                              |
| Isolation          | High                              | Low                                         |
| Failure Impact     | Usually affects only that process | May affect entire process                   |
| Overhead           | Higher                            | Lower                                       |

---

## Context Switching

### Process Context Switch

The OS must switch:

- Memory mappings
- Registers
- Process state
- Resource information

This makes process switching relatively costly.

### Thread Context Switch

The OS mainly switches:

- Registers
- Program counter
- Stack pointer

Since memory is shared, switching is faster.

---

## Real-World Example

### Process-Based

Suppose you open:

- A browser
- A media player
- A calculator

Each runs as a separate process.

### Thread-Based

Inside the browser process:

- UI thread
- Network thread
- Rendering thread
- JavaScript execution thread

All work together within the same process.

---

## Summary

| Process                       | Thread                                |
| ----------------------------- | ------------------------------------- |
| Heavyweight                   | Lightweight                           |
| Own memory and resources      | Shares process memory and resources   |
| Better isolation and security | Better performance and responsiveness |
| Higher overhead               | Lower overhead                        |
| Communication is slower       | Communication is faster               |

In short, a **process** provides isolation and resource ownership, while **threads** provide efficient concurrent execution
within a process. Modern applications typically use **multiple threads inside a process** to improve performance and
responsiveness.
