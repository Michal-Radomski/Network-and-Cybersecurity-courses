The **Instruction Life Cycle in a CPU** (also called the **Instruction Cycle** or **Fetch–Decode–Execute Cycle**) describes
the sequence of steps a processor follows to execute each machine instruction.

### 1. Fetch

The CPU retrieves the next instruction from main memory.

- The **Program Counter (PC)** contains the address of the next instruction.
- The address is sent to memory.
- The instruction is loaded into the **Instruction Register (IR)**.
- The PC is updated to point to the next instruction.

**Example:** If the instruction is stored at memory location 1000, the CPU fetches it and then increments the PC to 1001 (or
the next instruction address).

---

### 2. Decode

The CPU interprets the fetched instruction.

- The **Control Unit (CU)** examines the instruction in the IR.
- It determines:
  - What operation must be performed (add, subtract, load, store, jump, etc.).
  - Which operands are needed.
  - Where the operands are located.

**Example:** Instruction: `ADD R1, R2`

The CPU decodes it as:

- Operation: Addition
- Operands: Contents of registers R1 and R2

---

### 3. Operand Fetch (if required)

The CPU obtains the data needed for execution.

- Data may come from:
  - CPU registers
  - Cache memory
  - Main memory
  - Input devices

**Example:** For `LOAD R1, [2000]`, the CPU fetches the value stored at memory address 2000.

---

### 4. Execute

The instruction is carried out.

- The **Arithmetic Logic Unit (ALU)** or other execution units perform the operation.
- Arithmetic, logical, comparison, or control operations occur here.

**Example:** `ADD R1, R2`

- ALU adds the values in R1 and R2.
- Result is generated.

---

### 5. Memory Access (if required)

Some instructions need to read from or write to memory.

- **Load instruction:** Read data from memory.
- **Store instruction:** Write data to memory.

**Example:** `STORE R1, [3000]`

- The value in R1 is written to memory location 3000.

---

### 6. Write Back

The result is stored in its destination.

- Result may be written to:
  - A register
  - Memory
  - Special CPU status registers

**Example:** After addition, the sum is written back into R1.

---

### 7. Interrupt Check

Before starting the next instruction, the CPU checks for interrupts.

- If no interrupt exists, the next instruction cycle begins.
- If an interrupt is pending:
  - The CPU saves its current state.
  - Executes the interrupt service routine.
  - Returns to the interrupted program afterward.

---

### Summary

The instruction life cycle consists of:

1. **Fetch** – Get the instruction from memory.
2. **Decode** – Determine what the instruction means.
3. **Operand Fetch** – Obtain required data.
4. **Execute** – Perform the operation.
5. **Memory Access** – Read/write memory if needed.
6. **Write Back** – Store the result.
7. **Interrupt Check** – Handle any pending interrupts.

This cycle repeats continuously, allowing the CPU to execute programs instruction by instruction.
