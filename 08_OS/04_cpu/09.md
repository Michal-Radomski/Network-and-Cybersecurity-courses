## Pipelining

**Pipelining** is a CPU technique that improves performance by overlapping the execution of multiple instructions. Instead of
waiting for one instruction to complete all stages before starting the next, different instructions are processed
simultaneously in different stages.

### Typical Pipeline Stages

1. **Fetch (IF)** – Retrieve instruction from memory.
2. **Decode (ID)** – Interpret the instruction.
3. **Execute (EX)** – Perform the operation.
4. **Memory Access (MEM)** – Read/write memory if needed.
5. **Write Back (WB)** – Store the result.

### Example

Without pipelining:

```text
Instruction 1: Fetch → Decode → Execute → Memory → Write Back
Instruction 2:                           Fetch → Decode → Execute → ...
```

With pipelining:

```text
Cycle 1: I1 Fetch
Cycle 2: I1 Decode, I2 Fetch
Cycle 3: I1 Execute, I2 Decode, I3 Fetch
Cycle 4: I1 Memory, I2 Execute, I3 Decode, I4 Fetch
Cycle 5: I1 Write Back, I2 Memory, I3 Execute, I4 Decode
```

### Advantages

- Increases instruction throughput.
- Makes better use of CPU resources.
- Improves overall performance.

### Limitations (Pipeline Hazards)

- **Data Hazards:** One instruction depends on the result of another.
- **Control Hazards:** Caused by branch instructions.
- **Structural Hazards:** Two instructions need the same hardware resource simultaneously.

## Parallelism

**Parallelism** is the execution of multiple tasks or operations at the same time using multiple processing units.

### Types of Parallelism

### 1. Instruction-Level Parallelism (ILP)

Multiple instructions are executed simultaneously.

**Example:** Superscalar processors can execute several instructions in one clock cycle.

### 2. Data Parallelism

The same operation is performed on multiple data elements at once.

**Example:** Adding two arrays element-by-element using vector processors.

```text
A = [1,2,3,4]
B = [5,6,7,8]

Result = [6,8,10,12]
```

### 3. Task Parallelism

Different tasks run concurrently on different processors or cores.

**Example:**

- One core plays music.
- Another core downloads a file.
- A third core runs a web browser.

### 4. Thread-Level Parallelism

A program is divided into multiple threads that execute simultaneously.

**Example:** A web server handling many client requests at the same time.

---

## Difference Between Pipelining and Parallelism

| Feature              | Pipelining                                              | Parallelism                                                   |
| -------------------- | ------------------------------------------------------- | ------------------------------------------------------------- |
| Concept              | Overlaps stages of instruction execution                | Executes multiple tasks/instructions simultaneously           |
| Goal                 | Increase throughput                                     | Increase overall computational power                          |
| Hardware Requirement | Single CPU can implement it                             | Often requires multiple execution units, cores, or processors |
| Execution            | Different stages of different instructions run together | Multiple instructions/tasks run at the same time              |
| Example              | Assembly line in a factory                              | Multiple workers performing different jobs simultaneously     |

### Analogy

**Pipelining:** Like a car assembly line where different cars are at different stages (painting, engine fitting, inspection)
simultaneously.

**Parallelism:** Like having several assembly lines working at the same time, each producing cars independently.

### Summary

- **Pipelining** improves CPU efficiency by overlapping instruction stages.
- **Parallelism** improves performance by executing multiple operations concurrently.
- Modern processors combine both techniques: each core uses pipelining, while multiple cores provide parallel execution.
