Here is a TypeScript snippet for Node.js that will heavily load the processor. It works by running an infinite loop that
constantly performs mathematical calculations ($10^7$ iterations per cycle) to keep the CPU core busy.

### CPU Burner Code

```typescript
import { Worker, isMainThread } from "worker_threads";
import os from "os";

function burnCPU(): void {
  console.log(`Worker ${process.pid} started to load the CPU...`);
  let x = 0;
  while (true) {
    // Perform intense math calculations to prevent compiler optimization
    for (let i = 0; i < 10000000; i++) {
      x += Math.sqrt(Math.random() * 123.45);
    }
  }
}

if (isMainThread) {
  // Get the number of available CPU cores
  const numCPUs = os.cpus().length;
  console.log(`Spawning ${numCPUs} workers to maximize all CPU cores...`);
  console.log("Press Ctrl+C to stop the load test.");

  // Spawn a worker for each CPU core to hit 100% total usage
  for (let i = 0; i < numCPUs; i++) {
    new Worker(__filename);
  }
} else {
  // This runs inside the worker threads
  burnCPU();
}
```

### How It Works

- **Multi-Threading:** Node.js is single-threaded by default. If you only run a `while(true)` loop normally, it will only
  maximize **one** of your CPU cores.
- **`worker_threads` Module:** This script detects how many logical CPU cores your computer has using `os.cpus().length` and
  spawns a worker thread for each one. This ensures your overall CPU utilization spikes toward 100%.
- **Mathematical Operations:** The `Math.sqrt` and `Math.random` functions force the CPU to perform floating-point arithmetic
  constantly, preventing Node's V8 engine from optimizing or skipping the loop.
