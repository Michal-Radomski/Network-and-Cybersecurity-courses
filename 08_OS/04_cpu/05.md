When looking at the number of instructions in **CISC** (Complex Instruction Set Computer) and **RISC** (Reduced Instruction
Set Computer—_note: it's spelled RISC with a 'C' for Computer_), the difference isn't just a random number. It reflects two
completely opposite design philosophies.

Here is a breakdown of how many instructions they have, along with concrete examples.

---

## 1. CISC (Complex Instruction Set Computer)

The goal of CISC is to complete a task in as few lines of assembly code as possible. To do this, the processor hardware is
made highly complex, featuring single instructions that can perform multiple operations (like loading from memory, doing
math, and saving back to memory all at once).

- **Number of Instructions:** Typically **between 1,000 and 3,000+ instructions**, and the list grows with every new
  generation of processors.
- **Instruction Length:** Variable. An instruction can be anywhere from 1 byte to 15 bytes long depending on how complex it
  is.

### Real-World Architecture Examples

- **x86 and x86-64:** The processors made by **Intel** and **AMD** that power most modern laptops, desktops, and heavy-duty
  servers.

### Assembly Code Example

Imagine you want to multiply two numbers together, but one of them is tucked away in the main RAM. A CISC processor can do
this in **one single instruction**:

```assembly
MULT 2:3, 5:2

```

_What this single instruction does:_ It goes to memory address 2:3, fetches that number, multiplies it by the number at
memory address 5:2, and saves the result directly back into memory. The CPU hardware handles all the stepping back and forth.

---

## 2. RISC (Reduced Instruction Set Computer)

The goal of RISC is the exact opposite: make the hardware as simple and fast as possible. Instead of giving the CPU complex,
multi-step instructions, RISC only gives it highly optimized, simple instructions that can each be executed in **one clock
cycle**.

- **Number of Instructions:** Typically **fewer than 100 to around 250 instructions**.
- **Instruction Length:** Fixed. Every single instruction is exactly the same size (usually 4 bytes / 32 bits), making it
  incredibly easy for the CPU to pipeline and decode.

### Real-World Architecture Examples

- **ARM:** Powers practically all smartphones (iPhones, Androids), tablets, Apple Silicon Macs (M1, M2, M3, etc.), and
  Nintendo Switch.
- **RISC-V:** An increasingly popular, open-source architecture used in microcontrollers and modern tech infrastructure.

### Assembly Code Example

Because a RISC processor keeps its hardware simple, it cannot look up memory and do math at the same time. To perform that
exact same multiplication task from above, RISC breaks it down into **four separate instructions**:

```assembly
LOAD A, 2:3
LOAD B, 5:2
PROD A, B
STORE 2:3, A

```

_What this code does:_ It explicitly tells the CPU to load the first number into an internal register ($A$), load the second
number into another register ($B$), multiply the two registers together, and then save the result back out to memory.

---

## Summary Comparison

| Feature                  | CISC (e.g., Intel x86)                          | RISC (e.g., Apple M-Series/ARM)                        |
| ------------------------ | ----------------------------------------------- | ------------------------------------------------------ |
| **Instruction Count**    | **Thousands** (Huge vocabulary)                 | **~100 to 250** (Small, efficient vocabulary)          |
| **Execution Time**       | One instruction takes **multiple clock cycles** | One instruction takes **exactly 1 clock cycle**        |
| **Code Size**            | Smaller assembly files (dense instructions)     | Larger assembly files (more lines of simple code)      |
| **Hardware vs Software** | Puts the burden of work on the **Hardware**     | Puts the burden of work on the **Software (Compiler)** |
| **Power Efficiency**     | Lower (uses more energy to handle complexity)   | Higher (great for battery-powered devices)             |
