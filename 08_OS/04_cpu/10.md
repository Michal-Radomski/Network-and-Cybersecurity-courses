## Hyper-Threading

**Hyper-Threading (HT)** is a technology developed by Intel that allows a single physical CPU core to act like **two logical
(virtual) cores**. This enables the processor to execute multiple threads simultaneously and make better use of CPU
resources.

### How It Works

A CPU core contains execution units, registers, caches, and control logic. During normal operation, some resources may sit
idle while a thread waits for data from memory.

With Hyper-Threading:

- One physical core maintains the state of **two threads**.
- The operating system sees **two logical processors** instead of one.
- When one thread is stalled (for example, waiting for memory), the core can execute instructions from the other thread.

### Example

Suppose a processor has:

- 4 physical cores
- Hyper-Threading enabled

The operating system will see:

```text
4 Physical Cores × 2 Threads per Core = 8 Logical Processors
```

So a task manager may report **8 CPUs** even though there are only 4 physical cores.

---

## Simple Illustration

Without Hyper-Threading:

```text
Core 1 → Thread A
Core 2 → Thread B
Core 3 → Thread C
Core 4 → Thread D
```

With Hyper-Threading:

```text
Core 1 → Thread A + Thread B
Core 2 → Thread C + Thread D
Core 3 → Thread E + Thread F
Core 4 → Thread G + Thread H
```

The CPU can schedule and switch between these threads more efficiently.

---

## Advantages

### 1. Better CPU Utilization

Idle execution units can be used by another thread.

### 2. Improved Multitasking

Multiple applications can run more smoothly at the same time.

### 3. Higher Throughput

Workloads involving many threads often complete faster.

### 4. Enhanced Performance for Parallel Workloads

Useful for:

- Video rendering
- Scientific computing
- Virtual machines
- Software development
- Database servers

---

## Limitations

### 1. Not Equivalent to Extra Physical Cores

Two logical threads share the same physical core resources.

### 2. Performance Gain Varies

Typical gains are often around **10–30%**, depending on the workload.

### 3. Resource Contention

If both threads need the same execution units, performance improvement may be small.

### 4. Power and Heat

Running more active threads can increase power consumption and heat generation.

---

## Hyper-Threading vs Multicore

| Feature            | Hyper-Threading                    | Multicore                       |
| ------------------ | ---------------------------------- | ------------------------------- |
| Physical Cores     | No increase                        | Increases number of cores       |
| Logical Processors | Increases                          | Also increases                  |
| Resource Sharing   | Threads share one core's resources | Each core has its own resources |
| Performance Gain   | Moderate                           | Usually much larger             |
| Cost               | Lower                              | Higher hardware cost            |

---

## Real-World Example

A processor such as the Intel Core i7 with:

- 8 physical cores
- Hyper-Threading enabled

may appear as:

```text
8 Cores / 16 Threads
```

The operating system can schedule up to 16 threads concurrently, although they ultimately share the resources of 8 physical
cores.

## Summary

**Hyper-Threading** is a form of **simultaneous multithreading (SMT)** that allows one physical CPU core to execute two
threads at the same time. It improves CPU utilization and throughput by keeping execution resources busy, especially in
multitasking and highly threaded workloads. However, it does not double performance because the logical threads share the
same physical core resources.
