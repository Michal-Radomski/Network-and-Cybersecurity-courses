When you press the power button on a PC, a highly choreographed sequence of events happens behind the scenes to bring your
operating system to life.

Here is exactly what happens from the moment electricity flows to the moment your desktop is ready, focusing on **Ubuntu
(Linux)**, followed by how **Windows** handles the same process.

---

## The Universal Starting Point: Hardware Initiation

Before any operating system (like Ubuntu or Windows) can load, the computer's hardware has to wake up. This phase is
identical for both OS options.

1. **Power-On:** Pressing the button sends electricity to the motherboard.
2. **UEFI/BIOS Initialization:** The motherboard's firmware—modern computers use **UEFI** (Unified Extensible Firmware
   Interface), while older ones use **BIOS**—wakes up.
3. **POST (Power-On Self-Test):** The UEFI quickly checks your hardware (RAM, CPU, keyboard, storage drives) to make sure
   everything is working. If you hear a weird beep or see a motherboard logo (like ASUS, Dell, or HP), POST is happening.
4. **Finding the Bootloader:** The UEFI looks at its NVRAM settings to find the designated "boot device" (usually your main
   SSD) and locates a specific file called a **bootloader**.

---

## The Ubuntu Boot Process

Once the UEFI hands off control to Ubuntu, the Linux-specific boot process begins.

### 1. GRUB 2 (The Bootloader)

UEFI loads **GRUB** (Grand Unified Bootloader).

- If you only have Ubuntu installed, GRUB usually hides and automatically loads the default settings.
- If you are dual-booting, GRUB pauses and shows you a menu, letting you choose whether to boot into Ubuntu or Windows.
- Once Ubuntu is selected, GRUB loads the Linux **Kernel** and a temporary filesystem called **initramfs** into the
  computer's RAM.

### 2. The Kernel and initramfs

The **Kernel** is the absolute core of the Linux operating system.

- Because the kernel doesn't initially know everything about your specific hardware, it uses the **initramfs** (initial RAM
  filesystem) to load essential drivers (like your SSD controller drivers).
- The kernel then mounts the actual root filesystem (`/`) on your drive and prepares to start the real operating system
  processes.

### 3. systemd (The Master Process)

The kernel starts the very first software process, known as `systemd` (Process ID 1).

- `systemd` is responsible for bringing the rest of Ubuntu to life. It starts services in parallel to speed things up:
  network managers, sound drivers, Bluetooth, and printing services.
- During this time, you usually see the **Plymouth** splash screen (the black screen with the "Ubuntu" logo and a spinning
  wheel) hiding all the text-based code rolling by in the background.

### 4. Display Manager and Desktop Environment

- `systemd` launches the **Display Manager** (Ubuntu uses one called **GDM** - GNOME Display Manager). This generates your
  graphical login screen.
- Once you type your password, the system launches your **Desktop Environment** (GNOME). Your wallpaper, docks, and top bar
  load up.

**Ubuntu is ready!**

---

## How Windows Differs

While the hardware phase is identical, Windows handles the software hand-off differently.

```
[UEFI/BIOS] ➔ [Windows Boot Manager] ➔ [winload.efi] ➔ [NT Kernel] ➔ [Session Manager] ➔ [Login/Desktop]

```

### 1. The Bootloader (Windows Boot Manager)

Instead of GRUB, UEFI looks for `bootmgfw.efi` (Windows Boot Manager). Windows Boot Manager reads the Boot Configuration Data
(BCD) store. If it's a standard boot, it immediately calls `winload.efi` to start loading the operating system.

### 2. The NT Kernel and Fast Startup

Windows loads the **Windows NT Kernel** (`ntoskrnl.exe`) and essential system drivers.

- _The Big Difference:_ Modern Windows defaults to a feature called **Fast Startup**. When you shut down Windows, it doesn't
  actually do a fresh boot from scratch next time. It saves a snapshot of the kernel and loaded drivers into a file on your
  hard drive (similar to hibernation). When you turn the PC on, it just copy-pastes that snapshot back into your RAM, which
  makes it boot incredibly fast. Ubuntu always performs a fresh, clean initialization of the kernel by default.

### 3. System Session and Smss.exe

Instead of `systemd`, Windows uses the **Session Manager Subsystem** (`smss.exe`). This process creates the user environment,
starts the Windows subsystem (`win32k.sys`), and brings up the graphics hardware. This is when you see the Windows logo and
the spinning dots.

### 4. LSASS and Explorer

Windows starts the Local Security Authority Subsystem (`lsass.exe`), which brings up the familiar Windows lock screen. Once
you log in, `userinit.exe` launches **Windows Explorer** (`explorer.exe`), which builds your taskbar, desktop icons, and
start menu.

**Windows is ready!**

## Summary Comparison

| Feature                 | Ubuntu                          | Windows                             |
| ----------------------- | ------------------------------- | ----------------------------------- |
| **Bootloader**          | GRUB 2 (Great for dual-booting) | Windows Boot Manager                |
| **Core Core**           | Linux Kernel                    | Windows NT Kernel                   |
| **Primary Init System** | `systemd`                       | `smss.exe` (Session Manager)        |
| **Default Boot Style**  | Fresh initialization every time | "Fast Startup" (Hybrid hibernation) |
| **Interface Loader**    | GDM (GNOME Display Manager)     | LogOnUI & Explorer.exe              |
