To understand **machine code** and **assembly language**, it helps to think of them as the lowest two rungs on the ladder of
how humans communicate with computers.

They are the languages closest to the actual physical hardware of your computer.

---

## 1. Machine Code: The Computer's Native Language

**Machine code** (or machine language) is the absolute lowest-level software. It consists entirely of binary numbers—**1s and
0s** (or hexadecimal numbers used as a shorthand).

It is the _only_ language a computer's Central Processing Unit (CPU) can actually understand and execute directly.

- **What it looks like:** A tiny snippet of machine code might look like this: `10110000 01100001` (or `B0 61` in hex).
- **How it works:** Each combination of 1s and 0s corresponds to a specific physical command inside the CPU. One sequence
  might tell the CPU to "add two numbers," while another means "store this number in memory."
- **The Catch:** Machine code is impossible for humans to read or write efficiently. If you miss a single `0` or `1`, the
  entire program crashes.

---

## 2. Assembly Language: The Human-Readable Bridge

Because writing 1s and 0s is brutal, computer scientists invented **Assembly Language**.

Assembly is a low-level programming language that is a direct, 1-to-1 translation of machine code, but it replaces the
confusing binary numbers with short, human-readable words called **mnemonics** (like `ADD`, `MOV`, `SUB`, or `PUSH`).

- **What it looks like:** Instead of typing `10110000 01100001`, an assembly programmer writes: `MOV AL, 61h` (which means
  "Move the hexadecimal value 61 into a CPU storage slot called the AL register").
- **How it works:** Because the CPU still can't read words, a lightweight program called an **Assembler** is used to
  translate the assembly words back into the raw 1s and 0s of machine code.

## Summary: The Code Hierarchy

To see how they connect, look at how a simple instruction ("Add 5 to a number") moves through the chain:

| Level          | Type of Code      | Example                | Who Understands It?                         |
| -------------- | ----------------- | ---------------------- | ------------------------------------------- |
| **High-Level** | C++, Python, Java | `x = x + 5;`           | Humans (Easy to read and write)             |
| **Low-Level**  | **Assembly**      | `ADD EAX, 5`           | Programmers who need total hardware control |
| **Binary**     | **Machine Code**  | `00000101 00000101...` | The CPU chip itself                         |

### Why do we still use Assembly?

While 99% of software today is written in high-level languages like Python or C++, programmers still use Assembly when they
need **absolute maximum speed** or need to talk directly to hardware components—such as when writing operating system
kernels, device drivers, or programming tiny microchips in smart appliances.
