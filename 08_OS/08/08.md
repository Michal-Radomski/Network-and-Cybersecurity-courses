**Just-In-Time (JIT) compilation** is a feature used by modern language runtimes (like JavaScript's V8, Java's JVM, or .NET)
to blend the speed of a compiled language with the flexibility of an interpreted language.

Instead of translating your code all at once before you run it (like C++), or translating it line-by-line every single time
it runs (like traditional Python), a JIT compiler translates code **while the program is actively running**—literally "just
in time."

---

## How JIT Works: The Workflow

When you launch a program that uses a JIT compiler (like a JavaScript web app), the engine goes through a multi-step
evolution:

1. **The Interpreter Stage:** The engine starts up instantly by turning your source code into **bytecode** and interpreting
   it. This gets the application running without waiting.
2. **The Profiler Stage:** While the code runs, a "profiler" watches closely. It looks for **hot spots**—pieces of code (like
   a `for` loop or a specific function) that are being executed thousands of times.
3. **The JIT Compiler Stage:** Once a function is marked as "hot," the JIT compiler kicks in. It takes that specific chunk of
   bytecode, optimizes it heavily based on how the program is currently behaving, and compiles it directly.

---

## What is the Output of JIT?

The output of a JIT compiler is **raw, highly optimized Machine Code (native binary instructions)** that goes directly into
your computer's RAM.

Because this machine code is stored temporarily in memory, it is executed straight by your physical CPU (Intel, AMD, or Apple
Silicon) at maximum hardware speed.

### Key Characteristics of JIT Output:

- **Ephemeral (Temporary):** The outputted machine code only exists in the computer's volatile memory (RAM) while the program
  is running. It is never saved to your hard drive as a `.exe` or `.out` file. When you close the browser or stop the
  program, that machine code vanishes.
- **Highly Tailored:** Because the JIT compiler compiles the code _while_ it runs, it can see exactly what kind of data is
  passing through. For example, if a JavaScript function can technically accept strings or numbers, but the JIT sees you have
  passed integers into it 5,000 times in a row, it will output machine code specifically optimized _only_ for integers.

---

## Summary: A Real-World Analogy

Imagine you are an English-speaking chef trying to cook a massive recipe written in French:

- **Traditional Interpretation:** You look at line 1, translate it to English, chop an onion. You look at line 2, translate
  it, stir the pot. If a loop tells you to "stir 100 times," you translate the word "stir" 100 separate times. (Slow)
- **Ahead-of-Time Compilation (C++):** You pay a professional translator to translate the entire recipe book into English
  before you ever enter the kitchen. (Fast cooking, but you have to wait for the translation first).
- **JIT Compilation (JS/Java):** You start translating line-by-line to get moving. But suddenly you notice a section that
  says, _"Repeat this exact whisking technique for 20 minutes."_ Instead of translating "whisk" over and over, you stop,
  translate that entire paragraph into perfect English, memorize it, and execute it at full speed for the next 20 minutes.
