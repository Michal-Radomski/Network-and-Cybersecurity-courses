To compile a C file on Ubuntu using GCC, you will use the terminal. By default, Ubuntu creates executables without a specific
file extension (or with a `.out` extension), but you can name the output file whatever you want.

Before starting, make sure your terminal is opened to the folder where your C file (e.g., `main.c`) is saved.

---

## 1. Compiling to a `.out` File

In Linux/Ubuntu, the standard default name for a compiled executable is `a.out`.

### The Default Method:

If you don't specify an output name, GCC will automatically create a file named `a.out`:

```bash
gcc main.c

```

To run this file, you type:

```bash
./a.out

```

### The Custom `.out` Method:

If you want to explicitly name the file something else ending in `.out`, use the `-o` (output) flag:

```bash
gcc main.c -o program.out

```

To run it:

```bash
./program.out

```

---

## 2. Compiling to a `.exe` File

Here is an important distinction: **Linux doesn't use `.exe` files.** The `.exe` extension is a Windows format. On Ubuntu, a
file is made executable by system permissions, not by its file extension.

However, you can still compile a file and _name_ it with a `.exe` extension on Ubuntu. There are two ways to interpret this:

### Scenario A: You just want the file on Ubuntu to have `.exe` in its name

If you just want the extension for cosmetic reasons or specific project requirements, you can use the `-o` flag just like
before:

```bash
gcc main.c -o program.exe

```

Even though it says `.exe`, it is still a Linux binary under the hood. You run it normally in your Ubuntu terminal:

```bash
./program.exe

```

### Scenario B: You want to compile a true Windows `.exe` while using Ubuntu

If you are trying to build a program on Ubuntu that you can copy-paste onto a Windows computer to run, standard `gcc` won't
work. You need to **cross-compile** using a tool called MinGW.

1. **Install the Windows cross-compiler:**

```bash
sudo apt update
sudo apt install mingw-w64

```

2. **Compile the true Windows executable:**

```bash
x86_64-w64-mingw32-gcc main.c -o program.exe

```

_Note: This `program.exe` will now run perfectly on Windows, but it will no longer run directly on Ubuntu without a
compatibility layer like Wine._

---

## Quick Command Summary

| Goal                      | Command                                         | How to Run it on Ubuntu      |
| ------------------------- | ----------------------------------------------- | ---------------------------- |
| Default output            | `gcc main.c`                                    | `./a.out`                    |
| Custom `.out`             | `gcc main.c -o my_prog.out`                     | `./my_prog.out`              |
| Linux binary named `.exe` | `gcc main.c -o my_prog.exe`                     | `./my_prog.exe`              |
| True Windows `.exe`       | `x86_64-w64-mingw32-gcc main.c -o win_prog.exe` | _Transfer to Windows to run_ |
