A **.dll** file stands for **Dynamic Link Library**. It is a file format used in Windows that contains shared code and data
that multiple programs can use at the same time.

Instead of every single software program containing its own copy of standard code (like how to print a document or open a
window), Windows programs share these `.dll` files. This saves a massive amount of hard drive space and RAM.

---

## The Ubuntu (Linux) Equivalent: `.so` Files

On Ubuntu and other Linux-based operating systems, the exact equivalent of a `.dll` file is a **`.so` file**, which stands
for **Shared Object**.

Just like their Windows cousins, `.so` files are libraries of compiled machine code that different programs can call upon
dynamically while running.

---

## Static vs. Dynamic Libraries: The Two Types

In both Windows and Linux, there are two main ways to use code libraries. Understanding the difference explains exactly why
`.dll` and `.so` files are so popular.

### 1. Static Libraries (`.lib` on Windows / `.a` on Linux)

- **How they work:** During the **linking** stage of compiling a program, the linker copies the actual code from the library
  file and pastes it directly inside the final executable program.
- **Pros:** The program is entirely self-contained. You don't have to worry about missing files.
- **Cons:** The executable file size becomes huge. If you have 10 programs that use the same library, you have 10 copies of
  that code wasting space on your drive.

### 2. Dynamic Libraries (`.dll` on Windows / `.so` on Linux)

- **How they work:** The linker does _not_ copy the code into the executable. Instead, it just leaves a note saying, _"Hey
  program, when you launch, go look for this library file on the computer."_
- **Pros:** File sizes stay very small. If the library gets a security update or a bug fix, you only have to update that one
  `.dll` or `.so` file, and every program using it instantly gets the fix.
- **Cons:** If the library file gets deleted, renamed, or corrupted, the program will crash instantly on startup with a
  "Library Not Found" error.

## Quick Reference Table

| Library Type         | Windows Extension             | Ubuntu (Linux) Extension  | What it does                                                           |
| -------------------- | ----------------------------- | ------------------------- | ---------------------------------------------------------------------- |
| **Dynamic / Shared** | `.dll` (Dynamic Link Library) | **`.so`** (Shared Object) | Loaded into memory only when the program runs; shared among many apps. |
| **Static**           | `.lib` (Static Library)       | **`.a`** (Archive)        | Baked directly into the program file when it is built.                 |

If you look inside the `/usr/lib` or `/lib` folders on your Ubuntu system, you will see thousands of `.so` files keeping your
operating system running smoothly behind the scenes!
