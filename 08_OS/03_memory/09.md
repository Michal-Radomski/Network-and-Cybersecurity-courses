**Copy-on-Write (CoW)**, sometimes called implicit sharing, is an optimization strategy used in computer programming and
operating systems to efficiently duplicate modifiable resources (like memory pages or files).

Instead of making an actual copy of the resource right away, the system points both the original and the new reference to the
_same data_. The actual copying is delayed until one of them tries to modify (write to) the data.

Here is a breakdown of how it works, why it’s useful, and where it is used.

---

## How It Works (Step-by-Step)

1. **The Share:** When a resource is duplicated, the system creates a new pointer or reference that points to the exact same
   location in memory as the original. The resource is marked as **read-only** for both.
2. **The Read:** As long as both processes or variables are only _reading_ the data, no copy is ever made. They happily share
   the same memory space.
3. **The Interception:** The moment one process tries to _modify_ or write to the data, the system intercepts this request
   (usually causing a page fault in operating systems).
4. **The Write:** The system duplicates the specific piece of data being changed, applies the modification to the new copy,
   updates the pointer for the writing process, and marks it as writeable. The original data remains unchanged for everyone
   else.

---

## Why Use It? (The Benefits)

- **Memory Efficiency:** If a program duplicates a massive block of data but only needs to read it, CoW saves you from
  wasting RAM on a duplicate.
- **Speed:** Allocating memory and copying data takes time. By delaying or completely avoiding the copy, operations become
  nearly instantaneous.

---

## Common Real-World Examples

### 1. Operating Systems (`fork` in Unix/Linux)

When a process creates a child process using the `fork()` system call, the OS doesn't copy the parent's entire memory space
(which could be gigabytes of data). Instead, it uses CoW. The child process shares the parent's memory. If the child
immediately calls `exec()` to run a completely different program, the old memory is never copied, saving massive amounts of
time and RAM.

### 2. Programming Languages (Swift, C++, Java)

Many languages use CoW for data structures like strings, arrays, or collections.

- In **Swift**, standard collections like `Array` and `Dictionary` are value types but use CoW under the hood. If you pass an
  array to a function, it isn't copied unless that function tries to append or change an element.

### 3. File Systems (Btrfs, ZFS, APFS)

Modern file systems use CoW for snapshots and file cloning. If you "copy" a 10 GB video file on an Apple File System (APFS)
drive, it happens instantly and takes up 0 KB of extra space. Space is only consumed if you edit one of those video files
later.

---

## A Quick Summary Analog

Imagine you and a friend need to read a 500-page textbook for homework. Instead of buying two copies (expensive and heavy),
you share one copy. You both just read it.

However, if your friend wants to highlight a sentence or take notes in the margin, _that_ is the moment they photocopy just
that single page, make their notes on the copy, and binder it into their own notebook. You keep the clean original page, and
they have their modified page. You only paid for a single photocopy of one page, rather than a whole new book.
