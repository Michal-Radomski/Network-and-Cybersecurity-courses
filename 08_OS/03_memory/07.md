In Linux, virtual memory is referred to as **Swap**. When your physical RAM fills up, the Linux kernel moves inactive pages
of memory into designated storage space on your drive.

Linux gives you two different ways to set up this emergency workspace: a **Swap File** or a **Swap Partition**.

---

## 1. What is a Swap Partition?

A swap partition is a dedicated, separate section of your hard drive or SSD formatted exclusively to act as virtual memory.

- **The Setup:** During the installation of Linux (like Ubuntu, Fedora, or Arch), you allocate a specific chunk of storage
  (e.g., 8 GB) to be a "Swap" type partition. The rest of your files go onto a separate partition (like `ext4` or `Btrfs`).
- **How Linux Sees It:** To the operating system, it looks like a completely separate drive that regular files and folders
  cannot access. It is managed directly by the Linux kernel.

### Pros & Cons

- **Pro:** Historically slightly faster and less prone to fragmentation because it's a raw block of data unburdened by a
  standard filesystem.
- **Pro:** Essential if you want to use **Hibernation** (Suspend to Disk), as the system needs a reliable, raw space to dump
  the entire contents of your RAM when powering down.
- **Con:** Highly inflexible. If you run out of swap space, resizing a partition requires unmounting drives, shrinking other
  partitions, and using tools like `gparted`—which carries a risk of data loss.

---

## 2. What is a Swap File?

A swap file is a single, large file that sits right alongside your normal documents, photos, and system files inside your
regular filesystem (usually located at `/swapfile`).

- **The Setup:** Instead of carving out a whole section of the drive, you create a massive, empty file (for example, using
  the `dd` or `fallocate` command) and tell Linux to treat that specific file as virtual memory.
- **How Linux Sees It:** It acts exactly like a swap partition, but it lives within the standard filesystem structure.

### Pros & Cons

- **Pro:** Incredible flexibility. If you suddenly need more swap space because you are compiling a massive program, you can
  create a new swap file or expand the existing one in seconds without rebooting.
- **Pro:** Easy to delete if you decide you have upgraded your physical RAM and no longer need it.
- **Con:** Does not always support hibernation out-of-the-box (it requires extra configuration with offset values).
- **Con:** On older copy-on-write filesystems (like Btrfs), it requires special flags to prevent performance degradation.

---

## Quick Comparison

| Feature                 | Swap Partition                            | Swap File                                            |
| ----------------------- | ----------------------------------------- | ---------------------------------------------------- |
| **Location**            | Its own isolated storage zone.            | Inside your existing filesystem (e.g., `/swapfile`). |
| **Flexibility**         | Rigid. Hard to resize after installation. | Fluid. Can be resized or deleted easily.             |
| **Hibernation Support** | Excellent and native.                     | Requires complex configuration.                      |
| **Modern Default**      | Common in older distros/expert setups.    | Default in modern distros (like Ubuntu and Fedora).  |

## The Modern Verdict

For a long time, system administrators preferred **Swap Partitions** because bypass-the-filesystem speeds were noticeably
faster on old, spinning hard drives (HDDs).

However, with modern, ultra-fast **NVMe SSDs**, the performance difference between a swap file and a swap partition is
practically zero. Because of this, most modern Linux distributions default to using a **Swap File** because the convenience
of being able to resize it on the fly far outweighs the microscopic speed benefit of a partition.
