To understand the difference between **Asynchronous DRAM (ADRAM)** and **Synchronous DRAM (SDRAM)**, you have to look at how
memory communicates with the computer’s processor (CPU).

In short, it all comes down to **timing and coordination**.

---

## 1. Asynchronous DRAM (The Old Way)

Asynchronous DRAM was the standard for computers up until the mid-1990s.

- **How it works:** "Asynchronous" means **not synchronized**. This type of RAM operates independently of the computer's
  system clock. When the CPU wants data from the RAM, it sends a request and then simply has to _wait_ for the RAM to find
  the data and send it back.
- **The Problem:** Because the RAM doesn't know when the CPU is going to ask for data, and the CPU doesn't exactly know when
  the RAM will finish finding it, they have to use a complex system of "handshaking" signals. The CPU often ends up sitting
  idle, waiting for the memory to catch up. This creates a massive performance bottleneck.

---

## 2. Synchronous DRAM (The Modern Way)

Introduced to solve the bottleneck, SDRAM became the foundation for all modern computer memory (including the DDR4 and DDR5
RAM used today).

- **How it works:** "Synchronous" means **in perfect time**. SDRAM is locked directly into the **system clock** of the
  computer. It synchronizes its response times with the CPU’s clock cycles.
- **The Advantage:** Because the RAM and the CPU are ticking to the exact same beat, the CPU knows precisely when the data
  will be ready. It doesn't have to wait or handshake; it can send a request and move on to another task, knowing the data
  will arrive exactly a certain number of clock cycles later. This allows for pipelining, where the RAM can look up the next
  piece of data while sending the current one.

---

## Key Differences At a Glance

| Feature              | Asynchronous DRAM                                | Synchronous DRAM (SDRAM)                                    |
| -------------------- | ------------------------------------------------ | ----------------------------------------------------------- |
| **Clock Alignment**  | Operates independently of the CPU system clock.  | Locked into the CPU system clock.                           |
| **CPU Waiting Time** | High. CPU must wait for the RAM to respond.      | Minimal. CPU knows exactly when data will arrive.           |
| **Efficiency**       | Low. Can only handle one request at a time.      | High. Uses pipelining to queue up multiple requests.        |
| **Era**              | Used in older systems (e.g., 386/486 computers). | Used in all modern systems (DDR memory is a type of SDRAM). |

### The Evolution: What is DDR?

If you buy RAM today, you buy **DDR** (Double Data Rate). DDR is just an advanced type of SDRAM. While original SDRAM sent
data once per clock tick, DDR sends data _twice_ per clock tick (once on the rise of the signal, and once on the fall),
effectively doubling the speed without changing the clock rate.
