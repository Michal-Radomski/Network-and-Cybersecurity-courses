To read or write data, the processor (CPU) and the memory (RAM) have to engage in a highly coordinated dance. Because SDRAM
is synchronized with the system clock, this process happens in precise steps measured in clock cycles.

Here is exactly how a computer reads and writes data at the hardware level.

---

## The Infrastructure: The Three Buses

Data doesn't just teleport; it travels across three physical sets of wires on the motherboard called **buses**:

1. **Address Bus:** The CPU uses this to specify _where_ in the memory the data is located. Think of it like writing an
   address on an envelope.
2. **Data Bus:** The actual highway where the data travels back and forth between the CPU and RAM.
3. **Control Bus:** The command wire. It carries signals that tell the memory what to do (e.g., "Read", "Write", or
   "Refresh").

---

## 1. How the CPU READS from Memory

When the CPU needs a piece of information (like a line of code or an image file), it performs a **Read Operation**:

- **Step 1: Addressing:** The CPU places the exact memory address of the required data onto the **Address Bus**.
- **Step 2: Signaling:** The CPU sends a "Read" command along the **Control Bus**. This wakes up the memory controller and
  tells it to get ready to output data.
- **Step 3: Access Time:** The memory module decodes the address, locates the specific row and column in its transistor grid,
  and retrieves the electrical charge.
- **Step 4: Delivery:** The RAM places the data onto the **Data Bus**. The CPU then opens its internal gates, grabs the data
  off the bus, and moves it into its own super-fast internal registers to work on it.

---

## 2. How the CPU WRITES to Memory

When you save a file, change a variable in a game, or copy text, the CPU performs a **Write Operation**:

- **Step 1: Addressing:** Just like reading, the CPU starts by placing the target destination address onto the **Address
  Bus**.
- **Step 2: Loading Data:** Simultaneously, the CPU places the actual data it wants to save onto the **Data Bus**.
- **Step 3: Signaling:** The CPU sends a "Write" command along the **Control Bus**.
- **Step 4: Storage:** The memory controller detects the write signal, opens the gates to the specified address, and channels
  the electricity from the data bus into the RAM's tiny capacitors, charging them (storing a `1`) or draining them (storing a
  `0`).

---

## Memory Latency: The Tiny Delays

Even though this happens billions of times per second, it isn't instant. You might have seen numbers like **CL16** or
**CL18** when looking at RAM specs. This stands for **CAS Latency** (Column Address Strobe).

Latency is the exact number of clock cycles that pass between the moment the CPU sends the "Read" command (Step 2) and the
moment the RAM actually delivers that data to the Data Bus (Step 4). Lower latency means the RAM responds faster to the CPU's
commands.
