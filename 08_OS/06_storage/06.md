## 1. What is FAT32?

**FAT32 (File Allocation Table 32)** is one of the oldest, most widely compatible file systems in computing history.
Introduced by Microsoft in 1996 with Windows 95 OSR2, it was designed to replace the older FAT16 system and allow computers
to handle larger hard drives.

The name tells you exactly how it works:

- **File Allocation Table:** The system maintains a physical table (a master ledger) near the beginning of the drive. This
  table acts as an index, tracking where every piece of every file is stored.
- **32:** It uses a 32-bit binary number to address data units (though only 28 bits are actually used for addressing, with 4
  bits reserved).

Because of its age, FAT32 lacks modern features like journaling (making it prone to corruption if powered off abruptly) and
file-level security permissions. However, because nearly every operating system—Windows, macOS, Linux, Android, and even game
consoles—can read and write to it, it remains a standard choice for USB flash drives and SD cards.

## 2. Understanding Clustering (The "Cardboard Box" Principle)

To understand how FAT32 manages data, you have to understand **clustering**.

A storage drive is physically divided into tiny sectors (usually 512 bytes each). However, managing millions of tiny 512-byte
sectors individually would overwhelm a file system's index table. To stay efficient, the file system groups these sectors
together into larger, manageable blocks called **clusters** (also known as allocation units).

> **The Cluster Rule:** A cluster is the **smallest allocation unit** of storage. The operating system cannot write a
> fraction of a cluster; it can only assign data to whole clusters.

### The Cardboard Box Analogy

Think of a cluster as a cardboard box of a fixed size—for example, a box that can hold 4 KB of paperwork.

- If you have a tiny receipt that is only **1 KB** in size, you still have to put it in its own 4 KB box. The remaining 3 KB
  of space in that box is wasted and cannot be used by any other file. This wasted space is called **slack space**.
- If you have a massive report that is **10 KB** in size, it won't fit in one box. The file system will break it up, filling
  two 4 KB boxes completely, and putting the remaining 2 KB into a third box.

## 3. How FAT32 Uses Clustering

Under FAT32, the size of a cluster changes depending on how large the storage drive is. Smaller drives use smaller clusters
to save space; larger drives use larger clusters to keep the index table manageable.

Typical FAT32 cluster sizes look like this:

- **Drives under 8 GB:** 4 KB clusters (8 sectors per cluster)
- **Drives 8 GB to 16 GB:** 8 KB clusters (16 sectors per cluster)
- **Drives 16 GB to 32 GB:** 16 KB clusters (32 sectors per cluster)

### How FAT32 Chains Clusters Together

Files are rarely stored in one neat, continuous line on a drive. They get scattered. FAT32 handles this by creating a
**cluster chain** inside the File Allocation Table.

If the operating system wants to read a file called `VacationPhoto.jpg`, the process works like this:

1. The OS looks at the directory entry for `VacationPhoto.jpg` and sees that the file starts at **Cluster #102**.
2. The OS reads the data inside physical **Cluster #102**.
3. The OS then checks the _File Allocation Table_ entry for slot #102 to see where to go next. The table says: _"The next
   part is in Cluster #105."_
4. The OS reads **Cluster #105**, checks slot #105 in the table, and sees: _"Next is Cluster #110."_
5. This continues until the OS hits a special marker in the table (usually written as `0xFFFFFFF`), which translates to **End
   of File (EOF)**. The OS now knows the entire file has been successfully retrieved.

## 4. The Hard Limitations of FAT32

Because FAT32 is bound by its 32-bit architecture and its clustering system, it suffers from two major limitations that make
it obsolete for modern internal drives:

- **The 4 GB File Size Limit:** This is the most famous FAT32 limitation. Because of the way file sizes are recorded in a
  32-bit architecture, a single individual file **cannot be larger than $4\text{ GB} - 1\text{ Byte}$
  ($4,294,967,295\text{ bytes}$)**. If you try to download a 5 GB 4K movie or a large zip file onto a FAT32 drive, the
  transfer will fail, even if the drive has 1 TB of free space.
- **The 32 GB Partition Limit (Windows Native):** By default, the maximum total number of clusters FAT32 can comfortably
  track is about 268 million ($2^{28}$). Because of this, Microsoft artificially limits the creation of FAT32 partitions to
  **32 GB** within the Windows formatting tool (forcing users toward NTFS or exFAT instead). While the format _can_
  technically support up to 2 TB or even 16 TB partitions using third-party tools, doing so requires massive cluster sizes
  (up to 64 KB), which leads to extreme amounts of wasted "slack space."
