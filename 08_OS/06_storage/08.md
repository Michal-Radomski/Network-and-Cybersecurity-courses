The **OS Page Cache** (often called the disk cache) is a crucial performance optimization feature built into modern operating
systems like Linux, Windows, and macOS.

In short: **It uses your computer’s unused RAM to store copies of data recently read from or written to your storage drive.**
Because RAM is thousands of times faster than an SSD (and millions of times faster than a mechanical HDD), the page cache
prevents your system from becoming bottlenecked by slow disk I/O operations.

---

## 1. How It Works: The Core Mechanisms

The page cache operates on a simple principle of physics: reading from memory is fast; reading from storage is slow. The OS
divides physical memory into uniform chunks called **pages** (usually 4KB).

The cache handles two primary operations: **Cached Reads** and **Cached Writes**.

### Cached Reads (Page Faults vs. Cache Hits)

When a program requests data from a file, the OS doesn't immediately go to the drive. Instead:

1. It checks the Page Cache to see if those specific file pages are already in RAM.
2. **Cache Hit:** If the data is there, the OS copies it directly from RAM to the program. This happens almost instantly.
3. **Cache Miss (Page Fault):** If the data isn't there, the OS hits the slow drive, copies the data into the Page Cache for
   future use, and then hands it to the program.

### Cached Writes (Dirty Pages)

When a program writes data to a file, the OS optimizes for speed using a technique called **write-back caching**:

1. The OS writes the data directly into the Page Cache in RAM and tells the program, _"Done!"_ (The program thinks the data
   is safely on the disk, but it isn't yet).
2. The pages in RAM that have been modified but not yet saved to the physical disk are marked as **"dirty pages."**
3. In the background, a kernel thread (like `flusher` threads in Linux) quietly gathers these dirty pages and flushes
   (writes) them down to the physical storage drive.

---

## 2. Memory Management: "Free RAM is Wasted RAM"

A common point of confusion for users looking at Task Manager or Resource Monitor is seeing that their operating system is
using almost all of their RAM, even when no heavy programs are running. This is the Page Cache at work.

Operating systems operate on the philosophy that **free RAM is wasted RAM**.

- If you have 16GB of RAM and your apps are only using 6GB, the OS will aggressively fill the remaining 10GB with cached
  files you've recently used.
- **The Cache is Low-Priority:** The moment a program needs more memory (e.g., you launch a heavy game or a video editor),
  the OS instantly drops chunks of the page cache to free up physical RAM for the application. No closing or manual clearing
  required.

---

## 3. The Eviction Policy (LRU)

Because RAM is finite, the OS must eventually kick older data out of the cache to make room for new data. It does this using
the **Least Recently Used (LRU)** algorithm or variations of it.

The OS maintains a list of pages. Pages that are actively being read stay at the top. Pages that haven’t been touched in a
long time sink to the bottom. When the cache fills up, the pages at the bottom are evicted (discarded if clean, or flushed to
disk and then discarded if dirty).

---

## 4. The Risk: Sudden Power Loss & `fsync`

While write-back caching makes your computer incredibly snappy, it introduces a dangerous vulnerability: **data loss during a
power outage or system crash.**

If a program writes an important database entry, the OS marks it as a "dirty page" in RAM. If the power cord is pulled a
second later, before the background flusher can write it to the SSD, that data is gone forever.

### The Solution: `fsync()`

To protect critical data, operating systems provide a system call called `fsync` (File Sync).

- When an application uses `fsync`, it forces the OS to bypass the lazy background flushing mechanism.
- The OS halts the application and immediately pushes those specific dirty pages directly to the physical storage hardware.
- Databases (like SQLite, MySQL, or PostgreSQL) use `fsync` constantly to ensure that when they say a transaction is
  completed, it is safely committed to physical platters or flash cells.

---

## Summary of Benefits

- **Massive Speedups:** Frequently read applications, system libraries, and game files load instantly on subsequent boots
  because they are pulled straight from RAM.
- **Reduced Drive Wear:** By batching multiple small writes into RAM and flushing them all at once, the page cache minimizes
  write cycles, which extends the lifespan of SSDs.
