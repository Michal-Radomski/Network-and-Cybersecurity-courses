Checking partition alignment on Ubuntu is quick and straightforward. You can do it using either the command line (which is
the most precise method) or a graphical interface.

Here is how to check your alignment and verify if your drives are performing optimally.

---

## Method 1: Using the Command Line (Recommended)

The most reliable tool for checking partition alignment is `fdisk`.

### Step 1: List your partitions

Open your terminal and run the following command with `sudo` privileges:

```bash
sudo fdisk -l

```

### Step 2: Analyze the "Start" sector

Look at the output table for your drive (e.g., `/dev/sda` or `/dev/nvme0n1`). You will see a column labeled **Start**.

```text
Device          Start        End    Sectors   Size Type
/dev/nvme0n1p1   2048    2099199    2097152     1G EFI System
/dev/nvme0n1p2  2099200  4194303    2095104     1G Linux filesystem
/dev/nvme0n1p3  4194304 976771071  972576768 463.8G Linux filesystem

```

### Step 3: Do the math

To verify if a partition is aligned, **divide the number in the "Start" column by 8**.

- If the result is a **whole number** (no decimals), your partition is **perfectly aligned**.
- If the result has a decimal portion (e.g., `.5` or `.25`), your partition is **misaligned**.

> **Why 8?** `fdisk` displays sizes in legacy 512-byte sectors. Modern physical sectors and SSD blocks are 4096 bytes
> ($4\text{ KB}$). Since $4096 / 512 = 8$, the starting sector number must be perfectly divisible by 8 to line up with the
> hardware blocks. In the example above, the first partition starts at **2048**. $2048 / 8 = 256$ (a whole number, so it is
> perfectly aligned).

---

## Method 2: Automatic Verification via `parted`

If you don't want to do the math yourself, the `parted` utility can check alignment for you and give a simple "optimal" or
"not aligned" verdict.

1. Start `parted` for the drive you want to check (replace `/dev/sda` with your actual drive identifier):

```bash
sudo parted /dev/sda

```

2. Switch the display unit to sectors by typing:

```text
unit s

```

3. Print the partition table to see your partition numbers:

```text
print

```

4. Ask it to check the alignment of a specific partition number (e.g., partition 1):

```text
align-check optimal 1

```

5. It will output either `1 aligned` (Success) or `1 not aligned` (Misaligned).
6. Type `quit` to exit.

---

## Method 3: The Graphical Way (GUI)

If you prefer a visual tool, you can use Ubuntu’s built-in **Disks** utility or install **GParted**.

### Using GParted

1. Install it via the terminal if you don't have it:

```bash
sudo apt update && sudo apt install gparted

```

2. Launch **GParted** from your application menu.
3. Select your drive from the dropdown menu in the top-right corner.
4. Right-click on a partition and select **Information**.
5. Look at the **First sector** value. Just like the `fdisk` method, if this number is cleanly divisible by 8 (or if GParted
   shows it starts exactly at `1.00 MiB`), it is correctly aligned.

---

## What should you do if a drive is misaligned?

If you discovered that an older HDD or a cloned SSD is misaligned, you will notice degraded read/write speeds.

To fix it without losing data, you generally have to use a tool like **GParted Live USB** to safely resize/move the
partition, ensuring that the "Align to" dropdown is set to **MiB** (which forces the modern 1MB alignment standard). Always
back up your important data before modifying partition boundaries!
