**VFAT (Virtual File Allocation Table)** is an extension of the original FAT16 file system, introduced by Microsoft with
**Windows 95**.

To understand what VFAT is, you have to look at the massive limitation it solved back in the mid-1990s: the dreaded **8.3
filename rule**.

---

## 1. The Problem: The 8.3 Rule

In the days of MS-DOS and early Windows (like Windows 3.1), file systems could only handle filenames that were a maximum of
**8 characters long**, followed by a **3-character extension** (e.g., `DOCUMENT.TXT` or `MYREPORT.DOC`).

If you tried to name a file `FinancialBudget2026.xlsx`, the system simply wouldn't allow it. Users had to resort to confusing
abbreviations like `FINBUD26.XLS`.

---

## 2. The Solution: How VFAT Works

When Windows 95 rolled around, users expected to use normal, human-readable names. Microsoft created VFAT to allow **Long
File Names (LFN)**—up to 255 characters—while remaining completely backward compatible with old MS-DOS systems.

VFAT achieved this using a clever, invisible trick in the file system structure:

- **The Dual-Identity Trick:** When you save a file named `Quarterly_Report_2026.pdf`, VFAT actually creates **two** entries
  in the directory index.
- **The Real Name:** It saves the full name (`Quarterly_Report_2026.pdf`) across a series of hidden, specially marked
  directory slots that older OS versions are programmed to ignore.
- **The "Alias" (8.3 Name):** In the main slot, it automatically generates a shortened, MS-DOS-friendly alias using a tilde
  (`~`) and a number—for example, `QUARTE~1.PDF`.

Because of this dual setup, if you took that drive and plugged it into an ancient computer running MS-DOS 6.22, the old
machine wouldn't crash. It would just look at the drive, ignore the long-name slots, and display `QUARTE~1.PDF`. If you
plugged it back into Windows 95 or newer, you'd see `Quarterly_Report_2026.pdf` again.

---

## 3. VFAT vs. FAT32 (Common Confusion)

People often get VFAT and FAT32 confused because they were introduced around the same time, but they solve entirely different
problems:

- **VFAT is an _extension_ that allows long filenames.** It can sit on top of FAT12, FAT16, or FAT32.
- **FAT32 is a _layout structure_ that allows larger drive sizes.** > 📁 **In the real world:** When you format a USB drive
  or an SD card as "FAT32" today, you are automatically using VFAT. The underlying architecture is FAT32 (allowing the drive
  to be large), but the mechanism handling your filenames is VFAT (allowing you to use capital letters, spaces, and long
  names like `My Vacation Video.mp4`).

---

## 4. VFAT Limitations

While VFAT was a massive leap forward in 1995, it carries the exact same structural limitations as the FAT systems it
attaches to:

- **No Journaling:** It doesn't log writes, making it vulnerable to corruption if a drive is pulled out mid-transfer.
- **No Permissions:** It lacks modern security, encryption, or compression features.
- **The 4GB Barrier:** When paired with FAT32, it still cannot handle individual files larger than 4GB.

Today, VFAT is strictly a background technology. While we don't think about it anymore, it is the reason you can use spaces
and long names on standard USB flash drives.
