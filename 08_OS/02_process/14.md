By default, the call stack size limit for Node.js running on a 64-bit operating system (like 64-bit Ubuntu) is **984 KB**
(often rounded to **1 MB**).

This limit is defined internally by Google's V8 engine to prevent infinite recursion from freezing your machine.

---

### How to Check It on Your System

Because Node.js releases shift over time, you can verify the exact default value configured on your current environment by
running this terminal command:

```bash
node --v8-options | grep -B0 -A1 "stack-size"

```

The output will typically show the value in kilobytes:

```text
  --stack-size (default stack size limit in KiB)
        type: int  default: 984

```

### Can You Change It?

Yes. If your application relies on deeply nested recursion that legitimately requires a larger call stack, you can bump it up
manually using the `--stack-size` flag when executing your script.

For example, to double it to roughly 2 MB (2000 KB), use:

```bash
node --stack-size=2000 index.js

```

> ⚠️ **Note:** Do not confuse **Stack memory** with **Heap memory**. The stack tracks active function calls and primitive
> local variables. If you are running into memory crashes due to massive objects, arrays, or streams, you are likely hitting
> the V8 **Heap** limit (which dynamically scales but traditionally limits to ~2 GB to 4 GB depending on your Node.js version
> and host limits). You adjust that limit using `--max-old-space-size` instead.
