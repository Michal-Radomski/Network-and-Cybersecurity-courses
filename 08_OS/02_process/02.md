An **ELF (Executable and Linkable Format) header** is the data structure at the very beginning of an ELF file. You can think
of it as the "table of contents" or the "ID card" for a compiled program or library on operating systems like Linux.

Its primary job is to tell the operating system's loader and the linker exactly how to read and interpret the rest of the
binary file.

---

## What Information is Inside an ELF Header?

The header contains critical metadata. If you look at it in code (usually defined in `elf.h`), it includes fields for:

- **The Magic Number:** The first 4 bytes are always `0x7F`, followed by `'E'`, `'L'`, and `'F'` (in hex: `7f 45 4c 46`).
  This tells the OS, "Hey, I am definitely an ELF file."
- **Architecture & Class:** Identifies whether the binary is for a **32-bit** or **64-bit** system, and whether it uses
  Big-Endian or Little-Endian byte ordering.
- **Target Machine:** Specifies the CPU architecture the file was compiled for (e.g., x86-64, ARM, MIPS).
- **File Type:** Defines what kind of ELF file it is:
- `ET_EXEC` (Executable file)
- `ET_DYN` (Shared library / Position Independent Executable)
- `ET_REL` (Relocatable object file, before linking)
- `ET_CORE` (Core dump from a crash)

- **Entry Point Address:** The memory address where the CPU should start executing code when the program runs.
- **Table Offsets and Sizes:** It points to where the **Program Header Table** and **Section Header Table** start later in
  the file, and how large they are.

---

## How to View an ELF Header

If you are on a Linux or Unix-like system, you can easily peek inside an ELF header using the command-line tool `readelf`.

By running the following command on any compiled binary (like `/bin/ls`):

```bash
readelf -h /bin/ls

```

You will get an output that looks something like this:

```text
ELF Header:
  Magic:   7f 45 4c 46 02 01 01 00 00 00 00 00 00 00 00 00
  Class:                             ELF64
  Data:                              2's complement, little endian
  Version:                           1 (current)
  OS/ABI:                            UNIX - System V
  Type:                              DYN (Position-Independent Executable file)
  Machine:                           Advanced Micro Devices X86-64
  Version:                           0x1
  Entry point address:               0x6ab0
  Start of program headers:          64 (bytes into file)
  Start of section headers:          143248 (bytes into file)
  ...

```

Without this header, the operating system would just see a massive, unorganized blob of 1s and 0s and wouldn't know how to
execute the program.
