In computing, the terms **program** and **process** are closely related, but they represent two entirely different stages of
execution.

Think of it this way: a **program** is like a recipe in a cookbook, while a **process** is the actual act of baking the cake.

---

## 1. What is a Program?

A program is a passive entity. It is a file containing a set of written instructions (code) stored on your hard drive or
solid-state drive (SSD).

- **State:** Passive (it just sits there doing nothing until you open it).
- **Lifespan:** Persistent. It stays on your disk until you explicitly delete it.
- **Memory:** It does not consume RAM (Random Access Memory) or CPU power while stored.
- **Examples:** `chrome.exe`, `spotify.app`, or a Python script like `script.py`.

---

## 2. What is a Process?

A process is an active entity. When you double-click a program, the operating system loads those instructions into the
computer's RAM and begins executing them. A process is a "program in execution."

- **State:** Active (it is actively running, using the CPU, and changing data).
- **Lifespan:** Transient. It begins when started and disappears when closed or terminated.
- **Memory:** It requires a dedicated block of RAM and resources to run.

### Components of a Process

When a program becomes a process, the operating system allocates a specific memory structure for it, which includes:

- **Text Section:** The compiled machine code.
- **Data Section:** Global and static variables.
- **Heap:** Memory dynamically allocated during runtime.
- **Stack:** Temporary data like function parameters, return addresses, and local variables.

---

## Summary of Key Differences

| Feature          | Program                                | Process                                |
| ---------------- | -------------------------------------- | -------------------------------------- |
| **Nature**       | Passive entity (stored on disk).       | Active entity (executing in RAM).      |
| **Lifespan**     | Long-lasting (stored until deleted).   | Short-lived (ends when closed).        |
| **Resources**    | Consumes disk space only.              | Consumes RAM, CPU, and I/O resources.  |
| **Relationship** | One program can create many processes. | A process belongs to a single program. |

> 💡 **Fun Fact:** You can open multiple windows of the same web browser (one program) to look at different websites. Each of
> those windows or tabs runs as a separate **process**, all originating from that single **program** file on your computer.
