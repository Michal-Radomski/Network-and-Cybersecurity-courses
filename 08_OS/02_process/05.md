A **Program Counter (PC)**—also sometimes called an **instruction pointer**—is a dedicated, high-speed storage location (a
register) inside a computer's Central Processing Unit (CPU).

Think of it as the CPU's digital bookmark. Its sole job is to **hold the memory address of the next instruction** that the
computer needs to execute.

---

## How It Works in the Fetch-Decode-Execute Cycle

To understand the Program Counter, you have to look at how a CPU processes information. A computer program is just a long
list of instructions stored in the computer's memory (RAM). The CPU goes through these instructions one by one using a
continuous loop:

1. **Fetch:** The CPU looks at the Program Counter to see where to find the next instruction in the RAM. It goes to that
   memory location and grabs (fetches) the instruction.
2. **Increment:** As soon as the instruction is fetched, the Program Counter automatically updates (increments) to point to
   the _very next_ instruction in line.
3. **Decode & Execute:** The CPU figures out what the fetched instruction means and actually carries it out.

---

## Changing the Flow (Jumps and Branches)

Normally, the PC just counts upward sequentially ($PC + 1$, $PC + 2$, etc.), moving through a program in a straight line.
However, programs often have logic like loops (`for`, `while`) or conditions (`if/else`).

When the CPU encounters a "jump" or "branch" instruction, the normal sequence is broken:

- The CPU overwrites the Program Counter with a completely new memory address.
- On the next cycle, the CPU fetches from this new address, effectively "jumping" to a different part of the code.

> **Analogy:** Imagine reading a choose-your-own-adventure book. Normally, you read page 1, then page 2, then page 3 (like
> the PC incrementing normally). But if page 3 says, _"If you want to fight the dragon, go to page 50,"_ you change your
> mental bookmark straight to page 50. That is exactly what a program counter does during a branch instruction.
