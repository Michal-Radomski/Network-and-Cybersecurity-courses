It is absolutely possible! Node.js provides a few built-in ways to check the heap memory size either programmatically from
inside your script or globally from the command line.

Here are the most common and effective methods to do it:

### 1. Programmatically via `process.memoryUsage()`

If you want to check the memory from within your running JavaScript code, you can use `process.memoryUsage()`. This returns
an object containing memory metrics in **bytes**.

```javascript
const memoryData = process.memoryUsage();

// Function to convert bytes to Megabytes (MB)
const toMB = (bytes) => `${(bytes / 1024 / 1024).toFixed(2)} MB`;

console.log({
  rss: toMB(memoryData.rss), // Total memory allocated for the process
  heapTotal: toMB(memoryData.heapTotal), // Total size of the allocated heap
  heapUsed: toMB(memoryData.heapUsed), // Memory actually being used right now
  external: toMB(memoryData.external), // Memory used by C++ objects bound to V8
});
```

- **`heapTotal`**: The amount of memory the V8 engine has allocated for the heap. It can grow as needed.
- **`heapUsed`**: The actual amount of memory your variables, objects, and closures are occupying right now. **This is
  usually the number you care about most.**

---

### 2. Detailed Statistics via the `v8` Module

If you want to know the absolute maximum limit your heap can reach before Node.js crashes with an "Out of Memory" error, you
can use the built-in `v8` module.

```javascript
const v8 = require("v8");

const heapStats = v8.getHeapStatistics();
const limitInMB = (heapStats.heap_size_limit / 1024 / 1024).toFixed(2);

console.log(`Your current JS Heap Limit is: ${limitInMB} MB`);
```

---

### 3. Print Memory Summary on Exit

If you don't want to modify your script's code but want to see a summary of the memory allocation when the script finishes
running, you can pass a flag via your terminal:

```bash
node --v8-options | grep heap
# Or run your script and force it to print basic V8 internal stats:
node --trace-gc your-script.js

```

---

### Bonus: Need to increase it?

By default, Node.js limits the heap size based on your available system memory (historically defaulting around 1.5 GB on
older 64-bit systems). If your script is processing massive amounts of data and crashes, you can expand the heap limit using
the `--max-old-space-size` flag when executing your script:

````bash
# Sets the maximum heap limit to 4096 Megabytes (4 GB)
node --max-old-space-size=4096 your-script.js
````
