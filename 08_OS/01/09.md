**CPU Registers** are the smallest, fastest, and most immediate storage locations available to a computer's processor.

If system RAM is like a storage warehouse down the street, and Cache is like a bookshelf in your room, **Registers are like
the tools currently in your hands**. They are built directly into the CPU core itself and operate at the exact speed of the
processor, allowing data to be accessed in a fraction of a nanosecond.

The CPU cannot perform any math or logic operations on data while it sits in RAM or Cache; the data must first be loaded into
a register.

---

## Key Types of CPU Registers

Registers are generally divided into two main categories: **General-Purpose** (which software can use freely) and
**Special-Purpose** (which have dedicated hardware jobs).

### 1. Special-Purpose Registers

These registers have fixed, specific roles dictated by the CPU architecture to keep programs running in the correct order.

- **Program Counter (PC) / Instruction Pointer (IP):** This holds the memory address of the _next_ instruction waiting to be
  executed. As soon as the current instruction finishes, the PC updates to point to the next step.
- **Instruction Register (IR):** Holds the instruction that is _currently_ being decoded and executed by the Control Unit.
- **Memory Address Register (MAR):** Holds the physical memory address that the CPU currently wants to read from or write to.
- **Memory Buffer Register (MBR) / Data Register (MDR):** Holds the actual data that was just read from memory, or the data
  waiting to be written out to memory.
- **Stack Pointer (SP):** Points to the top of the "stack" in memory, which keeps track of temporary data, local variables,
  and return addresses when functions are called.

### 2. General-Purpose Registers (GPRs)

These are the workhorses used by software developers and compilers to hold temporary data during calculations.

If a program wants to calculate $X = A + B$, the CPU will load the value of $A$ into one GPR, load $B$ into a second GPR,
tell the ALU to add them, and store the result in a third GPR.

Depending on your computer's architecture, these have different names:

- **In 32-bit systems (x86):** They are named `EAX`, `EBX`, `ECX`, `EDX`, etc.
- **In 64-bit systems (x64):** They are expanded and named `RAX`, `RBX`, `RCX`, `RDX`, and `R8` through `R15`.
- **In ARM systems (Phones/Macs):** They are simply numbered, like `X0`, `X1`, `X2`, etc.

---

## Summary: The Memory Hierarchy

To see where registers fit into the overall speed and size hierarchy of a computer:

$$\text{Registers (Fastest / Smallest)} \rightarrow \text{L1/L2/L3 Cache} \rightarrow \text{System RAM} \rightarrow \text{SSD/Storage (Slowest / Largest)}$$

While a modern CPU might have gigabytes of RAM and megabytes of Cache, it usually only has a few dozen registers, totaling
just a few hundred bytes of storage. However, without them, the CPU wouldn't have a place to actually do its work.
