A **file system** is the method and data structure that an operating system uses to control how data is stored and retrieved.
Without a file system, information placed in a storage medium would be one large body of data with no way to tell where one
piece of information stops and the next begins.

Think of it like a massive digital library: the storage drive is the empty building, and the file system is the system of
bookshelves, catalog cards, and indexing rules that keep every book organized and easy to find.

---

## Core Functions of a File System

Every file system handles a few fundamental responsibilities:

- **Space Management:** It divides the physical space of a storage drive into small, manageable blocks and tracks which
  blocks are holding data and which are empty.
- **Filename and Directory Organization:** It allows you to give files human-readable names and group them into folders
  (directories) rather than remembering raw memory addresses.
- **Metadata Tracking:** It stores extra information about every file, such as its size, the date it was created, when it was
  last modified, and file permissions (who is allowed to read or write to it).

---

## Common Types of File Systems

Different operating systems use different file systems tailored to their specific architecture and performance needs.

### 1. ext4 (Fourth Extended Filesystem)

- **Primary OS:** Linux (including Ubuntu).
- **Key Feature:** It is a **journaling** file system. It keeps a temporary log (journal) of changes it intends to make
  before actually writing them to the disk. If your computer suddenly loses power, the kernel reads the journal to fix any
  corrupted or half-written files instantly upon reboot.

### 2. NTFS (New Technology File System)

- **Primary OS:** Windows.
- **Key Feature:** Highly secure, supports large file sizes, and features advanced file permissions, encryption, and
  compression capabilities.

### 3. APFS (Apple File System)

- **Primary OS:** macOS and iOS.
- **Key Feature:** Optimized specifically for Solid State Drives (SSDs) and flash storage. It features strong encryption,
  space-sharing, and fast directory sizing.

### 4. FAT32 / exFAT

- **Primary OS:** Universal (Windows, Mac, Linux, gaming consoles, cameras).
- **Key Feature:** Max compatibility. **FAT32** is an older standard that cannot hold individual files larger than
  $4\text{GB}$. **exFAT** is the modern upgrade that removes this limitation, making it the default choice for USB thumb
  drives and SD cards that need to work seamlessly across different devices.

---

## The Abstract Layer: Pathnames

To make navigating files identical across different hardware, the file system creates an abstract hierarchy of paths.

- **In Windows:** The architecture uses drive letters as roots (e.g., `C:\Users\Name\Documents\file.txt`).
- **In Linux/Ubuntu:** Everything starts from a single root directory denoted by a forward slash (e.g.,
  `/home/name/documents/file.txt`). Even separate physical drives are "mounted" as folders inside this single root tree.
