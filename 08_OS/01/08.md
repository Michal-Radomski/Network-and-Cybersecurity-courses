Inside the CPU, these components work together like a highly synchronized factory to execute instructions and process data.

Here is what each of these components does, broken down by its role:

---

## The Control and Processing Units

### 1. ALU (Arithmetic Logic Unit)

The **ALU** is the brain's calculator. It is the specific part of the CPU that performs all mathematical and logical
operations.

- **Arithmetic:** Addition, subtraction, multiplication, and division.
- **Logic:** Comparisons like `AND`, `OR`, `NOT`, and checking if one number is greater than, less than, or equal to another
  (`A > B`).

### 2. CU (Control Unit)

The **CU** acts as the manager or director of the CPU. It doesn't process data itself; instead, it directs the flow of data.

- It fetches instructions from the system memory.
- It decodes those instructions to figure out what needs to be done.
- It signals the ALU, registers, and other components to execute the instruction.

---

## The Memory Managers

### 3. MMU (Memory Management Unit)

The **MMU** is a hardware component responsible for handling the CPU's access to memory. Its primary job is
**Virtual-to-Physical Address Translation**.

- Modern operating systems use "virtual memory" so apps think they have a massive, continuous block of RAM.
- The MMU instantly translates those virtual addresses into the actual, physical locations on your physical RAM chips.
- It also enforces memory protection, ensuring one application isn't allowed to spy on or overwrite the memory of another
  app.

---

## The Cache Hierarchy (L1, L2, L3)

System RAM is relatively slow compared to the blazing speed of a CPU. To prevent the CPU from sitting idle while waiting for
data, CPUs use ultra-fast, built-in memory called **Cache**. Cache stores copies of the most frequently used data.

It is organized into three levels, moving from fastest/smallest to slowest/largest:

| Cache Level              | Speed                                                  | Size                                                   | Location & Access                                                                                                                                              |
| ------------------------ | ------------------------------------------------------ | ------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **L1 Cache** _(Level 1)_ | **Blazing Fast** (Matches CPU speed)                   | **Very Small** (usually 32KB to 128KB per core)        | Built directly into each individual CPU core. It is divided into two parts: L1i (for instructions) and L1d (for data).                                         |
| **L2 Cache** _(Level 2)_ | **Fast** (Slower than L1, faster than L3)              | **Medium** (usually 512KB to a few Megabytes per core) | Built into each core (or shared between a tiny cluster of cores). It holds data the CPU thinks it might need next if it isn't in L1.                           |
| **L3 Cache** _(Level 3)_ | **Moderate** (Slower than L2, but way faster than RAM) | **Large** (usually 16MB to 96MB+ total)                | Shared across **all** CPU cores. If Core 1 loads data from RAM into the L3 cache, Core 4 can grab it quickly without going all the way back to the system RAM. |

### How the Cache Search Works:

When the CPU needs a piece of data, it checks **L1** first. If it's there (a _cache hit_), it processes it instantly. If not
(a _cache miss_), it checks **L2**, then **L3**. If the data isn't in any of the caches, it has to make the long journey out
to the system **RAM**, which slows things down temporarily.
