To understand the difference between **user space** and **kernel space**, it helps to think of your computer's operating
system as a high-security building.

- **User space** is the public lobby where anyone can hang out.
- **Kernel space** is the secure vault in the back where the most critical operations happen.

Here is a breakdown of how they work, why they are separated, and how they talk to each other.

---

## 1. What is User Space?

User space is the area of memory where all your daily applications run. When you open a web browser, play a video game, or
write a document, those programs live and play in user space.

- **Low Privilege:** Applications here have limited permissions. They cannot directly touch the computer's hardware (like the
  hard drive or network card).
- **Safety Net:** If an application in user space crashes (like your browser freezing), it only crashes _that_ specific
  application. The rest of the computer keeps running smoothly.

## 2. What is Kernel Space?

The **kernel** is the absolute core of the operating system. Kernel space is the privileged area of memory reserved
exclusively for the kernel, device drivers, and core OS extensions.

- **High Privilege:** The kernel has unrestricted access to all hardware, CPU instructions, and memory. It manages everything
  from memory allocation to telling the CPU which task to process next.
- **High Risk:** Because it has total control, a crash in kernel space is catastrophic. If something goes wrong here, the
  entire system crashes—resulting in the infamous "Blue Screen of Death" (BSOD) on Windows or a "Kernel Panic" on Mac/Linux.

---

## How They Compare

| Feature              | User Space                                        | Kernel Space                          |
| -------------------- | ------------------------------------------------- | ------------------------------------- |
| **Privilege Level**  | Low (User Mode / Ring 3)                          | High (Kernel Mode / Ring 0)           |
| **What Runs Here**   | Web browsers, games, media players, text editors. | The core OS kernel, hardware drivers. |
| **Hardware Access**  | Indirect (must ask permission).                   | Direct and unrestricted.              |
| **Crash Impact**     | Only the specific app closes.                     | The entire system crashes.            |
| **Memory Available** | Limited to what the OS allocates.                 | Can access all physical memory.       |

---

## The Guard at the Gate: System Calls

Applications in user space constantly need to use hardware. For example, when you want to save a file in a text editor, the
editor (in user space) needs to write data to the hard drive (controlled by kernel space).

Because user space apps aren't allowed to touch the hardware directly, they use a mechanism called a **System Call
(Syscall)**.

1. **The Request:** The user space app triggers a system call (essentially knocking on the vault door).
2. **The Switch:** The CPU switches from "User Mode" to "Kernel Mode."
3. **The Execution:** The kernel verifies the request is safe, performs the task (writing to the disk), and finishes.
4. **The Return:** The CPU switches back to "User Mode" and hands control back to the app.

This strict separation is what keeps modern computers stable and secure. Without it, a buggy website could accidentally wipe
your entire hard drive!
