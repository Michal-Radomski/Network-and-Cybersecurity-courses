The `.well-known` directory is a standardized, publicly accessible folder that web servers use to demonstrate ownership of a
domain or to share important metadata about a website with automated tools.

If you look at the root directory of a modern website, you will often see it tucked inside as `/.well-known/`. It was
established by the Internet Engineering Task Force (IETF) under **RFC 8615** to solve a specific problem: giving external
services a predictable, uniform place to find site-wide configuration files.

---

## Why does it exist?

Before `.well-known`, if a service (like an SSL certificate provider or an app) needed to verify something about your
website, it had to guess where to look. One service might look for `yourdomain.com/verify.txt`, while another might look for
`yourdomain.com/security/config`.

The `.well-known` directory standardizes this. Because it always sits at the root of a domain, automated bots and services
know exactly where to go.

---

## Common Uses of `.well-known`

You will most frequently encounter this directory when dealing with security, encryption, and web standards. Here are the
most common files found inside it:

### 1. Let's Encrypt / SSL Verification (`/.well-known/acme-challenge/`)

When you get a free SSL/TLS certificate from Let's Encrypt, their automated system needs to prove that you actually own the
domain. It does this by asking your server to place a unique cryptographic token inside the `acme-challenge` folder. Let's
Encrypt's bots check that URL, and if the token matches, your SSL certificate is issued.

### 2. Security Contact Info (`/.well-known/security.txt`)

This file is the internet equivalent of a "See Something, Say Something" sign. It contains contact information (like an email
address or PGP key) for the website's security team. If an ethical hacker finds a security vulnerability on the site, they
look here first to find out how to safely report it.

### 3. Passwords and Managers (`/.well-known/change-password`)

Many modern web browsers and password managers (like Apple Keychain or 1Password) look for this specific URL. If you click
"Change Password" inside your password manager, it will automatically redirect you to
`yourdomain.com/.well-known/change-password`, sending you straight to the website's actual password reset page instead of
making you hunt for it.

### 4. Mobile App Linking (`/.well-known/assetlinks.json` or `apple-app-site-association`)

If you have a mobile app (iOS or Android) associated with your website, these files tell the smartphone operating system that
your app and your website are legally connected. This is what allows a link clicked in a mobile browser to seamlessly open
right inside your phone's native app.

---

## Do you need to create it manually?

Usually, **no**.

Most of the time, content management systems (like WordPress), security plugins, or server automation tools (like Certbot for
SSL) will automatically create the `.well-known` directory and manage the files inside it for you. However, if you are
building a custom website and need to verify your domain with a third-party service, you can safely create a folder named
`.well-known` right in your main web root directory (like `/var/www/html/` or `public_html/`).
