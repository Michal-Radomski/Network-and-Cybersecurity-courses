Here is how to configure Apache and Nginx to use the Let's Encrypt certificates generated by Certbot.

The easiest way to do this is to let Certbot handle it automatically, but you can also configure the files manually.

---

## Method 1: Automatic Configuration (Recommended)

If you haven't obtained the certificates yet, Certbot can automatically update your server configuration files for you.

### For Nginx:

Run the following command on your server terminal:

```bash
sudo certbot --nginx

```

Certbot will look at your existing Nginx configuration blocks, ask which domains you want to secure, fetch the certificates,
edit your configuration files, and reload Nginx automatically.

### For Apache:

Run the following command:

```bash
sudo certbot --apache

```

Just like with Nginx, Certbot will locate your virtual hosts, insert the necessary certificate paths, and restart Apache.

---

## Method 2: Manual Configuration

If you already have the certificates (found in `/etc/letsencrypt/live/yourdomain/`) and prefer to edit your server
configuration files manually, follow these steps.

### 1. Configuring Nginx

Open your Nginx configuration file for your website (usually located in `/etc/nginx/sites-available/` or
`/etc/nginx/nginx.conf`).

Update your `server` block to listen on port 443 (HTTPS) and point to the Certbot files:

```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com www.yourdomain.com;

    # Point to the Certbot files
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    # Recommended basic SSL settings
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;

    root /var/www/html;
    index index.html;
}

```

**Test and reload Nginx:**

```bash
sudo nginx -t
sudo systemctl reload nginx

```

---

### 2. Configuring Apache

Open your Apache virtual host configuration file (usually located in `/etc/apache2/sites-available/` on Debian/Ubuntu or
`/etc/httpd/conf.d/` on RHEL/CentOS).

Ensure the virtual host block is set to port 443 and includes the paths to your certificate files:

```apache
<VirtualHost *:443>
    ServerName yourdomain.com
    ServerAlias www.yourdomain.com
    DocumentRoot /var/www/html

    SSLEngine on

    # Point to the Certbot files
    SSLCertificateFile /etc/letsencrypt/live/yourdomain.com/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/yourdomain.com/privkey.pem

    # Optional: If using an older version of Apache, you might need:
    # SSLCertificateChainFile /etc/letsencrypt/live/yourdomain.com/chain.pem
</VirtualHost>

```

> **Note:** Ensure Apache's SSL module is enabled before restarting. On Ubuntu/Debian, run `sudo a2enmod ssl`.

**Test and reload Apache:**

```bash
sudo apachectl configtest
sudo systemctl restart apache2   # (or httpd on RHEL/CentOS)

```

---

## Automated Renewal Check

Regardless of whether you configure manually or automatically, Certbot sets up a system timer to handle renewals. You can
verify that the automated renewal process works smoothly by running a dry run:

```bash
sudo certbot renew --dry-run

```

If this command completes without errors, your web server will automatically pick up the renewed certificates every 90 days
without further manual editing.
