The `nslookup` (Name Server Lookup) command is a network administration tool used to query the Domain Name System (DNS) to
obtain mapping between domain names and IP addresses.

When you run a standard `nslookup` command, the output is generally divided into two main sections: **the server that
answered the query** and **the answer itself**.

Here is a breakdown of what the output typically looks like and what each part means:

---

### Example Output

If you type `nslookup google.com` into your terminal, you will see an output that looks similar to this:

```text
Server:		192.168.1.1
Address:	192.168.1.1#53

Non-authoritative answer:
Name:	google.com
Address: 142.250.190.46
Address: 2607:f8b0:4005:804::200e

```

---

### Deconstructing the Output

#### 1. The Resolver / DNS Server Info (The first two lines)

This section shows you _which_ DNS server your computer asked to get this information.

- **Server:** The hostname or IP address of the DNS server that handled your request (in the example above, `192.168.1.1` is
  usually a local home router).
- **Address:** The IP address of that DNS server, followed by `#53`. Port **53** is the standard networking port used for DNS
  traffic.

#### 2. The Answer Header

- **Non-authoritative answer:** This is a very common label. It means the DNS server you asked didn't own the original master
  records for `google.com`. Instead, your local DNS server looked it up from somewhere else previously, saved it in its
  temporary **cache**, and handed it to you quickly.
- _Note:_ If it says **Authoritative answer**, it means you queried the exact, official nameserver that directly manages the
  domain.

#### 3. The Target Data (The final lines)

This is the actual information you were looking for.

- **Name:** The domain name you queried (`google.com`).
- **Address (IPv4):** The current IPv4 address associated with that domain (e.g., `142.250.190.46`). This is the "phone
  number" your browser actually uses to connect to the website.
- **Address (IPv6):** If the website supports the newer IPv6 protocol, you will see a long, hexadecimal address listed right
  below the IPv4 address.

---

### Other Types of Output

Depending on the flags you use, `nslookup` can return different types of data:

- **Reverse Lookup:** If you type an IP address instead of a domain name (e.g., `nslookup 8.8.8.8`), the output will reverse
  the "Name" and "Address" fields, showing you the domain name attached to that IP address.
- **Mail Servers (MX Records):** If you run `nslookup -type=mx google.com`, the output will display the names of the specific
  servers responsible for handling email traffic for that domain.
