When you run Certbot to obtain an SSL/TLS certificate from Let's Encrypt, it creates and manages a specific set of files and
directories on your server. By default, on Linux systems, all of these files are stored inside the `/etc/letsencrypt/`
directory.

Here is a breakdown of the critical files and directories Certbot creates.

---

## 1. The Live Certificates Directory (`/etc/letsencrypt/live/`)

This is the most important folder for your web server configuration. Inside this directory, Certbot creates a subfolder named
after your domain (e.g., `/etc/letsencrypt/live/example.com/`).

Instead of storing the actual files here, this folder contains **symlinks (shortcuts)** that always point to the most recent
version of your certificates. This means you never have to update your Nginx or Apache configuration files when a certificate
renews; your server just reads the updated files through these shortcuts.

Inside your domain's `live` folder, you will find four specific files:

- **`privkey.pem` (The Private Key):** This is the private key for your certificate. **This file must be kept completely
  secret.** Your web server (like Nginx or Apache) needs to read this to decrypt data, but it should never be shared or made
  public.
- **`cert.pem` (The Server Certificate):** This is your domain's specific public certificate. It is what your server sends to
  browsers to prove its identity.
- **`chain.pem` (The Intermediate Certificate):** This contains the intermediate certificates provided by Let's Encrypt. It
  helps web browsers link your specific certificate back to a trusted Root Certificate Authority.
- **`fullchain.pem` (The Bundled Certificate):** This is a single file that combines `cert.pem` and `chain.pem`. **This is
  the file most modern web servers (like Nginx) require** for the certificate directive, alongside the `privkey.pem`.

---

## 2. The Archive Directory (`/etc/letsencrypt/archive/`)

This is where the actual, physical certificate files are stored.

Every time Certbot successfully renews a certificate, it saves the new files here and appends a number to them (e.g.,
`cert1.pem`, `cert2.pem`, `cert3.pem`).

- Certbot automatically updates the shortcuts in the `/live/` directory to point to the highest-numbered files in this
  archive folder.
- This architecture ensures that you have a history of past certificates to fall back on if something goes wrong during a
  renewal.

---

## 3. The Renewal Configurations (`/etc/letsencrypt/renewal/`)

Inside this directory, Certbot creates a configuration file for each of your domains (e.g., `example.com.conf`).

This is a text file that keeps track of the specific settings used when you first generated the certificate. It includes
information like:

- The renewal mechanism used (e.g., webroot, standalone, standalone standalone, or DNS challenge).
- The locations of the web root folders or specific plugin configurations.
- The parameters required by Certbot's background renewal script to successfully renew the certificate without human
  interaction.

---

## 4. Account and Key Storage (`/etc/letsencrypt/accounts/`)

This folder stores your Let's Encrypt account information. When you first run Certbot, you register an account (usually tied
to an email address) with the Let's Encrypt ACME server.

This directory saves the account registration keys and server metadata. If you manage certificates across multiple servers or
use multiple accounts, they are managed via these files.

---

## Quick Reference Summary

| File Path                                        | Description                                              | What to point your Web Server to     |
| ------------------------------------------------ | -------------------------------------------------------- | ------------------------------------ |
| `/etc/letsencrypt/live/yourdomain/fullchain.pem` | Server certificate combined with the intermediate chain. | **Yes** (Standard for Nginx/Apache)  |
| `/etc/letsencrypt/live/yourdomain/privkey.pem`   | The secret private key. Keep highly secure.              | **Yes** (Cryptographic key)          |
| `/etc/letsencrypt/live/yourdomain/cert.pem`      | Only the specific server certificate.                    | Seldom used alone on modern setups.  |
| `/etc/letsencrypt/live/yourdomain/chain.pem`     | Only the intermediate authority chain.                   | Used in older Apache configurations. |
