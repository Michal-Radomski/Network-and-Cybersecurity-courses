## How to Generate RSA Keys Using OpenSSL

To generate a secure, encrypted RSA key pair, you can use the standard `openssl` command-line tool.

### Step 1: Generate the Encrypted Private Key

Run the following command to generate an RSA private key that is encrypted with a passphrase:

```bash
openssl genrsa -aes256 -out private_key.pem 2048

```

- **`genrsa`**: Tells OpenSSL to generate an RSA private key.
- **`-aes256`**: This flag ensures the private key is **encrypted** using the AES-256 algorithm. OpenSSL will immediately
  prompt you to enter and verify a secure password/passphrase.
- **`-out private_key.pem`**: Specifies the filename for the output key.
- **`2048`**: The key size in bits. 2048-bit is the standard minimum for secure RSA keys.

### Step 2: Extract the Public Key

Now, you extract the public key from the private key you just created. Run this command:

```bash
openssl rsa -in private_key.pem -pubout -out public_key.pem

```

- **`-in private_key.pem`**: Specifies the input private key (it will ask for the password you created in Step 1 to read it).
- **`-pubout`**: Tells OpenSSL to extract and output the public key.
- **`-out public_key.pem`**: Specifies the filename for the public key.

## Why Private Keys Are Encrypted

In public-key cryptography (like RSA), you have two keys: a **public key** (which anyone can see) and a **private key**
(which only _you_ should control).

Here is why encrypting that private key "at rest" (while stored on your computer or server) is crucial:

- **Defense Against Theft:** If a hacker breaches your server, malware infects your computer, or someone physically steals
  your laptop, they can easily copy your key files. If the private key is encrypted, it is completely useless to them without
  your master passphrase.
- **Access Control:** Even inside an organization, encrypting the key ensures that only authorized administrators who know
  the passphrase can actually use the key to deploy software, decrypt sensitive data, or sign code.
- **The "Secret" Rules the System:** If someone gets your public key, they can only encrypt messages for you. But if someone
  gets your unencrypted private key, they can impersonate you perfectly, decrypt all your private messages, and bypass your
  security entirely. Encrypting it adds a vital second layer of defense (something you _have_—the file, plus something you
  _know_—the password).
