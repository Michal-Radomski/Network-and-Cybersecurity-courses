Configuring these redirections is a web development best practice. It boils down to three main reasons: **Security**, **SEO
(Search Engine Optimization)**, and **User Experience**.

Here is why you should always set up these redirects.

---

## 1. Security (HTTP to HTTPS)

The most critical redirection is from HTTP to HTTPS.

- **Encryption:** HTTP sends data in plain text. If a user types passwords, credit card numbers, or personal info into an
  HTTP site, anyone snooping on the network can see it. HTTPS encrypts this data.
- **Trust:** Modern browsers (like Chrome, Safari, and Firefox) mark HTTP websites as "Not Secure" in the address bar.
  Forcing a redirect to HTTPS ensures your users always see the secure padlock icon, building immediate trust.

---

## 2. SEO and "Duplicate Content" (WWW vs. Non-WWW)

To a human, `http://www.example.com` and `https://example.com` look like the exact same website. To a search engine engine
like Google, they are treated as **two completely separate websites**.

If you don't configure a redirect, you run into two major issues:

- **Split Search Authority:** If half of your backlinks point to the `www` version and half point to the non-`www` version,
  your search engine ranking power is split in half.
- **Duplicate Content Penalties:** Search engines dislike indexing identical content on different URLs. Forcing one primary
  version (canonicalization) consolidates all your SEO ranking power into a single URL.

---

## 3. User Experience and Consistency

Users rarely type out the full URL. They usually just type `example.com` into their browser.

- By default, a browser might try to load the HTTP version first.
- By handling the redirection instantly on the server side, you ensure that no matter how a user types your address (with
  `www`, without `www`, or with `http`), they always land on the exact same, secure page.

---

## How to do it in Apache

Since you were just looking at Apache configurations, here is the standard way to handle both of these redirections at once
using the `mod_rewrite` module in your Apache virtual host file (or `.htaccess` file):

```apache
RewriteEngine On

# 1. Redirect HTTP to HTTPS
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# 2. Redirect WWW to Non-WWW (or vice versa)
RewriteCond %{HTTP_HOST} ^www\.(.*)$ [NC]
RewriteRule ^(.*)$ https://%1%{REQUEST_URI} [R=301,L]

```

> **Note:** The `301` redirect is a "Permanent Redirect." This tells search engines to permanently transfer all SEO credit
> from the old URL to the new one.
