Generating a self-signed certificate with OpenSSL is a quick way to get HTTPS working for local development, testing, or
internal staging servers.

Here is the most efficient way to generate one using a single command, along with a breakdown of how it works.

---

### The One-Line Command

Open your terminal and run the following command. This will generate both the **Private Key** and the **Certificate** at the
same time:

```bash
openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout myserver.key -out myserver.crt

```

---

### What is happening in this command?

- **`req`**: Tells OpenSSL to use the Certificate Signing Request (CSR) management tool.
- **`-x509`**: This is the magic flag. It tells OpenSSL to output a **self-signed certificate** instead of a certificate
  request.
- **`-nodes`**: Short for "no DES". It means "don't encrypt the private key with a passphrase." This is important for web
  servers like Apache or Nginx, because if the key has a password, the server will freeze and demand you type it in every
  time it reboots.
- **`-days 365`**: Sets how long the certificate will be valid (1 year, in this case). You can change this to `825` (about 2
  years) if you want it to last longer.
- **`-newkey rsa:2048`**: Generates a new certificate request _and_ a new private key simultaneously using an RSA 2048-bit
  key size (the current security standard).
- **`-keyout myserver.key`**: The filename where your private key will be saved. **Keep this file secure and private.**
- **`-out myserver.crt`**: The filename where your public certificate will be saved.

---

### What happens next?

After you press Enter, OpenSSL will ask you a series of questions to populate the certificate's details.

The most important field is the **Common Name (CN)**:

```text
Country Name (2 letter code) [AU]:US
State or Province Name (full name) [Some-State]:California
Locality Name (eg, city) []:San Francisco
Organization Name (eg, company) [Internet Widgits Pty Ltd]:My Company
Organizational Unit Name (eg, section) []:IT
Common Name (e.g. server FQDN or YOUR name) []:localhost  <-- CRITICAL
Email Address []:admin@example.com

```

> 💡 **Crucial Step:** For the **Common Name**, type the exact domain name or IP address you will use to access the site
> (e.g., `localhost`, `127.0.0.1`, or `dev.local`). If this doesn't match your browser URL, the browser will throw an even
> harsher error.

---

### The "Not Secure" Warning

Because this certificate is self-signed (meaning it wasn't issued by a trusted authority like Let's Encrypt or DigiCert),
your browser **will** show a "Your connection is not private" warning when you visit your site.

This is completely normal for local development. You can safely click **Advanced** -> **Proceed to localhost (unsafe)** to
view your page over HTTPS.
