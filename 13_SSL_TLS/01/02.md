When you type **[www.google.com](https://www.google.com)** into your browser and hit Enter, a complex sequence of events
happens in a matter of milliseconds to bring the page to your screen.

Here is the step-by-step breakdown of that journey:

---

## Step 1: The DNS Lookup (Finding the Address)

Computers don't understand names like "google.com"; they communicate using numbers called **IP addresses** (like
`142.250.190.46`). Your browser needs to find the IP address associated with the domain name you typed. This process is
called **DNS resolution**.

- **Checking the Cache:** Your browser first checks its own memory (cache) to see if you've visited Google recently. If it's
  not there, it checks your computer's Operating System (OS) cache.
- **Asking the Router and ISP:** If still not found, your request travels to your router, and then to your Internet Service
  Provider's (ISP) **DNS Recursive Resolver**.
- **The DNS Tree:** If the ISP doesn't know it, the resolver queries the internet's DNS hierarchy (Root servers, Top-Level
  Domain servers like `.com`, and finally Authoritative Name Servers) until it finds the exact IP address for Google.

---

## Step 2: The TCP Connection (Establishing a Connection)

Now that your browser has Google's IP address, it needs to establish a connection to transport data. It does this using a
protocol called **TCP (Transmission Control Protocol)** via a process known as the **TCP Three-Way Handshake**.

1. **SYN (Synchronize):** Your browser sends a packet to Google's server asking to connect.
2. **SYN-ACK (Synchronize-Acknowledge):** Google's server responds saying it is ready.
3. **ACK (Acknowledge):** Your browser replies back confirming the connection is established.

---

## Step 3: The TLS Handshake (Securing the Connection)

Because Google uses **HTTPS**, the connection must be encrypted to keep your data private.

- Your browser and Google's server exchange cryptographic keys, verify Google's security certificate, and establish a secure,
  encrypted tunnel for all future data to pass through.

---

## Step 4: Sending the HTTP Request

With a secure connection established, your browser sends an **HTTP/HTTPS request** (specifically a `GET` request) to Google's
server. This request basically says: _"Hey, please give me the web page content for the homepage."_

Included in this request is extra data like your browser type (User-Agent), cookies, and language preferences.

---

## Step 5: The Server Processes and Responds

Google’s web server receives the request, processes it, and decides how to handle it.

- The server pulls the necessary files and sends back an **HTTP response**.
- This response includes a status code (like `200 OK`, meaning everything went perfectly) and the actual contents of the
  page, usually starting with an **HTML** file.

---

## Step 6: Rendering the Page (What You See)

Your browser receives the raw code from the server and begins turning it into a visual webpage. This happens in phases:

1. **Parsing HTML:** The browser reads the HTML code to build the structure of the page (called the **DOM tree**).
2. **Fetching Resources:** As it reads the HTML, it discovers links to other files like **CSS** (for styling and colors),
   **JavaScript** (for interactive features), and images. It sends separate requests to download those files.
3. **Painting:** The browser calculates where everything goes on your screen and "paints" the pixels, finally displaying the
   Google search bar and logo you are familiar with.
