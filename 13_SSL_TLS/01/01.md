To understand the difference between **HTTP** and **HTTPS**, think of HTTP as sending a postcard through the mail (anyone
handling it can read it), while HTTPS is like putting that letter into a tamper-proof, locked steel box before sending it.

Here is a breakdown of how they compare.

---

## The Core Difference

- **HTTP (Hypertext Transfer Protocol):** The foundational protocol used to transfer data across the web. It transmits data
  in **plain text**, meaning anyone snooping on your network (like on public Wi-Fi) can intercept and read it.
- **HTTPS (Hypertext Transfer Protocol Secure):** The secure version of HTTP. It uses an encryption protocol to scramble the
  data so that even if someone intercepts it, they cannot read it.

---

## Head-to-Head Comparison

| Feature               | HTTP                                        | HTTPS                                                             |
| --------------------- | ------------------------------------------- | ----------------------------------------------------------------- |
| **Security**          | Non-secure. Data is sent in plain text.     | Highly secure. Data is encrypted.                                 |
| **Port**              | Uses **Port 80** by default.                | Uses **Port 443** by default.                                     |
| **Encryption**        | None.                                       | Uses **TLS/SSL** (Transport Layer Security).                      |
| **Browser Indicator** | May show a "Not Secure" warning.            | Shows a padlock icon in the address bar.                          |
| **SEO & Speed**       | No SEO boost; can actually be slower today. | Preferred by Google (SEO boost) and faster due to HTTP/2 support. |

---

## How HTTPS Works (The TLS Handshake)

HTTPS doesn't replace HTTP; rather, it wraps HTTP inside a protective layer using **TLS (Transport Layer Security)**.

When you visit an HTTPS website, a quick "handshake" happens behind the scenes:

1. **The Request:** Your browser asks the website to identify itself.
2. **The Certificate:** The website sends over its **SSL/TLS Certificate** (which contains its public key).
3. **The Verification:** Your browser checks if the certificate is valid and trusted.
4. **The Encryption:** Once verified, your browser and the server agree on unique cryptographic keys to encrypt all data
   moving back and forth.

---

## Why HTTPS is Essential Today

While HTTP was fine for the early days of the web when we only read static text pages, HTTPS is now the mandatory standard
for several reasons:

- **Privacy:** It protects sensitive information like passwords, credit card numbers, and personal data from
  "man-in-the-middle" attacks.
- **Data Integrity:** It prevents hackers from altering the data in transit (for example, injecting malicious ads into a
  legitimate webpage).
- **Authentication:** It proves that you are actually connected to the real website (e.g., the actual bank website) and not a
  fake look-alike.
