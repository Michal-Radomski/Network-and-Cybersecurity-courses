## What is HashiCorp Vault?

**HashiCorp Vault** is an open-source, enterprise-grade tool designed specifically for security automation and secrets
management. In modern development, hardcoding passwords, API keys, or database credentials directly into source code (or
storing them in plain text `.env` files) is a massive security risk.

Vault solves this problem by acting as a highly secure, centralized "safe" for sensitive data.

### Key Capabilities

- **Centralized Secrets Management:** Securely stores static tokens, passwords, and API keys. Every piece of data is
  encrypted at rest and in transit.
- **Dynamic Secrets:** Instead of sharing a permanent database password, Vault can generate a unique, temporary credential
  for an application on the fly and automatically revoke it when its time-to-live (TTL) expires.
- **Data Encryption (Encryption as a Service):** Vault can encrypt and decrypt data for applications without storing the data
  itself. This offloads the mathematical heavy lifting of cryptography from your app code.
- **Leasing and Renewal:** All secrets have an expiration time. Applications must actively renew their lease to keep using
  them.
- **Detailed Audit Logs:** Vault maintains a strict, tamper-proof log of exactly who accessed what secret, when, and from
  where.

## How to Install Vault on Ubuntu

The cleanest way to install Vault on Ubuntu is using HashiCorp’s official package repository. This ensures you get the latest
stable updates.

### Step 1: Install Prerequisites

Open your terminal and ensure your local package index is up-to-date and that you have `curl` and `gpg` installed:

```bash
sudo apt update && sudo apt install -y gpg coreutils curl

```

### Step 2: Add the HashiCorp GPG Key

Download and add the official GPG key to verify the integrity of the downloaded packages:

```bash
curl -fsSL https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg

```

### Step 3: Add the Official Repository

Add the official HashiCorp repository to your system sources:

```bash
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list

```

### Step 4: Install Vault

Update your package list one more time to fetch metadata from the new repository, then install Vault:

```bash
sudo apt update
sudo apt install vault

```

### Step 5: Verify the Installation

Check that the binary was installed successfully by checking its version:

```bash
vault --version

```

## How to Use Vault (Quick Start Guide)

For learning and local experimentation, Vault includes a built-in **Development Mode** (`-dev`).

> ⚠️ **Warning:** Never use `-dev` mode in production. It runs unencrypted in-memory, automatically initializes, and
> automatically unseals itself, which defeats production security protocols but is perfect for practicing.

### 1. Start the Dev Server

Launch the server by running:

```bash
vault server -dev

```

Look closely at the terminal output. You will see a block of text containing a **Unseal Key** and a **Root Token**. Keep this
terminal open, and copy the **Root Token** (it will look something like `hvs.xxxxxxxxxxxx...`).

### 2. Configure Your Environment

Open a **new, separate terminal window** and map the Vault address and your root token so the CLI tool knows how to
communicate with your running server:

```bash
export VAULT_ADDR='http://127.0.0.1:8200'
export VAULT_TOKEN="YOUR_COPIED_ROOT_TOKEN_HERE"

```

Verify your connection by checking the server status:

```bash
vault status

```

### 3. Store and Retrieve Secrets (KV Engine)

Vault uses "Secrets Engines" to handle data. By default, dev mode enables a Key-Value (KV) version 2 engine at the path
`secret/`.

- **Write a Secret:** To store a secret containing a username and password for an app called `myapp`:

```bash
vault kv put secret/myapp username="admin" password="SuperSecretPassword123"

```

- **Read the Secret:** To retrieve the secret you just stored:

```bash
vault kv get secret/myapp

```

- **Read a Single Field:** If your script or application only needs the password field:

```bash
vault kv get -field=password secret/myapp

```

- **Delete the Secret:** To remove the secret data:

```bash
vault kv delete secret/myapp

```

Once you are done experimenting, you can go back to your first terminal window and press `Ctrl + C` to shut down the
temporary development server.
