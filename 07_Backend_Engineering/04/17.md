Integrating **Kerberos** and **HashiCorp Vault** into a React.js (Frontend) and Node.js (Backend) application requires a
solid understanding of architectural boundaries.

The most important fundamental rule to remember: **Never run Kerberos or Vault directly in the React frontend.** \* The
browser cannot access a machine's local Kerberos ticket cache.

- Storing a Vault token or secrets in the frontend exposes them to Cross-Site Scripting (XSS) attacks.

Instead, your **Node.js backend acts as the secure orchestrator**. It handles Kerberos Single Sign-On (SSO) with the user's
browser, communicates securely with Vault to fetch credentials, and passes data safely to React.

---

## Architecture Flow

1. **Authentication (Kerberos SPNEGO):** React requests a resource $\rightarrow$ Node.js challenges with `401 Unauthorized`
   and `WWW-Authenticate: Negotiate` $\rightarrow$ The browser fetches a Kerberos ticket from the OS and sends it back in the
   `Authorization` header.
2. **Secrets/Config Management (Vault):** The Node.js server authenticates itself to Vault (using AppRole or AWS/Kubernetes
   auth) $\rightarrow$ Node.js fetches runtime configurations (like database strings) securely from Vault.
3. **Session Management:** Once Node.js validates the Kerberos ticket, it establishes a secure session (via HttpOnly cookies)
   with React.

---

## Step 1: Setting up Kerberos SSO on the Node.js Backend

To implement Kerberos SSO (Spnego) in Express/Node.js, you'll need the native `kerberos` library. Ensure your hosting server
has Kerberos client libraries installed (e.g., `krb5-libs` or `krb5-workstation` on Linux) and a valid system `krb5.conf`
file.

### 1. Install Dependencies

```bash
npm install express kerberos

```

### 2. Implement the Kerberos Middleware

Your Node.js app needs a **Service Principal Name (SPN)** (e.g., `HTTP/my-app.domain.com`) and its corresponding **keytab**
file registered on the server environment.

```javascript
const express = require("express");
const kerberos = require("kerberos");
const app = express();

app.get("/api/auth/sso", async (req, res) => {
  const authHeader = req.headers.authorization;

  // 1. Send the Negotiate challenge if no token is present
  if (!authHeader || !authHeader.startsWith("Negotiate ")) {
    res.setHeader("WWW-Authenticate", "Negotiate");
    return res.status(401).send("Authentication Required");
  }

  // 2. Extract the Kerberos ticket from the header
  const ticket = authHeader.substring(10);

  try {
    // Initialize Kerberos server context
    // "HTTP" matches the service class of your SPN: HTTP/my-app.domain.com
    const server = await kerberos.initializeServer("HTTP");

    // Validate the ticket
    await server.step(ticket);

    // If successful, you can retrieve the authenticated username/principal
    const username = server.username;

    // 3. Establish a secure, HTTP-only session cookie for React
    // (Do not send the raw Kerberos data to the frontend)
    req.session.user = { username };

    res.json({ authenticated: true, user: username });
  } catch (error) {
    console.error("Kerberos validation failed:", error);
    res.status(401).json({ error: "Authentication failed" });
  }
});
```

## Step 2: Integrating HashiCorp Vault on the Node.js Backend

Your backend should use **AppRole** authentication (standard for machine-to-machine integration) to log into Vault and read
secrets.

### 1. Install Dependencies

```bash
npm install node-vault

```

### 2. Connect and Retrieve Secrets

Do not hardcode your Vault token. Instead, pass the environment-isolated `VAULT_ROLE_ID` and `VAULT_SECRET_ID` to your
application container during deployment.

```javascript
const vault = require("node-vault")({
  endpoint: process.env.VAULT_ADDR || "http://127.0.0.1:8200",
});

async function getBackendSecrets() {
  try {
    // 1. Authenticate with Vault via AppRole
    const authResult = await vault.approleLogin({
      role_id: process.env.VAULT_ROLE_ID,
      secret_id: process.env.VAULT_SECRET_ID,
    });

    // 2. Set the client token for subsequent requests
    vault.token = authResult.auth.client_token;

    // 3. Fetch secrets (e.g., database credentials)
    const secret = await vault.read("secret/data/myapp/config");
    return secret.data.data; // Returns your key-value pairs
  } catch (err) {
    console.error("Failed to retrieve secrets from Vault:", err.message);
    throw err;
  }
}
```

## Step 3: Configuring the React.js Frontend

Because Kerberos negotiation is handled natively by the browser, React doesn't need external Kerberos libraries. It just
needs to ensure it passes credentials correctly via `fetch` or `axios`.

### Important Browser Limitation

> ⚠️ **Note:** Kerberos SSO will **not** trigger if you access your app via an IP address or `localhost`. The browser
> requires a valid Domain Name System (DNS) address (e.g., `http://my-app.localdomain:3000`) that maps to an Intranet/Trusted
> zone in your OS settings.

### Implementing the API Call in React

When making requests to your backend, you must explicitly tell the browser to include credentials/cookies (and natively
attach the Kerberos header during the initial 401 challenge).

Using the standard `fetch` API:

```javascript
import React, { useEffect, useState } from "react";

function App() {
  const [user, setUser] = useState(null);

  const triggerSSO = () => {
    // 'credentials: "include"' forces the browser to handle cookies
    // and automatically attach the Kerberos SPNEGO tokens if challenged.
    fetch("http://my-app.localdomain:5000/api/auth/sso", { credentials: "include" })
      .then((res) => {
        if (res.status === 401) throw new Error("Kerberos Handshake failed");
        return res.json();
      })
      .then((data) => setUser(data.user))
      .catch((err) => console.error("SSO Error:", err));
  };

  return (
    <div style={{ padding: "20px" }}>
      <h1>Enterprise App Dashboard</h1>
      {user ? (
        <p>
          Welcome, <strong>{user}</strong>
        </p>
      ) : (
        <button onClick={triggerSSO}>Login with SSO</button>
      )}
    </div>
  );
}

export default App;
```

## Summary Summary Checklist

- [ ] **DNS Mapping:** Ensure your development/production URLs are fully-qualified domain names (FQDN) rather than IPs.
- [ ] **SPN & Keytab:** Ensure your Node.js infrastructure environment has the matching service principal registered in
      Active Directory/MIT Kerberos.
- [ ] **Network Settings:** Verify that the user's browser has your domain added to its "Local Intranet" or
      `network.negotiate-auth.trusted-uris` (Firefox) configuration.
- [ ] **Vault Isolation:** Ensure the React UI never sees the Vault address or token. Node.js should fetch secrets on
      startup/request and pass only sanitized data to the client.
