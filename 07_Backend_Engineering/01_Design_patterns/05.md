In Node.js, the biggest difference between synchronous and asynchronous code is **blocking** vs. **non-blocking** behavior.

- **Synchronous code** executes line-by-line. Each line must finish completely before the next one starts. If a line takes a
  long time (like reading a massive file), the whole application freezes and waits.
- **Asynchronous code** schedules a task to run in the background and moves on to the next line immediately. When the
  background task finishes, a callback, promise, or `async/await` block handles the result.

Here are examples of both approaches using Node.js's built-in File System (`fs`) module.

---

## 1. Synchronous Example (Blocking)

In this example, Node.js waits for `readFileSync` to completely finish reading the file before it logs
`"Moving on to other tasks..."`.

```javascript
const fs = require("fs");

console.log("1. Starting to read file...");

try {
  // Node.js stops here and waits until the entire file is read
  const data = fs.readFileSync("example.txt", "utf8");
  console.log("2. File Content:", data);
} catch (err) {
  console.error("Error reading file:", err.message);
}

// This will NOT run until the file is fully read
console.log("3. Moving on to other tasks...");
```

### Output Order:

1. `1. Starting to read file...`
2. `2. File Content: [Actual content of the file]`
3. `3. Moving on to other tasks...`

---

## 2. Asynchronous Example (Non-Blocking)

Modern Node.js heavily utilizes Promises and `async/await` for asynchronous code because it looks clean and is easy to read.

In this example, Node.js triggers the file read but doesn't wait around. It immediately moves on to log
`"3. Moving on to other tasks..."`. When the file is finally done being read, the code inside the `try` block resumes.

```javascript
const fs = require("fs").promises; // Using the promise-based version of fs

async function readFileAsync() {
  console.log("1. Starting to read file...");

  try {
    // Node.js hands this task to the background and frees up the thread
    const data = await fs.readFile("example.txt", "utf8");
    console.log("2. File Content:", data);
  } catch (err) {
    console.error("Error reading file:", err.message);
  }
}

readFileAsync();

// This runs IMMEDIATELY after the file read is kicked off,
// without waiting for the file to actually finish reading.
console.log("3. Moving on to other tasks...");
```

### Output Order:

1. `1. Starting to read file...`
2. `3. Moving on to other tasks...`
3. `2. File Content: [Actual content of the file]`

---

## Quick Comparison

| Feature           | Synchronous                                           | Asynchronous                                                       |
| ----------------- | ----------------------------------------------------- | ------------------------------------------------------------------ |
| **Execution**     | Line-by-line (Sequential)                             | Concurrent (Non-sequential)                                        |
| **Performance**   | Slower for I/O tasks; blocks the server               | Faster for I/O tasks; keeps the server responsive                  |
| **Best Used For** | Initial setup (e.g., loading config files at startup) | Database queries, API requests, reading files during user requests |
