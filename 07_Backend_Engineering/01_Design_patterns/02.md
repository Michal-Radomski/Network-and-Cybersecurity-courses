Implementing chunked file uploads is a brilliant way to handle large images or files. It prevents server timeouts, bypasses
payload size limits, and allows you to add pause/resume features later.

Here is a complete, streamlined guide to building a chunked upload system using **React** (with the standard `Fetch API`) and
**Express.js** (using **Multer** and Node's native **File System** module).

---

## The Workflow Blueprint

Instead of sending the whole file at once, the React client slices the file into smaller binary blobs using `File.slice()`.
Each chunk is sent sequentially (or concurrently) with metadata telling the server which part it is. Once the last chunk
arrives, the server stitches them back together.

---

## 1. The Express.js Server

First, let's set up the backend to receive individual chunks and append them to a temporary file. Once all chunks are
received, you can save the file metadata to your database.

```javascript
const express = require("express");
const multer = require("multer");
const fs = require("fs");
const path = require("path");
const cors = require("cors");

const app = express();
app.use(cors());
app.use(express.json());

// Configure multer to store chunks in a temporary directory
const upload = multer({ dest: "uploads/tmp/" });

app.post("/upload-chunk", upload.single("chunk"), (req, res) => {
  const { fileName, chunkIndex, totalChunks } = req.body;
  const chunkPath = req.file.path;
  const targetDir = path.join(__dirname, "uploads");
  const finalFilePath = path.join(targetDir, fileName);

  // Ensure the final uploads directory exists
  if (!fs.existsSync(targetDir)) {
    fs.mkdirSync(targetDir, { recursive: true });
  }

  // Append the chunk to the final file
  const chunkBuffer = fs.readFileSync(chunkPath);
  fs.appendFileSync(finalFilePath, chunkBuffer);

  // Delete the temporary chunk file to save space
  fs.unlinkSync(chunkPath);

  // Check if this was the last chunk
  if (parseInt(chunkIndex) === parseInt(totalChunks) - 1) {
    // ---------------------------------------------------------
    // DATABASE OPERATIONS GO HERE
    // Example: const imageUri = `/uploads/${fileName}`;
    // await MyImageModel.create({ name: fileName, url: imageUri });
    // ---------------------------------------------------------

    return res.status(200).json({
      success: true,
      message: "File fully uploaded and saved to DB!",
      url: `/uploads/${fileName}`,
    });
  }

  // If not the last chunk, acknowledge success for this piece
  res.status(200).json({ success: true, message: `Chunk ${chunkIndex} uploaded.` });
});

app.listen(5000, () => console.log("Server running on port 5000"));
```

## 2. The React Client

On the frontend, we use a loop to slice the file into a predefined chunk size (e.g., 1MB) and send them one by one using
`FormData`.

```jsx
import React, { useState } from "react";

const CHUNK_SIZE = 1024 * 1024; // 1MB per chunk

export default function ChunkedUpload() {
  const [file, setFile] = useState(null);
  const [progress, setProgress] = useState(0);

  const handleFileChange = (e) => {
    setFile(e.target.files[0]);
    setProgress(0);
  };

  const uploadFile = async () => {
    if (!file) return alert("Please select a file first.");

    const totalChunks = Math.ceil(file.size / CHUNK_SIZE);
    const uniqueFileName = `${Date.now()}-${file.name}`;

    for (let chunkIndex = 0; chunkIndex < totalChunks; chunkIndex++) {
      // Calculate the slice boundaries
      const start = chunkIndex * CHUNK_SIZE;
      const end = Math.min(start + CHUNK_SIZE, file.size);
      const chunk = file.slice(start, end);

      // Prepare payload
      const formData = new FormData();
      formData.append("chunk", chunk);
      formData.append("fileName", uniqueFileName);
      formData.append("chunkIndex", chunkIndex);
      formData.append("totalChunks", totalChunks);

      try {
        const response = await fetch("http://localhost:5000/upload-chunk", {
          method: "POST",
          body: formData,
        });

        if (!response.ok) throw new Error("Chunk upload failed.");

        const data = await response.json();

        // Update progress bar
        setProgress(Math.round(((chunkIndex + 1) / totalChunks) * 100));

        if (data.url) {
          alert("Upload complete! File URL tracked in DB.");
        }
      } catch (error) {
        console.error("Error uploading chunk:", error);
        alert("Upload failed. Check console.");
        break;
      }
    }
  };

  return (
    <div style={{ padding: "20px" }}>
      <h2>Chunked Image Upload</h2>
      <input type="file" accept="image/*" onChange={handleFileChange} />
      <button onClick={uploadFile} style={{ marginLeft: "10px" }}>
        Upload
      </button>

      {progress > 0 && (
        <div style={{ marginTop: "20px" }}>
          <progress value={progress} max="100" /> {progress}%
        </div>
      )}
    </div>
  );
}
```

## 💡 Key Considerations for Production

While the code above provides a working core foundation, production environments usually require a few extra safeguards:

- **Concurrency & Identification:** If two users upload files with the same name simultaneously, `fs.appendFileSync` will mix
  up their data. The code above uses `Date.now()` to prevent this locally, but using a unique session ID or a **UUID**
  generated before the upload starts is highly recommended.
- **Database Strategy:** Best practice dictates that you **do not store raw binary image data (BLOBs) directly in your
  SQL/NoSQL database**, as it degrades database performance. Instead, save the file to disk (or a cloud bucket like AWS S3)
  and store the resulting _string URL path_ in your database schema.
- **Cleaning Up Failures:** If a user closes their browser halfway through an upload, you will be left with an incomplete
  file. Consider running a daily cron job on your Express server to wipe files in the `uploads/tmp` directory older than 24
  hours.
