An HTTP response is the message a server sends back to your computer. [1] It delivers the answer to your request. For
example, it sends the webpage you asked to see, or a message saying if your login worked. [2, 3, 4, 5]

## 🧱 The 3 Main Parts of a Response

Every HTTP response has three main pieces:

- Status Line: The first line that says if the request worked or failed.
- Headers: Extra information about the server and the data being sent.
- Body: The actual content, like the text of a webpage, an image, or a file. [6, 7, 8, 9, 10]

---

## 🚦 Common Status Codes [11]

The status line uses a three-digit number to tell you what happened: [12, 13]

- 200 OK: Everything worked perfectly, and the server found your data.
- 301 Moved Permanently: The webpage moved to a new web address.
- 404 Not Found: The server could not find the page you asked for.
- 500 Internal Server Error: The server crashed or ran into a glitch. [14, 15, 16, 17, 18]

---

## 📂 Example of a Raw HTTP Response

Here is what a response looks like behind the scenes when a server sends a webpage:

HTTP/1.1 200 OK Content-Type: text/html Content-Length: 45 Server: Apache

<html><body><h1>Hello World!</h1></body></html>

What this means:

1.  HTTP/1.1 200 OK is the Status Line. It says the request worked perfectly.
2.  Content-Type: text/html tells your browser that the data is a regular web page.
3.  Content-Length: 45 tells the browser exactly how big the file is in bytes.
4.  Server: Apache says the server is running Apache software.
5.  <html>...</html> is the Body. This is the code that draws the page on your screen. [19, 20, 21, 22, 23]

---

## 🔗 Connection to TLS

Just like the request, if you use HTTPS, this entire response is encrypted by TLS. The server scrambles the status, headers,
and body before sending them over the internet so nobody can spy on the data you receive. [24, 25]
