HTTP pipelining is an older technology that allows a browser to send multiple HTTP requests over a single network connection
all at once, without waiting for the server to reply to each one first. [1, 2] Normally, a browser has to work in a strict
loop: send request 1, wait for the response, send request 2, wait for the response. Pipelining breaks that loop to try and
save time. [2, 3, 4]

---

## ⏳ Old Way vs. Pipelining

Think of a webpage that needs an HTML file, a CSS style file, and a picture. [4]

- Without Pipelining (Standard):
- Browser: "Give me the HTML." → Server sends HTML.
  - Browser: "Give me the CSS." → Server sends CSS.
  - Browser: "Give me the picture." → Server sends picture.
  - Result: Lots of waiting for data to travel back and forth.
- With Pipelining:
- Browser sends all three requests in one quick burst: "Give me the HTML, the CSS, and the picture!"
  - Server processes them and sends the answers back.
  - Result: Saves time because you do not wait for a response before asking for the next file. [1, 2, 3, 4, 5]

---

## 🚨 The Big Problem: Head-of-Line Blocking [6]

While pipelining sounds great, it has a major design flaw: The server must send the responses back in the exact same order
the requests were received. [5, 7, 8] If the server gets stuck trying to generate the first file (like a slow database
search), it cannot send the other files. The CSS and the picture get stuck waiting behind the slow HTML file. This traffic
jam is called Head-of-Line (HoL) Blocking. [7, 8] Because of this flaw, and because many older web systems broke when trying
to use it, modern web browsers do not use HTTP pipelining anymore. [1, 2]

---

## 🚀 What We Use Today: Multiplexing

To fix this traffic jam, the web moved to newer protocols like HTTP/2 and HTTP/3. [1] They use a feature called Multiplexing.
Multiplexing allows the browser to send all requests at once, but the server can send the answers back in any order it wants.
If a picture is ready before a slow text file, the server sends the picture first. No more traffic jams! [1, 3, 8, 9] You can
read the deep history of this feature on the
[MDN Web Docs Connection Management Guide](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Connection_management_in_HTTP_1.x)
or see how modern standards evolved on Wikipedia's HTTP/2 Page.
