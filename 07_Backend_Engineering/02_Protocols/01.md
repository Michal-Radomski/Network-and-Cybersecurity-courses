The **OSI (Open Systems Interconnection) Model** is a conceptual framework used to understand how data moves across a
network. It breaks down the complex process of communication into seven distinct layers.

Think of it like sending a physical letter: you write the message, put it in an envelope, address it, and hand it to the
postal service. Each step is a "layer" that ensures the message gets to the right person.

---

### Layer 7: Application Layer

This is the layer closest to the user. It provides network services directly to software applications (like your web browser
or email client).

- **Function:** Resource sharing, remote file access, and error handling for applications.
- **Protocols:** HTTP, HTTPS, FTP, SMTP, DNS, DHCP, Telnet.

### Layer 6: Presentation Layer

This layer acts as the "translator." It ensures that data sent from the application layer of one system can be read by the
application layer of another.

- **Function:** Data encryption/decryption, compression, and translation (e.g., converting EBCDIC to ASCII).
- **Protocols:** SSL/TLS, JPEG, GIF, MPEG, ASCII.

### Layer 5: Session Layer

This layer manages the "conversation" between computers. It starts, maintains, and ends the connection.

- **Function:** Session setup, authentication, and "checkpointing" (so a file transfer can resume if interrupted).
- **Protocols:** NetBIOS, SAP, PPTP, RPC.

### Layer 4: Transport Layer

This layer is responsible for the end-to-end delivery of data. It decides how much data to send and at what speed.

- **Function:** Segmentation, flow control, and error correction.
- **Protocols:** TCP (Transmission Control Protocol), UDP (User Datagram Protocol).

### Layer 3: Network Layer

The Network Layer is the "traffic controller." It determines the best physical path for the data to take (routing).

- **Function:** Logical addressing (IP addresses) and routing packets across different networks.
- **Protocols:** IPv4, IPv6, ICMP, IPsec, IGMP, RIP.

### Layer 2: Data Link Layer

This layer handles the physical movement of data across a single link. It packages data into "frames."

- **Function:** Physical addressing (MAC addresses) and error detection on the physical medium.
- **Protocols:** Ethernet, PPP, Switch, Bridge, Frame Relay.

### Layer 1: Physical Layer

The bottom layer deals with the actual hardware and the raw transmission of bits (0s and 1s) over a physical medium.

- **Function:** Electrical signals, cables, connectors, and light pulses.
- **Technologies:** Fiber optics, Coaxial cable, Wi-Fi (Radio waves), Hubs, Repeaters.

| Layer | Name             | Unit     | Primary Responsibility                 |
| ----- | ---------------- | -------- | -------------------------------------- |
| **7** | **Application**  | Data     | User Interface & Network Services      |
| **6** | **Presentation** | Data     | Data Representation & Encryption       |
| **5** | **Session**      | Data     | Interhost Communication                |
| **4** | **Transport**    | Segments | End-to-End Connections & Reliability   |
