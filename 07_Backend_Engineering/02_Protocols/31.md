Network Address Translation (NAT) is a method used to remap one IP address space into another while packets are in transit
across a traffic routing device. It’s essentially the receptionist of the internet world, directing incoming and outgoing
data to the right internal departments.

NAT is generally categorized into **three traditional types**, along with a few **specific variations** based on how ports
are handled.

---

## The 3 Traditional Types of NAT

### 1. Static NAT (One-to-One)

In Static NAT, a single private IP address is mapped to a single, dedicated public IP address. This mapping is permanent and
unchanging.

- **How it works:** Every time Device A (private) communicates with the outside world, it _always_ uses Public IP X.
- **Best for:** Hosting servers inside a private network (like a web server or mail server) that need to be consistently
  accessible from the outside internet.
- **Downside:** It doesn’t save public IP addresses, as it requires a 1:1 ratio.

### 2. Dynamic NAT (Many-to-Many)

Dynamic NAT maps an unregistered (private) IP address to a registered (public) IP address from a **pool of available public
IPs**.

- **How it works:** When an internal device wants to access the internet, the router assigns it an unused public IP from the
  pool. Once the device is done communicating, that public IP goes back into the pool for another device to use.
- **Best for:** Networks where the number of concurrent internet users is less than or equal to the number of public IPs
  owned.
- **Downside:** If the pool runs out of public IPs, any additional internal devices will be blocked from accessing the
  internet until an IP becomes free.

### 3. PAT / NAT Overload (Many-to-One)

**Port Address Translation (PAT)**, also known as NAT Overload, is the form of NAT used in almost all home networks and most
businesses. It maps multiple private IP addresses to a **single public IP address** by using unique port numbers to
distinguish between the different traffic streams.

- **How it works:** If three computers inside a house browse the web at the same time, they all share the same public IP. The
  router assigns a unique source port number (e.g., `204.56.7.8:1025`, `204.56.7.8:1026`) to each connection so it knows
  exactly which computer should receive the returning data.
- **Best for:** Conserving IPv4 addresses. A single public IP can theoretically support over 60,000 concurrent connections.

## Implementations of NAT (Cone vs. Symmetric)

When dealing with firewalls and more complex routing (especially for gaming, VoIP, and Peer-to-Peer networks), NAT is often
classified by how strictly it filters incoming traffic based on ports and destinations:

| NAT Type                     | Description                                                                                                                                                                                                                               | Strictness                                                  |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------- |
| **Full Cone NAT**            | Once an internal IP/port is mapped to a public IP/port, _any_ external host can send packets to it using that public mapping.                                                                                                             | Most Open                                                   |
| **Restricted Cone NAT**      | An external host can only send packets to the internal device if the internal device has previously sent a packet to that external host's _IP address_.                                                                                   | Medium                                                      |
| **Port-Restricted Cone NAT** | Same as Restricted Cone, but the external host must match both the _IP address and the port number_ that the internal device previously communicated with.                                                                                | Medium-Strict                                               |
| **Symmetric NAT**            | Every time the internal device sends a packet to a _new_ destination, a brand new public IP/port mapping is generated, even if it uses the same internal source. Only the specific external host receiving the packet can send data back. | Most Strict (Causes the most issues for online gaming/VoIP) |

## Other Notable NAT Variants

- **Destination NAT (DNAT):** Used to change the destination IP address in the packet header. This is commonly used for port
  forwarding, redirecting incoming traffic destined for a public IP down to a specific internal server.
- **Source NAT (SNAT):** Used to change the source IP address in the packet header. Traditional NAT and PAT are forms of
  SNAT, typically used when internal users are outgoing to the internet.
- **Carrier-Grade NAT (CGNAT):** An implementation where internet service providers (ISPs) use NAT within their own networks.
  Instead of giving your home router a public IP address, the ISP gives your router a private IP and handles the NAT at the
  massive provider level. (If you've ever had trouble setting up port forwarding for gaming, CGNAT is usually the culprit).
