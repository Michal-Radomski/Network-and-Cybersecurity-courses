Here’s a simple **TypeScript WebRTC** example that captures camera/mic, creates a peer connection, and wires up local/remote
video. `getUserMedia()` gets the media stream, and `RTCPeerConnection` handles the peer-to-peer connection and tracks.
[onsip](https://www.onsip.com/voip-resources/voip-fundamentals/getusermedia)

## TypeScript example

```ts
const localVideo = document.getElementById("localVideo") as HTMLVideoElement;
const remoteVideo = document.getElementById("remoteVideo") as HTMLVideoElement;

const pc = new RTCPeerConnection({
  iceServers: [{ urls: "stun:stun.l.google.com:19302" }],
});

async function startWebRTC() {
  const stream = await navigator.mediaDevices.getUserMedia({
    audio: true,
    video: true,
  });

  localVideo.srcObject = stream;

  stream.getTracks().forEach((track) => {
    pc.addTrack(track, stream);
  });

  pc.ontrack = (event: RTCTrackEvent) => {
    const [remoteStream] = event.streams;
    if (remoteStream) remoteVideo.srcObject = remoteStream;
  };

  pc.onicecandidate = (event) => {
    if (event.candidate) {
      // Send this candidate to the other peer via signaling
      console.log("ICE candidate:", event.candidate);
    }
  };

  const offer = await pc.createOffer();
  await pc.setLocalDescription(offer);

  // Send pc.localDescription to the other peer via signaling
  console.log("Offer:", pc.localDescription);
}

startWebRTC().catch(console.error);
```

## Minimal HTML

```html
<video id="localVideo" autoplay playsinline muted></video> <video id="remoteVideo" autoplay playsinline></video>
```

## What’s missing

This is only the browser-side WebRTC part. A real app also needs **signaling** to exchange the offer, answer, and ICE
candidates between peers; WebRTC itself does not define how that message exchange happens. In practice, signaling is usually
done with WebSocket or a similar channel. [webrtc.github](https://webrtc.github.io/samples/)
