## What is TCP?

**TCP (Transmission Control Protocol)** is one of the foundational protocols of the Internet protocol suite. Operating at the
transport layer, it sits on top of IP (Internet Protocol) and is responsible for ensuring that data is delivered from one
device to another **reliably, accurately, and in the correct order.**

If UDP is like sending an un-tracked postcard, TCP is like sending a high-value package via a premium courier service that
requires a signature, tracks the package every step of the way, and guarantees insurance if something goes wrong.

---

## How TCP Works: The Core Mechanisms

Unlike UDP's "fire and forget" approach, TCP is a **connection-oriented** protocol. It establishes a formal connection before
sending data and meticulously tracks the transmission.

### 1. The Three-Way Handshake

Before any data (like a webpage or an email) can be sent, TCP establishes a stable connection between the client and the
server using a process called the **Three-Way Handshake**:

1. **SYN (Synchronize):** The client sends a packet to the server asking to connect.
2. **SYN-ACK (Synchronize-Acknowledgment):** The server responds, agreeing to connect and acknowledging the request.
3. **ACK (Acknowledgment):** The client sends a final packet back, confirming the connection is established.

Once this handshake is complete, data transfer can begin.

### 2. Reliability and Acknowledgment

TCP guarantees that every single byte of data reaches its destination. It achieves this through **positive acknowledgment
with retransmission**:

- Every time the receiver gets a packet of data, it must send an **ACK** packet back to the sender to say, _"I got it!"_
- If the sender doesn't receive an ACK within a certain timeframe, it assumes the packet was lost in transit and
  **retransmits** the missing data automatically.

### 3. In-Order Delivery

The internet is chaotic; packets can take different physical routes and arrive out of order. TCP solves this by assigning a
**Sequence Number** to every packet. If Packet 3 arrives before Packet 2, the receiving computer's TCP layer will hold Packet
3 in a buffer, wait for Packet 2 to arrive, and put them back in the correct order before handing the data over to the
application.

### 4. Flow Control and Congestion Control

TCP is smart enough to manage _how fast_ data is sent:

- **Flow Control:** Prevents a fast sender from overwhelming a slow receiver. The receiver constantly tells the sender how
  much buffer space it has left (called the "Window Size").
- **Congestion Control:** Prevents the network itself from getting choked. If TCP notices that packets are being dropped or
  delayed, it automatically throttles back its transmission speed and gradually speeds up again as network congestion clears.

## The TCP Packet Structure (Header)

Because TCP does so much heavy lifting, its header is much larger and more complex than UDP's. A standard TCP header is at
least **20 bytes** long.

Key fields inside a TCP header include:

- **Source & Destination Ports:** Directs the data to the correct application.
- **Sequence Number:** Tracks the order of the data packets.
- **Acknowledgment Number:** Tells the sender which packet the receiver expects next.
- **Window Size:** Used for flow control (how much data the receiver can handle).
- **Flags (SYN, ACK, FIN, RST):** Control state flags used to manage the connection (e.g., initiating, acknowledging, or
  tearing down the connection).

## Why Use TCP? (The Advantages and Disadvantages)

### Advantages

- **Guaranteed Delivery:** Data will not be lost.
- **Data Integrity:** Built-in error checking ensures data isn't corrupted.
- **Order Preservation:** Files arrive exactly as they were sent.

### Disadvantages

- **Higher Latency:** The three-way handshake and constant ACK packets add time delays.
- **More Overhead:** The 20-byte header and constant bidirectional chatter consume more network bandwidth than UDP.

---

## Common Use Cases

TCP is the go-to choice for any application where **data accuracy is critical and data loss is unacceptable.**

- **Web Browsing (HTTP/HTTPS):** When loading a website, losing a few packets could mean missing paragraphs of text or broken
  page styling.
- **File Downloads (FTP/SFTP):** If a single bit of a downloadable software file or PDF is missing, the entire file becomes
  corrupted and unusable.
- **Email (SMTP/IMAP):** Messages must arrive fully intact and readable.
- **Database Queries:** Financial transactions and user data lookups require absolute accuracy.
