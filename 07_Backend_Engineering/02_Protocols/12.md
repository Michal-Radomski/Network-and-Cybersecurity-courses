TLS uses a mix of multiple cryptographic algorithms to secure communications. These algorithms work together during a
connection and are grouped into sets called Cipher Suites. Each part of the TLS process uses a specific type of algorithm.

## 🔑 Key Exchange Algorithms

These algorithms allow the client and server to safely share a secret key over an insecure network.

- ECDHE (Elliptic Curve Diffie-Hellman Ephemeral) - Used in modern TLS 1.3 for temporary, secure keys.
- DHE (Diffie-Hellman Ephemeral) - An older method to share keys safely.
- RSA - Used in older versions like TLS 1.2 to share the secret key.

## 🔏 Authentication & Digital Signatures

These algorithms prove that the website or server you are connecting to is actually who they claim to be.

- ECDSA (Elliptic Curve Digital Signature Algorithm) - A modern, fast way to verify identities.
- RSA - The traditional, widely compatible way to sign digital certificates.
- Ed25519 - A very fast and secure modern signature method.

## 🔒 Symmetric Encryption Algorithms

Once a secure connection is made, these algorithms encrypt the actual data being sent back and forth. They use a single
shared key.

- AES-GCM (Advanced Encryption Standard) - The most common and secure algorithm used today.
- ChaCha20-Poly1305 - A fast alternative, great for mobile devices without specialized hardware.

## 🏷️ Integrity & Hashing Algorithms

These algorithms create a unique digital fingerprint for data. They make sure nobody tampers with your information while it
travels.

- SHA-256 (Secure Hash Algorithm) - Creates a 256-bit fingerprint.
- SHA-384 - Creates a stronger 384-bit fingerprint.

---

## 🔄 Algorithm Differences: TLS 1.2 vs TLS 1.3

The two most common versions of TLS handle algorithms differently:

| Feature / Version       | TLS 1.2                                    | TLS 1.3 (Modern)                |
| ----------------------- | ------------------------------------------ | ------------------------------- |
| Number of Cipher Suites | Hundreds of combinations                   | Only 5 streamlined options      |
| Older Algorithms        | Allowed weak options like MD5, SHA-1, 3DES | Completely banned weak options  |
| Speed                   | Needs extra round-trips to pick algorithms | Instantly picks secure defaults |
