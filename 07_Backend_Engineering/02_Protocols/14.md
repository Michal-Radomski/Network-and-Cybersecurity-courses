An HTTP request is a message sent from a computer to a server. [1] It asks the server to send a file, save information, or do
a task [1]. For example, your web browser sends an HTTP request every time you click a link [1]. [2, 3, 4, 5, 6]

## 🧱 The 3 Main Parts of a Request

Every HTTP request has three main pieces: [7, 8]

- Request Line: The first line that says what the browser wants to do [2, 3].
- Headers: Extra information about the browser, like what language it uses [2, 3].
- Body: The actual data or message being sent, though it is often empty [3]. [9, 10, 11, 12, 13]

---

## 🛠️ Common Request Methods

The request line uses a specific action word called a Method to tell the server what to do [1]: [14, 15]

- GET: Asks the server to send back a page or file [1, 2].
- POST: Sends new data to the server, like a filled-out form or password [1, 2].
- PUT: Replaces an old file or data on the server with new data [4].
- DELETE: Tells the server to remove a specific file or piece of data [4]. [16, 17, 18, 19, 20]

---

## 📂 Example of a Raw HTTP Request

Here is what a simple request looks like behind the scenes when you visit a website:

GET /index.html HTTP/1.1 Host: ://example.com User-Agent: Mozilla/5.0 Accept-Language: en-US

What this means:

1.  GET /index.html HTTP/1.1 is the Request Line. It asks for the page /index.html using version 1.1 of HTTP [2, 3].
2.  Host: ://example.com tells the server which website you want [2, 3].
3.  User-Agent tells the server you are using a Mozilla web browser [2].
4.  Accept-Language tells the server you prefer the text in English [2]. [21, 22, 23, 24, 25]

---

## 🔗 Connection to TLS

If you use HTTPS (the secure version of HTTP), this whole request is scrambled using the TLS algorithms we talked about
before. The request is completely encrypted before it leaves your computer so hackers cannot read it. [26, 27, 28, 29, 30]
