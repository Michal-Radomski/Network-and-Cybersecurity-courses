Full Cone NAT and PAT (Port Address Translation) are two entirely different networking concepts that operate on different
dimensions of translation. Full Cone NAT defines filtering strictness (who can send traffic back in), while PAT defines
resource sharing (how multiple devices share an IP address). Here is the direct breakdown of their differences.

## The Core Difference

- PAT (Port Address Translation) is a resource-saving mechanism. It allows thousands of internal devices to share a single
  public IP address by assigning each device a unique source port number. It is also known as "Many-to-One" NAT.
- Full Cone NAT is an access policy. It establishes a 1-to-1 mapping where any external internet host can send unsolicited
  traffic to an internal device once a port mapping is open.

## Side-by-Side Comparison

| Feature          | Full Cone NAT                                                 | PAT (Port Address Translation)                            |
| ---------------- | ------------------------------------------------------------- | --------------------------------------------------------- |
| Primary Purpose  | Ensure uninterrupted, open bidirectional traffic.             | Conserve public IP addresses.                             |
| Mapping Ratio    | 1:1 (One private endpoint to one public endpoint).            | Many:1 (Many private IPs to one public IP).               |
| Inbound Security | Low. Any random external IP can send data to the mapped port. | High. Rejects all unsolicited inbound traffic by default. |
| Common Use Case  | Online gaming (Open NAT), VoIP, and P2P applications.         | Standard home routers and enterprise internet access.     |

## How They Interact

They are not mutually exclusive. A router can actually run Full Cone PAT.

- In a standard Restricted PAT router (common), if Device A and Device B both browse the web, they share one public IP using
  different ports. If an outside hacker tries to send traffic to one of those ports, the router blocks it because the
  internal device never talked to that hacker.
- In a Full Cone PAT router, the devices still share one public IP using different ports. However, if Device A opens a port
  for a game, any external player on the internet can send data directly to Device A using that shared public IP and specific
  port, without being blocked.
