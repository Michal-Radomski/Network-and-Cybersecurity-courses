The **Internet Protocol (IP)** is the primary set of rules for routing and addressing packets of data so that they can travel
across networks and arrive at the correct destination. In the OSI model, it operates at **Layer 3 (Network Layer)**.

Think of IP as the **logistics system** of the internet. If the data is a letter, the IP protocol is responsible for putting
the address on the envelope and deciding which "roads" it should take to get to the recipient.

---

### Key Characteristics of IP

- **Connectionless:** IP does not establish a "handshake" before sending data. It just sends the packet and hopes it gets
  there. (Reliability is handled by **TCP** at Layer 4).
- **Best-Effort Delivery:** IP doesn't guarantee that data will arrive, that it will arrive in order, or that it won't be
  corrupted.
- **Media Independent:** It doesn't care if the data is traveling over fiber optics, copper wire, or Wi-Fi.

---

### The IP Packet Structure

Every piece of data sent over the internet is broken into **packets**. An IP packet consists of two main parts:

1. **The Header:** Contains "meta-data" like the Source IP, Destination IP, Version (IPv4/IPv6), and TTL (Time to Live).
2. **The Payload:** The actual data being transported (e.g., a piece of an email or a snippet of a YouTube video).

---

### IPv4 vs. IPv6

There are currently two versions of IP in use today:

| Feature             | IPv4                          | IPv6                                     |
| ------------------- | ----------------------------- | ---------------------------------------- |
| **Address Size**    | 32-bit                        | 128-bit                                  |
| **Address Format**  | Numeric (e.g., `192.168.1.1`) | Alphanumeric (e.g., `2001:0db8:85a3...`) |
| **Total Addresses** | ~4.3 Billion                  | ~340 Undecillion (virtually infinite)    |
| **Security**        | Optional (IPsec)              | Built-in (IPsec support mandated)        |
| **Configuration**   | Manual or DHCP                | Self-configuration (SLAAC)               |

---

### How IP Works (The Routing Process)

1. **Addressing:** Every device on a network is assigned a unique IP address.
2. **Encapsulation:** The IP protocol takes a segment from the Transport Layer and wraps it in an IP header.
3. **Routing:** When a packet reaches a **Router**, the router looks at the **Destination IP**. It checks its "Routing Table"
   to see which path is the fastest or shortest to get that packet closer to its home.
4. **Fragmentation:** If a packet is too large for a specific network link to handle, the IP protocol can break it into
   smaller pieces and reassemble them at the destination.

### Essential Fields in the IP Header

- **Source IP Address:** Where the packet came from.
- **Destination IP Address:** Where the packet is going.
- **TTL (Time to Live):** A counter that prevents packets from looping forever. Every time a packet hits a router, the TTL
  drops by 1. If it hits 0, the packet is discarded.
- **Protocol:** Tells the receiving computer which Layer 4 protocol is inside (usually TCP or UDP).
