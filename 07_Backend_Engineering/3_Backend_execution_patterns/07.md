In Node.js, `UV_THREADPOOL_SIZE` and `worker_threads` handle concurrency in completely different ways, and they have distinct
default configurations.

Here is the breakdown of their default settings and how they operate.

---

## 1. UV_THREADPOOL_SIZE (libuv Thread Pool)

The `UV_THREADPOOL_SIZE` environment variable controls the size of the thread pool managed by **libuv**, the C++ library that
handles Node.js's asynchronous I/O operations. This pool is used for heavy tasks that cannot be done asynchronously at the OS
level (like file system operations, certain cryptographic functions, and compression).

- **Default Size:** **`4`**
- **Maximum Limit:** `1024`
- **Behavior:** All standard Node.js applications share these 4 threads by default for background tasks like `fs.readFile()`
  or `crypto.pbkdf2()`. If all 4 threads are busy, a 5th file system request will block and wait until one of the threads
  becomes free.

> **Note:** You can increase this by setting the environment variable before Node.js starts (e.g.,
> `process.env.UV_THREADPOOL_SIZE = 64;` at the very top of your entry file, or via the terminal
> `UV_THREADPOOL_SIZE=8 node app.js`).

---

## 2. worker_threads (JavaScript Workers)

The `worker_threads` module allows you to run CPU-intensive JavaScript code in parallel on multiple threads. Unlike libuv,
these threads run actual JavaScript code instances.

- **Default Size:** **`0`** (None are created automatically)
- **Maximum Limit:** No hardcoded limit in Node.js; it is limited only by your system's memory and CPU resources.
- **Behavior:** Node.js runs on a single main event loop thread by default. Worker threads are **explicitly managed**. A new
  thread is only created when you manually instantiate it in your code using `new Worker()`.

---

## Summary Comparison

| Feature              | `UV_THREADPOOL_SIZE`                 | `worker_threads`                                      |
| -------------------- | ------------------------------------ | ----------------------------------------------------- |
| **Default Value**    | **`4`**                              | **`0`** (Must be manually created)                    |
| **Managed By**       | libuv (C++ / Internal Node.js)       | V8 / Developer (JavaScript layer)                     |
| **Purpose**          | Asynchronous I/O, Crypto, Zlib tasks | CPU-bound JavaScript execution                        |
| **Memory Isolation** | Shared across the Node.js process    | Isolated V8 environments (communicates via Messaging) |
