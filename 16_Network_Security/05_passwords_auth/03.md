---

## Why You Shouldn't "Encrypt" Hashes

Encryption is **two-way** (reversible with a key), while hashing is **one-way** (irreversible).

If you encrypt a password hash, you introduce a massive security headache: **key management**.

- If a hacker breaches your database _and_ steals the encryption key from your server config, your encryption is useless.
- If you lose the encryption key, you can no longer verify any user passwords, effectively locking everyone out of your
  system.

Instead of encrypting hashes, rely on a mathematically strong hashing algorithm with a **salt**.

### The Exception: "Peppering"

If you want an extra layer of defense, you can use a **pepper**. A pepper is a secret string stored on your server (not in
the database). Instead of encrypting the final hash, you run your password through an HMAC (Hash-based Message Authentication
Code) using the pepper _before_ passing it to your hashing algorithm. If the database is leaked but the server config is
safe, the attacker cannot attempt to crack the hashes.

## How to Securely Store Passwords in Node.js

The industry standard algorithms for password hashing in Node.js are **Argon2** (the modern gold standard) and **bcrypt**
(extremely reliable and widely used).

Below is how to securely hash and verify passwords in Node.js using the popular `bcrypt` library.

### 1. Install Bcrypt

In your Node.js project, install the library:

```bash
npm install bcrypt

```

### 2. Hashing a Password (During Registration)

When a user signs up, you hash their password before saving it to the database. `bcrypt` handles generating a unique salt
automatically.

```javascript
const bcrypt = require("bcrypt");

async function registerUser(plainPassword) {
  // The "salt rounds" determines how secure (and slow) the hash is.
  // 10 to 12 rounds is currently the industry sweet spot.
  const saltRounds = 10;

  try {
    const hashedPassword = await bcrypt.hash(plainPassword, saltRounds);

    // Save 'hashedPassword' to your database.
    // NEVER save the 'plainPassword'.
    return hashedPassword;
  } catch (error) {
    console.error("Error hashing password:", error);
  }
}
```

### 3. Verifying a Password (During Login)

When a user attempts to log in, you retrieve their hashed password from the database and use `bcrypt.compare()` to check if
the plain-text password they typed matches.

```javascript
async function loginUser(enteredPassword, storedHashFromDatabase) {
  try {
    // bcrypt automatically extracts the salt from the stored hash
    // to securely compare the entered password.
    const isMatch = await bcrypt.compare(enteredPassword, storedHashFromDatabase);

    if (isMatch) {
      console.log("Login successful!");
      return true;
    } else {
      console.log("Invalid password.");
      return false;
    }
  } catch (error) {
    console.error("Error verifying password:", error);
  }
}
```

## Summary Checklist for Node.js Password Security

- [ ] **Never write your own hashing algorithm.**
- [ ] **Never use fast algorithms** like MD5, SHA-1, or SHA-256/512 directly for passwords (they are too fast and easily
      cracked on modern GPUs).
- [ ] **Use Bcrypt or Argon2** with an appropriate cost/work factor.
- [ ] **Ensure every password has a unique salt** (Bcrypt and Argon2 handle this automatically).
- [ ] **Transmit passwords over HTTPS** so they cannot be intercepted in transit before they even reach your Node.js server.
