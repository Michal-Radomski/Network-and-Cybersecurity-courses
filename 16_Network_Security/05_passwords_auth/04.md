To understand how passwords are cracked, it helps to understand how they are stored in the first place.

Most modern systems do not store your actual password. Instead, they run your password through a cryptographic algorithm to
generate a **hash** (a unique, fixed-length string of characters). When you log in, the system hashes what you typed and
compares it to the stored hash.

Therefore, "cracking" a password usually means taking a list of stolen hashes (typically obtained from a database breach) and
trying to figure out what original plain-text passwords created them.

---

## The Core Concept: Guess and Check

Because cryptographic hashes are designed to be **one-way** (you cannot reverse a hash back into a password), attackers
cannot just "decrypt" a hash. Instead, they must guess.

1. The attacker's computer generates a guess (e.g., `Password123`).
2. The computer hashes that guess using the same algorithm as the target database.
3. The computer compares the resulting hash to the stolen hash.
4. If they match, the password is cracked. If not, it tries another guess.

---

## 4 Common Password Cracking Methods

### 1. Dictionary Attacks

This is the simplest method. Attackers use a pre-compiled list of common words, phrases, and previously leaked passwords
(often called a "wordlist").

- **How it works:** Instead of guessing randomly, the cracking software quickly runs through millions of real words (like
  `admin`, `qwerty`, or `iloveyou`) to see if any of them match the target hash.

### 2. Rule-Based Attacks

Since most people know they shouldn't use simple dictionary words, they often modify them (e.g., changing "monkey" to
"M0nkey!"). Attackers anticipate this using rules.

- **How it works:** The attacker takes a dictionary list and applies rules to automatically generate variations. The software
  will automatically capitalize the first letter, append common number sequences (like `2026` or `123`), or swap letters for
  symbols (like `@` for `a`).

### 3. Brute-Force Attacks

This is a brute-strength method where the computer systematically tests every possible combination of characters.

- **How it works:** It starts with `a`, then `b`, then `c`... eventually moving to `aa`, `ab`, `ac`, and so on.
- **The limitation:** While guaranteed to work eventually, brute-force attacks are highly inefficient. A completely random,
  12-character password containing uppercase letters, lowercase letters, numbers, and symbols could take billions of years to
  crack using standard hardware.

### 4. Rainbow Tables

A rainbow table is a massive, pre-computed database of plain-text passwords and their corresponding hashes.

- **How it works:** Instead of calculating hashes on the fly, the attacker simply looks up the stolen hash in the table to
  see if the plain-text password is already listed.
- **The defense:** This attack is easily defeated by **salting**—adding a unique, random string of characters to each
  password before it is hashed, which makes pre-computed tables completely useless.

---

## The Hardware Behind the Speed

To run these attacks at scale, hackers don't use standard computer processors (CPUs). Instead, they use **Graphics Processing
Units (GPUs)** or specialized chips (ASICs).

A standard CPU is built to handle a few complex tasks at once. A GPU, however, has thousands of smaller cores designed to do
simple math in parallel. Because calculating a hash is a very simple mathematical operation, a high-end consumer GPU can
guess **billions** of simple hashes (like MD5 or SHA-1) per second.

---

## How to Make Passwords Uncrackable

Understanding how hackers crack passwords makes it easy to see how to defend against them:

- **Length over Complexity:** A longer password (or passphrase made of 4-5 random words, like `correct-horse-battery-staple`)
  is exponentially harder to brute-force than a short password with complex symbols (like `P@$$w0rd!`).
- **Avoid Predictability:** Do not use personal information (birthdays, pet names) or common substitution patterns that
  rule-based attacks easily guess.
- **Use a Password Manager:** Generate unique, completely random passwords for every single account so that a breach at one
  website doesn't compromise your other accounts.
- **Enable Multi-Factor Authentication (MFA):** Even if an attacker successfully cracks your password hash, MFA acts as a
  second lock that prevents them from gaining access to your account.
