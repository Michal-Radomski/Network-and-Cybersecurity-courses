Operating systems (like Windows, macOS, Linux, and mobile OSes like Android and iOS) handle passwords differently than
standard websites. Because they control the physical device and all the data on it, their password security models are deeply
integrated with hardware and local encryption.

Here is how modern operating systems store, protect, and verify your passwords.

---

## 1. How OS Passwords are Stored

Operating systems do not store your password in plain text. Instead, they use specialized database files and strong hashing
algorithms.

### Windows (Active Directory & Local)

- **Local Accounts (SAM Database):** Windows stores local user passwords in a protected database file called the **SAM
  (Security Accounts Manager)**.
- **The Hashing Algorithm:** Modern Windows versions hash passwords using the **NTLM** (NT LAN Manager) hashing algorithm
  (specifically, Unicode MD4).
- _Security Note:_ NTLM is an older, relatively fast hashing algorithm. Because it lacks modern "memory-hard" protections,
  Windows relies heavily on restricting access to the SAM file itself to prevent attackers from dumping and cracking the
  hashes.

- **Microsoft Accounts:** If you sign in with a Microsoft account, Windows uses modern, cloud-based authentication protocols
  (like OAuth) rather than storing a local NTLM hash of your actual Microsoft password.

### macOS and Linux

- **Linux (`/etc/shadow`):** Linux systems store user account details in `/etc/passwd` (readable by anyone), but the actual
  password hashes are kept in a highly restricted file called `/etc/shadow` (readable only by the root user). Modern Linux
  distributions typically use **yescrypt**, **SHA-512**, or **bcrypt** to hash these passwords, making them incredibly
  difficult to crack even if the file is stolen.
- **macOS (Directory Services):** macOS stores password hashes in its Open Directory structure under
  `/var/db/dslocal/nodes/Default/users/`. macOS uses a highly secure, salted SHA-512 based hashing system (specifically
  PBKDF2 with SHA-512) to protect local passwords.

## 2. The Shift to Hardware-Backed Security

Because local software-based files (like the SAM file or `/etc/shadow`) can sometimes be extracted if an attacker gets
physical access to a hard drive, modern operating systems have shifted toward **hardware-backed security**.

Instead of relying solely on a software database, modern OS security is anchored in physical security chips on your device’s
motherboard:

- **Windows:** Uses the **TPM (Trusted Platform Module)** chip.
- **macOS & iOS:** Uses Apple's custom **Secure Enclave**.
- **Android:** Uses a hardware-backed **Keystore** (often run inside a Trusted Execution Environment, or TEE).

These hardware chips act as a vault inside your device. When you set up a PIN, fingerprint, or facial recognition (like
Windows Hello, Touch ID, or Face ID), the cryptographic keys associated with your account are locked inside this hardware
chip. The operating system itself cannot see the raw keys; it simply asks the chip, _"Did the user provide the correct
PIN/biometric?"_, and the chip returns a simple _"Yes"_ or _"No"_.

---

## 3. Local Passwords vs. Full Disk Encryption

It is important to know that a standard operating system login password **only protects your system while it is running**.

- **The Vulnerability:** If your computer is stolen and does not have **Full Disk Encryption (FDE)** enabled, an attacker
  doesn't need to guess your password. They can simply pull the hard drive out of your computer, plug it into another
  machine, and read all your files directly.
- **The Solution:** Modern operating systems use FDE to encrypt the entire hard drive.
- **Windows** uses **BitLocker**.
- **macOS** uses **FileVault**.
- **Android & iOS** encrypt user data partitions by default.

When FDE is enabled, your login password (or the hardware keys unlocked by your password/PIN) is used to derive the
cryptographic key needed to decrypt the hard drive. Without your password, the files on the drive look like complete
gibberish, even if the drive is physically removed.
