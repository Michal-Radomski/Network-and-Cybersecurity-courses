The **HTTP Referer** (yes, it is officially misspelled in the web specifications!) is an HTTP header field that identifies
the address of the web page that linked to the resource being requested.

In simple terms: it tells a website **where a visitor came from**.

---

### How It Works

When you click a link on a webpage, your web browser automatically sends a request to the new website's server. Inside that
request, it includes the `Referer` header containing the URL of the page you were just on.

#### A Quick Example:

1. You are reading an article on `blogs.com/tech-news`.
2. You click a link in that article that takes you to `shop.com/gadget`.
3. Your browser sends a request to `shop.com`. Inside that request, the `Referer` header looks like this:
   `Referer: https://blogs.com/tech-news`
4. `shop.com` now knows exactly which blog post sent them a visitor.

---

### Why is it used?

Websites use the Referer header for several practical reasons:

- **Analytics and Marketing:** It helps website owners see which external sites, search engines, or social media platforms
  are driving traffic to their pages.
- **Optimizing UX:** If a user gets a "404 Not Found" error, the server can look at the referer to see which broken link
  directed them there and fix it.
- **Security (CSRF Protection):** Some sites check the referer header to ensure a request (like changing a password or making
  a purchase) actually originated from their own website and not a malicious third-party site.

---

### Privacy and Security Concerns

Because the Referer header passes along the exact URL you visited, it can sometimes accidentally leak private information.
For instance, if a URL contains a reset token or personal query parameters (like `?email=user@example.com`), that data gets
sent to the next website you click on.

To fix this, modern web standards introduced the **Referrer-Policy** header. This allows websites to control how much
information browser sharing sends. For example:

- **`no-referrer`:** The browser will never send the Referer header.
- **`same-origin`:** The browser only sends the referer if you are staying on the same website.
- **`strict-origin-when-cross-origin`:** (The modern default) The browser sends the full URL when staying on the same site,
  but _only_ sends the domain name (e.g., `https://blogs.com`) when leaving for a different website, stripping out the
  specific page path.
