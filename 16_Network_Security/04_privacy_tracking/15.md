The **Browser Exploitation Framework (BeEF)** is an open-source penetration testing tool tailored specifically to exploit
weaknesses in web browsers. Unlike operating system-level frameworks (such as Metasploit), BeEF operates strictly within the
client-side context of the browser.

It allows security professionals to assess the security posture of an environment by bypassing network-level firewalls and
focusing entirely on the "open window" that is a user's web browser.

---

## How BeEF Works

The framework functions on a hook-and-control mechanism:

1. **The Hook:** A JavaScript file (`hook.js`) hosted by the BeEF server is executed by a target browser. This usually occurs
   through a Cross-Site Scripting (XSS) vulnerability, a compromised website, or via authorized social engineering testing.
2. **The Command Center:** Once the browser executes the script, it becomes "hooked" and establishes a persistent, silent
   communication channel back to the BeEF control panel.
3. **Execution:** The tester can then push modules to the hooked browser to gather system details, execute social engineering
   tricks (like fake login prompts), map internal networks, or fingerprint the device.

## Setting Up BeEF on Kali Linux

Kali Linux includes BeEF in its package repositories, making installation and startup straightforward.

### Step 1: Install BeEF

Open your terminal and update your package lists, then install the `beef-xss` package:

```bash
sudo apt update
sudo apt install beef-xss

```

### Step 2: Start the Service

You can start BeEF using the designated Kali helper script:

```bash
sudo beef-xss-start

```

> **Note:** On your very first run, the terminal will prompt you to change the default password. For security reasons, the
> framework will not launch if the password remains set to `beef`.

### Step 3: Access the Control Panel

Once initiated, your terminal will provide the local interface URL. Open your browser and navigate to:

```text
http://127.0.0.1:3000/ui/panel

```

Log in using the username `beef` and the new password you configured.

## Setting Up BeEF on Ubuntu

Because BeEF is not in the default Ubuntu repositories, you must clone it from source and install its prerequisites (Ruby and
Node.js).

### Step 1: Install Dependencies

Open your terminal and install the required build tools and libraries:

```bash
sudo apt update
sudo apt install git ruby ruby-dev nodejs libsqlite3-dev build-essential

```

### Step 2: Clone the Repository

Navigate to the directory where you want to keep the tool (e.g., `/opt`) and clone BeEF from GitHub:

```bash
cd /opt
sudo git clone https://github.com/beefproject/beef.git
cd beef

```

### Step 3: Install Ruby Bundler & Gems

Install Bundler, which manages the specific Ruby gems required by BeEF, and run the installation script:

```bash
sudo gem install bundler
sudo ./install

```

### Step 4: Configure Credentials

Before launching, you must manually change the default password in the configuration file:

```bash
nano config.yaml

```

Look for the `credentials` section under `beef:`, change the `passwd: "beef"` line to a secure custom password, and save the
file (`Ctrl+O`, then `Ctrl+X`).

### Step 5: Launch BeEF

Run the startup script:

```bash
sudo ./beef

```

Open your browser and navigate to `http://127.0.0.1:3000/ui/panel` to log in.

## Verifying the Setup (Basic Testing)

To verify everything is working safely without targeting outside systems, you can hook your own local browser:

1. Look at the terminal output or the **Getting Started** tab in the BeEF interface to find the demonstration page link
   (typically `http://127.0.0.1:3000/demos/basic.html`).
2. Open that link in a separate browser window or tab.
3. Return to your BeEF Control Panel. You should see your local machine's IP address populate under the **Online Browsers**
   tree on the left sidebar.
4. Clicking on the hooked IP allows you to explore the **Commands** tab to see what diagnostic modules are available.
