**WPA2 Enterprise** is a high-security flavor of the WPA2 wireless standard designed for businesses, schools, and large
organizations.

While standard home Wi-Fi (WPA2 Personal) uses a single pre-shared key (PSK)—meaning everyone connects using the exact same
password—WPA2 Enterprise provides **individualized authentication** for every single user and device.

---

## How It Works: The Three-Component System

Instead of just checking your laptop against the router, WPA2 Enterprise relies on an industry-standard framework called
**802.1X**, which divides the connection process into three distinct parts:

1. **The Supplicant (The Client):** The user's device (like your phone or laptop) requesting access to the network.
2. **The Authenticator (The Access Point):** The Wi-Fi router or access point. It acts as a gatekeeper, blocking the device
   from the internet until its identity is verified.
3. **The Authentication Server (The Brains):** Usually a **RADIUS server** (Remote Authentication Dial-In User Service). This
   server holds the database of allowed users or links directly to an organization's central directory (like Microsoft Active
   Directory or Google Workspace).

---

## WPA2 Personal vs. WPA2 Enterprise

| Feature                   | WPA2 Personal (PSK)                                                            | WPA2 Enterprise (802.1X)                                                 |
| ------------------------- | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------ |
| **Authentication Method** | Single, shared password.                                                       | Unique credentials per user (Username/Password or Digital Certificates). |
| **Server Required?**      | No.                                                                            | Yes (RADIUS Server).                                                     |
| **Encryption Keys**       | Every device derives its encryption from the same shared password.             | Every device gets its own unique, dynamic encryption key.                |
| **Revoking Access**       | You must change the Wi-Fi password on _every single device_ if someone leaves. | You simply disable that specific user's account in the central database. |
| **Target Audience**       | Homes and small offices.                                                       | Enterprises, universities, and government networks.                      |

---

## Why Organizations Use It

- **Centralized Control:** If an employee leaves a company, IT can immediately revoke their network access by disabling their
  corporate account. The remaining employees don't have to change their Wi-Fi settings.
- **Protection Against Internal Snooping:** Because every device gets its own unique encryption key generated by the RADIUS
  server, users on the same Wi-Fi network cannot easily intercept or decrypt each other's traffic.
- **Accountability:** Network administrators can look at the logs and see exactly _who_ logged into the network, on _what_
  device, and _when_, rather than just seeing a generic MAC address.
- **Flexible Credentials:** It supports multiple authentication methods (via protocols known as EAP), allowing users to log
  in with standard usernames/passwords, smart cards, or secure digital certificates pre-installed on corporate hardware.
