In computer networking, a **DMZ** (short for **Demilitarized Zone**, borrowing a military term for a neutral buffer zone) is
a physical or logical subnetwork that acts as a secure perimeter between an organization's internal, private network and an
untrusted, public network—usually the internet.

The main goal of a DMZ is to allow external users access to certain public-facing services (like a web server or email
server) while keeping the organization's critical internal data safe behind a firewall.

---

## How a DMZ Works

Think of a DMZ like the lobby of a high-security office building. Anyone from the public can walk into the lobby to drop off
mail or talk to a receptionist. However, the lobby is completely separated from the rest of the building by security
turnstiles and locked doors. Visitors cannot access the main offices or the vault without explicit permission.

In a network layout:

1. **The Internet (Public):** External users connect to the servers inside the DMZ.
2. **The DMZ (Buffer):** Hosts public-facing servers. If a hacker successfully compromises a web server in the DMZ, they are
   trapped in the "lobby."
3. **The Local Network (Private):** Hidden safely behind another layer of firewall protection, completely isolated from
   direct access from the internet.

---

## What Usually Goes Inside a DMZ?

Any service that needs to be accessed by the general public over the internet should be placed in the DMZ. Common examples
include:

- **Web Servers:** The public needs to visit your website, but the website doesn't need access to internal corporate files.
- **Email (SMTP) Servers:** External mail servers must be able to send emails to your domain.
- **FTP/File Servers:** Used for exchanging files with outside clients.
- **DNS Servers:** Publicly resolves domain names to IP addresses.

---

## Common DMZ Architectures

There are two primary ways network administrators set up a DMZ:

### 1. The Single Firewall (Three-Legged) Model

This setup uses a single firewall with at least three network interfaces (or "legs").

- **Leg 1** connects to the internet.
- **Leg 2** connects to the DMZ.
- **Leg 3** connects to the internal network. The firewall manages the traffic rules for all three zones simultaneously. If
  the firewall itself is completely breached, the entire network could be at risk, but it is a cost-effective and common
  setup.

### 2. The Dual Firewall (Back-to-Back) Model

This is a more secure, enterprise-grade architecture that uses two separate firewalls.

- **The Front-End Firewall:** Sits between the internet and the DMZ. It only allows traffic destined for the public servers.
- **The Back-End Firewall:** Sits between the DMZ and the internal private network. It only allows very specific traffic from
  the DMZ into the local network (for instance, a web server pulling data from an internal database). For a hacker to reach
  your private data, they have to breach two entirely different firewall systems.

---

## DMZ in Home Routers: A Word of Warning

Many consumer home Wi-Fi routers have a setting labeled **"DMZ"** or **"DMZ Host."** It is important to know that a home
router's DMZ is **not** a true DMZ. In a home router, assigning a device (like a gaming console or a PC) as the DMZ Host
simply means **forwarding every single incoming internet port to that device**, completely bypassing the router's built-in
firewall.

While gamers sometimes do this to fix connection/NAT issues, placing a standard computer in a home DMZ exposes it directly to
the internet without any protection, making it highly vulnerable to attacks.
