When it comes to securing wireless networks, Wi-Fi security protocols have evolved significantly over the years to fix
critical vulnerabilities. Here is a breakdown of WEP, WPA, WPA2, and the encryption protocols they use (TKIP and CCMP).

---

## 1. WEP (Wired Equivalent Privacy)

Introduced in 1997 as part of the original 802.11 standard, WEP was designed to give wireless networks the same level of
security as a wired network.

- **How it works:** It uses the RC4 stream cipher for encryption and a static key.
- **The Flaw:** The implementation of its Initialization Vector (IV) was deeply flawed. Because the IVs were short and
  frequently reused, attackers could easily intercept data packets, analyze them, and crack the Wi-Fi password in minutes.
- **Status:** **Obsolescent and highly insecure.** It should never be used today.

---

## 2. WPA (Wi-Fi Protected Access)

Released in 2003, WPA was developed as a temporary, stop-gap measure to replace WEP without requiring users to buy new
hardware (it could be deployed via firmware updates).

- **The Fix:** WPA introduced **TKIP** (Temporal Key Integrity Protocol) to fix WEP’s vulnerabilities.
- **Status:** **Deprecated.** While much safer than WEP, it was still vulnerable to advanced attacks and has been phased out.

### What is TKIP?

> **TKIP (Temporal Key Integrity Protocol)** was designed to wrap around WEP's flawed RC4 encryption. Instead of using a
> static key like WEP, TKIP automatically generates a unique, dynamic 128-bit key for every single data packet sent. It also
> included a message integrity check to prevent attackers from altering packets in transit.

---

## 3. WPA2 (Wi-Fi Protected Access 2)

Released in 2004, WPA2 became the official, long-term successor to WEP and WPA. It required a complete overhaul of the
underlying hardware to support stronger cryptographic algorithms.

- **The Fix:** WPA2 dropped TKIP in favor of **CCMP**, a much more powerful encryption mechanism based on AES (Advanced
  Encryption Standard).
- **Status:** **Mainstream.** While WPA3 is the current modern standard, WPA2 is still widely deployed and considered secure
  for most general uses (provided it uses a strong password to mitigate certain handshake vulnerabilities like KRACK).

### What is CCMP?

> **CCMP (Counter Mode with Cipher Block Chaining Message Authentication Code Protocol)** is the encryption protocol used by
> WPA2. It is built on the **AES** algorithm, which is a military-grade block cipher. CCMP provides significantly stronger
> data confidentiality and integrity than TKIP, ensuring that data cannot be decrypted or tampered with by unauthorized
> parties.

---

## Quick Comparison Summary

| Protocol | Release Year | Encryption Cipher | Key Management Protocol | Security Level       |

| **WEP**  | 1997         | RC4 (Static)      | None                    | **None (Broken)**    |

| **WPA2** | 2004         | AES               | **CCMP**                | **High (Standard)**  |
