**UFW**, or **Uncomplicated Firewall**, is a user-friendly frontend for managing firewall rules on Linux. It was developed by
Canonical for Ubuntu, but it is available on many other Linux distributions as well.

While tools like `iptables` and `nftables` are incredibly powerful, their syntax can be complex and difficult to memorize.
UFW acts as a simplified wrapper around those backend tools, allowing you to secure your system using straightforward,
easy-to-read commands.

---

## How UFW Works

UFW focuses on managing basic incoming and outgoing traffic rules. By default, when you enable UFW, it blocks all incoming
connections and allows all outgoing connections. You then open specific "ports" to allow external traffic in.

### Example Syntax Comparison

To see how much simpler UFW is, look at how you would allow web traffic (port 80) compared to `iptables`:

- **`iptables` syntax:** `iptables -A INPUT -p tcp --dport 80 -j ACCEPT`
- **`UFW` syntax:** `ufw allow 80/tcp` (or simply `ufw allow http`)

---

## Common UFW Commands

Because UFW is designed to be simple, managing it usually only requires a handful of basic commands:

- **Check the status:** `sudo ufw status` (shows if the firewall is active and lists current rules).
- **Enable the firewall:** `sudo ufw enable`
- **Disable the firewall:** `sudo ufw disable`
- **Allow specific traffic:** `sudo ufw allow 22/tcp` (allows SSH traffic).
- **Block specific traffic:** `sudo ufw deny 80/tcp` (blocks HTTP traffic).
- **Delete a rule:** `sudo ufw delete allow 22/tcp`

---

## Summary

Think of `nftables` or `iptables` as the advanced, under-the-hood engine that does the heavy lifting of filtering network
packets, while **UFW is the clean dashboard** that lets you control that engine without needing to know all the complex
mechanics. It is ideal for home users, developers, and small server administrators who need quick, reliable security without
a steep learning curve.
