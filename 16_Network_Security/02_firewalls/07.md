On macOS, firewalls work a bit differently than on Linux. While Linux firewalls like `iptables` or `ufw` are heavily focused
on managing specific network ports and IP addresses, the built-in macOS firewall is **application-based**.

Instead of asking you which port number to block, macOS asks you which _app_ should be allowed to talk to the internet.

Here is a breakdown of how firewalls work on macOS, both built-in and third-party.

---

## 1. The Built-In macOS Application Firewall

Every Mac comes with a built-in firewall that is disabled by default. When enabled, it monitors incoming connections to
applications and services.

### How it Works:

- **App-Centric:** When a new application tries to accept incoming connections from the network, macOS will pop up a prompt
  asking: _"Do you want the application to accept incoming network connections?"_
- **Code Signing:** macOS uses "code signing" to verify the identity of apps. If a trusted, signed app (like a native Apple
  service or an app from the Mac App Store) needs network access, macOS often allows it automatically to reduce popup
  fatigue.
- **Stealth Mode:** The built-in firewall includes a "Stealth Mode" feature. When enabled, your Mac will not respond to
  "ping" requests or connection attempts from hackers scanning a network for vulnerable devices. To the outside world, your
  Mac acts as if it doesn't exist.

### How to Turn It On:

1. Open **System Settings** (or System Preferences on older macOS versions).
2. Click on **Network** in the sidebar, then select **Firewall**.
3. Toggle the **Firewall** switch to on.
4. Click **Options** to customize which apps are allowed or denied, or to enable Stealth Mode.

---

## 2. The Under-the-Hood Firewall: PF (Packet Filter)

While the user interface gives you an application firewall, macOS actually has a second, incredibly powerful network-level
firewall hidden deep inside the operating system called **PF (Packet Filter)**.

PF is inherited from BSD Unix (the foundation macOS is built on). It works exactly like `nftables` or `iptables` on
Linux—filtering raw network packets based on IP addresses, ports, and protocols.

- **Why you don't usually see it:** It has no official Apple graphical interface. It is completely controlled via the
  Terminal using the `pfctl` command.
- **When it's used:** Network administrators and advanced users use PF to create complex rules (like blocking a specific
  malicious IP address or setting up advanced routing).

---

## 3. Third-Party macOS Firewalls (Outbound Filtering)

The biggest limitation of the built-in macOS firewall is that it **only filters incoming traffic**. If a piece of malware or
a sketchy app already on your Mac tries to send your data _out_ to an external server, the built-in firewall will not stop
it.

Because of this, many Mac users look to third-party firewalls that specialize in **outbound filtering**.

Popular options include:

- **Little Snitch / LuLu:** These tools monitor your Mac in real-time. The moment any app tries to connect to a server on the
  internet, a popup asks you to approve or deny the outbound connection. (LuLu is a popular free, open-source alternative to
  Little Snitch).
- **IceFloor / Murus:** These are graphical frontends that give you a visual dashboard to control the hidden, built-in **PF**
  firewall mentioned above, without needing to use the Terminal.

---

## Summary: Do you need to turn it on?

If you are on a secure home Wi-Fi network behind a router, your router's firewall is already doing most of the heavy lifting.
However, if you frequently take your MacBook to public places like coffee shops, airports, or school networks, **turning on
the built-in macOS firewall (and enabling Stealth Mode)** is highly recommended to protect your device from other users on
the same public network.
