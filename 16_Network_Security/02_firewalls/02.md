Yes, **Ubuntu has a built-in SPI (Stateful Packet Inspection) firewall active at the kernel level.** In Linux, the core
firewall engine is built directly into the operating system's kernel through a framework called **Netfilter**. Netfilter
includes a module called `conntrack` (Connection Tracking), which is exactly what provides stateful inspection capabilities.
It tracks the state of network connections (e.g., `NEW`, `ESTABLISHED`, `RELATED`) to intelligent block or allow traffic.

While the backend engine is always there, you interact with it using various user-friendly frontend tools.

---

### The Default Choice: UFW (Uncomplicated Firewall)

Ubuntu comes pre-installed with **UFW**, which serves as a simplified interface to manage the kernel's complex filtering
rules.

- **Status:** It is installed by default but is usually **disabled** initially to avoid accidentally locking users out of a
  headless system or remote server.
- **How to enable it:** You can turn on the SPI firewall features by opening a terminal and running:

```bash
sudo ufw enable

```

- **Checking its status:**

```bash
sudo ufw status verbose

```

### Graphical Interface: GUFW

If you prefer not to use the terminal, you can install a graphical user interface for UFW.

- **How to install:** ```bash sudo apt install gufw

### Advanced Options: `nftables` and `iptables`

For advanced network administrators, the underlying tools that communicate directly with the Netfilter kernel are `nftables`
(the modern standard) and `iptables` (the legacy framework). You don't need to touch these for everyday security, as UFW
safely manages them in the background.
