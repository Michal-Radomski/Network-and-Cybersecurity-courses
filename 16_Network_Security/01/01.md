A **Canary Token** (sometimes called a honeytoken) is a digital tripwire used in cybersecurity to detect unauthorized access.

The name comes from the historical practice of miners carrying a canary into coal mines; if toxic, odorless gases leaked into
the cave, the canary would die first, giving the miners an early warning to evacuate. In the digital world, a canary token
acts as that early warning system for a network or system breach.

---

### How They Work

Canary tokens operate on a simple principle: **"If it is touched, alert the defenders."** 1. **Creation:** A security team
generates a unique, fake asset (like a document, an API key, or a URL) that embeds a unique tracking mechanism (usually a
custom DNS or HTTP callback). 2. **Placement:** The token is strategically placed in a location where legitimate users have
no reason to look, but a malicious attacker or malicious insider executing reconnaissance would find enticing (e.g., a file
named `2026_Salaries.xlsx` or a fake AWS key left in a code repository). 3. **Trigger:** Because the asset is fake, it sits
dormant. However, the moment an attacker finds it and interacts with it (opens the file, clicks the link, or tries to
authenticate with the key), the token silently "phones home" to a logging server. 4. **Alert:** The server immediately sends
a high-fidelity alert to the security team containing forensic metadata, such as the attacker’s public IP address, the
timestamp, and exactly which token was tripped.

---

### Common Types of Canary Tokens

Canary tokens are highly versatile and can take many digital forms:

- **File Tokens:** Word documents, PDFs, or Excel spreadsheets. When opened, they force the application to fetch a remote
  image or resource, triggering the alert.
- **Credential/API Tokens:** Fake AWS keys, Azure service principals, or database passwords placed in configuration files or
  GitHub repositories. The alert fires the instant someone attempts to log in with them.
- **URL/Web Bugs:** Unique links planted in internal documentation or emails. An alert triggers if the link is clicked.
- **DNS Tokens:** Fake hostnames or subdomains. If an attacker runs a network scan and tries to resolve the domain name, the
  custom DNS server catches it.
- **Database Tokens:** Decoy rows or tables inside a database. If a malicious actor dumps the database, the interaction
  triggers a notification.

---

### Why Security Teams Use Them

- **High Signal, Low Noise:** Traditional security tools sift through millions of logs trying to guess if behavior is
  malicious, leading to alert fatigue. Canary tokens yield almost **zero false positives**—because no legitimate user should
  ever be touching them.
- **Early Detection:** They catch attackers during the reconnaissance or lateral movement phase of a breach, allowing
  defenders to contain the threat before any actual data is stolen or ransomware is deployed.
- **Cost-Effective and Lightweight:** Unlike massive "honeypots" (which mimic entire servers or networks), canary tokens are
  tiny, require virtually no computing resources, and can be deployed by the thousands across cloud and on-prem environments.
