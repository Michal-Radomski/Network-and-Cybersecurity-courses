## What is OpenVAS?

**OpenVAS** (Open Vulnerability Assessment Scanner) is a full-featured, open-source software framework used for vulnerability
scanning and vulnerability management. Originally branched from Nessus when it went closed-source, it is now maintained and
developed by Greenbone as part of the **Greenbone Vulnerability Management (GVM)** framework.

It tests endpoints, servers, and network devices against a database of tens of thousands of known security vulnerabilities
(known as Network Vulnerability Tests, or NVTs) and provides detailed reports and remediation guidance.

## How to Install OpenVAS on Ubuntu

The easiest native way to install OpenVAS on modern Ubuntu systems is using the `gvm` meta-package via a dedicated
repository, as native repository packages can sometimes be outdated or broken.

> **System Requirements:** OpenVAS is highly resource-intensive during feed synchronization and scanning. Ensure your system
> has at least **4 vCPUs, 8 GB RAM, and 20 GB of free disk space**.

### Step 1: Update the System and Add the Repository

First, ensure your system is completely up-to-date, then add the GVM community PPA repository:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install software-properties-common -y
sudo add-apt-repository ppa:mrazavi/gvm -y
sudo apt update

```

### Step 2: Install GVM (OpenVAS)

Install the primary toolchain package along with PostgreSQL and Redis, which act as the backend database and temporary
storage:

```bash
sudo apt install gvm -y

```

### Step 3: Run the Setup Script

Once the packages are installed, execute the automated setup script.

- **Note:** This step downloads the latest vulnerability feeds (NVTs, SCAP data, and CERT data). It can take **up to an
  hour** depending on your internet connection and CPU speed.

```bash
sudo gvm-setup

```

_Pay close attention to the end of this command's output, as it will auto-generate an initial **admin password**._

### Step 4: Verify the Installation

Verify that all background daemons (`gvmd`, `openvas-scanner`, `gsad`) are running correctly:

```bash
sudo gvm-check-setup

```

## Accessing the Web Interface

OpenVAS uses the Greenbone Security Assistant (GSAD) web UI, which typically runs on port `9392`.

1. Open your browser and navigate to `https://localhost:9392` (or use your server's IP address instead of `localhost`).
2. Your browser will display a certificate warning because OpenVAS uses a self-signed SSL certificate by default. You can
   safely bypass this warning.
3. Log in using the default username: **admin** and the password generated during `gvm-setup`.

### Quick Tips

- **To change your admin password via the terminal:**

```bash
sudo gvmd --user=admin --new-password=YourNewSecurePassword

```

- **To manually update vulnerability definitions (Feeds) in the future:**

```bash
sudo greenbone-feed-sync

```

> 💡 **Alternative Route:** Greenbone officially recommends running OpenVAS inside **Docker containers** via their official
> Docker Compose files for production or modern setups, as containerized deployment avoids configuration issues with local
> Linux dependencies.
