ARP spoofing (also called ARP poisoning) is a network attack where a hacker sends fake Address Resolution Protocol (ARP)
messages to link their physical hardware (MAC address) with a legitimate IP address on a local network. This tricks devices
into sending their data to the attacker instead of the intended recipient, such as a router or server. [1, 2, 3, 4]

## How the Attack Works

1.  Scanning: The attacker scans the local network to find the IP and MAC addresses of targets, typically a user's computer
    and the network gateway.
2.  Spoofing: Using tools like arpspoof or Ettercap, the attacker broadcasts forged ARP replies that claim the attacker's MAC
    address belongs to the target's IP.
3.  Poisoning: Devices receive these fake messages and update their ARP cache (a local "contacts list" of IP-to-MAC mappings)
    with the fraudulent info.
4.  Interception: Future data meant for the legitimate device is now routed directly to the attacker. [1, 3, 4, 5, 6, 7, 8]

## Why It Is Dangerous

Because the original ARP protocol lacks authentication, devices "trust" these fake updates blindly. This allows hackers to
perform: [1, 9]

- Man-in-the-Middle (MitM): Intercepting and secretly reading or modifying your traffic.
- Data Theft: Stealing unencrypted passwords, session cookies, or financial info.
- Denial of Service (DoS): Dropping all intercepted packets to cut off a device's internet access. [1, 7, 10, 11, 12]

## Detection and Prevention

- Check for Duplicates: Running the command arp -a in a terminal can reveal if multiple IP addresses are mapped to the same
  MAC address, a common sign of spoofing.
- Static ARP Entries: For critical devices like routers, you can manually set a permanent IP-to-MAC mapping that cannot be
  changed by dynamic updates.
- Dynamic ARP Inspection (DAI): Many modern network switches can automatically validate ARP packets and block suspicious
  ones.
- Use Encryption: Using a [VPN](https://en.wikipedia.org/wiki/Virtual_private_network) or HTTPS ensures that even if a hacker
  intercepts your data, they cannot read its contents.
- IPv6: Switching to IPv6 largely eliminates this threat, as it uses the more secure Neighbor Discovery Protocol (NDP)
  instead of ARP. [4, 7, 8, 10, 13, 14, 15]
