Using Wireshark on Ubuntu involves three main steps: capturing live traffic, filtering the noise, and analyzing specific data
packets.

## 1. Capture Traffic

When you open [Wireshark](https://www.wireshark.org/), the home screen displays a list of your network interfaces (like eth0
for wired or wlan0 for Wi-Fi) with activity sparklines next to them.

- Select Interface: Double-click the interface with the most activity to start a live capture.
- Stop Capture: Click the Red Square icon in the top-left toolbar once you have gathered enough data.

## 2. Filter the Results

By default, Wireshark captures everything, which is often overwhelming. Use the Display Filter bar at the top to isolate
specific traffic.

- By Protocol: Type http, dns, icmp, or tcp and press Enter.
- By IP Address: Type ip.addr == 192.168.1.1 to see traffic involving a specific machine.
- By Port: Type tcp.port == 443 for HTTPS traffic.
- Logic: You can combine filters, such as http and ip.addr == 192.168.1.1.

## 3. Analyze the Interface

The main window is divided into three panes that help you dig into the data:

1.  Packet List (Top): A summary of every packet (No., Time, Source, Destination, Protocol, Length, Info).
2.  Packet Details (Middle): A "drill-down" view where you can expand layers (Ethernet, IP, TCP) to see specific headers and
    flags.
3.  Packet Bytes (Bottom): The raw data of the packet in hexadecimal and ASCII text.

## Pro Tip: Follow the Stream

If you find a packet from a website or chat, right-click it and select Follow > TCP Stream. Wireshark will open a new window
showing the entire conversation in plain text (if unencrypted), making it easy to read what was actually sent.
