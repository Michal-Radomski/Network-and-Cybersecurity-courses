Sniffing (also known as packet sniffing) is a technique used to capture and monitor data as it travels across a network. It
works like a digital eavesdropper, allowing an observer to see the content of data "packets". [1, 2, 3, 4]

## How it Works Internally

Normally, your network card only pays attention to data addressed specifically to your computer and ignores everything else.
[5, 6]

1.  Promiscuous Mode: A sniffer reconfigures your Network Interface Card (NIC) into "promiscuous mode". In this state, the
    card becomes a "busybody" and listens to all traffic passing through that part of the network, regardless of its
    destination.
2.  Packet Capture: The tool grabs raw data packets in real-time as they zip by.
3.  Decoding: Since raw packets are just binary data, the sniffer uses a decoder to translate them into a human-readable
    format, showing you details like source/destination IP addresses and actual message contents. [2, 3, 6, 7, 8]

## Passive vs. Active Sniffing

- Passive Sniffing: The tool simply "listens" to traffic without interacting with it. This is very hard to detect but only
  works on older hardware (hubs) or specifically configured switches.
- Active Sniffing: Used on modern "switched" networks where traffic is isolated to specific ports. Attackers use techniques
  like ARP Spoofing to trick the network into sending everyone else's traffic to their machine instead of the real recipient.
  [1, 2, 3, 9, 10, 11]

## Detection on Ubuntu

- chkrootkit sniffer: This specific test scans for hidden processes using "raw sockets" (the low-level tools needed for
  sniffing).
- Common False Positives: You may see a warning about dhclient or systemd-networkd being a packet sniffer. This is usually
  normal because these system tools must listen to raw network traffic to manage your IP address. [12, 13, 14, 15, 16]

## Why it's Dangerous

If a network is unencrypted (using http instead of https), a sniffer can see everything in plain text, including usernames,
passwords, and personal emails. [6, 10, 17] Would you like to see how to use tcpdump—the built-in Ubuntu tool—to safely
"sniff" your own network traffic for troubleshooting?
