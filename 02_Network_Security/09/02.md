To monitor and analyze your home network on Linux, you’ll typically combine command-line tools for quick checks with more
robust software for long-term data.

Since you are at home, you’ll likely be looking at your own traffic or using a **Network Tap** or **Managed Switch** to see
traffic from other devices (like smart TVs or phones).

---

## 1. Real-Time Monitoring (The Command Line)

These tools are great for a "quick look" to see what is eating your bandwidth right now.

### **Nload** (Bandwidth Usage)

Displays incoming and outgoing traffic using easy-to-read ASCII graphs.

- **Install:** `sudo apt install nload`
- **Run:** `nload`

### **Iptraf-ng** (Protocol Breakdown)

Provides a more detailed look at which protocols (TCP, UDP, ICMP) are active and which ports are being used.

- **Install:** `sudo apt install iptraf-ng`
- **Run:** `sudo iptraf-ng`

### **Bmon** (Interface Statistics)

A powerful, list-based monitor that shows traffic across all your network interfaces (Wi-Fi, Ethernet, Virtual Bridges)
simultaneously.

- **Install:** `sudo apt install bmon`
- **Run:** `bmon`

---

## 2. Deep Traffic Analysis (Packet Inspection)

When you need to see exactly what data is being sent (e.g., "Is my smart fridge sending data to a server in another
country?"), you need these tools.

### **Wireshark** (The Gold Standard)

Wireshark is the most popular GUI-based packet analyzer. It allows you to capture traffic and filter it by IP, protocol, or
keyword.

- **Install:** `sudo apt install wireshark`
- **Pro-Tip:** During installation, allow non-superusers to capture packets so you don't have to run the GUI as `root`.

### **Tcpdump** (The Terminal Alternative)

If you are working on a headless Linux server (like a Raspberry Pi), `tcpdump` is your best friend. It captures packets to a
`.pcap` file that you can move to your main computer to open in Wireshark later.

- **Example command:** `sudo tcpdump -i eth0 -w network_dump.pcap`
  - `-i eth0`: Listen on the Ethernet interface.
  - `-w`: Write the output to a file.

## 3. Advanced Home Analysis (The "Home Lab" Setup)

If you want a professional-grade dashboard for your home network, look into **Zeek** or **Suricata**.

### **Zeek (formerly Bro)**

Zeek isn't just a packet sniffer; it’s a network security monitor. It turns raw traffic into compact "connection logs" that
tell a story.

- **What it shows:** Every DNS query made, every HTTP connection requested, and every SSL certificate exchanged on your
  network.

### **Darkstat**

If you want a persistent, lightweight web interface to see which devices on your home network use the most data over time,
Darkstat is excellent.

- **Install:** `sudo apt install darkstat`
- **How it works:** It runs as a background service and serves a simple website (usually on port 667) showing a breakdown of
  traffic by host IP.

## The "How-To" Strategy

To effectively monitor a **whole** home network from one Linux machine, you face a hurdle: **Switches don't broadcast all
traffic to every port.**

1.  **The "Man-in-the-Middle" (Bridge):** If you have two Ethernet ports on your Linux PC, you can bridge them and put the PC
    between your router and your main switch. All traffic must then pass _through_ your Linux machine to be analyzed.
2.  **Port Mirroring:** If you have a "Managed Switch," configure a **SPAN port** to mirror all traffic to the port where
    your Linux machine is plugged in.
3.  **Raspberry Pi Gateway:** Many enthusiasts use a Raspberry Pi as their home router (using OpenWrt or Ubuntu Server).
    Since it handles all the routing, it can log every single packet that enters or leaves the house.
