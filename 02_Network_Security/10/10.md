When you "ping" a server, you are actually using the **ICMP (Internet Control Message Protocol)**. Specifically, you are
sending an **ICMP Echo Request** and hoping to receive an **ICMP Echo Reply**.

Since ICMP sits directly on top of the IP layer (it doesn't use TCP or UDP), the packet is very lean.

---

### The Anatomy of a Ping Packet

A standard ping packet consists of the **IP Header** followed by the **ICMP Message**.

#### 1. The IP Header (20 Bytes)

This is the "envelope" that gets the packet to the right house. It contains:

- **Source IP:** Your address.
- **Destination IP:** The address you are pinging.
- **Protocol:** Set to `1` (which tells the computer this is an ICMP packet).

#### 2. The ICMP Header (8 Bytes)

This is the actual "letter" inside. Here is how those 8 bytes are broken down:

| Field               | Size    | Description                                      |
| :------------------ | :------ | :----------------------------------------------- |
| **Type**            | 8 bits  | `8` for Request, `0` for Reply.                  |
| **Code**            | 8 bits  | Always `0` for pings.                            |
| **Checksum**        | 16 bits | Ensures the ICMP header wasn't corrupted.        |
| **Identifier**      | 16 bits | A unique ID to help match requests with replies. |
| **Sequence Number** | 16 bits | Starts at 1 and increases with each ping sent.   |

#### 3. The Payload (Optional Data)

After the header, there is usually some "filler" data to give the packet a standard size (often **32 bytes** on Windows or
**56 bytes** on Linux/macOS).

- **Fun Fact:** This data is often just the alphabet (`abcd...`) or a timestamp used to calculate the round-trip time
  precisely.

---

### Total Visual Structure

If you were to look at this in a packet sniffer like Wireshark, the stack looks like this:

> **[ Ethernet Header ]** — _Hardware addresses (MAC)_
>
> **[ IP Header ]** — _Source/Dest IP addresses_
>
> **[ ICMP Header ]** — _Type 8 (Request), ID, Seq Number_
>
> **[ Data Payload ]** — _Alphabet or timestamp_

---

### How the "Handshake" works (The Echo)

Ping doesn't have a formal handshake like TCP, but it follows a strict **Request/Response** cycle:

1.  **Request:** You send an ICMP Type 8 (Echo Request) with `Sequence: 1`.
2.  **Wait:** Your computer starts a stopwatch.
3.  **Reply:** The target receives it and immediately flips the Type to 0 (Echo Reply) and sends it back with the exact same
    `Identifier` and `Sequence: 1`.
4.  **Result:** Your computer stops the watch, sees the sequence numbers match, and reports: _"Reply from 8.8.8.8: bytes=32
    time=12ms"_

If the reply never comes back, you get the infamous **"Request timed out."**
