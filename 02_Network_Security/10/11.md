If Nmap is a Swiss Army knife, the **Nmap Scripting Engine (NSE)** is the secret compartment that turns it into a full-blown
power tool.

The NSE allows users to write and share simple scripts (using a programming language called **Lua**) to automate a wide
variety of networking tasks. Instead of just seeing if a port is open, the NSE lets you "interrogate" that port to see what’s
really going on inside.

---

### How it Works

The NSE runs scripts alongside a standard Nmap scan. These scripts are generally categorized into **Libraries** (to handle
things like SSL or SMB protocols) and **Scripts** (the actual tasks you want to perform).

#### The 4 Stages of a Script:

1.  **Prerule:** Runs before any scanning (e.g., generating new targets).
2.  **Hostrule:** Runs after Nmap discovers a host.
3.  **Portrule:** Runs after Nmap identifies an open port (the most common stage).
4.  **Postrule:** Runs after all scanning is finished (e.g., to summarize the data).

---

### Script Categories

Nmap comes with hundreds of built-in scripts. They are organized into categories so you don't accidentally do something you
didn't intend:

| Category          | Description                                                             |
| :---------------- | :---------------------------------------------------------------------- |
| **Discovery**     | Maps out the network, finds hidden shares, and lists active services.   |
| **Safe**          | Scripts that are unlikely to crash a service or use too much bandwidth. |
| **Default**       | The "standard" set of scripts run when you use the `-sC` flag.          |
| **Vulnerability** | Checks if a service is vulnerable to a specific known bug (CVE).        |
| **Exploit**       | Attempts to actually prove a vulnerability exists (use with caution!).  |
| **Brute**         | Attempts to guess passwords for services like SSH, FTP, or SQL.         |

---

### What Can You Actually Do With It?

The NSE moves Nmap from "Basic Mapping" to "Advanced Intelligence":

- **HTTP Header Grabbing:** Don't just find a web server; see what software version it’s running and if it has a `robots.txt`
  file.
- **SSL/TLS Auditing:** Automatically check if a server is using old, insecure versions of SSL or if its certificate is about
  to expire.
- **Database Discovery:** Find out the name of an instance or see if it allows "anonymous" logins.
- **Network Inventory:** Pull the exact OS version and hardware details from a printer or a router.

---

### Basic Syntax

To run a script, you use the `--script` flag.

- **Run a single script:** `nmap --script http-title <target>` (This grabs the title of the webpage).
- **Run a whole category:** `nmap --script "discovery" <target>` (Runs every script in the discovery category).
- **The "Default" scan:** `nmap -sC <target>` (The most common way to use NSE; runs the standard safe scripts).

---

### A Word of Advice

Because NSE scripts can be very active—meaning they send specific, sometimes complex data to a server—they are much noisier
than a simple "ping." If you're running vulnerability or brute-force scripts on a network you don't own, it will be very
obvious to anyone looking at the security logs.
