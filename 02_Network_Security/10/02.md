A TCP (Transmission Control Protocol) header is a 20-to-60-byte control structure added to the beginning of every TCP segment
to ensure reliable, ordered, and error-checked data delivery. It functions like a "passport" for data, containing vital
instructions for the network to manage the connection. [1, 2, 3]

## Header Structure

The standard header is 20 bytes long, divided into several fixed fields aligned in 32-bit words. [2, 4]

- Source Port (16 bits): Identifies the sending application's port.
- Destination Port (16 bits): Identifies the receiving application's port (e.g., port 80 for HTTP).
- Sequence Number (32 bits): Numbers each byte sent so the receiver can reassemble them in the correct order.
- Acknowledgment Number (32 bits): If the ACK flag is set, this indicates the next byte the receiver expects to get.
- Data Offset (4 bits): Also known as Header Length, it specifies the size of the header in 32-bit words.
- Reserved (3 or 6 bits): Unused bits set to zero for future standards.
- Control Flags (9 bits): Individual bits used to manage the connection state.
- Window Size (16 bits): Used for flow control to tell the sender how many bytes the receiver can currently accept without
  being overwhelmed.
- Checksum (16 bits): Used for error detection to verify the integrity of both the header and the data payload.
- Urgent Pointer (16 bits): Points to the end of urgent data if the URG flag is set.
- Options (0-40 bytes): Optional fields used for advanced features like Maximum Segment Size (MSS) or Timestamps. [2, 3, 4,
  5, 6, 7, 8, 9]

## Key Control Flags

Flags are 1-bit "switches" that trigger specific TCP behaviors: [10, 11, 12, 13, 14]

- SYN (Synchronize): Initiates a connection and synchronizes sequence numbers.
- ACK (Acknowledgment): Confirms that the data was received and that the acknowledgment number field is valid.
- FIN (Finish): Signals a graceful termination of the connection.
- RST (Reset): Abruptly terminates a connection due to an error.
- PSH (Push): Tells the receiver to pass the data to the application immediately rather than buffering it.
- URG (Urgent): Indicates that certain data should be processed with high priority. [8, 10, 15, 16, 17]
