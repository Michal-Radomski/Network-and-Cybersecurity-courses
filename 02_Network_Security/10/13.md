**Banner grabbing** is a technique used to gain information about a computer system on a network and the services running on
its open ports. Think of it like walking up to a shop and looking at the sign in the window to see what they sell and what
version of "business" they are running.

In technical terms, a "banner" is a text message sent by a service (like a web server or an email server) that often contains
its name, version number, and operating system.

---

### How Nmap Performs Banner Grabbing

Nmap has two primary ways to "grab" these banners: using its built-in service detection or using specific scripts from the
Nmap Scripting Engine (NSE).

#### 1. Service and Version Detection (`-sV`)

This is the most common method. When you use the `-sV` flag, Nmap doesn't just look for open ports; it sends specific
"probes" to the port to see how it responds.

- **The Process:** Nmap sends a packet, waits for the service to respond, and compares that response against a database of
  thousands of known service signatures.
- **The Result:** Instead of just saying "Port 80 is open," it will tell you "Port 80 is running **Apache httpd 2.4.41
  (Ubuntu)**."

#### 2. The `banner.nse` Script

If you want to see the "raw" banner exactly as the service sends it without Nmap trying to interpret it, you can use a
specific script: `nmap --script banner <target>` This script simply connects to the port and prints out whatever string of
text the service sends back immediately upon connection.

---

### Common Banner Examples

Different services reveal different amounts of information:

- **SSH (Port 22):** Usually very chatty. It might return `SSH-2.0-OpenSSH_8.2p1 Ubuntu-4ubuntu0.5`.
- **FTP (Port 21):** Often identifies the software, such as `220 (vsFTPd 3.0.3)`.
- **HTTP (Port 80/443):** Requires a "GET" request, but will return a `Server:` header like `Server: nginx/1.18.0`.

---

### Why Banner Grabbing is Useful

For someone curious about their network or a security professional, banner grabbing serves several purposes:

1.  **Inventory Management:** Knowing exactly which versions of software are running across a large network.
2.  **Vulnerability Research:** If you see a service is running _Software X version 1.0_, and you know version 1.0 has a
    security bug, you know that system needs an update.
3.  **OS Fingerprinting:** Sometimes the banner includes the name of the operating system (like "Debian" or "CentOS"),
    helping you map out the network.

### How to Defend Against It

If you are a system administrator, you often want to "suppress" or "mask" these banners so you aren't giving away too much
information to potential attackers.

- **Configuration:** Most software (like Apache or Nginx) has a setting to turn off version headers (e.g.,
  `ServerTokens Prod`).
- **Obfuscation:** Some people even change their banners to say something fake, like an old Linux server pretending to be a
  Windows machine, just to confuse automated scanners.
