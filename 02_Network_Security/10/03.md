The TCP 3-way handshake is the process used to establish a reliable, full-duplex connection between a client and a server
before any actual data is exchanged. It ensures both sides are ready and have synchronized their Initial Sequence Numbers
(ISNs). [1, 2, 3, 4]

## The 3-Way Handshake Steps

1.  SYN (Synchronize): The client sends a packet with the SYN flag set to the server. It includes a random Sequence Number
    (e.g., $X$) to start the conversation.
2.  SYN-ACK (Synchronize-Acknowledge): The server responds with a packet where both SYN and ACK flags are set. It sends its
    own random Sequence Number ($Y$) and an Acknowledgment Number ($X + 1$) to confirm it received the client's request.
3.  ACK (Acknowledge): The client sends a final ACK packet back to the server. It increments the server's sequence number to
    $Y + 1$ as its Acknowledgment Number. The connection is now established. [2, 4, 5, 6, 7, 8]

## TCP Header 32-Bit Word Structure

Computers process data in 32-bit "words" for efficiency. The TCP header is organized into five mandatory rows (20 bytes) of
32 bits each. [9, 10, 11]

| Word (32 bits) [9, 12, 13, 14, 15, 16, 17] | 0–15 bits                                                                                       | 16–31 bits                                                 |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| Row 1                                      | Source Port: Sending app's port.                                                                | Destination Port: Receiving app's port.                    |
| Row 2                                      | Sequence Number: Tracks byte order (32 bits).                                                   |                                                            |
| Row 3                                      | Acknowledgment Number: Next expected byte (32 bits).                                            |                                                            |
| Row 4                                      | Data Offset / Reserved / Flags: Header length, future bits, and control flags (SYN, ACK, etc.). | Window Size: Flow control (how much data can be accepted). |
| Row 5                                      | Checksum: Integrity check.                                                                      | Urgent Pointer: Priority data location.                    |
| Row 6+                                     | Options: Variable length (up to 40 bytes).                                                      |                                                            |
