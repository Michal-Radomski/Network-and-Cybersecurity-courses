On Ubuntu, you can perform a ping sweep using professional network tools or built-in terminal commands. Below are the most
common methods starting with the industry standard. [1, 2]

## 1. Using Nmap (Recommended) [3]

[Nmap](https://nmap.org/) is the most efficient tool for this. It is specifically designed to handle large ranges and can
even find hosts that block standard pings by using alternative discovery methods. [3, 4]

- Install: sudo apt install nmap
- Run Sweep: nmap -sn 192.168.1.0/24
- The -sn flag (formerly -sP) performs "host discovery" only, skipping the slower port scanning phase. [3, 5, 6, 7]

## 2. Using fping (High Performance) [8]

[fping](https://fping.org/) is built for speed and scripting. Unlike the standard ping command, it doesn't wait for one host
to respond before moving to the next; it sends requests in parallel. [9, 10]

- Install: sudo apt install fping
- Run Sweep: fping -a -g 192.168.1.0/24 2>/dev/null
- -a: Show only "alive" hosts.
  - -g: Generate a range from the provided subnet.
  - 2>/dev/null: Hides "unreachable" errors for a cleaner list. [8, 10, 11]

## 3. Using Built-in Bash Script (No Installation)

If you cannot install new tools, you can use a simple "for loop" in the terminal to iterate through a range of IP addresses.
[12, 13]

for i in {1..254}; do (ping -c 1 192.168.1.$i | grep "bytes from" &); done

- How it works: This command pings every address from .1 to .254 once. The & symbol sends each request to the background so
  they run simultaneously, significantly speeding up the process. [12, 14]

## 4. Using arp-scan (Local Network Only) [15]

For local networks, arp-scan is often faster and more accurate because it uses ARP packets, which are rarely blocked by local
firewalls. [15, 16]

- Install: sudo apt install arp-scan
- Run Sweep: sudo arp-scan --localnet [15, 16]
