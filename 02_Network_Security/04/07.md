To "crack" a hash back into a plain-text password, you typically use a tool that guesses potential passwords, hashes them,
and compares the result to your target hash. Since hashing is a one-way process, you cannot simply "reverse" it; you must
find a match. [1, 2, 3, 4]

## Common Cracking Methods

- Dictionary Attack: The tool tests a list of common words and previously leaked passwords (called a "wordlist").
- Brute-Force: The tool systematically tries every possible combination of characters (e.g., aaaa, aaab, aaac...). This is
  very slow for long passwords.
- Rainbow Tables: These are massive precomputed tables of hashes and their corresponding plain-text passwords, used to speed
  up the process. [5, 6, 7]

## Standard Tools & Workflow## 1. Linux: John the Ripper (JtR) [8]

For Linux hashes, you must first combine /etc/passwd and /etc/shadow into a format the cracker understands. [9, 10]

1.  Prepare the file: Run unshadow /etc/passwd /etc/shadow > combined_hashes.txt.
2.  Crack it: Use a wordlist like the famous rockyou.txt by running: john --wordlist=rockyou.txt combined_hashes.txt
3.  View results: Run john --show combined_hashes.txt. [9, 11, 12, 13, 14]

## 2. Windows: Hashcat

[Hashcat](https://hashcat.net/hashcat/) is often preferred for Windows NTLM hashes because it can use your computer's GPU to
guess millions of times faster than a CPU. [15, 16]

1.  Identify Hash Type: NTLM hashes (from the SAM file) use "mode 1000" in Hashcat.
2.  Run the command: hashcat -m 1000 -a 0 ntlm_hashes.txt wordlist.txt [16, 17]

## Ethical & Legal Warning

Password cracking should only be performed on systems you own or have explicit, written permission to test. Unauthorized
access is a criminal offense in most jurisdictions. To protect your own hashes from these tools, use long passphrases (20+
characters) and Multi-Factor Authentication (MFA). [4, 7, 18, 19, 20]
