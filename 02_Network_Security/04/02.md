In cybersecurity, IAAA stands for Identification, Authentication, Authorization, and Accountability. It is the standard
framework for managing identity and access control within a system. Think of it as a four-step security gate that every user
must pass through to access a network.

## 1. Identification

This is the process of a user claiming an identity. It is the "Who are you?" phase.

- Example: Typing your username into a login field or swiping an ID card.
- Key Requirement: Usernames must be unique so the system knows exactly which individual is making the request.

## 2. Authentication

This is the "Prove it" phase. The system verifies that you are actually who you claim to be.

- Common Factors:
- Something you know: Password or PIN.
  - Something you have: A physical token or a code sent to your phone (MFA).
  - Something you are: Fingerprint or facial recognition (biometrics).

## 3. Authorization

Once your identity is proven, the system determines what you are allowed to do. This is the "What can you access?" phase.

- Example: An intern might only have "Read-Only" access to files, while a manager has "Read/Write" access.
- Principle: Usually follows the Principle of Least Privilege, meaning you only get the minimum access required to do your
  job.

## 4. Accountability

This is the "What did you do?" phase. It involves logging and auditing all user actions.

- Mechanism: System logs, audit trails, and timestamps.
- Purpose: If a security breach occurs, accountability ensures that the action can be traced back to a specific identified
  user, preventing them from denying their actions (non-repudiation).
