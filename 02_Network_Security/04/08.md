Steganography is the practice of hiding a secret message inside a non-secret file—like a photo or song—so that nobody even
realizes a message exists. While cryptography scrambles data to make it unreadable, steganography hides the data "in plain
sight" to avoid drawing any attention to it. [1, 2, 3]

## What is OpenStego?

[OpenStego](https://www.openstego.com/) is a free, open-source tool used to perform steganography on your computer. It is
written in Java, meaning it works on Windows, macOS, and Linux. [4, 5, 6, 7] It offers two primary features: [4]

- Data Hiding: It takes a "secret" file (the message) and embeds it inside a "cover" file (usually an image like a JPG or
  PNG).
- Watermarking: It can add an invisible signature to your images to help you track unauthorized copying or prove you are the
  owner. [4, 8, 9, 10]

## How it Works

1.  Select Files: You choose the secret message (e.g., a text file) and the cover image you want to hide it in.
2.  Encrypt (Optional): You can add a password to the secret file using encryption (like AES-256) so even if someone suspects
    something is hidden, they can't read it without the key.
3.  Embed: OpenStego creates a new image file that looks identical to the original but contains the hidden data buried within
    its pixels.
4.  Extract: The intended recipient uses OpenStego to "extract" the secret file from the image back into its original form.
    [7, 8, 9, 11, 12, 13]

Least Significant Bit (LSB) is the most common technique in digital steganography for hiding data inside images. It works by
replacing the "last" bit of a pixel's color value with a bit of your secret message. [1, 2, 3, 4]

## How the Math Works

Digital images are made of pixels, and each pixel's color (Red, Green, or Blue) is represented by a byte (8 bits) with a
value between 0 and 255. [3, 5]

- The "Significant" Bits: The bits on the left (the "Most Significant Bits") have the highest mathematical weight (e.g., 128,
  64). Changing these would drastically change the color.
- The "Least Significant" Bit: The bit on the far right has a weight of only 1. Changing this bit only changes the color
  intensity by 1/255th. [5, 6, 7, 8, 9]

## The Embedding Process

To hide a message like the letter "A" (binary 01000001), a tool like [OpenStego](https://www.openstego.com/) would:

1.  Break the message into bits: 0, 1, 0, 0, 0, 0, 0, 1.
2.  Pick 8 pixels in the cover image.
3.  Swap the LSBs: It replaces the last bit of each pixel's color value with one bit from the secret message.
4.  Result: The image looks exactly the same to the human eye because the change is too subtle to detect. [5, 6, 8, 10, 11,
    12]

## Key Strengths and Weaknesses

- Capacity: You can hide a lot of data. Since every pixel has 3 color channels (RGB), you can hide 3 bits per pixel.
- Fragility: LSB is easily destroyed. If you convert the image to a JPEG or resize it, the subtle LSB changes are often lost
  because of lossy compression.
- Detection: While invisible to the eye, security experts use steganalysis tools (like StegExpose) to look for statistical
  anomalies in the LSBs to prove a message is hidden. [5, 12, 13, 14, 15]
