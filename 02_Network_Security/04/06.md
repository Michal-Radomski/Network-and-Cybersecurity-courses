In simple terms, SAM is where Windows hides your passwords, and pwdump is a tool used to dig them out. [1, 2, 3]

## What is a SAM File?

The Security Account Manager (SAM) is a critical database file in Windows that stores the credentials for local user
accounts. [4, 5]

- What's inside: It contains usernames and passwords, though the passwords aren't stored in plain text; they are stored as
  cryptographic "hashes" (specifically LM or NTLM hashes) to keep them secure.
- Where it lives: You can find it at C:\Windows\System32\config\SAM.
- Security: Windows locks this file while the computer is running so that regular users (or even administrators) can't just
  copy it. It requires special system-level privileges to access. [1, 3, 5, 6, 7]

## What is pwdump?

pwdump is the general name for a family of Windows programs designed to extract (or "dump") those password hashes from the
SAM database. [6, 8]

- How it works: It bypasses the standard Windows locks to pull the hashes out of the SAM file or directly from the system's
  memory while Windows is running.
- Who uses it: It is a popular tool for both system administrators (to audit password strength) and hackers (to steal
  credentials).
- What happens next: Once the hashes are dumped into a text file, they can be fed into a "cracker" tool (like John the
  Ripper) to reveal the actual plain-text passwords. [2, 3, 6, 9, 10, 11]

In Linux, the architecture is different, but you can find direct functional equivalents for both the SAM file and the pwdump
tool. [1, 2]

## 1. The SAM Equivalent: /etc/shadow

While Windows uses the SAM database, Linux splits user information across two main files: [1, 3]

- /etc/passwd: This is the equivalent of the "username" part of the SAM file. It is public-readable and lists usernames, user
  IDs, and home directories.
- /etc/shadow: This is the equivalent of the "secret" part of the SAM file. It stores the actual password hashes and
  sensitive account aging information.
- Access: Like the SAM file, /etc/shadow is heavily protected. Only the root user (administrator) has permission to read it.
  [1, 4, 5]

## 2. The pwdump Equivalent: unshadow

In the Linux world, there isn't one single "pwdump" tool because the files are simpler to access if you have root privileges.
However, the most direct equivalent is a utility called unshadow (part of the John the Ripper suite).

- How it works: unshadow combines the information from /etc/passwd and /etc/shadow into a single file format.
- Why use it: This "combined" file is exactly what password-cracking tools need to start trying to guess the passwords.
- Other Tools:
- Mimikatz-like: For advanced memory dumping (similar to how some versions of pwdump work), Linux users might use Mimipy or
  Linikatz.
  - Cross-Platform: If you are on a Linux machine trying to dump a Windows SAM file from a mounted drive, tools like
    [samdump2](https://www.kali.org/tools/samdump2/) or secretsdump.py are the go-to choices. [2, 6, 7, 8, 9]

| Feature [1, 7, 10] | Windows           | Linux Equivalent |
| ------------------ | ----------------- | ---------------- |
| Storage File       | C:\...\config\SAM | /etc/shadow      |
| Dump Tool          | pwdump            | unshadow         |
| Common Cracker     | Hashcat           | John the Ripper  |
