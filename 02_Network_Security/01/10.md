These protocols form the "engine" of the Internet Layer (Layer 3) of the networking stack, handling how data is addressed,
diagnosed, and secured. [1, 2, 3, 4]

## IPv4 vs. IPv6 (Addressing & Routing)

Internet Protocol (IP) is the primary routing protocol used to identify devices and move data across the web. [5, 6]

- IPv4 (Internet Protocol version 4): The traditional addressing system used since the early 1980s.
- Format: 32-bit addresses, usually written in "dotted decimal" (e.g., 192.168.1.1).
  - Capacity: Supports ~4.3 billion unique addresses, which are now largely exhausted.
  - Features: Often requires NAT (Network Address Translation) to share a single public IP among multiple devices.
- IPv6 (Internet Protocol version 6): The modern successor designed to solve the address shortage.
- Format: 128-bit addresses, written in hexadecimal (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334).
  - Capacity: Provides ~340 undecillion addresses, effectively limitless for the foreseeable future.
  - Features: Includes built-in security (IPsec), simpler headers for faster routing, and SLAAC (Stateless Address
    Autoconfiguration) so devices can set their own IPs without a DHCP server. [7, 8, 9, 10, 11, 12]

## ICMP (Internet Control Message Protocol) [13]

ICMP is a supporting protocol used by network devices to send error messages and operational information. [14, 15]

- Purpose: It doesn't move user data but acts as a diagnostic tool to report if a destination is unreachable or if a packet
  timed out.
- Common Tools:
- Ping: Uses ICMP "Echo Request" and "Echo Reply" messages to test if a device is online and measure latency.
  - Traceroute: Uses ICMP to map the physical path packets take across routers to reach a destination. [4, 14, 15, 16, 17]

## IGMP (Internet Group Management Protocol) [18]

IGMP is used specifically for multicasting on IPv4 networks. [18, 19]

- Purpose: It allows devices to join or leave a "multicast group." This is essential for applications where one sender
  transmits data to many specific receivers simultaneously without flooding the entire network.
- Use Cases: Live video streaming (IPTV), online gaming, and web conferencing.
- Note: In IPv6, this function is handled by MLD (Multicast Listener Discovery) instead of IGMP. [18, 20, 21, 22, 23]

## IPsec (Internet Protocol Security)

IPsec is a suite of protocols that adds encryption and authentication directly to the IP layer. [1, 5]

- Purpose: Since standard IP is unencrypted (like writing a message on the outside of an envelope), IPsec provides the
  "envelope" to keep data private.
- Components:
- AH (Authentication Header): Verifies the sender and ensures data hasn't been tampered with.
  - ESP (Encapsulating Security Payload): Provides the actual encryption to hide the data from onlookers.
- Common Use: It is the foundation for most Site-to-Site VPNs, allowing branch offices to connect securely over the public
  internet. [5, 24, 25, 26]
