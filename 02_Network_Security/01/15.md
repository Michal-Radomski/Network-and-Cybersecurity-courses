The difference in names is due to the separate origins and naming conventions of the operating systems they were developed
for.

- ifconfig (Linux/Unix): Stands for "Interface Configuration". It was originally developed for BSD Unix in 1983, roughly nine
  years before Linux existed. It focuses on the status and control of the network interfaces (like your Ethernet or Wi-Fi
  card) and is not strictly limited to Internet Protocol (IP).
- ipconfig (Windows): Stands for "Internet Protocol Configuration". It was introduced by Microsoft starting with Windows NT
  as a utility specifically designed to display and manage TCP/IP settings. [1, 2, 3, 4, 5, 6]

## Key Differences at a Glance

| Feature [1, 4, 6, 7, 8, 9, 10] | ifconfig                                                | ipconfig                                |
| ------------------------------ | ------------------------------------------------------- | --------------------------------------- |
| Origin                         | 1983 (4.2BSD Unix)                                      | 1993 (Windows NT)                       |
| Primary Meaning                | Interface Configuration                                 | Internet Protocol Configuration         |
| System Focus                   | Managing the hardware interface                         | Managing the IP software settings       |
| Current Status                 | Deprecated in modern Linux; replaced by the ip command. | Still the standard command for Windows. |

## The Modern Linux Alternative

While many still use ifconfig, it is officially deprecated on most modern Linux distributions. The new standard is the
[ip command](https://www.redhat.com/en/blog/ifconfig-vs-ip) (part of the iproute2 package), which is more powerful and
provides consistent syntax for advanced networking. [9, 11, 12, 13]

- To see your IP on modern Linux, you would now use: ip addr (or just ip a). [6, 14]
