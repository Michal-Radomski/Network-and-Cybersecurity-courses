A subnet mask is a 32-bit number used in IPv4 networking to distinguish between the network and host portions of an IP
address. It effectively acts as a "filter" that tells your device which part of its IP refers to the specific subnetwork it
belongs to and which part identifies the unique device on that network. [1, 2, 3, 4, 5]

## Core Concept: The "Slicing" Effect

Think of an IP address like a house address. The subnet mask tells the computer where to "slice" the string of numbers to
separate the "Street Name" (Network ID) from the "House Number" (Host ID). [6, 7]

- Network Portion: Identified by a continuous string of binary 1s.
- Host Portion: Identified by a continuous string of binary 0s. [8, 9, 10, 11]

## Why it Matters

- Direct vs. Routed Traffic: A device uses its mask to determine if a destination IP is local. If the network portions match,
  it sends the data directly; if not, it sends the data to the default gateway (router) to be sent elsewhere.
- Organization: It allows large networks to be broken into smaller, more manageable subnets, which improves security and
  reduces unnecessary "broadcast" traffic. [1, 2, 3, 12, 13, 14]

## Common Examples

| Decimal Notation [2, 7, 15, 16, 17] | CIDR Notation (Prefix) | Number of Useable Hosts | Use Case                            |
| ----------------------------------- | ---------------------- | ----------------------- | ----------------------------------- |
| 255.255.255.0                       | /24                    | 254                     | Most common for home/small office.  |
| 255.255.0.0                         | /16                    | 65,534                  | Medium to large enterprises.        |
| 255.0.0.0                           | /8                     | ~16.7 million           | Massive global or carrier networks. |

## Visualizing the Binary Process

When a device processes an IP, it performs a Bitwise AND operation between the IP and the mask to isolate the Network ID. [1,
6, 17]
