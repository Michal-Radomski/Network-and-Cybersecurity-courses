**NetBIOS** (Network Basic Input/Output System) is an industry-standard networking protocol and API developed in the 1980s.
While it is considered "legacy" today, it played a massive role in how Windows computers talked to each other for decades.

Think of NetBIOS as a **digital directory service** for a local neighborhood. It allows computers to identify each other by
"friendly names" rather than just hard-to-remember IP addresses.

---

## What is NetBIOS For?

NetBIOS provides three primary services to a local area network (LAN):

1.  **Name Service (NetBIOS-NS):** This allows a computer to register a name (like `OFFICE-PC-01`) and let other computers
    know it exists. When you type a computer's name into a file explorer to find a shared folder, NetBIOS is often what
    resolves that name to an IP address.
2.  **Datagram Service:** Used for "connectionless" communication. This handles simple tasks like broadcasting information to
    all computers on the network at once.
3.  **Session Service:** This allows two computers to establish a stable "conversation" for things like file sharing and
    printer sharing.

---

## How it Works

NetBIOS operates on the principle of **Broadcasting**.

- **The Shout:** When a computer starts up, it "shouts" its name to the entire network: _"I am LAPTOP-B, does anyone else
  have this name?"_
- **The Claim:** If no one objects, it claims that name.
- **The Request:** When you want to find `LAPTOP-B`, your computer shouts: _"Who is LAPTOP-B?"_ and `LAPTOP-B` responds with
  its address.

### The Shift to TCP/IP

Originally, NetBIOS was its own networking protocol (NetBEUI). However, NetBIOS couldn't "route" data across the internet—it
only worked in small, local circles. To fix this, Microsoft created **NetBIOS over TCP/IP (NBT)**, which wraps the old
NetBIOS commands inside modern internet packets.

---

## Why is it Still Around? (and the Security Risks)

You’ll still find NetBIOS active on most Windows machines for **backward compatibility**. If you have an old printer or a
legacy server from 15 years ago, NetBIOS ensures they can still talk to modern Windows 11 machines.

### The Security Catch:

NetBIOS is a frequent target for attackers because:

- **Information Leakage:** It gives away computer names, usernames, and share names to anyone on the local network.
- **Spoofing:** Because it relies on "shouting" (broadcasting), an attacker can pretend to be the computer you’re looking
  for. This is a common tactic in **Man-in-the-Middle** attacks (often called LLMNR/NetBIOS Poisoning).

## Should You Use It?

In a modern home or office environment:

- **Disable it if possible:** Most modern networks use **DNS** (Domain Name System) or **mDNS** (Apple's Bonjour) for name
  resolution, which are faster and more secure.
- **Keep it on ONLY if:** You have very old hardware (Windows XP/2000 era) or specific legacy industrial software that
  requires it to function.

To see if NetBIOS is active on your Windows machine right now, you can open a command prompt and type: `nbtstat -n` (This
shows your local NetBIOS name table).

---

Technically, Linux **does not** use NetBIOS natively. Linux was built on standard internet protocols (TCP/IP), whereas
NetBIOS was a proprietary Microsoft creation.

However, Linux is a "chameleon" in networking. Because Linux servers often need to live in Windows-heavy environments,
developers created tools to make Linux **act** like it has NetBIOS.

---

## The Bridge: Samba

If you have ever shared a folder from Linux to a Windows PC, you likely used **Samba**.

Samba is the software suite that allows Linux to speak the **SMB/CIFS** language (the protocol Windows uses for file
sharing). Included in the Samba suite is a specific daemon called `nmbd`.

- **`nmbd` (NetBIOS Message Block Daemon):** This is the part of Samba that handles NetBIOS name service requests. It makes
  your Linux machine show up by its "friendly name" (e.g., `UBUNTU-SERVER`) in the "Network" section of a Windows File
  Explorer.

---

## How Linux Interacts with NetBIOS

On a modern Linux system, you generally interact with NetBIOS in three ways:

### 1. Name Resolution

When you type `ping windows-pc`, Linux usually checks its local `/etc/hosts` file or asks a DNS server. If those fail, and
you have the **Winbind** service installed, Linux can use NetBIOS to "shout" on the network to find the IP address of that
Windows machine.

### 2. Scanning (nbtscan)

Linux users often use a tool called `nbtscan` to find NetBIOS information on a network. This is common in network
administration and security auditing.

- **Command:** `sudo nbtscan 192.168.1.0/24`
- **Result:** It returns a list of IP addresses, NetBIOS names, and MAC addresses for every Windows-compatible device on the
  subnet.

### 3. Mounting Shares

When you mount a Windows share on Linux:

```bash
sudo mount -t cifs //WINDOWS-PC/ShareName /mnt/win -o guest
```

The `cifs-utils` package handles the connection. While modern versions of this command use **DNS** or **IP addresses** to
find the target, it can still fall back to NetBIOS protocols if the network is older.

---

## Is it necessary on Linux today?

**Mostly, no.**

Modern networking has moved away from NetBIOS in favor of:

- **DNS:** The standard for the entire internet.
- **mDNS (Avahi/Bonjour):** This is what Linux (and Mac) uses for "Zero-Config" networking. It’s why you can find
  `printer.local` or `raspberrypi.local` without any setup.

**The Verdict:** Linux only "uses" NetBIOS if you specifically install and configure **Samba** to maintain compatibility with
older Windows systems or specific legacy network environments. On a "vanilla" Linux install with no file-sharing software,
NetBIOS is completely absent.
