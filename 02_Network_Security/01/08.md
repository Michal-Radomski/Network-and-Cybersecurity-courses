Understanding network protocols and their ports is fundamental to how data moves across the internet. Here is a breakdown of
the protocols you requested, categorized by their primary function: [1, 2]

## Web & Security Protocols

- HTTP (Hypertext Transfer Protocol): The foundational protocol for the World Wide Web. It is used to transfer web pages from
  a server to your browser but lacks encryption, making it insecure.
- Port: 80.
- HTTPS (HTTP Secure): The secure version of HTTP that uses [SSL/TLS](https://www.sectigo.com/blog/what-is-port-443) to
  encrypt data, protecting sensitive information like login credentials.
- Port: 443.
- TLS/SSL (Transport Layer Security / Secure Sockets Layer): These are cryptographic protocols rather than application-layer
  services themselves. They provide the "S" in HTTPS and secure many other protocols (like IMAPS or FTPS). TLS is the modern,
  more secure successor to the now-deprecated SSL. [3, 4, 5, 6, 7, 8]

## File Transfer & Management

- FTP (File Transfer Protocol): A standard protocol for moving files between a client and a server. It is unencrypted and
  transmits data in plain text.
- Ports: 21 (Control) and 20 (Data).
- FTPS (FTP over SSL): An extension of FTP that adds support for TLS and SSL cryptographic protocols to secure file
  transfers.
- Ports: Typically 990 (Implicit) or 21 (Explicit via STARTTLS).
- SSH (Secure Shell): Provides a secure, encrypted channel for remote command-line access and
  [secure file transfers](https://www.facebook.com/AhmedNazmyOfficial/posts/essential-network-protocols-and-their-port-numbers-2025-editionevery-network-eng/717153240811614/)
  (via SFTP).
- Port: 22. [3, 7, 9, 10, 11, 12, 13, 14, 15, 16]

## Remote Access & Directory Services [17]

- Telnet: A legacy protocol for remote access to servers. Because it transmits all data (including passwords) in plain text,
  it has largely been replaced by SSH.
- Port: 23.
- LDAP (Lightweight Directory Access Protocol): Used to access and manage distributed directory information services, such as
  Microsoft Active Directory.
- Port: 389. [3, 7, 9, 11, 16, 17, 18, 19, 20]

## Email & Infrastructure

- IMAP (Internet Message Access Protocol): Used by email clients to retrieve messages from a server. Unlike POP3, IMAP
  synchronizes mail across multiple devices.
- Ports: 143 (Default/Unencrypted) or 993 (IMAPS/Encrypted).
- DHCP (Dynamic Host Configuration Protocol): Automatically
  [assigns IP addresses](https://blog.certcube.com/top-40-protocols-a-comprehensive-guide/) and other network configuration
  details to devices when they connect to a network.
- Ports: 67 (Server) and 68 (Client).
- NTP (Network Time Protocol): Synchronizes the clocks of computers and devices across a network to ensure consistent timing.
- Port: 123. [5, 7, 11, 16, 17, 18, 20, 21, 22, 23]
