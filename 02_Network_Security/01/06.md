While there is no single command named exactly like getmac, several standard Linux commands provide the same information. [1]

## Primary Commands

The most modern and standard way to view MAC addresses on Linux is using the ip suite:

- ip link: This is the most direct equivalent. It lists all network interfaces and their corresponding physical (MAC)
  addresses.
- Look for the value following link/ether.
- ip addr (or ip a): Displays IP addresses alongside the MAC address for each interface. [2, 3, 4, 5]

## Alternative & Older Commands

Depending on your distribution or preference, you might also use:

- ifconfig -a: A traditional tool that displays detailed interface information.
- The MAC address is typically listed next to HWaddr or ether.
  - Note: ifconfig is considered deprecated in favor of the ip command in many modern distributions.
- cat /sys/class/net/\*/address: A quick way to list only the MAC addresses for every interface on the system without extra
  network details.
- nmcli device show: If you use NetworkManager, this provides a detailed summary, with the MAC listed under GENERAL.HWADDR.
  [2, 5, 6, 7, 8, 9, 10, 11]

## Remote MAC Discovery

If you used getmac /s to find addresses of other computers on your local network:

- arp -a: Shows the ARP cache, mapping local IP addresses to MAC addresses.
- nmap -sn [subnet]: A more powerful tool that can scan your entire network to list the MAC and IP of every connected device.
  [9, 12, 13, 14]
