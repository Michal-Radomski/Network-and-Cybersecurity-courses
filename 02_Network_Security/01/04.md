A VPN (Virtual Private Network) works by creating a secure, encrypted "tunnel" between your device and a remote server.
Instead of your data traveling directly to its destination over the open internet, it is first scrambled and sent to the VPN
server, which then interacts with the web on your behalf. [1, 2, 3, 4]

## How a VPN Works Step-by-Step

1.  Authentication: When you tap "Connect" in your VPN app, your device and the VPN server verify each other's identities to
    ensure the connection is legitimate.
2.  Encryption: Your device and the server agree on a protocol (like OpenVPN, WireGuard, or IKEv2) and unique encryption
    keys.
3.  Tunneling: Your data is wrapped inside a "capsule" (another network packet). To anyone watching—like your ISP or a hacker
    on public Wi-Fi—your data looks like an unreadable jumble of code.
4.  IP Masking: The VPN server decrypts your request, replaces your real IP address with its own, and sends it to the
    destination website.
5.  Return Path: When the website responds, it sends the data to the VPN server, which re-encrypts it and sends it back
    through the secure tunnel to you. [1, 3, 4, 5, 6, 7]

## Relationship to the OSI Model

While a VPN is an application (Layer 7), it mostly operates at Layer 3 (Network Layer) by creating a virtual network adapter
that reroutes all your computer's IP traffic through the tunnel. Some protocols, like SSL VPNs, can also work at Layer 4
(Transport Layer) or Layer 7 depending on how they are implemented. [8, 9, 10, 11, 12]

## Common Benefits

- Privacy: Your
  [Internet Service Provider (ISP)](https://azure.microsoft.com/en-ca/resources/cloud-computing-dictionary/what-is-vpn) can
  see that you are using a VPN, but they cannot see which specific sites you visit or what you do there.
- Security: It protects your sensitive info (like bank logins) from "Man-in-the-Middle" attacks, especially on insecure
  public Wi-Fi.
- Location Spoofing: Since you use the VPN server's IP address, you can appear to be in a different country to access
  regional content or bypass censorship.
- Secure Remote Access: Companies use VPNs to allow employees to securely log into the office network from home. [1, 3, 6,
  13, 14, 15, 16]

OpenVPN is a highly flexible, open-source VPN protocol that has been a security standard since 2001. It is widely trusted for
its ability to bypass strict firewalls by masquerading as standard web traffic and its support for a variety of encryption
algorithms. [1, 2, 3]

## OpenVPN vs. WireGuard (Quick Comparison)

While OpenVPN is the veteran standard, many modern users are switching to WireGuard for daily use. [4]

| Feature [1, 2, 4, 5, 6, 7, 8, 9, 10] | OpenVPN                                  | WireGuard                                    |
| ------------------------------------ | ---------------------------------------- | -------------------------------------------- |
| Speed                                | Moderate; higher overhead.               | Blazing fast; 2–4x faster throughput.        |
| Complexity                           | Large codebase (steeper learning curve). | Tiny codebase (~4,000 lines); easy to audit. |
| Bypassing Firewalls                  | Excellent: Can run over TCP port 443.    | Poor: Restricted to UDP.                     |
| Best For                             | Restricted networks/Legacy systems.      | Speed, mobile devices, and gaming.           |

## How to use OpenVPN on Ubuntu

You can use OpenVPN on Ubuntu either through a graphical interface (GUI) or the terminal. [11, 12]

## Method 1: Using the GUI (Easiest for Desktop)

This method is ideal if you have a .ovpn configuration file from a VPN provider. [12]

1.  Install the plugin: Open your terminal and run: sudo apt install network-manager-openvpn-gnome.
2.  Open Settings: Go to Settings > Network.
3.  Import File: Click the + button in the VPN section and select "Import from file...".
4.  Add Profile: Select your .ovpn file and enter your username/password if prompted.
5.  Connect: Toggle the switch to On to start your secure tunnel. [12, 13, 14, 15]

## Method 2: Using the Terminal (Best for Servers)

1.  Install OpenVPN: sudo apt update && sudo apt install openvpn.
2.  Run the connection: Navigate to where your .ovpn file is located and run: sudo openvpn --config your-file.ovpn.
3.  Keep it running: Keep the terminal window open; closing it will disconnect the VPN. [13, 16, 17, 18]

## Setting Up Your Own OpenVPN Server

If you want to host your own VPN server on Ubuntu to access your home network remotely, the easiest way is to use a
community-vetted installation script. [19]

1.  Download the script: wget https://raw.githubusercontent.com/angristan/openvpn-install/master/openvpn-install.sh.
2.  Make it executable: chmod +x openvpn-install.sh.
3.  Run it: sudo ./openvpn-install.sh. Follow the on-screen prompts to set your IP and port. It will generate a .ovpn file
    for you to use on your client devices. [19, 20, 21]
