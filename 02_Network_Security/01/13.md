In the legacy system of Classful Addressing, the IPv4 address space was divided into five classes (A through E). While
Classes A, B, and C are used for standard unicast communication (sending data to one specific device), Classes D and E are
reserved for specialized purposes.

## Class D: Multicasting

Class D addresses range from 224.0.0.0 to 239.255.255.255.

- Purpose: These are used for Multicasting, which allows a single sender to transmit data to a group of interested receivers
  simultaneously.
- How it works: Instead of assigning these IPs to a specific computer, devices "subscribe" to a Class D address. When data is
  sent to that address, the network hardware ensures everyone in the group receives a copy.
- Real-World Examples:
- Routing Protocols: Routers use them to talk to each other (e.g., OSPF or RIPv2).
  - Streaming: IPTV and digital radio broadcasts.
  - Local Discovery: Apple’s AirPlay and "mDNS" use multicast to find devices on your Wi-Fi.

## Class E: Experimental/Reserved

Class E addresses range from 240.0.0.0 to 255.255.255.254.

- Purpose: These were officially set aside by the IETF (Internet Engineering Task Force) for Experimental, Research, and
  Development purposes.
- Current Status: They are strictly reserved and are not used on the public internet. In fact, most operating systems and
  routers are programmed to drop any traffic coming from or going to a Class E address.
- Fun Fact: Despite the global shortage of IPv4 addresses, Class E remains "locked away." There have been many debates about
  "unlocking" them for general use, but the compatibility issues with older hardware make this unlikely.
