Testing your home Wi-Fi security involves checking who is on your network, scanning for technical weaknesses, and auditing
your router's configuration.

## 1. Identify Connected Devices (Intruder Check)

The most common "security breach" is simply an unauthorized person using your bandwidth.

- Use a Network Scanner: Download a free app like [Fing](https://www.fing.com/news/best-network-scanner-apps/) (available for
  iOS, Android, and Desktop) to scan your network [1, 2] . It will list every phone, laptop, and smart device currently
  connected.
- Audit the List: Look for "Generic" or "Unknown" devices [3] . If you see something you don't recognize, it could be a rogue
  connection from a neighbor or a hacker.
- Router Admin Panel: Log into your router’s web interface (usually by typing 192.168.1.1 into your browser) and look for the
  DHCP Client List or Attached Devices [4, 5] . This is the most authoritative list of who is on your network.

## 2. Scan for Technical Vulnerabilities

You can use specialized tools to see if your router has known security holes or "open doors."

- F-Secure Router Checker: Use the web-based [F-Secure Router Checker](https://www.f-secure.com/en/router-checker) to see if
  your router’s DNS settings have been tampered with or if it has known exploits listed in public databases [6] .
- Port Scanning: Use [Angry IP Scanner](https://www.lifewire.com/free-wifi-network-apps-3984721) or Nmap to check for "Open
  Ports" [1] . An open port is like an unlocked window; if you don't need it for a specific service, it should be closed to
  prevent intrusions.
- Vulnerability Testing: For advanced users, tools like [Aircrack-ng](https://www.youtube.com/watch?v=2bhuZTUkPUA) can be
  used to test if your Wi-Fi password can be easily cracked, though this requires a technical background in Linux [7, 8, 9] .

## 3. The "Secure Settings" Audit

Run through this quick checklist within your router’s settings to ensure your defenses are active:

- Encryption Check: Ensure you are using WPA3 or WPA2 (AES). If you see "WEP" or "WPA (TKIP)", your network is highly
  vulnerable [10, 11] .
- Firmware Version: Check if a "Firmware Update" is available. Updates often patch critical security flaws that hackers use
  to take control of routers.
- Disable WPS: If your router has a "WPS" (Wi-Fi Protected Setup) button feature enabled in the software, turn it off. It is
  a major entry point for attackers [12] .
- Remote Management: Ensure "Remote Management" or "WAN Admin" is Disabled so that your router settings cannot be accessed
  from outside your home [13, 14] .

On Ubuntu, testing your wireless network security involves using the terminal to audit your connections, find your router’s
admin gateway, and scan for unauthorized devices.

## 1. Audit Your Wi-Fi Security Standard

Before logging into your router, check if your current connection is using a strong security protocol (like WPA3 or WPA2).

- Command Line: Open the terminal (Ctrl+Alt+T) and run: nmcli dev wifi Look at the SECURITY column on the far right. It
  should ideally show WPA3 or WPA2 [1] .
- GUI Method: Click the Wi-Fi icon in the top right, go to Wi-Fi Settings, click the gear icon next to your network, and
  check the Security tab [2, 3, 4, 5] .

## 2. Find Your Router’s Admin Page

To change your settings, you first need your router's "Default Gateway" IP address.

- Find the IP: Run this command in the terminal: ip route | grep default The number after via (e.g., 192.168.1.1) is your
  router's address [6, 7] .
- Access the Router: Open your web browser and type that IP address into the URL bar. Log in using the admin credentials
  (often found on a sticker on your router) [8, 9] .

## 3. Scan for Unauthorized Devices

You can use Nmap (Network Mapper), a powerful industry-standard tool, to see every device currently on your network.

1.  Install Nmap: sudo apt update && sudo apt install nmap
2.  Run a Scan: If your router IP is 192.168.1.1, scan the whole network range by running: sudo nmap -sn 192.168.1.0/24 This
    will list every active device, their IP addresses, and sometimes their manufacturer (e.g., "Apple", "Samsung", "TP-Link")
    [10, 11] .

## 4. Advanced Security Auditing Tools

If you want to perform a deeper security audit, Ubuntu supports several professional-grade tools:

- Wireshark: A protocol analyzer to inspect data packets for unencrypted traffic [12] .
- [Aircrack-ng](https://github.com/david-palma/wifi-cracking): A suite used specifically to test the strength of Wi-Fi
  encryption and "pentest" your own network [13, 14] .
- [Lynis](https://www.coursehero.com/tutors-problems/Computer-Science/59611330-1-Identify-5-Web-Application-Penetration-Testing-tools-and/):
  A security auditing tool that can scan your entire system and network for misconfigurations [13] .
