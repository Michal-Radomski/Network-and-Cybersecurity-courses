Defense in Depth is a cybersecurity strategy that uses multiple layers of security controls to protect data. The core idea is
that if one layer fails, others are in place to stop the threat. [1, 2, 3, 4, 5] Think of it like a medieval castle: even if
an invader crosses the moat, they still have to deal with the high walls, the gatehouse, and finally the inner keep.

## The Layers of Defense

In a digital environment, these layers are typically broken down into several categories:

1.  Physical Security: Locks, security cameras, and biometric scanners that protect the actual servers and hardware.
2.  Perimeter Security: Firewalls and VPNs that guard the boundary between the private network and the internet.
3.  Network Security: VLANs, NAC, and internal firewalls that limit movement inside the network.
4.  Host Security: Antivirus, EDR (Endpoint Detection and Response), and regular OS patching (like the Ubuntu updates we
    discussed).
5.  Application Security: Secure coding practices and Web Application Firewalls (WAF) to protect specific software.
6.  Data Security: Encryption at rest and in transit (using TLS/SSL) to ensure that even if data is stolen, it is unreadable.
    [6, 7, 8, 9, 10]

## Why it is Necessary

No single security tool is 100% effective. A Defense in Depth approach accounts for:

- Human Error: A user clicking a phishing link.
- Software Vulnerabilities: A "Zero-Day" exploit in a web server.
- Insider Threats: A disgruntled employee with network access. [11, 12, 13, 14]

By layering controls, you increase the "cost" and effort required for an attacker to succeed, often causing them to give up
or be detected before they reach the "crown jewels" (your sensitive data). [15, 16]
