NAT (Network Address Translation) is a method that allows multiple devices on a private network to share a single public IP
address. Masquerade is a specific, dynamic form of NAT used when your public IP address changes frequently. [1, 2, 3, 4]

## How NAT Works

NAT acts as a translator at your network's gateway (the router). [5]

1.  Outbound: When an internal device sends a request to the internet, the router replaces the device's private IP with its
    own public IP.
2.  Tracking: The router records this translation in a NAT table, noting which internal device made the request and which
    port it used.
3.  Inbound: When the external server responds, the router checks its table, replaces the public destination address with the
    original private IP, and forwards the packet to the correct internal device. [3, 6, 7, 8, 9]

## NAT vs. Masquerade

| Feature [1, 4, 12, 13, 14, 15, 16, 17] | Source NAT (SNAT)                                                           | Masquerade                                                                      |
| -------------------------------------- | --------------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| IP Address Type                        | Used with static public IP addresses.                                       | Used with dynamic public IP addresses (DHCP/PPP).                               |
| IP Assignment                          | You must manually specify the translation IP in the rule.                   | The router automatically detects and uses the current IP of the exit interface. |
| Connection Handling                    | Maintains connection states even if the interface momentarily resets.       | Clears all connection entries if the interface goes down.                       |
| Performance                            | Slightly faster as it doesn't need to "lookup" the interface IP every time. | Slightly more overhead due to dynamic IP lookups.                               |

Key Takeaway: Use Masquerade for home internet connections where your ISP might change your IP address at any time. Use SNAT
in enterprise environments where you have a fixed, permanent public IP address. [9, 15, 18]
