PAT (Port Address Translation) is a specific type of NAT that allows thousands of devices on a private network to share a
single public IP address simultaneously by using different port numbers. It is sometimes called NAT Overload because it
"overloads" one public IP with many internal connections.

## How it Works

While standard NAT often maps one private IP to one public IP, PAT uses the Source Port to keep track of multiple
conversations.

1.  Request: Your laptop (192.168.1.5) and your phone (192.168.1.10) both try to access Google at the same time.
2.  Translation: The router changes both requests to use its one public IP (99.88.77.66), but assigns them unique unique
    source ports:

- Laptop request becomes: 99.88.77.66:10001
  - Phone request becomes: 99.88.77.66:10002

3.  Return: When Google sends data back to port 10001, the router checks its table and knows that specific data belongs to
    the Laptop.

## Why is PAT important?

- IP Conservation: Without PAT, every device in your home would need its own expensive, unique public IPv4 address. With PAT,
  a single home or office can support over 65,000 simultaneous connections using just one public IP.
- Security: Like NAT, PAT provides a layer of security because external servers only see the router's IP and port; they
  cannot see or initiate direct connections to your internal devices.

## Summary: NAT vs. PAT

- Static NAT: 1 Private IP = 1 Public IP (Used for hosting web servers).
- Dynamic NAT: A group of Private IPs share a group of Public IPs.
- PAT (NAT Overload): Many Private IPs = 1 Public IP (Used in almost all home and small office routers).
