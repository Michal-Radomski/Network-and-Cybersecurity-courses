To turn a PC into a replacement for your router that handles firewall, IDS, and IPS duties, you need to follow these steps.

## 1. Hardware Requirements

- The PC: Almost any old PC will work, but it must have two Network Interface Cards (NICs).
- WAN Port: Connects to your internet modem.
  - LAN Port: Connects to your home switch or Wi-Fi access point.
  - Note: Avoid USB Ethernet adapters; they are unstable for high-speed routing.
- A USB Drive: To create the installer. [1]

## 2. Choose Your Operating System

While you can use Ubuntu, it is difficult to manage as a primary router. Most pros use a specialized, free "Firewall OS" that
has a web interface.

- pfSense: The gold standard. It is rock solid and has a massive community.
- OPNsense: A user-friendly fork of pfSense with a more modern interface and better integration for Suricata. [2]

## 3. The Setup Process (Step-by-Step)## Step A: Install the OS

1.  Download the ISO (e.g., OPNsense) and "burn" it to your USB drive using Etcher.
2.  Boot the PC from the USB.
3.  Follow the prompts to install it onto the hard drive.
4.  Assign Interfaces: The installer will ask which port is WAN (internet) and which is LAN (home). [3]

## Step B: Connect the Cables

- Unplug your old router.
- Connect the Modem to your new PC's WAN port.
- Connect your Laptop to the LAN port to finish setup.

## Step C: Initial Configuration

1.  On your laptop, open a browser and go to the default IP (usually 192.168.1.1).
2.  Log in (default is usually root / opnsense or admin / pfsense).
3.  Follow the Setup Wizard to configure your internet connection (DHCP, PPPoE, etc.).

## 4. Enabling IDS/IPS (Suricata/Snort)

Once the router is working, you add the "security" layer:

1.  Install the Plugin: In the web UI, go to System > Firmware > Plugins. Search for os-suricata (on OPNsense) or snort (on
    pfSense) and click Install.
2.  Download Rules: Go to the Services > Intrusion Detection menu.

- Enable the Emerging Threats (ET) Open ruleset.
  - Click Download & Update.

3.  Monitor First (IDS): Enable the service on your WAN interface but do not check "IPS mode" yet. Let it run for 24 hours to
    see what it flags.
4.  Activate Prevention (IPS): Once you are sure it isn't blocking legitimate traffic, check the box for "IPS Mode" or "Block
    Offenders." Your PC will now actively drop malicious packets before they enter your home. [4, 5]

## 5. Adding Wi-Fi

Since your PC likely doesn't have good Wi-Fi antennas, you shouldn't use it as a wireless broadcaster. Instead:

- Take your old router, put it in "Access Point (AP) Mode," and plug it into the LAN port of your new security PC. [6]

To turn an Ubuntu PC into a replacement for your home router with built-in security, you must configure it to act as a
Network Gateway. This involves setting up packet forwarding, Network Address Translation (NAT), and then layering Suricata or
Snort on top. [1, 2]

## 1. Prepare the Network Interfaces

Ensure your PC has two physical network ports (NICs). Ubuntu identifies them with names like eth0 or enp0s3. [2, 3]

- WAN (Port 1): Connects to your internet modem.
- LAN (Port 2): Connects to your home switch or Wi-Fi Access Point. [1, 2, 4]

## 2. Enable IP Forwarding [5]

By default, Ubuntu will not pass traffic between two ports. You must enable routing at the kernel level: [2, 6]

1.  Open the config: sudo nano /etc/sysctl.conf.
2.  Uncomment this line: net.ipv4.ip_forward=1.
3.  Apply changes: sudo sysctl -p. [2, 7]

## 3. Configure NAT and Firewall (UFW) [5]

You need to allow devices on your LAN to "share" your WAN’s internet connection using NAT (Masquerading). [5, 7]

1.  Set Defaults: Tell UFW to block incoming but allow outgoing traffic.

sudo ufw default deny incoming sudo ufw default allow outgoing

2.  Edit NAT Rules: You must add NAT instructions to /etc/ufw/before.rules.

- Add these lines at the very top (replace eth0 with your WAN interface): \*nat :POSTROUTING ACCEPT [0:0] -A POSTROUTING -o
  eth0 -j MASQUERADE COMMIT 3. Enable Firewall: sudo ufw enable. [5, 7, 8, 9, 10]

## 4. Layering Suricata (IDS/IPS) [11]

Now that the PC is a router, install Suricata to scan the traffic flowing through it. [12, 13]

1.  Installation: Add the official repo and install: sudo add-apt-repository ppa:oisf/suricata-stable && sudo apt update &&
    sudo apt install suricata -y.
2.  Configuration: Edit /etc/suricata/suricata.yaml to define your protected network (HOME_NET) and select your monitoring
    interface.
3.  IPS Mode (Optional): To actively block threats instead of just alerting, you must configure Suricata to use NFQUEUE and
    set up iptables to send forwarded traffic to that queue. [11, 13, 14, 15, 16, 17]

## Summary of the Path

- Traffic Flow: Internet → Modem → Ubuntu (WAN Port) → UFW/NAT/Suricata → Ubuntu (LAN Port) → Home Devices.
- Wi-Fi: Since the PC is now your router, plug your old Wi-Fi router into the LAN port and set it to "Access Point" mode. [1,
  17]
