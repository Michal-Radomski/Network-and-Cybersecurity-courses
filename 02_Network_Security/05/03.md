An Intrusion Prevention System (IPS) is the proactive "big brother" to the IDS. While an IDS is a security camera that
records and alerts, an IPS is a security guard who sees the intruder and actively tackles them to stop the crime in progress.
[1, 2, 3, 4] It sits "inline"—meaning all network traffic must physically pass through it before reaching its destination.
[5, 6]

## How it Works: The "In-Line" Process

Because an IPS sits directly in the path of your data, it works in four immediate steps: [7, 8, 9]

1.  Traffic Inspection: Every packet entering the network is intercepted and scanned in real-time.
2.  Analysis: It uses the same methods as an IDS (Signatures, Anomalies, and Protocol Analysis) to check for threats.
3.  Action: If a packet matches a known threat, the IPS takes immediate action (see below).
4.  Reporting: It logs the incident and notifies the admin, just like an IDS. [10, 11, 12, 13, 14]

## How it Stops Attacks (Actions)

An IPS has several ways to neutralize a threat instantly:

- Dropping Packets: It simply deletes the malicious data packets so they never reach the target.
- Terminating Sessions: It sends a "Reset" command to both the sender and receiver to kill the connection entirely.
- Blocking IP Addresses: It can automatically update firewall rules to "shun" (block) the attacker's IP address for a set
  amount of time.
- Cleaning Traffic: In some cases, it can strip away the malicious part of a packet (like a virus attachment) and let the
  "clean" part of the message through. [15, 16, 17, 18]

## IDS vs. IPS: A Quick Comparison

| Feature [19, 20, 21, 22, 23] | IDS (Detection)               | IPS (Prevention)                   |
| ---------------------------- | ----------------------------- | ---------------------------------- |
| Analogy                      | Security Alarm / Camera       | Security Guard / Bouncer           |
| Placement                    | Out-of-band (monitors a copy) | In-line (traffic flows through it) |
| Response                     | Alerts and logs only          | Blocks, drops, and resets          |
| Risk                         | No impact on network speed    | Can cause "lag" if it's too slow   |
| False Positives              | Annoying (too many alerts)    | Critical (blocks legitimate users) |

## The "False Positive" Risk

The biggest challenge with an IPS is accuracy. If an IPS incorrectly identifies a legitimate business email as a virus (a
False Positive), it will block that email automatically. This is why many admins start by running their system in "IDS mode"
to ensure it's accurate before "flipping the switch" to IPS mode. [24, 25, 26]
