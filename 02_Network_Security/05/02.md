An Intrusion Detection System (IDS) works like a digital security alarm. Its primary job is to monitor network traffic or
system activity, identify suspicious patterns that indicate an attack, and alert administrators. [1, 2, 3] Unlike a firewall,
which acts as a bouncer at the door, an IDS acts as a detective already inside the building, looking for people who shouldn't
be there. [4, 5]

## How it Monitors (Placement)

An IDS can be deployed in two main ways:

- Network-based (NIDS): It is placed at strategic points in the network (like behind a firewall) to monitor all incoming and
  outgoing traffic. It typically uses a "TAP" or "SPAN" port to analyze a copy of the traffic so it doesn't slow down the
  actual network speed.
- Host-based (HIDS): It is installed directly on an individual device, such as a server or laptop. It monitors that specific
  device’s internal logs, file integrity, and user activity. [5, 6, 7, 8, 9, 10, 11]

## How it Detects (Analysis Methods)

The "brain" of the IDS uses several techniques to decide if something is dangerous: [12, 13]

1.  Signature-based Detection: This is the most common method. The IDS has a database of "fingerprints" (signatures) of known
    attacks—like specific code sequences in a virus or a known bad subject line in an email. If it finds a match, it triggers
    an alarm.
2.  Anomaly-based Detection: The system first spends time learning what "normal" looks like for your network (a baseline). It
    then uses machine learning to flag anything that deviates from that norm—such as a user accessing a sensitive database at
    3 AM for the first time. This is great for catching new, "zero-day" attacks that don't have a signature yet.
3.  Stateful Protocol Analysis: It looks at the actual state of communication protocols (like HTTP or DNS) to ensure they are
    following the rules. If a packet sequence is mathematically impossible for that protocol, the IDS flags it as an
    intrusion attempt. [3, 5, 6, 14, 15, 16, 17, 18, 19, 20]

## The Final Action

When an IDS identifies a threat, it generates an alert for security teams and records the event in a log. It does not stop
the attack itself; that is the job of an
[Intrusion Prevention System (IPS)](https://www.fortinet.com/resources/cyberglossary/intrusion-detection-system), which can
automatically block malicious traffic. [1, 21, 22, 23, 24]
