An HTTP RAT (Remote Access Trojan) is a type of malware that allows an attacker to covertly control a computer by using HTTP
or HTTPS protocols to communicate with their Command and Control (C2) server. Unlike traditional RATs that might use uncommon
ports (which are easily blocked by firewalls), an HTTP RAT "hides in plain sight" by disguising its malicious traffic as
normal web browsing.

## How an HTTP RAT Works

1.  Infection: The malware is installed on the victim's machine (via phishing, a malicious download, or an exploit).
2.  Beaconing: The RAT periodically sends an HTTP "GET" or "POST" request to the attacker's server. To a firewall, this looks
    like a user simply visiting a website.
3.  Command Retrieval: The attacker’s server responds to that request with instructions (e.g., "take a screenshot," "upload
    this file," or "log keystrokes").
4.  Data Exfiltration: The RAT executes the command and sends the stolen data back to the attacker, often encrypted within
    standard HTTP traffic.

## Why Attackers Use Them

- Firewall Bypassing: Most networks must allow HTTP (port 80) and HTTPS (port 443) traffic to function. An HTTP RAT uses
  these "open doors" to avoid being blocked.
- Stealth: Because the traffic looks like standard web requests, it is very difficult for security software to distinguish
  between a user reading the news and a RAT sending stolen passwords to a hacker.
- Ease of Use: Many HTTP RATs come with a web-based control panel, allowing the attacker to manage thousands of infected
  "zombie" computers from any browser.

## Key Capabilities

A typical HTTP RAT gives the attacker almost total control, including:

- Keylogging: Recording every keystroke you type.
- File Management: Downloading, uploading, or deleting your files.
- Remote Desktop: Seeing your screen in real-time.
- Webcam/Microphone Access: Secretly activating your camera or mic.

## Protection on Ubuntu

Since you are using Ubuntu, you can defend against these by:

- Monitoring Outbound Traffic: Use tools like opensnitch (an application-level firewall) which will alert you whenever a
  program tries to connect to a new website.
- Regular Audits: Use the netstat -tp command to see which programs are currently making web connections.
