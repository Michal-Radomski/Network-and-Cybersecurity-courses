HTTP Flooding is a type of Layer 7 (application layer) DDoS attack where an attacker overwhelms a web server or application
with a massive volume of seemingly legitimate HTTP requests. Because these requests use standard formatting and valid URLs,
they are difficult to distinguish from real user traffic, allowing them to "fly under the radar" of traditional network-level
defenses. [1, 2, 3, 4]

## How it Works

The goal of the attack is to force the server to allocate all its resources (CPU, RAM, or database connections) to processing
the flood of fake requests, making it unable to serve actual users. Attackers often use botnets—large networks of infected
"zombie" computers—to launch these requests simultaneously from thousands of different IP addresses. [1, 5, 6, 7]

## Main Types of HTTP Floods

- HTTP GET Attack: The most common form, where bots repeatedly request static content like images or large files. The server
  is forced to retrieve and send these files over and over until it is overloaded.
- HTTP POST Attack: A more resource-intensive attack where bots submit data to the server (e.g., filling out a login or
  search form). This is highly effective because processing form data often requires the server to perform complex tasks like
  database lookups, consuming more power per request than a simple GET. [1, 8, 9, 10, 11, 12, 13]

## Mitigation Strategies

Since these attacks look like normal traffic, they require specialized
[DDoS protection](https://www.cloudflare.com/learning/ddos/http-flood-ddos-attack/) tools to mitigate: [14, 15, 16]

- Web Application Firewall (WAF): Analyzes the behavior and headers of requests to identify and block abnormal patterns.
- Progressive Challenges: Uses tests like [CAPTCHAs](https://www.akamai.com/glossary/what-is-an-http-flood-ddos-attack) or
  hidden JavaScript challenges to verify if a requester is a human or a bot.
- IP Reputation Databases: Security teams track known malicious IP addresses and block them before they can reach the server.
- Rate Limiting: Restricts the maximum number of requests a single IP can make within a specific timeframe. [3, 16, 17, 18,
  19, 20, 21]
