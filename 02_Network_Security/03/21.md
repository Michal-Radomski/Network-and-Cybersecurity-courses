The Account Lockout Policy in Windows is a built-in security feature designed to protect user accounts from brute-force and
dictionary attacks. It automatically disables a user's account after a specified number of failed login attempts for a set
period. [1, 2, 3, 4]

## Windows Account Lockout Policy

Windows manages this through three primary settings found in Local Security Policy or Group Policy Management under Computer
Configuration\Windows Settings\Security Settings\Account Policies\Account Lockout Policy: [5, 6, 7, 8, 9]

- Account Lockout Threshold: The number of invalid logon attempts (0 to 999) that will trigger a lockout. A value of 0 means
  the account will never be locked.
- Account Lockout Duration: The number of minutes (1 to 99,999) the account remains locked before automatically unlocking.
  Setting this to 0 requires an administrator to manually unlock the account.
- Reset Account Lockout Counter After: The time in minutes that must pass after a failed attempt before the "failure counter"
  resets to zero. [2, 10, 11, 12, 13]

## Linux Equivalent: pam_faillock [14]

On Linux (including Ubuntu, RHEL, and Debian), the equivalent functionality is handled by PAM (Pluggable Authentication
Modules). Specifically, modern distributions use the
[pam_faillock](https://manpages.ubuntu.com/manpages/focal/man5/faillock.conf.5.html) module. [15, 16, 17, 18]

- Module: pam_faillock.so replaced the older pam_tally2 module to provide better security and flexibility.
- Configuration: Settings are typically defined in /etc/security/faillock.conf or directly within PAM service files like
  /etc/pam.d/common-auth.
- Parameters:
- deny=n: Equivalent to the Windows Threshold.
  - unlock_time=n: Equivalent to Duration (measured in seconds in Linux).
  - even_deny_root: A specific Linux feature that allows the root (administrator) account to be locked as well, which is
    often disabled by default to prevent permanent lockout. [16, 17, 19, 20, 21, 22]

## Managing Lockouts on Linux

Unlike Windows' graphical interface, Linux uses command-line tools for management: [23, 24]

- Check failed attempts: Run the faillock command to see a list of failed login attempts for users.
- Manual unlock: Use faillock --user <username> --reset to immediately unlock a specific user. [18, 25, 26]
