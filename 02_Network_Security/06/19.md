To build a robust defense, you should follow the principle of Least Privilege: block everything by default and only open the
specific "doors" you absolutely need.

## 1. The "Default Deny" Rule

The most important best practice is the Implicit Deny. Your last rule should always be to block all traffic that wasn't
specifically allowed by the rules above it.

- Inbound: Deny all by default (UFW does this automatically).
- Outbound: For high security, deny all outbound and only allow specific ports (80, 443, etc.) to prevent malware from
  "phoning home."

## 2. Clean Up Your Rules

- Remove "Shadowed" Rules: Because firewalls read from top to bottom, a broad "Allow All" rule at the top will make a "Block
  IP" rule at the bottom useless.
- Delete Unused Rules: If you no longer use Plex or an old printer, delete those specific UFW rules immediately. Open ports
  are "attack surfaces."
- Use Descriptive Labels: In UFW, use the comment feature to remember why a rule exists: sudo ufw allow 32400/tcp comment
  'Plex Media Server'.

## 3. Secure Remote Access

- Never open Port 22 (SSH) to the world: If you must access your Ubuntu machine from outside your house, use a VPN (like
  WireGuard or Tailscale) or limit SSH access to a specific IP address.
- Use Non-Standard Ports: While not a complete security solution (security by obscurity), moving services like SSH to a
  random port (e.g., 2222) can stop 99% of automated bot scans.

## 4. Enable Logging and Alerts

- Monitor "Denies": Turn on logging (sudo ufw logging on) to see who is trying to hit your firewall.
- Use an IPS/IDS: As discussed earlier, tools like Suricata or Snort add a layer of intelligence that a standard firewall
  lacks by looking for patterns of an active attack.

## 5. Layered Defense (Defense in Depth)

- Host + Network: Always run a firewall on your computer (UFW/ESET) even if your router (pfSense/OPNsense) has one.
- VLANs: If you use a network firewall like OPNsense, put your "untrusted" IoT devices (smart cameras, bulbs) on a separate
  VLAN so they can't talk to your Ubuntu PC.

## 6. Regular Audits

Once a month, run sudo ufw status numbered to review your active rules. If you don't recognize a rule or a port, block it
until you've confirmed it's necessary.
