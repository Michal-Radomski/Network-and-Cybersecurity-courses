Both Suricata and Snort are primarily designed to be installed on a network gateway, though they can also be used as
host-based protection. [1, 2, 3]

## Primary Deployment: Network Gateway

In this setup, the software runs on a machine that acts as your router (like a pfSense or OPNsense box). [3, 4, 5]

- Protection: It secures all devices behind it (phones, laptops, smart home tech) by inspecting traffic before it reaches
  them.
- Performance: It uses its own dedicated CPU/RAM, so it doesn't slow down your gaming or work computer.
- Visibility: It sees all network traffic but cannot see specific "internal" events on your PC, like a file being modified.
  [1, 3, 6, 7]

## Secondary Deployment: Host-Based (HIDS/HIPS) [7]

You can install them directly on an Ubuntu workstation or server. [8, 9]

- Protection: It protects only that specific machine.
- Context: It can be more effective for detecting threats that originate from within the machine itself or encrypted traffic
  that was decrypted by the host.
- Setup: On Linux, this usually requires configuring iptables or nftables to send packets to a specific queue (NFQUEUE) so
  Suricata/Snort can inspect them before they are processed by the OS. [1, 10, 11]

## Which is better where?

- Suricata: Highly recommended for gateways because its multi-threaded architecture allows it to handle massive amounts of
  high-speed network traffic efficiently.
- Snort: Often preferred for individual hosts or low-powered devices because it typically has a smaller resource footprint in
  single-core environments. [12, 13, 14, 15]

| Deployment Type [3] | Focus                 | Example Use Case                                                                  |
| ------------------- | --------------------- | --------------------------------------------------------------------------------- |
| Network Gateway     | The "Border Guard"    | An OPNsense box protecting a home Wi-Fi network.                                  |
| Host-Based          | The "Internal Sentry" | Installing Suricata[](https://suricata.io/) on a public-facing Ubuntu web server. |
