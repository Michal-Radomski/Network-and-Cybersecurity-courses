For your Ubuntu setup with Apache2 as a reverse proxy for your Node.js/React app, the most effective way to implement a WAF
is by using [ModSecurity](https://modsecurity.org/). Since Apache is already handling your traffic before it hits your
PM2-managed Node service, ModSecurity can sit directly inside Apache to inspect and filter requests. [1, 2]

## 1. Install ModSecurity [3]

Install the ModSecurity module for Apache using the standard package manager. [4]

sudo apt update sudo apt install libapache2-mod-security2 -y

## 2. Enable "Blocking" Mode

By default, ModSecurity only logs attacks without stopping them. You must switch it to On to actually protect your app. [4,
5]

1.  Copy the config: sudo cp /etc/modsecurity/modsecurity.conf-recommended /etc/modsecurity/modsecurity.conf
2.  Edit the file: sudo nano /etc/modsecurity/modsecurity.conf
3.  Change the rule engine: Find SecRuleEngine DetectionOnly and change it to SecRuleEngine On.
4.  Restart Apache: sudo systemctl restart apache2. [2, 4, 5]

---

## 3. Deploy the OWASP Core Rule Set (CRS) [6]

ModSecurity is an engine, but it needs "rules" to know what to block. The OWASP CRS provides a pre-configured set of rules to
stop common attacks like SQL Injection and XSS. [1, 2, 5, 6, 7]

1.  Download the rules:

git clone https://github.com/coreruleset/coreruleset.git

2.  Move the config:

cd coreruleset sudo mv crs-setup.conf.example /etc/modsecurity/crs-setup.conf sudo mv rules/ /etc/modsecurity/

3.  Link the rules to Apache: Edit /etc/apache2/mods-enabled/security2.conf and ensure it includes the rules:

IncludeOptional /etc/modsecurity/_.conf Include /etc/modsecurity/rules/_.conf

---

## 4. Verify the Connection

Since you are using a reverse proxy (Apache to Node.js), check your logs to ensure legitimate traffic isn't being blocked.
[8]

- WAF Logs: Located at /var/log/apache2/modsec_audit.log.
- PM2 Logs: Keep an eye on your app logs with pm2 logs to ensure requests are still reaching your Node.js backend. [3, 8]

🛡️ Essential Tip: If your React app uses a lot of large JSON payloads or file uploads, you may need to increase the
SecRequestBodyLimit in modsecurity.conf. The default is often too low for modern web apps. [9]
