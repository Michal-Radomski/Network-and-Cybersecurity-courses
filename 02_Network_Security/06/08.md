To allow services like Plex or a network printer through your Ubuntu firewall (UFW), you need to open specific ports. Because
UFW's default policy is to block all incoming traffic, these services will fail to connect unless you explicitly permit them.
[1, 2]

## Allowing Plex Media Server

Plex requires several ports for basic streaming and device discovery. [3]

- Primary Streaming Port: This is the most critical port for accessing your library.
- sudo ufw allow 32400/tcp
- Discovery & DLNA (Optional): If you use a smart TV or DLNA-compatible devices that "find" the server automatically, you may
  also need:
- sudo ufw allow 1900/udp (DLNA)
  - sudo ufw allow 32410:32414/udp (GDM Network Discovery) [3, 4, 5]

## Allowing Network Printers

Printing usually involves two parts: Discovery (finding the printer) and Communication (sending the job). [6, 7]

- Printer Communication (CUPS/IPP): Most modern printers use the Internet Printing Protocol (IPP).
- sudo ufw allow 631/tcp
- Discovery (Avahi/mDNS): To let your computer "see" the printer on the local network automatically (Bonjour).
- sudo ufw allow 5353/udp [7, 8, 9]

## Pro-Tip: Restrict by Local Network

Since you only want your family or housemates to access these services, it is safer to only allow your local network rather
than the whole world. If your router uses a common IP range like 192.168.1.x, you can restrict the rule:

- sudo ufw allow from 192.168.1.0/24 to any port 32400 [4, 10]

## Summary Checklist

| Service | Action          | Command                  |
| ------- | --------------- | ------------------------ |
| Plex    | Allow Streaming | sudo ufw allow 32400/tcp |
| Printer | Allow Jobs      | sudo ufw allow 631/tcp   |
| Printer | Allow Discovery | sudo ufw allow 5353/udp  |
| SSH     | DO THIS FIRST   | sudo ufw allow ssh       |

After running these commands, verify your active rules by typing sudo ufw status numbered. [11, 12]
