In Ubuntu, you manage the firewall using UFW (Uncomplicated Firewall), which is a user-friendly interface for the more
complex underlying iptables. [1, 2]

## How to Enable and Disable UFW

Open your terminal and use the following commands:

- To Enable: sudo ufw enable
- Warning: If you are connecting remotely via SSH, you must run sudo ufw allow ssh first to avoid locking yourself out.
- To Disable: sudo ufw disable
- To Check Status: sudo ufw status
- For a GUI: If you prefer a visual interface, install GUFW with sudo apt install gufw. [2, 3, 4, 5, 6, 7, 8]

## Should You Enable it if Your Router has SPI?

Yes, you should still enable it. This is a security principle called Defense in Depth. While a router's Stateful Packet
Inspection (SPI) firewall is your "front gate," the Ubuntu firewall acts as your "locked bedroom door." [9, 10, 11, 12, 13]
Here is why having both is beneficial:

- Lateral Movement Protection: If another device on your Wi-Fi (like a smart bulb or a guest's phone) is compromised, the
  router firewall cannot stop it from attacking your computer. UFW protects you from threats already inside your network.
- Outbound Control: While routers mostly block incoming threats, a host-based firewall like UFW can be configured to alert
  you or block suspicious outgoing traffic, which is a common sign of malware "phoning home".
- Portability: If you ever take your laptop to a coffee shop or use a different network, your router's protection stays at
  home. UFW stays with your device wherever you go.
- Process Awareness: Host firewalls can sometimes distinguish between different applications, allowing you to block specific
  software from accessing the internet while letting others through. [9, 14, 15, 16, 17, 18, 19]

Since Ubuntu's default policy is to deny all incoming and allow all outgoing traffic, enabling it typically has zero impact
on your normal browsing but provides that critical extra layer of safety. [5, 20]
