Setting up TLS 1.3 on an Apache web server running on Ubuntu ensures your website uses the latest, fastest, and most secure
encryption protocol.

Here is a step-by-step guide to getting it configured.

### Prerequisites

Before you start, ensure your system meets the minimum version requirements, as older versions do not support TLS 1.3:

- **Ubuntu 20.04 LTS** or newer (Ubuntu 22.04 or 24.04 is ideal).
- **Apache v2.4.37** or higher.
- **OpenSSL v1.1.1** or higher.

You can check your versions by running:

```bash
apache2 -v
openssl version

```

---

### Step 1: Update Your System

Ensure your repositories and packages are fully up to date.

```bash
sudo apt update && sudo apt upgrade -y

```

### Step 2: Configure Apache for TLS 1.3

You need to modify your Apache SSL configuration file. If you are using the default Let's Encrypt configuration or the
standard Apache SSL module, you will edit the primary SSL configuration file.

1. Open the global SSL configuration file using a text editor like `nano`:

```bash
sudo nano /etc/apache2/mods-available/ssl.conf

```

2. Search for the `SSLProtocol` directive. If it is commented out or already exists, modify it to look like this:

```apache
SSLProtocol all -SSLv3 -TLSv1 -TLSv1.1 -TLSv1.2

```

> **Note:** The configuration above restricts connections **strictly** to TLS 1.3. If you still need to support older clients
> (like older phones or legacy browsers) while prioritizing TLS 1.3, use this instead:
>
> ```apache
> SSLProtocol all -SSLv3 -TLSv1 -TLSv1.1
>
> ```

3. Next, find or add the `SSLCipherSuite` directive to specify secure ciphers. For TLS 1.3, modern secure ciphers are handled
   automatically by OpenSSL, but for the best compatibility block, paste the following:

```apache
SSLCipherSuite EECDH+AESGCM:EDH+AESGCM
SSLHonorCipherOrder on

```

4. Save and close the file (in nano, press `Ctrl + O`, `Enter`, then `Ctrl + X`).

---

### Step 3: Update Your Virtual Host (If Applicable)

If you have a specific Virtual Host file for your website (e.g., `/etc/apache2/sites-available/yourdomain-le-ssl.conf`), the
lines inside that file might override the global settings you just changed.

1. Open your specific site's SSL configuration file:

```bash
sudo nano /etc/apache2/sites-available/yourdomain-ssl.conf

```

2. Look for `SSLProtocol` inside the `<VirtualHost *:443>` block. If it exists, update it to match what you configured in
   Step 2:

```apache
SSLProtocol all -SSLv3 -TLSv1 -TLSv1.1

```

3. Save and close the file.

---

### Step 4: Test and Restart Apache

Before restarting Apache, always test your configuration files for syntax errors to avoid crashing your web server.

```bash
sudo apache2ctl configtest

```

If the output says `Syntax OK`, restart Apache to apply the changes:

```bash
sudo systemctl restart apache2

```

---

### Step 5: Verify TLS 1.3 is Working

You can verify that your server successfully negotiates TLS 1.3 using an external tool or the command line.

#### Method A: Using OpenSSL via Terminal

Run the following command from another machine (or locally) to test the connection:

```bash
openssl s_client -connect yourdomain.com:443 -tls1_3

```

Look through the output. If successful, you will see a line stating: `New, TLSv1.3, Cipher is TLS_AES_256_GCM_SHA384`

#### Method B: Browser DevTools

1. Open your website in Google Chrome or Firefox.
2. Press `F12` to open Developer Tools and navigate to the **Security** tab.
3. Refresh the page, and it will show the connection protocol (e.g., "Connection - secure TLS 1.3").
