When a device sends a **Multicast** packet, it doesn't just use a multicast IP address (Layer 3)—it also needs a
corresponding **Multicast MAC address** (Layer 2) so that network switches know exactly which physical ports to send the data
to.

Unlike standard MAC addresses, which are uniquely burned into a device's network card at the factory, multicast MAC addresses
are **special, shared addresses** generated automatically based on the multicast IP address.

Here is how they work, how they are structured, and how they are calculated.

---

## The Core Rule: The `01:00:5E` Prefix

Every single IPv4 multicast MAC address in the world starts with the exact same **24-bit prefix** (the first 3 bytes):

$$\mathbf{01:00:5E}$$

When a network switch sees a MAC address starting with `01:00:5E`, it instantly knows, _"Ah, this isn't a single device, and
it's not a broadcast to everyone. This is a multicast group."_

---

## How an IPv4 Multicast MAC Address is Built

A MAC address is 48 bits (6 bytes) long. An IPv4 multicast MAC address is split into three specific parts:

1. **The Vendor Prefix (24 bits):** Always `01:00:5E`.
2. **The Guide Bit (1 bit):** The 25th bit is always set to **0**.
3. **The IP Mapping Bits (23 bits):** The remaining 23 bits are copied directly from the last 23 bits of the destination
   Multicast IP address.

Because only 23 bits of the IP address are copied over, this creates a famous quirk in networking known as the **32-to-1
overlap**.

> ### ⚠️ The 32-to-1 Overlap Problem
>
> Multicast IP addresses use 28 bits to define a group, but the MAC address only has room for 23 of those bits. Because 5
> bits of the IP address are "lost" during the conversion, **32 different multicast IP addresses will map to the exact same
> multicast MAC address.** For example, `224.1.1.1` and `225.1.1.1` will share the identical MAC address:
> `01:00:5E:01:01:01`. If both are running on the same local network, the network card has to look up at Layer 3 to filter
> out the stream it didn't ask for.

---

## Example: Mapping an IP to a MAC

Let's look at a real-world example using the popular routing protocol address **`224.0.0.5`** (used by OSPF).

1. Start with the mandatory prefix: `01:00:5E:00:00:00`
2. Convert the last three octets of the IP address (`0.0.5`) into hexadecimal:

- `0` becomes `00`
- `0` becomes `00`
- `5` becomes `05`

3. Combine them together to get the final Multicast MAC address:

$$\mathbf{01:00:5E:00:00:05}$$

---

## What about IPv6 Multicast MACs?

IPv6 handles multicast a bit differently and completely skips the `01:00:5E` prefix. Instead, all IPv6 multicast MAC
addresses start with the prefix **`33:33`**.

The remaining 32 bits of the MAC address are simply copied directly from the last 32 bits of the IPv6 multicast address. For
example, the IPv6 multicast address `FF02::1` maps to the MAC address `33:33:00:00:00:01`.
