In networking, an **EtherType** is a two-byte (16-bit) field located inside a standard Layer 2 Ethernet frame. Its primary
job is to tell the receiving network card **which protocol is hidden inside the payload** of the frame so it knows how to
process the data next.

Think of an Ethernet frame like a shipping box. The EtherType is the shipping label on the outside that says _"Fragile:
Electronics Inside"_ or _"Perishable: Food Inside,"_ so the person opening the box knows exactly which department to send it
to.

---

## Where is it located?

The EtherType field sits right after the Source MAC Address and immediately before the actual data payload.

Once a network switch or computer reads the destination and source MAC addresses, it hits the EtherType field to figure out
what to do with the incoming bytes.

---

## Common EtherType Values

EtherType values are represented as 4-digit hexadecimal numbers (prefixed by `0x`). The IEEE is the organization that assigns
these numbers. Here are some of the most common ones you'll see if you look at network traffic in a tool like Wireshark:

| EtherType (Hex) | Protocol                   | What it means                                                            |
| --------------- | -------------------------- | ------------------------------------------------------------------------ |
| **`0x0800`**    | IPv4                       | The payload contains a standard Internet Protocol version 4 packet.      |
| **`0x86DD`**    | IPv6                       | The payload contains a modern Internet Protocol version 6 packet.        |
| **`0x0806`**    | ARP                        | Address Resolution Protocol (used to map IP addresses to MAC addresses). |
| **`0x8100`**    | VLAN-tagged frame (802.1Q) | Indicates that the frame belongs to a specific Virtual LAN.              |
| **`0x8864`**    | PPPoE Session              | Used by many internet service providers for DSL connections.             |

---

## How it works in action

Imagine your computer receives an Ethernet frame.

1. The network interface card (NIC) checks the destination MAC address. It matches your computer, so it accepts the frame.
2. The NIC looks at the **EtherType** field and sees **`0x0800`**.
3. Because `0x0800` stands for **IPv4**, the NIC immediately strips off the Ethernet outer wrapping and hands the raw payload
   directly up to the computer's **IPv4 software software stack** (Layer 3).

If the EtherType had been `0x86DD`, it would have handed it to the IPv6 engine instead. Without the EtherType, the computer
would just see a chaotic pile of 1s and 0s and have no idea how to decode them.

---

## The "Length" Quirks

Historically, in older IEEE 802.3 Ethernet standards, this exact same 2-byte field was used to describe the _Length_ of the
frame rather than the protocol type.

To prevent confusion, modern networks use a clever trick based on the size of the number:

- If the value in that field is **1500 or less** (decimal), the field is interpreted as the **Length** of the data.
- If the value is **1536 or greater** (which equals `0x0600` or higher in hex), it is interpreted as an **EtherType**.

Since all modern protocols have hex values well above `0x0600`, the field easily pulls double-duty without breaking older
systems.
