In networking, a loop is a disaster. If you connect multiple network switches together in a circle to create backup paths
(redundancy), data packets can get trapped. They will circulate forever, multiplying and creating a **broadcast storm** that
completely crashes the network.

**Spanning Tree Protocol (STP)** is the savior that prevents this. It is a Layer 2 network protocol designed to prevent loops
while still allowing you to keep backup cables in place for safety.

Here is how STP works and how it keeps a network running smoothly.

---

## The Problem: The Infinite Loop

Imagine you have three switches connected in a triangle. If Switch A sends out a broadcast message, it goes to Switch B and
Switch C. Switch B forwards it to Switch C, and Switch C forwards it back to Switch A.

Because Ethernet packets do not have a built-in "expiration date" (like a Time-to-Live counter), this data will loop
infinitely at lightning speed, crushing your bandwidth and knocking the network offline.

---

## The Solution: How STP Works

STP acts like a digital lumberjack. It looks at the network, detects any loops, and **strategically shuts down specific
ports** to break the loop, turning a circular network into a single, clean tree structure (a "spanning tree").

If a primary cable breaks later on, STP automatically turns the blocked port back on to restore connection.

Here is the step-by-step process of how switches negotiate STP:

### 1. Electing the "Root Bridge"

When switches are turned on, they talk to each other using special messages called **BPDUs (Bridge Protocol Data Units)**.
They use these messages to elect a leader, called the **Root Bridge**.

- The switch with the lowest **Bridge ID** (a combination of a priority number and its MAC address) wins the election and
  becomes the center of the network.

### 2. Determining the Best Paths

Once the Root Bridge is chosen, every other switch looks at the available paths to get to it. They calculate the "cost" of
each path based on the speed of the cables (e.g., a 10 Gbps cable is faster and has a lower "cost" than a 1 Gbps cable).

### 3. Assigning Port Roles

Based on those calculations, STP assigns roles to every single switch port on the network:

- **Root Port (RP):** The port on a non-root switch that has the fastest path to the Root Bridge. (Always active).
- **Designated Port (DP):** The port on a network segment that is allowed to forward traffic. All ports on the Root Bridge
  are Designated Ports. (Always active).
- **Non-Designated / Blocked Port:** The port that loses the calculation. STP puts this port into a **Blocking State**. It
  cannot send regular network traffic, but it sits quietly listening for BPDUs in case something breaks.

---

## STP States: From Plugged In to Active

When you plug a cable into an STP-enabled switch, it doesn't instantly turn green and start working. It safely cycles through
several states to ensure it won't cause a loop:

1. **Blocking:** The port is disabled from sending data to prevent loops; it only listens for BPDUs.
2. **Listening:** The switch determines if there are any loops on the path.
3. **Learning:** The port starts building its MAC address table but still doesn't forward user data.
4. **Forwarding:** The port is fully operational, sending and receiving data safely.

---

## Evolution of STP

The original STP (IEEE 802.1D) was invented in 1985 by Radia Perlman. While brilliant, it was slow—taking up to 30 to 50
seconds to fix the network if a cable was pulled.

Because of this, modern networks use **Rapid Spanning Tree Protocol (RSTP - 802.1w)**, which can detect a failure and open a
backup path in less than **6 seconds**, making network hiccups almost unnoticeable to users.
