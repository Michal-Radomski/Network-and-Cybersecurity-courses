Think of a network switch as the **smart traffic controller** of a local computer network.

While a home router connects your house to the outside internet, a network switch connects devices _inside_ your network—like
computers, printers, game consoles, and servers—so they can talk to each other.

Here is a guide to what a network switch is and exactly how it works.

---

## What is a Network Switch?

A network switch is a physical hardware device that operates at **Layer 2 (Data Link Layer)** of the OSI model. Its primary
job is to receive data from one connected device and forward it _only_ to the specific device that needs it.

Before switches became popular, networks used a device called a **hub**.

- **The Hub Problem:** When a hub received data, it screamed it out to _every single device_ connected to it. This was
  inefficient and caused a lot of data collisions.
- **The Switch Solution:** A switch is much smarter. It creates a temporary, direct "tunnel" between the sender and the
  receiver, keeping the rest of the network clear.

---

## How a Network Switch Works (Step-by-Step)

To direct traffic efficiently, a switch relies on physical addresses built into every device, known as **MAC (Media Access
Control) addresses**.

Here is the step-by-step process of how a switch handles data:

### 1. Learning (Building the MAC Address Table)

When you first plug devices into a switch, it doesn't know where anything is. It has an empty lookup table called a **MAC
Address Table**. As soon as a device (let's say, Computer A) sends a piece of data, the switch looks at the incoming packet,
notes Computer A’s MAC address, and records which physical port it is plugged into.

### 2. Flooding (If the Destination is Unknown)

If Computer A wants to talk to Computer B, the switch looks at its table to see where Computer B is. If the switch has never
heard from Computer B before, it doesn't know which port it's on. In this rare case, it will **flood** the data to every
single port except the one it came from.

### 3. Forwarding (The Smart Route)

When Computer B receives that flooded message, it replies, "Hey, that's me!" The switch immediately logs Computer B's MAC
address and port number into its table. From that moment on, whenever any device wants to send data to Computer B, the switch
**forwards** the data directly to Computer B's port. No more flooding.

### 4. Filtering

Because the switch knows exactly where data needs to go, it **filters** it out from all other ports. If Computer A is talking
to Computer B, Computers C and D can talk to each other at the exact same time on the same switch without their data getting
mixed up.

---

## Summary of Key Functions

To recap, a network switch performs three core duties:

- **Learning:** It reads the source MAC address of every incoming packet to map out the network.
- **Forwarding:** It sends the data directly to the correct destination port based on its map.
- **Loop Prevention:** It uses protocols (like Spanning Tree Protocol) to make sure data doesn't get trapped in infinite
  circles if switches are hooked up to each other incorrectly.
