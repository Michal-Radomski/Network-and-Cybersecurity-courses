A router is a fundamental networking device that acts as a traffic controller for data on the internet. It connects different
networks together—such as linking your home local network to the broader internet—and ensures that data packets find the most
efficient path to their destination.

Here is a breakdown of what a router is and how it interconnects Local Area Networks (LANs).

---

## What is a Router?

At its core, a router operates at the **Network Layer (Layer 3)** of the OSI (Open Systems Interconnect) model. Unlike a
switch, which connects devices _within_ a single network, a router connects **entirely separate networks**.

To do this, routers use **IP (Internet Protocol) addresses** to identify devices and networks, allowing them to forward data
across different network boundaries.

---

## How Routers Interconnect LANs

When you want to connect two or more distinct LANs (for example, connecting the Accounting department's LAN to the Sales
department's LAN, or connecting a school's LAN to the Internet), a router is required.

Here is step-by-step how they make that interconnection happen:

### 1. Breaking Broadcast Domains

Every LAN is a single "broadcast domain," meaning if a device sends out a broadcast message, every other device on that
specific LAN hears it. If you connected thousands of devices together without a router, the network would quickly bog down
from too much background chatter.

- Routers **do not forward broadcast traffic**. By placing a router between LANs, it keeps local traffic local, preventing
  network congestion.

### 2. Reading IP Addresses and Packet Headers

Data travels across networks in small chunks called **packets**. Each packet has a header containing the sender's IP address
and the destination's IP address.

- When a device on LAN A wants to send data to a device on LAN B, it realizes the destination IP is on a different network.
- The device sends the packet to its **Default Gateway** (which is simply the router's interface connected to LAN A).

### 3. Maintaining Routing Tables

Every router maintains a built-in database called a **Routing Table**. This table is a map of the network universe. It
contains lists of known network paths and instructions on where to send data next.

- When the router receives the packet from LAN A, it strips away the local delivery information (MAC addresses) and looks at
  the destination IP address.
- It checks its routing table to find which of its interfaces connects to LAN B (or which next router will get it closer to
  LAN B).

### 4. Forwarding the Data (Path Determination)

Once the router determines the best path, it repackages the data packet and sends it out through the specific physical port
connected to LAN B. If the destination is across the world, the packet might hop through multiple routers, each doing this
exact same check, until it reaches the final target LAN.

---

## Summary: Switch vs. Router in a LAN Environment

To visualize how a router fits into the picture, it helps to contrast it with a switch:

| Feature             | Switch                                    | Router                                         |
| ------------------- | ----------------------------------------- | ---------------------------------------------- |
| **Primary Job**     | Connects devices _within_ a single LAN.   | Connects multiple distinct LANs together.      |
| **OSI Layer**       | Data Link Layer (Layer 2).                | Network Layer (Layer 3).                       |
| **Addressing Used** | MAC Addresses (Physical device IDs).      | IP Addresses (Logical network IDs).            |
| **Analogy**         | An apartment building's internal hallway. | The postal service moving mail between cities. |
