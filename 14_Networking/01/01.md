Welcome to the world of computer networking! At its core, networking is just about getting two or more devices to talk to
each other, share resources, and pass data back and forth.

Think of it like the global postal system, but operating at the speed of light.

---

## 1. The Core Components

To build a network, you need a few essential pieces of hardware:

- **Clients/Hosts:** The devices using the network (like your phone, laptop, or smart TV).
- **Servers:** Powerful computers that "serve" data (like websites, video streams, or games) to the clients.
- **Routers:** The traffic cops of the internet. They look at where data is trying to go and direct it along the best path.
- **Switches:** These connect devices _within_ the same local network (like connecting all the computers in a single school
  lab).
- **Transmission Media:** The physical paths data travels through—either cables (Fiber-optic, Ethernet) or wireless signals
  (Wi-Fi, Cellular).

---

## 2. Types of Networks

Networks are generally categorized by how much geographic space they cover:

| Network Type | Full Name                 | Description                                           | Example                                   |
| ------------ | ------------------------- | ----------------------------------------------------- | ----------------------------------------- |
| **LAN**      | Local Area Network        | Covers a small, specific area.                        | Your home Wi-Fi or a school network.      |
| **WAN**      | Wide Area Network         | Connects smaller networks across cities or countries. | **The Internet** is the ultimate WAN.     |
| **MAN**      | Metropolitan Area Network | Covers a town or a specific city.                     | A smart-city traffic light camera system. |

---

## 3. How Data Travels: Packets and Protocols

Data doesn't travel as one giant file. If you download a photo, the network chops it up into tiny pieces called **packets**.

Each packet gets a digital "shipping label" containing the sender's address, the destination address, and instructions on how
to put the photo back together.

### IP Addresses

Every device on a network needs a unique identifier, known as an **IP (Internet Protocol) Address**.

- **IPv4:** The older, classic style (e.g., `192.168.1.1`).
- **IPv6:** The newer style created because the world ran out of IPv4 addresses (e.g., `2001:db8::ff00:42:8329`).

## 4. The Rules of the Road: The OSI Model

To make sure a Mac, a Windows PC, an iPhone, and a Linux server can all understand each other, networking relies on a
standardized 7-layer framework called the **OSI (Open Systems Interconnection) Model**.

It breaks down the journey of data from the screen to the physical wire:

1. **Application Layer:** Where you interact with the network (e.g., your web browser using HTTP/HTTPS).
2. **Presentation Layer:** Translates and encrypts data (e.g., SSL/TLS, JPEG conversion).
3. **Session Layer:** Opens, manages, and closes the connection between devices.
4. **Transport Layer:** Decides how to send the data (using **TCP** for reliable, perfect delivery, or **UDP** for fast, live
   streaming where a few lost drops don't matter).
5. **Network Layer:** Handles routing and IP addresses.
6. **Data Link Layer:** Handles physical addressing (MAC addresses) between devices on the same local network.
7. **Physical Layer:** The actual hardware—electrical pulses, fiber-optic light, or radio waves.
