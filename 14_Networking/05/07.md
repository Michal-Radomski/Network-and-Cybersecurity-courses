While they sound almost identical and are often used interchangeably, **URI** and **URL** have distinct meanings. The
simplest way to remember the difference is: **A URL is a specific type of URI.**

Think of a **URI** as a person's identity, and a **URL** as their home address.

---

## 1. What is a URI? (Uniform Resource Identifier)

A **URI** is a string of characters used to identify a resource (like a webpage, a document, an image, or a book) either by
its **name**, its **location**, or both.

URI is the "umbrella term" that covers two sub-categories: **URLs** and **URNs**.

```text
               +-----------------------------------+
               |      URI (Resource Identifier)     |
               +-----------------------------------+
                                 |
         +-----------------------+-----------------------+
         |                                               |
         v                                               v
+------------------+                            +------------------+
|  URL (Location)  |                            |   URN (Name)     |
|  How to find it  |                            |  What it is      |
+------------------+                            +------------------+

```

---

## 2. What is a URL? (Uniform Resource Locator)

A **URL** is a specific type of URI that identifies a resource by explaining **how to find it** on the internet. It provides
the exact network location or web address.

If it tells a browser how to fetch the resource (using protocols like `http`, `https`, or `ftp`), it is a URL.

- **Example:** `https://www.example.com/index.html`
- This is a URL because it identifies the resource and specifies _where_ it is located and _how_ to get it (`https`).

---

## 3. What is a URN? (Uniform Resource Name)

To fully understand the difference, you need to know the second sibling under the URI umbrella: the **URN**. A URN identifies
a resource by a **unique name** in a specific space, but it does _not_ tell you how to find it or where it lives.

- **Example:** `urn:isbn:9780131103628`
- This is a URN. It identifies a specific book (C Programming Language) by its unique ISBN number. It is a permanent
  identifier, but it doesn't tell you _where_ to buy or download the book.

---

## Key Differences Summary

| Feature           | URI                                               | URL                                                           |
| ----------------- | ------------------------------------------------- | ------------------------------------------------------------- |
| **Definition**    | Identifies a resource by name, location, or both. | Identifies a resource by its web location/address.            |
| **Scope**         | The broad, overarching category.                  | A specific subset of URI.                                     |
| **Component**     | Contains URLs, URNs, or both.                     | Contains the protocol (`http`, `ftp`), domain name, and path. |
| **Goal**          | To identify.                                      | To locate.                                                    |
| **Rule of Thumb** | **All URLs are URIs.**                            | **Not all URIs are URLs** (some are just URNs).               |

### Quick Analogy:

- **URI:** John Doe (Identifies the person).
- **URN:** John Doe's Social Security Number (Unique name/ID, doesn't change, but doesn't tell you where he is).
- **URL:** 123 Main Street, New York, NY (Tells you exactly how to find him).
