In networking, **PAT (Port Address Translation)** is a feature of NAT (Network Address Translation) that allows multiple
devices on a private local network to share a single public IP address (or a small pool of them) by differentiating the
traffic using **Layer 4 port numbers**.

Here is the breakdown of the two main types: Dynamic PAT and Static PAT.

---

## Dynamic PAT (Hide NAT / Many-to-One)

Dynamic PAT is the most common form of NAT. It is what your home router uses to connect all your smartphones, laptops, and
smart TVs to the internet using just the one public IP address provided by your Internet Service Provider (ISP).

- **How it works:** When an internal device wants to access the internet, the router changes the device's private source IP
  address to the router's public IP address. To keep track of which internal device sent the traffic, the router also assigns
  a unique **source port number** to that specific session.
- **Direction:** It is **unidirectional**. Connections must originate from the _inside_ network. Outside users on the
  internet cannot initiate a connection to an internal device because the port mappings are created dynamically and
  temporary.

> **Example:** > \* Laptop A ($192.168.1.5$) wants to visit a website. The router translates its IP to $203.0.113.1$ and
> assigns port `50001`.
>
> - Laptop B ($192.168.1.6$) visits a website at the same time. The router translates its IP to the same $203.0.113.1$ but
>   assigns port `50002`.
> - When the website responds, the router looks at the destination port (`50001` or `50002`) to know which laptop should
>   receive the data.

---

## Static PAT (Port Forwarding)

Static PAT maps a specific public IP address and port combination to a specific private IP address and port combination.

- **How it works:** Instead of creating a temporary mapping when someone goes online, a network administrator manually
  configures a permanent, fixed rule in the router or firewall.
- **Direction:** It is **bidirectional**, but it is primarily used to allow external users on the internet to initiate
  connections to a specific service hosted inside the private network.

> **Example:** > You are hosting a web server inside your home network at private IP $192.168.1.100$. Web traffic uses port
> `80`. You create a Static PAT rule stating that any traffic arriving from the internet at your public IP ($203.0.113.1$) on
> port `80` must be forwarded directly to $192.168.1.100$ on port `80`.

---

## Summary Comparison

| Feature             | Dynamic PAT                                  | Static PAT                                       |
| ------------------- | -------------------------------------------- | ------------------------------------------------ |
| **Common Name**     | Hide NAT, Many-to-One NAT                    | Port Forwarding                                  |
| **Mapping Type**    | Dynamic, temporary, created on-the-fly       | Permanent, manually configured                   |
| **Primary Purpose** | Allows internal users to browse the internet | Allows external users to access internal servers |
| **Initiation**      | Outbound only                                | Inbound (and outbound)                           |
| **Use Case**        | Home networks, office workstations           | Hosting web, mail, or gaming servers             |
