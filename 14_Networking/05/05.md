**DHCP (Dynamic Host Configuration Protocol)** is a network protocol used to automatically assign IP addresses and other
network configuration parameters to devices joining a network.

Without DHCP, you would have to manually enter an IP address, subnet mask, gateway, and DNS server every time you connected
your phone or laptop to a new Wi-Fi network.

---

## The 4-Step DHCP Process (DORA)

DHCP works using a simple four-step handshake, easily remembered by the acronym **DORA**: **D**iscover, **O**ffer,
**R**equest, **A**cknowledge.

```text
 [ Client ]                               [ DHCP Server ]
     |                                          |
     | ------- 1. DISCOVER (Broadcast) -------->|
     |                                          |
     | <------- 2. OFFER (Unicast) -------------|
     |                                          |
     | ------- 3. REQUEST (Broadcast) --------->|
     |                                          |
     | <------- 4. ACKNOWLEDGE (Unicast) -------|
     v                                          v

```

### 1. Discover (**D**)

When a device (the client) connects to a network, it doesn't have an IP address yet. It sends out a **DHCP Discover** message
to the entire network (a broadcast message) asking, _"Is there a DHCP server out there? I need an IP address."_

### 2. Offer (**O**)

Any DHCP server on the network receives the broadcast and responds with a **DHCP Offer** message. The server looks at its
pool of available IP addresses and tells the device, _"I have an address available for you. You can use `192.168.1.50` along
with these other network settings."_

### 3. Request (**R**)

Once the client receives the offer, it accepts it by sending back a **DHCP Request** message. This message is broadcast to
the whole network to let the chosen server know its offer was accepted, and to subtly let any other DHCP servers know their
offers were declined. The client says, _"Great, I'll take that `192.168.1.50` address you offered."_

### 4. Acknowledge (**A**)

Finally, the DHCP server sends a **DHCP Acknowledgment (ACK)** message. This confirms the lease of the IP address, finalizes
the configuration, and says, _"You're all set. The address is yours for the next 24 hours."_ The device can now officially
communicate on the network.

---

## Beyond the IP: What Else Does DHCP Provide?

A DHCP server doesn't just hand out an IP address. It provides a complete "network starter pack" that includes:

- **Subnet Mask:** Defines the size and boundaries of the local network.
- **Default Gateway:** The IP address of the router, telling the device how to send data outside the local network (to the
  internet).
- **DNS Servers:** The addresses of Domain Name System servers, allowing the device to translate website names (like
  `google.com`) into IP addresses.
- **Lease Time:** The duration for which the device is allowed to keep the assigned IP address.

---

## What is an IP Lease?

IP addresses assigned by DHCP are not permanent; they are leased.

- **Renewal:** When the lease time is halfway over (usually 50%), the client will automatically attempt to renew the lease
  with the DHCP server.
- **Expiration:** If a device leaves the network (like you leaving your house) and the lease expires, the DHCP server puts
  that IP address back into its pool to give to another device. This prevents the network from running out of addresses.
