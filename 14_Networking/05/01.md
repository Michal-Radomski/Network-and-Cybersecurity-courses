**NAT**, which stands for **Network Address Translation**, is a method used in networking to map one IP address space into
another. It essentially acts as a translator or a traffic cop between your local private network (like your home Wi-Fi) and
the public internet.

Here is a breakdown of why it is used and how the different types work.

---

## Why Do We Need NAT?

- **IP Address Conservation:** The older version of internet addresses (IPv4) has a limited number of unique addresses (about
  4.3 billion). Because the world ran out of these, NAT allows thousands of devices on a private network to share a single
  public IP address.
- **Security:** NAT hides the internal IP addresses of your devices from the public internet. External hackers can only see
  the router's public IP address, making it much harder to target specific devices directly.

---

## Types of NAT

There are three main types of Network Address Translation, each handling traffic a bit differently:

### 1. Static NAT

In Static NAT, a single **private** IP address is permanently mapped to a single **public** IP address. It is a one-to-one
relationship.

- **How it works:** Every time Device A communicates with the outside world, it always uses the exact same public IP address.
- **Best for:** Web servers, email servers, or devices inside a company network that need to be consistently accessible from
  the outside internet using a fixed address.

### 2. Dynamic NAT

Dynamic NAT maps an unregistered private IP address to a public IP address from a **pool of available public IP addresses**.
It is still a one-to-one relationship, but the pairing isn't permanent.

- **How it works:** When a device wants to access the internet, the router checks the "pool" of public IPs and assigns an
  unused one to that device. Once the device is done communicating, that public IP goes back into the pool for another device
  to use.
- **Downside:** If you have 10 public IP addresses in your pool and 11 devices try to access the internet at the exact same
  time, the 11th device will be blocked until someone else disconnects.

### 3. PAT (Port Address Translation) / NAT Overload

This is the most common type of NAT and is almost certainly what your home Wi-Fi router uses. It maps **multiple private IP
addresses to a single public IP address** by using different **port numbers**. It is a many-to-one relationship.

- **How it works:** When your phone and your laptop both visit a website at the same time, they share the exact same public
  IP address. To keep the traffic separate, the router assigns a unique "port number" to each connection.
- _Example:_ Your phone's data might be sent out on Port 2001, while your laptop's data goes out on Port 2002. When the
  website replies, the router looks at the port number to figure out which device should get the data.

---

## Summary Comparison

| NAT Type           | Relationship                           | Public IP Requirement                         | Common Use Case                    |
| ------------------ | -------------------------------------- | --------------------------------------------- | ---------------------------------- |
| **Static NAT**     | 1 Private to 1 Public (Fixed)          | High (Needs a dedicated public IP per server) | Hosting a company web server       |
| **Dynamic NAT**    | 1 Private to 1 Public (From a pool)    | Medium (Needs a small block of public IPs)    | Internal corporate networks        |
| **PAT (Overload)** | Many Private to 1 Public (Using ports) | Very Low (Only needs 1 public IP total)       | Home networks and small businesses |
