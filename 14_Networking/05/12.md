**DNS caching** is a temporary storage mechanism used by devices and network systems to save the results of recent DNS
lookups.

Instead of forcing your computer to journey across the global internet to ask Root, TLD, and Authoritative servers for a
website's IP address every single time you click a link, DNS caching keeps a quick "cheat sheet" locally. This makes web
browsing incredibly fast and drastically reduces global internet traffic.

---

## The Layers of DNS Caching

DNS caching happens at multiple levels between your web browser and the authoritative servers. When you type in a URL, your
computer checks these caches in order:

```text
[Browser Cache] ➔ [OS Cache] ➔ [Router Cache] ➔ [Resolver (ISP) Cache]

```

### 1. Browser Cache

Modern browsers (like Chrome, Firefox, or Safari) maintain their own internal DNS cache. Because it is built right into the
app you are using, this is the absolute fastest lookup layer.

### 2. Operating System (OS) Cache

If the browser doesn't have the record, it makes a system call to your computer's OS (Windows, macOS, or Linux). The OS
maintains its own local cache of recent network connections.

### 3. Router Cache

If your computer doesn't know the IP, it sends the request to your home Wi-Fi router. Routers often maintain a small DNS
cache for all the devices connected to your home network.

### 4. Recursive Resolver (ISP) Cache

If the router doesn't have it, the request goes to your Internet Service Provider's DNS resolver (or public ones like
Cloudflare or Google). These resolvers handle millions of requests, meaning their caches are massive. If someone else down
your street visited `example.com` five minutes ago, the resolver will instantly serve that cached IP address to you.

---

## How Long Do Records Stay Cached? (TTL)

Caches cannot store IP addresses forever. Websites change servers, move hosting providers, or update their security
configurations, which changes their IP addresses. If a cache kept an old IP indefinitely, you would get a "Website Not Found"
error.

To solve this, every DNS record includes a **TTL (Time to Live)** value, set by the website owner.

- **What it is:** TTL is a countdown timer measured in seconds.
- **How it works:** If a website sets a TTL of `3600` (1 hour), any DNS server or device that caches that IP address will
  only keep it for 60 minutes. Once the timer hits zero, the record is marked as expired ("stale"), and the next request will
  trigger a fresh, live lookup from the authoritative servers.

---

## Clearing the Cache ("Flushing")

Sometimes, a website changes its IP address, but your computer is still trying to use the old, cached IP because the TTL
hasn't expired yet. This causes the site to fail to load.

To fix this, you can manually force your Operating System to clear out its memory, a process called **flushing your DNS**.

- **On Windows:** Open Command Prompt and type `ipconfig /flushdns`
- **On macOS:** Open Terminal and type `sudo killall -HUP mDNSResponder`

This wipes the local cheat sheet clean, forcing your computer to fetch the most up-to-date IP address on its next request.
