The **13 root DNS servers** are the foundational servers at the very top of the Domain Name System (DNS) hierarchy. They are
named using the letters **A through M** (e.g., `a.root-servers.net`, `b.root-servers.net`, up to `m.root-servers.net`).

Despite their names, there aren't actually just 13 physical computers in the world handling this traffic. Because of a
technical technique called **Anycast routing**, those 13 unique IP addresses are actually shared across **over 1,500 physical
server locations** spread across the globe.

---

## Why Exactly 13?

The limit of 13 names comes down to an old technical constraint in early internet design. Originally, DNS data had to fit
inside a single network packet using the UDP protocol, which had a strict limit of **512 bytes**. Fitting the network
addresses, server names, and protocol overhead neatly allowed for exactly 13 server addresses.

---

## What is the Difference Between A, B, C, and the Rest?

From a functional standpoint, **there is no difference in the data they provide.** Every single root server contains the
exact same authoritative list of Top-Level Domains (like `.com`, `.org`, `.uk`).

However, they differ significantly in three main ways behind the scenes:

### 1. The Operators (Administrative Diversity)

To prevent a single company or government from controlling the entire internet, the 13 addresses are managed by a mix of
independent universities, government agencies, and non-profit organizations.

- **A** & **J:** Verisign, Inc.
- **B:** Information Sciences Institute (USC)
- **C:** Cogent Communications
- **D:** University of Maryland
- **E:** NASA (Ames Research Center)
- **F:** Internet Systems Consortium (ISC)
- **G:** US Department of Defense (DISA)
- **H:** US Army Research Lab
- **I:** Netnod (Sweden)
- **K:** RIPE NCC (Europe/Middle East)
- **L:** ICANN
- **M:** WIDE Project (Japan)

### 2. Software & Hardware ("Genetic" Diversity)

If all 13 root servers ran identical software on the exact same operating system, a single software bug or hack could
theoretically take down the entire internet at once.

To prevent this, the different operators purposefully use a diverse mix of hardware and different DNS software applications
(such as BIND, NSD, and Knot DNS). If a critical bug is found that crashes the software running on the **A** server, the
**K** server will likely be completely unaffected because it runs a different code base.

### 3. Physical Locations and Routing

While you might send a request to the **I** root server (managed in Sweden) or the **M** root server (managed in Japan),
Anycast routing ensures that your request doesn't actually travel around the world. It automatically routes your computer's
request to the closest physical copy of that server.

---

## Summary: Which one does your computer use?

Your computer or local network provider chooses which root server to talk to based entirely on **network speed and
proximity**. It will naturally favor whichever root server letter has a physical location closest to your location at that
exact millisecond.
