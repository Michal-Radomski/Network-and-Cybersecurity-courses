**Reverse DNS (rDNS)** is exactly what it sounds like—the Domain Name System operating in reverse.

While standard (forward) DNS looks up a domain name (like `google.com`) to find its IP address, Reverse DNS looks up an **IP
address** to find its associated domain name or hostname.

---

## How Reverse DNS Works (The Pointer Record)

In standard DNS, you use an **A Record** (or AAAA for IPv6) to map a name to an IP. In Reverse DNS, a special type of DNS
record called a **PTR (Pointer) Record** is used to map the IP back to the name.

Because DNS is designed to search by text names from right to left (like `.com` $\rightarrow$ `example` $\rightarrow$ `www`),
searching for a raw IP address natively doesn't work. To fix this, rDNS uses a clever trick involving a special, hidden
domain:

### For IPv4: `in-addr.arpa`

To perform a reverse lookup, the IP address is **reversed**, and the suffix `.in-addr.arpa` is attached to the end.

- **The IP address:** `192.0.2.55`
- **The Reverse DNS lookup query:** `55.2.0.192.in-addr.arpa`

By reversing the numbers, the DNS system can read the IP address hierarchically, just like a regular domain name, allowing it
to easily find the authoritative server that holds the PTR record.

---

## Why is Reverse DNS Used?

You rarely use Reverse DNS when simply browsing the web, but it is incredibly important for backend server management,
security, and verification.

### 1. Anti-Spam Verification (Email)

This is the most common use case for rDNS. When an email server receives a message from an IP address claiming to be
`mail.bank.com`, it will run a Reverse DNS lookup on that IP address.

- If the PTR record matches `mail.bank.com`, the email is accepted.
- If the PTR record is empty, or points to something completely different (like a hacked home internet connection), the
  receiving server will block the email or mark it as spam.

### 2. Security and Logging

Network administrators and security tools use rDNS to make sense of traffic logs. Looking at a log filled with thousands of
raw IP addresses is difficult to analyze. rDNS automatically translates those IPs into recognizable hostnames, making it much
easier to spot unusual patterns or malicious actors.

### 3. System Diagnostics

Network tools like `traceroute` use Reverse DNS to show you the human-readable names of the routers your data packets are
bouncing through as they travel across the internet.

## Quick Comparison

| Feature                 | Forward DNS                          | Reverse DNS (rDNS)                               |
| ----------------------- | ------------------------------------ | ------------------------------------------------ |
| **Direction**           | Domain Name $\rightarrow$ IP Address | IP Address $\rightarrow$ Domain Name             |
| **Primary Record Type** | **A** (IPv4) / **AAAA** (IPv6)       | **PTR** (Pointer Record)                         |
| **Main Use Case**       | Helping humans load websites.        | Helping servers verify identity and filter spam. |
| **Example Query**       | `google.com`                         | `55.2.0.192.in-addr.arpa`                        |
