A **NAT (Network Address Translation) table** is a database maintained by a router or firewall. It acts like a routing
directory that allows multiple devices on a private local network (like your home Wi-Fi) to share a single public IP address
to communicate with the internet.

Because private IP addresses (e.g., `192.168.1.X`) cannot be routed over the public internet, the router uses the NAT table
to map internal addresses to public ones and keep track of where internet traffic needs to go.

---

## How a NAT Table Works

When a device on a local network wants to access a website, the router intercepts the request and performs the following
steps:

1. **Translation:** The router replaces the device's private IP address and source port with its own public IP address and a
   unique assigned port.
2. **Recording:** The router records this mapping (Private IP + Private Port $\leftrightarrow$ Public IP + Public Port) as an
   entry in the **NAT table**.
3. **Forwarding:** The router sends the request out to the internet.
4. **Receiving:** When the website responds, it sends data back to the router's public IP and the unique assigned port.
5. **Lookup & Delivery:** The router looks at the NAT table to see which private IP and port match that specific public port,
   translates it back, and forwards the data to the correct local device.

---

## Example of a NAT Table

If you are browsing a website on your laptop while someone else is gaming on a console, a simplified **PAT (Port Address
Translation)** version of a NAT table—the most common type of NAT—looks like this:

| Inside Local (Private Device)     | Inside Global (Router's Public Face) | Outside Global (Destination/Website)  | Protocol |
| --------------------------------- | ------------------------------------ | ------------------------------------- | -------- |
| **192.168.1.10** : 5021 (Laptop)  | **203.0.113.1** : 10001              | **93.184.216.34** : 443 (Web Server)  | TCP      |
| **192.168.1.15** : 6100 (Console) | **203.0.113.1** : 10002              | **198.51.100.5** : 3074 (Game Server) | UDP      |

### Key Terms in a NAT Table:

- **Inside Local:** The actual private IP address and port of the device on your home network.
- **Inside Global:** The public IP address assigned by your Internet Service Provider (ISP) and the unique port the router
  chose for that specific connection.
- **Outside Global:** The public IP address and port of the destination server on the internet (like Google, Netflix, or a
  gaming server).

---

## Why is the NAT Table Important?

- **IP Address Conservation:** It allows thousands of devices to share one public IPv4 address, which delayed the exhaustion
  of the IPv4 address space.
- **Security:** It acts as a natural barrier. Because external servers only see the router's public IP, outsiders cannot
  directly initiate a connection to a private device inside the network unless an explicit entry exists in the NAT table.
- **State Management:** NAT entries are dynamic. Once a connection is closed or times out (e.g., you close the browser tab),
  the router removes the entry from the table to free up that public port for other connections.
