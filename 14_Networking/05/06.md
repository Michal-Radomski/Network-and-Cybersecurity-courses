## DHCP Ports

DHCP relies on the **UDP (User Datagram Protocol)** transport protocol because it needs to send fast, connectionless
broadcast messages before a device even has an IP address.

It uses two specific, dedicated ports:

- **UDP Port 67:** Used by the **DHCP Server** to listen for requests and send data to clients.
- **UDP Port 68:** Used by the **DHCP Client** to listen for responses from the server.

## How the IP Lease Renewal Works

A device doesn't wait until its IP lease completely runs out to ask for a new one. If it did, it might temporarily lose its
internet connection. Instead, it goes through a quiet, two-stage renewal process while you are using the network.

Here is exactly how it looks:

### Stage 1: The T1 Timer (50% of Lease Time)

When **50%** of the lease time has passed (for example, after 12 hours of a 24-hour lease), the client enters the **Renewing
State**.

1. **DHCP Request (Unicast):** The client sends a _DHCP Request_ directly to the specific DHCP server that originally gave it
   the IP address. It says, _"Hey, my lease is half over. Can I keep this IP address longer?"_
2. **DHCP Acknowledgment (ACK):** If the server is online and available, it replies directly with a _DHCP ACK_. This resets
   the lease clock back to 100%, and the process is complete.

### Stage 2: The T2 Timer (87.5% of Lease Time)

If the original DHCP server was offline or down at the 50% mark, the client keeps using its IP address but tries again when
**87.5%** of the lease has passed (21 hours into a 24-hour lease). This is the **Rebinding State**.

1. **DHCP Request (Broadcast):** Since the original server didn't answer before, the client now broadcasts a _DHCP Request_
   to the _entire network_. It says, _"Can ANY DHCP server out there let me keep my current IP address?"_
2. **DHCP Acknowledgment (ACK):** Any available DHCP server on the network can step in, verify the address is valid, and send
   a _DHCP ACK_ to renew the lease.

### What if it completely expires?

If 100% of the time passes and no server has responded, the client must immediately stop using that IP address. It drops off
the network and starts the entire **DORA** process over from scratch to get a completely new IP address.
