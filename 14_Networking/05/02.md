In networking, **NAT (Network Address Translation)** is a method used to rewrite the source or destination IP addresses of an
IP packet while it's in transit through a router or firewall.

The primary difference between **Source NAT (SNAT)** and **Destination NAT (DNAT)** depends on which part of the packet
header is being modified and the direction of the traffic.

---

## 1. Source NAT (SNAT)

**Source NAT** changes the _source_ IP address of a packet.

- **When it's used:** Typically when a device on a private local network (LAN) needs to initiate a connection to a device on
  a public network (like the Internet).
- **How it works:** Private IP addresses (e.g., `192.168.1.X`) are not routable on the public internet. When you try to
  access a website, your router changes your private source IP to its own public IP address before sending the request out.
  When the website replies, the router remembers who made the request and passes the data back to your device.
- **Common Example:** Home internet routers use a specific type of SNAT called **PAT (Port Address Translation)** or "NAT
  Overload," allowing hundreds of local devices to share a single public IP address.

---

## 2. Destination NAT (DNAT)

**Destination NAT** changes the _destination_ IP address of a packet.

- **When it's used:** Typically when an external device on the internet needs to initiate a connection to a specific device
  inside a private network.
- **How it works:** An external user sends a request to a public IP address. When the router/firewall receives this packet,
  it changes the destination IP to the internal, private IP address of the target server and forwards the traffic inward.
- **Common Example:** **Port Forwarding**. If you host a gaming server or a web server at home, you configure DNAT so that
  any outside traffic hitting your public IP on a specific port gets redirected straight to your specific internal server.

---

## Summary Comparison

| Feature            | Source NAT (SNAT)                                                          | Destination NAT (DNAT)                                              |
| ------------------ | -------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| **What changes?**  | The **Source** IP address                                                  | The **Destination** IP address                                      |
| **Who initiates?** | The **Internal** client (going out)                                        | The **External** client (coming in)                                 |
| **Primary Goal**   | To let internal devices browse the internet safely using fewer public IPs. | To expose an internal service (like a server) to the outside world. |
| **Everyday Term**  | Outbound NAT / IP Masquerading                                             | Port Forwarding / Port Mapping                                      |
