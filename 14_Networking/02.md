When networking professionals talk about how data moves across the internet, they rely on two primary architectural
frameworks: the **OSI (Open Systems Interconnection) Model** and the **TCP/IP Model**.

Think of them as blueprints for the internet. While they share a lot of similarities, they approach the concept of networking
from slightly different angles.

---

## 1. The OSI Model (The Theoretical Standard)

Developed by the International Organization for Standardization (ISO) in 1984, the OSI model is a **seven-layer** conceptual
framework. It was designed to be a universal standard, breaking down network communication into highly specific, distinct
steps.

Here are the 7 layers, from top to bottom:

- **Layer 7: Application** – Where the user interacts with the network (e.g., HTTP, FTP, SMTP).
- **Layer 6: Presentation** – Translates, encrypts, and compresses data so the application layer can understand it (e.g.,
  SSL/TLS, JPEG).
- **Layer 5: Session** – Manages and controls the connections (sessions) between local and remote applications.
- **Layer 4: Transport** – Manages end-to-end communication, flow control, and error detection (e.g., TCP, UDP).
- **Layer 3: Network** – Handles the routing of data packets across different networks using logical addressing (e.g., IP
  addresses).
- **Layer 2: Data Link** – Manages the physical node-to-node transfer of data and handles physical addressing (e.g., MAC
  addresses, Ethernet switches).
- **Layer 1: Physical** – The actual physical hardware, cables, and electrical signals that transmit raw bitstreams (e.g.,
  fiber optic cables, Wi-Fi radio waves).

---

## 2. The TCP/IP Model (The Practical Standard)

Developed by the US Department of Defense (DARPA) in the 1970s, the TCP/IP model is older than the OSI model and is the
**actual foundation of the modern internet**. It is a more practical, streamlined framework consisting of **four layers**.

- **Application Layer** – Combines OSI's Application, Presentation, and Session layers. It handles high-level protocols and
  user interaction.
- **Transport Layer** – Directly matches the OSI Transport layer, managing host-to-host communication and data reliability
  (TCP/UDP).
- **Internet Layer** – Equivalent to the OSI Network layer. It handles packet routing and addressing across networks (IP).
- **Network Access Layer** (or Link Layer) – Combines OSI's Data Link and Physical layers. It deals with how data is
  physically sent over the network hardware.

---

## Comparison: OSI vs. TCP/IP

While both models use a layered approach to handle data encapsulation (wrapping data with protocol headers at each step),
they differ significantly in their execution.

| Feature              | OSI Model                                                           | TCP/IP Model                                                        |
| -------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------- |
| **Development**      | Theoretical blueprint created _before_ protocols were designed.     | Practical framework built _around_ existing protocols (TCP and IP). |
| **Number of Layers** | 7 Layers                                                            | 4 Layers                                                            |
| **Approach**         | **Vertical** and rigid; strict separation of duties between layers. | **Horizontal** and flexible; looser grouping of functions.          |
| **Usage**            | Used primarily for teaching, troubleshooting, and conceptualizing.  | The actual operational standard used by modern internet devices.    |
| **Protocols**        | Protocols are hidden and can be easily replaced.                    | Protocols are the core foundation of the model.                     |
