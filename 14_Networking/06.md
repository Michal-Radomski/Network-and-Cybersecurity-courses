In computer networking, data can't just be dumped onto a wire or sent through the air all at once. It has to be broken down,
wrapped up, and managed.

A **Protocol Data Unit (PDU)** is the specific form that a block of data takes at any given layer of a networking model (like
the OSI or TCP/IP models).

As your data travels down through the network layers to be sent, or up through the layers when received, it changes its
"clothes" (headers and footers). At each stage, this package gets a different name—a different PDU.

---

## The "Mailing a Letter" Analogy

Think of a PDU like sending a letter through the mail:

1. **The Letter:** You write a message (Data).
2. **The Envelope:** You put it in an envelope and write the recipient's name and address on it (Segments/Packets).
3. **The Mail Truck:** The envelope is put into a shipping crate with other letters heading to the same city (Frames).
4. **The Road:** The crate is moved along the highway (Bits).

Each of these stages represents a different PDU.

---

## PDUs Across the OSI Model

The **OSI Model** breaks networking down into 7 layers. The most common PDUs you will hear about exist from Layer 4 down to
Layer 1.

### 1. Data (Layers 5, 6, 7 - Application, Presentation, Session)

At the top layers, the PDU is simply called **Data**. This is the raw information your app creates, like an email message, a
webpage request, or a streaming video chunk.

### 2. Segment (Layer 4 - Transport)

When the data moves down to the Transport Layer (using protocols like TCP), it is broken into manageable pieces. The layer
adds a header containing port numbers (so the receiving computer knows which app should get the data).

- **PDU Name:** **Segment** (or _Datagram_ if using UDP).

### 3. Packet (Layer 3 - Network)

The segment moves down to the Network Layer (where IP routing happens). This layer adds a header containing the **Source IP
address** and **Destination IP address**.

- **PDU Name:** **Packet**.

### 4. Frame (Layer 2 - Data Link)

The packet moves to the Data Link Layer (your network card, Wi-Fi, or Ethernet). This layer wraps the packet by adding a
header with the physical **MAC addresses** of the hardware and a "trailer" (footer) at the end to check for data errors.

- **PDU Name:** **Frame**.

### 5. Bits (Layer 1 - Physical)

Finally, the frame is converted into raw electrical impulses, light pulses, or radio waves to travel across a physical wire
or through the air.

- **PDU Name:** **Bits**.

---

## Summary of the PDU Life Cycle

| OSI Layer   | Layer Name | PDU Name               | Key Information Added                          |
| ----------- | ---------- | ---------------------- | ---------------------------------------------- |
| **Layer 4** | Transport  | **Segment** / Datagram | Source/Destination Ports                       |
| **Layer 3** | Network    | **Packet**             | Source/Destination IP Addresses                |
| **Layer 2** | Data Link  | **Frame**              | Source/Destination MAC Addresses & Error Check |
| **Layer 1** | Physical   | **Bits**               | Raw binary (0s and 1s)                         |

This process of wrapping the data in headers as it moves down is called **Encapsulation**. When the receiving computer gets
the bits, it does the reverse (**Decapsulation**), stripping away the headers layer by layer until it is left with the
original Data.
