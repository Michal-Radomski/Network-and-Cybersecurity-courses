The **Transport Layer** is Layer 4 of the OSI model and Layer 3 of the TCP/IP model.

If the Network Layer (IP) is responsible for getting a package from one _building_ to another, the Transport Layer is
responsible for getting that package to the exact _person_ inside the building who ordered it, and making sure it arrived
intact.

It manages **end-to-end communication**, error-checking, and data flow between specific applications running on different
hosts.

---

## The Two Titans of the Transport Layer

The Transport Layer relies almost entirely on two contrasting protocols. Depending on what an application needs (speed vs.
accuracy), it will choose one of these two:

### 1. TCP (Transmission Control Protocol) – _The Certified Mail_

TCP is a **connection-oriented** protocol. It prioritizes reliability and accuracy over absolute speed. Before sending any
data, it establishes a formal connection with the destination.

- **Used for:** Web browsing (HTTP/HTTPS), email (SMTP), file transfers (FTP), and remote access (SSH). If a single bit is
  lost in these applications, the final data becomes corrupted.

### 2. UDP (User Datagram Protocol) – _The Regular Postcard_

UDP is a **connectionless**, "best-effort" protocol. It prioritizes speed and efficiency over reliability. It fires packets
at the destination without checking if the receiver is ready or if the data actually arrives.

- **Used for:** Video streaming (Netflix/YouTube live streams), online gaming, VoIP (voice calls), and DNS queries. In these
  applications, a dropped packet just means a brief pixel glitch or audio skip—speed is what matters.

## Key Responsibilities of the Transport Layer

To manage how data is moved, the Transport Layer performs four critical functions:

### 1. Port Numbers (Tracking the Applications)

A single computer can run a web browser, a game, and a music stream simultaneously. The Transport Layer uses **Port Numbers**
to direct incoming data to the correct application.

- **Source Port:** A random number generated by your device to track your specific request.
- **Destination Port:** A standardized "well-known" port that tells the server what service you want (e.g., Port `80` or
  `443` for web traffic, Port `25` for email).

### 2. Segmentation and Reassembly

Data from applications can be massive (like a 5GB file). The Transport Layer breaks this large data down into smaller,
manageable chunks called **Segments** (in TCP) or **Datagrams** (in UDP) so they can travel across the network. On the
receiving end, it pieces them back together in the correct order.

### 3. Connection Establishment (The 3-Way Handshake)

Exclusive to TCP, devices must agree to talk before sending data. They do this using a process called the **Three-Way
Handshake**:

```text
[ Client ]                             [ Server ]
    |                                      |
    |  ---- SYN (Let's synchronize) ---->  |
    |                                      |
    |  <--- SYN-ACK (I agree, do you?) --  |
    |                                      |
    |  ---- ACK (Yes, let's go!) --------> |

```

### 4. Flow Control and Error Recovery

Also exclusive to TCP, the Transport Layer ensures a fast sender doesn't overwhelm a slow receiver.

- **Flow Control (Windowing):** The receiver tells the sender, _"Hey, my buffer is getting full, slow down your transmission
  speed."_
- **Error Recovery:** If a segment is corrupted or dropped in transit, the receiver notes the missing sequence number and
  asks the sender to retransmit that specific chunk.

## OSI vs. TCP/IP: Is there a difference at this layer?

While the OSI model and the TCP/IP model treat other layers differently, they are perfectly aligned here:

[Image comparing the OSI model 7 layers to the TCP/IP model 4 layers, highlighting the Transport Layer in both]

- In the **OSI Model**, the Transport Layer sits cleanly as **Layer 4**, acting as the bridge between the lower
  hardware-focused layers (Physical, Data Link, Network) and the upper software-focused layers (Session, Presentation,
  Application).
- In the **TCP/IP Model**, it is called either the **Transport Layer** or the **Host-to-Host Layer**. Its functional
  responsibilities are exactly the same.
