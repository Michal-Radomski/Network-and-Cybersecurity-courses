A **routing table** is a digital database or data file stored in a router or a network-connected computer. It lists the paths
(routes) to various network destinations and acts like a map or a GPS for data packets, telling the device exactly where to
send data next so it can reach its final destination.

Here is a breakdown of how it works, what it contains, and how it handles traffic.

---

## How a Routing Table Works

When a data packet arrives at a router, the router examines the packet's **destination IP address**. It then cross-references
this address with its routing table to find the best match.

1. **The Match:** The router looks for the most specific network address that matches the destination IP.
2. **The Forwarding:** Once a match is found, the router forwards the packet out of the specific interface or to the next-hop
   router indicated by the table.
3. **The Drop (if no match):** If there is no specific match, the router sends the packet to a "Default Gateway." If no
   default gateway is configured, the packet is dropped.

---

## Core Components of a Routing Table

A standard routing table consists of several key columns of information:

| Component                 | Description                                                                                                                                  | Example                    |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------- |
| **Network Destination**   | The IP address of the destination network or host.                                                                                           | `192.168.1.0`              |
| **Subnet Mask (Netmask)** | Defines the size of the destination network, helping the router distinguish the network portion of the IP from the host portion.             | `255.255.255.0` (or `/24`) |
| **Gateway / Next Hop**    | The IP address of the next router along the path. If the network is directly connected, this usually reads "On-link" or shows `0.0.0.0`.     | `10.0.0.1`                 |
| **Interface**             | The local physical or virtual network card (NIC) the router must use to send the data out.                                                   | `eth0` or `wlan0`          |
| **Metric**                | A cost value assigned to the route. If multiple paths exist to the same destination, the router chooses the path with the **lowest** metric. | `20`                       |

---

## How Routes Get into the Table

Routing tables don't just appear out of nowhere; they are built using three types of routes:

- **Directly Connected Routes:** These are networks physically attached to the router's interfaces. The router discovers them
  automatically as soon as the interface is configured and turned on.
- **Static Routes:** These are paths manually configured by a network administrator. They are highly reliable and don't
  consume network bandwidth, but they don't adapt automatically if the network layout changes.
- **Dynamic Routes:** These are discovered automatically by routers using **Routing Protocols** (like OSPF, BGP, or EIGRP).
  Routers talk to each other, share layout updates, and dynamically update their tables if a link goes down or a better path
  opens up.

---

## The Default Route (`0.0.0.0/0`)

One of the most important entries in a routing table is the **Default Route**, often listed as destination `0.0.0.0` with a
netmask of `0.0.0.0`.

Think of this as the "catch-all" or the "exit ramp to the Internet." If a packet is trying to reach a website across the
world, the local router won't have a specific entry for that exact website. Instead, it matches the default route and sends
the packet to the Internet Service Provider’s (ISP) router, which has a much larger routing table to handle the traffic from
there.
