To get the same output on Linux that you get from the Windows `netstat` command, you have a couple of great options.

While Linux _does_ have a traditional `netstat` command, it has been deprecated for years in favor of a modern, much faster
tool called **`ss`** (socket statistics).

Here is how to match the most common Windows `netstat` behaviors on Linux:

### 1. The Standard Windows Equivalent (`netstat -an` or `-ano`)

In Windows, you usually run `netstat -an` to see all active connections and listening ports numerically, or `-ano` to include
the Process ID (PID).

On Linux, the ultimate equivalent is:

```bash
ss -tunapl

```

**What those flags mean:**

- **`-t`**: Displays **TCP** sockets.
- **`-u`**: Displays **UDP** sockets.
- **`-n`**: Shows **numeric** port numbers and IP addresses (e.g., `80` instead of `http`).
- **`-a`**: Shows **all** sockets (both listening and established).
- **`-p`**: Shows the **process** name and PID using the socket (requires `sudo` for full details).
- **`-l`**: Shows **listening** sockets.

> 💡 **Note:** You will need to run this with **`sudo`** if you want to see the Process IDs and names (the `-p` flag), just
> like opening an Administrator Command Prompt in Windows.

---

### 2. Quick Cheat Sheet: Windows vs. Linux

| If you use this in Windows... | Use this in Linux (`ss`) | What it does                                    |
| ----------------------------- | ------------------------ | ----------------------------------------------- |
| `netstat`                     | `ss`                     | Shows established connections                   |
| `netstat -a`                  | `ss -a`                  | Shows all connections (listening & established) |
| `netstat -n`                  | `ss -n`                  | Shows addresses and ports as numbers            |
| `netstat -o`                  | `ss -p`                  | Shows which PID/Process is using the port       |
| `netstat -p tcp`              | `ss -t`                  | Filters for TCP only                            |

---

### 3. The Old-School Way (Literal `netstat`)

If your Linux system still has the old `net-tools` package installed, you can actually just run `netstat` directly using
almost the exact same flags as Windows:

```bash
sudo netstat -tunap

```

However, if you get a `command not found` error, it just means your system relies on `ss` now. Stick with `ss`—it's faster
and installed by default on almost every modern Linux distribution (Ubuntu, Debian, Fedora, CentOS, etc.).
