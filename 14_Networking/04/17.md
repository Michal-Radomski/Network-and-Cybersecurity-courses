**Dynamic Routing Protocols** are the software languages and rules that routers use to communicate with one another. Instead
of a human manually typing paths into a routing table, these protocols allow routers to automatically map out the network,
share information about available paths, and calculate the most efficient route for data packets.

If a network link goes down or a new router is added, dynamic routing protocols instantly recalculate the path, ensuring the
network automatically heals itself without human intervention.

---

## How Dynamic Routing Protocols Work

Dynamic routing relies on a continuous three-step cycle:

```text
[ Discover Neighbors ] ---> [ Share Network Info ] ---> [ Calculate Best Paths ]
                                   ^                            |
                                   |____________________________|
                                     (Repeats if topology changes)

```

1. **Discovery:** Routers send out periodic "Hello" packets to find other routers directly connected to them (neighbors).
2. **Sharing:** Routers share their known networks and path details with their neighbors.
3. **Choosing (The Metric):** When a router learns multiple ways to get to the same destination, it uses a **Metric** (a cost
   value based on distance, speed, or reliability) to determine the absolute best path and inserts it into the routing table.

## Classification of Routing Protocols

Routing protocols are broadly categorized based on **where** they operate and **how** they calculate paths.

### 1. By Location: IGP vs. EGP

Networks are broken down into **Autonomous Systems (AS)**, which are collections of routers under a single administrative
control (like a corporate network or an ISP).

- **Interior Gateway Protocols (IGP):** Used to route data _inside_ a single company or organization. Examples: OSPF, EIGRP,
  RIP.
- **Exterior Gateway Protocols (EGP):** Used to route data _between_ different Autonomous Systems across the global internet.
  There is only one modern EGP: **BGP**.

### 2. By Behavior: Distance-Vector vs. Link-State

IGPs are further split by the algorithm they use to map the network:

#### Distance-Vector Protocols

Routers using distance-vector protocols do not know the full map of the entire network. Instead, they pass their entire
routing tables to their immediate neighbors at fixed intervals. It is often described as "routing by rumor."

- **Metric:** Usually based on **Hop Count** (how many routers a packet must cross).
- **Analogy:** Following road signs. You don't see the whole map; you just trust the sign that says "New York: 50 miles this
  way."
- **Examples:** RIP (Routing Information Protocol).

#### Link-State Protocols

Routers using link-state protocols build a complete, synchronized 3D map (topology database) of the entire network area.
Every router runs an algorithm called **SPF (Shortest Path First)** or Dijkstra's Algorithm to independently calculate the
absolute fastest path to every destination.

- **Metric:** Based on **Cost** (which correlates to link speed/bandwidth).
- **Analogy:** Using a GPS map. The router can see every road and traffic condition before making a decision.
- **Examples:** OSPF (Open Shortest Path First), IS-IS.

#### Hybrid Protocols

Some protocols combine the best of both worlds. For example, Cisco's **EIGRP** (Enhanced Interior Gateway Routing Protocol)
acts like a distance-vector protocol but uses advanced calculations (bandwidth, delay, load) and triggers updates instantly
only when changes happen, making it as fast as a link-state protocol.

## The Major Dynamic Routing Protocols

Here is a summary of the most common protocols used in networking today:

| Protocol                                   | Type                  | Metric Used                    | Best Used For                                                                   |
| ------------------------------------------ | --------------------- | ------------------------------ | ------------------------------------------------------------------------------- |
| **RIP** _(Routing Information Protocol)_   | IGP / Distance-Vector | Hop Count (Max 15 hops)        | Legacy system; very small, simple networks.                                     |
| **OSPF** _(Open Shortest Path First)_      | IGP / Link-State      | Cost (Inverse of Bandwidth)    | Enterprise internal networks of almost any size. Very fast convergence.         |
| **EIGRP** _(Enhanced Interior Gateway...)_ | IGP / Advanced Hybrid | Composite (Bandwidth + Delay)  | Fast internal networks, traditionally heavily favored in Cisco environments.    |
| **BGP** _(Border Gateway Protocol)_        | EGP / Path-Vector     | Path Attributes (Policy-based) | **The Internet.** Connects massive ISPs and global corporate networks together. |

## Key Terminology

- **Convergence:** The time it takes for all routers in a network to agree on the network topology after a change occurs.
  Fast convergence means minimal dropped packets during a network failure.
- **Flapping:** A problematic condition where a broken network interface rapidly goes up and down. A good routing protocol
  will suppress a flapping link to prevent the entire network from constantly recalculating routes.
- **Administrative Distance (AD):** If a router runs _two_ different dynamic protocols (e.g., OSPF and RIP) and both find a
  path to the same destination, the router uses AD to determine which protocol to trust more. OSPF has a lower AD than RIP
  because it is considered more reliable.
