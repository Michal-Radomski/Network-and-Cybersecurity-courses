**SpiderMonkey** is the official name of the **JavaScript engine** developed by Mozilla. It is the piece of software under
the hood of the **Firefox** web browser that reads, compiles, and executes JavaScript code.

Fun fact: It was actually the world's very first JavaScript engine, originally written by Brendan Eich (the creator of
JavaScript) in 1995 for Netscape Navigator.

---

### What Exactly Does It Do?

When you visit a website, SpiderMonkey takes the raw text of the JavaScript code and does three main things:

1. **Parses it:** Breaks the code down into a tree structure that a computer can understand (an Abstract Syntax Tree).
2. **Compiles it:** Uses a **JIT (Just-In-Time) compiler** to instantly translate that JavaScript into lightning-fast machine
   code (zeros and ones) that your computer's CPU can run directly.
3. **Manages Memory:** It automatically cleans up unused variables and data so your browser doesn't crash or hog all your RAM
   (a process called Garbage Collection).

---

### How to Use It

For 99% of people, you use SpiderMonkey implicitly every single day just by opening **Firefox** or using the **Firefox
Developer Tools** (by pressing `F12` and clicking on the "Console" tab).

However, if you are a developer, you can actually use SpiderMonkey as a standalone tool _outside_ of the browser, much like
people use Node.js (which runs Google's V8 engine). Here is how you can play with it:

#### 1. The Javascript Shell (js-shell)

Mozilla provides a standalone command-line environment for SpiderMonkey.

If you are on Linux (like Ubuntu/Debian), you can install the standalone SpiderMonkey shell using your package manager:

```bash
sudo apt install jsshell

```

Once installed, you can start an interactive JavaScript session right in your terminal by typing:

```bash
js

```

This opens a prompt where you can run pure JavaScript without a browser:

```javascript
js> let message = "Hello from SpiderMonkey!";
js> print(message);
Hello from SpiderMonkey!

```

_(Note: Standard browser features like `window` or `document` won't work here, because those belong to the browser web API,
not the JavaScript engine itself)._

#### 2. Running a JS Script File

You can also pass a file directly to the engine to execute it, just like you would with Python or Node.js.

If you have a file named `script.js` with this code:

```javascript
function greet(name) {
  print("Hello, " + name);
}
greet("Developer");
```

You can run it in your Linux terminal like this:

```bash
js script.js
# Output: Hello, Developer

```

#### 3. Embedding it in C++ Applications

Because SpiderMonkey is written in C/C++, advanced software engineers can download the SpiderMonkey source code and embed it
into their own desktop applications or games. This allows their C++ apps to run custom JavaScript plugins or mods written by
users.
