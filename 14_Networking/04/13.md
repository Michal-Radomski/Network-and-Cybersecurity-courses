When data needs to travel from a device on one network to a device on a completely different network, it cannot be delivered
directly. It requires a layer 3 device—most commonly a **router**—to act as a bridge between them.

Here is a step-by-step breakdown of how routing between two networks works, using the standard TCP/IP and OSI models.

---

## The Scenario

Imagine we have two separate local area networks (LANs) connected by a single router:

- **Network A:** IP range `192.168.1.0/24` (Host PC-1: `192.168.1.5`)
- **Network B:** IP range `10.0.0.0/24` (Host PC-2: `10.0.0.20`)
- **The Router:** Has two physical interfaces:
- Interface 1 (Gateway for Network A): `192.168.1.1`
- Interface 2 (Gateway for Network B): `10.0.0.1`

---

## Step-by-Step Data Journey

When PC-1 wants to send a packet to PC-2, the following process takes place:

### 1. Determining the Destination (Local vs. Remote)

PC-1 looks at the destination IP address (`10.0.0.20`) and compares it to its own IP address and subnet mask
(`192.168.1.5 / 255.255.255.0`).

- **The Realization:** PC-1 realizes that `10.0.0.20` is **not** on its local network.
- **The Decision:** Because the destination is remote, PC-1 knows it cannot send the data directly to PC-2. Instead, it must
  send the packet to its **Default Gateway** (the router's local IP: `192.168.1.1`).

### 2. Finding the Gateway's MAC Address

Before PC-1 can transmit the data over the physical wire/Wi-Fi, it needs the hardware (MAC) address of the router's
interface.

- PC-1 checks its **ARP Cache** (Address Resolution Protocol table) for the MAC address belonging to `192.168.1.1`.
- If it isn't there, PC-1 sends out an **ARP Request** asking, _"Who has 192.168.1.1?"_ The router replies with its MAC
  address.

### 3. Packet Construction and Transmission (Layer 2 vs. Layer 3)

PC-1 builds the data package:

- **IP Packet (Layer 3):** Source IP = `192.168.1.5` | Destination IP = `10.0.0.20` _(Note: IP addresses do not change during
  transit)._
- **Ethernet Frame (Layer 2):** Source MAC = PC-1's MAC | Destination MAC = **Router's Interface 1 MAC**.
- PC-1 fires the frame across the wire.

### 4. The Router Processes the Packet

The router receives the frame because the destination MAC address matches its own.

- **Decapsulation:** The router strips away the outer Layer 2 Ethernet frame, leaving just the inner Layer 3 IP packet.
- **Table Lookup:** The router inspects the destination IP (`10.0.0.20`). It checks its **routing table** and finds that the
  `10.0.0.0/24` network is directly connected to its second interface (Interface 2).

### 5. Forwarding to the Destination Network

The router must now send the packet out of Interface 2 onto Network B.

- **ARP Lookup:** The router checks its own ARP table to find the hardware MAC address of PC-2 (`10.0.0.20`).
- **Encapsulation:** The router wraps the original IP packet in a **brand new Ethernet frame**.
- New Source MAC = Router's Interface 2 MAC.
- New Destination MAC = PC-2's MAC address.

- The router transmits the frame onto Network B.

### 6. Arrival at PC-2

PC-2 receives the frame, verifies the destination MAC address matches its own, strips the frame, and reads the IP packet. It
sees the source IP is `192.168.1.5` and processes the data.

---

## Summary of Key Rules

- **IP Addresses stay the same:** From the moment the packet leaves PC-1 to the moment it hits PC-2, the Source and
  Destination IP addresses never change.
- **MAC Addresses change at every hop:** The Layer 2 Ethernet frame is completely destroyed and rebuilt every time the data
  passes through a router. MAC addresses are only used to get data from one device to the immediate _next_ device on the same
  local physical wire.
- **The Default Gateway is crucial:** Without a configured default gateway, a host device will simply drop any packet
  destined for an outside network because it doesn't know where to send it.
