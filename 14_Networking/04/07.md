The `lsof -i -n -P` command is used in Linux and macOS to **list all open network connections and the processes running
them**.

Think of it as a specialized, highly detailed version of Task Manager or Resource Monitor, specifically focused on your
internet and network activity.

Here is exactly what the command and its specific flags do:

### What the Flags Mean

- **`lsof`**: Stands for **L**ist **S**o**f**iles. In Linux, almost everything (including network connections) is treated as
  a file.
- **`-i`**: Filters the list to show **only internet and network connections** (IPv4 and IPv6). If you don't specify a port
  or IP after it, it shows everything.
- **`-n`**: Inhibits the conversion of network numbers to host names. In plain English: **it shows raw IP addresses** (like
  `192.168.1.1`) instead of trying to look up domain names (like `router.local`). This makes the command run much faster.
- **`-P`**: Inhibits the conversion of port numbers to port names. It forces the command to **show the actual port numbers**
  (like `80` or `443`) instead of the service names (like `http` or `https`).

---

### Understanding the Output

When you run `sudo lsof -i -n -P`, you will see a table that looks like this:

| COMMAND  | PID    | USER   | FD    | TYPE   | DEVICE | SIZE/OFF | NODE  | NAME                                               |
| -------- | ------ | ------ | ----- | ------ | ------ | -------- | ----- | -------------------------------------------------- |
| `sshd`   | `1234` | `root` | `3u`  | `IPv4` | `4321` | `0t0`    | `TCP` | `*:22 (LISTEN)`                                    |
| `chrome` | `5678` | `user` | `42u` | `IPv4` | `8765` | `0t0`    | `TCP` | `192.168.1.5:52341->142.250.1.1:443 (ESTABLISHED)` |

**Key columns to look at:**

- **COMMAND**: The name of the application or process using the network (e.g., `sshd`, `chrome`, `nginx`).
- **PID**: The Process ID. Crucial if you want to force-close a stubborn application using `kill <PID>`.
- **NODE**: The protocol being used (`TCP` or `UDP`).
- **NAME**: Where the connection is coming from and going to.
- `*:22 (LISTEN)` means the application is waiting for connections on port 22.
- `->142.250.1.1:443 (ESTABLISHED)` means you are actively connected to that IP address on port 443.

> 💡 **Pro-Tip:** Like the previous netstat/ss commands, you should usually run this with **`sudo`** (`sudo lsof -i -n -P`).
> If you don't use `sudo`, the command will only show network connections owned by your specific user account, hiding
> system-level processes.
