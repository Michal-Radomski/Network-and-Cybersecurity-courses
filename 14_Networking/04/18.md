**CIDR**, which stands for **Classless Inter-Domain Routing**, is the standard system used to allocate IP addresses and route
data packets on the internet.

Introduced in 1993, CIDR replaced an older, wasteful system called "Classful routing." It allows network administrators to
precisely size networks based on their actual needs, preventing the internet from running out of available IP addresses
prematurely.

---

## The Problem CIDR Solved: The Old "Classful" System

Before CIDR, IP addresses were handed out in rigid, fixed blocks called classes:

- **Class A:** Handed out networks of **16.7 million** addresses. (Too big for almost everyone).
- **Class B:** Handed out networks of **65,536** addresses. (Still too big for most companies).
- **Class C:** Handed out networks of **256** addresses. (Too small for medium businesses).

If a company needed 500 IP addresses, Class C was too small, so they were given a Class B block. This meant over 65,000 IP
addresses were wasted on a single company. The internet was rapidly running out of space.

---

## How CIDR Works: Flexible Subnetting

CIDR completely eliminated these rigid classes. Instead of forcing a network to fit into a pre-made box, CIDR allows you to
draw the boundary line of a network anywhere you want.

An IP address is 32 bits long, split into a **Network ID** (the street name) and a **Host ID** (the house number). CIDR uses
a concept called **Variable-Length Subnet Masking (VLSM)** to let an administrator decide exactly how many bits belong to the
network and how many belong to the hosts.

---

## CIDR Notation (The "Slash" Slang)

You have likely seen CIDR notation without realizing it. It looks like a standard IP address followed by a forward slash and
a number:

$$\mathbf{192.168.1.0 / 24}$$

The **`/24`** (pronounced "slash 24") is the **CIDR prefix**. It tells the router exactly how many bits are locked down for
the network portion of the address.

- Because an IPv4 address has 32 bits in total, if 24 bits are used for the network, you have $32 - 24 = 8$ bits left over
  for unique host devices.
- $2^8 = 256$ total IP addresses available in that specific block.

### Common CIDR Examples

By shifting that slash number up or down, you can create networks of any size:

| CIDR Notation | Traditional Subnet Mask | Total Available IP Addresses | Best Used For                                     |
| ------------- | ----------------------- | ---------------------------- | ------------------------------------------------- |
| `/32`         | `255.255.255.255`       | 1                            | A single, specific device (Host route).           |
| `/30`         | `255.255.255.252`       | 4 _(2 usable)_               | A direct point-to-point link between two routers. |
| `/24`         | `255.255.255.0`         | 256 _(254 usable)_           | A standard small business or home office network. |
| `/22`         | `255.255.262.0`         | 1,024 _(1,022 usable)_       | A large corporate office or university building.  |
| `/16`         | `255.255.0.0`           | 65,536                       | A massive enterprise network or cloud VPC.        |

> **Note:** In practice, you always subtract 2 from the total addresses. The very first address is reserved as the Network
> ID, and the very last address is reserved as the Broadcast Address.

---

## Route Aggregation (Supernetting)

Beyond saving IP addresses, CIDR kept the global internet stable through **Route Aggregation**.

Before CIDR, if an ISP owned 50 small Class C networks, it had to advertise all 50 individual networks to the rest of the
world, making global routing tables staggeringly massive.

With CIDR, that ISP can combine (or "collapse") those 50 continuous networks into a single, broader CIDR block (like a single
`/16`). The ISP tells the world's core routers: _"Send anything matching this one big prefix to me,"_ and the ISP handles the
internal sorting. This drastically reduced the memory burden on global routers.
