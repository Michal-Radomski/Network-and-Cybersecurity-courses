**Route Aggregation** (also known as **Supernetting** or **Route Summarization**) is the process of combining multiple
detailed network routes into a single, shorter routing entry.

It is the networking equivalent of a zip code. Instead of a mail sorting facility keeping track of every single street name
in a distant city, it looks at the zip code and sends all those letters in one big box to that city's main post office.

---

## The Problem It Solves: Routing Table Explosion

Every time a new network is added to the internet or a large corporate network, routers have to add that network to their
routing tables.

Without aggregation, core internet routers would have to keep track of every individual small office, home network, and
website block. This would cause routing tables to grow so massive that routers would run out of memory (RAM) and slow down
significantly trying to process data packets.

---

## How Route Aggregation Works

Route aggregation relies entirely on **CIDR** (Classless Inter-Domain Routing). To aggregate networks, they must be
**contiguous** (sequential/back-to-back) and share the same starting binary bits.

Imagine an Internet Service Provider (ISP) allocates four small, consecutive networks to four different customers:

- Customer 1: `192.168.0.0/24`
- Customer 2: `192.168.1.0/24`
- Customer 3: `192.168.2.0/24`
- Customer 4: `192.168.3.0/24`

Instead of the ISP telling the rest of the internet about four separate `/24` networks, it looks at the binary structure of
these addresses to find where they match.

### The Binary Breakdown

If we convert the third octet (the number that changes) into binary, we can see exactly where the addresses align:

- `192.168.000000 00.0` (`192.168.0.0`)
- `192.168.000000 01.0` (`192.168.1.0`)
- `192.168.000000 10.0` (`192.168.2.0`)
- `192.168.000000 11.0` (`192.168.3.0`)

Notice that the first **22 bits** of all four networks are exactly the same (`192.168.000000`). Only the last two bits of the
third octet change.

### The Aggregated Result

The ISP can collapse all four entries into a single **summary route** by locking down only the matching 22 bits:

$$\mathbf{192.168.0.0 / 22}$$

Now, the ISP advertises just this _one_ route to the core internet routers. When a packet arrives at the ISP destined for
`192.168.2.5`, the ISP's internal routers handle the final local delivery.

---

## Major Benefits of Route Aggregation

- **Reduces Router Overhead:** It keeps routing tables small, saving RAM and CPU processing power on routers worldwide.
- **Improves Network Stability (Flap Dampening):** If Customer 1’s network constantly breaks and goes offline (known as
  "route flapping"), a local router would normally blast updates to the entire internet every time it changes. With
  aggregation, the summary route (`192.168.0.0/22`) stays online and stable because the other three customers are still up.
  The rest of the world never hears about the local instability.
- **Speeds up Convergence:** When a network layout changes, routers can recalculate smaller tables much faster than massive
  ones.

---

## The Golden Rules of Aggregation

For route aggregation to work without causing network chaos, two conditions must be met:

1. **The networks must be contiguous:** You cannot easily summarize `192.168.1.0` and `192.168.200.0` together without
   accidentally including hundreds of other networks in the middle.
2. **The routers must use Classless Routing:** Older protocols (like RIP v1) do not support CIDR slash notation and cannot
   handle custom summarized routes. Modern protocols like OSPF, BGP, and EIGRP handle this natively.
