When building or managing a network, administrators have two primary ways to populate a router's routing table: **Static
Routing** and **Dynamic Routing**.

The fundamental difference lies in **how** the router learns about network paths: static routes are hardcoded by a human,
while dynamic routes are learned automatically by the routers talking to each other.

---

## Static Routing

In static routing, a network administrator manually types the routes into the router’s configuration. It is a "set it and
forget it" approach that requires manual intervention whenever the network changes.

### How It Works

If Network A wants to talk to Network B, the admin must log into Router A and explicitly command it: _"To reach Network B,
send data to Router B's IP address."_ If a return path is needed, the admin must also log into Router B and configure the
reverse route manually.

### Pros & Cons

- **$\oplus$ Predictable and Secure:** The admin has total control over the exact path data takes. No unexpected route
  changes can occur.
- **$\oplus$ Zero Overhead:** It consumes no CPU power, RAM, or network bandwidth because the routers don't need to exchange
  routing updates.
- **$\ominus$ No Redundancy:** If a link breaks, a static route remains in the table, sending data into a "dead end" until an
  admin manually changes it.
- **$\ominus$ Not Scalable:** Managing static routes for 2 or 3 routers is easy. Managing them for 50 routers requires
  tracking thousands of manual entries, making it highly prone to human error.

---

## Dynamic Routing

Dynamic routing relies on specialized software called **Routing Protocols** (such as OSPF, EIGRP, BGP, or RIP). Routers use
these protocols to discover the network layout automatically and share information with their neighbors.

### How It Works

Routers constantly exchange "keep-alive" messages and topology updates. If a new network is added, routers tell their
neighbors about it. If a cable is cut, the affected router immediately alerts the rest of the network, and all routers
calculate a new detour path in real-time.

### Pros & Cons

- **$\oplus$ Automatic Rerouting (Failover):** If a primary path goes down, the routing protocol instantly calculates a
  backup path, resulting in minimal network downtime.
- **$\oplus$ Highly Scalable:** Excellent for large, complex networks. You only need to configure the protocol once, and the
  routers do the heavy lifting of mapping the network.
- **$\ominus$ Resource Intensive:** Exchanging updates consumes network bandwidth, and calculating the quickest paths
  requires router CPU and RAM.
- **$\ominus$ Complex Security:** Because routers automatically trust updates from neighbors, they are vulnerable to routing
  loops or malicious configuration injections if not properly authenticated.

---

## Comparison Table

| Feature                  | Static Routing                         | Dynamic Routing                             |
| ------------------------ | -------------------------------------- | ------------------------------------------- |
| **Configuration**        | Manually entered by an administrator.  | Automatically discovered by protocols.      |
| **Topology Changes**     | Requires manual updates by an admin.   | Adapts automatically in real-time.          |
| **Best Used For**        | Small, simple, or unchanging networks. | Large, complex, or growing networks.        |
| **Resource Consumption** | None (Low CPU/RAM/Bandwidth).          | High (Requires CPU/RAM/Bandwidth).          |
| **Path Selection**       | Chosen strictly by the admin.          | Calculated using metrics (bandwidth, hops). |
| **Failure Recovery**     | None (Traffic drops until fixed).      | High (Finds alternative paths instantly).   |

---

## When to Use Which?

Most enterprise networks actually use a hybrid approach called **Complementary Routing**:

- **Use Static Routes** for connecting a small branch office to the main corporate headquarters via a single internet line (a
  "Stub Network"), or for setting up a permanent **Default Route** (`0.0.0.0/0`) to send all unknown internet traffic to an
  ISP.
- **Use Dynamic Routes** for the internal core network where multiple routers are cross-connected, ensuring that if one
  internal switch or router fails, internal data traffic is automatically rerouted without dropping active connections.
