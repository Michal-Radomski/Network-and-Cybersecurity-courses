### What is STARTTLS?

**STARTTLS** is a protocol command used to upgrade an existing, unencrypted communication channel into a secure, encrypted
one (using TLS or SSL).

Instead of using a separate port for secure communications (like HTTPS does on port 443), STARTTLS allows a server and a
client to start talking in plain text on the **original, standard port**, and then seamlessly "switch on" encryption
mid-conversation.

---

### How STARTTLS Works (The Handshake)

Imagine a conversation between an email client (like Outlook or phone mail) and an email server:

1. **The Plaintext Connection:** The client connects to the server on a standard unencrypted port (for example, SMTP port
   587).
2. **The Announcement:** The server tells the client, _"Hey, I support encryption."_
3. **The Command:** The client sends the specific command: **`STARTTLS`**.
4. **The Upgrade:** Both sides perform a cryptographic handshake. From that exact moment onward, all data sent back and forth
   (including your login credentials and email content) is encrypted.

---

### STARTTLS vs. Implicit TLS

When configuring software (especially email), you will often see two choices for security: **STARTTLS** and **SSL/TLS**
(sometimes called _Implicit TLS_). Here is how they differ:

| Feature               | STARTTLS (Opportunistic/Explicit)                                                                                | Implicit TLS                                                                    |
| --------------------- | ---------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| **How it starts**     | Starts as **unencrypted text**, then upgrades to encrypted.                                                      | Starts as **encrypted** from the very first second.                             |
| **Ports Used**        | Uses the traditional, standard ports (e.g., SMTP **587**, IMAP **143**).                                         | Requires dedicated, separate secure ports (e.g., SMTPS **465**, IMAPS **993**). |
| **Fallback Behavior** | If the encryption upgrade fails, it might fall back to sending data in plain text (unless configured otherwise). | If the encryption handshake fails, the connection immediately drops.            |

### The "Opportunistic" Risk

STARTTLS is often referred to as **Opportunistic TLS**. Because it starts in plain text, it is vulnerable to a cyberattack
known as a **STRIPTLS attack** (a type of Man-in-the-Middle attack).

In this scenario, an attacker intercepts the initial plain text conversation and alters the server's message to hide the fact
that it supports encryption. The client, thinking the server is old and doesn't support security, falls back to sending your
password and data in clear text.

> 💡 **Best Practice:** Modern email applications usually have a setting to **"Require STARTTLS"**. This ensures that if the
> encryption upgrade fails or is blocked, the app will refuse to send your data rather than falling back to unencrypted text.
