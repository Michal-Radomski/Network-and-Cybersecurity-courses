Here is a list of common networking protocols, comparing their original (unencrypted) versions with their modern, secure
(encrypted) counterparts.

---

### Common Network Protocols: Unsecure vs. Secure

| Service / Use Case         | Original (Unsecure) Protocol | Default Port | Secure Protocol | Default Port       | Encryption/Security Mechanism               |
| -------------------------- | ---------------------------- | ------------ | --------------- | ------------------ | ------------------------------------------- |
| **Web Browsing**           | HTTP                         | **80**       | HTTPS           | **443**            | SSL/TLS                                     |
| **File Transfer**          | FTP                          | **20, 21**   | SFTP / FTPS     | **22 / 990**       | SSH (SFTP) or SSL/TLS (FTPS)                |
| **Remote Terminal Access** | Telnet                       | **23**       | SSH             | **22**             | Public-key cryptography                     |
| **Email Sending**          | SMTP                         | **25**       | SMTPS           | **465** or **587** | SSL/TLS / STARTTLS                          |
| **Email Retrieval**        | POP3                         | **110**      | POP3S           | **995**            | SSL/TLS                                     |
| **Email Retrieval**        | IMAP                         | **143**      | IMAPS           | **993**            | SSL/TLS                                     |
| **Directory Services**     | LDAP                         | **389**      | LDAPS           | **636**            | SSL/TLS                                     |
| **Network Management**     | SNMPv1 / SNMPv2              | **161, 162** | SNMPv3          | **161, 162**       | Cryptographic authentication and encryption |

---

### Key Concepts to Remember

- **Why the change?** Original protocols transmit data—including passwords and sensitive information—in **cleartext** (plain
  text). Anyone intercepting the traffic can read it. Secure versions encrypt the data so it looks like gibberish to
  eavesdroppers.
- **STARTTLS vs. Implicit SSL/TLS:** Some secure protocols use an entirely different port (like HTTPS on 443). Others use the
  **STARTTLS** command, which allows a connection to start on the original unsecure port and then upgrade to a secure,
  encrypted connection automatically.
