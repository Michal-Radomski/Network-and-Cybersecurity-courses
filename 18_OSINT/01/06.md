While the term "dorking" originated with Google, almost all major search engines support advanced search operators. However,
because each search engine uses its own indexing engine, the available operators, syntax, and behaviors vary.

---

## 1. Bing

Bing supports many standard operators, but with a few unique additions and differences in syntax.

| Operator              | Function                                                | Example                        |
| --------------------- | ------------------------------------------------------- | ------------------------------ |
| `site:`               | Restricts results to a specific domain or TLD.          | `site:gov "report"`            |
| `filetype:`           | Filters by file extension.                              | `filetype:pdf "annual report"` |
| `ip:`                 | Finds sites hosted on a specific IP address.            | `ip:192.0.2.1`                 |
| `contains:`           | Identifies pages linking to specific file types.        | `contains:pdf`                 |
| `feed:`               | Finds RSS or Atom feeds on a topic or site.             | `feed:cybersecurity`           |
| `loc:` or `location:` | Restricts results to a specific country or region code. | `loc:uk "press release"`       |

_Note: Bing also powers Yahoo search, so Bing operators generally work identically on Yahoo._

---

## 2. DuckDuckGo

DuckDuckGo uses a mix of its own crawler and index sources from partners (like Bing). It supports standard search syntax
along with specialized shortcut features.

| Operator    | Function                                                           | Example                                            |
| ----------- | ------------------------------------------------------------------ | -------------------------------------------------- |
| `site:`     | Limits results to a specific domain.                               | `site:github.com "readme"`                         |
| `filetype:` | Restricts results to specific file extensions.                     | `filetype:docx`                                    |
| `intitle:`  | Looks for terms within the page title.                             | `intitle:"index of"`                               |
| `!bangs`    | Redirects your query directly to an external site's search engine. | `!w cybersecurity` _(searches Wikipedia directly)_ |

---

## 3. Brave Search

Brave Search relies on its own independent index and supports standard logical operators and site filtering.

| Operator    | Function                               | Example                          |
| ----------- | -------------------------------------- | -------------------------------- |
| `site:`     | Limits search to a specific domain.    | `site:archive.org`               |
| `filetype:` | Filters results by file extension.     | `filetype:pdf`                   |
| `title:`    | Searches for terms within page titles. | `title:"privacy policy"`         |
| `-` (minus) | Excludes terms or sites from results.  | `encryption -site:wikipedia.org` |

---

## Key Differences to Keep in Mind

- **Strictness:** Google is typically the most precise with strict quotes (`"..."`) and negative terms (`-term`). Engines
  like Bing or DuckDuckGo may occasionally include soft matches even when quotes are used.
- **Specialized Indexing Engines:** Search engines tailored for specific internet layers (such as specialized IoT/network
  search platforms) use entirely different query syntax based on banners, ports, certificates, and headers rather than
  traditional webpage text.
- **Combining Operators:** Most standard web search engines allow you to chain operators together (e.g.,
  `site:example.com filetype:pdf`).
