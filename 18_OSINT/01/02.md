**Google Dorking** (also known as **Google Hacking**) refers to the technique of using advanced search operators in Google to
find information that is publicly indexed on the web, but difficult to find with standard search queries.

It relies on standard search commands built into Google—not actual hacking or exploiting software. However, because Google
indexes virtually everything it can access, precise queries can reveal unintentionally exposed data.

---

## Common Search Operators Used

| Operator    | Function                                                          | Example                 |
| ----------- | ----------------------------------------------------------------- | ----------------------- |
| `site:`     | Restricts results to a specific domain or website.                | `site:wikipedia.org`    |
| `filetype:` | Searches for specific file extensions (e.g., PDF, DOCX, XLSX).    | `filetype:pdf`          |
| `intitle:`  | Searches for pages with specific text in the page title.          | `intitle:"Index of"`    |
| `inurl:`    | Finds pages containing specific words in the website address/URL. | `inurl:login`           |
| `intext:`   | Searches for specific text within the body of a webpage.          | `intext:"confidential"` |

Combining these operators creates a "dork"—a tailored search string. For example, a query like
`site:example.com filetype:pdf` will filter Google results to show only PDF files hosted on that specific domain.

---

## What Google Dorks Are Used For

- **Cybersecurity Research & OSINT:** Security professionals and ethical hackers use search operators during reconnaissance
  to find misconfigured servers, exposed files, or accidental data leaks so organizations can fix them.
- **Refined Everyday Searching:** Students, researchers, and journalists use operators daily to cut through clutter and
  instantly isolate exact documents or specific domains.
- **Malicious Reconnaissance:** Threat actors can use advanced queries to scout for exposed login portals, outdated software,
  or unlinked directories before attempting an attack.

> **Note:** Performing a search using advanced operators is completely legal. However, accessing non-public, sensitive data
> or using discovered information to access unauthorized systems crosses legal and ethical boundaries.
