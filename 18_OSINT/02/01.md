While the terms **breached database** and **leaked database** are often used interchangeably in casual news reports, they
refer to different stages and mechanisms of data compromise.

The primary distinction comes down to **how the data was accessed** (an active attack vs. an accidental exposure) and **how
the data became public**.

---

## Key Definitions

### 1. Breached Database

A **data breach** occurs when an unauthorized actor actively bypasses security controls to gain access to a private system or
database.

- **Mechanism:** Involves an intentional, unauthorized intrusion—such as exploiting a software vulnerability (e.g., SQL
  injection), using stolen credentials, or executing malware.
- **Focus:** Describes the **attack event** itself and the unauthorized intrusion into a secure network.
- **Example:** An attacker uses compromised administrator credentials to log into a company's internal server and download
  customer tables.

---

### 2. Leaked Database

A **data leak** occurs when sensitive data is exposed to the public or unauthorized parties without requiring an active
intrusion, often due to misconfiguration, human error, or deliberate insider release.

- **Mechanism:** Frequently caused by cloud storage misconfigurations (e.g., an open Amazon S3 bucket or unauthenticated
  database instance), accidental publication, or an insider dumping data publicly.
- **Focus:** Describes the **state of exposure**—the data is freely accessible to anyone who finds it.
- **Example:** A database backup file is accidentally saved to a publicly accessible web directory with no password
  protection, allowing search engines to index it.

---

## Direct Comparison

| Feature                 | Breached Database                                                   | Leaked Database                                                     |
| ----------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------- |
| **Primary Cause**       | Active security bypass or cyberattack                               | Misconfiguration, human error, or accidental exposure               |
| **Attacker Action**     | Requires breaking through security barriers                         | Requires minimal effort (often just finding an open link/port)      |
| **Public Availability** | Stolen data may remain private to the attacker or be sold privately | Data is exposed to the open internet (or indexed by search engines) |
| **Common Trigger**      | Exploits, phishing, credential stuffing                             | Unsecured cloud buckets, missing authentication, accidental uploads |

---

> **Summary:** A **breach** is an unauthorized breaking-and-entering into a system to steal data, whereas a **leak** is an
> instance where data is left exposed or made public due to a mistake or lack of access controls.
