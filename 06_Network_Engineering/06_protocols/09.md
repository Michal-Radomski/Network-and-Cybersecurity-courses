Public and private keys are used to secure modern digital communication. They solve two critical problems: confidentiality
(stopping eavesdroppers) and authenticity (proving who sent a message). Here is exactly how and when they are used in
everyday digital life.

---

## 1. Data Encryption (Confidentiality)

When it is used: Sending private messages, processing credit card payments, or protecting data stored in the cloud.

- How it works:
- Anyone can use your public key to encrypt a message.
  - Once encrypted, the data becomes unreadable scrambled text.
  - Only you possess the corresponding private key required to decrypt and read it.
- Real-world example: If Alice wants to send Bob a secret file, she encrypts it using Bob’s public key. Even if a hacker
  intercepts the file, they cannot open it without Bob’s private key.

## 2. Digital Signatures (Authenticity & Integrity)

When it is used: Signing legal e-documents (like DocuSign), authenticating software updates, and authorizing cryptocurrency
transactions.

- How it works:
- You sign a document by hashing it and encrypting that hash with your private key.
  - Anyone can use your public key to decrypt the signature and verify it matches the document.
  - This proves the file genuinely came from you and was not altered in transit.
- Real-world example: When you send Bitcoin, your wallet uses your private key to sign the transaction. The blockchain
  network uses your public key to verify that you authorized the transfer.

## 3. Secure Web Browsing (HTTPS)

When it is used: Every time you visit a secure website (indicated by the padlock icon in your browser URL bar).

- How it works:
- Your browser requests the website's public key (embedded in an SSL/TLS certificate).
  - Your browser uses that public key to securely send a temporary, random "session key" to the website.
  - The website decrypts it using its private key.
  - Both sides now use that temporary session key for ultra-fast, symmetric encryption for the rest of your browsing session.

## 4. SSH Remote Access

When it is used: When software developers and system administrators securely log into remote servers or push code to
platforms like GitHub.

- How it works:
- You paste your public key onto the remote server.
  - When you try to log in, the server sends your computer a random challenge encrypted with that public key.
  - Your computer automatically decrypts it using your local private key and sends it back.
  - This logs you in securely without ever typing a password.
