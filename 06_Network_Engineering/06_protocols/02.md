The Domain Name System (DNS) header is a critical **12-byte (96-bit)** structure found at the beginning of every DNS query
and response. It provides the essential metadata needed for the client and server to understand what the packet is for and
how to process it.

### 1. The Header Fields

The header is divided into six 16-bit fields. Here is the breakdown:

| Field              | Size    | Description                                                                            |
| ------------------ | ------- | -------------------------------------------------------------------------------------- |
| **Transaction ID** | 16 bits | A unique identifier assigned by the client to match a response with a specific query.  |
| **Flags**          | 16 bits | A collection of control bits that define the message type and status (detailed below). |
| **Questions**      | 16 bits | The number of entries in the Question section (usually 1).                             |
| **Answer RRs**     | 16 bits | The number of Resource Records in the Answer section.                                  |
| **Authority RRs**  | 16 bits | The number of Resource Records in the Authority section.                               |
| **Additional RRs** | 16 bits | The number of Resource Records in the Additional section.                              |

---

### 2. Deep Dive: The Flags Field

The **Flags** field is the "brain" of the header. It’s a 16-bit sequence where specific bits have distinct meanings:

- **QR (Query/Response):** 1 bit. `0` for a query, `1` for a response.
- **Opcode:** 4 bits. Defines the kind of query (standard query is `0`).
- **AA (Authoritative Answer):** 1 bit. In a response, `1` means the responding server is an authority for the domain.
- **TC (TrunCation):** 1 bit. If `1`, the message was too large for the transmission channel (common in UDP).
- **RD (Recursion Desired):** 1 bit. If set by the client, it asks the server to pursue the query recursively.
- **RA (Recursion Available):** 1 bit. Set by the server to indicate it supports recursion.
- **Z (Reserved):** 3 bits. Historically reserved for future use (usually `0`).
- **RCODE (Response Code):** 4 bits. Indicates the status of the response (e.g., `0` for No Error, `3` for NXDOMAIN/Name
  Error).

---

### Why is the Header important?

- **Security:** The **Transaction ID** helps prevent DNS spoofing (the response must match the ID of the request).
- **Efficiency:** The **TC (Truncation)** bit tells the client if it needs to switch from UDP to TCP to get the full data.
- **Navigation:** The **Count** fields (QDCOUNT, ANCOUNT, etc.) tell the parser exactly where the header ends and where the
  variable-length data sections begin.
