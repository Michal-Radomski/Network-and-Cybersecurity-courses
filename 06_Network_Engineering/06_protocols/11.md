Generating keys and certificates with OpenSSL depends on what you need them for. The most common use case is creating a
**Self-Signed Certificate** (great for testing and local development) or generating a **Private Key and a Certificate Signing
Request (CSR)** to get a certificate from an official Certificate Authority (CA).

Here is a quick, practical guide to the most common OpenSSL commands.

---

## 1. Create a Self-Signed Certificate (Perfect for Testing)

If you just need a certificate for a local development server (like HTTPS on `localhost`), you can generate a private key and
a self-signed certificate in a single command.

```bash
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -sha256 -days 365 -nodes

```

### What this command means:

- **`-x509`**: Tells OpenSSL to output a self-signed certificate instead of a certificate request.
- **`-newkey rsa:4096`**: Creates a new 4096-bit RSA private key at the same time.
- **`-keyout key.pem`**: The filename where your private key will be saved.
- **`-out cert.pem`**: The filename where your certificate will be saved.
- **`-days 365`**: The certificate will be valid for 1 year.
- **`-nodes`**: Short for "no DES". This skips encrypting the private key with a password, which is usually preferred for
  automated servers so they can restart without you typing a password. _Remove this if you want to secure your key with a
  passphrase._

---

## 2. Generate a Key and CSR (For Production/Official CAs)

If you are buying a certificate from a trusted authority (like DigiCert, Namecheap, etc.), you must keep your private key
secret and send them a CSR.

### Step 1: Generate the Private Key

```bash
openssl genrsa -out private.key 4096

```

### Step 2: Generate the CSR from that Key

```bash
openssl req -new -key private.key -out request.csr

```

> ⚠️ **Note:** When you run this, it will ask you a series of questions (Country, State, Organization, etc.). The most
> important field is the **Common Name (CN)**, which must exactly match your domain name (e.g., `example.com` or
> `*.example.com`).

---

## 3. Extracting or Verifying Your Files (Handy Tips)

Once you have your files, you might want to double-check them or convert their formats.

- **View the details of a Certificate:**

```bash
openssl x509 -in cert.pem -text -noout

```

- **Verify a CSR:**

```bash
openssl req -in request.csr -text -noout -verify

```

- **Combine Key and Certificate into a `.pfx` / `.p12` file (often required for IIS or Windows servers):**

```bash
openssl pkcs12 -export -out certificate.pfx -inkey key.pem -in cert.pem

```
