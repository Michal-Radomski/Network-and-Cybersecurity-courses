A **Network Address Translation (NAT) Table** is a database maintained by a router or firewall. It acts like a routing
directory or a "switchboard operator" that keeps track of connections between devices on a private local network (LAN) and
servers on the public internet (WAN).

Because local devices use private IP addresses (which cannot be routed on the public internet), the router uses the NAT table
to map those private addresses to its single public IP address, ensuring data safely finds its way back to the right device.

---

## How a NAT Table Works (The Airport Analogy)

Think of an office building with one main phone number (the Public IP) and hundreds of employee desks with unique extension
numbers (Private IPs).

- When an employee calls out, the receptionist routes the call through the main number and notes down who made the call.
- When the external party calls back, the receptionist checks their log and connects the call to the correct desk extension.

The NAT table is that receptionist's log.

---

## Structure of a NAT Table

In a standard home or office setup (specifically using **PAT - Port Address Translation**, the most common type of NAT), a
NAT table consists of four main columns:

1. **Inside Local:** The private IP address and port number of the device on the internal network (e.g., your laptop).
2. **Inside Global:** The public IP address assigned by your ISP, combined with a unique port number assigned by the router
   for that specific session.
3. **Outside Global:** The public IP address and port number of the destination server on the internet (e.g., Google's web
   server).
4. **Outside Local:** The IP address of the outside device as seen by the internal network (almost always identical to the
   Outside Global address).

### Example of a NAT Table Entry

Imagine your laptop ($192.168.1.5$) wants to visit a website ($203.0.113.50$). Your router's public IP is $93.184.216.34$.

The NAT table entry will look like this:

| Inside Local (Your Device) | Inside Global (Router's Public Face) | Outside Global (The Website) | Protocol |
| -------------------------- | ------------------------------------ | ---------------------------- | -------- |
| `192.168.1.5:54321`        | `93.184.216.34:40001`                | `203.0.113.50:443`           | TCP      |
| `192.168.1.9:61200`        | `93.184.216.34:40002`                | `142.250.190.46:80`          | TCP      |

> **What happens here:** When the website responds, it sends data back to `93.184.216.34:40001`. The router looks at its
> table, sees that port `40001` is mapped to your laptop at `192.168.1.5:54321`, rewrites the destination header, and
> forwards the data to you.

---

## Why is the NAT Table Important?

- **Conserving IPv4 Addresses:** It allows thousands of devices across a local network to share a single public IP address.
- **Security:** Because external traffic cannot pass through the router unless there is an active, matching entry in the NAT
  table, it acts as a natural, basic firewall. Unsolicited inbound traffic is simply dropped.
- **Tracking State:** It ensures that your sibling’s Netflix stream, your phone's app updates, and your computer's web
  browsing don't get mixed up, even though they are all utilizing the exact same public internet connection.

## Table Maintenance (Aging Out)

NAT tables do not hold onto entries forever. Because memory on routers is finite, entries have a **timeout period**.

- **TCP connections** (like web browsing) might stay in the table for hours until the connection is formally closed.
- **UDP connections** (like DNS queries or live gaming) might "age out" and be deleted after just a few minutes of inactivity
  to free up space.
