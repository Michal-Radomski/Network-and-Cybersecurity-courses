The Transmission Control Protocol (TCP) uses a complex state machine to manage the lifecycle of a connection—from creation to
data transfer, and finally to termination.

Understanding these states is crucial for network troubleshooting and analyzing traffic. Here is a breakdown of the TCP
connection states, organized by the phases of a connection.

---

## 1. Connection Establishment (The 3-Way Handshake)

Before data can flow, the client and server must synchronize their sequence numbers through a three-step process.

- **LISTEN:** The server is waiting for an incoming connection request from any remote client. It is in a passive state.
- **SYN-SENT:** The client has sent a synchronization request (`SYN` packet) to the server and is waiting for a matching
  connection request and acknowledgment.
- **SYN-RECEIVED:** The server has received the client's `SYN`, sent back a `SYN-ACK` (Synchronization-Acknowledgment), and
  is waiting for the final acknowledgment (`ACK`) from the client.
- **ESTABLISHED:** The connection is fully open. The client received the `SYN-ACK`, sent the final `ACK`, and data transfer
  can now begin. This is the normal operating state for an active connection.

---

## 2. Active Data Transfer

- **ESTABLISHED:** As mentioned above, both hosts remain in this state while exchanging data.

---

## 3. Connection Termination (The 4-Wave Close)

Closing a connection can be initiated by either party (active close), while the other party responds (passive close). Because
TCP is full-duplex, each side of the connection must be closed independently.

### The Active Close (Initiating the teardown)

- **FIN-WAIT-1:** The application on this host decided it was done and sent a `FIN` (Finish) packet. It is now waiting for an
  acknowledgment (`ACK`) from the remote host.
- **FIN-WAIT-2:** This host received the `ACK` for its `FIN`. It is now waiting for the remote host to finish its data
  transfer and send its own `FIN` packet.
- **CLOSING:** A rare state where both hosts tried to close the connection at the same time. This host sent a `FIN`, received
  a `FIN` from the remote host, but hasn't received the `ACK` for its own `FIN` yet.
- **TIME-WAIT:** The host has received the remote `FIN`, sent the final `ACK`, and is now waiting for a period equal to 2MSL
  (Maximum Segment Lifetime, usually 1–4 minutes). This ensures the final `ACK` was received and prevents old, delayed
  packets from interfering with new connections.

### The Passive Close (Responding to the teardown)

- **CLOSE-WAIT:** This host received a `FIN` from the remote host and sent an `ACK`. It is now waiting for its local
  application to finish sending data and issue its own close command.
- **LAST-ACK:** The local application finally closed, and this host sent its `FIN` packet. It is now waiting for the final
  `ACK` from the remote host.

---

## 4. The Final State

- **CLOSED:** This represents no connection state at all. The connection is completely dead, and the resources (ports and
  memory buffers) are released back to the operating system.

---

## Quick Reference Summary Table

| State            | Who enters it? | What is it waiting for?                                      |
| ---------------- | -------------- | ------------------------------------------------------------ |
| **LISTEN**       | Server         | A connection request (`SYN`) from a client.                  |
| **SYN-SENT**     | Client         | A `SYN-ACK` reply from the server.                           |
| **SYN-RECEIVED** | Server         | The final `ACK` from the client to establish the connection. |
| **ESTABLISHED**  | Both           | Normal data transfer phase.                                  |
| **FIN-WAIT-1**   | Initiator      | An `ACK` for its own `FIN` packet.                           |
| **FIN-WAIT-2**   | Initiator      | The `FIN` packet from the responder.                         |
| **CLOSE-WAIT**   | Responder      | The local application to finish up and close.                |
| **LAST-ACK**     | Responder      | The final `ACK` for its own `FIN` packet.                    |
| **TIME-WAIT**    | Initiator      | A timeout buffer (2MSL) to ensure the network clears out.    |
| **CLOSED**       | Both           | None. The connection is entirely closed.                     |
