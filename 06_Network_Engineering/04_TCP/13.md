When you run an Express.js app, there is a fascinating stack of technologies connecting your physical network cards to your
JavaScript code.

Here is the breakdown of how that connection works, followed by how you can restrict your app to _only_ listen to your
Ethernet card.

---

## The Connection Stack: Hardware to Express.js

When a packet of data arrives at your laptop, it travels through several layers to reach your app:

1. **The Physical Layer (Hardware):** Your Ethernet card (NIC) or Wi-Fi card physically receives the data packets.
2. **The Operating System (Kernel & TCP/IP Stack):** The OS manages these network interfaces and assigns them IP addresses.
   It maintains a table of which applications are listening to which ports and IP addresses.
3. **The Node.js Runtime:** Express.js is built on top of Node.js. Node has a core module called `net` (written in C++ and
   JavaScript) that interacts with the OS using **network sockets**.
4. **The Express.js App:** Express sits at the very top, providing a smooth wrapper around Node's native server so you can
   easily handle requests.

---

## Binding to Specific Interfaces (The IP Address Trick)

By default, when you write `app.listen(3000)`, Express tells the OS to listen on the special IP address `0.0.0.0`.

> `0.0.0.0` means **"listen on all available network interfaces."** This is why your app is currently accessible via both
> Wi-Fi and Ethernet.

To make Express listen **only** to your Ethernet card, you must bind it specifically to the local IP address assigned to that
Ethernet card.

### Step 1: Find your Ethernet IP address

You need to find the specific IPv4 address of your Ethernet card.

- **Windows (Command Prompt):** Run `ipconfig` and look for "Ethernet adapter".
- **Mac/Linux (Terminal):** Run `ifconfig` or `ip a` and look for your wired interface (usually named `en0`, `eth0`, or
  `eth1`).

Let's assume your Ethernet IP address is `192.168.1.50`, and your Wi-Fi IP is `192.168.1.75`.

### Step 2: Pass the IP to `app.listen()`

The `app.listen()` method accepts a hostname/IP address as its second argument. You just need to explicitly pass your
Ethernet IP there:

```javascript
const express = require("express");
const app = express();

const PORT = 3000;
// Replace this with your actual Ethernet IP address
const ETHERNET_IP = "192.168.1.50";

app.get("/", (req, res) => {
  res.send("Hello from the Ethernet card!");
});

// Binding specifically to the Ethernet IP
app.listen(PORT, ETHERNET_IP, () => {
  console.log(`Server running strictly on Ethernet at http://${ETHERNET_IP}:${PORT}`);
});
```

### What happens now?

- If someone tries to access your laptop via your **Ethernet IP** (`http://192.168.1.50:3000`), the connection will succeed.
- If someone tries to access your laptop via your **Wi-Fi IP** (`http://192.168.1.75:3000`), the OS will reject the
  connection, and they will get a "Connection Refused" error.
- **Note:** You will also lose the ability to access it via `http://localhost:3000` or `http://127.0.0.1:3000` from your own
  machine, because `localhost` goes through the loopback interface, not the Ethernet card.

## A Dynamic Alternative (The `os` module)

If your Ethernet IP changes frequently because of DHCP, hardcoding the IP is a pain. You can use Node's built-in `os` module
to dynamically find your Ethernet card's IP at startup:

```javascript
const express = require("express");
const os = require("os");
const app = express();

// Find the Ethernet interface dynamically
const interfaces = os.networkInterfaces();
let ethernetIp = "127.0.0.1"; // fallback

// Look for your specific interface name (e.g., 'Ethernet', 'en0', 'eth0')
const ethernetInterface = interfaces["Ethernet"] || interfaces["eth0"] || interfaces["en0"];

if (ethernetInterface) {
  const ipv4 = ethernetInterface.find((info) => info.family === "IPv4");
  if (ipv4) ethernetIp = ipv4.address;
}

app.listen(3000, ethernetIp, () => {
  console.log(`Server dynamically bound to Ethernet at http://${ethernetIp}:3000`);
});
```
