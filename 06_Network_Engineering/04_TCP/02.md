**TCP**, which stands for **Transmission Control Protocol**, is one of the foundational building blocks of the internet.
Operating at the Transport Layer (Layer 4) of the network stack, it is the protocol responsible for ensuring that data is
delivered reliably, accurately, and in the correct order from one device to another.

If UDP is like dropping a letter in the mail and hoping for the best, TCP is like sending a package via registered mail with
a tracking number, delivery confirmation, and a contract that says: _"If any piece of this package is damaged or missing, we
will stop everything and resend it."_

---

## 1. The Core Concept: Connection-Oriented & Reliable

Unlike connectionless protocols, TCP never just blindly starts throwing data into the network. It requires a formal
relationship to be established between the sender and the receiver before a single byte of actual data is transferred.

Once that relationship is built, TCP acts as a meticulous project manager for your data, guaranteeing that:

1. **No data is lost:** Every packet is tracked.
2. **No data is corrupted:** Everything is checked for errors.
3. **Data arrives in order:** Packets are stitched back together in the exact sequence they were sent.

---

## 2. The 3-Way Handshake (How it Starts)

Before your browser loads a webpage using TCP, your computer and the web server perform a strict, three-step opening ceremony
known as the **3-Way Handshake**.

- **Step 1: SYN (Synchronize):** The client sends a packet to the server saying, _"Hey, I want to talk. Let's synchronize our
  sequence numbers. My starting number is X."_
- **Step 2: SYN-ACK (Synchronize-Acknowledgment):** The server receives it and replies, _"Got your request! I acknowledge
  your number X. I also want to talk, let's sync my numbers too. My starting number is Y."_
- **Step 3: ACK (Acknowledgment):** The client replies, _"Got it! I acknowledge your number Y. We are now connected."_

Only after this three-step exchange is complete does the actual data transfer begin.

---

## 3. Key Features of TCP

To ensure perfect reliability, TCP uses several sophisticated mechanisms:

- **Sequence Numbers (Ordering):** TCP breaks large files into smaller chunks called segments. Each segment gets a sequence
  number. If packet #3 arrives at your computer before packet #2, your operating system will hold packet #3 in memory and
  wait for #2 to arrive before showing the data to your application.
- **Acknowledgments (ACK) & Retransmissions:** Every time the receiver gets a packet, it sends an "ACK" message back to the
  sender. If the sender doesn't receive an ACK within a certain timeframe, it assumes the packet was lost in the depths of
  the internet and automatically sends it again.
- **Flow Control (Windowing):** TCP prevents a fast sender from overwhelming a slow receiver. The receiver constantly tells
  the sender, _"I only have room in my memory buffer for X amount of data right now."_ The sender adjusts its speed
  accordingly.
- **Congestion Control:** TCP is a good internet citizen. If it notices that packets are being dropped frequently, it deduces
  that the network routers are clogged. It will automatically throttle its own transmission speed to help clear the network
  traffic jam.

---

## 4. The TCP Segment Structure

Because TCP does so much heavy lifting, its header is complex and significantly larger than a UDP header (minimum 20 bytes
vs. UDP's fixed 8 bytes).

Key fields inside a TCP header include:

- **Source & Destination Ports:** To route the data to the correct application.
- **Sequence Number:** Tracks the order of the data bytes.
- **Acknowledgment Number:** Tracks the next expected byte from the other side.
- **Flags (SYN, ACK, FIN, RST):** 1-bit triggers used to manage the connection state (e.g., establishing, acknowledging, or
  tearing down the connection).
- **Window Size:** Used for flow control to dictate how much data the receiver can handle.
- **Checksum:** Ensures the header and data weren't corrupted by electrical noise or hardware glitches.

---

## 5. When is TCP Used?

TCP is used for any application where **data integrity is non-negotiable**. If a single bit of data is altered or missing,
the entire file or action becomes broken.

### Common Use Cases:

- **Web Browsing (HTTP/HTTPS):** Missing pixels on a live video stream is fine, but a missing snippet of code on a website
  could cause the page to crash or fail to load.
- **File Transfers (FTP/SFTP):** If you download a software installer or a ZIP file, a single missing byte can corrupt the
  entire file, rendering it unopenable.
- **Email (SMTP/IMAP):** You need to guarantee that every sentence of your message arrives exactly as written.
- **Secure Shell (SSH) & Database Queries:** Remote terminal commands and banking database updates must be flawless and
  perfectly ordered.
