The standard algorithm used to generate a public and private key pair is the RSA (Rivest-Shamir-Adleman) algorithm. It relies
on prime factorization, a mathematical one-way function where multiplying two large prime numbers is easy, but factoring
their product back into primes is computationally nearly impossible. [1, 2, 3, 4, 5] Here is the exact step-by-step
mathematical algorithm used to generate the keys. [3, 6, 7]

## 1. Select Two Distinct Prime Numbers [8]

Choose two incredibly large, random prime numbers, $p$ and $q$. In real-world security systems, these primes are thousands of
bits long. [3, 7, 9, 10]

- Condition: $p \neq q$ [3]
-

## 2. Calculate the Modulus

Multiply the two prime numbers together to find the modulus $n$. This number will dictate the maximum size of data that can
be encrypted and forms part of both keys. $$n = p \times q$$

## 3. Compute Euler's Totient Function

Calculate the value of $\phi(n)$ (also called the totient), which represents how many integers up to $n$ are coprime to $n$.
$$\phi(n) = (p - 1) \times (q - 1)$$

## 4. Choose the Public Exponent [11]

Select an integer $e$, which will serve as the public exponent (used for encryption). [12, 13]

- Conditions:
- $1 < e < \phi(n)$
  - $\gcd(e, \phi(n)) = 1$ (meaning $e$ and $\phi(n)$ must be coprime)
- Standard practice: In modern implementations, $e$ is almost universally set to $65537$ because it optimizes computing speed
  and security. [3, 7, 13, 14, 15]
-

## 5. Calculate the Private Exponent [16]

Compute the private exponent $d$ (used for decryption). It is the modular multiplicative inverse of $e$ modulo $\phi(n)$. [3,
7, 12, 13, 17]

- Condition: Solve for $d$ using the Extended Euclidean Algorithm so that: $$d \times e \equiv 1 \pmod{\phi(n)}$$ [18, 19]
-

## 6. Formulate the Key Pairs

The generated variables are grouped to form the final keys: [3, 7]

- Public Key: Consists of the pair $(e, n)$
- Private Key: Consists of the pair $(d, n)$ [3, 7]

## Once $d$ is successfully generated, the prime components $p$, $q$, and $\phi(n)$ must be securely destroyed to prevent anyone from mathematically recreating the private key from the public key. [13, 20]

## Step-by-Step Toy Example

To see how the algorithm behaves with small integers, here is a working example: [3, 21]

1.  Select Primes: Let $p = 61$ and $q = 53$.
2.  Calculate Modulus: $$n = 61 \times 53 = 3233$$
3.  Compute Totient: $$\phi(n) = (61 - 1) \times (53 - 1) = 60 \times 52 = 3120$$
4.  Choose Public Exponent: Let's pick $e = 17$ (since 17 is coprime to 3120).
5.  Calculate Private Exponent: Find $d$ such that $(d \times 17) \pmod{3120} = 1$. Using the Extended Euclidean Algorithm,
    we find: $$d = 2753$$ (Verification: $2753 \times 17 = 46801$, and $46801 \div 3120 = 15$ with a remainder of $1$) [22,
    23, 24, 25, 26]

- Public Key: $(17, 3233)$
- Private Key: $(2753, 3233)$ [3, 7, 27]
