To clear up a common point of confusion: **there is actually no difference between a TCP/UDP port in IPv4 and a port in
IPv6.** Ports work exactly the same way in both versions of the Internet Protocol. However, the _context_ around how they are
used changes because IPv4 and IPv6 addresses look completely different.

Here is a breakdown of how ports interact with IPv4 vs. IPv6 and what you need to look out for.

---

## 1. The Port Number is Identical

A port number is controlled by the Transport Layer protocols (**TCP** and **UDP**), which sit on top of the Network Layer
(IPv4/IPv6).

Because TCP and UDP don't care which version of IP is running underneath them, the technical specifications for ports remain
unchanged:

- Both IPv4 and IPv6 use a **16-bit integer** for the port number.
- Both have an identical range of available ports: **`0` to `65535**`.
- Standard port assignments are exactly the same (e.g., HTTP is always port `80`, HTTPS is always port `443`, and SSH is
  always port `22`) whether you connect over IPv4 or IPv6.

---

## 2. The Difference is in the Syntax (Notation)

Because an IPv6 address contains colons (`:`), putting a port number at the end of an address can confuse software.

For example, if you typed `2001:db8::1:80`, a computer wouldn't know if the `:80` is part of the IPv6 address or if it's port
`80`.

To solve this, **IPv6 requires square brackets around the IP address** whenever you append a port number.

| Protocol | Address Type          | Syntax with Port   |
| -------- | --------------------- | ------------------ |
| **IPv4** | 32-bit dotted-decimal | `192.168.1.1:80`   |
| **IPv6** | 128-bit hexadecimal   | `[2001:db8::1]:80` |

---

## 3. How Operating Systems Handle "Dual-Stack" Ports

If you are writing code or configuring a server (like Nginx or Apache), you have to tell the operating system how to bind to
ports across IPv4 and IPv6. This is where practical differences appear:

- **Separate Sockets:** By default, an application might open an IPv4 socket on `0.0.0.0:80` and a completely separate IPv6
  socket on `[::]:80`.
- **IPv4-Mapped IPv6 Addresses:** Many modern operating systems support a feature called "dual-stack sockets." This allows a
  server application to listen _only_ on the IPv6 port (`[::]:80`). If an old IPv4 client tries to connect, the OS
  automatically converts the client's IPv4 address into a special IPv6 format (e.g., `[::ffff:192.168.1.1]`) so the
  application can handle both types of traffic through a single port configuration.

---

## 4. No More "Port Forwarding" in IPv6

If you are dealing with home networking or firewalls, the way ports behave feels very different between v4 and v6 due to
**NAT (Network Address Translation)**.

- **In IPv4:** Your router gets one public IP. To host a game server or web server inside your house, you have to configure
  **Port Forwarding** (e.g., tell the router to forward any traffic hitting public port `25565` to a specific internal
  computer).
- **In IPv6:** NAT is obsolete because there are plenty of IP addresses to go around. Every single device on your home
  network gets its own globally unique public IPv6 address. You don't "forward" ports anymore. Instead, you just open the
  specific port on your router's **IPv6 firewall** to allow traffic directly through to that specific machine.
