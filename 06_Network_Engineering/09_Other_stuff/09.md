A **Host header** is a mandatory field in an **HTTP/1.1 (and newer)** request that tells the server which specific domain
name the client is trying to reach.

Because many different websites can live on a single server with a single IP address, the Host header acts as the "label"
that ensures your request lands on the correct site.

---

### Why It Exists: Virtual Hosting

In the early days of the web, one IP address usually equaled one website. As the internet grew, this became inefficient. Web
servers started using **Virtual Hosting** to host multiple domains (e.g., `site-a.com` and `site-b.com`) on one machine.

When a request arrives at the server's IP address, the server looks at the Host header to decide which folder or application
should handle the request.

---

### How It Works in a Request

When you type `www.example.com` into your browser, the browser performs a DNS lookup to find the IP address. Once connected,
it sends a plain-text request that looks like this:

```http
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0...
Accept: text/html...

```

- **The Problem:** The server knows you want `/index.html`, but it doesn't know _which_ site's index page you want based on
  the IP alone.
- **The Solution:** The `Host: www.example.com` line tells the server exactly which virtual host to wake up.

---

### Common Use Cases

- **Shared Hosting:** Thousands of small websites often share a few dozen IP addresses. The Host header is the only way the
  server knows which customer's site to display.
- **Reverse Proxies & Load Balancers:** Tools like **Nginx** or **HAProxy** use the Host header to decide which internal
  "backend" server should receive the traffic.
- **CDNs (Content Delivery Networks):** When you request a file from a provider like Cloudflare, they use the Host header to
  identify which client’s cached data to serve you.

---

### Security Risks: Host Header Injection

If a web application isn't configured correctly, it might trust the Host header too much. An attacker can manually change the
Host header in a request to something else.

> **Example:** If a website uses the Host header to generate "Password Reset" links in emails, an attacker could inject their
> own domain. When the victim clicks the link, their secret reset token is sent to the attacker's server instead of the real
> one.

---

### Host Header vs. SNI

People often confuse these two because they both deal with domain names, but they happen at different layers of the
connection:

| Feature     | Host Header                                     | SNI (Server Name Indication)                        |
| ----------- | ----------------------------------------------- | --------------------------------------------------- |
| **Layer**   | **Application Layer** (HTTP)                    | **Transport Layer** (TLS/SSL)                       |
| **Timing**  | Sent _after_ the connection is secure.          | Sent _before_ the connection is secure.             |
| **Purpose** | Tells the web server which **content** to show. | Tells the server which **SSL certificate** to show. |
