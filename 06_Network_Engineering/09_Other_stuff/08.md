**Server Name Indication (SNI)** is an extension of the TLS (Transport Layer Security) protocol that allows a client (like
your browser) to tell the server exactly which hostname it is trying to connect to at the start of the "handshake" process.

It’s essentially the digital equivalent of putting a **"Attention: [Name]"** line on an envelope delivered to a large
apartment building.

---

### The Problem SNI Solves

Before SNI, a server identified which SSL/TLS certificate to show based solely on the **IP address**.

This worked fine when one server (one IP) hosted exactly one website. However, modern web servers use **Virtual Hosting** to
host hundreds or thousands of websites on a single IP address.

Without SNI:

1. The client connects to an IP address and asks for an encrypted session.
2. The server doesn't know _which_ of its 500 hosted websites the client wants yet (because the specific HTTP "Host" header
   is still encrypted).
3. The server sends a default certificate.
4. If the default certificate doesn't match the website the user typed in, the browser throws a "Your connection is not
   private" security warning.

---

### How SNI Works

SNI inserts the domain name (e.g., `example.com`) into the **Client Hello** message—the very first step of the TLS handshake.

1. **Client Hello:** The browser sends a message saying: _"I'd like to start an encrypted connection, and by the way, I'm
   looking for **example.com**."_
2. **Server Selection:** The server sees the name "example.com," looks through its library of certificates, and picks the one
   that matches.
3. **Server Hello:** The server sends back the correct certificate.
4. **Encryption:** The handshake continues, and the rest of the communication is secured.

---

### Technical Constraints and Privacy

- **Cleartext Leakage:** Because the SNI extension is sent in the _Client Hello_ (before encryption is established), anyone
  monitoring the network (like an ISP or a firewall) can see which website you are visiting, even though they can't see the
  content of your visit.
- **The Fix (ECH):** To solve this privacy leak, a newer standard called **Encrypted Client Hello (ECH)** is being rolled
  out. It encrypts the SNI information so that not even the hostname is visible to prying eyes.

---

### Why it Matters Today

- **IPv4 Scarcity:** SNI is the reason we don't need a unique, expensive IPv4 address for every single website on the
  internet.
- **Cloud & CDNs:** Services like Cloudflare, Akamai, and AWS CloudFront rely entirely on SNI to route traffic for millions
  of customers using a shared pool of IP addresses.
- **Microservices:** It allows internal load balancers to route encrypted traffic to the correct backend service based on the
  requested name.

### Summary Table

| Aspect               | Without SNI                   | With SNI                          |
| -------------------- | ----------------------------- | --------------------------------- |
| **IP Usage**         | 1 IP per Certificate          | Many Certificates per IP          |
| **Server Knowledge** | Only knows the IP address     | Knows the specific Domain Name    |
| **Initial Header**   | Generic/Default Certificate   | Correct Site-Specific Certificate |
| **Privacy**          | Hidden (but connection fails) | Hostname is visible in cleartext  |
