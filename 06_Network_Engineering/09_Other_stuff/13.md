To get a free, globally trusted SSL certificate for your Apache website on Ubuntu, the gold standard is **Let's Encrypt**.

The process is fully automated using a tool called **Certbot**. It will verify that you own the domain, issue the
certificate, configure Apache to use it, and handle automatic renewals.

Before you begin, ensure your domain name (e.g., `example.com`) is already pointed to your Ubuntu server's public IP address
via your DNS provider.

---

### Step 1: Verify your Apache Virtual Host

Certbot needs to look at your Apache configuration to know what domain it is securing.

Open your site's configuration file (replace `your_domain.conf` with your actual file name):

```bash
sudo nano /etc/apache2/sites-available/your_domain.conf

```

Ensure the `ServerName` (and optionally `ServerAlias`) matches your actual domain name:

```apache
ServerName your_domain.com
ServerAlias www.your_domain.com

```

Save and close the file (`Ctrl+X`, then `Y`, then `Enter`). If you made changes, reload Apache:

```bash
sudo systemctl reload apache2

```

### Step 2: Install Certbot

Update your local package index and install Certbot along with its Apache plugin:

```bash
sudo apt update
sudo apt install certbot python3-certbot-apache

```

### Step 3: Adjust the Firewall (If Active)

If you have `ufw` enabled, you need to make sure HTTPS traffic is allowed through. Check your status:

```bash
sudo ufw status

```

If it is active, allow the "Apache Full" profile, which opens both port 80 (HTTP) and port 443 (HTTPS):

```bash
sudo ufw allow 'Apache Full'

```

### Step 4: Run Certbot to Get and Install the Certificate

Run the interactive Certbot script for Apache:

```bash
sudo certbot --apache

```

**What happens next:**

1. **Email:** It will ask for your email address (used for renewal warnings if something goes wrong).
2. **Terms of Service:** Agree to the terms by typing `A`.
3. **Domain Selection:** Certbot will read your Apache configs and list your domains. Press **Enter** to select all of them,
   or type the numbers separated by commas.
4. **Automation:** Certbot will talk to Let's Encrypt, complete a quick verification background challenge, generate your
   certificates, and rewrite your Apache configuration to activate HTTPS.

Once complete, it will output a success message showing where your certificate files were saved.

---

### Step 5: Verify Auto-Renewal

Let's Encrypt certificates are valid for 90 days, but the Certbot installation automatically installs a background system
timer that checks and renews your certificates twice a day when they have less than 30 days left.

You can verify that the renewal service works flawlessly by running a dry run:

```bash
sudo certbot renew --dry-run

```
