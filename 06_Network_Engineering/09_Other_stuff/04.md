**MD5** (Message Digest Algorithm 5) is a widely used cryptographic hash function that takes an input (a string, a file, or
any data) and produces a **128-bit hash value**, typically rendered as a 32-digit hexadecimal number.

In the past, it was the gold standard for security, but today it is considered **cryptographically broken** and should never
be used for protecting sensitive data like passwords.

---

### How It Works

MD5 acts like a "digital fingerprint." If you change even a single character in a 500-page book and run it through the MD5
algorithm, the resulting hash will look completely different.

1. **Padding:** The algorithm adds bits to the input data so its length is congruent to 448 modulo 512.
2. **Appended Length:** A 64-bit representation of the original message length is added.
3. **Initialization:** It uses four 32-bit variables (A, B, C, and D) as a starting buffer.
4. **Processing:** The data is processed in 512-bit blocks. Each block goes through four rounds of complex mathematical
   operations (using logical functions like `AND`, `OR`, and `XOR`).
5. **Output:** The final values of the four variables are concatenated to create the 128-bit hash.

---

### Current Use Cases

Because MD5 is very fast to calculate, it is still used in non-security contexts:

- **Checksums:** To verify that a file wasn't corrupted during a download (e.g., "Does the MD5 of the file on my disk match
  the one on the website?").
- **Data Indexing:** Generating unique keys for database lookups or "fingerprinting" non-sensitive assets to check for
  duplicates.

---

### Why It Is Insecure

MD5 suffers from a fatal flaw known as **Collision Vulnerability**. A collision occurs when two different inputs produce the
exact same hash.

- **Collision Attacks:** Researchers have demonstrated that they can create two different files (e.g., a "good" software
  installer and a "malicious" one) that have the same MD5 hash.
- **Speed:** Because it is computationally "cheap," hackers can use consumer-grade GPUs to guess billions of MD5 hashes per
  second, making "brute-forcing" passwords trivial.

### Comparison Table

| Feature             | MD5                                  | SHA-256 (Modern Standard)      |
| ------------------- | ------------------------------------ | ------------------------------ |
| **Output Size**     | 128 bits                             | 256 bits                       |
| **Security Status** | **Broken** (Do not use for security) | Secure                         |
| **Speed**           | Extremely Fast                       | Slower (More complex)          |
| **Common Use**      | File integrity, Legacy systems       | SSL/TLS, Passwords, Blockchain |

---

### Summary

Think of MD5 like a **padlock from the 1970s**. It’s fine for keeping a locker shut at the gym so your gym bag doesn't fall
out, but you wouldn't use it to lock a vault containing millions of dollars. For anything involving security, encryption, or
sensitive data, you should use **SHA-256** or **SHA-3** instead.
