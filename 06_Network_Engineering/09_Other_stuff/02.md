The command `export SSLKEYLOGFILE` is used to tell your operating system (and specific applications) where to save the
**secrets** used to encrypt your internet traffic.

Essentially, it creates a "cheat sheet" that allows diagnostic tools to decrypt and read your HTTPS/TLS traffic.

---

### How It Works

When you set this environment variable, browsers like **Chrome** or **Firefox** and libraries like **OpenSSL** will
automatically write the per-session secrets (specifically the "client random" and "pre-master secret") into a text file.

> **Note:** Without this file, your traffic looks like gibberish to anyone watching. With this file, tools like **Wireshark**
> can use those keys to turn that gibberish back into readable text (HTML, JSON, passwords, etc.).

### Common Usage

Most people use this command for **troubleshooting or security analysis**. A typical workflow looks like this:

1. **Set the file path:** `export SSLKEYLOGFILE=~/Documents/mykeys.log`
2. **Launch the application:** Open Chrome or Firefox from that same terminal.
3. **Capture traffic:** Run Wireshark, point it to `mykeys.log` in the settings, and watch the encrypted traffic become plain
   text in real-time.

### Security Warning

**This is a powerful debugging tool, but it is also a security risk.** \* **Don't leave it on:** If you forget this variable
is active, every "secure" site you visit will have its session keys saved in plain text on your hard drive.

- **Privacy:** Anyone with access to that log file and a capture of your network traffic can see exactly what you were doing,
  even on HTTPS sites.

---

### Technical Context

In the context of the TLS handshake, the keylog file follows a specific format (usually the **NSS Key Log Format**). This
allows the tool to match a specific network packet to the math required to unlock it using the following formula:

$$\text{Pre-Master Secret} + \text{Client Random} + \text{Server Random} \rightarrow \text{Master Secret}$$

The `SSLKEYLOGFILE` provides the **Master Secret** or the components needed to derive it, bypassing the need to "break" the
encryption through brute force.
