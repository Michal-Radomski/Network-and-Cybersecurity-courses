`nc` (Netcat) is often called the "Swiss Army knife" of networking. It’s a powerful command-line utility used on Ubuntu for
reading from and writing to network connections using TCP or UDP.

On Ubuntu, `nc` comes pre-installed by default (usually the OpenBSD variant). If for some reason it's missing, you can
install it using:

```bash
sudo apt update && sudo apt install netcat-openbsd

```

Here is a guide on how to use `nc` for the most common networking tasks.

---

## 1. Quick Banner Grabbing / Testing a Port

The simplest use of Netcat is to check if a specific port on a remote server is open and see what service is running behind
it.

```bash
nc [IP_or_hostname] [port]

```

- **Example:** Connect to a web server on port 80:

```bash
nc www.google.com 80

```

If you type `GET / HTTP/1.1` and hit Enter twice, the server will dump the raw HTML/header response to your terminal.

- **Example with a timeout (`-w`):** If you don't want Netcat hanging forever waiting for a connection, add a timeout in
  seconds:

```bash
nc -w 5 192.168.1.50 22

```

---

## 2. Scanning Ports

While `nmap` is the king of port scanning, Netcat can do quick, lightweight scans using the **`-z`** flag (zero-I/O mode,
which reports status without sending data) and **`-v`** (verbose).

- **Scan a specific port range:**

```bash
nc -zv 192.168.1.10 20-80

```

- **Scan using UDP instead of TCP (`-u`):**

```bash
nc -zuv 192.168.1.10 53

```

---

## 3. Creating a Simple Chat Server

You can use Netcat to pass text instantly between two machines on the same network.

1. **On Machine A (The Listener/Server):** Tell Netcat to listen (**`-l`**) on a specific port (e.g., `4444`):

```bash
nc -l 4444

```

2. **On Machine B (The Client):** Connect to Machine A's IP address and that same port:

```bash
nc 192.168.1.20 4444

```

Now, anything typed into the terminal on Machine A will show up on Machine B, and vice versa. Press `Ctrl + C` to end the
session.

---

## 4. Transferring Files

Netcat is incredibly handy for moving a file between two Linux machines without dealing with SSH, FTP, or USB drives.

1. **On the Receiving Machine:** Set up Netcat to listen on a port and redirect (`>`) the incoming data stream into a new
   file:

```bash
nc -l 5555 > received_file.txt

```

2. **On the Sending Machine:** Pipe (`<`) the file you want to send directly into Netcat, pointed at the receiver's IP:

```bash
nc 192.168.1.30 5555 < file_to_send.txt

```

Once the transfer finishes, Netcat will automatically close.

---

## 5. Serving a Basic Web Page

You can use Netcat to quickly host an HTML file for testing purposes.

1. Create a simple HTML file:

```bash
echo "<h1>Hello from Ubuntu Netcat Server</h1>" > index.html

```

2. Run a loop that serves this file over port 8080 whenever someone connects:

```bash
while true; do printf "HTTP/1.1 200 OK\r\n\r\n"; cat index.html; done | nc -l 8080

```

Now open a web browser and go to `http://localhost:8080` (or your Ubuntu machine's IP) to see the page.

## Common `nc` Flags Cheat Sheet

| Flag            | Description                                                                                  |
| --------------- | -------------------------------------------------------------------------------------------- |
| **`-l`**        | **Listen mode.** Used to turn Netcat into a server waiting for incoming connections.         |
| **`-p [port]`** | **Port.** Explicitly specifies the source port (usually paired with `-l`).                   |
| **`-v`**        | **Verbose.** Prints more details about what Netcat is doing (crucial for troubleshooting).   |
| **`-n`**        | **Numeric only.** Disables DNS lookup to speed up connection testing.                        |
| **`-u`**        | **UDP mode.** Switches Netcat from standard TCP to UDP.                                      |
| **`-w [secs]`** | **Timeout.** Drops the connection attempt after the specified number of seconds.             |
| **`-z`**        | **Zero-I/O mode.** Used exclusively for scanning (reports port status without sending data). |
