To understand where the IP packet sits, you have to look at **Encapsulation**. Think of it like a Russian nesting doll: the
data is put inside a TCP segment, which is put inside an IP packet, which is finally tucked inside a **Network Frame**.

The Frame is the outermost "envelope" used at the **Data Link Layer (Layer 2)** to move data across physical wires or Wi-Fi
signals.

---

## 1. The Anatomy of an Ethernet Frame

Since Ethernet is the most common wired standard, let's look at how an Ethernet II frame is structured. It "wraps" around the
IP packet.

### The Frame Header

- **Preamble & SFD:** A series of bits that tell the receiving hardware, "Get ready, a frame is coming!"
- **Destination MAC Address:** The hardware ID of the next "hop" (like your router).
- **Source MAC Address:** The hardware ID of the device sending the frame.
- **EtherType:** A code that tells the receiver what is inside the payload. For an IP packet, the code is usually `0x0800`.

### The Payload (Where the IP Packet lives)

- **This is the "cargo" area.** The entire **IP Packet** (Header + Data) is placed right here. The frame doesn't care what's
  in the packet; it just carries it.

### The Frame Trailer

- **FCS (Frame Check Sequence):** A mathematical checksum (CRC) used to see if the frame was corrupted or "glitched" during
  its physical travel over the wire. If the math doesn't add up, the receiver throws the frame away.

---

## 2. The "Nesting" Visual

If you were to peel back the layers of a single transmission, it would look like this from the outside in:

| Layer                   | Unit Name     | What's added?                         |
| ----------------------- | ------------- | ------------------------------------- |
| **Layer 2 (Link)**      | **Frame**     | MAC Addresses + Error Check (FCS)     |
| **Layer 3 (Network)**   | **IP Packet** | IP Addresses + TTL + IHL              |
| **Layer 4 (Transport)** | **Segment**   | Port Numbers (e.g., 80 or 443)        |
| **Layer 7 (App)**       | **Data**      | The actual message (HTTP, Image bits) |

---

## 3. Why the distinction matters

A **Router** stripped away the Frame to look at the **IP Packet** inside. It reads the IP address, decides where the packet
needs to go next, and then wraps it in a **brand new Frame** with new MAC addresses to send it to the next stop.

### Summary: Where is the IP Packet?

The IP packet is the **Payload** of the Network Frame.

- The **Frame** gets it from one physical network card to the next.
- The **IP Packet** gets it from the original sender to the final destination.
