Think of a **Subnet Mask** as a filter or a divider for an IP address. Its entire job is to tell a computer which part of an
IP address refers to the **network** (the neighborhood) and which part refers to the **host** (the specific house).

Without a subnet mask, a device wouldn't know if it’s talking to a neighbor on its own local network or if it needs to send
the data to a router to reach a different network across the internet.

---

## 1. How It Works

An IP address is 32 bits long. A subnet mask is also 32 bits long. When you "overlay" the mask onto the IP address, the
**1s** in the mask lock in the network portion, and the **0s** identify the host portion.

### The Visual Breakdown

Imagine the IP address `192.168.1.50` with a standard subnet mask of `255.255.255.0`.

- **IP Address:** `192.168.1.50`
- **Subnet Mask:** `255.255.255.0`

In this case:

- **Network Portion:** `192.168.1.x` (The first three segments are the "street name")
- **Host Portion:** `.50` (The specific "house number")

---

## 2. Common Subnet Masks

You will usually see these three types, which correspond to the size of the network:

| Class       | Subnet Mask     | Number of Available Hosts | Use Case                        |
| ----------- | --------------- | ------------------------- | ------------------------------- |
| **Class A** | `255.0.0.0`     | 16,777,214                | Massive corporations or ISPs    |
| **Class B** | `255.255.0.0`   | 65,534                    | Large universities or hospitals |
| **Class C** | `255.255.255.0` | 254                       | Home Wi-Fi and small offices    |

---

## 3. CIDR Notation (The Shorthand)

Technicians often find typing out `255.255.255.0` tedious, so they use **CIDR (Classless Inter-Domain Routing)** notation.
This is just a slash followed by the number of "1" bits in the mask.

- `255.255.255.0` becomes **`/24`** (because there are 24 "ones" in binary).
- `255.0.0.0` becomes **`/8`**.

---

## 4. Why Do We Use Them?

1. **Organization:** It allows a large network (like a big company) to be broken down into smaller, manageable departments
   (Accounting, HR, IT).
2. **Efficiency:** It limits "broadcast traffic." If every computer in a 10,000-person company were on the same subnet, the
   network would crawl because of all the background "noise" and chatter.
3. **Security:** You can use subnets to wall off sensitive data. For example, the Guest Wi-Fi shouldn't be on the same subnet
   as the Server Room.
