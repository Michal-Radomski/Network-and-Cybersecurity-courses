In the context of the IPv4 packet, **Header Options** are like "add-ons" or "accessories." While the standard IP header
contains all the essential info for routing, the Options field allows for extra features that aren't needed for every single
packet.

As we discussed with the **Internet Header Length (IHL)**, the presence of these options is what causes the header size to
grow beyond the standard 20 bytes.

---

## 1. Why use Header Options?

Options are generally used for network testing, debugging, or specific security requirements. They are rarely seen in
standard web traffic today because they take extra processing power for routers to read, which can slow down the network.

---

## 2. Common Types of Options

Here are the most well-known options you might encounter in a networking lab or a packet capture:

### Record Route

This tells every router that handles the packet to "sign" its IP address in the header. By the time the packet reaches the
destination, you have a list of every stop it made.

- **Use Case:** Mapping the path of a packet (similar to how `tracert` or `traceroute` works, but at the IP level).

### Timestamp

Similar to Record Route, but instead of just writing the IP address, each router also records the **exact time** it handled
the packet.

- **Use Case:** Measuring delay (latency) at specific points along a network path.

### Loose & Strict Source Routing

Usually, routers decide the best path for a packet. Source routing allows the **sender** to specify the exact path the packet
must take.

- **Strict:** The sender lists every single jump. If a router isn't on that list, the packet is dropped.
- **Loose:** The sender lists a few "must-visit" checkpoints, but the packet can take any path to get between them.
- **Security Note:** Most modern routers disable this because hackers can use it to bypass firewalls or "spoof" their
  location.

### Router Alert

This tells the router to look more closely at the packet's content rather than just passing it along as fast as possible.

- **Use Case:** Used for protocols like RSVP (Resource Reservation Protocol).

---

## 3. The "Padding" Problem

IP headers must end on a **32-bit boundary**. If the options you add don't end perfectly at a multiple of 4 bytes, the sender
adds "Padding" (extra zeros).

Think of it like shipping a small item in a large box—you add bubble wrap (padding) to make sure everything fits snugly and
the box keeps its shape.

---

## 4. Why are they disappearing?

In the transition to **IPv6**, the "Options" field was completely removed from the main header.

Modern networking prioritizes speed. Reading variable-length options in IPv4 requires routers to use their CPU (software)
instead of their fast-path hardware (ASICs). To fix this, IPv6 uses **Extension Headers**, which are much more efficient and
don't slow down the main routing process.
