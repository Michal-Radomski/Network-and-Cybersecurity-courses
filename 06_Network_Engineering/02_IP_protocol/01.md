# Internet Protocol (IP)

The **Internet Protocol (IP)** is a set of rules used to send and receive data across networks and the Internet. It is a core
protocol of the **TCP/IP** networking model and operates at the **Network Layer (Layer 3)** of the OSI model.

IP is responsible for:

- Addressing devices
- Packaging data into packets
- Routing packets between networks

---

# Main Purpose of IP

Internet Protocol helps:

1. Identify devices uniquely
2. Transfer data from source to destination
3. Route packets across interconnected networks

---

# How IP Works

When data is sent over a network:

1. Data is divided into small units called **packets**
2. Each packet gets:
   - Source IP address
   - Destination IP address

3. Routers examine the destination IP
4. Packets travel through different paths
5. Destination device reassembles packets

---

# IP Address

An **IP address** is a unique logical address assigned to a device on a network.

Example:

```text id="xv7akd"
192.168.1.10
```

# Functions of Internet Protocol

## 1. Addressing

Assigns unique IP addresses to hosts.

## 2. Routing

Determines the path packets follow.

## 3. Packet Fragmentation

Breaks large packets into smaller ones if needed.

## 4. Encapsulation

Encapsulates transport-layer data into packets.

# IP Packet Structure

An IP packet contains:

| Field               | Purpose             |
| ------------------- | ------------------- |
| Source Address      | Sender IP           |
| Destination Address | Receiver IP         |
| Header              | Control information |
| Data                | Actual message      |

---

# Characteristics of IP

## Connectionless

IP does not establish a dedicated connection before sending data.

## Best-Effort Delivery

IP does not guarantee:

- Delivery
- Order
- Error recovery

Reliability is handled by TCP.

---


The best way to think about the difference between the **Internet Protocol (IP)** and **TCP/UDP** is to imagine a postal
service. IP is the **envelope and the address** that gets the mail to the right house, while TCP and UDP are the
**instructions inside** that tell the recipient how to handle the contents.

---

## 1. The Internet Protocol (IP)

IP operates at the **Network Layer** (Layer 3). Its primary job is addressing and routing. It doesn't care what's inside the
"package"; it only cares about getting it to the correct destination.

- **Responsibility:** Delivering packets from Point A to Point B using IP addresses.
- **Analogy:** A GPS system. It knows how to find the house, but it doesn't care who is inside or if they actually open the
  door.
- **Reliability:** None. IP is "best-effort." If a packet gets lost in the mail, IP doesn't try to find it.

---

## 2. TCP and UDP (The Transport Layer)

These protocols live at the **Transport Layer** (Layer 4). They sit "on top" of IP and define how data is exchanged between
applications.

### TCP (Transmission Control Protocol)

TCP is the "perfectionist." It ensures that every single bit of data arrives correctly and in the right order.

- **Connection-Oriented:** It performs a "three-way handshake" to establish a connection before sending data.
- **Error Checking:** If a packet is lost, TCP asks for it to be resent.
- **Use Cases:** Web browsing (HTTP), Email, File transfers.

### UDP (User Datagram Protocol)

UDP is the "speedster." It throws data at the destination without checking if it arrived.

- **Connectionless:** No handshake; it just starts sending.
- **No Recovery:** If data is lost, it stays lost. This is fine for things where speed is more important than 100% accuracy.
- **Use Cases:** Video streaming, Gaming, VoIP (calls).

---

## 3. Comparison Table

| Feature          | IP (Internet Protocol)             | TCP / UDP                           |
| ---------------- | ---------------------------------- | ----------------------------------- |
| **Layer**        | Network Layer (Layer 3)            | Transport Layer (Layer 4)           |
| **Primary Goal** | **Where** the data goes (Routing)  | **How** the data is sent (Delivery) |
| **Unit of Data** | Packets                            | Segments (TCP) / Datagrams (UDP)    |
| **Addressing**   | IP Addresses (e.g., `192.168.1.1`) | Ports (e.g., `Port 80`, `Port 443`) |
| **Analogy**      | The address on the envelope        | The letter/instructions inside      |

---

### How they work together

They aren't competitors; they are partners. In a standard web request, your data is wrapped in a **TCP segment** (to ensure
it arrives intact), which is then tucked inside an **IP packet** (to ensure it finds the right server).
