`tcpdump` is a powerful command-line packet analyzer that allows you to capture and inspect network traffic going to and from
your system. It is an essential tool for network troubleshooting and security analysis.

Because it interacts directly with network interfaces, you will almost always need to run it with administrative privileges
(using `sudo`).

---

## 1. The Basic Syntax

The fundamental structure of a `tcpdump` command is:

```bash
sudo tcpdump [options] [filters]

```

- **Options:** Flags that change how `tcpdump` behaves (e.g., saving to a file, changing verbosity).
- **Filters:** Expressions that limit which packets are captured (e.g., only traffic from a specific IP or port).

---

## 2. Common Options (How it captures)

By default, running just `sudo tcpdump` will flood your screen with every packet it sees on your default interface. Use these
options to control the output:

- **`-i [interface]`**: Specify which network interface to listen on (e.g., `eth0`, `wlan0`).
- _Tip:_ Use `sudo tcpdump -D` to list all available interfaces.

- **`-c [number]`**: Stop capturing after a specific number of packets.
- `sudo tcpdump -c 10` (Captures exactly 10 packets and stops).

- **`-n`**: Don't resolve IP addresses to hostnames (speeds up the process and shows raw IPs).
- **`-nn`**: Don't resolve IPs _or_ port numbers (e.g., shows `80` instead of `http`).
- **`-v`, `-vv`, `-vvv**`: Increase the amount of packet detail displayed (verbosity).
- **`-A`**: Print the packet body in ASCII text (great for reading unencrypted HTTP traffic or payloads).

---

## 3. Using Filters (What it captures)

Filters are the most powerful part of `tcpdump`. They allow you to isolate the exact traffic you care about.

### Filter by Host (IP Address)

- Traffic to or from a specific IP:

```bash
sudo tcpdump host 192.168.1.50

```

- Traffic coming _from_ a specific IP only:

```bash
sudo tcpdump src 192.168.1.50

```

- Traffic going _to_ a specific IP only:

```bash
sudo tcpdump dst 192.168.1.50

```

### Filter by Port or Service

- Traffic on a specific port (like HTTP):

```bash
sudo tcpdump port 80

```

- Traffic on a range of ports:

```bash
sudo tcpdump portrange 21-25

```

### Filter by Protocol

- Capture only ICMP (ping) traffic, UDP, or TCP:

```bash
sudo tcpdump icmp
sudo tcpdump udp

```

### Combining Filters (`and`, `or`, `not`)

You can combine filters to get highly specific.

- Capture traffic from IP `192.168.1.50` **and** using port `443` (HTTPS):

```bash
sudo tcpdump src 192.168.1.50 and port 443

```

- Capture all traffic **except** SSH (port 22) so your own terminal session doesn't clutter the screen:

```bash
sudo tcpdump not port 22

```

---

## 4. Saving and Reading Captures (PCAP)

Reading packets in real-time on a terminal can be overwhelming. It is often better to save the capture to a `.pcap` file so
you can analyze it later or open it in a graphical tool like **Wireshark**.

- **To write (save) a capture to a file (`-w`):**

```bash
sudo tcpdump -i eth0 -w network_traffic.pcap

```

_(Note: This saves the raw packets, so nothing will print to your screen while it's running. Press `Ctrl + C` to stop it.)_

- **To read (play back) a saved file (`-r`):**

```bash
tcpdump -r network_traffic.pcap

```

_(Note: You do not need `sudo` to read a saved file)._

---

## 5. Practical Examples to Try

1. **The "What's happening right now?" command:** See clean, numeric IP/port data on interface `eth0`, limiting it to 20
   packets so it doesn't overwhelm you:

```bash
sudo tcpdump -i eth0 -nn -c 20

```

2. **Inspect website traffic text:** Watch port 80 traffic and print the payload in plain text to see HTTP headers:

```bash
sudo tcpdump -i any port 80 -A

```

3. **Isolate a specific device:** See if your router (`192.168.1.1`) is talking to your machine:

```bash
sudo tcpdump host 192.168.1.1

```
