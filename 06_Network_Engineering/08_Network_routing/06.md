A **routing table** is a digital roadmap stored in the memory of a network device—typically a router or a layer 3 switch. It
consists of a list of rules and destinations that the device uses to determine where to forward IP packets traveling across a
network.

Without a routing table, a router wouldn't know what to do with incoming data; it would be like a postal worker trying to
sort mail without a map of the city.

---

## How a Routing Table Looks (The Core Fields)

Every row in a routing table represents a specific network path. While different operating systems (like Cisco IOS, Linux, or
Windows) format them slightly differently, they all contain the same fundamental fields:

| Destination Network | Subnet Mask / CIDR      | Next Hop (Gateway) | Interface | Metric / Cost |
| ------------------- | ----------------------- | ------------------ | --------- | ------------- |
| `192.168.1.0`       | `255.255.255.0` (`/24`) | `On-link` (Direct) | `eth0`    | `0`           |
| `10.0.0.0`          | `255.0.0.0` (`/8`)      | `192.168.1.254`    | `eth0`    | `20`          |
| `0.0.0.0`           | `0.0.0.0` (`/0`)        | `73.55.100.1`      | `WAN`     | `1`           |

### Breakdown of the Fields:

1. **Destination Network:** The target IP network address that a packet is trying to reach.
2. **Subnet Mask / CIDR:** Defines the size of the target network so the router knows exactly which bits of the IP address
   represent the network ID.
3. **Next Hop (Gateway):** The IP address of the _next_ router along the path. If the destination network is directly plugged
   into the router, this will say "Direct" or "On-link".
4. **Interface:** The physical or virtual network port (e.g., `eth0`, `Wi-Fi`, `WAN`) that the router must shoot the packet
   out of to reach the next hop.
5. **Metric / Cost:** A value used to break ties. If there are three different paths to the exact same destination, the
   router will choose the path with the lowest metric (usually based on speed, bandwidth, or hop count).

## How a Router Uses the Table: The Matching Process

When a packet arrives at a router port, the router extracts the **Destination IP Address** from the packet's header. It then
goes through a strict logic sequence:

### 1. Longest Match Prefix Rule (The Priority Rule)

The router compares the destination IP against all entries in its table. If the IP matches multiple rows, it **always**
chooses the most specific route—meaning the one with the longest subnet mask (highest CIDR prefix).

- _Example:_ If a packet is going to `192.168.1.50`, and the table has a route for `192.168.1.0/24` and another for
  `192.168.0.0/16`, the router chooses `192.168.1.0/24` because it is a tighter, more specific match.

### 2. Direct Delivery

If the destination network is directly connected to one of the router's interfaces, the router skips looking for another
gateway. It uses ARP (Address Resolution Protocol) to find the exact hardware MAC address of the destination device and
delivers it natively.

### 3. The Default Route (The Catch-All)

If the destination IP does not match _any_ specific network listed in the routing table, the router falls back to the
**Default Route** (represented as `0.0.0.0/0`). This is essentially the rule that says: _"If you don't know where it goes,
send it to the Internet Gateway."_

> 🛑 **What happens if there is no match and no default route?** The router will immediately drop the packet and send an ICMP
> error message back to the sender, typically stating: `"Destination Host Unreachable."`

## How Routing Tables are Populated

Routers don't just inherently know the layout of the world; their tables must be filled using three distinct methods:

- **Connected Routes:** These are added automatically. The moment you plug a live network cable into a router's interface and
  give it an IP address, the router adds that local subnet to its routing table.
- **Static Routes:** These are typed in manually by a network administrator. They are highly secure and don't change unless
  an admin changes them.
- **Dynamic Routes:** These are learned automatically via routing protocols (like OSPF, BGP, or EIGRP). Routers constantly
  talk to neighboring routers, exchanging information to update their tables dynamically if a network segment goes down or a
  faster path opens up.

## How to See Your Own Device's Routing Table

Your computer or phone has its own routing table built right into the operating system so it knows whether to talk to a local
device on your Wi-Fi or send data to your router.

You can view it right now using your terminal:

- **Windows (Command Prompt):** Type `route print` or `netstat -r`
- **Linux / macOS (Terminal):** Type `ip route` or `netstat -nr`
