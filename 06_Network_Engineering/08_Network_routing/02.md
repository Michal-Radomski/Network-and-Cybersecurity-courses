In computer networking, the **data link** refers to the connection between two physically adjacent devices that allows them
to transfer data back and forth.

More formally, it represents **Layer 2 (the Data Link Layer)** of the Open Systems Interconnection (OSI) model. While the
Network Layer (Layer 3) handles the big picture of routing data across the entire internet, the Data Link Layer is strictly
responsible for moving data across a **single physical link**—such as a cable or a wireless signal—from one specific device
to the very next.

---

## The Two Sublayers of the Data Link Layer

To handle its responsibilities, the Data Link Layer is split into two distinct sublayers:

### 1. Logical Link Control (LLC)

The LLC sublayer acts as the interface between the network software above it and the physical hardware below it. It is
responsible for identifying the network layer protocol being used (like IPv4 or IPv6) and managing flow control to ensure a
fast sender doesn't overwhelm a slow receiver.

### 2. Media Access Control (MAC)

The MAC sublayer deals with the actual physical hardware. It manages physical addressing (MAC addresses) and controls how
devices share access to a common transmission medium (like Wi-Fi radio waves or a shared Ethernet switch) to prevent packets
from colliding with one another.

---

## Core Functions of the Data Link Layer

The Data Link Layer performs several critical tasks to ensure data moves reliably from point A to point B:

### 1. Framing

The network layer hands down packets of data. The Data Link Layer takes these packets and encapsulates them into **frames**.
A frame adds a header (containing the starting marker and physical addresses) and a trailer (containing error-checking data)
around the original packet.

### 2. Physical Addressing (MAC Addresses)

While routers use IP addresses to find networks, the Data Link layer uses **MAC (Media Access Control) addresses** to find
specific hardware. A MAC address is a permanent, 48-bit physical address burned into a device's Network Interface Card (NIC)
at the factory.

> **The Difference:** An IP address is like your mailing address (it changes if you move). A MAC address is like your
> fingerprint (it stays with the device forever).

### 3. Error Detection

As data travels over wires or through the air, electrical interference or signal degradation can corrupt the data (flipping a
`1` to a `0`). The Data Link layer uses a mechanism called a **Cyclic Redundancy Check (CRC)**.

- The sender calculates a mathematical code based on the frame data and puts it in the trailer.
- The receiver runs the same calculation.
- If the codes don't match, the receiver knows the frame was corrupted and discards it.

### 4. Media Access Control (Flow & Collision Management)

If two devices on the same Wi-Fi network try to transmit data at the exact same millisecond, their signals crash into each
other, resulting in corrupted data. The Data Link layer uses protocols like **CSMA/CD** (for wired Ethernet) or **CSMA/CA**
(for Wi-Fi) to sense when the line is clear before transmitting.

---

## Common Data Link Protocols and Technologies

Depending on how you are connecting to a network, you are interacting with different data link technologies:

- **Ethernet (IEEE 802.3):** The standard for wired local area networks (LANs).
- **Wi-Fi (IEEE 802.11):** The standard for wireless local area networks (WLANs).
- **PPP (Point-to-Point Protocol):** Used to establish a direct connection between two nodes, often used by ISPs to connect
  home routers to their network.
- **Switches:** A network switch is a "Layer 2 device." It inspects the MAC addresses inside incoming data frames to forward
  them precisely to the correct destination device on a local network.
