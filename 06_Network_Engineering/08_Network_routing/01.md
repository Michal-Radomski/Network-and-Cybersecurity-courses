At its core, **network routing** is the process of selecting paths in a network along which to send data packets from a
source to a destination.

Think of it as the GPS system for the internet. When you type a website URL into your browser, your data doesn't travel in a
straight, direct line. Instead, it is broken down into small pieces called **packets**, and routers determine the most
efficient street-by-street path to get those packets to their destination.

---

## How Routing Works (The Step-by-Step Journey)

To understand routing, it helps to look at the physical journey a packet takes:

1. **Packet Creation:** Your device chops your data into packets. Each packet contains a **header** (which includes the
   source IP address and the destination IP address) and the **payload** (the actual data).
2. **The Local Gateway:** Your device realizes the destination IP is not on its local network (e.g., your home Wi-Fi), so it
   forwards the packet to its **Default Gateway** (your local router).
3. **Hop-by-Hop Evaluation:** The router inspects the destination IP address. It looks at its internal **Routing Table**—a
   map of known networks—to find the best next stop (called a "hop").
4. **The Hand-off:** The router forwards the packet to the next router. This process repeats across multiple routers across
   the internet.
5. **Arrival:** The final router delivers the packets to the destination server, which reassembles them into the original
   file or webpage.

---

## The Core Components of Routing

Network routing relies on a combination of hardware and data structures to function seamlessly.

### 1. Routers

Routers are specialized network devices operating at **Layer 3 (Network Layer)** of the OSI model. Their primary job is to
connect different networks (e.g., connecting your home network to your Internet Service Provider's network) and forward
packets between them.

### 2. Routing Tables

A routing table is a database stored in a router's memory that lists the routes to particular network destinations. It
typically contains:

- **Destination Network:** The IP address range the router knows how to reach.
- **Next Hop:** The IP address of the next router along the path.
- **Interface:** The physical network port the router should use to send the packet out.
- **Metric/Cost:** A value that helps the router decide which path is "best" if multiple paths exist to the same destination.

---

## Types of Routing

There are three primary ways a router learns about where to send packets:

### Static Routing

An administrator manually configures the routes in the routing table.

- **Pros:** Highly secure, predictable, and consumes no network overhead or CPU.
- **Cons:** Terrible for large networks. If a link goes down, the router doesn't know how to reroute traffic automatically;
  an admin must fix it manually.

### Dynamic Routing

Routers use specialized **routing protocols** to talk to each other, share information about network health, and
automatically discover the best paths.

- **Pros:** Highly scalable and adaptive. If a cable is cut or a router fails, dynamic protocols immediately calculate a
  detour.
- **Cons:** Consumes router CPU and network bandwidth to send updates.

### Default Routing

A catch-all route configured when the router doesn't have a specific path listed for a destination. It essentially tells the
router: _"If you don't know where to send this packet, just send it out of this specific interface (usually toward the
internet)."_

---

## Key Dynamic Routing Protocols

Dynamic routing is divided into two main categories depending on where the routing takes place:

### 1. Interior Gateway Protocols (IGP)

Used to route data _within_ a single organization or private network (like a corporate campus or a data center).

- **OSPF (Open Shortest Path First):** A widely used protocol that builds a complete map of the network topology and uses
  Dijkstra's algorithm to find the fastest path based on bandwidth.
- **EIGRP (Enhanced Interior Gateway Routing Protocol):** A proprietary Cisco protocol (though now partially open) known for
  incredibly fast convergence times.

### 2. Exterior Gateway Protocols (EGP)

Used to route data _between_ different distinct networks or Autonomous Systems (AS) across the global internet.

- **BGP (Border Gateway Protocol):** The literal backbone of the internet. BGP manages how packets travel between massive
  internet service providers (ISPs), tech giants, and universities globally. It bases its decisions on path, network
  policies, and rule-sets rather than just speed.
