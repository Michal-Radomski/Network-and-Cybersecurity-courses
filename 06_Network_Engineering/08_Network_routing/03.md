To determine if two IP addresses belong to the same network, computers use a mathematical process called a **bitwise AND
operation** using the subnet mask.

If you apply the subnet mask to both IP addresses and the resulting **Network ID** is exactly the same, they are on the same
network.

Here is the step-by-step guide on how computers do this, and how you can do it manually.

---

## The Concept: What the Subnet Mask Does

An IP address is made up of two parts: the **Network ID** (the street name) and the **Host ID** (the house number). The
subnet mask tells you exactly where the street name ends and the house number begins.

- A `255` in a subnet mask means _"this part is the network."_
- A `0` in a subnet mask means _"this part belongs to the individual host."_

---

## Method 1: The Easy Way (For Simple Subnet Masks)

If your subnet mask uses standard numbers like `255.255.255.0`, you can check it just by looking at it.

### Example 1: Same Network

- **IP 1:** `192.168.1.50`
- **IP 2:** `192.168.1.75`
- **Subnet Mask:** `255.255.255.0`

Because the first three octets of the subnet mask are `255.255.255`, the first three numbers of the IP addresses **must match
exactly**.

- IP 1 Network part: **`192.168.1`**`.50`
- IP 2 Network part: **`192.168.1`**`.75`

They match perfectly. **Result: Same network.**

### Example 2: Different Networks

- **IP 1:** `192.168.1.50`
- **IP 2:** `192.168.2.75`
- **Subnet Mask:** `255.255.255.0`
- IP 1 Network part: **`192.168.1`**`.50`
- IP 2 Network part: **`192.168.2`**`.75`

Because `192.168.1` does not equal `192.168.2`, they are on different networks. To communicate, they will need a router.

---

## Method 2: The Precise Way (Bitwise AND Operation)

When a subnet mask doesn't end cleanly in `.0` (for example, a custom subnet mask like `255.255.255.224`), you must convert
the numbers into binary and perform a **Logical AND** operation.

> **The Rule of AND:**
>
> - $1 \text{ AND } 1 = 1$
> - $1 \text{ AND } 0 = 0$
> - $0 \text{ AND } 0 = 0$

Let's test if **`192.168.1.35`** and **`192.168.1.60`** are on the same network using the subnet mask **`255.255.255.224`**.

### Step 1: Convert the changing octet to binary

The first three octets (`192.168.1`) match, and the mask is `255.255.255`, so we already know that part is the same. Let's
look at the tricky 4th octet:

- Subnet Mask 4th octet (`224`): **`11100000`**
- IP 1 4th octet (`35`): **`00100011`**
- IP 2 4th octet (`60`): **`00111100`**

### Step 2: Perform the AND operation on IP 1

Line up IP 1 and the Subnet Mask, then multiply down:

```text
  00100011  (IP 1: 35)
  11100000  (Subnet Mask: 224)
  --------
  00100000  (Resulting Network ID block = 32 in decimal)

```

The Network ID for IP 1 is **`192.168.1.32`**.

### Step 3: Perform the AND operation on IP 2

Line up IP 2 and the Subnet Mask, then multiply down:

```text
  00111100  (IP 2: 60)
  11100000  (Subnet Mask: 224)
  --------
  00100000  (Resulting Network ID block = 32 in decimal)

```

The Network ID for IP 2 is **`192.168.1.32`**.

### Conclusion

Because both operations resulted in the exact same Network ID (`192.168.1.32`), **both IP addresses belong to the same
network.** They can talk to each other directly without a router.

## Shortcut Tools

If you are doing this for real-world network engineering rather than an exam, don't do binary math by hand. You can use free
online tools or command-line utilities:

- **Linux:** Use the `ipcalc` tool (e.g., `ipcalc 192.168.1.35/27`).
- **Web:** Search for a "CIDR Subnet Calculator."
