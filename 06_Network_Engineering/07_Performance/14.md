To view and edit the `pg_hba.conf` file on a Linux server, you first need to locate it, as its directory varies depending on
your Linux distribution and PostgreSQL version.

Here is the step-by-step process to find and view the file.

---

## Step 1: Find the File Location

If you don't know exactly where the file is, the most reliable way to find it is by asking PostgreSQL directly via SQL.

Log into your PostgreSQL prompt (psql) and run:

```sql
SHOW hba_file;

```

This will return the absolute path, which typically looks like one of these standard locations:

- **Ubuntu/Debian:** `/etc/postgresql/16/main/pg_hba.conf` (replace `16` with your version)
- **RHEL/CentOS/Rocky Linux:** `/var/lib/pgsql/16/data/pg_hba.conf`

Alternatively, you can search for it using the Linux command line:

```bash
sudo find / -name pg_hba.conf 2>/dev/null

```

---

## Step 2: View the File

Because `pg_hba.conf` contains sensitive security configurations, it is owned by the `postgres` system user. You will usually
need `sudo` (root) privileges to read it.

### Option A: View without editing (Safe)

If you just want to read the file without making changes, use `cat` or `less`:

```bash
sudo cat /etc/postgresql/16/main/pg_hba.conf

```

_(Make sure to replace the path above with the actual path you found in Step 1)._

### Option B: Filter out the comments (Recommended)

The default `pg_hba.conf` file has dozens of lines of documentation comments (lines starting with `#`). To see **only** your
actual, active configuration rules, use `grep` to filter out the comments and empty lines:

```bash
sudo grep -v -e '^#' -e '^$' /etc/postgresql/16/main/pg_hba.conf

```

---

## Step 3: Edit the File (If Needed)

To make changes to the file, open it with a terminal text editor like `nano` or `vi`:

```bash
sudo nano /etc/postgresql/16/main/pg_hba.conf

```

- **To save and exit in Nano:** Press `Ctrl + O`, hit `Enter`, then press `Ctrl + X`.

---

## Step 4: Apply Your Changes

As mentioned before, PostgreSQL won't notice your changes until you explicitly tell it to reload. You can do this right from
the Linux terminal:

```bash
sudo systemctl reload postgresql

```
