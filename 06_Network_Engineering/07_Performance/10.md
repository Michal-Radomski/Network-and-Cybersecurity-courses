Think of proxies as the ultimate middlemen of the internet. They sit between a client (like your laptop) and a server (like a
website), but they serve opposite masters.

Here is the breakdown of how **Forward Proxies** and **Reverse Proxies** work, how they differ, and why we use them.

---

## The Quick Analogy

- **Forward Proxy (The Guard for the Client):** Imagine you are a kid who wants a cookie from a bakery, but your parents
  don’t want the baker to know who you are (or want to check if the cookie is healthy first). Your parent goes to the bakery,
  gets the cookie, and hands it to you. The baker only ever sees your parent.
- **Reverse Proxy (The Guard for the Server):** Imagine a fancy restaurant. You don't walk into the kitchen and talk to the
  chef. You talk to the waiter (the reverse proxy). The waiter takes your order, brings it to the kitchen, and brings your
  food back. You have no idea which specific chef actually cooked your meal.

---

## 1. Forward Proxy (Closer to the Client)

A **Forward Proxy** (often just called a "proxy") acts on behalf of the **client** (user). When you try to visit a website,
your request goes to the forward proxy first, which then forwards the request to the internet.

### Key Uses:

- **Anonymity:** The destination website only sees the IP address of the proxy, not your actual IP address.
- **Bypassing Restrictions:** If a government or school blocks a website, a forward proxy can be used to bypass those
  filters.
- **Content Filtering:** Conversely, employers use them to block employees from accessing certain sites (e.g., blocking
  social media at work).
- **Caching:** It can save copies of frequently visited sites to speed up browsing for everyone on the local network.

---

## 2. Reverse Proxy (Closer to the Server)

A **Reverse Proxy** acts on behalf of the **web server**. When a user types in a website URL, the request hits the reverse
proxy first. The proxy decides which backend server should handle the request and sends it there. The client thinks they are
talking directly to the website.

### Key Uses:

- **Load Balancing:** If a website gets millions of visitors, a single server will crash. A reverse proxy distributes the
  traffic evenly across a cluster of backend servers.
- **Security (DDoS Protection):** Since hackers only see the reverse proxy’s IP address, they can't attack the actual
  database or application servers directly.
- **SSL Encryption/Decryption:** Handling SSL/TLS encryption takes a lot of computing power. A reverse proxy can handle this
  overhead ("SSL termination"), freeing up backend servers to just focus on processing data.
- **Caching:** It can cache static content (like images and videos) so the main servers don't have to rebuild the page every
  single time someone visits.

---

## Summary of Differences

| Feature                  | Forward Proxy                               | Reverse Proxy                                    |
| ------------------------ | ------------------------------------------- | ------------------------------------------------ |
| **Who it protects**      | The Client (User)                           | The Server (Website)                             |
| **Who knows it's there** | The client configures it/knows it's there.  | The client has no idea it exists.                |
| **Visible IP Address**   | Hides the client's IP from the internet.    | Hides the backend server's IP from the internet. |
| **Primary Goal**         | Privacy, content control, bypassing blocks. | Load balancing, security, speed optimization.    |
| **Common Example**       | A corporate web filter or a VPN-like proxy. | Nginx, HAProxy, Cloudflare.                      |

---

To put it in one sentence: A **forward proxy** hides the identity of the **user**, while a **reverse proxy** hides the
identity (and complexity) of the **web server**.
