The **OSI (Open Systems Interconnection) Model** is a conceptual framework used to understand how data travels through a
network. It divides network communication into **7 layers**, where each layer performs a specific function.

The model was developed by the International Organization for Standardization.

---

# OSI Model Layers

```text
+-------------------+ 7. Application
+-------------------+ 6. Presentation
+-------------------+ 5. Session
+-------------------+ 4. Transport
+-------------------+ 3. Network
+-------------------+ 2. Data Link
+-------------------+ 1. Physical
```

Data travels:

- **Downward** from sender
- **Upward** at receiver

---

# 1. Physical Layer (Layer 1)

## Function

Handles the **physical transmission of data** through cables, radio waves, or hardware.

## Responsibilities

- Transmitting raw bits (0s and 1s)
- Electrical signals
- Cable connections
- Data rates

## Devices

- Hubs
- Repeaters
- Cables

## Example

Ethernet cable carrying signals.

---

# 2. Data Link Layer (Layer 2)

## Function

Provides **node-to-node delivery** and error detection.

## Responsibilities

- Framing
- MAC addressing
- Error detection
- Flow control

## Devices

- Switches
- Bridges

## Example

Ethernet MAC addresses.

---

# 3. Network Layer (Layer 3)

## Function

Handles **routing and logical addressing**.

## Responsibilities

- IP addressing
- Path selection
- Packet forwarding

## Devices

- Routers

## Protocols

- IP
- ICMP

## Example

Finding the best route for internet traffic.

---

# 4. Transport Layer (Layer 4)

## Function

Provides **end-to-end communication**.

## Responsibilities

- Segmentation
- Reliability
- Error recovery
- Flow control

## Protocols

- TCP
- UDP

## Example

TCP ensures reliable delivery of webpages.

---

# 5. Session Layer (Layer 5)

## Function

Establishes, manages, and terminates communication sessions.

## Responsibilities

- Session setup
- Session maintenance
- Synchronization

## Example

Keeping a login session active.

---

# 6. Presentation Layer (Layer 6)

## Function

Translates data between application and network formats.

## Responsibilities

- Encryption
- Compression
- Data translation

## Example

SSL/TLS encryption for secure websites.

---

# 7. Application Layer (Layer 7)

## Function

Provides services directly to end users.

## Responsibilities

- Web browsing
- Email
- File transfer

## Protocols

- HTTP
- HTTPS
- FTP
- SMTP

## Example

Opening a webpage in a browser.

---

# Data Flow Example

When opening a website:

1. **Application Layer** → Browser creates request
2. **Presentation Layer** → Data encrypted
3. **Session Layer** → Session established
4. **Transport Layer** → Data split into segments
5. **Network Layer** → IP address added
6. **Data Link Layer** → MAC address added
7. **Physical Layer** → Bits transmitted

At the receiver side, the process happens in reverse.

# Advantages of OSI Model

- Standardizes networking concepts
- Simplifies troubleshooting
- Encourages interoperability
- Modular design
- Easier learning and development

---

# OSI vs TCP/IP Model

| OSI Model         | TCP/IP Model             |
| ----------------- | ------------------------ |
| 7 layers          | 4 layers                 |
| Reference model   | Practical internet model |
| More detailed     | Simpler                  |
| Used for learning | Used in real networks    |

---

# Conclusion

The OSI Model explains how network communication works by dividing tasks into seven layers. Each layer has a dedicated role,
making networking easier to design, troubleshoot, and understand.
