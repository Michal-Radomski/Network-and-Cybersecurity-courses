Client–Server Architecture is a computing model where multiple **clients** request services or resources from a centralized
**server** over a network.

## Basic Idea

- **Client** → A device or application that requests data or services.
- **Server** → A system that provides data, services, or resources to clients.

The communication usually happens through a network such as the Internet or a local network.

---

## Simple Example

When you open [Google](https://www.google.com?utm_source=chatgpt.com) in a browser:

1. Your browser acts as the **client**.
2. Google’s web systems act as the **servers**.
3. The browser sends a request for a webpage.
4. The server processes the request and sends back the webpage data.

---

## Architecture Diagram

```text
+-----------+         Request          +-----------+
|  Client   | -----------------------> |  Server   |
| (Browser) |                          | (Web App) |
|           | <----------------------- |           |
+-----------+         Response         +-----------+
```

---

## Components

### 1. Client

The client:

- Initiates communication
- Sends requests
- Displays results to users

Examples:

- Web browsers
- Mobile apps
- Desktop applications

---

### 2. Server

The server:

- Waits for requests
- Processes data
- Sends responses

Examples:

- Web servers
- Database servers
- File servers
- Mail servers

---

## How It Works

### Step-by-Step Process

1. Client sends a request
2. Server receives the request
3. Server processes it
4. Server sends back the response
5. Client displays the result

Example:

- Client asks for a video
- Server locates the video
- Server streams it back

---

## Types of Client–Server Architecture

### 1. Two-Tier Architecture

Client communicates directly with the server.

```text
Client  <---->  Database Server
```

Used in:

- Small business applications

---

### 2. Three-Tier Architecture

Includes:

- Presentation layer (client)
- Application layer (business logic)
- Database layer

```text
Client <--> Application Server <--> Database Server
```

Used in:

- Web applications
- Enterprise systems

---

### 3. Multi-Tier Architecture

Uses multiple servers for scalability and security.

Examples:

- Cloud systems
- Large e-commerce platforms

---

## Advantages

- Centralized data management
- Better security control
- Easier maintenance
- Resource sharing
- Scalable

---

## Disadvantages

- Server failure can affect all clients
- Network dependency
- Heavy server load may reduce performance
- Higher setup cost for large systems

---

## Real-World Examples

- Email systems
- Banking applications
- Online games
- Social media platforms
- Cloud storage services

---

## Difference Between Client and Server

| Client                  | Server                |
| ----------------------- | --------------------- |
| Requests services       | Provides services     |
| User-facing             | Backend system        |
| Initiates communication | Responds to requests  |
| Usually less powerful   | Usually more powerful |

---

## Conclusion

Client–Server Architecture is one of the foundational models of modern computing. It separates service providers (servers)
from service requesters (clients), enabling efficient communication, centralized management, and scalable application
development.
