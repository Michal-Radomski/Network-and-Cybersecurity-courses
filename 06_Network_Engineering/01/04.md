# Host-to-Host Communication

**Host-to-host communication** refers to the process of exchanging data between two devices (hosts) connected through a
network.

A **host** can be:

- Computer
- Smartphone
- Server
- Printer
- IoT device

The communication happens using networking protocols such as TCP/IP.

---

# Basic Concept

One host acts as the **sender** and another as the **receiver**.

```text
+-----------+                      +-----------+
|  Host A   |  ---- Data ----->   |  Host B   |
+-----------+                      +-----------+
```

Example:

- Your laptop accesses a website hosted on a remote server.

---

# Requirements for Host-to-Host Communication

To communicate successfully, hosts need:

1. Unique IP addresses
2. Network connectivity
3. Communication protocols
4. Proper routing

---

# Role of Different OSI Layers

## 1. Application Layer

Creates data for communication.

Examples:

- Browser
- Email app
- Chat app

---

## 2. Transport Layer

Provides **process-to-process delivery**.

Main protocols:

- TCP
- UDP

### TCP

- Reliable
- Connection-oriented
- Error checking

### UDP

- Faster
- Connectionless
- Less reliable

---

## 3. Network Layer

Handles:

- Logical addressing
- Routing

Uses:

- IP addresses

Routers help move packets between networks.

---

## 4. Data Link Layer

Handles:

- MAC addresses
- Local delivery

Switches operate here.

---

## 5. Physical Layer

Transfers bits as:

- Electrical signals
- Optical signals
- Wireless signals

---

# Communication Process

## Step-by-Step

### Step 1: Data Creation

Application creates data.

### Step 2: Segmentation

Transport layer divides data into segments.

### Step 3: IP Addressing

Network layer adds source and destination IP addresses.

### Step 4: Framing

Data link layer adds MAC addresses.

### Step 5: Transmission

Physical layer sends bits through media.

### Step 6: Routing

Routers forward packets to destination.

### Step 7: Reception

Destination host reassembles and processes data.

# Addressing in Host-to-Host Communication

| Address Type | Used At         | Example           |
| ------------ | --------------- | ----------------- |
| MAC Address  | Data Link Layer | 00:1A:2B:3C:4D:5E |
| IP Address   | Network Layer   | 192.168.1.10      |
| Port Number  | Transport Layer | 80, 443           |

---

# Types of Host Communication

## 1. Unicast

One sender → One receiver

## 2. Broadcast

One sender → All devices in network

## 3. Multicast

One sender → Selected group
