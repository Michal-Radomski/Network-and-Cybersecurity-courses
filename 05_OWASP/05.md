## What is XML External Entity (XXE)?

An **XML External Entity (XXE)** injection is a type of web security vulnerability that allows an attacker to interfere with
an application's processing of XML data. It occurs when an XML parser, configured to allow external entities, is used to
process input from an untrusted source.

Think of it as a "hidden instruction" tucked inside a seemingly harmless XML file that tells the server to fetch data from a
location it shouldn't access.

---

### How the Vulnerability Works

XML allows the definition of **entities**, which are essentially variables used within the document. An **External Entity**
is a specific type of entity whose value is loaded from outside the main XML file (like a local file on the server or a
remote URL).

When a vulnerable parser encounters an external entity, it attempts to resolve the reference and include the content in the
XML document. If an attacker can control this reference, they can trick the server into doing things it wasn't designed to
do.

---

### Common Attack Scenarios

1. **Retrieving Local Files:** The attacker defines an entity that points to a sensitive file on the server's filesystem,
   such as `/etc/passwd` on Linux or `C:\Windows\win.ini` on Windows.

- **Example Payload:**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE data [
  <!ENTITY file SYSTEM "file:///etc/passwd">
]>
<user>
  <name>&file;</name>
</user>

```

2. **Server-Side Request Forgery (SSRF):** The attacker points the entity to an internal URL that is not accessible from the
   outside. The server, acting as a proxy, fetches the internal resource and returns it to the attacker.
3. **Exfiltrating Data (Out-of-Band XXE):** If the application doesn't return the XML response directly, attackers can use
   "blind" XXE. They force the server to make a request to a server they control, appending the stolen data as a URL
   parameter.

---

### Potential Impact

The consequences of a successful XXE attack can be severe:

- **Data Theft:** Access to configuration files, source code, or sensitive system files.
- **Internal Reconnaissance:** Scanning internal networks and ports that are protected by firewalls.
- **Denial of Service (DoS):** Using the "Billion Laughs" attack (a variation involving nested entities) to crash the server
  by exhausting memory.

---

### Prevention and Mitigation

The most effective way to prevent XXE is to **disable the processing of external entities** in your XML parser.

| Method                        | Description                                                                                             |
| ----------------------------- | ------------------------------------------------------------------------------------------------------- |
| **Disable DTDs**              | Completely disable Document Type Definitions (DTDs) if they aren't necessary.                           |
| **Disable External Entities** | Specifically instruct the parser to ignore `SYSTEM` and `PUBLIC` entities.                              |
| **Use Less Complex Formats**  | Switch to JSON where possible, as it is inherently less complex and doesn't support entities.           |
| **Update Libraries**          | Ensure XML parsing libraries are up to date, as many modern versions disable these features by default. |

> **Security Note:** While input validation and "allow-listing" can help, they are often bypassed. Hardening the parser
> configuration is the industry-standard defense.
