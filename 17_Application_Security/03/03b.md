**Subresource Integrity (SRI)** is a powerful browser security feature that directly defends against the exact type of
software supply chain failure seen in the polyfill.io attack.

It allows web developers to guarantee that the files (like JavaScript or CSS) fetched from a third-party Content Delivery
Network (CDN) haven't been secretly altered or hacked.

---

## How It Works

When you use SRI, you don't just tell the browser _where_ to find a file; you also provide a cryptographic hash (a unique
digital fingerprint) of what that file is supposed to look like.

The process follows these steps:

1. **Generation:** The developer hashes the legitimate file using an algorithm like SHA-256, SHA-384, or SHA-512.
2. **Implementation:** The developer adds an `integrity` attribute containing this hash to the HTML tag.
3. **Verification:** When a user visits the website, their browser downloads the file from the CDN, calculates its hash on
   the fly, and compares it to the `integrity` hash provided in the code.
4. **Enforcement:** If the hashes match, the browser runs the script. If they don't match (meaning someone modified the
   file), the browser instantly **blocks** the script from executing, protecting the user.

---

## What the Code Looks Like

Here is a comparison of a standard script tag versus one secured with SRI:

### Insecure (Standard)

If a hacker compromises the server hosting `analytics.js`, they can inject malware, and your users' browsers will execute it
without question.

```html
<script src="https://example-cdn.com/js/analytics.js"></script>
```

### Secure (With SRI)

Even if a hacker modifies `analytics.js` on the CDN, the browser will catch the discrepancy and block it.

```html
<script
  src="https://example-cdn.com/js/analytics.js"
  integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/uxy9rx7HNQlGYl1kPzQho1wx4JwY8wC"
  crossorigin="anonymous"
></script>
```

_(Note: The `crossorigin="anonymous"` attribute is required because it tells the browser to fetch the script with CORS
enabled, allowing it to safely read the file contents to calculate the hash.)_

---

## Why SRI is Vital

SRI bridges a major trust gap in modern web development. It allows you to leverage the speed and cost benefits of using
public CDNs while retaining absolute control over the code running in your users' browsers. If a service like polyfill.io
gets sold or compromised, an SRI attribute ensures that the sudden change in the script's code breaks the connection
immediately, neutralizing the attack before it can touch a single user.
