The situation with **polyfill.io** is one of the most textbook examples of a **Software Supply Chain Failure** in recent web
history.

In mid-2024, the popular community-run service was caught injecting malicious code into more than 100,000 websites, forcing
major tech companies to step in and shut down the traffic.

---

## What Actually Happened?

Polyfill.io was a highly trusted service used by developers to ensure modern JavaScript features could run smoothly on older
web browsers. For years, thousands of major websites linked directly to `cdn.polyfill.io`.

### 1. The Acquisition (February 2024)

The original maintainer sold the domain and the associated GitHub account to a China-based content delivery network (CDN)
company called **Funnull**. Immediately, the original creator of Polyfill posted on social media warning developers to remove
the library from their sites, fearing the new owners could not be trusted.

### 2. The Malware Injection (June 2024)

A few months after the buyout, cybersecurity researchers at Sansec discovered that the new owners had weaponized the CDN. The
server began injecting **malicious JavaScript code** into the polyfill scripts sent to web browsers.

To hide their tracks, the attackers used sophisticated evasion techniques:

- **Targeting Mobile Devices:** The code often checked if the visitor was on a mobile device before striking.
- **Dynamic Payloads:** The server evaluated HTTP headers to only send the malicious code to specific, real user sessions,
  making it hard for automated security scanners to detect.
- **Malicious Redirects:** Victims' browsers were secretly redirected to scam, sports betting, and adult websites, and the
  code opened up vectors for data theft.

---

## How the Tech Industry Responded

Because the blast radius was so massive—affecting major brands, academic platforms, and government sites—the tech community
took drastic measures:

- **Domain Takedown:** The registrar Namecheap officially suspended the `polyfill.io` domain to immediately break the attack
  chain.
- **Automatic Rewrites:** Cloudflare launched an automatic feature for its users that intercepted requests aimed at
  `polyfill.io` and seamlessly rerouted them to a safe, trusted mirror hosted by Cloudflare. Fastly did the same.
- **Google Intervened:** Google began blocking ads and warning advertisers whose landing pages were caught executing code
  from the compromised domain.

---

## The Takeaway

The polyfill.io incident highlights why **Software Supply Chain** security is now the #1 priority for many organizations. It
proved that even if your own code is completely secure, relying on an external, live-loaded third-party script means your
security is only as good as the person who owns that domain. Today, developers avoid linking directly to unverified external
CDNs, preferring to bundle dependencies locally or use cryptographic safeguards like **Subresource Integrity (SRI)**.
