**Identification and Authentication Failures** (formerly known as _Broken Authentication_) refers to vulnerabilities that
occur when an application incorrectly validates a user's identity.

When authentication mechanisms are weak, attackers can compromise user accounts, steal active sessions, or completely assume
the identities of legitimate users, often gaining administrative control.

---

## Common Causes of Authentication Failures

Authentication is the front door of your application. If that door is poorly constructed, attackers will find a way through.

### 1. Permitting Automated Attacks (Credential Stuffing)

When an application doesn't limit login attempts, attackers can use automated tools to test millions of known
username/password combinations leaked from other data breaches. This is known as **Credential Stuffing** or **Brute-Force**
attacks.

### 2. Allowing Weak Passwords

Applications that fail to enforce strong password complexity or don't check passwords against lists of commonly used, weak
passwords (like `password123` or `12345678`) make it incredibly easy for attackers to guess their way in.

### 3. Weak Session Management

Once a user logs in, the application issues a "session token" (like a cookie) so the user doesn't have to re-enter their
password on every page. Authentication fails if:

- Session identifiers are predictable or easily guessed.
- Sessions do not time out after inactivity, leaving them open indefinitely.
- The session ID is exposed in the URL string (e.g., `example.com/dashboard?session_id=XYZ`).

### 4. Missing Multi-Factor Authentication (MFA)

Relying solely on a username and password (single-factor authentication) means that if those credentials are leaked or
guessed, the account is immediately compromised.

---

## Real-World Impact

- **Account Takeover (ATO):** Attackers gain full control of user accounts, allowing them to steal personal data, make
  unauthorized purchases, or change account details.
- **Privilege Escalation:** If an administrative account is compromised due to weak authentication, the attacker can gain
  full access to the backend systems.
- **Mass Identity Fraud:** Attackers can script the takeover of thousands of accounts simultaneously, severely damaging user
  trust and corporate reputation.

---

## How to Prevent It

Securing authentication requires building layered defenses around how identities are verified and managed.

> 🛡️ **The Golden Rule:** Implement multi-factor authentication (MFA) wherever possible, and ensure the application properly
> tracks, secures, and destroys session tokens.

- **Enforce Multi-Factor Authentication (MFA):** MFA adds an extra layer of defense (like a mobile authenticator app or
  hardware key). Even if an attacker steals a password, they cannot access the account without the second factor.
- **Implement Rate Limiting and Lockouts:** Block or slow down IP addresses that generate an excessive number of failed login
  attempts to stop automated brute-force attacks.
- **Validate Against Common Passwords:** Use a library or service to reject weak passwords or those found in historical data
  breach dumps (such as the _Have I Been Pwned_ API).
- **Use Secure Session Managers:** Rely on built-in, secure server-side session management tools. Ensure session cookies use
  the `Secure`, `HttpOnly`, and `SameSite=Strict` attributes to protect them from being stolen via scripts or network
  interception.
- **Properly Log Out:** Ensure that when a user clicks "Log Out," the session token is fully invalidated and destroyed on
  both the client browser and the backend server.
