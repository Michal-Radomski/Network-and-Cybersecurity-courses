## What is AWS Application Security?

**AWS Application Security (AppSec)** involves protecting applications built, deployed, and run on Amazon Web Services from
threats at the code, package, infrastructure, and runtime levels.

Because modern applications are complex and continuously updated, AWS provides tools that fit into every phase of the
development lifecycle—shifting security "left" (during coding) and extending it "right" (during live production).

---

## 1. Security During Code & Build (Shift Left)

These tools help developers find and fix vulnerabilities _before_ code is deployed to production.

- **Amazon Inspector:** Traditionally used to scan virtual servers, Inspector automatically scans application code,
  open-source dependencies (SCA), and container images (e.g., inside Amazon ECR or CI/CD pipelines) for known
  vulnerabilities.
- **Amazon Q Developer:** An AI-powered assistant that can perform automated code reviews directly inside a developer's IDE
  (Integrated Development Environment) to flag security flaws and offer immediate remediation suggestions.
- **AWS Continuum & Security Agent:** AWS's AI-native security platform. It allows teams to run automated design security
  reviews, analyze threat modeling, and schedule context-aware **penetration testing** directly within CI/CD pipelines to
  find deep business logic flaws.

---

## 2. Security at the Edge & Runtime (Production Defense)

Once an application is live on the internet, it needs protection from real-time exploits, bots, and DDoS attacks.

- **AWS WAF (Web Application Firewall):** Monitored right at the edge (often via Amazon CloudFront or Application Load
  Balancers), AWS WAF filters incoming HTTP/HTTPS traffic. It blocks common web exploits like SQL Injections, Cross-Site
  Scripting (XSS), and malicious bots before they ever reach the application server.
- **AWS Shield:** A managed Distributed Denial of Service (DDoS) protection service. _Shield Standard_ is automatically
  turned on for all AWS customers for free to protect against common network layer attacks. _Shield Advanced_ provides
  custom, 24/7 mitigation against massive, sophisticated DDoS attacks.
- **Amazon API Gateway Security:** If your application relies on microservices and APIs, API Gateway acts as the secure front
  door, managing traffic throttling, API keys, and integration with authorization layers.

---

## 3. Application Authentication & Authorization

Controlling _who_ can access the application and _what_ they are allowed to do inside it.

- **Amazon Cognito:** A customer identity management service. It easily adds user sign-up, sign-in, and access control to web
  and mobile apps, supporting multi-factor authentication (MFA) and social identity providers (like Apple, Google, or
  Amazon).
- **Amazon Verified Permissions:** A scalable permissions management service for building fine-grained authorization rules
  inside custom applications using human-readable policies (written in the Cedar policy language).

---

## The AppSec Summary Checklist

| Stage           | Threat                                          | AWS Tool                                   |
| --------------- | ----------------------------------------------- | ------------------------------------------ |
| **Design**      | Bad Architecture / Flawed Logic                 | AWS Security Agent (Threat Modeling)       |
| **Write/Build** | Vulnerable Open-Source Libraries / Bad Code     | Amazon Inspector / Amazon Q Developer      |
| **Deploy**      | Publicly exposed endpoints / Bad infrastructure | AWS Config / Security Hub Network Scanning |
| **Run**         | SQL Injection, XSS, Heavy Bot Traffic           | AWS WAF                                    |
| **Scale**       | Volumetric DDoS Attacks                         | AWS Shield                                 |
