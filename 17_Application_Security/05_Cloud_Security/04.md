**AWS Data Protection** focuses on ensuring that customer data remains private, secure, and available, both while it is
traveling across the internet and while it is stored on AWS hardware.

Under the AWS Shared Responsibility Model, AWS secures the underlying infrastructure (the data centers and physical drives),
while the customer is responsible for configuring the tools to protect their actual data.

AWS breaks down data protection into three main pillars: **Data Classification & Discovery**, **Data at Rest**, and **Data in
Transit**.

---

## 1. Data Classification and Discovery

Before protecting data, an organization must know what data it has and how sensitive it is (e.g., public vs. highly
confidential personal data).

- **Amazon Macie:** A fully managed security and data privacy service that uses machine learning and pattern matching to
  automatically discover, classify, and protect sensitive data (like credit card numbers or Social Security numbers) stored
  in Amazon S3 buckets.

---

## 2. Protecting Data at Rest

Data at rest refers to data that is physically stored on digital media, such as hard drives, solid-state drives, or archival
tapes.

- **AWS Key Management Service (KMS):** The foundational service for data encryption. It allows users to easily create,
  manage, and rotate cryptographic keys. KMS integrates natively with almost every major AWS service (like S3, EBS, and RDS)
  to encrypt data automatically with a single click.
- **AWS CloudHSM:** For organizations with strict regulatory compliance needs (like government or finance), CloudHSM provides
  dedicated Hardware Security Modules within the AWS cloud to manage encryption keys on tamper-resistant physical hardware.
- **Workload Confidentiality (AWS Nitro System):** Modern AWS virtual servers are built on the Nitro System. Its design
  prevents anyone—including AWS operators—from accessing customer memory or data stored on local instance storage.

---

## 3. Protecting Data in Transit

Data in transit refers to data that is actively moving across a network—such as between a user's browser and a cloud server,
or between different cloud services.

- **AWS Certificate Manager (ACM):** Automatically provisions, manages, and deploys public and private Secure Sockets
  Layer/Transport Layer Security (SSL/TLS) certificates. This ensures all web traffic coming into an application is encrypted
  using standard `https://` protocols.
- **Virtual Private Cloud (VPC) Endpoints:** Allows traffic between a private network and AWS services to travel entirely
  over the secure, private AWS network infrastructure, without ever touching the public internet.

---

## 4. Resilience and Backup

Data protection also means protecting against accidental deletion, corruption, or ransomware attacks.

- **AWS Backup:** A fully managed backup service that centralizes and automates the backup of data across multiple AWS
  services (like databases, block storage volumes, and file systems), allowing for quick recovery if data is lost or
  compromised.
