AWS global infrastructure determines how applications achieve global scale, high availability, and low latency. Instead of
relying on a single data center, AWS distributes its hardware across a highly redundant, multi-layered worldwide network.

The physical architecture breaks down into four core building blocks:

---

## 1. AWS Regions

An **AWS Region** is a physical, geographic location anywhere in the world where AWS clusters its data centers (e.g.,
`us-east-1` in Northern Virginia or `eu-west-1` in Ireland).

- **Isolation:** Every region is entirely independent and completely isolated from the others. AWS will never move your data
  out of a chosen region unless you explicitly configure a backup or replication feature.
- **Selection Criteria:** Companies choose their primary region based on three main factors:

1. **Compliance/Data Sovereignty:** Laws requiring financial or healthcare data to stay within a specific country's borders.
2. **Latency:** Choosing the region physically closest to the end users for faster response times.
3. **Cost & Features:** Pricing varies slightly by region, and brand-new AWS features typically roll out to core regions
   first.

## 2. Availability Zones (AZs)

Inside every AWS Region is a collection of **Availability Zones (AZs)**. An AZ consists of one or more discrete, physical
data centers. Every AWS region is mandated to have at least three AZs.

- **Physical Separation:** AZs within a region are separated by a meaningful distance (often miles apart) to ensure that a
  localized disaster (like a flood or power grid failure) affecting one AZ won't impact the others.
- **High-Speed Connectivity:** Despite being physically isolated, all AZs in a region are interconnected through a private,
  ultra-low-latency fiber-optic network.
- **High Availability Design:** Architects build resilient apps by launching their servers across multiple AZs. If AZ-1 goes
  dark, traffic automatically switches to the identical setup running seamlessly in AZ-2.

## 3. Edge Locations & Points of Presence (PoPs)

**Edge Locations** are smaller deployments located in major cities worldwide, separate from the primary AWS data centers.

- **The Cache Layer:** These are used by **Amazon CloudFront** (a Content Delivery Network, or CDN). When a user in London
  requests a video hosted on a server in California, the file is retrieved once and cached at the London Edge Location.
  Subsequent European users download the video instantly from the local cache rather than making the long journey back to the
  US.
- **DNS Services:** They also house **Amazon Route 53** infrastructure to translate web addresses quickly at the network
  edge.

## 4. Extended Infrastructures (Hybrid & 5G Edge)

AWS provides specialized hardware frameworks to stretch cloud services into physical on-premises facilities or cellular
networks:

- **AWS Outposts:** A physical rack of AWS-designed servers delivered directly to a company's private on-premises data
  center. It allows businesses to run AWS compute and storage locally while managing it entirely through the standard AWS
  cloud console.
- **AWS Local Zones:** Puts compute and storage resources very close to massive metropolitan areas (like Los Angeles or New
  York) that don't have a full AWS Region, targeting single-digit millisecond latency applications like media editing or
  real-time gaming.
- **AWS Wavelength:** Plugs AWS hardware directly into telecom providers' **5G cell towers**, allowing mobile developers to
  process data locally at the absolute edge of the mobile network.
