## What is "The Cloud"?

At its simplest, **the cloud** refers to software and services that run on the internet instead of locally on your computer
or phone.

In the past, if you wanted to play a video game, write a document, or store a photo, you had to install the software on your
hard drive and save the file to your physical device. With the cloud, those files and programs live on massive, secure
servers located in data centers all over the world.

When you stream a movie, back up your photos, or open a shared document, your device is simply communicating with a cloud
server managed by companies like Amazon (AWS), Microsoft (Azure), or Google (Google Cloud).

## What is Cloud Security?

**Cloud security** is the suite of technologies, policies, controls, and services deployed to protect data, applications, and
infrastructure from cyber threats, theft, and unauthorized access.

Because data in the cloud is stored remotely and accessed over the internet, traditional security methods (like putting a
firewall around an office building) don't work. Cloud security requires a completely different approach.

### The "Shared Responsibility Model"

The most important concept in cloud security is the **Shared Responsibility Model**. Security in the cloud is a team effort
between the cloud provider and the customer:

- **Security OF the Cloud (The Provider's Job):** Companies like Amazon, Google, or Microsoft are responsible for securing
  the physical infrastructure. This means protecting the actual data centers with guards, backup generators, and biometric
  locks, as well as maintaining the physical servers and virtualization layers.
- **Security IN the Cloud (The Customer's Job):** The person or business using the cloud is responsible for protecting
  whatever they put _into_ it. This includes managing who has access to the data, setting strong passwords, encrypting files,
  and configuring permissions correctly. If a company leaves a cloud database completely open to the public without a
  password, the provider cannot prevent a data breach.

## Core Pillars of Cloud Security

To keep data safe, cloud security relies on several overlapping layers of defense:

- **Identity and Access Management (IAM):** Ensuring that only authorized people can access specific cloud resources. This
  relies heavily on **Multi-Factor Authentication (MFA)** and the principle of least privilege (only giving people access to
  the files they absolutely need to do their job).
- **Data Encryption:** Scrambling data so that even if a hacker intercepts it, they cannot read it. Cloud security mandates
  encrypting data both **in transit** (as it travels across the internet) and **at rest** (while it is sitting on the
  server's hard drive).
- **Network Security:** Using virtual firewalls and segmented networks to control the flow of traffic into and out of cloud
  environments, blocking unauthorized connections.
- **Continuous Monitoring and Visibility:** Because cloud environments can change in seconds (e.g., spin up new servers or
  delete data), automated tools continuously scan the environment to detect misconfigurations, abnormal user behavior, or
  potential malware.
