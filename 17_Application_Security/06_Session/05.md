Implementing a hybrid approach (Session + JWT) in Node.js gives you the best of both worlds: the **security and server-side
control** of sessions (like instant revocation and device limiting) combined with the **stateless efficiency** of JWTs for
microservices or API communication.

Here is a step-by-step guide to building a production-ready hybrid authentication system using **Express**, **Redis** (for
session management), and **JSON Web Tokens**.

---

## The Architecture Flow

1. **Login:** The user logs in. The server creates a session in Redis and generates a JWT.
2. **Delivery:** The JWT is encrypted and stored inside a secure, `httpOnly` cookie. The session ID can either be embedded
   inside the JWT payload or handled via standard session cookies.
3. **Verification:** For every request, the server verifies the JWT signature _and_ checks Redis to ensure the session is
   still active/valid.

---

## 1. Project Setup & Dependencies

First, install the necessary packages. We will use `jsonwebtoken` for the tokens, `redis` for the fast session store, and
`cookie-parser` to handle cookies securely.

```bash
npm install express jsonwebtoken redis cookie-parser dotenv bcryptjs

```

## 2. The Implementation

Here is a complete, modular example of how to implement this architecture.

### Environment Setup (`.env`)

```env
PORT=3000
JWT_SECRET=your_super_secret_jwt_key_change_this
REDIS_URL=redis://localhost:6379

```

### Server Code (`server.js`)

```javascript
import express from "express";
import jwt from "jsonwebtoken";
import cookieParser from "cookie-parser";
import { createClient } from "redis";
import dotenv from "dotenv";

dotenv.config();

const app = express();
app.use(express.json());
app.use(cookieParser());

// Initialize Redis Client for Session Management
const redisClient = createClient({ url: process.env.REDIS_URL });
redisClient.on("error", (err) => console.error("Redis Client Error", err));
await redisClient.connect();

const JWT_SECRET = process.env.JWT_SECRET;
const SESSION_EXPIRY = 3600; // 1 hour in seconds

// --- MIDDLEWARE: The Hybrid Guard ---
async function authenticateHybrid(req, res, next) {
  const token = req.cookies.auth_token;

  if (!token) {
    return res.status(401).json({ message: "Unauthorized: No token provided" });
  }

  try {
    // 1. Verify the JWT (Stateless check)
    const decoded = jwt.verify(token, JWT_SECRET);

    // 2. Verify against Redis (Stateful check)
    const sessionKey = `session:${decoded.userId}:${decoded.sessionId}`;
    const sessionExists = await redisClient.exists(sessionKey);

    if (!sessionExists) {
      return res.status(401).json({ message: "Unauthorized: Session expired or invalidated" });
    }

    // Attach user data to request object
    req.user = decoded;
    next();
  } catch (err) {
    return res.status(401).json({ message: "Unauthorized: Invalid token" });
  }
}

// --- ROUTES ---

// 1. Login Route (Creates Session & Issues JWT)
app.post("/api/login", async (req, res) => {
  const { username, password } = req.body;

  // Hardcoded user for demonstration purposes
  if (username === "user123" && password === "password") {
    const userId = "user_99";
    const sessionId = Math.random().toString(36).substring(2, 15); // Unique session ID

    // A. Store Session in Redis (Stateful)
    const sessionKey = `session:${userId}:${sessionId}`;
    await redisClient.set(sessionKey, JSON.stringify({ device: req.headers["user-agent"] }), {
      EX: SESSION_EXPIRY,
    });

    // B. Generate JWT containing both User ID and Session ID (Stateless token)
    const token = jwt.sign({ userId, sessionId }, JWT_SECRET, { expiresIn: "1h" });

    // C. Send JWT inside a highly secure cookie
    res.cookie("auth_token", token, {
      httpOnly: true, // Prevents XSS attacks
      secure: true, // Enforces HTTPS (set to false only in local dev without HTTPS)
      sameSite: "strict", // Prevents CSRF attacks
      maxAge: SESSION_EXPIRY * 1000,
    });

    return res.json({ message: "Login successful" });
  }

  res.status(400).json({ message: "Invalid credentials" });
});

// 2. Protected Route (Requires Authentication)
app.get("/api/dashboard", authenticateHybrid, (req, res) => {
  res.json({
    message: "Welcome to your secure dashboard!",
    user: req.user,
  });
});

// 3. Logout Route (Instantly destroys session)
app.post("/api/logout", authenticateHybrid, async (req, res) => {
  const { userId, sessionId } = req.user;

  // Kill the session instantly in Redis
  const sessionKey = `session:${userId}:${sessionId}`;
  await redisClient.del(sessionKey);

  // Clear the client cookie
  res.clearCookie("auth_token");
  res.json({ message: "Logged out successfully" });
});

app.listen(process.env.PORT, () => {
  console.log(`Server running on http://localhost:${process.env.PORT}`);
});
```

---

## Why this pattern works exceptionally well:

- **Instant Logout / Revocation:** Pure JWTs cannot be invalidated before they expire without a blacklist. In this hybrid
  setup, if a user clicks "Logout" or an admin bans a user, you simply delete the key from Redis. The next API request will
  fail immediately, even if the JWT signature is valid.
- **Granular Session Control:** Because the Redis key contains both `userId` and a unique `sessionId`, you can easily build a
  feature that shows **"Log out of all other devices"** by querying all Redis keys starting with `session:user_99:*` and
  deleting them.
- **XSS & CSRF Protection:** Storing the JWT inside an `httpOnly`, `secure`, `sameSite: 'strict'` cookie protects the token
  from being stolen via malicious JavaScript injections.
