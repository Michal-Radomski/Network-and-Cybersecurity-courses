To create a JWE (JSON Web Encryption) in Node.js, the most robust and widely used standard library is **`jose`**. It supports
modern cryptographic algorithms, works seamlessly in Node.js, and doesn't rely on native C++ bindings that can break during
installations.

Here is a step-by-step guide to generating a JWE using a shared secret key (Symmetric Encryption).

---

## 1. Install the `jose` package

First, install the library in your project:

```bash
npm install jose

```

---

## 2. Generating the JWE

In this example, we will encrypt a JSON payload using the **`A256GCM`** (AES-GCM with a 256-bit key) encryption algorithm and
**`dir`** (Direct encryption using a shared symmetric key).

```javascript
import * as jose from "jose";

async function createJWE() {
  // 1. Define the payload you want to keep secret
  const payload = {
    userId: "user_987654321",
    role: "premium_user",
    ssn: "000-12-3456", // Sensitive PII data
  };

  // 2. Generate or provide a 256-bit secret key (32 bytes)
  // In production, load this safely from your environment variables: process.env.JWE_SECRET
  // For AES-256, the key MUST be exactly 32 bytes long.
  const secretKey = jose.base64url.decode("zH2N3y5bV8e1mK4p7q9r2s5t8wLxPzQe1r2s3t4u5v6");

  // 3. Encrypt the payload into a JWE string
  const jwe = await new jose.CompactEncrypt(new TextEncoder().encode(JSON.stringify(payload)))
    .setProtectedHeader({
      alg: "dir", // Direct encryption (using the shared key directly)
      enc: "A256GCM", // Content encryption algorithm
    })
    .encrypt(secretKey);

  console.log("Generated JWE Token:\n", jwe);
  return jwe;
}

createJWE();
```

---

## 3. Decrypting the JWE

To read the data back on your receiving server, you use the exact same shared key to decrypt it:

```javascript
async function decryptJWE(jweString) {
  const secretKey = jose.base64url.decode("zH2N3y5bV8e1mK4p7q9r2s5t8wLxPzQe1r2s3t4u5v6");

  try {
    // Decrypt the token
    const { plaintext, protectedHeader } = await jose.compactDecrypt(jweString, secretKey);

    // Decode the plaintext back into a JSON object
    const decryptedPayload = JSON.parse(new TextDecoder().decode(plaintext));

    console.log("Decrypted Payload:", decryptedPayload);
  } catch (error) {
    console.error("Failed to decrypt or verify JWE:", error.message);
  }
}
```

---

## Best Practices for JWE in Node.js

- **Key Management:** Never hardcode your keys. Use a 256-bit string encoded in `base64url` and store it in a secure
  environment variable manager or vault.
- **Algorithm Choice:** `A256GCM` or `A256CBC-HS512` are standard, highly secure choices for the encryption (`enc`) layer.
- **Asymmetric Alternative:** If you are sending data to a third party where you shouldn't share a secret key, switch the
  algorithm (`alg`) to **`RSA-OAEP-256`**. This allows you to encrypt the token using _their public key_, ensuring only
  _their private key_ can decrypt it.
