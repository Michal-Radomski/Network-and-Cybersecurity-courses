Generating an SBOM from a Node.js `package.json` file is a straightforward, automated process. Because SBOMs need to be
machine-readable, you can use open-source CLI tools to scan your dependencies and output the file in seconds.

The industry standard tool for Node.js is the **CycloneDX Node.js module**, which generates a security-focused SBOM in JSON
or XML format.

Here is how to do it step-by-step:

---

### Step 1: Install the Generator Tool

You can install the CycloneDX tool globally on your machine, or run it directly using `npx` without installing it
permanently.

To install it globally via npm:

```bash
npm install -g @cyclonedx/cyclonedx-npm

```

### Step 2: Install your Project Dependencies

Before running the generator, ensure you have run `npm install` inside your project directory. The tool relies on both the
`package.json` and the `package-lock.json` (or `node_modules` folder) to accurately map out **transitive dependencies** (the
libraries that your libraries depend on).

```bash
npm install

```

### Step 3: Generate the SBOM

Run the tool from the root directory of your project where your `package.json` sits.

**To generate a JSON format SBOM (Recommended):**

```bash
cyclonedx-npm --output-format JSON --output-file sbom.json

```

**To generate an XML format SBOM:**

```bash
cyclonedx-npm --output-format XML --output-file sbom.xml

```

_(Note: If you didn't install the tool globally, you can just prefix the command with npx, like this:
`npx @cyclonedx/cyclonedx-npm --output-format JSON`)_

---

## What to Do with the Resulting `sbom.json`

Once the file is generated, it will contain a structured list of every single package in your ecosystem, their version
numbers, licenses, and unique Package URLs (purls). You can use this file in a few ways:

- **Vulnerability Scanning:** You can upload this `sbom.json` to vulnerability management tools (like Dependency-Track or
  Snyk) to instantly check if any of your open-source packages have known security flaws.
- **CI/CD Integration:** You can add the generation command to your GitHub Actions or GitLab CI pipeline so that a fresh,
  accurate SBOM is generated automatically every time you build or deploy your application.
- **Compliance Sharing:** If you are building a product for a client or organization that requires supply chain transparency,
  you can hand over this file alongside your software release.
