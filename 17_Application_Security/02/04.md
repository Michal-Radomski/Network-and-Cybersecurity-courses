While the **Web Application** and **API** lists are the most famous, OWASP maintains several other "Top 10" projects
targeting specific technologies, architectures, and emerging threats.

The most prominent sibling projects in the OWASP ecosystem include:

---

## 1. OWASP Top 10 for LLM Applications (AI Security)

As generative AI exploded, OWASP quickly established a dedicated list to address vulnerabilities specific to Large Language
Models (LLMs) and AI integrations.

- **LLM01: Prompt Injection:** Attackers craft inputs that trick the LLM into ignoring its system instructions and executing
  unintended actions.
- **LLM02: Insecure Output Handling:** Accepting LLM-generated output blindly without sanitizing it, potentially leading to
  XSS or remote code execution.
- **LLM06: Sensitive Data Disclosure:** The LLM revealing proprietary, private, or sensitive data that was used in its
  training dataset.
- **LLM09: Overreliance:** Users or systems trusting LLM outputs blindly without validation, leading to security oversights
  or spreading misinformation.

## 2. OWASP Top 10 CI/CD Security Risks

With the rise of DevOps, the pipelines used to build and deploy software have become high-value targets (e.g., the SolarWinds
attack). This list secures the software delivery pipeline.

- **CICD-SEC-01: Insufficient Flow Control Mechanisms:** A single malicious developer or compromised account being able to
  push code straight to production without peer reviews.
- **CICD-SEC-03: Dependency Chain Abuse:** Attackers tricking build servers into downloading malicious packages from public
  registries instead of secure internal ones.
- **CICD-SEC-04: Poisoned Pipeline Execution (PPE):** Attackers modifying build configuration files (like
  `.github/workflows`) within a repository to execute malicious code on the build runner.
- **CICD-SEC-06: Insufficient Credential Hygiene:** Leaving hardcoded API keys, SSH keys, or cloud credentials exposed inside
  pipeline logs or configuration files.

## 3. OWASP Mobile Top 10

Mobile apps (iOS and Android) behave differently than web apps because much of the code is downloaded and runs directly on
the user's physical device.

- **M1: Improper Credential Usage:** Hardcoding API keys or hardcoding passwords directly inside the binary.
- **M2: Inadequate Supply Chain Security:** Importing compromised third-party SDKs or libraries into the mobile app project.
- **M3: Insecure Data Storage:** Storing sensitive user data (like session tokens or personal info) in local unencrypted
  files or databases on the device.
- **M9: Insecure Data Transmission:** Failing to use SSL/TLS properly, allowing attackers on the same Wi-Fi network to
  intercept traffic (Man-in-the-Middle).

---

## 4. OWASP Kubernetes Top 10

Container orchestration is highly complex, and minor misconfigurations in Kubernetes (K8s) clusters can compromise entire
cloud infrastructures.

- **K1: Insecure Workload Configurations:** Running containers with unnecessary "root" privileges or host-level access.
- **K2: Supply Chain Vulnerabilities:** Deploying container images sourced from untrusted registries or containing unpatched
  vulnerabilities.
- **K5: Inadequate Logging and Monitoring:** Failing to audit API server logs, making it impossible to detect cluster
  intrusion.

## 5. OWASP Top 10 Proactive Controls

Unlike the other lists that catalog _vulnerabilities_ (what can go wrong), this is a sister project that outlines the **top
10 defensive techniques** developers should write into every application.

- **C1:** Define Security Requirements
- **C2:** Leverage Security Frameworks and Libraries
- **C3:** Secure Database Access
- **C4:** Encode and Validate Data
- **C5:** Implement Digital Identity (Authentication)
- **C6:** Enforce Access Controls (Authorization)
- **C7:** Protect Data (Encryption)
- **C8:** Implement Security Logging and Monitoring
- **C9:** Handle All Errors and Exceptions
- **C10:** Validate Architecture and Design (Threat Modeling)
