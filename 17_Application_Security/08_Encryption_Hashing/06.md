In Node.js, `bcrypt` handles the **salt** automatically as part of its algorithm. To add a **pepper**, you pre-hash the
password with a secret key (stored in an environment variable, not the database) using Node’s built-in `crypto` module before
passing it to `bcrypt`.

Here is the step-by-step implementation.

---

### Step 1: Set Your Pepper Secret

Store your pepper in a `.env` file or environment variable so it stays out of your codebase and database:

```env
# .env
PEPPER_SECRET="your-ultra-secret- pepper-key-keep-this-safe!"

```

---

### Step 2: Hashing the Password (Registration)

We use `crypto.createHmac('sha256', PEPPER)` to combine the password with the secret pepper first, then pass the result to
`bcrypt.hash()`. `bcrypt` generates and manages the unique salt automatically.

```javascript
const crypto = require("crypto");
const bcrypt = require("bcrypt");

// Load your secret pepper from environment variables
const PEPPER = process.env.PEPPER_SECRET;
const SALT_ROUNDS = 12; // Standard work factor for bcrypt

/**
 * Pre-hashes password with pepper, then hashes with bcrypt (salt included)
 */
async function hashPassword(plainPassword) {
  // 1. Pepper the password using HMAC-SHA256
  const pepperedPassword = crypto.createHmac("sha256", PEPPER).update(plainPassword).digest("hex");

  // 2. Hash with bcrypt (generates a unique salt automatically)
  const hashedPassword = await bcrypt.hash(pepperedPassword, SALT_ROUNDS);

  return hashedPassword;
}
```

---

### Step 3: Verifying the Password (Login)

During login, repeat the exact same pepper step before passing the result to `bcrypt.compare()`.

```javascript
/**
 * Verifies an incoming password against the stored bcrypt hash
 */
async function verifyPassword(plainPassword, storedHash) {
  // 1. Apply the exact same pepper transformation
  const pepperedPassword = crypto
    .createHmac('sha256', PEPPER)
    .update(plainPassword)
    {
      /* digest hex */
    }
    .digest('hex');

  // 2. Compare using bcrypt
  const isMatch = await bcrypt.compare(pepperedPassword, storedHash);

  return isMatch;
}

```

### Complete Clean Example

```javascript
require("dotenv").config();
const crypto = require("crypto");
const bcrypt = require("bcrypt");

const PEPPER = process.env.PEPPER_SECRET || "fallback-secret-for-dev";
const SALT_ROUNDS = 12;

function applyPepper(password) {
  return crypto.createHmac("sha256", PEPPER).update(password).digest("hex");
}

async function registerUser(password) {
  const peppered = applyPepper(password);
  const hash = await bcrypt.hash(peppered, SALT_ROUNDS);

  // Store `hash` in your database
  return hash;
}

async function loginUser(inputPassword, storedHash) {
  const peppered = applyPepper(inputPassword);
  return await bcrypt.compare(peppered, storedHash);
}

// Quick Test
(async () => {
  const userPassword = "MySuperSecretPassword123!";

  // 1. Register: Hash password
  const storedDbHash = await registerUser(userPassword);
  console.log("Stored DB Hash:", storedDbHash);

  // 2. Login Check (Correct Password)
  const isValid = await loginUser(userPassword, storedDbHash);
  console.log("Correct password matches?", isValid); // true

  // 3. Login Check (Wrong Password)
  const isInvalid = await loginUser("WrongPassword!", storedDbHash);
  console.log("Wrong password matches?", isInvalid); // false
})();
```

### Why Combine HMAC-SHA256 with `bcrypt`?

1. **Automatic Salt:** `bcrypt.hash()` creates a unique, 128-bit salt for every single password and bakes it into the
   resulting hash output string (`$2b$12$...`). You don't need a separate database column for the salt.
2. **Bypasses bcrypt's 72-Byte Limit:** `bcrypt` silently ignores characters after 72 bytes. HMAC-SHA256 converts any length
   password into a fixed 64-character hex string, ensuring long passphrases aren't truncated.
3. **Defense-in-Depth:** If your database is leaked via SQL injection, an attacker gets the hashes and the salts, but they
   **cannot** attempt offline cracking without the `PEPPER_SECRET` stored on your application server or key manager.
