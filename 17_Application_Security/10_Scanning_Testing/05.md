A **Web Application Firewall (WAF)** is a security solution that monitors, filters, and blocks HTTP/HTTPS traffic travelling
to and from a web application or service.

Positioned at the network boundary (Layer 7 / Application Layer of the OSI model), a WAF acts as an external shield
protecting applications from malicious external attacks, bad bots, and exploit attempts before they ever reach the web
server.

---

## 1. How a WAF Works

A WAF operates as a reverse proxy, inspecting all incoming client requests and outgoing server responses against a configured
policy rule set.

```
┌──────────┐     HTTP Request      ┌──────────────────┐     Clean Request     ┌──────────────────┐
│  Client  │ ────────────────────> │  Web Application │ ────────────────────> │   Web Application│
│ (User/   │                       │  Firewall (WAF)  │                       │   Server         │
│ Attacker)│ <──────────────────── │                  │ <──────────────────── │                  │
└──────────┘     Blocked / 403     └──────────────────┘     HTTP Response     └──────────────────┘
                 (If Malicious)

```

### Detection Models

- **Negative Security Model (Blacklisting):** Uses known threat signatures to block specific malicious inputs (e.g., matching
  known SQL injection payloads or attack strings).
- **Positive Security Model (Whitelisting):** Defines strictly what traffic is allowed (e.g., valid paths, expected query
  parameters) and blocks everything else.
- **Behavioral / Anomaly Detection:** Utilizes machine learning to analyze baseline traffic patterns and block sudden spikes,
  unusual user-agent strings, or irregular request structures.

---

## 2. Key Threat Coverage

A WAF protects applications primarily against the **OWASP Top 10** risks and volumetric attacks:

- **Injection Exploits:** SQL Injection (SQLi), Cross-Site Scripting (XSS), Command Injection.
- **Bad Bots & Scraping:** Automated credentials stuffing, brute-force login attempts, content scraping.
- **Layer 7 DDoS Attacks:** Volumetric HTTP floods aimed at overwhelming application endpoints.
- **File Inclusion & Directory Traversal:** Attempts to read restricted system files via manipulated paths.

---

## 3. WAF Deployment Modes

| Deployment Type                                            | How It Operates                                                                      | Key Advantage                                                                   |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------- |
| **Cloud-Based / SaaS** (e.g., Cloudflare, AWS WAF, Akamai) | Managed DNS or CDN edge service that filters traffic before reaching your network.   | Easy to set up, minimal maintenance, handles large-scale DDoS attacks.          |
| **Network-Based / Hardware**                               | Dedicated physical appliance installed on-premises in the data center.               | High throughput, low latency, full internal control over data payload.          |
| **Host-Based / Software**                                  | Integrated directly as a plugin/module in the web server (e.g., NGINX, Apache, IIS). | Highly customizable for specific applications; cost-effective for small setups. |

---

## 4. Next-Gen Evolution: WAAP

Traditional rule-based WAFs can struggle with API-heavy architectures and false positives. As a result, the industry has
evolved modern WAFs into **WAAP (Web Application and API Protection)** platforms, which integrate four capabilities into a
single solution:

1. **Next-Gen WAF:** Context-aware filtering for web apps.
2. **API Security:** Automatically discovers API endpoints, parses JSON/XML payloads, and enforces schema validation.
3. **Bot Management:** Differentiates between beneficial crawlers (e.g., Googlebot) and malicious automated bots.
4. **DDoS Protection:** Mitigates high-volume application-layer floods.

---

## 5. How WAF Fits with Other Security Tools

- **WAF vs. Network Firewall:** A standard firewall operates at Layers 3 and 4 (IP addresses, ports, protocols). A WAF
  inspects Layer 7 data payloads (URL parameters, cookies, JSON bodies).
- **WAF vs. RASP:** A WAF sits **outside** the application at the network perimeter filtering incoming traffic. **RASP**
  (Runtime Application Self-Protection) sits **inside** the application runtime analyzing how inputs are actually executed in
  memory.
