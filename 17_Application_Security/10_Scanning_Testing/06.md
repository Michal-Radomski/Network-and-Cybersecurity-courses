Adding **RASP (Runtime Application Self-Protection)** to a Node.js application typically involves installing a security agent
library and requiring/initializing it at the **very top** of your application entry point—before any other modules or
framework code (like Express or Fastify) are loaded.

This allows the RASP agent to hook into critical Node.js runtime APIs (like HTTP request handlers, database drivers, file
system methods, and child process execution) to detect and block malicious payloads in real time.

---

## 1. How RASP Integrates into Node.js

Node.js RASP solutions use runtime instrumentation (monkey-patching or using the native `diagnostics_channel` / `async_hooks`
APIs) to monitor application calls.

```
┌──────────────────────────────────────────────────────────┐
│                   Node.js Process                        │
│                                                          │
│  ┌──────────────────┐        ┌────────────────────────┐  │
│  │   RASP Agent     │ ─────> │ Application Code       │  │
│  │ (Loads First)    │ Hooks  │ (Express, Fastify, etc)│  │
│  └────────┬─────────┘        └────────────────────────┘  │
│           │                                              │
│           ▼ Inspects & Blocks                            │
│  ┌────────────────────────────────────────────────────┐  │
│  │ Core Node APIs & Drivers (fs, http, child_process) │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘

```

---

## 2. General Implementation Steps

While commercial vendors (such as **Datadog Application Security Management**, **Contrast Security**, or **Imperva**) supply
their own proprietary SDKs, the integration pattern remains consistent across most providers.

### Step 1: Install the RASP Package

Install the vendor's RASP/ASM module via `npm`:

```bash
npm install @vendor/rasp-agent

```

---

### Step 2: Initialize the Agent at App Entry

The agent **must** be loaded before any other dependencies to ensure it can intercept runtime calls.

#### Option A: In Code (`server.js` / `app.js`)

Place the initialization call at the absolute top of your entry file:

```javascript
// MUST BE THE VERY FIRST IMPORT
const rasp = require("@vendor/rasp-agent").init({
  apiKey: process.env.RASP_API_KEY,
  environment: process.env.NODE_ENV,
  mode: "block", // Options: 'monitor' (log only) or 'block' (active defense)
  rules: {
    sqli: true, // SQL Injection protection
    rce: true, // Remote Code Execution protection
    pathTraversal: true,
    ssrf: true,
  },
});

// Import your web framework AFTER RASP is initialized
const express = require("express");
const app = express();

app.get("/user", (req, res) => {
  // RASP monitors the incoming request and DB query execution
  res.send("Hello World");
});

app.listen(3000);
```

#### Option B: Preload via Node CLI Flag (Recommended)

To prevent developers from accidentally moving or removing the RASP import during code refactoring, preload the agent using
the Node.js `--require` (`-r`) flag:

```bash
# Start your app with preloaded RASP instrumentation
node -r @vendor/rasp-agent app.js

```

In a **Dockerfile**, this would look like:

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .

# Pass the require flag at startup
ENV NODE_OPTIONS="-r @vendor/rasp-agent"
CMD ["node", "app.js"]

```

## 3. Light-weight Alternative: Modern Security SDKs (e.g., Arcjet)

If you don't want a full heavy enterprise RASP suite, modern security SDKs like **Arcjet** bring RASP-like capabilities (bot
detection, SQLi protection, rate limiting, and sensitive data protection) directly into Node.js middleware:

```javascript
import express from "express";
import arcjet, { shield, detectBot } from "@arcjet/node";

const app = express();

const aj = arcjet({
  key: process.env.ARCJET_KEY,
  rules: [
    shield({ mode: "LIVE" }), // Blocks SQLi, XSS, and common attack vectors
    detectBot({ mode: "LIVE", allow: [] }), // Blocks malicious bots
  ],
});

// Apply RASP protection middleware
app.use(async (req, res, next) => {
  const decision = await aj.protect(req);
  if (decision.isDenied()) {
    return res.status(403).json({ error: "Forbidden - Threat Detected" });
  }
  next();
});

app.listen(3000);
```

## 4. Best Practices for Deploying RASP in Node.js

- **Start in Monitoring Mode First:** Deploy RASP in `log-only` or `audit` mode in staging/production for a few days. Review
  logged alerts to ensure no legitimate user actions or complex database queries trigger false positives before enabling
  `block` mode.
- **Monitor CPU and Memory Overhead:** RASP agents inspect requests and function calls in real time, which adds a slight
  latency overhead (typically 1–5ms). Test performance under load.
- **Combine with a WAF:** Use a WAF at the edge (e.g., Cloudflare, AWS WAF) for volumetric DDoS and broad filtering, and rely
  on RASP inside Node.js for deep payload inspection and context-aware defense.
