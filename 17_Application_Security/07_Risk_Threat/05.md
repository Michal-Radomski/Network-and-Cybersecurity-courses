**Threagile** is an open-source, code-based toolkit designed to bring **threat modeling** directly into agile software
development and DevSecOps workflows.

Instead of drawing complex diagrams or filling out long forms in traditional threat-modeling tools, Threagile allows
engineering and security teams to treat threat models as **declarative code** (specifically YAML).

---

## Key Features & How It Works

### 1. Threat-Model-as-Code

You define your system's architecture—including technical assets, data assets, communication links, and trust boundaries—in a
simple YAML file. Because it’s code, it can live inside your Git repository alongside your application code, making it
version-controlled and diff-able.

### 2. Automated Analysis

When you run Threagile (via CLI, Docker, or server mode), it automatically parses your YAML model. It executes built-in (and
custom) security risk rules against the architecture to identify missing controls, vulnerabilities, and data flow risks.

### 3. Comprehensive Output Generation

In seconds, Threagile automatically generates detailed outputs, including:

- **Data Flow Diagrams (DFDs):** Clear visual maps of components and boundaries.
- **Risk Reports:** Detailed documentation (in PDF, Excel, or JSON formats) highlighting identified threats, risk levels, and
  actionable mitigation advice.
- **Risk Metrics:** Metrics like Relative Attacker Attractiveness (RAA) and Data Loss Probability (DLP).

### 4. DevSecOps Pipeline Ready

Because Threagile generates JSON reports and runs seamlessly in Docker, it can easily be embedded into automated CI/CD
pipelines (such as GitHub Actions or Jenkins) to block or flag new high-risk architecture changes before deployment.

---

> **Summary:** Threagile bridges the gap between developers and security teams. It shifts threat modeling "left" into the
> IDE, automating time-consuming diagramming and risk analysis so security models stay updated alongside fast-moving
> codebases.
